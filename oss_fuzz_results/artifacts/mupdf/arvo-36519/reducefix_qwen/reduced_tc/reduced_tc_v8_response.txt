After analyzing the crash and code, I can identify the root cause of the heap-use-after-free READ 8 issue.

The problem occurs in the `fz_draw_clip_image_mask` function when accessing `pixmap->w` and `pixmap->h` after potentially dropping the pixmap. The crash happens because:

1. `pixmap` is obtained from `fz_get_pixmap_from_image`
2. Later, in the interpolation block, if `scaled` is successfully created, we call `fz_drop_pixmap(ctx, pixmap)` and set `pixmap = scaled`
3. However, immediately after this, we access `pixmap->w` and `pixmap->h` in the `ctx->tuning->image_scale` call parameter list
4. This creates a use-after-free condition since `pixmap` may have been dropped

The fix is to store the width and height values before potentially dropping the pixmap, as these values are needed for the image scaling decision.
