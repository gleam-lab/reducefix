### Bug Analysis

The crash is a **Use-of-uninitialized-value** detected by MSan (MemorySanitizer), occurring in `pdf_load_embedded_font`. The stack trace shows:

```
fz_drop_buffer
pdf_load_embedded_font
pdf_load_font_descriptor
```

This indicates that an uninitialized buffer is being dropped.

Looking at the code:
- `buf` is assigned via `pdf_load_stream(ctx, stmref);`
- Then inside `fz_try`, we call `fz_buffer_storage(ctx, buf, &data)` which uses `buf`
- However, **there is no check whether `pdf_load_stream` succeeded**
- If `pdf_load_stream` returns NULL or a partially initialized buffer, then:
  - `fz_buffer_storage` will read from uninitialized memory → triggers MSan
  - Later, `fz_drop_buffer(ctx, buf)` may drop an invalid pointer
  - Even worse: after `fz_always`, we do `fz_buffer_storage(ctx, buf, NULL)` even if everything failed

But critically: **after the `fz_catch`, execution continues and accesses `buf` again**, which might have already been dropped or was never valid.

### Root Cause

Even though `fz_rethrow` is called in `fz_catch`, it's *after* the `fz_always` block. The real problem is this line at the end:

```c
fontdesc->size += fz_buffer_storage(ctx, buf, NULL);
```

At this point, `buf` has **already been dropped** in the `fz_always` block. Using a dropped buffer here causes use-after-free-like behavior (or use-of-invalid-memory), and since the buffer could be null or corrupt, it leads to use-of-uninitialized-value.

Moreover, if `pdf_load_stream` fails and throws an exception, `buf` might not be properly initialized — but we still try to access it in the `fz_always`.

So the core issue is:  
> **We are using `buf` after it has been dropped, and potentially without verifying it was successfully loaded.**

We must ensure:
1. We don't touch `buf` after dropping it.
2. We only update `fontdesc->size` if the operation succeeded.

### Fix Strategy

Move the `fontdesc->size += ...` line **inside the `fz_try` block**, right after creating the font, so it only runs on success, and before the buffer is dropped.

Also, make sure we don’t access `buf` after `fz_always`.
