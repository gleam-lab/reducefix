After analyzing the crash and code, the issue is a **use-after-free** in `update_errors_warnings`. The function calls `timelib_error_container_dtor(*last_errors)` after potentially assigning it to `DATEG(last_errors)`, which creates a dangling pointer if `DATEG(last_errors)` holds the same pointer.

The bug occurs because:
1. If `(*last_errors)` has warnings/errors, it's stored in `DATEG(last_errors)`
2. Then `*last_errors` is passed to `timelib_error_container_dtor()` and freed
3. But `DATEG(last_errors)` still points to the now-freed memory
4. Later access to `DATEG(last_errors)` causes heap-use-after-free

The fix is to avoid storing the pointer right before freeing it, or make a copy. Since we only need to preserve error info when there are actual errors/warnings, we should duplicate the container instead of keeping a reference to freed memory.
