The crash is a **heap-use-after-free** in `update_errors_warnings`, specifically during a `timelib_error_container_dtor` call. The issue arises because:

1. The function receives a double pointer `timelib_error_container **last_errors`.
2. It conditionally assigns `*last_errors` to `DATEG(last_errors)` (a global).
3. Then it calls `timelib_error_container_dtor(*last_errors)` — but if that pointer was just stored globally, the destructor frees memory that may still be referenced by `DATEG(last_errors)`.

However, the real bug is:  
👉 **After assigning `*last_errors` to `DATEG(last_errors)`, the code *still* calls `timelib_error_container_dtor(*last_errors)` and sets `*last_errors = NULL`.**  
But since `DATEG(last_errors)` now points to the same container, this creates a **dangling pointer** — leading to a **use-after-free** when later accessed (e.g., in `timelib_error_container_dtor` again via `php_date_initialize`).

### Root Cause
- If `(*last_errors)` has warnings/errors, we store it in `DATEG(last_errors)` **without incrementing any refcount or copying**.
- Then we immediately destroy `*last_errors` — even though it's now held by the global!
- This makes `DATEG(last_errors)` point to freed memory → **Use-after-free**.

### Fix
We must **not destroy** the error container if we are storing it in the global. So:
- Only destroy `*last_errors` if we did **not** assign it to `DATEG(last_errors)`.

Also note: Line 2273 checks `if (last_errors == NULL || (*last_errors) == NULL)` — but `last_errors` itself cannot be NULL at this point because it's dereferenced earlier as `DATEG(last_errors)` (which expands to a global access based on the pointer value). However, the check for `(*last_errors)` being NULL is valid.

But more importantly: **The current logic destroys the container regardless of whether it was saved to the global.**

✅ Correct behavior:
- If the container has errors/warnings, transfer ownership to `DATEG(last_errors)` → do **not** destroy it here.
- Otherwise, destroy it.
