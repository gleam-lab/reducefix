The crash is a **heap-use-after-free** in `update_errors_warnings`, triggered via `timelib_error_container_dtor` in the call stack. The issue arises because the function:

1. Checks and destroys `DATEG(last_errors)` if it exists.
2. Then later, after potentially assigning `*last_errors` to `DATEG(last_errors)`, it proceeds to destroy `*last_errors` unconditionally.

But here's the critical bug:  
If `(*last_errors)` has warnings/errors, it gets assigned to `DATEG(last_errors)` at line 2278:
```c
DATEG(last_errors) = *last_errors;
```
Then, on line 2281, it calls:
```c
timelib_error_container_dtor(*last_errors);
```
This **frees memory that is now also pointed to by `DATEG(last_errors)`**, creating a **dangling pointer** — hence the **use-after-free** when later accessed (e.g., in `timelib_error_container_dtor` during next invocation or cleanup).

Furthermore, there's a logical flaw: we are destroying `*last_errors` **even if we just saved it into `DATEG(last_errors)`**. That’s incorrect — if we are saving the error container for later use (via `DATEG(last_errors)`), we should **not** destroy it here.

### ✅ Fix:
Only destroy `*last_errors` **if it was not saved** (i.e., if it doesn't have errors/warnings). If it **does** have errors/warnings, we've promoted it to `DATEG(last_errors)` and must **not** destroy it.

Thus, we need to move the destruction **before** the assignment, and only if we are **not** going to save it.

But note: the current logic already clears any previous `DATEG(last_errors)` at the top. So we must:
- Destroy old global errors.
- Save new ones if non-empty.
- Only destroy input `*last_errors` if we didn’t save them.

However, since `*last_errors` might be same as `DATEG(last_errors)` (same object), we cannot destroy it after assigning.

Therefore, correct behavior:
- If `*last_errors` contains errors/warnings → save it to `DATEG(last_errors)` and **do not destroy** it.
- Otherwise → destroy it.

So the final fix is to **conditionally destroy** `*last_errors` only when it is **not** being retained.
