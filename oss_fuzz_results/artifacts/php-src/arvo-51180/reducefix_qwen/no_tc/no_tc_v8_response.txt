The crash is a **heap-use-after-free** occurring in `update_errors_warnings`, specifically during the call to `timelib_error_container_dtor(*last_errors)` on line 2281. The issue arises because:

1. If `(*last_errors)` has warnings or errors, it's assigned to `DATEG(last_errors)` on line 2278.
2. Then, on line 2281, we **destroy** `*last_errors` via `timelib_error_container_dtor`.
3. However, `DATEG(last_errors)` now points to that same freed memory — creating a **dangling pointer**.
4. Later, when `DATEG(last_errors)` is accessed (e.g., in `timelib_error_container_dtor` from `php_date_initialize`), it results in a use-after-free.

But more critically: **we are double-freeing** if `DATEG(last_errors)` was set to `*last_errors` and then later destroyed again — because both `DATEG(last_errors)` and `*last_errors` point to the same object.

### Root Cause:
We are transferring ownership of `*last_errors` to `DATEG(last_errors)` **without cloning**, but then still calling `timelib_error_container_dtor` on the original pointer — which invalidates the data that `DATEG(last_errors)` now points to.

### Fix:
We should **only destroy** `*last_errors` **if we are not saving it** in `DATEG(last_errors)`. So, move the destruction inside an `else` block.
