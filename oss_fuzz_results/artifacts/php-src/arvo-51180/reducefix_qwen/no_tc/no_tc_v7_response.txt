The crash is a **heap-use-after-free** in `update_errors_warnings`, triggered during `DateTime` construction. The issue occurs because we're calling `timelib_error_container_dtor(*last_errors)` after potentially storing the same pointer in `DATEG(last_errors)`. This creates a use-after-free: if the error container is non-null and has warnings/errors, it's stored in `DATEG(last_errors)`, but then immediately freed — leaving `DATEG(last_errors)` pointing to freed memory.

### Root Cause:
- Line 2278: `DATEG(last_errors) = *last_errors;` stores the pointer.
- Line 2281: `timelib_error_container_dtor(*last_errors);` **frees** that same memory.
- Later access via `DATEG(last_errors)` leads to **use-after-free**.

### Fix:
Only destroy `*last_errors` if it was **not** saved into `DATEG(last_errors)`. So we should skip destruction when we transfer ownership.
