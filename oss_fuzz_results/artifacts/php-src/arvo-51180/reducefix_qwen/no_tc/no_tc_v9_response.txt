The crash is a **heap-use-after-free** occurring in `update_errors_warnings`, specifically due to an incorrect double call to `timelib_error_container_dtor`. The issue arises because:

1. The function receives `*last_errors` which may point to memory that's already being managed or freed.
2. After conditionally assigning `*last_errors` to `DATEG(last_errors)`, the code proceeds to call `timelib_error_container_dtor(*last_errors)` — but if `DATEG(last_errors)` now points to the same memory, and later tries to free it again (e.g., during `zim_DateTime___construct`), this results in **use-after-free**.

### Root Cause:
- On line 2281: `timelib_error_container_dtor(*last_errors);` unconditionally destroys the container.
- However, on line 2278: `DATEG(last_errors) = *last_errors;` transfers ownership without transferring responsibility for cleanup.
- But then we destroy `*last_errors` anyway — even though `DATEG(last_errors)` now points to that same freed memory.
- Later, when `timelib_error_container_dtor(DATEG(last_errors))` is called (as seen in stack trace), it attempts to free already-freed memory → **use-after-free**.

### Fix:
Only destroy `*last_errors` **if** it wasn't saved into `DATEG(last_errors)` (i.e., only if its errors/warnings were empty). Otherwise, let `DATEG(last_errors)` own and eventually free it.

We must skip the destruction if we transferred ownership (i.e., if there were warnings/errors).
