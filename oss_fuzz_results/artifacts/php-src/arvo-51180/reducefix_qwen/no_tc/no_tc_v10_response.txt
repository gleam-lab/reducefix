The crash is a **heap-use-after-free** in `update_errors_warnings`, triggered via `timelib_error_container_dtor` in the call stack. The issue arises because we are calling `timelib_error_container_dtor(*last_errors)` **after already potentially assigning it to `DATEG(last_errors)`**, and then later dereferencing it.

### 🔍 Root Cause:
In this code:
```c

    DATEG(last_errors) = *last_errors;  // Stored raw pointer
}
timelib_error_container_dtor(*last_errors);  // Frees the memory!
*last_errors = NULL;
```

We store `*last_errors` into global `DATEG(last_errors)` **without incrementing any reference count or making a copy**. Then we immediately call `timelib_error_container_dtor(*last_errors)`, which **frees the structure**. Later, if someone tries to access `DATEG(last_errors)`, it points to **freed memory** — hence the **use-after-free**.

However, the real bug is:

> We are storing a pointer to memory we are about to free, without cloning it.
