After analyzing the crash and code, the root cause is a **heap-use-after-free** in `update_errors_warnings`. The issue occurs because:

1. The function receives a double pointer `timelib_error_container **last_errors`
2. It conditionally assigns `*last_errors` to `DATEG(last_errors)` (global storage)
3. Then unconditionally calls `timelib_error_container_dtor(*last_errors)` and sets `*last_errors = NULL`
4. **BUT**: If `(*last_errors)` was stored in `DATEG(last_errors)`, then the subsequent `timelib_error_container_dtor(*last_errors)` frees memory that may still be referenced by the global

The crash happens when later code tries to access the already-freed error container.

The fix is to avoid double-free: only destroy the container if it wasn't saved to the global storage.
