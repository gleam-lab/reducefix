After analyzing the crash and code, the root cause is a **heap-use-after-free** in `update_errors_warnings`. The issue occurs because `*last_errors` is being passed to `timelib_error_container_dtor()` after potentially being assigned to `DATEG(last_errors)`. This creates a use-after-free: if the container is stored in `DATEG(last_errors)`, it's still referenced there but then freed by the final `timelib_error_container_dtor(*last_errors)` call.

The bug is in the logic flow:
1. We conditionally assign `*last_errors` to `DATEG(last_errors)`
2. Then unconditionally destroy `*last_errors`
3. But if it was stored in `DATEG`, that global now holds a dangling pointer

However, looking at the stack trace and function purpose, the **real issue** is that we're destroying the container twice: once via `DATEG(last_errors)` and once directly. The fix is to avoid double destruction.

The minimal fix is to **only destroy `*last_errors` if it wasn't saved to `DATEG(last_errors)`**.
