### Bug Analysis

The crash is a **heap-use-after-free** occurring in `update_errors_warnings`, triggered via `timelib_error_container_dtor` during `php_date_initialize`. The stack trace shows it happens when constructing a `DateTime` object.

Looking at the function:

```c
static void update_errors_warnings(timelib_error_container **last_errors)
{
	if (DATEG(last_errors)) {
		timelib_error_container_dtor(DATEG(last_errors));
		DATEG(last_errors) = NULL;
	}

	if (last_errors == NULL || (*last_errors) == NULL) {
		return;
	}

	if ((*last_errors)->warning_count || (*last_errors)->error_count) {
		DATEG(last_errors) = *last_errors;  // ⚠️ Use after free potential
	}

	timelib_error_container_dtor(*last_errors);  // ⚠️ Free
	*last_errors = NULL;
}
```

#### Root Cause:
- The function conditionally assigns `*last_errors` to `DATEG(last_errors)` **before** calling `timelib_error_container_dtor(*last_errors)`
- But `timelib_error_container_dtor(*last_errors)` **frees the memory**
- Then, if there were warnings/errors, we do: `DATEG(last_errors) = *last_errors;` — but this points to **already-freed memory**
- Later, when `DATEG(last_errors)` is accessed (e.g., in destructor), it causes **use-after-free**

Additionally, the check `if (last_errors == NULL)` is flawed because `last_errors` is a double pointer passed as `&some_ptr`. It cannot be `NULL` itself unless caller passes a null address — which would be invalid. So checking `last_errors == NULL` is likely unnecessary and distracts from correct logic.

But the real issue: **storing a pointer to an error container just before freeing it**, leading to dangling pointer.

We must:
1. Only assign `DATEG(last_errors)` if we are keeping the container.
2. **Not store a pointer to memory we are about to free.**
3. Fix ordering: if we want to keep the errors, don't destroy them.

---

### ✅ Fix Strategy

Change semantics:
- If `(*last_errors)` has errors/warnings, **transfer ownership** to `DATEG(last_errors)` → so **don't destroy it**
- Otherwise, destroy it

So:
- Move the `timelib_error_container_dtor(*last_errors)` **inside an else branch**
- Or skip destruction if we are adopting the container

This prevents use-after-free by ensuring that if we save the pointer in `DATEG(last_errors)`, we don’t destroy it.

### Explanation of Fix

- We now **only** call `timelib_error_container_dtor(*last_errors)` when we are **not** saving the container in `DATEG(last_errors)`
- When we **do** want to preserve the errors (`warning_count` or `error_count` > 0), we transfer ownership to `DATEG(last_errors)` and null out the source
- Prevents double-free and use-after-free
- Ensures `DATEG(last_errors)` never holds a pointer to freed memory

This matches safe resource management patterns: **either adopt or destroy, not both.**

Also note: The earlier block already nulls `DATEG(last_errors)` if it existed — so no leak risk.
