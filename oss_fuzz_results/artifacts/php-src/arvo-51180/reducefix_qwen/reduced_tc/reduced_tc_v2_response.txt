The crash is a **heap-use-after-free** in `update_errors_warnings`, specifically a READ 4 after free. The issue occurs because the code:

1. Conditionally assigns `*last_errors` to `DATEG(last_errors)` (a global),
2. Then calls `timelib_error_container_dtor(*last_errors)` — which **frees the memory** pointed to by `*last_errors`,
3. But if `(*last_errors)` was assigned to `DATEG(last_errors)`, then `DATEG(last_errors)` now points to **freed memory**.

Then later, when `timelib_error_container_dtor(DATEG(last_errors))` is called (e.g., in future calls), it attempts to free already-freed memory → **use-after-free**.

### Root Cause:
- On line 2278: `DATEG(last_errors) = *last_errors;` makes the global point to the same container.
- Then on line 2281: `timelib_error_container_dtor(*last_errors);` **frees that container**.
- But `DATEG(last_errors)` still holds a pointer to the freed memory.
- When `update_errors_warnings` is called again, it checks `if (DATEG(last_errors))` and tries to destroy it again → double-free or use-after-free.

This is a classic **ownership transfer bug**: we are storing a pointer to memory we are about to free, without transferring ownership safely.

---

### Fix:
We must **only assign to `DATEG(last_errors)` if we are not going to free it immediately**, or **duplicate it**. But since this is an error container used temporarily during parsing, the correct behavior is:

> If there are warnings/errors, **transfer ownership** of `*last_errors` to `DATEG(last_errors)` — so **don't free it locally**.

Thus, we should **avoid calling `timelib_error_container_dtor(*last_errors)` when we've stored it in `DATEG(last_errors)`**.

### Minimal Fix:
