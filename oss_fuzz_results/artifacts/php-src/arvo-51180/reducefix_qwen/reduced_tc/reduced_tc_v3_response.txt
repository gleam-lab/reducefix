### Bug Analysis

The crash is a **heap-use-after-free** occurring in `update_errors_warnings`, triggered during `DateTime::__construct`. The ASan stack trace shows:

```
timelib_error_container_dtor
php_date_initialize
zim_DateTime___construct
```

This indicates that after freeing an error container via `timelib_error_container_dtor`, it's being accessed again.

Looking at the code:

```c
if ((*last_errors)->warning_count || (*last_errors)->error_count) {
	DATEG(last_errors) = *last_errors;
}

timelib_error_container_dtor(*last_errors);
*last_errors = NULL;
```

**Root Cause**:  
We conditionally assign `*last_errors` to `DATEG(last_errors)` (a global), but then **immediately destroy** `*last_errors`. However, if we assigned it to `DATEG(last_errors)`, then `DATEG(last_errors)` now points to freed memory — a dangling pointer. Later use (e.g., in destructor or subsequent calls) will cause a **use-after-free**.

Additionally, line 2273:
```c
if (last_errors == NULL || (*last_errors) == NULL)
```
is unsafe: if `last_errors` is `NULL`, dereferencing it (`*last_errors`) causes undefined behavior.

---

### Fix Strategy

1. **Avoid dangling pointer**: Only assign `*last_errors` to `DATEG(last_errors)` **after** ensuring we won’t free it.
2. **Fix null check order**: Check `last_errors == NULL` before any dereference.
3. **Do not free if transferred**: If we are transferring ownership to `DATEG(last_errors)`, **don't destroy** it.

Thus, the fix:
- Move assignment to `DATEG(last_errors)` **before** destruction.
- Only assign and preserve if valid and non-empty.
- Do **not** call `timelib_error_container_dtor` on the container if we're keeping it in `DATEG`.

But note: current logic **always** destroys `*last_errors`, even when assigning it to `DATEG`. That’s wrong — we should **transfer ownership**, not copy and destroy.

Since `timelib_error_container` is not reference counted, assigning it without copying means we must **not** destroy the original if we keep it.

Hence: only destroy `*last_errors` if we **do not** assign it to `DATEG`.
