The bug is a **heap-use-after-free** in the `update_errors_warnings` function. The issue occurs because `*last_errors` is being used after it has been freed via `timelib_error_container_dtor(*last_errors)` on line 2281, but then potentially assigned to `DATEG(last_errors)` on line 2278 *after* that free if the error/warning count check passes.

However, note the execution flow:
- If `(*last_errors)` has warnings or errors, we assign it to `DATEG(last_errors)` **then**
- We call `timelib_error_container_dtor(*last_errors)` — which frees the same memory now pointed to by `DATEG(last_errors)`
- This creates a **dangling pointer** in `DATEG(last_errors)`, leading to **use-after-free** when later accessed (e.g., in `timelib_error_container_dtor(DATEG(last_errors))`)

### Root Cause
We are transferring ownership of `*last_errors` to `DATEG(last_errors)` only **if** there are warnings/errors, but then unconditionally destroying `*last_errors`. This results in `DATEG(last_errors)` holding a pointer to freed memory.

### Fix Strategy
Only assign `*last_errors` to `DATEG(last_errors)` **after** we have duplicated or safely taken ownership — but since no duplication happens, and `timelib` expects ownership transfer, we must **not destroy `*last_errors` if we assign it to `DATEG(last_errors)`**.

So: **only destroy `*last_errors` if we did NOT transfer it to `DATEG(last_errors)`**

### Explanation of Fix

- If there are warnings/errors: transfer ownership to `DATEG(last_errors)` and nullify `*last_errors` → **do not dtor**
- Otherwise: safely destroy the container since nothing else owns it
- Prevents double-free and use-after-free by ensuring:
  - Memory is destroyed only once
  - `DATEG(last_errors)` never holds a dangling pointer

This matches expected ownership semantics: `DATEG(last_errors)` takes ownership of meaningful error containers, others are cleaned up immediately.
