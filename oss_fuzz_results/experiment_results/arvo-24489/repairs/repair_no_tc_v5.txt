### 崩溃原因分析
从崩溃信息来看，`hb_vector_t<CFF::dict_val_t>::resize` 函数在尝试进行内存操作时引发了段错误（segmentation fault）。具体来说，问题发生在 `__msan_memset` 函数中，这表明在访问未知地址（即 `0x0000000e680b`）时发生了写操作。通常，这种错误是由于以下原因之一引起的：

1. **越界访问**：可能是由于向量的容量不足，导致在尝试扩展向量时访问了未分配的内存。
2. **空指针**：可能是由于向量中的某个成员为空指针，在尝试对其进行操作时导致了段错误。
3. **堆损坏**：可能是由于之前的内存操作破坏了堆结构，导致后续的内存访问出错。

### 修复策略
为了修复这个崩溃问题，需要确保在访问向量及其成员时不会发生越界访问，并且避免空指针解引用。具体步骤如下：

1. **检查向量容量**：在每次扩容之前，确保有足够的空间来存储新元素。
2. **初始化指针**：确保向量中的所有指针都已正确初始化，避免空指针解引用。
3. **堆验证**：使用工具（如 Valgrind 或 AddressSanitizer）来验证堆的完整性，确保没有堆损坏的情况发生。

template <typename T>
class hb_vector_t {
public:
    void resize(size_t new_size) {
        if (new_size > capacity_) {
            // 确保有足够大的容量
            size_t new_capacity = std::max(capacity_ * 2, new_size);
            reserve(new_capacity);

            // 初始化新增加的部分
            for (size_t i = size_; i < new_size; ++i) {

            }

    }

    void push(const T& value) {
        if (size_ >= capacity_) {
            resize(size_ + 1); // 如果容量不足，先扩容
        }
        data_[size_] = value; // 直接赋值
        ++size_;
    }

private:
    T* data_ = nullptr;
    size_t size_ = 0;
    size_t capacity_ = 0;

    void reserve(size_t new_capacity) {
        if (new_capacity <= capacity_) return;

        T* new_data = static_cast<T*>(std::malloc(new_capacity * sizeof(T)));
        if (!new_data) throw std::bad_alloc();

        for (size_t i = 0; i < size_; ++i) {
            new (&new_data[i]) T(std::move(data_[i])); // 使用移动语义

            data_[i].~T(); // 析构原数据
        }

        std::free(data_);
        data_ = new_data;
        capacity_ = new_capacity;
    }
};
