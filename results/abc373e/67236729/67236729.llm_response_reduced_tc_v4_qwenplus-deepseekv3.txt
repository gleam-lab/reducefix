To fix this code, we need to ensure that the algorithm correctly calculates the minimum number of additional votes each candidate needs to guarantee their victory. The key insight is that a candidate i will be elected if the number of candidates with more votes than them is less than M. 

### Approaches
1. **Sorting and Prefix Sums**: First, we sort the list of votes to efficiently compute the sum of votes for candidates that might surpass candidate i when given additional votes.
2. **Binary Search**: For each candidate, we perform a binary search to determine the minimum number of additional votes needed. This involves checking whether adding a certain number of votes to candidate i ensures that no more than M-1 candidates can have more votes, considering the remaining votes.
3. **Condition Check**: For each candidate, the condition to check is whether the sum of votes that other candidates can receive (considering the remaining votes) doesn't allow more than M-1 candidates to surpass candidate i's votes after adding X votes.

### Solution Code
```cpp
#include <bits/stdc++.h>
using namespace std;
using i64 = long long;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);
    
    int N, M;
    i64 K;
    cin >> N >> M >> K;
    
    vector<i64> A(N);
    for (auto &a : A) cin >> a;
    
    vector<i64> B = A;
    sort(B.begin(), B.end());
    
    vector<i64> prefix(N + 1);
    for (int i = 0; i < N; ++i) {
        prefix[i + 1] = prefix[i] + B[i];
    }
    
    vector<i64> res(N);
    for (int i = 0; i < N; ++i) {
        i64 a = A[i];
        i64 low = 0, high = K, ans = -1;
        
        while (low <= high) {
            i64 mid = (low + high) / 2;
            i64 new_a = a + mid;
            
            int pos = upper_bound(B.begin(), B.end(), new_a) - B.begin();
            int cnt = N - pos;
            
            if (cnt < M) {
                i64 sum = prefix[N] - prefix[pos];
                i64 needed = new_a * cnt - sum;
                if (needed <= K - mid) {
                    ans = mid;
                    high = mid - 1;
                } else {
                    low = mid + 1;
                }
            } else {
                low = mid + 1;
            }
        }
        res[i] = ans;
    }
    
    for (int i = 0; i < N; ++i) {
        cout << res[i] << " \n"[i == N - 1];
    }
    
    return 0;
}
```

### Explanation
1. **Input Handling**: The code reads the number of candidates (N), the maximum allowed candidates to have more votes (M), and the total votes (K). It also reads the current votes for each candidate (A).
2. **Sorting and Prefix Sum**: The votes are sorted (B), and a prefix sum array (prefix) is created to quickly compute the sum of votes from any range.
3. **Binary Search for Each Candidate**: For each candidate, the code performs a binary search to find the minimum additional votes (X) needed. The binary search checks if adding X votes to candidate i ensures that no more than M-1 candidates can have more votes.
4. **Condition Check**: The condition involves calculating the sum of votes that would allow other candidates to surpass the new vote count of candidate i. If the required votes (needed) to surpass are within the remaining votes (K - X), then X is a valid solution.
5. **Output**: The results for each candidate are printed, indicating the minimum additional votes needed or -1 if it's impossible.

This approach ensures that the solution is efficient and correct, handling all edge cases and constraints effectively.
