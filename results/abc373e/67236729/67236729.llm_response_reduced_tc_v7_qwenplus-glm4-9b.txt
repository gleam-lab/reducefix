
### Problem Analysis

The problem involves an election with multiple candidates and partial voting results. Given the number of candidates \( N \), the number of votes \( K \), the number of top candidates \( M \), and the votes each candidate has received so far \( A \), we need to determine the minimum number of additional votes required for each candidate to guarantee their election. A candidate can secure their victory if they are within the top \( M \) candidates after all votes are counted.

### Solution Strategy

1. **Sort Candidates by Votes**: Start by sorting the candidates based on their current votes in descending order. This helps in easily identifying the top candidates.

2. **Determine Votes Needed for Top Candidates**: Calculate the minimum votes required for a candidate to be in the top \( M \) positions. This is done by considering the votes of the \( M^{th} \) candidate in the sorted list and adding the number of candidates beyond \( M \) (i.e., \( N - M \)) multiplied by the difference between the votes of the \( (M+1)^{th} \) candidate and the \( M^{th} \) candidate.

3. **Calculate Additional Votes Required**:
   - For each candidate, if they are already in the top \( M \) positions, they need 0 additional votes.
   - If not, calculate the additional votes required to either join the top \( M \) or surpass the \( (M+1)^{th} \) candidate, whichever is lesser.

4. **Edge Cases**:
   - If a candidate already has more votes than the \( M^{th} \) candidate, they need 0 additional votes.
   - If adding even the maximum possible votes (i.e., \( K \)) cannot secure a candidate's position within the top \( M \), then they cannot secure their victory, and the result should be -1 for that candidate.

### Implementation

```cpp
#include <iostream>
#include <vector>
#include <algorithm>

using namespace std;

int main() {
    int N, M, K;
    cin >> N >> M >> K;

    vector<int> A(N);
    for (int i = 0; i < N; ++i) {
        cin >> A[i];
    }

    // Sort the votes in descending order
    sort(A.rbegin(), A.rend());

    // Prepare to calculate the minimum additional votes needed
    vector<long long> min_votes_needed(N, 0);

    // The candidate at position M-1 needs to secure the M-th position
    if (M - 1 < N) {
        long long votes_needed = (long long)M - 1 - (A[M - 1] - A[M]);
        min_votes_needed[M - 1] = votes_needed;
    }

    // Calculate for the rest of the candidates
    for (int i = 0; i < M - 1; ++i) {
        long long votes_needed = K - A[i] + 1 - (N - M);
        min_votes_needed[i] = votes_needed > (A[M - 1] - A[i]) ? votes_needed : A[M - 1] - A[i];
    }

    // Output the results
    for (int i = 0; i < N; ++i) {
        cout << min_votes_needed[i] << " ";
    }
    cout << endl;

    return 0;
}
```

This refined solution follows the outlined strategy, ensuring that each candidate's additional vote requirement is calculated based on their current standing and the requirements of the top \( M \) positions.
