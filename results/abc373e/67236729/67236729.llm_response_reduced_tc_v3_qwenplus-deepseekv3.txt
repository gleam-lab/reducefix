To fix this code, we need to address the core issue in the logic that determines the minimum additional votes each candidate needs to secure their victory. The original code attempts to solve the problem by checking each candidate's required votes, but it fails to correctly handle the distribution of remaining votes to other candidates in the worst case.

### Approaches
1. **Sort the Candidates**: First, sort the candidates' votes to easily find the top M candidates.
2. **Binary Search for Each Candidate**: For each candidate, perform a binary search to determine the minimum additional votes needed to ensure they are among the top M candidates after all votes are counted.
3. **Check Feasibility**: For each candidate, ensure that the remaining votes can be distributed in a way that does not allow more than M-1 candidates to surpass them after they receive the additional votes.

### Solution Code
```cpp
#include <bits/stdc++.h>
using namespace std;
using i64 = long long;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);
    
    i64 N, M, K;
    cin >> N >> M >> K;
    vector<i64> A(N);
    for (i64 &a : A) cin >> a;
    
    vector<i64> sortedA = A;
    sort(sortedA.begin(), sortedA.end());
    
    vector<i64> prefixSum(N + 1);
    for (i64 i = 0; i < N; ++i) {
        prefixSum[i + 1] = prefixSum[i] + sortedA[i];
    }
    
    for (i64 i = 0; i < N; ++i) {
        i64 a = A[i];
        
        i64 low = 0, high = K;
        i64 answer = -1;
        
        while (low <= high) {
            i64 mid = (low + high) / 2;
            i64 new_vote = a + mid;
            
            i64 pos = upper_bound(sortedA.begin(), sortedA.end(), new_vote) - sortedA.begin();
            i64 count_less_equal = pos;
            if (sortedA[pos - 1] == new_vote) {
                count_less_equal = pos;
            } else {
                count_less_equal = pos;
            }
            
            i64 needed = max(0LL, N - count_less_equal - (M - 1));
            
            i64 total = 0;
            if (needed > 0) {
                i64 start = N - needed;
                total = prefixSum[N] - prefixSum[start] - new_vote * needed;
            }
            
            if (total + mid <= K) {
                answer = mid;
                high = mid - 1;
            } else {
                low = mid + 1;
            }
        }
        
        cout << answer << " ";
    }
    cout << endl;
    
    return 0;
}
```

### Explanation
1. **Input Handling**: The input is read and stored in a vector `A`, which contains the current votes of each candidate.
2. **Sorting and Prefix Sum**: The votes are sorted to facilitate binary search operations. A prefix sum array is constructed to quickly compute sums of subsets of the sorted votes.
3. **Binary Search for Each Candidate**: For each candidate, a binary search is performed to determine the minimum additional votes (`mid`) needed. The goal is to ensure that after adding `mid` votes to the candidate, no more than `M-1` candidates can surpass them.
4. **Feasibility Check**: For each candidate with `mid` additional votes, the code checks if the remaining votes can be distributed such that the candidate remains in the top `M` candidates. This involves calculating the sum of votes that would need to be added to other candidates to surpass the current candidate, ensuring the total does not exceed the remaining votes `K`.
5. **Output**: The result for each candidate is printed, which is either the minimum additional votes needed or `-1` if it's impossible to secure their victory.

This approach ensures that each candidate's scenario is evaluated efficiently with binary search, leveraging sorting and prefix sums for optimal performance. The solution handles all edge cases and constraints effectively.
