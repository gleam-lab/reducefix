### Approaches
1. **Sorting and Prefix Sums**: First, sort the candidates based on their current votes. This helps in efficiently determining the number of candidates that can surpass a given candidate's vote count.
2. **Binary Search for Minimum Votes**: For each candidate, use binary search to find the minimum additional votes needed such that no more than `M-1` candidates can have more votes than this candidate after distributing the remaining votes.
3. **Validation**: During the binary search, validate whether a candidate can secure their position by ensuring that the number of candidates that can potentially surpass them is less than `M`.

### Solution Code

```cpp
#include <bits/stdc++.h>
using namespace std;
using ll = long long;

int N, M;
ll K;
vector<ll> A;

int main() {
    cin >> N >> M >> K;
    A.resize(N);
    for (int i = 0; i < N; ++i) {
        cin >> A[i];
    }
    
    vector<ll> B = A;
    sort(B.begin(), B.end());
    vector<ll> prefix(N + 1, 0);
    for (int i = 0; i < N; ++i) {
        prefix[i + 1] = prefix[i] + B[i];
    }
    
    ll total = prefix[N];
    ll remaining = K - total;
    
    vector<ll> ans(N, -1);
    for (int i = 0; i < N; ++i) {
        ll low = 0, high = remaining;
        ll best = -1;
        while (low <= high) {
            ll mid = (low + high) / 2;
            ll new_val = A[i] + mid;
            
            int idx = upper_bound(B.begin(), B.end(), new_val) - B.begin();
            int cnt = N - idx;
            
            if (cnt < M) {
                best = mid;
                high = mid - 1;
            } else {
                low = mid + 1;
            }
        }
        ans[i] = best;
    }
    
    for (int i = 0; i < N; ++i) {
        cout << ans[i] << ' ';
    }
    cout << '\n';
    
    return 0;
}
```

### Explanation
1. **Sorting and Prefix Sums**: The votes of the candidates are sorted to facilitate binary search operations. Prefix sums are computed to quickly calculate the sum of votes up to any candidate.
2. **Binary Search**: For each candidate, perform a binary search to determine the minimum additional votes (`mid`) needed such that when added to the candidate's current votes, no more than `M-1` candidates can have more votes. This is done by checking the position of the candidate's new vote count in the sorted list.
3. **Validation**: The binary search checks if the candidate's new vote count ensures that the number of candidates with higher votes is below `M`. If so, it adjusts the search range to find the minimum such `mid`.

This approach ensures that the solution is both efficient and correct, handling all edge cases and constraints effectively.
