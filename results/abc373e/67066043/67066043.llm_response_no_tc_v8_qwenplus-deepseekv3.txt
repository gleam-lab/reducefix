### Approaches
1. **Sorting and Preprocessing**: Sort the candidates' votes in ascending order to facilitate binary search and prefix sum calculations.
2. **Binary Search for Minimum Votes**: For each candidate, perform a binary search to determine the minimum number of additional votes needed such that the candidate is among the top candidates who can be elected.
3. **Prefix Sums**: Use prefix sums to efficiently calculate the total votes required to ensure that the candidate's position is secure.
4. **Edge Cases**: Handle cases where candidates are already secure or cannot possibly secure a position by checking the bounds and conditions early.

### Solution Code

```cpp
#include <bits/stdc++.h>
using namespace std;
typedef long long ll;
#define all(x) (x).begin(), (x).end()

int main() {
    ios_base::sync_with_stdio(0);
    cin.tie(0);
    int n, m;
    ll k;
    cin >> n >> m >> k;
    vector<ll> a(n);
    for (int i = 0; i < n; i++) {
        cin >> a[i];
    }
    vector<int> ord(n);
    iota(all(ord), 0);
    sort(all(ord), [&](int i, int j) { return a[i] < a[j]; });
    auto b = a;
    sort(all(b));
    vector<ll> pref(n + 1);
    for (int i = 0; i < n; i++) {
        pref[i + 1] = b[i] + pref[i];
    }
    vector<ll> ans(n, -1);
    for (int idx = 0; idx < n; idx++) {
        int i = ord[idx];
        ll low = 0, high = k;
        ll res = -1;
        while (low <= high) {
            ll mid = (low + high) / 2;
            ll new_votes = a[i] + mid;
            int pos = upper_bound(all(b), new_votes) - b.begin();
            if (pos > n - m) {
                res = mid;
                high = mid - 1;
                continue;
            }
            ll total = pref[n] - pref[pos] - (new_votes * (n - pos)) + mid;
            if (total <= k) {
                res = mid;
                high = mid - 1;
            } else {
                low = mid + 1;
            }
        }
        ans[i] = res;
    }
    for (int i = 0; i < n; i++) {
        cout << ans[i] << " ";
    }
    cout << endl;
    return 0;
}
```

### Explanation
1. **Sorting and Preprocessing**: The candidates' votes are sorted to allow efficient binary search operations. The prefix sums array is constructed to quickly compute the sum of votes up to any given candidate.
2. **Binary Search**: For each candidate, we perform a binary search to determine the minimum additional votes needed to ensure their election. The binary search checks if the candidate's new vote count places them in a position where they are among the top candidates who can be elected.
3. **Prefix Sums Calculation**: The prefix sums array helps in efficiently calculating the total votes required to adjust the votes of other candidates so that they do not surpass the current candidate's vote count by too much.
4. **Edge Cases Handling**: The binary search directly checks if the candidate's position after adding the votes is within the top remaining candidates. If not, it adjusts the search range accordingly. The result is printed for each candidate, ensuring correctness and efficiency within the problem constraints.

This approach ensures that we efficiently and correctly determine the minimum additional votes each candidate needs to secure their election, handling all edge cases and constraints properly.
