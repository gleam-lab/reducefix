To fix this code, we need to ensure that for each candidate, we correctly determine the minimum number of additional votes they need to guarantee their election. The original code had some logical issues in the `check` function and the binary search loop, which didn't correctly account for the constraints and conditions for a candidate to be elected.

Here’s the corrected approach:

1. **Sort the Votes**: First, sort the array of initial votes to easily determine the rankings.
2. **Binary Search for Each Candidate**: For each candidate, perform a binary search to find the minimum additional votes `X` such that, after adding `X` votes, the candidate is among the top `M` candidates, even if all other remaining votes are given to the other candidates in the worst possible way.
3. **Check Condition**: The condition to check is whether the candidate's votes after adding `X` is such that at least `M` candidates (including themselves) have votes less than or equal to their new vote count. This involves calculating the sum of votes needed for other candidates to not surpass the candidate in question.

The fixed code is as follows:

```cpp
#include<bits/stdc++.h>
using namespace std;
#define int long long
const int N=2e5+5;
int n, m, k, a[N], b[N], c[N];

bool check(int x, int y) {
    int total = x + y;
    int pos = upper_bound(b + 1, b + n + 1, total) - b - 1;
    if (n - pos >= m) return false;
    int needed = 0;
    int start = n - m + 1;
    if (start <= 0) return true;
    needed = total * (start - 1) - c[start - 1];
    needed += max(0LL, (total - b[start]) * (n - start + 1) - (c[n] - c[start - 1]));
    return needed <= k - y;
}

signed main() {
    ios::sync_with_stdio(0), cin.tie(0), cout.tie(0);
    cin >> n >> m >> k;
    for (int i = 1; i <= n; i++) {
        cin >> a[i];
        b[i] = a[i];
    }
    sort(b + 1, b + n + 1);
    for (int i = 1; i <= n; i++) {
        c[i] = c[i - 1] + b[i];
    }
    for (int i = 1; i <= n; i++) {
        int l = 0, r = k, ans = -1;
        while (l <= r) {
            int mid = (l + r) / 2;
            if (check(a[i], mid)) {
                ans = mid;
                r = mid - 1;
            } else {
                l = mid + 1;
            }
        }
        cout << ans << " ";
    }
    return 0;
}
```

### Explanation:
1. **Sorting and Prefix Sum**: The array `b` is a sorted version of the initial votes `a`, and `c` is the prefix sum array of `b`. This helps in efficiently calculating the sum of votes needed.
2. **Binary Search**: For each candidate, we perform a binary search to find the minimum `X` (additional votes) such that the candidate can be in the top `M` candidates.
3. **Check Function**: The `check` function determines if adding `X` votes to candidate `i` ensures their election. It calculates the position of the candidate in the sorted array after adding `X` votes and checks if the sum of votes required to keep other candidates below or equal to this new count is within the remaining votes `k - y`.

This approach ensures that we correctly and efficiently find the minimum additional votes needed for each candidate to guarantee their election under the worst-case scenario of vote distribution among other candidates.
