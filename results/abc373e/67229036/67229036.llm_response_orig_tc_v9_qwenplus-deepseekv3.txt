### Approaches
1. **Understanding the Problem**: Each candidate will be elected if the number of candidates who have more votes than them is less than M. The goal is to find the minimum additional votes each candidate needs to ensure this condition, regardless of how the remaining votes are distributed among other candidates.
2. **Key Insight**: To guarantee a candidate's victory, their total votes (including additional votes) must be such that at most M-1 candidates can have more votes than them. This involves:
   - Sorting the current votes to efficiently determine the threshold.
   - Calculating the required votes for each candidate to be among the top M candidates after distribution of remaining votes.
3. **Algorithm**:
   - Sort the array of current votes.
   - For each candidate, determine the minimum votes needed so that even if all other candidates get the maximum possible votes from the remaining, the candidate still has at least M-1 candidates with fewer or equal votes.

### Solution Code
```cpp
#include <bits/stdc++.h>
using namespace std;
#define int long long

signed main() {
    ios::sync_with_stdio(0);
    cin.tie(0);
    cout.tie(0);
    
    int N, M, K;
    cin >> N >> M >> K;
    vector<int> A(N);
    for (int i = 0; i < N; ++i) {
        cin >> A[i];
        K -= A[i];
    }
    
    vector<int> sortedA = A;
    sort(sortedA.begin(), sortedA.end(), greater<int>());
    
    int threshold = (M == N) ? 0 : sortedA[M - 1];
    
    for (int i = 0; i < N; ++i) {
        if (M == N) {
            cout << "0 ";
            continue;
        }
        int required = max(0LL, threshold - A[i] + 1);
        if (required <= K) {
            cout << required << " ";
        } else {
            cout << "-1 ";
        }
    }
    
    return 0;
}
```

### Explanation
1. **Input Handling**: The code reads the number of candidates (N), the threshold M, and the total votes (K). It then reads the current votes for each candidate.
2. **Sorting**: The current votes are sorted in descending order to easily identify the top M candidates.
3. **Threshold Calculation**: The threshold is set to the vote count of the M-th candidate in the sorted list (if M is not equal to N). This threshold determines the minimum votes a candidate needs to be in the top M.
4. **Determining Additional Votes**:
   - For each candidate, the required additional votes are calculated as the difference between the threshold and their current votes plus one (to ensure they surpass the threshold).
   - If the required votes can be met with the remaining votes (K), the result is printed; otherwise, -1 is printed indicating it's impossible.
5. **Edge Case Handling**: If M equals N, every candidate is guaranteed to win, so no additional votes are needed (output 0 for all candidates).

This solution efficiently checks the condition for each candidate and ensures correct results within the problem constraints.
