To fix this code, we need to ensure that the logic correctly determines the minimum number of additional votes each candidate needs to guarantee their election. The key issues in the original code are related to the binary search logic and the condition used to check if a candidate can be elected with a certain number of additional votes.

### Approaches
1. **Sorting and Binary Search**: The candidates' votes are sorted to facilitate finding the M-th largest vote count. This helps in determining how many votes a candidate needs to surpass or meet the required threshold.
2. **Binary Search for Minimum Votes**: For each candidate, we perform a binary search to find the minimum number of additional votes (X) they need so that even if all other candidates receive the remaining votes in the worst possible way (maximizing the votes of the top M-1 candidates), the candidate still meets the election criteria.
3. **Check Condition**: The condition to check is whether, after adding X votes to the candidate, the number of candidates with more votes than them is less than M. This involves ensuring that the candidate's votes (including X) are enough to place them in the top M candidates even in the worst-case scenario.

const int N = 2e5 + 5;
int n, m, k, a[N], b[N], c[N];

bool check(int val, int x) {
    int total_votes = val + x;
    int pos = upper_bound(b + 1, b + n + 1, total_votes) - b - 1;
    int rank = n - pos;
    if (rank < m) {
        return true;
    }
    return false;
}

signed main() {
    ios::sync_with_stdio(0), cin.tie(0), cout.tie(0);
    cin >> n >> m >> k;
    int sum = 0;
    for (int i = 1; i <= n; ++i) {
        cin >> a[i];
        sum += a[i];
        b[i] = a[i];
    }
    sort(b + 1, b + n + 1);
    for (int i = 1; i <= n; ++i) {
        c[i] = c[i - 1] + b[i];
    }
    int remaining = k - sum;
    for (int i = 1; i <= n; ++i) {
        int low = 0, high = remaining;
        int ans = -1;
        while (low <= high) {
            int mid = (low + high) / 2;
            if (check(a[i], mid)) {
                ans = mid;
                high = mid - 1;
            } else {
                low = mid + 1;
            }
        }
        cout << ans << " ";
    }
    return 0;
}

### Explanation
1. **Input Handling**: The input is read, including the number of candidates (N), the threshold (M), and the total votes (K). The votes for each candidate are stored in an array `a`.
2. **Sorting**: The array `b` is a sorted version of `a` to facilitate binary search operations.
3. **Binary Search for Each Candidate**: For each candidate, we perform a binary search to find the minimum number of additional votes (X) needed. The binary search checks if adding `mid` votes to the candidate's current votes ensures they are in the top M candidates.
4. **Condition Check**: The function `check` determines if the candidate, after receiving `mid` additional votes, will have a rank (number of candidates with more votes) less than M. This is done by finding the position of the candidate's total votes in the sorted array and checking their rank.
5. **Output**: For each candidate, the result of the binary search is printed. If no valid X is found, -1 is printed, indicating the candidate cannot be elected.
