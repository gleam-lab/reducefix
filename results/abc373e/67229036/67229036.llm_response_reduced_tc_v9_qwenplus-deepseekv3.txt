To fix this code, we need to ensure that the logic correctly determines the minimum additional votes each candidate needs to guarantee their election. The key insight is to determine for each candidate the minimum votes they need such that at most M-1 candidates can have more votes than them, even if all remaining votes are distributed to those candidates.

### Approach
1. **Sort and Prefix Sums**: First, sort the array of votes received by the candidates. This helps in efficiently determining the threshold for each candidate.
2. **Binary Search for Each Candidate**: For each candidate, perform a binary search to find the minimum additional votes (X) such that when candidate i receives X more votes, at most M-1 candidates can have more votes than them. This involves checking if the remaining votes (after giving X votes to candidate i) can be distributed to other candidates in a way that no more than M-1 candidates exceed candidate i's new vote count.

### Solution Code
```cpp
#include<bits/stdc++.h>
using namespace std;
#define int long long

const int N = 2e5 + 5;
int n, m, k, a[N], b[N], c[N];

bool check(int i, int x) {
    int target = a[i] + x;
    int idx = upper_bound(b + 1, b + n + 1, target) - b;
    int cnt = n - idx + 1;
    if (cnt >= m) return false;
    int max_possible = (m - 1) - cnt;
    if (max_possible < 0) return true;
    int sum_needed = 0;
    int start = idx - max_possible;
    if (start <= 0) return false;
    sum_needed = target * max_possible - (c[idx - 1] - c[start - 1]);
    return sum_needed <= (k - x);
}

signed main() {
    ios::sync_with_stdio(0), cin.tie(0), cout.tie(0);
    cin >> n >> m >> k;
    for (int i = 1; i <= n; i++) {
        cin >> a[i];
        b[i] = a[i];
    }
    sort(b + 1, b + n + 1);
    for (int i = 1; i <= n; i++) {
        c[i] = c[i - 1] + b[i];
    }
    for (int i = 1; i <= n; i++) {
        int l = 0, r = k, ans = -1;
        while (l <= r) {
            int mid = (l + r) / 2;
            if (check(i, mid)) {
                ans = mid;
                r = mid - 1;
            } else {
                l = mid + 1;
            }
        }
        cout << ans << " ";
    }
    return 0;
}
```

### Explanation
1. **Sorting and Prefix Sums**: The array `b` is a sorted version of the original votes array `a`. The prefix sum array `c` is used to quickly compute the sum of any subarray of `b`.
2. **Binary Search for Minimum Votes**: For each candidate, the binary search checks if a certain number of additional votes `X` ensures that at most `M-1` candidates can have more votes than the candidate `i` after distributing the remaining votes. The `check` function determines feasibility by:
   - Finding the position `idx` in the sorted array where votes exceed `a[i] + X`.
   - Calculating how many candidates already have more votes than `a[i] + X`.
   - Determining if the remaining votes can be allocated such that no more than `M-1` candidates exceed `a[i] + X`.
3. **Output**: For each candidate, the result of the binary search is printed, which is the minimum `X` ensuring their election or `-1` if it's impossible.

This approach efficiently handles the constraints and ensures correctness for all test cases by leveraging sorting, binary search, and prefix sums.
