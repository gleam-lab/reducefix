To fix this code, we need to ensure that the logic correctly computes the minimum additional votes each candidate needs to guarantee their election. The key issues in the original code revolve around the incorrect handling of the sorted array and the conditions for determining the number of candidates that can surpass the given candidate's vote count after adding X votes.

### Approaches
1. **Sorting and Prefix Sums**: First, we sort the array of votes to easily compute the necessary conditions. We also compute prefix sums to facilitate efficient calculations.
2. **Binary Search for Each Candidate**: For each candidate, we perform a binary search to determine the minimum number of additional votes (X) needed such that no more than M-1 candidates can have votes strictly greater than the candidate's votes after adding X.
3. **Condition Check**: For each candidate and a given X, we check if the sum of votes required to make M-1 candidates surpass the candidate's new vote count (A_i + X) is feasible within the remaining votes (K).

### Solution Code
```cpp
#include<bits/stdc++.h>
using namespace std;
#define int long long

const int N = 2e5 + 5;
int n, m, k, a[N], b[N], c[N];

bool check(int idx, int x) {
    int new_vote = a[idx] + x;
    int pos = upper_bound(b + 1, b + n + 1, new_vote) - b;
    int cnt = n - pos + 1;
    if (cnt >= m) return false;
    
    int needed = 0;
    int start = pos - (m - cnt);
    if (start <= 0) return true;
    for (int i = start; i < pos; ++i) {
        needed += max(0LL, new_vote + 1 - b[i]);
    }
    return needed <= (k - x);
}

signed main() {
    ios::sync_with_stdio(0), cin.tie(0), cout.tie(0);
    cin >> n >> m >> k;
    int total = 0;
    for (int i = 1; i <= n; ++i) {
        cin >> a[i];
        total += a[i];
        b[i] = a[i];
    }
    sort(b + 1, b + n + 1);
    for (int i = 1; i <= n; ++i) {
        c[i] = c[i - 1] + b[i];
    }
    
    for (int i = 1; i <= n; ++i) {
        int l = 0, r = k - (total - a[i]);
        int ans = -1;
        while (l <= r) {
            int mid = (l + r) / 2;
            if (check(i, mid)) {
                ans = mid;
                r = mid - 1;
            } else {
                l = mid + 1;
            }
        }
        cout << ans << " ";
    }
    return 0;
}
```

### Explanation
1. **Sorting and Prefix Sums**: The array `b` is a sorted version of the original votes array `a`. The prefix sums array `c` helps in quickly calculating the sum of votes up to any index.
2. **Binary Search**: For each candidate, we perform a binary search to find the minimum `X` such that adding `X` votes to the candidate ensures that no more than `M-1` candidates can have more votes. This involves checking if the remaining votes after allocating `X` to the candidate can be distributed to other candidates in a way that allows the condition to be met.
3. **Condition Check**: The `check` function determines if, for a given candidate and `X`, the condition is satisfied. It calculates how many candidates would have more votes than the candidate's new total and checks if the remaining votes can be used to ensure that no more than `M-1` candidates surpass this total.

This approach ensures that the solution is both efficient and correct, handling the constraints and edge cases appropriately.
