To fix this code, we need to ensure that the logic correctly determines the minimum additional votes each candidate needs to secure their election. The key observation is that a candidate will be elected if there are fewer than M candidates with more votes than them. The solution involves sorting the votes, using prefix sums for efficient calculations, and binary search to find the minimal required votes for each candidate.

### Approaches
1. **Sorting and Prefix Sums**: First, sort the array of votes. This helps in quickly determining how many candidates have more votes than a given candidate after potential additional votes.
2. **Binary Search**: For each candidate, perform a binary search to find the minimal number of additional votes such that, even if the other candidates receive all remaining votes, the candidate will still be among the top candidates (with fewer than M candidates having more votes).
3. **Check Function**: The check function determines whether a candidate with a certain number of additional votes will secure their position. It involves checking if the candidate's total votes (original plus additional) are sufficient to ensure that no more than M-1 candidates can surpass them, considering the worst-case distribution of the remaining votes.

### Solution Code
```cpp
#include<bits/stdc++.h>
using namespace std;
#define int long long
const int N=2e5+5;
int n, m, k, a[N], b[N], c[N];

bool check(int i, int x) {
    int total = a[i] + x;
    int pos = upper_bound(b + 1, b + n + 1, total) - b;
    pos = n + 1 - pos;
    if (pos < m) return true;
    
    int worst_case = (n - pos) + 1;
    if (worst_case > m) return false;
    
    int sum_extra = 0;
    int start = upper_bound(b + 1, b + n + 1, total) - b;
    int end = n;
    if (start <= end) {
        sum_extra = (c[n] - c[start - 1]) - total * (end - start + 1);
    }
    return (sum_extra + x) <= k;
}

signed main() {
    ios::sync_with_stdio(0), cin.tie(0), cout.tie(0);
    cin >> n >> m >> k;
    for(int i = 1; i <= n; ++i) {
        cin >> a[i];
        b[i] = a[i];
    }
    sort(b + 1, b + n + 1);
    for(int i = 1; i <= n; ++i) {
        c[i] = c[i - 1] + b[i];
    }
    for(int i = 1; i <= n; ++i) {
        int l = 0, r = k, ans = -1;
        while (l <= r) {
            int mid = (l + r) / 2;
            if (check(i, mid)) {
                ans = mid;
                r = mid - 1;
            } else {
                l = mid + 1;
            }
        }
        cout << ans << " ";
    }
    return 0;
}
```

### Explanation
1. **Sorting and Prefix Sums**: The array `b` is a sorted copy of the original votes array `a`. The prefix sum array `c` allows quick summation of ranges of the sorted array.
2. **Binary Search**: For each candidate, we perform a binary search to find the minimal `x` (additional votes) that ensures the candidate's election. The range for `x` is from 0 to the remaining votes `k`.
3. **Check Function**: The function checks if the candidate's total votes (original plus mid) can ensure their position. It calculates how many candidates could potentially surpass them and verifies if the remaining votes can be distributed such that no more than M-1 candidates do so. The worst-case scenario is considered where other candidates receive as many votes as possible without surpassing the candidate's total.
4. **Output**: For each candidate, the result of the binary search is printed, which is either the minimal additional votes needed or -1 if it's impossible to secure their election.

This approach efficiently checks each candidate's condition using sorting, prefix sums, and binary search, ensuring correctness and optimal performance.
