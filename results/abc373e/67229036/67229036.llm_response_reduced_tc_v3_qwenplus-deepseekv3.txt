To fix this code, we need to address the core logic that ensures each candidate can secure their election with the minimum number of additional votes. The previous approach had some logical flaws in the `check` function and the overall binary search strategy. 

### Approaches
1. **Problem Analysis**: The problem requires determining for each candidate the minimum number of additional votes they need to secure their election, ensuring that no more than `M-1` candidates have strictly more votes than them. If it's impossible, we should return -1.

2. **Key Insight**: For candidate `i` to be elected, after adding `X` votes, their total votes must be such that at most `M-1` other candidates have more votes than them. To guarantee this, we need to ensure that even if the remaining votes are distributed to the other candidates in a way that maximizes the votes of the top `M-1` candidates, candidate `i`'s votes are still sufficient.

3. **Algorithm Selection**: 
   - **Sorting and Prefix Sums**: We sort the initial votes to easily identify the top candidates and compute cumulative sums for efficient calculations.
   - **Binary Search**: For each candidate, we perform a binary search to determine the minimum `X` such that, when added to their votes, they meet the election condition.

4. **Complexity Analysis**: Sorting the array takes O(N log N). For each candidate, the binary search and the checks inside it take O(log K) time, leading to an overall complexity of O(N log N + N log K), which is efficient for the given constraints.

### Solution Code

#include<bits/stdc++.h>
using namespace std;
#define int long long

const int N = 2e5 + 5;
int n, m, k, a[N], b[N], c[N];

bool check(int i, int x) {
    int total = a[i] + x;
    int pos = upper_bound(b + 1, b + n + 1, total) - b;
    int cnt = n + 1 - pos;
    if (cnt < m) return true;
    
    int sum = 0;
    int start = n - m + 1;
    if (start < pos) {
        sum = c[n] - c[start - 1] - (a[i] + x) * (n - start + 1);
    } else {
        sum = c[n] - c[pos - 1] - (a[i] + x) * (n - pos + 1);
    }
    return (sum <= k - x);
}

signed main() {
    ios::sync_with_stdio(0), cin.tie(0), cout.tie(0);
    cin >> n >> m >> k;
    for (int i = 1; i <= n; i++) {
        cin >> a[i];
        b[i] = a[i];
    }
    sort(b + 1, b + n + 1);
    for (int i = 1; i <= n; i++) {
        c[i] = c[i - 1] + b[i];
    }
    
    for (int i = 1; i <= n; i++) {
        int low = 0, high = k, ans = -1;
        while (low <= high) {
            int mid = (low + high) / 2;
            if (check(i, mid)) {
                ans = mid;
                high = mid - 1;
            } else {
                low = mid + 1;
            }
        }
        cout << ans << " ";
    }
    return 0;
}

### Explanation
1. **Sorting and Prefix Sums**: The array `b` is the sorted version of `a`, and `c` stores the prefix sums of `b`. This helps in quickly accessing the sum of top candidates.
2. **Binary Search**: For each candidate `i`, we perform a binary search to find the minimum `X` (additional votes) needed. The `check` function verifies if adding `X` votes to candidate `i` ensures their election.
3. **Check Function**: The function calculates how many candidates would still have more votes than candidate `i` after adding `X` votes. If fewer than `M` candidates have more votes, it returns true. Otherwise, it checks if the remaining votes are sufficient to reduce the top candidates' votes below the threshold.
4. **Output**: For each candidate, the binary search result is printed, indicating the minimum additional votes needed or -1 if it's impossible.

This approach ensures that we efficiently determine the solution for each candidate while handling the constraints correctly.
