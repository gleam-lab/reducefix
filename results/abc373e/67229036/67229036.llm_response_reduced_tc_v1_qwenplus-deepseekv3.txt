To fix this code, we need to ensure that the logic correctly determines the minimum number of additional votes each candidate needs to guarantee their victory. The key is to efficiently check for each candidate whether they can be among the top M candidates after receiving X additional votes, considering the worst-case scenario where other candidates receive the maximum possible votes.

### Approaches
1. **Sorting and Prefix Sums**: 
   - Sort the array of votes to facilitate quick comparison and sum calculations.
   - Compute prefix sums to efficiently calculate the sum of votes up to any index.

2. **Binary Search for Each Candidate**:
   - For each candidate, use binary search to determine the minimum X such that, when the candidate receives X additional votes, they will be in the top M candidates regardless of how the remaining votes are distributed among other candidates.
   - The binary search involves checking if the candidate's votes (original + X) are sufficient to ensure they are among the top M, even if other candidates receive all remaining votes in the worst possible way.

3. **Handling Edge Cases**:
   - If a candidate is already in the top M with their current votes, they need 0 additional votes.
   - If it's impossible for a candidate to be in the top M even with all remaining votes, they should output -1.

### Solution Code
```cpp
#include<bits/stdc++.h>
using namespace std;
#define int long long
const int N=2e5+5;
int n,m,k,a[N],b[N],c[N];

bool check(int x, int y) {
    int total = x + y;
    int pos = upper_bound(b+1, b+n+1, total) - b - 1;
    if (n - pos >= m) {
        return false;
    }
    int rem = m - (n - pos);
    int sum_extra = 0;
    if (pos > n - m) {
        sum_extra = max(0LL, rem * (total + 1) - (c[pos] - c[pos - rem]));
    } else {
        sum_extra = rem * (total + 1) - (c[n] - c[n - m]);
    }
    return sum_extra <= (k - y);
}

signed main() {
    ios::sync_with_stdio(0), cin.tie(0), cout.tie(0);
    cin >> n >> m >> k;
    for (int i = 1; i <= n; ++i) {
        cin >> a[i];
        b[i] = a[i];
    }
    sort(b+1, b+n+1);
    for (int i = 1; i <= n; ++i) {
        c[i] = c[i-1] + b[i];
    }
    for (int i = 1; i <= n; ++i) {
        int l = 0, r = k;
        int ans = -1;
        while (l <= r) {
            int mid = (l + r) / 2;
            if (check(a[i], mid)) {
                ans = mid;
                r = mid - 1;
            } else {
                l = mid + 1;
            }
        }
        if (ans != -1 && ans <= k) {
            cout << ans << " ";
        } else {
            cout << "-1 ";
        }
    }
    return 0;
}
```

### Explanation
1. **Sorting and Prefix Sums**: The votes are sorted to easily find positions and sums. The prefix sums array `c` allows quick calculation of the sum of the top votes.
2. **Binary Search**: For each candidate, we perform a binary search to find the minimum X such that adding X votes to the candidate ensures they are in the top M candidates. The `check` function verifies if the candidate can be in the top M by comparing against the sorted votes and using prefix sums to calculate the required additional votes.
3. **Edge Cases Handling**: The binary search handles cases where a candidate might already be in the top M (X=0) or where it's impossible to be in the top M (X=-1). The solution efficiently checks these conditions using sorted data and binary search for optimal performance.

This approach ensures the solution is both correct and efficient, handling all edge cases within the problem constraints.
