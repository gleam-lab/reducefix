1. **Sort and Calculate Prefix Sums**: First, sort the results of the candidates in ascending order to know the votes distribution as we iterate over them. Calculate prefix sums to keep track of the sum of votes for the top \( i \) candidates.

2. **Calculate the Vote Requirement**: For each candidate, calculate how many votes they would need to win. This is done by finding the votes they would need to surpass the candidate at the \( M \)th position (or the \( N \)th position if \( M = N \)).

3. **Check Feasibility**: For each candidate, check if the required votes can be achieved with the remaining votes \( K \). If a candidate's required votes exceed \( K \), they cannot win. If they are already guaranteed a win, their required votes will be zero.

4. **Output the Results**: For each candidate, output the minimum votes needed or -1 if it's not possible.
