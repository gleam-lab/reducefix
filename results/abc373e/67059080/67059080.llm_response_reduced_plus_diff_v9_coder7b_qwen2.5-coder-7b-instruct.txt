To solve this problem efficiently, we need to ensure that each candidate can secure their victory regardless of how the remaining votes are distributed among the other candidates. Here's a step-by-step approach to achieve this:

1. **Sort Candidates by Votes**: First, sort the candidates based on the number of votes they currently have. This helps in determining the worst-case scenario where the candidate needs to win against the least voted candidate.

2. **Calculate Remaining Votes Needed**: For each candidate, calculate the minimum number of additional votes required to ensure they are ahead of the next `M-1` candidates. This involves checking how many votes are needed to surpass the current lead of the candidate ranked at position `i + M`.

3. **Binary Search for Minimum Votes**: Use binary search to determine the minimum number of votes needed. The search range should be between 0 and the total remaining votes.

Here's the corrected and optimized C++ code:

```cpp
#include <bits/stdc++.h>
using namespace std;
using ll = long long;

int N, M;
ll K;

struct Node {
    ll val;
    int idx;
};

bool canWin(const vector<Node>& a, ll add, int i) {
    int cnt = 0;
    for (int j = 0; j < N; ++j) {
        if (j != i && a[j].val + add > a[i].val) {
            ++cnt;
        }
        if (cnt >= M) break;
    }
    return cnt == M - 1;
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    cin >> N >> M >> K;

    vector<Node> a(N + 5);

    for (int i = 1; i <= N; ++i) {
        cin >> a[i].val;
        a[i].idx = i;
    }

    sort(a.begin() + 1, a.begin() + N + 1);

    vector<ll> pre(N + 5);
    pre[0] = 0LL;
    for (int i = 1; i <= N; ++i) {
        pre[i] = pre[i - 1] + a[i].val;
    }

    K -= pre[N];

    vector<ll> ans(N + 5, -1);

    for (int i = 1; i <= N; ++i) {
        if (a[i].idx != 1) {
            ans[a[i].idx] = 0;
            continue;
        }

        ll low = 0, high = K;
        while (low <= high) {
            ll mid = (low + high) / 2;
            if (canWin(a, mid, i)) {
                ans[a[i].idx] = mid;
                high = mid - 1;
            } else {
                low = mid + 1;
            }
        }
    }

    for (int i = 1; i <= N; ++i) {
        cout << ans[i] << " ";
    }
    cout << endl;

    return 0;
}
```

### Explanation
1. **Sorting**: We sort the candidates based on their vote counts to handle the worst-case scenario.
2. **Prefix Sum Array**: We use a prefix sum array to quickly calculate the total votes up to any point.
3. **Binary Search**: For each candidate, we perform a binary search to find the minimum number of votes required to ensure they are ahead of the next `M-1` candidates.
4. **Edge Cases**: We handle the case where a candidate already has enough votes to secure their victory directly.

This solution ensures that we efficiently determine the minimum number of votes each candidate needs, adhering to the constraints and requirements of the problem.
