3. **Checking Feasibility**: For each candidate, we calculate the smallest index `u` such that if the candidate with the vote count at `u` receives all remaining votes, the candidate in question cannot be elected. This is done by checking the number of candidates who have more votes than the current candidate and ensuring that this number is less than `M`.

4. **Edge Cases**: If a candidate already has more votes than `M-1` other candidates, they are guaranteed to win and do not need any more votes. If it's impossible for a candidate to win regardless of the distribution of remaining votes, the result is `-1`.
