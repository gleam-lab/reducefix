To fix this code, we need to address several issues related to the handling of the priority queue and the counting of the remaining island area. The main problem lies in how the sea level rise affects the island sections and how the sinking process is simulated.

### Approaches
1. **Initialization**:
   - We begin by reading the island dimensions (H, W) and the number of years (Y) for which we need to compute the remaining area.
   - We then read the elevation matrix of the island.

2. **Priority Queue Setup**:
   - The priority queue (min-heap) is initialized to process sections in order of their elevation. This helps simulate the sinking process as the sea level rises year by year.
   - The initial sections added to the queue are those on the borders of the island. These sections are the first candidates to sink as the sea level rises.

3. **Simulation of Sinking Process**:
   - For each sea level from 1 to Y, we process all sections in the priority queue that have an elevation less than or equal to the current sea level.
   - For each such section, we decrement the remaining area count and explore its adjacent sections. If these adjacent sections haven't been processed yet and their elevation is less than or equal to the current sea level, they are added to the priority queue to be processed in subsequent iterations.

4. **Output**:
   - After processing each sea level, the remaining area is printed.

### Solution Code
```cpp
#include <bits/stdc++.h>
using namespace std;
#define ll long long

ll dx[4] = {0, 1, 0, -1};
ll dy[4] = {1, 0, -1, 0};

int main() {
    ll h, w, y;
    cin >> h >> w >> y;
    ll ans = h * w;
    vector<vector<ll>> room(h, vector<ll>(w));
    for (ll i = 0; i < h; ++i) {
        for (ll j = 0; j < w; ++j) {
            cin >> room[i][j];
        }
    }
    priority_queue<vector<ll>, vector<vector<ll>>, greater<vector<ll>>> pq;
    vector<vector<bool>> used(h, vector<bool>(w, false));
    
    // Add border cells to the priority queue
    for (ll i = 0; i < h; ++i) {
        if (!used[i][0]) {
            pq.push({room[i][0], i, 0});
            used[i][0] = true;
        }
        if (!used[i][w - 1]) {
            pq.push({room[i][w - 1], i, w - 1});
            used[i][w - 1] = true;
        }
    }
    for (ll j = 1; j < w - 1; ++j) {
        if (!used[0][j]) {
            pq.push({room[0][j], 0, j});
            used[0][j] = true;
        }
        if (!used[h - 1][j]) {
            pq.push({room[h - 1][j], h - 1, j});
            used[h - 1][j] = true;
        }
    }
    
    for (ll current_level = 1; current_level <= y; ++current_level) {
        while (!pq.empty() && pq.top()[0] <= current_level) {
            ll elevation = pq.top()[0];
            ll i = pq.top()[1];
            ll j = pq.top()[2];
            pq.pop();
            ans--;
            for (ll k = 0; k < 4; ++k) {
                ll ni = i + dx[k];
                ll nj = j + dy[k];
                if (ni >= 0 && ni < h && nj >= 0 && nj < w && !used[ni][nj]) {
                    pq.push({room[ni][nj], ni, nj});
                    used[ni][nj] = true;
                }
            }
        }
        cout << ans << endl;
    }
    return 0;
}
```

### Explanation
1. **Initialization**:
   - The dimensions of the island (H, W) and the number of years (Y) are read.
   - The elevation matrix of the island is then read and stored.

2. **Priority Queue Setup**:
   - The priority queue is initialized to process sections in ascending order of their elevation. Sections on the island's borders are added to the queue first, as they are adjacent to the sea and will sink first when the sea level rises.

3. **Simulation of Sinking Process**:
   - For each year (sea level from 1 to Y), the code processes sections in the priority queue whose elevation is less than or equal to the current sea level. Each processed section's adjacent sections are checked and added to the queue if they meet the criteria (not processed and elevation ≤ current sea level).

4. **Output**:
   - After processing each sea level, the remaining area of the island is printed, which is the count of sections not yet sunk.

This approach efficiently simulates the sinking process using a min-heap to handle sections in order of their elevation, ensuring correctness and efficiency even for large input sizes.
