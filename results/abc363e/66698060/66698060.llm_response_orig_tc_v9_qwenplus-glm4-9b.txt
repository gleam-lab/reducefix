
The problem is about modeling and simulating the flooding of an island with increasing sea levels. The island is represented as a grid of cells with given elevations. Each year, the sea level rises by one unit, and cells that are adjacent to the sea or already flooded cells (with elevations not greater than the current sea level) will sink. This sinking process is recursive, as adjacent cells also sink.

Given the constraints, the problem is computationally challenging due to the large number of possible sinks and the number of years to simulate (up to 100,000). The provided code appears to use a priority queue to simulate the process, but it has several issues:

1. The code does not handle the recursive nature of the sinking process correctly. It only adds adjacent cells that have not been flooded yet to the priority queue, but it does not ensure that the sinking process continues to propagate through the grid.

2. The code does not efficiently keep track of which cells have already been flooded or which cells are adjacent to the sea.

3. The output format is incorrect as it prints the area of the island in an undefined format.

### Proposed Solution

To address these issues, we can use a flood fill algorithm to simulate the sinking process. Here's a high-level description of the approach:

1. Use a 2D array (or map) to represent the island, where each cell has a value indicating its current state (e.g., not flooded, flooded, or sunk).

2. Initialize the boundary cells along the edges with a special marker indicating they are part of the sea.

3. Use a flood fill algorithm starting from the boundary cells to mark all flooded cells.

4. Recursively apply the flood fill algorithm to any adjacent cells that are adjacent to a flooded cell and have not already been marked.

5. After each year, recalculate the area of the island that remains above sea level by counting the cells that are not marked as sunk.

6. Repeat the process for each of the Y years.
