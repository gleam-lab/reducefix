To fix this code, we need to ensure that the algorithm correctly simulates the sinking process of the island sections as the sea level rises each year. The initial approach uses a priority queue to process sections in order of their elevation, but there appears to be an issue in how the sections are initially added to the queue and how the sinking process propagates.

### Approaches
1. **Initialization**: Start by adding all border sections of the island to a priority queue. These sections are the first to potentially sink as the sea level rises.
2. **Priority Queue**: Use a min-heap to process sections in increasing order of their elevation. This ensures that sections that should sink earliest are processed first.
3. **BFS Propagation**: When a section sinks, check its adjacent sections (up, down, left, right). If an adjacent section's elevation is not greater than the current sea level and hasn't been processed yet, add it to the priority queue to be processed.
4. **Result Tracking**: For each year, after processing all sections that sink at that sea level, record the remaining area of the island.

#include <bits/stdc++.h>

using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int H, W, Y;
    cin >> H >> W >> Y;

    vector<vector<int>> A(H, vector<int>(W));
    for (int i = 0; i < H; ++i) {
        for (int j = 0; j < W; ++j) {
            cin >> A[i][j];
        }
    }

    const int dx[] = {0, 0, 1, -1};
    const int dy[] = {1, -1, 0, 0};

    priority_queue<pair<int, pair<int, int>>, vector<pair<int, pair<int, int>>>, greater<pair<int, pair<int, int>>>> pq;
    vector<vector<bool>> visited(H, vector<bool>(W, false));

    // Push all border cells into the priority queue
    for (int i = 0; i < H; ++i) {
        if (!visited[i][0]) {
            pq.push({A[i][0], {i, 0}});
            visited[i][0] = true;
        }
        if (!visited[i][W - 1]) {
            pq.push({A[i][W - 1], {i, W - 1}});
            visited[i][W - 1] = true;
        }
    }
    for (int j = 1; j < W - 1; ++j) {
        if (!visited[0][j]) {
            pq.push({A[0][j], {0, j}});
            visited[0][j] = true;
        }
        if (!visited[H - 1][j]) {
            pq.push({A[H - 1][j], {H - 1, j}});
            visited[H - 1][j] = true;
        }
    }

    int remaining = H * W;
    vector<int> result(Y);

    for (int year = 1; year <= Y; ++year) {
        while (!pq.empty() && pq.top().first <= year) {
            auto current = pq.top();
            pq.pop();
            int h = current.first;
            int x = current.second.first;
            int y = current.second.second;
            --remaining;
            for (int dir = 0; dir < 4; ++dir) {
                int nx = x + dx[dir];
                int ny = y + dy[dir];
                if (nx >= 0 && nx < H && ny >= 0 && ny < W && !visited[nx][ny]) {
                    visited[nx][ny] = true;
                    pq.push({A[nx][ny], {nx, ny}});
                }
            }
        }
        result[year - 1] = remaining;
    }

    for (int i = 0; i < Y; ++i) {
        cout << result[i] << '\n';
    }

    return 0;
}

### Explanation
1. **Initialization**: The input is read, including the island dimensions (H, W), the number of years (Y), and the elevation matrix (A).
2. **Priority Queue Setup**: Border sections of the island are added to a priority queue (min-heap) to start the sinking simulation. These sections are marked as visited to avoid reprocessing.
3. **Yearly Simulation**: For each year, the algorithm processes all sections in the priority queue that have an elevation less than or equal to the current year's sea level. Each processed section is removed from the island (remaining area decreases), and its adjacent sections are checked and added to the queue if they meet the sinking criteria.
4. **Result Tracking**: The remaining area after processing each year is stored in a result vector, which is printed at the end.

This approach efficiently simulates the sinking process using a priority queue to handle sections in the correct order and ensures that each section is processed only once, maintaining optimal performance.
