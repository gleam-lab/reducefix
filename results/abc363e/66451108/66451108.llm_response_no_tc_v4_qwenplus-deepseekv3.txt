The problem involves simulating the sinking of island sections as the sea level rises each year, using a priority queue to manage the sections that are adjacent to the sea or already sunk. The key issues in the original code include incorrect initialization of the priority queue, inefficient handling of the sinking process, and potential overflow or incorrect grid dimensions.

### Approaches
1. **Initialization**: The priority queue should be initialized with all boundary sections of the grid, as these are the first to be considered for sinking when the sea level rises.
2. **Priority Queue Management**: The priority queue should prioritize sections with the smallest elevation values, as these will sink first when the sea level reaches their elevation.
3. **Sinking Process**: For each year, sections with elevation less than or equal to the current sea level should be processed. Each time a section sinks, its adjacent sections should be checked and added to the queue if they haven't been processed yet.
4. **Efficiency**: Given the constraints, the algorithm must efficiently handle up to \(10^5\) years and a grid size of up to \(1000 \times 1000\). Using a min-heap (priority queue) ensures that each section is processed in logarithmic time relative to the queue size.

#include <bits/stdc++.h>
using namespace std;
#define int long long
#define close ios::sync_with_stdio(false); cin.tie(0); cout.tie(0);
#define MAXN 1000
#define LIMIT 22
#define inf 0x3f3f3f3f3f3f
#define endl '\n'
const double eps = 1e-7;
int dx[] = {0, 0, 1, -1};
int dy[] = {1, -1, 0, 0};
const int MOD = 1e9 + 7;

struct Node {
    int x, y, val;
    bool operator<(const Node &a) const {
        return val > a.val;
    }
};

signed main() {
    close;
    int h, w, Y;
    cin >> h >> w >> Y;
    vector<vector<int>> grid(h, vector<int>(w));
    vector<vector<bool>> vis(h, vector<bool>(w, false));
    priority_queue<Node> q;

    for (int i = 0; i < h; ++i) {
        for (int j = 0; j < w; ++j) {
            cin >> grid[i][j];
        }
    }

    // Push all boundary cells into the priority queue
    for (int i = 0; i < h; ++i) {
        for (int j = 0; j < w; ++j) {
            if (i == 0 || i == h - 1 || j == 0 || j == w - 1) {
                q.push({i, j, grid[i][j]});
                vis[i][j] = true;
            }
        }
    }

    int res = h * w;
    vector<int> ans(Y + 1, res); // 1-based indexing for years

    for (int level = 1; level <= Y; ++level) {
        while (!q.empty() && q.top().val <= level) {
            Node cur = q.top();
            q.pop();
            ans[level]--;
            for (int i = 0; i < 4; ++i) {
                int nx = cur.x + dx[i];
                int ny = cur.y + dy[i];
                if (nx >= 0 && nx < h && ny >= 0 && ny < w && !vis[nx][ny]) {
                    vis[nx][ny] = true;
                    q.push({nx, ny, grid[nx][ny]});
                }
            }
        }
        if (level < Y) {
            ans[level + 1] = ans[level];
        }
    }

    for (int i = 1; i <= Y; ++i) {
        cout << ans[i] << endl;
    }

    return 0;
}

### Explanation
1. **Initialization**: The grid and a priority queue are initialized. The priority queue is set up to process sections in order of their elevation (lowest first).
2. **Boundary Handling**: All boundary sections of the grid are added to the priority queue and marked as visited. These sections are the first to be checked for sinking as the sea level rises.
3. **Yearly Processing**: For each year from 1 to Y, the code checks all sections in the priority queue with elevation less than or equal to the current sea level. Each time a section is processed (sinks), its adjacent sections are added to the queue if they haven't been processed yet.
4. **Result Tracking**: The results for each year are stored in an array, with each year's result being the remaining area of the island after processing all sinking sections up to that year's sea level.
5. **Output**: Finally, the results for each year are printed.

This approach ensures that the sinking process is simulated efficiently, handling the constraints effectively and providing the correct remaining island area for each year.
