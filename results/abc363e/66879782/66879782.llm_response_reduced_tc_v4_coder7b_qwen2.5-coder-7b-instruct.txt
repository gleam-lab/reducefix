To solve this problem efficiently, we need to simulate the rising sea levels and determine the remaining land area after each level. Given the constraints, a breadth-first search (BFS) approach can be used to handle the spreading of water effectively. Here’s how you can implement it:

1. **Initialize the BFS Queue**: Start by adding all cells that are initially below the sea level to a queue.
2. **Simulate Sea Level Rise**: For each sea level rise, pop elements from the queue, mark them as submerged, and add their unvisited neighboring cells (above, below, left, right) to the queue if they are also below the new sea level.
3. **Count Remaining Land**: After processing all cells up to the current sea level, count the number of cells that remain unsubmerged.

This approach ensures that we only consider reachable land areas and efficiently track the spread of water using BFS.

Here's the corrected C++ code:

```cpp
#include <bits/stdc++.h>
using namespace std;

typedef pair<int, pair<int, int>> pi;

int main() {
    ios_base::sync_with_stdio(0);
    cin.tie(0);

    int h, w, y;
    cin >> h >> w >> y;
    
    vector<vector<int>> height(h, vector<int>(w));
    vector<vector<bool>> sink(h, vector<bool>(w, false));

    for(int i = 0; i < h; ++i) {
        for(int j = 0; j < w; ++j) {
            cin >> height[i][j];
            if(height[i][j] > y) sink[i][j] = true;
        }
    }

    vector<int> dir = {-1, 0, 1, 0, -1};
    int ans = h * w;

    for(int l = 1; l <= y; ++l) {
        int sz = ans;
        for(int i = 0; i < h; ++i) {
            for(int j = 0; j < w; ++j) {
                if(!sink[i][j] && height[i][j] <= l) {
                    queue<pi> q;
                    q.push({height[i][j], {i, j}});
                    sink[i][j] = true;

                    while(!q.empty()) {
                        auto [hl, p] = q.front(); q.pop();
                        int x = p.first, y = p.second;
                        for(int k = 0; k < 4; ++k) {
                            int nx = x + dir[k], ny = y + dir[k + 1];
                            if(nx >= 0 && nx < h && ny >= 0 && ny < w && !sink[nx][ny] && height[nx][ny] <= l) {
                                sink[nx][ny] = true;
                                q.push({height[nx][ny], {nx, ny}});
                            }
                        }
                    }
                }
            }
        }
        ans -= (sz - ans);
        cout << ans << '\n';
    }

    return 0;
}
```

### Explanation:
1. **Initialization**:
   - Read the dimensions `h` and `w`, and the number of queries `y`.
   - Initialize the `height` matrix to store the elevation of each cell.
   - Initialize the `sink` matrix to keep track of which cells are submerged.
   
2. **Reading Input**:
   - Populate the `height` matrix.
   - Mark cells that are already submerged due to initial conditions (`height[i][j] > y`) in the `sink` matrix.

3. **Simulation**:
   - Iterate over each sea level from 1 to `y`.
   - For each level, use BFS to mark all connected land cells as submerged.
   - Count the number of remaining unsubmerged cells.

4. **Output**:
   - Print the remaining land area after each sea level rise.

This solution ensures that we correctly simulate the rising sea levels and accurately compute the remaining land area efficiently.
