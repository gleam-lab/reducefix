To fix this code, we need to ensure that the algorithm correctly processes the sea level rise and accurately tracks the sections of the island that sink each year. The main issue with the initial code was that it did not properly handle the adjacency checks and the sinking propagation, leading to incorrect counts of remaining sections.

### Approaches
1. **Initialization**: We start by reading the input dimensions and elevation data.
2. **Priority Queue Setup**: We use a priority queue to process sections starting from the borders, as these are the first to be affected by rising sea levels.
3. **BFS for Sinking Propagation**: For each sea level, we process all sections that sink at that level using a BFS approach to propagate the sinking to adjacent sections.
4. **Tracking Remaining Sections**: We maintain a count of the remaining sections above sea level and output it for each year.

using vi = vc<int>;
using vvi = vc<vi>;
using Pi = pair<int, int>;

template<typename T> struct cmp {
    bool operator()(const T &l, const T &r) { return l[0] > r[0]; }
};

int INF = 1e9 + 10;

int main() {
    int H, W, Y;
    cin >> H >> W >> Y;
    vvi A(H, vi(W)), visited(H, vi(W));
    cin >> A;

    priority_queue<vi, vvi, cmp<vi>> pq;
    
    // Push border cells into the priority queue
    for (int i = 0; i < H; ++i) {
        pq.push({A[i][0], i, 0});
        visited[i][0] = 1;
        if (W > 1) {
            pq.push({A[i][W - 1], i, W - 1});
            visited[i][W - 1] = 1;
        }
    }
    for (int j = 1; j < W - 1; ++j) {
        pq.push({A[0][j], 0, j});
        visited[0][j] = 1;
        if (H > 1) {
            pq.push({A[H - 1][j], H - 1, j});
            visited[H - 1][j] = 1;
        }
    }

    int ans = H * W;
    vi dx = {1, 0, -1, 0}, dy = {0, 1, 0, -1};
    
    for (int year = 1; year <= Y; ++year) {
        queue<Pi> q;
        while (!pq.empty() && pq.top()[0] <= year) {
            auto cell = pq.top();
            pq.pop();
            int i = cell[1], j = cell[2];
            if (A[i][j] > year) continue; // This should not happen as per the priority queue
            --ans;
            q.emplace(i, j);
        }
        
        // BFS to propagate sinking to adjacent cells
        while (!q.empty()) {
            auto [i, j] = q.front();
            q.pop();
            for (int d = 0; d < 4; ++d) {
                int ni = i + dx[d], nj = j + dy[d];
                if (ni < 0 || nj < 0 || ni >= H || nj >= W || visited[ni][nj]) continue;
                if (A[ni][nj] <= year) {
                    visited[ni][nj] = 1;
                    --ans;
                    q.emplace(ni, nj);
                } else {
                    visited[ni][nj] = 1;
                    pq.push({A[ni][nj], ni, nj});
                }
            }
        }
        cout << ans << '\n';
    }
}

### Explanation
1. **Initialization**: The code starts by reading the island dimensions (H, W) and the number of years (Y), followed by the elevation data for each section of the island.
2. **Priority Queue Setup**: The priority queue is initialized with the border sections of the island, as these are the first to be affected by rising sea levels. This ensures we process sections in order of their elevation.
3. **Processing Each Year**: For each year, the code checks sections from the priority queue that will sink at the current sea level. These sections are processed, and their adjacent sections are checked. If an adjacent section's elevation is not greater than the current sea level, it sinks and is added to the queue for further propagation. Otherwise, it is added to the priority queue for future processing.
4. **Output**: After processing each year, the remaining sections above sea level are counted and printed.

This approach ensures that we correctly propagate the sinking effect and efficiently track the remaining sections above sea level for each year. The use of a priority queue and BFS ensures that the solution is both correct and efficient.
