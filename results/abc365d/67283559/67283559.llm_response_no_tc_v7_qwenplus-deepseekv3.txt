Here's the fixed solution that correctly implements the optimal strategy for maximizing the score in an RPS game where you can't use the same move consecutively:

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    int N;
    cin >> N;
    string S;
    cin >> S;
    
    int total = 0;
    char last = '\0';
    
    for (int i = 0; i < N; i++) {
        char current = S[i];
        char choice;
        
        if (last != 'R' && (current == 'S')) {
            choice = 'R';
        } else if (last != 'P' && (current == 'R')) {
            choice = 'P';
        } else if (last != 'S' && (current == 'P')) {
            choice = 'S';
        } else {
            // Choose any valid move that doesn't match last, prioritizing winning moves
            if (last != 'R') choice = 'R';
            else if (last != 'P') choice = 'P';
            else choice = 'S';
        }
        
        if ((choice == 'R' && current == 'S') || 
            (choice == 'P' && current == 'R') || 
            (choice == 'S' && current == 'P')) {
            total++;
        }
        
        last = choice;
    }
    
    cout << total;
    return 0;
}
```

This solution:
1. Iterates through each round of the game
2. For each round, selects the optimal move that:
   - Beats the opponent's current move (if possible)
   - Doesn't repeat the previous move
3. If no winning move is available that doesn't repeat the previous move, chooses any valid non-repeating move
4. Tracks the score based on whether the chosen move beats the opponent's move
5. Outputs the total score
