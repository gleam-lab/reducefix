### Revised Approach
To ensure the maximum score, we need to consider all possible sequences of moves that maximize the score at each step while adhering to the rule that the next move cannot be the same as the current move. This can be efficiently done using dynamic programming (DP) to track the maximum score achievable at each step with each possible move.

### Solution Code
```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    int N;
    cin >> N;
    string S;
    cin >> S;
    
    if (N == 0) {
        cout << 0 << endl;
        return 0;
    }
    
    // DP table: dp[i][c] represents the maximum score up to the i-th round using move c
    // c can be 'R', 'P', 'S'
    vector<unordered_map<char, int>> dp(N);
    
    // Initialize the first round
    dp[0]['R'] = (S[0] == 'S') ? 1 : 0;
    dp[0]['P'] = (S[0] == 'R') ? 1 : 0;
    dp[0]['S'] = (S[0] == 'P') ? 1 : 0;
    
    for (int i = 1; i < N; ++i) {
        // For each possible move in the current round
        for (char current : {'R', 'P', 'S'}) {
            int score = (current == 'R' && S[i] == 'S') ? 1 :
                        (current == 'P' && S[i] == 'R') ? 1 :
                        (current == 'S' && S[i] == 'P') ? 1 : 0;
            
            // The previous move cannot be the same as the current move
            for (char prev : {'R', 'P', 'S'}) {
                if (prev != current) {
                    if (dp[i][current] < dp[i-1][prev] + score) {
                        dp[i][current] = dp[i-1][prev] + score;
                    }
                }
            }
        }
    }
    
    // The answer is the maximum score in the last round considering all possible moves
    int max_score = max({dp[N-1]['R'], dp[N-1]['P'], dp[N-1]['S']});
    cout << max_score << endl;
    
    return 0;
}
```

### Explanation
1. **Initialization**: The DP table is initialized to store the maximum score achievable at each round for each possible move (R, P, S).
2. **First Round**: The score for the first round is calculated based on the opponent's first move.
3. **Subsequent Rounds**: For each subsequent round, the score is calculated based on the current move and the previous move (ensuring they are different). The DP table is updated to reflect the maximum score achievable.
4. **Result Extraction**: The maximum score from the last round is taken as the result, representing the highest possible score achievable under the game's rules.

This approach ensures that all possible optimal sequences are considered, leading to the correct maximum score. The time complexity is O(N), which is efficient for the problem constraints.
