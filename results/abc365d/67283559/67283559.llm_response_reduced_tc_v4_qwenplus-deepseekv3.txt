### Correct Approach
1. **Dynamic Programming**: We can use dynamic programming (DP) to keep track of the maximum score achievable at each step, considering the constraints of not using the same move consecutively.
2. **DP State**: For each position `i` in the sequence and for each possible move `m` (R, P, S), we can store the maximum score achievable up to that position if we choose move `m` at position `i`.
3. **Transition**: For each move at position `i`, the score is incremented if the move beats the opponent's move at `i`. The next move at `i+1` cannot be the same as the current move.

### Solution Code
```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    int N;
    cin >> N;
    string S;
    cin >> S;
    
    // DP table: dp[i][m] represents the max score up to position i with move m
    vector<vector<int>> dp(N, vector<int>(3, 0));
    
    // Initialize the first move
    for (int m = 0; m < 3; ++m) {
        char move = (m == 0) ? 'R' : (m == 1) ? 'P' : 'S';
        if (move == 'R' && S[0] == 'S') dp[0][m] = 1;
        else if (move == 'P' && S[0] == 'R') dp[0][m] = 1;
        else if (move == 'S' && S[0] == 'P') dp[0][m] = 1;
    }
    
    // Fill the DP table
    for (int i = 1; i < N; ++i) {
        for (int m = 0; m < 3; ++m) {
            char move = (m == 0) ? 'R' : (m == 1) ? 'P' : 'S';
            int current_score = 0;
            if (move == 'R' && S[i] == 'S') current_score = 1;
            else if (move == 'P' && S[i] == 'R') current_score = 1;
            else if (move == 'S' && S[i] == 'P') current_score = 1;
            
            // The previous move cannot be the same as the current move
            for (int prev_m = 0; prev_m < 3; ++prev_m) {
                if (prev_m != m) {
                    dp[i][m] = max(dp[i][m], dp[i-1][prev_m] + current_score);
                }
            }
        }
    }
    
    // The answer is the maximum score achievable at the last position
    int max_score = max({dp[N-1][0], dp[N-1][1], dp[N-1][2]});
    cout << max_score << endl;
    
    return 0;
}
```

### Explanation
1. **Initialization**: The DP table is initialized for the first move. Each possible move (R, P, S) is checked against the opponent's first move to determine if it wins, which would give a score of 1.
2. **DP Table Filling**: For each subsequent move, the score is calculated based on whether the current move beats the opponent's move at that position. The key constraint is that the same move cannot be used consecutively, so the previous move must be different from the current move.
3. **Result Extraction**: The maximum score is found by taking the maximum value from the last position in the DP table, representing the best score achievable after all moves.

This approach ensures that we optimally choose moves to maximize the score while adhering to the constraint of not using the same move consecutively. The time complexity is O(N), which is efficient for the problem constraints.
