To solve this problem, we need to determine the **minimum number of hand signs** required to beat a given sequence of Rock-Paper-Scissors moves (`R`, `P`, or `S`) in a game where:

- Each move you make must beat the corresponding move in the input string.
- You can repeat moves, but once you choose a move, you can continue using it until you change to a different one.

For example:
- If the opponent plays `'PPP'`, you can just use `'S'` once to beat all of them.
- If the opponent plays `'PRSP'`, you'd have to change your hand sign multiple times depending on what beats each character:
  - `'P'` → you play `'S'`
  - `'R'` → you play `'P'`
  - `'S'` → you play `'R'`
  - `'P'` → you play `'S'`

The core idea is that when consecutive characters require the same move from us, we count only once — so we want to **count transitions** between required winning moves.

---

### Fixing the Logic

In the original code, the logic was incorrect because it didn't correctly handle which move should be played to beat the current character. For instance:

```cpp
if(prev != win(s[i])){
    ans++;
    prev = win(s[i]);
}
```

This approach assumes that `win(s[i])` gives the correct move to beat `s[i]`, and counts transitions between successive winning moves.

However, there was no validation that the `win()` function actually returns the right move for each case. Also, some conditions were missing (like handling `'X'`).

---

### Final Corrected Code

```cpp
#include <iostream>
#include <string>
using namespace std;

// Returns the move that beats 'c'
char win(char c) {
    if (c == 'R') return 'P';
    if (c == 'S') return 'R';
    if (c == 'P') return 'S';
    return 'X'; // fallback
}

int main() {
    int n;
    string s;
    cin >> n >> s;

    int ans = 0;
    char prev_move = 'X'; // nothing initially

    for (int i = 0; i < n; ++i) {
        char current_move = win(s[i]);

        if (current_move != prev_move) {
            ans++;
            prev_move = current_move;
        }
    }

    cout << ans << endl;
    return 0;
}
```

---

### Explanation

- The `win()` function maps each character to the one that beats it.
- We iterate through the string and for each character, compute the required move to beat it.
- If the required move changes from the previous move, we increment our answer — since that means we had to switch handsigns.
- This results in an optimal solution with **O(n)** time complexity.
