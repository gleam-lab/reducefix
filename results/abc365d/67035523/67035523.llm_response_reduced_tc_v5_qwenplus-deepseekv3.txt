To fix this code, we need to ensure that the algorithm correctly counts the number of times the optimal strategy changes its move to beat the opponent's sequence. The initial approach had a flaw in how it tracked the previous move and decided when to increment the answer.

### Approaches
1. **Understanding the Problem**: The task is to determine the minimum number of changes needed in a sequence of moves (Rock, Paper, Scissors) such that every move in the sequence beats the corresponding move in the opponent's sequence. The key observation here is that whenever the current optimal move differs from the previous optimal move, we need to increment the count.

2. **Optimal Move Calculation**: For each move in the opponent's sequence, the optimal move is the one that beats it (e.g., 'P' beats 'R', 'R' beats 'S', 'S' beats 'P').

3. **Tracking Changes**: We need to keep track of the previous optimal move. If the current optimal move is different from the previous one, we increment our answer and update the previous move. Otherwise, we continue without incrementing.

### Solution Code
```cpp
#include <iostream>
#include <string>
using namespace std;

char win(char x) {
    if (x == 'R') return 'P';
    if (x == 'S') return 'R';
    if (x == 'P') return 'S';
    return 'X'; // default case, though not needed for correct input
}

int main() {
    int n;
    string s;
    cin >> n >> s;
    if (n == 0) {
        cout << 0 << endl;
        return 0;
    }
    char prev = win(s[0]);
    int ans = 1;
    for (int i = 1; i < n; ++i) {
        char current = win(s[i]);
        if (current != prev) {
            ans++;
            prev = current;
        }
    }
    cout << ans << endl;
    return 0;
}
```

### Explanation
1. **Input Handling**: The code reads the number of moves `n` and the sequence of moves `s` from the input.
2. **Edge Case Handling**: If `n` is 0, it directly outputs 0 since no moves are needed.
3. **Initial Setup**: The initial optimal move is set to the move that beats the first element of the sequence `s`.
4. **Loop Through Moves**: For each subsequent move in `s`, the code checks if the current optimal move (to beat `s[i]`) is different from the previous optimal move. If it is, the answer is incremented, and the previous move is updated.
5. **Output the Result**: The total number of changes (answer) is printed.

This approach ensures that we efficiently count the number of changes required by only incrementing the count when the optimal move changes, leading to the correct solution.
