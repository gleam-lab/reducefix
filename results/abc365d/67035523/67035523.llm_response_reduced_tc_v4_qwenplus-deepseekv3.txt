### Fixed Code
```cpp
#include <iostream>
#include <string>
using namespace std;

char win(char x) {
    if (x == 'R') return 'P';
    if (x == 'S') return 'R';
    if (x == 'P') return 'S';
    return 'X'; // default case, though not expected
}

int main() {
    int n;
    string s;
    cin >> n >> s;
    int changes = 0;
    
    for (int i = 0; i < n - 1; ++i) {
        if (s[i] == win(s[i+1])) {
            continue; // current move beats the next, no change needed
        } else {
            changes++; // need to change either current or next move
            i++; // skip the next move since we've accounted for it
        }
    }
    
    cout << changes << endl;
    return 0;
}
```

### Explanation
1. **Function `win`**: This helper function returns the move that beats the input move according to Rock-Paper-Scissors rules.
2. **Main Logic**: 
   - Iterate through the string `s` from the first move to the second last move.
   - For each move `s[i]`, check if it beats the next move `s[i+1]` using the `win` function.
   - If it does not beat the next move, increment the `changes` counter and skip the next move (since changing either move resolves the issue for the current pair).
3. **Output**: The total number of changes needed is printed.

This approach ensures that we correctly count the minimal number of changes required to make each move in the sequence beat the next move, handling all edge cases efficiently. The complexity is O(N), which is optimal for this problem.
