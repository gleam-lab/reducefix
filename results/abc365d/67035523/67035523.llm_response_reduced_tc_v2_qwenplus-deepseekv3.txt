### Correct Approach
1. **Understanding the Game**: The game is Rock-Paper-Scissors (RPS). To win against a sequence of moves, we need to choose a move that beats the current opponent's move. If consecutive moves are the same, we can stick with the same winning move, but if they differ, we need to change our move accordingly.
2. **Counting Changes**: The number of changes required is equal to the number of times the optimal winning move changes when moving from one character to the next in the string. For example, in "PPR", the optimal moves are 'S' (to beat 'P') and then 'P' (to beat 'R'), resulting in two changes.

### Solution Code
```cpp
#include <iostream>
#include <string>
using namespace std;

char getWinMove(char c) {
    if (c == 'R') return 'P';
    if (c == 'S') return 'R';
    if (c == 'P') return 'S';
    return 'X'; // should not happen
}

int main() {
    int n;
    string s;
    cin >> n >> s;
    
    if (n == 0) {
        cout << 0 << endl;
        return 0;
    }
    
    int changes = 0;
    char current = getWinMove(s[0]);
    
    for (int i = 1; i < n; ++i) {
        char next = getWinMove(s[i]);
        if (next != current) {
            changes++;
            current = next;
        }
    }
    
    cout << changes + 1 << endl;
    return 0;
}
```

### Explanation
1. **Function `getWinMove`**: This helper function returns the move that beats the given character in RPS (e.g., 'P' beats 'R', 'R' beats 'S', 'S' beats 'P').
2. **Initial Check**: If the input string is empty, the output is 0, as no moves are needed.
3. **Main Loop**: The loop starts from the second character in the string. For each character, it checks if the optimal winning move changes compared to the previous character. If it does, it increments the change count.
4. **Output**: The total number of changes is `changes + 1` because the first move is always counted as one change (from nothing to the first optimal move).

This approach ensures that we correctly count the number of times the optimal winning move changes, leading to the correct result for all test cases. The complexity is O(N), which is efficient for the given constraints.
