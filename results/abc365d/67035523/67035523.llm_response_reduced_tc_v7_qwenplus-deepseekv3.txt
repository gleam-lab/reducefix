To fix this code, we need to correctly calculate the number of times the optimal strategy changes its move to always win against the sequence of moves in the string. The initial approach was incorrect because it didn't account for consecutive moves that could be won with the same optimal response, leading to an underestimation of the required changes.

### Approaches
1. **Problem Analysis**: The problem requires determining the minimum number of times the optimal strategy changes its move to always win against the sequence of moves in the string. The optimal move against any move `s[i]` is `win(s[i])` (e.g., 'P' beats 'R').
2. **Key Insight**: The optimal strategy should change its move only when the current optimal move differs from the previous optimal move. If the current optimal move is the same as the previous one, no change is needed.
3. **Algorithm**: 
   - Iterate through each character in the string.
   - For each character, determine the optimal move (`win(s[i])`).
   - If the current optimal move is different from the previous move, increment the count and update the previous move.
4. **Complexity**: The algorithm runs in O(N) time, where N is the length of the string, as it processes each character exactly once.

### Solution Code
```cpp
#include <iostream>
#include <string>
using namespace std;

char win(char x) {
    if (x == 'R') return 'P';
    if (x == 'S') return 'R';
    if (x == 'P') return 'S';
    return 'X'; // default case, though not needed if input is correct
}

int main() {
    int n;
    string s;
    cin >> n >> s;
    int ans = 0;
    char prev = 'X'; // Initialize to an invalid move
    
    for (int i = 0; i < n; i++) {
        char current_win = win(s[i]);
        if (current_win != prev) {
            ans++;
            prev = current_win;
        }
    }
    
    cout << ans << endl;
    return 0;
}
```

### Explanation
1. **Function `win(char x)`**: This helper function returns the optimal move to beat the input move `x`. For example, if `x` is 'R', it returns 'P' (since paper beats rock).
2. **Initialization**: The variables `n` and `s` are read from standard input, representing the length of the string and the string itself, respectively. `ans` is initialized to 0 to count the number of changes in the optimal strategy, and `prev` is initialized to an invalid move ('X') to ensure the first move is always counted.
3. **Iteration and Counting**: For each character in the string, the optimal move is determined. If this move differs from the previous optimal move (`prev`), the count (`ans`) is incremented, and `prev` is updated to the current optimal move.
4. **Output**: The total count of changes in the optimal strategy is printed.

This approach efficiently counts the necessary changes by leveraging the optimal move for each character and ensuring changes are only counted when the optimal move changes, thus minimizing the total number of changes.
