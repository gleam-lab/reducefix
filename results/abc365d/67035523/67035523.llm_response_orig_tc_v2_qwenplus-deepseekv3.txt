To fix this code, we need to correctly calculate the number of times the optimal strategy changes to counter the opponent's moves in the Rock-Paper-Scissors game. The initial approach was incorrect because it didn't account for all possible optimal moves and their transitions properly.

### Approaches
1. **Understanding the Game**: The optimal strategy involves countering each opponent's move with a move that beats it. Specifically:
   - Rock (R) is beaten by Paper (P)
   - Paper (P) is beaten by Scissors (S)
   - Scissors (S) is beaten by Rock (R)
2. **Optimal Moves**: For each move in the sequence, the optimal counter is determined by the function `win(char x)` which returns the move that beats `x`.
3. **Transitions**: The key insight is that the optimal strategy changes only when the current optimal move differs from the previous one. If it's the same, we don't increment the count. Otherwise, we do.
4. **Edge Cases**: Handle sequences where consecutive moves might require the same optimal counter, ensuring transitions are counted correctly.

### Solution Code
```cpp
#include <iostream>
#include <string>
using namespace std;

char win(char x) {
    if (x == 'R') return 'P';
    if (x == 'P') return 'S';
    if (x == 'S') return 'R';
    return 'X'; // should not happen if input is valid
}

int main() {
    int n;
    string s;
    cin >> n >> s;
    
    if (n == 0) {
        cout << 0 << endl;
        return 0;
    }
    
    int changes = 0;
    char prev_optimal = 'X'; // initialize to an invalid move
    
    for (int i = 0; i < n; ++i) {
        char current_optimal = win(s[i]);
        if (current_optimal != prev_optimal) {
            changes++;
            prev_optimal = current_optimal;
        }
    }
    
    cout << changes << endl;
    return 0;
}
```

### Explanation
1. **Function `win(char x)`**: This helper function returns the move that beats the input move `x` (e.g., 'P' for 'R', 'S' for 'P', 'R' for 'S').
2. **Initialization**: The main function reads the input values `n` (length of the string) and `s` (the sequence of moves). If `n` is zero, it outputs zero immediately.
3. **Loop Through Moves**: For each move in the string, the optimal counter is determined. If this counter differs from the previous optimal move, the count of changes (`changes`) is incremented, and the previous optimal move is updated.
4. **Output**: The total number of changes in the optimal strategy is printed, which corresponds to the expected output.

This approach ensures that the solution is efficient (O(N) time complexity) and handles all edge cases correctly, including sequences where consecutive moves might require the same optimal counter.
