To fix this code, we need to correctly calculate the number of times the player changes their strategy to win against the sequence of opponent's moves. The initial approach was incorrect because it didn't properly account for consecutive identical opponent moves and the optimal strategy switches.

### Approaches
1. **Problem Analysis**: The problem involves a sequence of moves (R, P, S) representing Rock, Paper, Scissors. The player needs to choose a move that beats the opponent's current move. The goal is to determine the minimum number of times the player needs to change their strategy to always win.
2. **Insight**: The player should change their strategy only when the consecutive opponent's moves change in a way that the previous winning move no longer applies. For example, if the opponent's moves are "PR", the player would choose 'P' for 'R' and then switch to 'S' for 'P'.
3. **Algorithm**: 
   - Iterate through the sequence of opponent's moves.
   - For each move, determine the winning move (R -> P, P -> S, S -> R).
   - If the current winning move is different from the previous one, increment the change count.
   - Handle consecutive identical moves by only counting a change when the opponent's move changes to a different one that requires a different winning move.

### Solution Code
```cpp
#include <iostream>
#include <string>

using namespace std;

char winning_move(char x) {
    if (x == 'R') return 'P';
    if (x == 'P') return 'S';
    if (x == 'S') return 'R';
    return 'X'; // should not happen
}

int main() {
    int n;
    string s;
    cin >> n >> s;
    
    if (n == 0) {
        cout << 0 << endl;
        return 0;
    }
    
    int changes = 0;
    char current_win = winning_move(s[0]);
    
    for (int i = 1; i < n; ++i) {
        char next_win = winning_move(s[i]);
        if (next_win != current_win) {
            changes++;
            current_win = next_win;
        }
    }
    
    // The initial move counts as the first strategy, so changes+1 is the total strategies used
    // But the problem asks for the number of changes, so changes is correct if we start with the first move's winning strategy.
    cout << changes + 1 << endl;
    
    return 0;
}
```

### Explanation
- **Initialization**: The code reads the length of the sequence and the sequence itself.
- **Edge Case Handling**: If the sequence is empty (n=0), it immediately outputs 0.
- **Winning Move Determination**: The function `winning_move` returns the move that beats the opponent's move (R beats S, S beats P, P beats R).
- **Iteration and Counting Changes**: The loop iterates through the sequence, starting from the second move. For each move, it checks if the winning move changes compared to the previous one. If it does, it increments the change counter and updates the current winning move.
- **Output**: The result is the number of changes plus one (accounting for the initial strategy) printed out.

This approach efficiently counts the necessary strategy changes by focusing on transitions in the opponent's moves, ensuring optimal performance with O(N) time complexity.
