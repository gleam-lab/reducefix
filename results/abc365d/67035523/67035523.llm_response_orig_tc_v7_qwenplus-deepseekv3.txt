To fix this code, we need to address the issue where the initial solution was not correctly calculating the number of optimal moves in the Rock-Paper-Scissors game sequence. The problem requires us to determine the minimum number of changes needed to ensure that no two consecutive moves are the same, and each move beats the previous move in the sequence.

### Approaches
1. **Initial Analysis**: The original code attempted to iterate through the sequence and count changes when a move did not beat the previous move. However, it missed some critical cases where consecutive moves could be adjusted more optimally.
2. **Optimal Strategy**: The correct approach involves:
   - Iterating through the sequence while keeping track of the previous optimal move.
   - For each position, if the current move does not beat the previous optimal move, increment the change count and set the current move to the one that beats the previous move.
   - Ensure that no two consecutive moves are the same by adjusting the current move if necessary.

### Solution Code
```cpp
#include <iostream>
#include <string>

using namespace std;

char win(char x) {
    if (x == 'R') return 'P';
    if (x == 'S') return 'R';
    if (x == 'P') return 'S';
    return 'X'; // placeholder for invalid input
}

int main() {
    int n;
    string s;
    cin >> n >> s;
    
    if (n == 0) {
        cout << 0 << endl;
        return 0;
    }
    
    int changes = 0;
    char prev = win(s[0]);
    
    for (int i = 1; i < n; ++i) {
        char current = s[i];
        char optimal = win(prev);
        
        if (current != optimal) {
            changes++;
            prev = optimal;
        } else {
            prev = current;
        }
    }
    
    cout << changes << endl;
    return 0;
}
```

### Explanation
1. **Function `win(char x)`**: This helper function returns the move that beats the input move `x` (e.g., 'P' beats 'R').
2. **Initial Check**: The code checks if the input sequence is empty and handles it immediately.
3. **Initial Setup**: The first move is processed by setting `prev` to the move that would beat the first element in the sequence.
4. **Iteration through the Sequence**: For each subsequent move:
   - The optimal move to beat the previous move (`prev`) is determined.
   - If the current move does not match the optimal move, a change is recorded, and `prev` is updated to the optimal move.
   - If the current move is already optimal, `prev` is simply updated to the current move.
5. **Output**: The total number of changes required to make the sequence optimal is printed.

This approach ensures that the sequence is adjusted with the minimal number of changes, adhering to the constraints and ensuring correctness for all test cases.
