### Analysis

The logic of the code appears to be trying to count two different scenarios:
1. `total_1`: The number of moves where the current move beats the previous move.
2. `total_2`: The number of moves where the previous move beats the current move.

The code then takes the maximum of `total_1` and `total_2`.

### Incorrect Code Behavior

The logic in the provided code for updating `total_1` and `total_2` is flawed. Specifically, it incorrectly increments `total_1` and `total_2` inside the for-loop, without considering the whole sequence of moves. The variables should be incremented only when a turn is considered complete.

Here's how we can correct the code to properly count the number of turns where one player wins:

```cpp
#include <iostream>
#include <unordered_map>
using namespace std;
#define endl "\n"
#define MOD 1000000007

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(0);

    unordered_map<char, char> beats = {{'R', 'P'}, {'P', 'S'}, {'S', 'R'}};
    int n;
    cin >> n;
    string s;
    cin >> s;

    int total_1 = 0; // Total number of winning moves for the first player
    int total_2 = 0; // Total number of winning moves for the second player
    char current_player = s[0]; // The player who starts the game

    for (int i = 1; i < n; ++i) {
        if (beats[s[i]] == current_player) {
            // Current player wins, so switch the player
            current_player = s[i];
            // Increment total_2 because the second player wins this turn
            total_2++;
        } else {
            // First player wins this turn
            total_1++;
        }
    }

    // Output the maximum of total_1 and total_2
    cout << max(total_1, total_2) << endl;

    return 0;
}
```

If the current move is a winning move, it increments `total_2` (because the second player wins) and switches the current player. If the current move is not a winning move, it increments `total_1` (because the first player wins). Finally, it prints the maximum of `total_1` and `total_2`.
