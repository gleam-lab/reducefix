To fix this code, we need to correctly calculate the maximum number of wins a player can achieve by strategically choosing their moves against the given sequence of moves. The problem involves determining the optimal sequence of moves that beats the opponent's moves, changing moves only when necessary to avoid losing or to maximize wins.

### Approaches
1. **Initial Analysis**: The original code attempts to compute two different totals (`total_1` and `total_2`) by tracking the last move and incrementing the count when a new move is needed. However, it fails to correctly account for all scenarios where changing the move could lead to more wins.
2. **Optimal Strategy**: The optimal strategy involves choosing a move that beats the current opponent's move and continues with it until the opponent's move changes to one that would beat the player's current move. At that point, the player should switch to a move that beats the new opponent's move.
3. **Dynamic Tracking**: We need to dynamically track the current move and count the number of times we switch moves to ensure we always beat or at least tie with the opponent's move, maximizing the number of wins.

### Solution Code
```cpp
#include <bits/stdc++.h>
using namespace std;
#define endl "\n"
#define MOD 1000000007

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(0);

    map<char, char> beats = {{'R', 'P'}, {'P', 'S'}, {'S', 'R'}};
    int n;
    cin >> n;
    string s;
    cin >> s;
    if (n == 0) {
        cout << 0 << endl;
        return 0;
    }
    int total = 1;
    char current = beats[s[0]];
    for (int i = 1; i < n; ) {
        if (s[i] == s[i-1]) {
            i++;
            continue;
        }
        if (beats[s[i]] != current) {
            current = beats[s[i]];
            total++;
        }
        i++;
    }
    cout << total << endl;
    return 0;
}
```

### Explanation
1. **Initialization**: We start by defining the map `beats` which indicates which move beats which (Rock beats Scissors, Paper beats Rock, Scissors beats Paper).
2. **Input Handling**: The input consists of the number of moves `n` and the sequence of moves `s`.
3. **Edge Case Handling**: If there are no moves (`n == 0`), the output is `0`.
4. **Main Logic**: 
   - Initialize the total wins to 1 because the first move is always a win.
   - The current move is set to the move that beats the first opponent's move.
   - For each subsequent move in the sequence, if the opponent's current move is the same as the previous, skip it.
   - If the opponent's current move changes and the current player's move no longer beats it, switch to the move that beats the opponent's current move and increment the total wins.
5. **Output**: The total number of wins is printed, which is the maximum achievable by optimally switching moves.

This approach ensures that the player maximizes their wins by only switching moves when necessary to beat the opponent's current move, thus efficiently handling all possible sequences of moves.
