The problem involves determining the maximum number of wins in a Rock-Paper-Scissors game sequence where the player can choose their move based on the opponent's previous move. The initial code had logical flaws in how it counted the wins. 

### Approaches
1. **Understanding the Game Rules**: In Rock-Paper-Scissors, Rock (R) beats Scissors (S), Scissors (S) beat Paper (P), and Paper (P) beats Rock (R). The player wants to maximize wins by choosing the move that beats the opponent's current move.
2. **Tracking Wins**: The player can choose their move based on the opponent's move. We need to count the number of times the player's move beats the opponent's move.
3. **Handling Transitions**: The player can switch their strategy based on the opponent's moves, but we need to ensure that the counting logic correctly tracks the maximum possible wins.

### Solution Code
```cpp
#include <bits/stdc++.h>
using namespace std;
#define endl "\n"
#define MOD 1000000007

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(0);

    map<char, char> beats = {{'R', 'P'}, {'P', 'S'}, {'S', 'R'}};
    int n;
    cin >> n;
    string s;
    cin >> s;
    
    if (n == 0) {
        cout << 0 << endl;
        return 0;
    }

    int total = 1;
    char last_choice = beats[s[0]];
    
    for (int i = 1; i < n; i++) {
        if (s[i] != s[i-1]) {
            last_choice = beats[s[i]];
            total++;
        }
    }
    
    cout << total << endl;

    return 0;
}
```

### Explanation
1. **Initialization**: The `beats` map defines which move beats which: Rock (R) is beaten by Paper (P), Paper (P) by Scissors (S), and Scissors (S) by Rock (R).
2. **Input Handling**: The code reads the number of moves `n` and the sequence of opponent's moves `s`.
3. **Edge Case Handling**: If there are no moves (`n == 0`), the output is 0.
4. **Counting Wins**: The player starts by choosing the move that beats the first opponent's move. For each subsequent move, if the opponent changes their move, the player chooses the move that beats the new opponent's move, and the win count is incremented.
5. **Output**: The total number of wins is printed.

This approach ensures that the player maximizes wins by always choosing the correct move in response to the opponent's current move, and efficiently counts these wins in a single pass through the sequence.
