   - `total_2` starts by copying the first move.
2. The logic inside the loops incorrectly updates `last_move` and increments the count, leading to inaccurate results.

### Correct Approach
1. The player should switch their move only when the current opponent's move can be beaten by a different move than the one they are currently using.
2. We should track the current move of the player and count how many times they need to switch to beat or tie with the opponent's moves.

### Fixed Code
```cpp
#include <bits/stdc++.h>
using namespace std;
#define endl "\n"
#define MOD 1000000007

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(0);

    map<char, char> beats = {{'R', 'P'}, {'P', 'S'}, {'S', 'R'}};
    int n;
    cin >> n;
    string s;
    cin >> s;

    if (n == 0) {
        cout << 0 << endl;
        return 0;
    }

    int switches = 0;
    char current = s[0]; // Start by copying the first move

    for (int i = 1; i < n; ++i) {
        if (beats[s[i]] == current) {
            // Need to switch to beat the current move
            current = beats[s[i]];
            switches++;
        } else if (s[i] != current) {
            // Current move doesn't beat or tie, so switch to beat if advantageous
            // But to maximize switches, we can switch even if it's not necessary
            // So we switch to the move that beats the current opponent's move
            current = beats[s[i]];
            switches++;
        }
    }

    cout << switches << endl;

    return 0;
}
```

### Explanation
1. **Initialization**: The `beats` map defines which move beats which (e.g., 'P' beats 'R').
2. **Input Handling**: Reads the number of moves `n` and the sequence of moves `s`.
3. **Edge Case Handling**: If `n` is 0, output 0 immediately.
4. **Switch Counting**: 
   - The player starts by copying the opponent's first move.
   - For each subsequent move, if the opponent's current move can be beaten by a different move than the player's current move, the player switches to that move and increments the switch count.
   - If the current move ties with the opponent's move, the player may switch to maximize the count (though this is not necessary for correctness, it helps in maximizing the switches).
5. **Output**: The total number of switches is printed.

This approach ensures that the player switches moves optimally to either beat or tie with the opponent's moves, maximizing the number of switches while maintaining correctness.
