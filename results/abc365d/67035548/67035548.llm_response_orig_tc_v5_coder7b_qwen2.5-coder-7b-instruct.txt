To solve this problem, we need to determine the maximum number of consecutive wins that can be achieved by either playing optimally as "P" (paper), "R" (rock), or "S" (scissors). The key observation here is that we can always force a win by choosing the right move against the opponent's previous move.

Here's the corrected and optimized C++ code:

```cpp
#include <iostream>
#include <string>
using namespace std;

int main() {
    int n;
    string s;
    cin >> n >> s;

    // Initialize variables to keep track of the maximum wins
    int maxWins = 0;
    int currentWins = 0;

    // Iterate through the sequence of moves
    for (int i = 0; i < n; ++i) {
        char prevMove = (i == 0 ? 'A' : s[i - 1]);
        char currentMove = s[i];

        // Check if the current move beats the previous move
        if ((currentMove == 'P' && prevMove == 'S') ||
            (currentMove == 'R' && prevMove == 'P') ||
            (currentMove == 'S' && prevMove == 'R')) {
            currentWins++;
        } else {
            currentWins = 1;
        }

        // Update the maximum wins found so far
        maxWins = max(maxWins, currentWins);
    }

    cout << maxWins << endl;
    return 0;
}
```

### Explanation:
1. **Initialization**: We initialize `maxWins` to keep track of the maximum number of consecutive wins and `currentWins` to count the current streak of wins.
2. **Iteration**: We iterate through each move in the sequence.
3. **Determine Previous Move**: For the first move, we assume the previous move was something neutral ('A'). For subsequent moves, we take the previous move from the sequence.
4. **Check for Win**: We check if the current move beats the previous move based on the rules of Rock-Paper-Scissors. If it does, we increment the `currentWins` counter. Otherwise, we reset it to 1.
5. **Update Maximum Wins**: After updating `currentWins`, we update `maxWins` if `currentWins` is greater than `maxWins`.
6. **Output**: Finally, we output the maximum number of consecutive wins.

This approach ensures that we efficiently find the maximum number of consecutive wins with a time complexity of O(n), which is optimal for this problem.
