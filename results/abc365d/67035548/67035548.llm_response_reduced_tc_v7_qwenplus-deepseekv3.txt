The problem involves determining the maximum number of wins achievable by optimally choosing moves (Rock, Paper, Scissors) against a sequence of opponent moves. The initial approach attempted to count wins by alternating between the winning move and the opponent's move, but it missed some critical scenarios where consecutive wins could be achieved by choosing the same winning move multiple times.

### Approaches
1. **Problem Analysis**: The task is to find the maximum number of wins by selecting moves such that each selected move beats the corresponding opponent's move in the sequence. The key observation is that consecutive opponent moves can be beaten by the same winning move, allowing for multiple consecutive wins without needing to alternate moves.
2. **Algorithm Selection**: We can use a greedy approach where we iterate through the sequence of opponent moves and count the number of times we can switch to a winning move that beats the current opponent's move, without switching unnecessarily.
3. **Complexity Analysis**: The solution involves a single pass through the sequence of moves, making the time complexity O(N), which is efficient for the problem constraints.

### Solution Code

```cpp
#include <iostream>
#include <string>

using namespace std;

char getWinMove(char opponent) {
    if (opponent == 'R') return 'P';
    if (opponent == 'P') return 'S';
    if (opponent == 'S') return 'R';
    return ' '; // handle unexpected input
}

int main() {
    int n;
    string s;
    cin >> n >> s;
    
    if (n == 0) {
        cout << 0 << endl;
        return 0;
    }
    
    int maxWins = 0;
    char currentMove = getWinMove(s[0]);
    maxWins = 1;
    
    for (int i = 1; i < n; ++i) {
        if (s[i] == s[i-1]) {
            // Can continue with the same winning move
            maxWins++;
        } else {
            // Need to switch to the new winning move
            currentMove = getWinMove(s[i]);
            maxWins++;
        }
    }
    
    cout << maxWins << endl;
    return 0;
}
```

### Explanation
1. **Input Handling**: The code reads the number of moves `n` and the sequence of opponent's moves `s`.
2. **Edge Case Handling**: If there are no moves (`n == 0`), the output is `0` since no wins are possible.
3. **Initial Setup**: The first move's winning move is determined, and `maxWins` is initialized to `1` since the first move will always result in a win.
4. **Greedy Iteration**: For each subsequent move in the sequence:
   - If the current opponent's move is the same as the previous one, the same winning move continues to be effective, thus incrementing `maxWins`.
   - If the current opponent's move is different, the winning move is switched to the one that beats the current opponent's move, again incrementing `maxWins`.
5. **Output**: The total number of wins (`maxWins`) is printed.

This solution efficiently counts the maximum possible wins by leveraging the greedy approach, ensuring optimal performance and correctness for all test cases.
