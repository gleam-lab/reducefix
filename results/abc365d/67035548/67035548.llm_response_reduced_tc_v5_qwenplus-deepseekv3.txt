The problem involves determining the maximum number of wins achievable by optimally choosing the moves to counter the opponent's sequence of moves (Rock, Paper, Scissors). The initial approach tried to count the wins by alternating the moves, but it failed to account for sequences where consecutive optimal moves could be the same.

### Approaches
1. **Understanding the Game**: In Rock-Paper-Scissors, each move beats another. To maximize wins, we need to choose the move that beats the opponent's current move. If consecutive opponent moves are the same, we can use the same winning move consecutively without alternating.
2. **Optimal Strategy**: The optimal strategy involves selecting the move that beats the current opponent's move. If the next opponent's move is the same as the current one, we can continue using the same winning move. Otherwise, we switch to the new winning move.
3. **Counting Wins**: We traverse the string of opponent moves and count the number of times we need to switch our move to beat the opponent's move. Each switch indicates a new win.

### Solution Code
```cpp
#include <iostream>
#include <string>
using namespace std;

char win(char x) {
    if (x == 'R') return 'P';
    if (x == 'S') return 'R';
    if (x == 'P') return 'S';
    return 'X'; // dummy return
}

int main() {
    int n;
    string s;
    cin >> n >> s;
    
    if (n == 0) {
        cout << 0 << endl;
        return 0;
    }
    
    int ans = 1;
    char current = win(s[0]);
    
    for (int i = 1; i < n; ++i) {
        if (s[i] != s[i-1]) {
            char next = win(s[i]);
            if (next != current) {
                ans++;
                current = next;
            }
        }
    }
    
    cout << ans << endl;
    return 0;
}
```

### Explanation
1. **Initialization**: The code starts by reading the number of moves `n` and the sequence of moves `s`.
2. **Edge Case Handling**: If there are no moves (`n == 0`), the result is `0`.
3. **Initial Win Calculation**: The first move's winning counterpart is determined using the `win` function.
4. **Traversal and Counting**: The code traverses the sequence of moves from the second move onwards. For each move, if it differs from the previous move, the corresponding winning move is calculated. If this winning move is different from the current one, the count of wins (`ans`) is incremented, and the current move is updated.
5. **Output**: The total count of wins is printed.

This approach ensures that the solution optimally counts the maximum number of wins by minimizing unnecessary switches between winning moves, thus improving efficiency and correctness.
