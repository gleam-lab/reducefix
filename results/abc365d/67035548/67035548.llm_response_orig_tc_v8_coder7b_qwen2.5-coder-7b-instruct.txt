To solve this problem, we need to determine the maximum number of consecutive wins that can be achieved given a sequence of moves in Rock-Paper-Scissors (RPS). The key insight here is to count the longest sequences of consecutive wins and losses.

Here's the corrected and optimized C++ code:

```cpp
#include <iostream>
#include <string>
#include <vector>
#include <algorithm>

using namespace std;

int main() {
    int n;
    string s;
    cin >> n >> s;

    int maxWins = 0;
    int currentWins = 0;
    int maxLosses = 0;
    int currentLosses = 0;

    char prev = 'A';

    for (int i = 0; i < n; ++i) {
        char move = s[i];

        if (move == 'X') {
            break;
        }

        if (prev != move) {
            if (move == 'R' && prev == 'P') {
                currentWins = 1;
                currentLosses = 0;
            } else if (move == 'P' && prev == 'S') {
                currentWins = 1;
                currentLosses = 0;
            } else if (move == 'S' && prev == 'R') {
                currentWins = 1;
                currentLosses = 0;
            } else {
                currentWins = 0;
                currentLosses = 1;
            }
        } else {
            if (move == 'R' && prev == 'P') {
                currentWins++;
                currentLosses = 0;
            } else if (move == 'P' && prev == 'S') {
                currentWins++;
                currentLosses = 0;
            } else if (move == 'S' && prev == 'R') {
                currentWins++;
                currentLosses = 0;
            } else {
                currentWins = 0;
                currentLosses++;
            }
        }

        maxWins = max(maxWins, currentWins);
        maxLosses = max(maxLosses, currentLosses);

        prev = move;
    }

    cout << max(maxWins, maxLosses) << endl;

    return 0;
}
```

### Explanation:
1. **Initialization**: We initialize variables to keep track of the maximum number of wins (`maxWins`), the current number of consecutive wins (`currentWins`), the maximum number of losses (`maxLosses`), and the current number of consecutive losses (`currentLosses`).
2. **Loop through the sequence**: For each move in the sequence:
   - If the move is 'X', we break out of the loop as it indicates the end of the sequence.
   - If the current move is different from the previous move, we reset the current counters based on the type of transition.
   - If the current move is the same as the previous move, we increment the appropriate counter.
3. **Update maximum values**: After updating the current counters, we update the maximum values if the current counters are higher.
4. **Output the result**: Finally, we output the maximum of `maxWins` and `maxLosses`.

This approach ensures that we correctly identify the longest sequences of consecutive wins and losses, handling all edge cases and constraints efficiently.
