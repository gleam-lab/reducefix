The problem involves determining the maximum number of wins in a game where each move beats another move (Rock beats Scissors, Scissors beat Paper, Paper beats Rock). The incorrect solution attempts to count wins by choosing a move that beats the current move and avoiding consecutive same moves, but it fails to account for optimal strategies that maximize wins.

### Analysis
1. **Problem Understanding**: The game involves a sequence of moves (R, P, S). For each move in the sequence, we can choose a move that beats it (R -> P, P -> S, S -> R). The goal is to maximize the number of wins without choosing the same move consecutively.
2. **Incorrect Solution**: The provided solution iterates through the sequence, choosing the move that beats the current move and increments the win count if the move is different from the previous one. However, this approach is greedy and not optimal, as it doesn't consider future moves to maximize wins.
3. **Optimal Strategy**: To maximize wins, we need to choose moves that not only beat the current move but also ensure that the next move is different. This can be approached using dynamic programming to consider all possible choices at each step while avoiding consecutive same moves.

### Solution Code
```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(NULL);

    int N;
    string S;
    cin >> N >> S;

    if (N == 0) {
        cout << 0 << endl;
        return 0;
    }

    vector<vector<int>> dp(N, vector<int>(3, 0));

    char first_move = S[0];
    if (first_move == 'R') {
        dp[0][1] = 1; // P beats R
    } else if (first_move == 'P') {
        dp[0][2] = 1; // S beats P
    } else if (first_move == 'S') {
        dp[0][0] = 1; // R beats S
    }

    for (int i = 1; i < N; ++i) {
        char current = S[i];
        int beat_r = (current == 'S') ? 1 : 0;
        int beat_p = (current == 'R') ? 1 : 0;
        int beat_s = (current == 'P') ? 1 : 0;

        dp[i][0] = max(dp[i-1][1], dp[i-1][2]) + beat_r;
        dp[i][1] = max(dp[i-1][0], dp[i-1][2]) + beat_p;
        dp[i][2] = max(dp[i-1][0], dp[i-1][1]) + beat_s;
    }

    cout << max({dp[N-1][0], dp[N-1][1], dp[N-1][2]}) << endl;

    return 0;
}
```

### Explanation
1. **Dynamic Programming Setup**: We use a DP table `dp` where `dp[i][j]` represents the maximum wins up to the `i-th` move when choosing move `j` (0 for R, 1 for P, 2 for S).
2. **Initialization**: For the first move, we initialize `dp[0][j]` based on whether the chosen move beats the opponent's first move.
3. **DP Transition**: For each subsequent move, the value of `dp[i][j]` is computed by taking the maximum wins from the previous move's non-j choices and adding a win if the current move `j` beats the opponent's move.
4. **Result**: The result is the maximum value in the last row of the DP table, representing the maximum wins achievable by optimally choosing moves without consecutive repeats.

This approach ensures that we explore all possible optimal paths while avoiding consecutive same moves, thus maximizing the number of wins efficiently.
