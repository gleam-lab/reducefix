4. If `winmove` is equal to `prev`, it just updates `prev` to `c`.

- If `winmove` is equal to `prev`, the condition for increasing `wins` should not occur. However, the current logic would incorrectly increase `wins` if the current move and the winning move are the same as the previous move.
- The assignment of `prev` to `c` if `winmove != prev` should be outside the if condition to ensure `prev` is always set to the last move regardless of whether a win was counted.
