To fix this code, we need to correctly count the number of winning moves in the sequence, ensuring that consecutive wins are not counted if they are the same move. The original code incorrectly reset the previous move when a win wasn't possible, leading to an undercount of wins. 

### Approaches
The correct approach involves:
1. Iterating through each character in the string.
2. For each character, determine the winning move (Rock beats Scissors, Scissors beat Paper, Paper beats Rock).
3. Count a win if the winning move is different from the previous winning move. If it's the same, skip counting to avoid consecutive same moves.
4. Update the previous move to the current winning move whenever a win is counted.

### Solution Code
```cpp
#include <bits/stdc++.h>
using namespace std;

int main(){
    ios_base::sync_with_stdio(false);
    cin.tie(NULL);

    int N;
    string S;
    cin >> N;
    cin >> S;
    int wins = 0;
    char prev_winmove = '\0';
    for (char c : S){
        char winmove;
        if (c == 'R'){
            winmove = 'P';
        } else if (c == 'P'){
            winmove = 'S';
        } else if (c == 'S'){
            winmove = 'R';
        }
        if (winmove != prev_winmove){
            wins++;
            prev_winmove = winmove;
        }
    }   
    cout << wins;
}
```

### Explanation
1. **Initialization**: The code initializes `wins` to 0 and `prev_winmove` to a null character to keep track of the previous winning move.
2. **Iteration through the string**: For each character in the string, the corresponding winning move is determined:
   - 'R' (Rock) is beaten by 'P' (Paper).
   - 'P' (Paper) is beaten by 'S' (Scissors).
   - 'S' (Scissors) is beaten by 'R' (Rock).
3. **Counting wins**: If the current winning move is different from the previous winning move, the win count is incremented, and `prev_winmove` is updated to the current winning move. This ensures that consecutive same moves are not counted multiple times.
4. **Output**: The total number of wins is printed.

This approach ensures that all test cases are handled correctly, including edge cases, by efficiently tracking and counting distinct winning moves without overcounting consecutive same moves.
