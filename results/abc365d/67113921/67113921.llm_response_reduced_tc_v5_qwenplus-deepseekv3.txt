To fix this code, we need to correctly count the number of times Alice can win by choosing the optimal move against Bob's sequence of moves. The issue in the original code is that it doesn't properly account for consecutive moves that could be won by the same choice. The original code resets the previous move to 'c' when the same winning move is encountered again, which is incorrect.

### Approaches
1. **Understanding Winning Moves**: The winning moves are straightforward:
   - If Bob plays 'R', Alice should play 'P' to win.
   - If Bob plays 'P', Alice should play 'S' to win.
   - If Bob plays 'S', Alice should play 'R' to win.
2. **Counting Wins**: We need to count the number of times Alice can win by selecting the optimal move. However, if consecutive moves by Bob can be won by the same move by Alice, Alice can reuse her previous move without switching, as long as it beats Bob's current move. The original code incorrectly switched moves when the same winning move was encountered, leading to undercounting wins.

### Solution Code
```cpp
#include <bits/stdc++.h>
using namespace std;

int main(){
    ios_base::sync_with_stdio(false);
    cin.tie(NULL);

    int N;
    string S;
    cin >> N;
    cin >> S;
    int wins = 0;
    char prev_win_move = '\0';
    for (char c : S){
        char current_win_move;
        if (c == 'R'){
            current_win_move = 'P';
        } else if (c == 'P'){
            current_win_move = 'S';
        } else if (c == 'S'){
            current_win_move = 'R';
        }
        if (current_win_move != prev_win_move){
            wins++;
            prev_win_move = current_win_move;
        }
    }   
    cout << wins;
    return 0;
}
```

### Explanation
- **Initialization**: We read the input values `N` (number of moves) and `S` (Bob's sequence of moves).
- **Winning Moves Logic**: For each move `c` in `S`, we determine the winning move `current_win_move` that Alice should play to win against `c`.
- **Counting Wins**: If Alice's current winning move is different from her previous winning move (`prev_win_move`), we increment the win count and update `prev_win_move` to the current one. This ensures that Alice doesn't switch moves unnecessarily, allowing her to maximize her wins by reusing the same move when it continues to beat Bob's moves.
- **Output**: The total number of wins is printed.

This approach ensures that Alice maximizes her wins by only switching her move when necessary, thus efficiently counting all possible wins against Bob's sequence of moves.
