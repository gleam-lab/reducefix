To fix this code, we need to address the algorithm used to calculate the number of times the player can change their move to win against the opponent's previous move according to the Rock-Paper-Scissors rules. The original code attempted to keep track of the last move and increment the answer only when the current move could beat the last move, but it failed to account for optimal move changes that maximize the number of wins.

### Approaches
1. **Optimal Strategy**: The player should choose their move to beat the opponent's previous move whenever possible. If the opponent's previous move is the same as the current move, the player should choose a move that wins against it.
2. **Tracking Opponent's Moves**: We need to keep track of the opponent's previous move to determine the player's optimal current move.
3. **Initialization**: The first move doesn't have a previous move to compare, so we start the count from the second move.

### Solution Code
```cpp
#include <bits/stdc++.h>
using namespace std;
using ll = long long;

int main() {
    int N;
    string S;
    cin >> N >> S;
    if (N == 0) {
        cout << 0 << endl;
        return 0;
    }
    char last_opponent = 'N'; // Initialize to an invalid move
    int ans = 0;
    for (char c : S) {
        if (last_opponent == 'N') {
            // First move, choose any (but count if it beats the previous which doesn't exist)
            last_opponent = c;
        } else {
            if (c == 'P') {
                if (last_opponent == 'R') {
                    ans++;
                    last_opponent = 'S'; // Choose scissors to beat rock
                } else if (last_opponent == 'S') {
                    // Do nothing, opponent's last move was scissors, paper loses
                    last_opponent = 'P';
                } else if (last_opponent == 'P') {
                    // Choose scissors to beat paper
                    ans++;
                    last_opponent = 'S';
                }
            } else if (c == 'R') {
                if (last_opponent == 'S') {
                    ans++;
                    last_opponent = 'R'; // Choose rock to beat scissors
                } else if (last_opponent == 'P') {
                    // Do nothing, opponent's last move was paper, rock loses
                    last_opponent = 'R';
                } else if (last_opponent == 'R') {
                    // Choose paper to beat rock
                    ans++;
                    last_opponent = 'P';
                }
            } else if (c == 'S') {
                if (last_opponent == 'P') {
                    ans++;
                    last_opponent = 'S'; // Choose scissors to beat paper
                } else if (last_opponent == 'R') {
                    // Do nothing, opponent's last move was rock, scissors lose
                    last_opponent = 'S';
                } else if (last_opponent == 'S') {
                    // Choose rock to beat scissors
                    ans++;
                    last_opponent = 'R';
                }
            }
        }
    }
    cout << ans << endl;
    return 0;
}
```

### Explanation
1. **Initialization**: The variable `last_opponent` is initialized to 'N' (an invalid move) to indicate no previous move.
2. **First Move Handling**: The first move is processed without any comparison since there's no previous move to compare against.
3. **Subsequent Moves**: For each subsequent move, the code checks the opponent's previous move (`last_opponent`):
   - If the current move can beat the previous move, the answer is incremented, and the player's move is set to the one that wins.
   - If the current move loses or ties with the previous move, the player's move is updated to the current move without incrementing the answer.
4. **Output**: The total count of wins (`ans`) is printed.

This approach ensures that the player optimally changes their move to maximize the number of wins against the opponent's previous move, adhering to the Rock-Paper-Scissors rules. The algorithm efficiently processes each move in O(N) time, making it suitable for large inputs.
