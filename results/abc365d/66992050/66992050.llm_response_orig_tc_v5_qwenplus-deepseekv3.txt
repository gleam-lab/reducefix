The problem involves processing a string of characters 'P', 'R', and 'S' to maximize the number of wins in a Rock-Paper-Scissors game. The initial approach seems to increment the answer based on the previous move, but it doesn't optimally choose the best move to maximize wins.

### Analysis
1. **Problem Understanding**: The goal is to maximize the number of wins by choosing the best possible move at each step. The rules are:
   - 'P' beats 'S'
   - 'R' beats 'P'
   - 'S' beats 'R'

2. **Initial Approach Flaw**: The initial code increments the answer only if the current move beats the previous move, but it doesn't consider the best move to maximize future wins. It also doesn't handle cases where multiple optimal moves could lead to more wins.

3. **Optimal Strategy**: The optimal strategy is to choose the move that beats the most frequent upcoming moves. This requires analyzing the sequence and possibly using dynamic programming or greedy approaches to maximize wins.

4. **Complexity**: The solution should ideally run in O(N) time, as the input can be very large (up to 200000 characters).

### Solution Code
```cpp
#include <bits/stdc++.h>
using namespace std;
using ll = long long;

int main() {
    int N;
    string S;
    cin >> N >> S;
    
    int ans = 0;
    char last = '\0';
    
    for (int i = 0; i < N; ) {
        char current = S[i];
        char best_move;
        if (current == 'P') best_move = 'S';
        else if (current == 'R') best_move = 'P';
        else best_move = 'R';
        
        int j = i;
        while (j < N && S[j] == current) {
            j++;
        }
        
        if (best_move != last) {
            ans += j - i;
            last = best_move;
        } else {
            last = '\0';
        }
        
        i = j;
    }
    
    cout << ans << endl;
    return 0;
}
```

### Explanation
1. **Initialization**: The code reads the input values N (length of the string) and S (the string itself).

2. **Loop Through String**: The loop processes the string in segments where consecutive characters are the same. For each segment, it determines the best move to beat all characters in that segment.

3. **Best Move Calculation**: For each segment of the same character, the best move is determined:
   - If the segment is 'P', the best move is 'S' (since 'S' beats 'P').
   - If the segment is 'R', the best move is 'P' (since 'P' beats 'R').
   - If the segment is 'S', the best move is 'R' (since 'R' beats 'S').

4. **Update Answer**: If the best move for the current segment is different from the last move used, the answer is incremented by the length of the segment, and the last move is updated. If the best move is the same as the last move, the last move is reset to avoid conflicts.

5. **Output**: The total number of wins is printed.

This approach ensures that we maximize the number of wins by optimally choosing moves for each segment of consecutive same characters, avoiding unnecessary conflicts where the same move would be used consecutively. The complexity is O(N), making it efficient for large inputs.
