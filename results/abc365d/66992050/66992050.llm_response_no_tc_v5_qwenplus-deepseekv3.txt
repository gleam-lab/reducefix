The problem seems to involve counting the number of times a character in the string `S` beats the previous character according to the Rock-Paper-Scissors rules (R beats S, S beats P, P beats R). The original code attempts to count these wins but has some logical errors in how it tracks the previous character and updates the count.

### Issues in the Original Code
1. The initial `past` is set to 'N' which isn't part of the game and can cause incorrect comparisons.
2. The logic for updating `past` and counting wins is flawed because it doesn't properly account for consecutive same characters.
3. The count increases even when the current character doesn't beat the previous one in some cases.

### Fixed Code
```cpp
#include <bits/stdc++.h>
using namespace std;
using ll = long long;

int main() {
    int N;
    string S;
    cin >> N >> S;
    
    if (N == 0) {
        cout << 0;
        return 0;
    }
    
    char past = S[0];
    int ans = 0;
    
    for (int i = 1; i < N; ++i) {
        char current = S[i];
        if (current == 'P' && past == 'R') {
            ans++;
            past = current;  // Only update past if current wins
        } else if (current == 'R' && past == 'S') {
            ans++;
            past = current;
        } else if (current == 'S' && past == 'P') {
            ans++;
            past = current;
        } else {
            past = current;  // Update past even if no win
        }
    }
    
    cout << ans;
    return 0;
}
```

### Explanation
1. **Initialization**: The `past` variable is initialized to the first character of the string `S` to ensure valid comparisons right from the start.
2. **Loop Through String**: The loop starts from the second character (index 1) and compares each character with `past`.
3. **Win Conditions**: The code checks if the current character beats `past` according to Rock-Paper-Scissors rules:
   - 'P' beats 'R'
   - 'R' beats 'S'
   - 'S' beats 'P'
4. **Update Past**: If the current character wins, `past` is updated to the current character. Otherwise, `past` is still updated to the current character to continue comparisons.
5. **Edge Case Handling**: If the string is empty (N=0), the output is 0.

This approach ensures that we correctly count each winning move while properly handling consecutive characters and initial conditions.
