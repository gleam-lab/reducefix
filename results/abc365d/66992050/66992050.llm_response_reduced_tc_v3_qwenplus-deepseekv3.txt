To fix this code, we need to correctly simulate the game rules where each character in the string represents a move (P for Paper, R for Rock, S for Scissors). The goal is to count the number of times the current move beats the previous move based on the game rules:
- Rock (R) beats Scissors (S)
- Scissors (S) beats Paper (P)
- Paper (P) beats Rock (R)

The issue in the original code is that it resets the `past` variable incorrectly, leading to missed counts when consecutive moves should be considered. 

### Approaches
1. **Initialization**: Start by reading the input values, the integer `N` and the string `S`.
2. **Tracking Previous Move**: Use a variable to keep track of the previous move. Initialize it to a dummy value (e.g., 'N' for None) to handle the first move correctly.
3. **Iterate Through Moves**: For each move in the string `S`, check if it beats the previous move according to the game rules. If it does, increment the answer and update the previous move to the current one. If it doesn't, set the previous move to the current one without incrementing the answer.
4. **Edge Cases**: Handle cases where the string is empty or has only one move.

### Solution Code
```cpp
#include <bits/stdc++.h>
using namespace std;
using ll = long long;

int main() {
    int N;
    string S;
    cin >> N >> S;
    char past = 'N'; // 'N' represents no previous move
    int ans = 0;
    for (char c : S) {
        if (past == 'N') {
            ans++;
            past = c;
        } else {
            if ((past == 'R' && c == 'P') || 
                (past == 'P' && c == 'S') || 
                (past == 'S' && c == 'R')) {
                ans++;
                past = c;
            } else {
                past = c;
            }
        }
    }
    cout << ans << endl;
    return 0;
}
```

### Explanation
1. **Initialization**: The variables `N` and `S` are read from the input. `past` is initialized to 'N' to denote no previous move initially.
2. **Loop Through Moves**: For each character `c` in string `S`:
   - If there is no previous move (`past == 'N'`), increment the answer and set `past` to the current move `c`.
   - Otherwise, check if the current move `c` beats the previous move `past`:
     - If `past` is 'R' (Rock) and `c` is 'P' (Paper), Paper beats Rock.
     - If `past` is 'P' (Paper) and `c` is 'S' (Scissors), Scissors beat Paper.
     - If `past` is 'S' (Scissors) and `c` is 'R' (Rock), Rock beats Scissors.
   - If any of these conditions are met, increment the answer and update `past` to `c`. Otherwise, simply update `past` to `c` without incrementing the answer.
3. **Output the Result**: After processing all moves, the total count of winning moves is printed.

This approach ensures that each move is correctly compared with the previous move, and the answer is incremented only when the current move beats the previous one, adhering to the game rules. The solution efficiently handles all edge cases and operates in O(N) time complexity, which is optimal for the given problem constraints.
