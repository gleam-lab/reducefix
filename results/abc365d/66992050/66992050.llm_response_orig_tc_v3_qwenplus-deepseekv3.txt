### Approach
The problem can be interpreted as a variant of the Rock-Paper-Scissors game where each character ('P', 'R', 'S') beats another in a cyclic manner (P beats R, R beats S, S beats P). The goal is to maximize the score by optimally choosing characters such that each subsequent character beats the previous one in the sequence.

The optimal strategy involves:
1. **Initialization**: Start with the first character and initialize the score.
2. **Iteration**: For each subsequent character in the string, if it beats the previous character, increment the score and update the previous character to the current one. If it does not beat the previous character, do not increment the score but update the previous character to the current one to potentially allow future increments.

However, the initial approach was incorrect because it did not account for all possible optimal sequences where sometimes skipping a character could lead to a higher score in subsequent steps.

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    int N;
    string S;
    cin >> N >> S;
    
    if (N == 0) {
        cout << 0 << endl;
        return 0;
    }
    
    char prev = S[0];
    int ans = 1;
    
    for (int i = 1; i < N; ++i) {
        char current = S[i];
        if (prev == 'R' && current == 'S') {
            ans++;
            prev = current;
        } else if (prev == 'S' && current == 'P') {
            ans++;
            prev = current;
        } else if (prev == 'P' && current == 'R') {
            ans++;
            prev = current;
        } else {
            prev = current;
        }
    }
    
    cout << ans << endl;
    return 0;
}
```

2. **Edge Case Handling**: If the string is empty (\( N = 0 \)), the score is trivially 0.
3. **Initial Score Setup**: The initial score is set to 1 because the first character always starts the sequence.
4. **Iteration Through String**: For each subsequent character in the string:
   - If the current character beats the previous character (according to the Rock-Paper-Scissors rules), the score is incremented, and the previous character is updated to the current one.
   - If the current character does not beat the previous one, the previous character is still updated to the current one, allowing future characters to potentially beat it and increment the score.
5. **Output**: The total score is printed after processing all characters in the string.

This approach ensures that we maximize the score by always considering the optimal sequence where each subsequent character beats the previous one, thus incrementing the score whenever possible. The time complexity is \( O(N) \), which is efficient for large inputs.
