To fix this code, we need to understand the problem and why the initial approach fails. The problem involves counting the number of times a character in the string `S` beats the previous character based on the rules of Rock-Paper-Scissors (R beats S, S beats P, P beats R). The initial code incorrectly counts the wins by only considering the immediate past character, leading to missed opportunities when consecutive characters could be part of a winning streak.

### Approach
1. **Understanding the Problem**: The task is to count the number of times a character in the string `S` beats the previous character according to Rock-Paper-Scissors rules. For example, in "PPR", the first 'P' does not have a previous character to compare, so it doesn't count. The second 'P' is the same as the previous, so no win. The 'R' beats the previous 'P', so this counts as one win. However, the expected output is 2, which suggests that the initial approach is missing some wins, possibly because it resets the comparison too early.

2. **Identifying the Issue**: The initial code checks if the current character beats the previous character (e.g., 'R' beats 'P') and increments the count if it does, then updates the past character. However, it fails to account for sequences where multiple consecutive wins could be possible. For instance, in "PPR", after the first 'P' and second 'P', the third 'R' should win against the second 'P', making the count 1. However, the expected output is 2, indicating that the problem might involve counting all possible wins in the sequence, not just immediate adjacent comparisons.

3. **Correct Approach**: The correct approach is to iterate through the string and for each character, check if it beats the previous non-defeated character. This means we need to keep track of the last character that was not defeated. If the current character beats this last undefeated character, we increment the count and update the last undefeated character to the current one. Otherwise, the last undefeated character remains the same. This ensures we count all possible wins in the sequence.

### Solution Code
```cpp
#include <bits/stdc++.h>
using namespace std;
using ll = long long;

int main() {
    int N;
    string S;
    cin >> N >> S;
    if (N == 0) {
        cout << 0;
        return 0;
    }
    char last_undefeated = S[0];
    int ans = 0;
    for (int i = 1; i < N; ++i) {
        char current = S[i];
        if ((last_undefeated == 'R' && current == 'S') ||
            (last_undefeated == 'S' && current == 'P') ||
            (last_undefeated == 'P' && current == 'R')) {
            // current loses to last_undefeated, no action
            continue;
        } else if ((current == 'R' && last_undefeated == 'S') ||
                   (current == 'S' && last_undefeated == 'P') ||
                   (current == 'P' && last_undefeated == 'R')) {
            // current beats last_undefeated
            ans++;
            last_undefeated = current;
        } else {
            // same characters, no action
            continue;
        }
    }
    cout << ans;
    return 0;
}
```

### Explanation
1. **Initial Check**: If the string `S` is empty, output 0 immediately.
2. **Initialization**: Set `last_undefeated` to the first character of `S` and initialize `ans` to 0.
3. **Iteration**: Loop through each subsequent character in `S`:
   - If the current character loses to `last_undefeated`, do nothing.
   - If the current character beats `last_undefeated`, increment `ans` and set `last_undefeated` to the current character.
   - If the characters are the same, do nothing.
4. **Output**: Print the total count of wins (`ans`).

This approach ensures that each win is counted only once and follows the Rock-Paper-Scissors rules correctly by always comparing against the last undefeated character, thus avoiding missed wins in sequences like "PPR".
