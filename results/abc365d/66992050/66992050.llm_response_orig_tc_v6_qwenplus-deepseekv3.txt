To fix this code, we need to correctly simulate the game where each character in the string represents an action (R, P, or S) and the player earns points based on the previous action. The key insight is that the player can only earn a point if they perform an action that beats the previous action of their opponent, and they can't repeat the same action consecutively unless they lose a point.

### Approaches
1. **Initialization**: Start with the initial state where the previous action is 'N' (no previous action).
2. **Iterate through the string**: For each character in the string, determine if it can earn a point based on the previous action.
3. **Update the previous action**: If the current action beats the previous action of the opponent, increment the score and set the previous action to the current one. If not, set the previous action to the current action without incrementing the score.
4. **Handling edge cases**: Ensure that the first action always earns a point if it is valid.

### Solution Code
```cpp
#include <bits/stdc++.h>
using namespace std;
using ll = long long;

int main() {
    int N;
    string S;
    cin >> N >> S;
    char last = 'N'; // Indicates no previous move
    int ans = 0;
    for (char c : S) {
        if (c == 'R') {
            if (last != 'P') {
                ans++;
                last = 'P';
            } else {
                last = 'R';
            }
        } else if (c == 'P') {
            if (last != 'S') {
                ans++;
                last = 'S';
            } else {
                last = 'P';
            }
        } else if (c == 'S') {
            if (last != 'R') {
                ans++;
                last = 'R';
            } else {
                last = 'S';
            }
        }
    }
    cout << ans << endl;
    return 0;
}
```

### Explanation
1. **Initialization**: The variable `last` is initialized to 'N', indicating no previous move has been made.
2. **Loop through each character**: For each character in the string `S`, check if it can beat the previous move stored in `last`.
   - If the current move is 'R' (Rock), it beats 'S' (Scissors). If the previous move wasn't 'P' (Paper), the player earns a point, and `last` is updated to 'P' (the move that would beat the current move). Otherwise, `last` is set to 'R'.
   - Similarly, for 'P' (Paper), it beats 'R' (Rock). If the previous move wasn't 'S' (Scissors), the player earns a point, and `last` is updated to 'S'. Otherwise, `last` is set to 'P'.
   - For 'S' (Scissors), it beats 'P' (Paper). If the previous move wasn't 'R' (Rock), the player earns a point, and `last` is updated to 'R'. Otherwise, `last` is set to 'S'.
3. **Output the result**: After processing all characters, the total points earned are printed.

This approach ensures that the player maximizes their score by always choosing the optimal move based on the previous move of their opponent, without repeating the same move consecutively unless necessary. The algorithm efficiently processes each character in O(N) time, which is optimal for the given constraints.
