### Analysis
The problem involves determining the number of successful attacks based on a sequence of moves (P, R, S) where:
- 'P' (Paper) beats 'S' (Scissors) if the previous move was not 'S'
- 'R' (Rock) beats 'P' (Paper) if the previous move was not 'P'
- 'S' (Scissors) beats 'R' (Rock) if the previous move was not 'R'

The initial code incorrectly increments the answer only when the current move beats the previous move in a specific way, but it does not account for the sequence of moves properly. For example, in the input "PPR":
- The first 'P' does not increment the answer because there is no previous move (or it's 'N').
- The second 'P' does not increment the answer because the previous move was 'P', and the condition `!(past=='S')` is true, but the code sets `past = 'P'` if the condition fails, which is incorrect.
- The 'R' increments the answer because the previous move was 'P', and `!(past=='P')` is false, so it correctly increments and sets `past = 'P'`.

This results in an answer of 1, but the expected output is 2 because the second 'P' should also be considered a successful attack if the previous move was not 'S' (which it wasn't, since it was 'P').

### Solution Approach
To correctly solve the problem, we need to:
1. Initialize the previous move to a dummy value (like 'N') to handle the first move.
2. For each move in the sequence:
   - If the current move can beat the previous move according to the game rules, increment the answer and update the previous move to the beaten move.
   - Otherwise, set the previous move to the current move without incrementing the answer.
3. The key insight is that a move can only be a successful attack if it beats the previous move, and the previous move was not the one that the current move beats in the next round.

}
