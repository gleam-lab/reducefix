### Analysis
The original code attempts to calculate the number of wins by choosing the optimal move against each opponent's move in a Rock-Paper-Scissors game. However, the logic for counting consecutive wins is flawed. Specifically, the code doesn't correctly handle sequences where the same winning move can be used consecutively without being counted as a new win. 

### Key Issues
1. **Wrong Win Counting Logic**: The code only increments the win count when the current winning move is different from the previous one. This misses cases where the same winning move can be used consecutively against the same opponent's move.
2. **Incorrect Reset of `prev`**: When the winning move is the same as the previous, the code sets `prev` to the opponent's move, which doesn't logically follow the game's rules.

### Solution
To fix this, we should:
1. Count every instance where the chosen move beats the opponent's move, regardless of whether it's the same as the previous winning move.
2. Remove the unnecessary check and reset of `prev` since it's not needed for correct win counting.

### Fixed Code
```cpp
#include <bits/stdc++.h>
using namespace std;

int main(){
    ios_base::sync_with_stdio(false);
    cin.tie(NULL);

    int N;
    string S;
    cin >> N;
    cin >> S;
    int wins = 0;
    for (char c : S){
        if (c == 'R'){
            wins += 'P' > 'R'; // P beats R
        } else if (c == 'P'){
            wins += 'S' > 'P'; // S beats P
        } else if (c == 'S'){
            wins += 'R' > 'S'; // R beats S
        }
    }   
    cout << wins;
}
```

### Explanation
- **Input Handling**: The code reads the number of rounds `N` and the sequence of opponent's moves `S`.
- **Win Counting**: For each move in `S`, the code checks the opponent's move and increments the win count if the corresponding winning move is chosen (e.g., 'P' for 'R', 'S' for 'P', 'R' for 'S').
- **Output**: The total number of wins is printed.

This approach ensures that every possible win is counted correctly, regardless of consecutive moves, and efficiently computes the result in linear time.
