### Solution Approach
1. **Track the previous winning move**: For each move in the sequence, determine the winning move (e.g., 'R' -> 'P', 'P' -> 'S', 'S' -> 'R').
2. **Count wins when the winning move changes**: If the current winning move is different from the previous one, increment the win count and update the previous move. If it's the same, do not increment but still keep track of the last winning move.

### Final Corrected Code
```cpp
#include <bits/stdc++.h>
using namespace std;

int main(){
    ios_base::sync_with_stdio(false);
    cin.tie(NULL);

    int N;
    string S;
    cin >> N;
    cin >> S;
    int wins = 0;
    char prevWinMove = '\0'; // Initialize to an invalid move

    for (char c : S){
        char currentWinMove;
        if (c == 'R'){
            currentWinMove = 'P';
        } else if (c == 'P'){
            currentWinMove = 'S';
        } else if (c == 'S'){
            currentWinMove = 'R';
        }

        if (currentWinMove != prevWinMove){
            wins++;
            prevWinMove = currentWinMove;
        }
    }   
    cout << wins;
    return 0;
}
```

### Explanation
- **Initialization**: `prevWinMove` is initialized to an invalid character (`'\0'`) to ensure the first move is always counted.
- **Determining Winning Moves**: For each character in the string `S`, the corresponding winning move is determined (e.g., 'R' is beaten by 'P').
- **Counting Wins**: If the current winning move is different from the previous one, the win count is incremented, and `prevWinMove` is updated to the current winning move. This ensures that each change in the winning strategy is counted as a new win, even if the same move appears consecutively in the input string.

This approach ensures that all possible wins are counted correctly, handling both consecutive same moves and varying moves appropriately.
