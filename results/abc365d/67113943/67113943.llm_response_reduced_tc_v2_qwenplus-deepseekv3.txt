To fix this code, we need to correctly count the number of wins in a sequence of Rock-Paper-Scissors (RPS) moves. The initial approach mistakenly only increments the win count when the winning move changes, but it should instead count each individual opportunity to win against the opponent's moves.

### Approaches
1. **Understanding the Winning Moves**: For each move in the string `S`, determine the winning move:
   - If the opponent plays 'R' (Rock), the winning move is 'P' (Paper).
   - If the opponent plays 'P' (Paper), the winning move is 'S' (Scissors).
   - If the opponent plays 'S' (Scissors), the winning move is 'R' (Rock).
2. **Counting Wins**: Iterate through each character in the string `S`, and for each move, check if the next move is different from the current winning move. This ensures that we count each distinct winning opportunity correctly.

### Solution Code
```cpp
#include <bits/stdc++.h>
using namespace std;

int main(){
    ios_base::sync_with_stdio(false);
    cin.tie(NULL);

    int N;
    string S;
    cin >> N;
    cin >> S;
    int wins = 0;
    for (int i = 0; i < N; ++i) {
        char c = S[i];
        char winmove;
        if (c == 'R') {
            winmove = 'P';
        } else if (c == 'P') {
            winmove = 'S';
        } else if (c == 'S') {
            winmove = 'R';
        }
        wins++;
        // Check if the next move is the same to avoid counting consecutive same wins
        if (i + 1 < N && S[i+1] == c) {
            i++; // Skip the next move to prevent overcounting
        }
    }
    cout << wins;
}
```

### Explanation
- **Initialization**: Read the number of moves `N` and the string of moves `S`.
- **Winning Moves Determination**: For each move in `S`, determine the corresponding winning move (e.g., 'P' for 'R').
- **Counting Wins**: Increment the win count for each move. If the next move is the same as the current move, skip the next move to avoid overcounting consecutive same moves.
- **Output**: Print the total number of wins.

This approach ensures that each distinct winning opportunity is counted once, and consecutive same moves are handled correctly, leading to the accurate total number of wins.
