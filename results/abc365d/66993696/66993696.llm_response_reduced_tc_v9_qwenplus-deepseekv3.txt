The problem involves optimizing the choice of responses to a sequence of moves in a Rock-Paper-Scissors game to maximize the number of wins. The initial approach tried to greedily choose winning moves but failed to handle cases where consecutive moves could be optimized better.

### Approach
1. **Understanding the Problem**: The goal is to choose responses to each move in the sequence such that the number of wins is maximized. The responses must not have the same choice in consecutive positions unless necessary.
2. **Initial Strategy**: The initial strategy was to choose the move that beats the opponent's move (e.g., 'P' becomes 'S', 'R' becomes 'P', 'S' becomes 'R'). Then, it adjusted consecutive same moves by reverting to the opponent's move, reducing the win count.
3. **Identifying the Issue**: The initial approach did not consider the optimal way to break consecutive same moves. For instance, in the failing case "PPR", the initial approach would choose "SSP", then adjust the consecutive 'S's to "PSP", resulting in only one win (against 'P' with 'S'), but the optimal solution is "SSR" which wins twice (against both 'P's with 'S' and 'R' with 'P').
4. **Optimal Strategy**: The optimal strategy involves:
   - First, choose the move that beats each opponent's move.
   - Then, for any consecutive same moves, decide whether to change the current move or the previous move to maximize wins. This involves checking the surrounding moves to ensure the change does not create new consecutive same moves.

### Solution Code
```cpp
#include<bits/stdc++.h>
using namespace std;

char s[200011];
char c[200011];

int main() {
    int n;
    scanf("%d", &n);
    scanf("%s", s + 1);

    // Initialize c to the moves that beat s
    for (int i = 1; i <= n; ++i) {
        if (s[i] == 'P') c[i] = 'S';
        else if (s[i] == 'R') c[i] = 'P';
        else c[i] = 'R';
    }

    int sum = n;
    // First pass: handle consecutive same moves from left to right
    for (int i = 2; i <= n; ++i) {
        if (c[i] == c[i - 1]) {
            // Change c[i] to something that doesn't match c[i-1] and still tries to win or tie
            // We can try to set c[i] to s[i], which might not win, but avoids consecutive same moves
            c[i] = s[i];
            sum--;
        }
    }

    int ans = sum;
    sum = n;
    // Reset c
    for (int i = 1; i <= n; ++i) {
        if (s[i] == 'P') c[i] = 'S';
        else if (s[i] == 'R') c[i] = 'P';
        else c[i] = 'R';
    }

    // Second pass: handle consecutive same moves from right to left
    for (int i = n - 1; i >= 1; --i) {
        if (c[i] == c[i + 1]) {
            c[i] = s[i];
            sum--;
        }
    }

    ans = max(ans, sum);
    printf("%d\n", ans);

    return 0;
}
```

### Explanation
1. **Initialization**: The array `c` is initialized with moves that beat each corresponding move in `s`.
2. **First Pass (Left to Right)**: Iterate from the second element to the end. For any consecutive same moves in `c`, change the current element to the opponent's move (which doesn't win but avoids consecutive same moves), decrementing the win count.
3. **Second Pass (Right to Left)**: Reset `c` to the initial winning moves. Then iterate from the end to the beginning, changing any consecutive same moves similarly to the first pass but in the opposite direction.
4. **Result Calculation**: The maximum win count from both passes is taken as the result, ensuring the optimal solution by considering both directions of adjustment for consecutive moves.

This approach ensures that we maximize the number of wins while avoiding consecutive same moves by considering adjustments in both directions, thus handling cases like "PPR" optimally.
