1. First creating a counter-move sequence
2. Then trying to avoid consecutive same moves by reverting to original moves
3. But it doesn't properly account for all possible optimal sequences

Here's the corrected version that properly calculates the maximum number of winning moves while ensuring no two consecutive moves are the same:

```cpp
#include<bits/stdc++.h>
using namespace std;

char s[200011];
char c[200011];

int main() {
    int n;
    scanf("%d", &n);
    scanf("%s", s+1);
    
    // Initialize counter moves
    for(int i = 1; i <= n; i++) {
        if(s[i] == 'P') c[i] = 'S';
        else if(s[i] == 'R') c[i] = 'P';
        else c[i] = 'R';
    }
    
    int max_wins = 0;
    // Try two different approaches to maximize wins
    for(int approach = 0; approach < 2; approach++) {
        int wins = 0;
        char prev = '\0';
        
        for(int i = 1; i <= n; i++) {
            // For even positions in first approach, odd in second
            if((approach == 0 && i % 2 == 1) || (approach == 1 && i % 2 == 0)) {
                if(c[i] != prev) {
                    wins++;
                    prev = c[i];
                } else {
                    // Can't use this move, use original instead
                    prev = s[i];
                }
            } else {
                // Use original move
                prev = s[i];
            }
        }
        max_wins = max(max_wins, wins);
    }
    
    printf("%d", max_wins);
    return 0;
}
```

Key improvements:
1. The code now tries two different approaches (alternating starting positions) to maximize wins
2. It properly tracks the previous move to ensure no two consecutive moves are the same
3. It only counts a move as a win if it's different from the previous move and beats the opponent's move
4. It falls back to the original move if the winning move would create a consecutive duplicate
5. The solution is more systematic and handles edge cases better

This approach will correctly calculate the maximum number of winning moves while maintaining the constraint of no two consecutive same moves.
