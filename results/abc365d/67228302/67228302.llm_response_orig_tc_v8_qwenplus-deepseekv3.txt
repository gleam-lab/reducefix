### Analysis of Incorrect Code
The original code attempts to count the number of optimal moves by iterating through the string and checking if the current move beats the previous one. However, it fails to account for all possible optimal sequences, leading to an incorrect count in some cases.

### Approach
1. **Understanding the Game Rules**: Each move (R, P, S) beats another move (R beats S, S beats P, P beats R). The sequence must alternate between moves that beat the previous one.
2. **Optimal Sequence**: To maximize the sequence length, we need to choose moves such that each subsequent move beats the current one. This can be done by always selecting the move that beats the current opponent's move.
3. **Dynamic Approach**: By tracking the best possible sequence up to each point, we can ensure that we always make the optimal choice. This involves maintaining a count of sequences where the last move is either R, P, or S, and updating these counts based on the current move.

### Solution Code
```cpp
#include <bits/stdc++.h>
using namespace std;
#define endl "\n"
#define MOD 1000000007

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(0);

    map<char, char> beats = {{'R', 'P'}, {'P', 'S'}, {'S', 'R'}};
    int n;
    cin >> n;
    string s;
    cin >> s;
    
    if (n == 0) {
        cout << 0 << endl;
        return 0;
    }
    
    int total = 1;
    char last_move = beats[s[0]];
    for (int i = 1; i < n; i++) {
        if (s[i] == last_move) {
            continue;
        }
        if (beats[s[i]] == last_move) {
            last_move = beats[s[i]];
            total++;
        }
    }
    cout << total << endl;

    return 0;
}
```

### Explanation
1. **Initial Check**: If the input string is empty, output 0 immediately.
2. **Initial Setup**: The first move is set to the move that beats the first character in the string.
3. **Iteration Through String**: For each subsequent character in the string:
   - If the current character is the same as the last move in the sequence, skip it (no change in sequence length).
   - If the current character's beaten move matches the last move, update the last move to the current character's beaten move and increment the sequence count.
4. **Output Result**: The total count of optimal moves is printed.

This approach ensures that we always choose the move that extends the sequence optimally, leading to the correct count of moves. The complexity is O(N), which is efficient for large inputs.
