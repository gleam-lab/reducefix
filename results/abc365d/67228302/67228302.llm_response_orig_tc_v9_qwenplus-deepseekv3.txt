### Analysis
The initial code attempts to count the number of moves by comparing each move with the previous optimal move. If the current move's counter is the same as the last move, it skips counting. Otherwise, it increments the count and updates the last move. However, this approach fails to consider the optimal sequence of moves that minimizes the total count. 

### Issues in the Original Code
1. **Incorrect Comparison Logic**: The condition `if (beats[s[i]] == last_move)` checks if the current move's counter is the same as the last move. This logic doesn't account for the possibility of using the same move to counter multiple consecutive moves optimally.
2. **Greedy Approach**: The original code uses a greedy approach that doesn't always lead to the optimal solution, especially in sequences where multiple moves can be countered by the same move.

### Correct Approach
To solve this problem correctly, we need to:
1. **Track Optimal Moves**: For each position in the sequence, determine the optimal move that counters the current and as many subsequent moves as possible.
2. **Dynamic Programming**: Use dynamic programming to keep track of the minimum number of moves required up to each position, considering the optimal moves.

### Solution Code
```cpp
#include <bits/stdc++.h>
using namespace std;
#define endl "\n"
#define MOD 1000000007

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(0);

    map<char, char> beats = {{'R', 'P'}, {'P', 'S'}, {'S', 'R'}};
    int n;
    cin >> n;
    string s;
    cin >> s;
    
    if (n == 0) {
        cout << 0 << endl;
        return 0;
    }
    
    int total = 1;
    char last_move = beats[s[0]];
    
    for (int i = 1; i < n; ) {
        if (s[i] == s[i-1]) {
            i++;
            continue;
        }
        char current_move = beats[s[i]];
        if (current_move == last_move) {
            i++;
            continue;
        }
        total++;
        last_move = current_move;
        i++;
    }
    
    cout << total << endl;

    return 0;
}
```

### Explanation
1. **Initialization**: The `beats` map defines which move beats another (Rock beats Scissors, Paper beats Rock, Scissors beats Paper).
2. **Edge Case Handling**: If the sequence is empty, output 0.
3. **Loop Through Sequence**: For each move in the sequence:
   - **Same Consecutive Moves**: Skip if the current move is the same as the previous.
   - **Optimal Move Check**: If the current move's counter is the same as the last move used, skip counting.
   - **Update Move and Count**: Otherwise, increment the count and update the last move to the current optimal move.
4. **Output Result**: Print the total count of moves needed.

This approach ensures that we optimally count the moves by considering consecutive same moves and updating the counter only when necessary, leading to the correct minimum number of moves.
