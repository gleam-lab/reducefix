The problem appears to involve determining the number of optimal moves in a game where each move beats the previous one based on certain rules (Rock, Paper, Scissors). The incorrect solution attempts to count the optimal moves by checking if the current move beats the previous one, but it fails to account for all possible optimal sequences.

### Analysis
1. **Problem Understanding**: The game involves a sequence of moves (R, P, S) where each move beats another (R beats S, S beats P, P beats R). The goal is to determine the maximum number of optimal moves where each subsequent move beats the previous one.
2. **Incorrect Logic**: The original code increments the count only if the current move's counter is different from the last move. This logic misses cases where consecutive moves could be part of a longer optimal sequence.
3. **Correct Approach**: We need to traverse the sequence and count the number of times the optimal move changes. This involves tracking the current optimal move and updating it whenever a better move is found.

### Solution Code
```cpp
#include <bits/stdc++.h>
using namespace std;
#define endl "\n"
#define MOD 1000000007

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(0);

    map<char, char> beats = {{'R', 'P'}, {'P', 'S'}, {'S', 'R'}};
    int n;
    cin >> n;
    string s;
    cin >> s;
    if (n == 0) {
        cout << 0 << endl;
        return 0;
    }
    int total = 1;
    char last_move = beats[s[0]];
    for (int i = 1; i < n; i++) {
        if (beats[s[i]] != last_move) {
            last_move = beats[s[i]];
            total++;
        }
    }
    cout << total << endl;

    return 0;
}
```

### Explanation
1. **Initialization**: The `beats` map defines which move beats another (e.g., 'P' beats 'R').
2. **Input Handling**: The code reads the number of moves `n` and the sequence of moves `s`.
3. **Edge Case Handling**: If `n` is 0, it directly outputs 0 since there are no moves.
4. **Traversal and Counting**: The code initializes `total` to 1, representing the first move. It then iterates through the sequence, updating the `last_move` to the move that beats the current one if it differs from the previous `last_move`, incrementing `total` each time this happens.
5. **Output**: The total count of optimal moves is printed.

This approach ensures that we correctly count the number of optimal moves by always choosing the move that beats the current one, thus maximizing the sequence length. The complexity is O(N), which is efficient for the problem constraints.
