2. **Iteration**: For each subsequent character, if the current move also beats the next character, no change is needed. Otherwise, change to the move that beats the next character and increment the total count.

The key insight is that we should only change our move when the current move does not beat the next character in the sequence. This ensures we minimize the number of changes while still beating all moves.

```cpp
#include <bits/stdc++.h>
using namespace std;
#define endl "\n"
#define MOD 1000000007

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(0);

    map<char, char> beats = {{'R', 'P'}, {'P', 'S'}, {'S', 'R'}};
    int n;
    cin >> n;
    string s;
    cin >> s;
    if (n == 0) {
        cout << 0 << endl;
        return 0;
    }
    int total = 1;
    char current_move = beats[s[0]];
    for (int i = 1; i < n; i++) {
        if (beats[s[i]] != current_move) {
            current_move = beats[s[i]];
            total++;
        }
    }
    cout << total << endl;
    return 0;
}
```

1. **Initialization**: The `beats` map defines which move beats each possible move (Rock, Paper, Scissors).
2. **Edge Case Handling**: If the input string is empty, the output is 0.
3. **Initial Move**: The first move is set to beat the first character in the string.
4. **Iteration**: For each subsequent character in the string, if the current move does not beat the next character, we change our move to the one that does and increment the total count. This ensures that we always have the optimal move to beat as many consecutive characters as possible without unnecessary changes.
5. **Output**: The total number of changes is printed.

This approach efficiently counts the minimal number of changes required to beat all moves in the sequence, ensuring correctness for all test cases.
