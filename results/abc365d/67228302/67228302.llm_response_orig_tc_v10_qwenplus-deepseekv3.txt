### Approaches
1. **Problem Analysis**: The task is to find the minimal number of moves required such that each subsequent move beats the previous one. The predefined rules are Rock (R) beats Scissors (S), Paper (P) beats Rock (R), and Scissors (S) beats Paper (P).
2. **Initial Approach**: The initial code tries to traverse the sequence of moves and count the changes needed when the next move doesn't beat the current one. However, it fails to consider all optimal sequences where changing a move might not be necessary if the subsequent moves can form a longer sequence without changes.
3. **Optimal Approach**: We need to ensure that each move in our sequence beats the previous one. The optimal way is to iterate through the sequence and for each position, determine the best next move that beats the current one, thus minimizing the total number of changes.

### Solution Code
```cpp
#include <bits/stdc++.h>
using namespace std;
#define endl "\n"
#define MOD 1000000007

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(0);

    map<char, char> beats = {{'R', 'P'}, {'P', 'S'}, {'S', 'R'}};
    int n;
    cin >> n;
    string s;
    cin >> s;
    if (n == 0) {
        cout << 0 << endl;
        return 0;
    }
    int total = 1;
    char last_move = beats[s[0]];
    for (int i = 1; i < n; i++) {
        if (s[i] != last_move) {
            total++;
            last_move = beats[s[i]];
        }
    }
    cout << total << endl;

    return 0;
}
```

### Explanation
1. **Initial Check**: If the input string length `n` is zero, the output is zero since there are no moves to consider.
2. **Initialization**: The `beats` map defines which move beats which (e.g., 'P' beats 'R'). The `total` is initialized to 1 because the first move is always counted, and `last_move` is set to the move that beats the first move in the sequence.
3. **Iteration**: For each subsequent move in the sequence, if the current move does not match the `last_move` (the move that beats the previous one), we increment the `total` and update `last_move` to the move that beats the current one.
4. **Output**: The total count is printed, which represents the minimal number of moves needed to ensure each subsequent move beats the previous one.

This approach ensures that we consider each move in the sequence and adjust our count based on the optimal move that beats the previous one, leading to the correct minimal count of moves.
