To fix this code, we need to properly count the number of times the optimal strategy changes to beat the opponent's moves in a Rock-Paper-Scissors game. The initial approach was incorrect because it didn't account for consecutive moves that could be optimally countered with the same move. 

### Approach
1. **Initialization**: We start by mapping each move to the move that beats it (Rock beats Scissors, Scissors beats Paper, Paper beats Rock).
2. **Iteration through the string**: For each move in the string, we determine the optimal counter move. If the current optimal move is the same as the previous one, we do not increment the count. Otherwise, we update the last move and increment the count.
3. **Edge Cases**: Handle cases where the string is empty or has only one move.

### Solution Code
```cpp
#include <bits/stdc++.h>
using namespace std;
#define endl "\n"
#define MOD 1000000007

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(0);

    map<char, char> beats = {{'R', 'P'}, {'P', 'S'}, {'S', 'R'}};
    int n;
    cin >> n;
    string s;
    cin >> s;
    if (n == 0) {
        cout << 0 << endl;
        return 0;
    }
    int total = 1;
    char last_move = beats[s[0]];
    for (int i = 1; i < n; i++) {
        if (beats[s[i]] != last_move) {
            last_move = beats[s[i]];
            total++;
        }
    }
    cout << total << endl;

    return 0;
}
```

### Explanation
1. **Initialization**: The `beats` map is used to determine the optimal counter for each move. For example, 'R' (Rock) is beaten by 'P' (Paper), so the map entry is `{'R', 'P'}`.
2. **Reading Input**: The input consists of the number of moves `n` and the sequence of moves `s`.
3. **Edge Handling**: If there are no moves (`n == 0`), the output is `0` immediately.
4. **Counting Optimal Moves**: The initial count is set to `1` because the first move requires at least one counter. For each subsequent move, if the optimal counter is different from the last used counter, we update the last counter and increment the total. This ensures we count each distinct optimal counter once, even if it appears multiple times consecutively in the sequence.
5. **Output**: The total number of distinct optimal counters is printed.

This approach efficiently counts the optimal strategy changes by ensuring each new required counter is counted once, leading to the correct solution.
