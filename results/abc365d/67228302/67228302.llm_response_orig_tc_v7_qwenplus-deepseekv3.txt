The problem involves counting the minimum number of moves required to beat a sequence of Rock-Paper-Scissors moves. The initial solution attempted to count the moves by comparing each move with the previous one, but it missed some critical aspects of the optimal strategy.

### Approach
1. **Understanding the Problem**: The task is to determine the minimum number of moves needed to beat every consecutive move in the sequence. Each move in the sequence must be countered by a move that beats it (R loses to P, P loses to S, S loses to R).
2. **Optimal Strategy**: The optimal strategy involves choosing the best move that can beat as many consecutive moves as possible. For example, if the next few moves are the same, choosing the move that beats them all in one go reduces the total count.
3. **Algorithm Selection**: The algorithm should scan the sequence and group consecutive identical moves. For each group, it should count one move that beats the entire group. This approach ensures the minimum number of moves.

```cpp
#include <bits/stdc++.h>
using namespace std;
#define endl "\n"
#define MOD 1000000007

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(0);

    map<char, char> beats = {{'R', 'P'}, {'P', 'S'}, {'S', 'R'}};
    int n;
    cin >> n;
    string s;
    cin >> s;
    
    if (n == 0) {
        cout << 0 << endl;
        return 0;
    }
    
    int total = 1;
    char current = beats[s[0]];
    
    for (int i = 1; i < n; ) {
        if (s[i] == s[i-1]) {
            i++;
            continue;
        }
        total++;
        current = beats[s[i]];
        i++;
    }
    
    cout << total << endl;

    return 0;
}
```

### Explanation
1. **Initialization**: The `beats` map defines which move beats another (R → P, P → S, S → R).
2. **Edge Handling**: If the input string is empty, the output is 0.
3. **Main Loop**: The loop starts from the second character. If the current character is the same as the previous, it continues without incrementing the count. Otherwise, it increments the count and sets the current move to the one that beats the current character.
4. **Output**: The total count of moves is printed.

This approach ensures that we only count a new move when the sequence changes, thus minimizing the total number of moves required. This solution efficiently handles the problem by leveraging the optimal strategy of grouping consecutive identical moves and counting one move per group.
