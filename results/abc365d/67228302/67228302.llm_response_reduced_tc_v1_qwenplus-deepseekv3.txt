To fix this code, we need to correctly count the number of times a player needs to change their move to win against a sequence of opponent's moves. The original code incorrectly counted changes only when the next move differed from the previous one in a specific way, leading to an incorrect total count.

### Approaches
1. **Initialization**: We start by defining a map that shows which move beats another (Rock beats Scissors, Scissors beats Paper, Paper beats Rock).
2. **Reading Input**: Read the number of moves `n` and the sequence of moves `s`.
3. **Counting Changes**: Iterate through the sequence of moves. For each move, determine the winning move. If the current winning move is different from the previous one, increment the change count.
4. **Edge Cases**: Handle cases where the sequence is empty or has a single move.

### Solution Code
```cpp
#include <bits/stdc++.h>
using namespace std;
#define endl "\n"
#define MOD 1000000007

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(0);

    map<char, char> beats = {{'R', 'P'}, {'P', 'S'}, {'S', 'R'}};
    int n;
    cin >> n;
    string s;
    cin >> s;
    if (n == 0) {
        cout << 0 << endl;
        return 0;
    }
    int total = 1;
    char last_move = beats[s[0]];
    for (int i = 1; i < n; i++) {
        char current_move = beats[s[i]];
        if (current_move != last_move) {
            total++;
            last_move = current_move;
        }
    }
    cout << total << endl;

    return 0;
}
```

### Explanation
1. **Initialization**: The `beats` map is used to find the move that beats the current move in the sequence. For example, 'R' (Rock) is beaten by 'P' (Paper).
2. **Input Handling**: The input is read, including the number of moves `n` and the sequence `s`.
3. **Edge Case Handling**: If `n` is 0, the output is 0 since there are no moves to consider.
4. **Counting Changes**: The loop starts from the second move. For each move, it checks if the current winning move (from the `beats` map) differs from the last move used. If it does, it increments the `total` and updates `last_move` to the current winning move.
5. **Output**: The total number of changes is printed.

This approach ensures that we count each necessary change to consistently win against the sequence of moves, providing the correct result for all test cases.
