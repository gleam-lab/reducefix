The idea is that if the current move beats the previous move, the turn increment continues; otherwise, the turn is reset and starts counting again.

However, there is an error in the code that will cause it to fail for certain sequences of moves. Specifically, the check in the code:
```cpp
if (beats[s[i]] == last_move)
```
will incorrectly increment the `total` if the next move in the sequence beats the previous move, but the previous move was the same as the current move. This is not the intended behavior according to the problem description, because you should count a turn increment whenever the player makes a different move that beats the previous one, not just when it beats the previous move that was not the current one.

    map<char, char> beats = {{'R', 'P'}, {'P', 'S'}, {'S', 'R'}};
    int n;
    cin >> n;
    string s;
    cin >> s;
    int total = 1; // Start with 1 because the first move is always counted
    char last_move = 'N'; // Initialize to an invalid character to ensure the first comparison is valid
    for (int i = 1; i < n; ++i) {
        // Check if the current move beats the previous move
        if (beats[s[i]] == s[i-1]) {
            last_move = s[i]; // Update the last move
            total++; // Increment the total
        }
    }
    cout << total << endl;
    return 0;
}
