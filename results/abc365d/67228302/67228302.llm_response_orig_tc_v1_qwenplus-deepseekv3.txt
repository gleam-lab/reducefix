### Approaches
1. **Problem Analysis**: The problem requires finding the maximum number of non-overlapping segments in a sequence where each segment starts with a move that beats the previous segment's move. For example, if the previous segment ended with 'R' (Rock), the next segment must start with 'P' (Paper), which beats Rock.

2. **Initial Incorrect Approach**: The initial code tried to count segments by checking if the current move beats the last move of the previous segment. However, this approach fails to consider all possible optimal segmentations, leading to a suboptimal count.

3. **Correct Approach**: The correct approach involves:
   - Iterating through the sequence and dynamically determining the best move to start a new segment based on the previous segment's move.
   - Using a greedy algorithm to maximize the number of segments by always choosing the earliest possible move that beats the previous segment's move.

### Solution Code
```cpp
#include <bits/stdc++.h>
using namespace std;
#define endl "\n"
#define MOD 1000000007

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(0);

    map<char, char> beats = {{'R', 'P'}, {'P', 'S'}, {'S', 'R'}};
    int n;
    cin >> n;
    string s;
    cin >> s;
    if (n == 0) {
        cout << 0 << endl;
        return 0;
    }
    int total = 1;
    char last_beat = beats[s[0]];
    for (int i = 1; i < n; i++) {
        if (s[i] == last_beat) {
            total++;
            if (i + 1 < n) {
                last_beat = beats[s[i + 1]];
                i++;
            }
        }
    }
    cout << total << endl;

    return 0;
}
```

### Explanation
1. **Initialization**: The code starts by setting up a map `beats` that defines which move beats another (e.g., 'P' beats 'R'). This helps in determining the next valid move for a new segment.

2. **Input Handling**: The code reads the sequence length `n` and the sequence `s` itself. If the sequence is empty, it immediately returns 0.

3. **Segment Counting**: The algorithm initializes the segment count to 1 (since there's at least one segment) and sets the `last_beat` to the move that beats the first move in the sequence.

4. **Greedy Segmentation**: The algorithm iterates through the sequence, checking if the current move is the one that beats the previous segment's move (`last_beat`). If it is, it increments the segment count and updates `last_beat` to the move that beats the next move in the sequence (to start a new segment). This greedy approach ensures the maximum number of segments by always making the locally optimal choice.

5. **Output**: The total number of segments is printed.

This approach ensures that we maximize the number of segments by always choosing the earliest possible move that forms a valid segment, adhering to the problem constraints and requirements.
