To fix this code, we need to correctly count the number of times the optimal move changes in a sequence of Rock-Paper-Scissors (RPS) moves. The initial approach was flawed because it didn't account for consecutive moves that could be beaten by the same optimal response. The correct approach involves tracking when the optimal move changes based on the current move in the sequence.

### Approaches
1. **Initialization**: Start by mapping each move to its beating move (R to P, P to S, S to R).
2. **First Move Handling**: The first optimal move is always the move that beats the first move in the sequence.
3. **Iterate Through Sequence**: For each subsequent move in the sequence:
   - If the current optimal move (stored as `last_move`) can beat the current move in the sequence, continue without changing the optimal move.
   - If the current optimal move cannot beat the current move, update the optimal move to the one that beats the current move and increment the count of changes.
4. **Edge Cases**: Handle cases where the sequence length is 1 or empty appropriately.

### Solution Code
```cpp
#include <bits/stdc++.h>
using namespace std;
#define endl "\n"
#define MOD 1000000007

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(0);

    map<char, char> beats = {{'R', 'P'}, {'P', 'S'}, {'S', 'R'}};
    int n;
    cin >> n;
    string s;
    cin >> s;
    if (n == 0) {
        cout << 0 << endl;
        return 0;
    }
    int total = 1;
    char last_move = beats[s[0]];
    for (int i = 1; i < n; i++) {
        if (beats[s[i]] != last_move) {
            last_move = beats[s[i]];
            total++;
        }
    }
    cout << total << endl;

    return 0;
}
```

### Explanation
- **Initialization**: The `beats` map is created to know which move beats each of the RPS moves (Rock, Paper, Scissors).
- **First Move Handling**: The first optimal move (`last_move`) is set to the move that beats the first character in the input string.
- **Iterating Through the Sequence**: For each subsequent character in the string:
   - If the move that beats the current character is different from `last_move`, update `last_move` to the new optimal move and increment the change counter (`total`).
   - This ensures that we only count changes when the optimal response changes, not for every move in the sequence.
- **Edge Cases**: The code checks if the input string is empty (though constraints likely ensure `n > 0`) and handles it appropriately.

This approach efficiently counts the number of times the optimal response changes, ensuring correctness for all test cases, including the provided failing case. The complexity is O(N), which is optimal for this problem.
