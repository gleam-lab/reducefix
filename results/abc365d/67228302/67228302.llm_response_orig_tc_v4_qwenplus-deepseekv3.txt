The problem involves determining the minimum number of moves required to win a game where each move beats the previous one in a Rock-Paper-Scissors fashion. The initial code attempted to count the number of times a move needed to change to ensure a win, but it missed some critical details in the logic.

### Approaches
1. **Problem Analysis**: The game involves a sequence of moves (R, P, S). Each move must beat the previous move played by the opponent. The goal is to find the minimum number of changes required in the sequence to ensure that every consecutive move beats the previous one.
2. **Initial Insight**: The initial approach incorrectly assumed that the optimal strategy is to change the move only when the next move in the sequence does not beat the current move. This misses cases where a sequence of moves could be optimized further by changing multiple moves in a strategic way.
3. **Optimal Strategy**: The correct strategy involves ensuring that each move in the sequence beats the previous move in the original sequence. If consecutive moves in the original sequence are the same or lose to the previous move, we need to change them to ensure a win. This can be done by greedily selecting the next move that beats the previous move in the sequence.

### Solution Code
```cpp
#include <bits/stdc++.h>
using namespace std;
#define endl "\n"
#define MOD 1000000007

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(0);

    map<char, char> beats = {{'R', 'P'}, {'P', 'S'}, {'S', 'R'}};
    int n;
    cin >> n;
    string s;
    cin >> s;
    if (n == 0) {
        cout << 0 << endl;
        return 0;
    }
    int total = 1;
    char last_move = beats[s[0]];
    for (int i = 1; i < n; i++) {
        if (beats[s[i]] != last_move) {
            last_move = beats[s[i]];
            total++;
        }
    }
    cout << total << endl;

    return 0;
}
```

### Explanation
1. **Initial Setup**: The code starts by defining a map `beats` that specifies which move beats another (e.g., 'P' beats 'R'). It reads the input values for the number of moves `n` and the sequence of moves `s`.
2. **Edge Case Handling**: If the sequence is empty (`n == 0`), it immediately returns 0 as no moves are needed.
3. **Greedy Strategy**: The algorithm initializes the total moves to 1 (for the first move) and sets the last move to the one that beats the first move in the sequence. It then iterates through the sequence, checking if the current move in the sequence does not beat the last move. If it doesn't, it updates the last move to the one that beats the current move and increments the total count.
4. **Output**: Finally, the total number of changes required is printed.

This approach ensures that each move in the sequence is optimal to beat the previous move, minimizing the total number of changes required. The complexity is O(N), which is efficient for the given constraints.
