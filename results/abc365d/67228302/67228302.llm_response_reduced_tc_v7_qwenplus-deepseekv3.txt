To fix this code, we need to analyze why the initial solution failed for the given input. The problem involves determining the number of times a player needs to change their move to always beat the opponent's current move in a sequence of Rock-Paper-Scissors (RPS) moves. The initial approach incorrectly counts the number of changes by only considering the immediate next move and not the optimal sequence of moves that minimizes changes while always beating the opponent.

### Analysis of the Failing Case
**Input:**
```
3
PPR
```
The sequence is 'P', 'P', 'R'. The optimal moves to beat each are:
- First 'P' is beaten by 'S'.
- Second 'P' is also beaten by 'S' (no change needed).
- 'R' is beaten by 'P'. Since the previous move was 'S', we need to change to 'P', resulting in a total of 2 changes (initial move to 'S' and then to 'P').

The initial code incorrectly outputted 1 because it didn't account for the need to change from 'S' to 'P' when encountering 'R'.

### Solution Approach
1. **Initialization:** Start by determining the move that beats the first element in the sequence.
2. **Tracking Moves:** Iterate through the sequence, and for each subsequent move, check if the current optimal move (the one that beats the opponent's move) is the same as the last move used. If it is, no change is needed. If not, increment the change count and update the last move to the new optimal move.
3. **Edge Cases:** Handle cases where the sequence is empty or has only one move.

### Fixed Code
```cpp
#include <bits/stdc++.h>
using namespace std;
#define endl "\n"
#define MOD 1000000007

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(0);

    map<char, char> beats = {{'R', 'P'}, {'P', 'S'}, {'S', 'R'}};
    int n;
    cin >> n;
    string s;
    cin >> s;
    if (n == 0) {
        cout << 0 << endl;
        return 0;
    }
    int total = 1;
    char last_move = beats[s[0]];
    for (int i = 1; i < n; i++) {
        char current_beat = beats[s[i]];
        if (current_beat != last_move) {
            total++;
            last_move = current_beat;
        }
    }
    cout << total << endl;
    return 0;
}
```

### Explanation
- **Initialization:** The `beats` map defines which move beats each RPS move ('P' beats 'R', 'S' beats 'P', 'R' beats 'S').
- **Edge Handling:** If the input sequence is empty, the output is 0.
- **Iteration:** For each subsequent move in the sequence, the code checks if the optimal move to beat the current opponent's move is different from the last move used. If it is different, the change count is incremented, and the last move is updated to the new optimal move.
- **Output:** The total number of changes is printed, which represents the minimum number of moves needed to always beat the opponent's sequence.

This approach ensures that the solution is efficient (O(N) time complexity) and handles all edge cases correctly.
