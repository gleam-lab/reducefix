To fix this code, we need to correctly count the number of times the optimal move changes in a sequence of Rock-Paper-Scissors (RPS) moves. The initial approach had a logical flaw in how it determined when to increment the total. The correct approach involves checking when the optimal response to the current move differs from the optimal response to the previous move.

### Approaches
1. **Initialization**: Start by defining the beats map which indicates which move beats which (e.g., Rock ('R') is beaten by Paper ('P')).
2. **Initial Move**: The optimal move for the first character in the string is the move that beats it.
3. **Iterate Through the String**: For each subsequent character in the string, check if the optimal move to beat it is different from the previous optimal move. If it is, increment the total and update the last move to the current optimal move.
4. **Edge Cases**: Handle cases where the string is empty or has only one character.

### Solution Code
```cpp
#include <bits/stdc++.h>
using namespace std;
#define endl "\n"
#define MOD 1000000007

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(0);

    map<char, char> beats = {{'R', 'P'}, {'P', 'S'}, {'S', 'R'}};
    int n;
    cin >> n;
    string s;
    cin >> s;
    if (n == 0) {
        cout << 0 << endl;
        return 0;
    }
    int total = 1;
    char last_move = beats[s[0]];
    for (int i = 1; i < n; i++) {
        if (beats[s[i]] != last_move) {
            total += 1;
            last_move = beats[s[i]];
        }
    }
    cout << total << endl;

    return 0;
}
```

### Explanation
- **Initialization**: The `beats` map is initialized to show which move beats which. For example, 'P' beats 'R', 'S' beats 'P', and 'R' beats 'S'.
- **Reading Input**: The number of moves `n` and the string `s` representing the sequence of moves are read.
- **Edge Case Handling**: If `n` is zero, the output is zero since there are no moves to respond to.
- **Initial Move Handling**: The first optimal move is set to the move that beats the first character in the string.
- **Iteration and Counting**: For each subsequent move in the string, if the optimal move to beat it is different from the previous optimal move, the total is incremented and the last move is updated to the current optimal move.
- **Output**: The total count of changes in optimal moves is printed.

This approach ensures that we correctly count every time the optimal response changes, providing the correct result for all test cases.
