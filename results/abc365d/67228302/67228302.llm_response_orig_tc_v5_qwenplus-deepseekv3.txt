To fix this code, we need to correctly count the number of optimal moves in a Rock-Paper-Scissors game where each move should beat the previous move of the opponent. The initial approach had a flaw in how it determined when to switch moves, leading to incorrect counts.

### Approaches
1. **Initialization**: Start by defining what beats each move (Rock beats Scissors, Paper beats Rock, Scissors beats Paper).
2. **Iterate through the sequence**: For each move in the sequence, determine the optimal move that beats it.
3. **Counting moves**: Increment the count only when the current optimal move is different from the previous one. This ensures we switch moves only when necessary to beat the current move, avoiding unnecessary switches which could lead to incorrect counts.

### Solution Code
```cpp
#include <bits/stdc++.h>
using namespace std;
#define endl "\n"
#define MOD 1000000007

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(0);

    map<char, char> beats = {{'R', 'P'}, {'P', 'S'}, {'S', 'R'}};
    int n;
    cin >> n;
    string s;
    cin >> s;
    if (n == 0) {
        cout << 0 << endl;
        return 0;
    }
    int total = 1;
    char last_move = beats[s[0]];
    for (int i = 1; i < n; i++) {
        char current_optimal = beats[s[i]];
        if (current_optimal != last_move) {
            total++;
            last_move = current_optimal;
        }
    }
    cout << total << endl;

    return 0;
}
```

### Explanation
1. **Initialization**: The `beats` map is used to determine the move that beats each possible move (e.g., 'P' beats 'R').
2. **Reading Input**: The input consists of the length of the sequence `n` and the sequence of moves `s`.
3. **Edge Handling**: If the sequence is empty (`n == 0`), the output is immediately 0.
4. **Counting Optimal Moves**: The algorithm starts with the first optimal move to beat the first move in the sequence. For each subsequent move, it checks if the optimal move to beat it is different from the last optimal move used. If so, it increments the count and updates the last move to the current optimal move.
5. **Output**: The total count of optimal moves is printed.

This approach ensures that we only switch moves when necessary, thus optimizing the number of moves and correctly counting them. The complexity is O(N) per test case, which is efficient given the constraints.
