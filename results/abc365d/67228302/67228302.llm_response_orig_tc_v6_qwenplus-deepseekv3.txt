### Analysis
1. **Problem Understanding**: The game is based on Rock-Paper-Scissors rules where each move beats a specific other move (Rock beats Scissors, Scissors beat Paper, Paper beats Rock). The goal is to determine the minimum number of moves needed to ensure a win against each opponent's move in a sequence.
2. **Incorrect Logic**: The original code tried to count the number of times the optimal move changed, but it did not account for sequences where the same move could consecutively beat multiple opponent moves without changing.
3. **Correct Approach**: We need to iterate through the sequence of opponent moves and determine the optimal move that beats the current opponent's move. If the next opponent's move can be beaten by the same move, we continue without incrementing the count. Otherwise, we switch to the move that beats the next opponent's move and increment the count.

### Solution Code
```cpp
#include <bits/stdc++.h>
using namespace std;
#define endl "\n"
#define MOD 1000000007

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(0);

    map<char, char> beats = {{'R', 'P'}, {'P', 'S'}, {'S', 'R'}};
    int n;
    cin >> n;
    string s;
    cin >> s;
    if (n == 0) {
        cout << 0 << endl;
        return 0;
    }
    int total = 1;
    char last_move = beats[s[0]];
    for (int i = 1; i < n; i++) {
        if (s[i] == s[i-1]) {
            continue;
        }
        if (beats[s[i]] != last_move) {
            last_move = beats[s[i]];
            total++;
        }
    }
    cout << total << endl;
    return 0;
}
```

### Explanation
1. **Initialization**: We define a map `beats` that specifies which move beats each other move (e.g., 'P' beats 'R').
2. **Input Handling**: We read the number of moves `n` and the sequence of moves `s`.
3. **Edge Case Handling**: If there are no moves (n == 0), we immediately return 0.
4. **Counting Moves**: We initialize the total count to 1 because the first move always requires a response. We then iterate through the sequence:
   - If the current move is the same as the previous, we skip further checks (no need to change the move).
   - If the optimal move to beat the current opponent's move is different from the last move used, we switch to this new move and increment the count.
5. **Output**: The total count of moves is printed, which represents the minimum number of moves needed to win the game.

This approach ensures that we efficiently determine the optimal sequence of moves with minimal changes, adhering to the game's rules and constraints.
