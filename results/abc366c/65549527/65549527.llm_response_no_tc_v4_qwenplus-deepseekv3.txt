The provided code uses a `set` to keep track of the unique integers in the bag, which correctly handles the count of different integers (query type 3). However, the issue lies in handling multiple occurrences of the same integer when removing balls (query type 2). The current implementation removes the integer from the set entirely, which is incorrect if there are multiple balls with the same integer in the bag. 

To fix this, we need to track the count of each integer in the bag. We can use a `map` (or `unordered_map` for better performance) to keep count of each integer. When a ball is added (query type 1), we increment the count. When a ball is removed (query type 2), we decrement the count. If the count reaches zero, we remove the integer from the set. The set will always contain the integers that have a count greater than zero, so its size will correctly reflect the number of different integers in the bag.

### Fixed Code
```cpp
#include<bits/stdc++.h>
using namespace std;

void solve()
{
    int q;
    cin >> q;
    set<int> st;
    unordered_map<int, int> count;
    for (int i = 0; i < q; i++)
    {
        int a;
        cin >> a;
        if (a == 1)
        {
            int x;
            cin >> x;
            count[x]++;
            if (count[x] == 1)
            {
                st.insert(x);
            }
        }
        else if (a == 2)
        {
            int y;
            cin >> y;
            count[y]--;
            if (count[y] == 0)
            {
                st.erase(y);
            }
        }
        else if (a == 3)
        {
            cout << st.size() << endl;
        }
    }
}
```

### Explanation
1. **Data Structures**: 
   - `unordered_map<int, int> count` keeps track of how many balls of each integer are in the bag.
   - `set<int> st` maintains the unique integers present in the bag (those with count > 0).

2. **Query Handling**:
   - **Type 1 (`1 x`)**: Increment the count of `x`. If the count becomes 1 (meaning it was not in the bag before), add `x` to the set.
   - **Type 2 (`2 x`)**: Decrement the count of `x`. If the count drops to 0, remove `x` from the set.
   - **Type 3 (`3`)**: The size of the set gives the number of different integers in the bag, which is printed.

This approach ensures that the operations are efficient (O(1) average time for map and set operations) and correct for all cases.
