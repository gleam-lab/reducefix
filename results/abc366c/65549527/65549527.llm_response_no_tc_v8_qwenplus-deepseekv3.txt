The problem requires handling three types of queries on a bag of balls, where the balls have integers written on them. The operations include adding a ball, removing a ball, and querying the number of distinct integers in the bag. The initial solution uses a set to track the distinct integers, which is correct in terms of maintaining the count of distinct integers. However, the problem arises when multiple balls with the same integer are added or removed. The set only keeps track of whether an integer is present or not, not the count of how many balls there are with each integer. 

### Analysis of the Initial Code
1. **Addition (`1 x`)**: The code inserts `x` into the set, which correctly notes that `x` is present in the bag. However, if there are multiple balls with the same integer `x`, the set will still only show `x` once, which is correct for counting distinct integers.
2. **Removal (`2 x`)**: The code removes `x` from the set. This is incorrect because if there are multiple balls with `x`, removing one should not remove `x` from the set unless it was the last ball with `x`. The set doesn't track the counts of each integer, leading to incorrect distinct counts after removals.
3. **Query (`3`)**: The code prints the size of the set, which should represent the number of distinct integers. This part is correct if the additions and removals are handled properly.

### Solution
To fix this, we need to maintain both the count of each integer and the set of distinct integers:
1. Use a `map` or `unordered_map` to keep track of the count of each integer.
2. Use a `set` to keep track of the distinct integers present in the bag.
3. When adding a ball (`1 x`), increment the count of `x` in the map. If the count becomes 1, add `x` to the set.
4. When removing a ball (`2 x`), decrement the count of `x` in the map. If the count becomes 0, remove `x` from the set.
5. For the query (`3`), simply output the size of the set.

### Fixed Code
```cpp
#include<bits/stdc++.h>
using namespace std;

void solve() {
    int q;
    cin >> q;
    unordered_map<int, int> count;
    set<int> distinct;
    
    for (int i = 0; i < q; i++) {
        int a;
        cin >> a;
        if (a == 1) {
            int x;
            cin >> x;
            count[x]++;
            if (count[x] == 1) {
                distinct.insert(x);
            }
        } else if (a == 2) {
            int y;
            cin >> y;
            count[y]--;
            if (count[y] == 0) {
                distinct.erase(y);
            }
        } else if (a == 3) {
            cout << distinct.size() << endl;
        }
    }
}

int main() {
#ifndef ONLINE_JUDGE
    freopen("input.txt", "r", stdin);
    freopen("output.txt", "w", stdout);
#endif
    
    int t = 1;
    while (t--) {
        solve();
    }
    return 0;
}
```

### Explanation
- **Data Structures**: We use an `unordered_map` to count occurrences of each integer and a `set` to track which integers are present (i.e., have a count greater than zero).
- **Handling Queries**:
  - For `1 x`, we increment the count of `x`. If the count becomes 1 (meaning it wasn't present before), we add `x` to the set.
  - For `2 x`, we decrement the count of `x`. If the count drops to 0 (no more `x` in the bag), we remove `x` from the set.
  - For `3`, we simply print the size of the set, which represents the number of distinct integers in the bag.
- **Efficiency**: Both insertion and deletion operations on the set and map are O(1) average time, making the solution efficient for the given constraints (Q ≤ 2×10^5).
