The initial code uses a `set` to track the unique integers in the bag, which correctly handles the count of distinct integers for query type 3. However, the issue arises with query type 2, where the code removes the integer `y` from the set, regardless of how many balls with that integer are in the bag. This is incorrect because the bag can have multiple balls with the same integer, and removing one should not necessarily remove the integer from the set unless it was the last ball with that integer.

To fix this, we need to track the count of each integer in the bag. Only when the count of an integer drops to zero should we remove it from the set. This requires using a separate data structure, like a `map` or `unordered_map`, to keep track of the counts of each integer.

#include<bits/stdc++.h>
using namespace std;

void solve()
{
    int q;
    cin >> q;
    unordered_map<int, int> count_map;
    set<int> unique_set;
    for (int i = 0; i < q; i++)
    {
        int a;
        cin >> a;
        if (a == 1)
        {
            int x;
            cin >> x;
            count_map[x]++;
            if (count_map[x] == 1) {
                unique_set.insert(x);
            }
        }
        else if (a == 2)
        {
            int y;
            cin >> y;
            count_map[y]--;
            if (count_map[y] == 0) {
                unique_set.erase(y);
            }
        }
        else if (a == 3)
        {
            cout << unique_set.size() << endl;
        }
    }
}

### Explanation
1. **Data Structures**: 
   - `count_map` (an `unordered_map`) is used to keep track of how many balls of each integer are in the bag.
   - `unique_set` (a `set`) is used to keep track of which integers are currently in the bag (i.e., have a count greater than zero).

2. **Query Handling**:
   - **Type 1 (`1 x`)**: Increment the count of `x` in `count_map`. If this is the first occurrence of `x`, add it to `unique_set`.
   - **Type 2 (`2 x`)**: Decrement the count of `x` in `count_map`. If the count drops to zero, remove `x` from `unique_set`.
   - **Type 3 (`3`)**: Output the size of `unique_set`, which gives the number of distinct integers in the bag.

This approach ensures that we correctly handle multiple balls with the same integer and only update the set of unique integers when necessary, maintaining both correctness and efficiency.
