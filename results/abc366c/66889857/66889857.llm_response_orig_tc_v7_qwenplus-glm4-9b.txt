Since the set will automatically handle duplicates and the provided input guarantees the existence of an element before attempting to remove it, we can use the following approach:

1. **Initialization**: We initialize an empty set to store the integers written on the balls.
2. **Query Processing**:
   - For a type 1 query (`1 x`), we insert the integer `x` into the set.
   - For a type 2 query (`2 x`), we remove the integer `x` from the set. Since it's guaranteed that `x` exists, there's no need to check for its presence.
   - For a type 3 query (`3`), we output the size of the set, which gives us the number of different integers present in the bag.
3. **Output**: We print the results of the type 3 queries sequentially.

Here is the corrected code:

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    int Q;
    cin >> Q;
    set<int> a; // Use set to manage unique integers

    while (Q--) {
        int query_type;
        cin >> query_type;

        if (query_type == 1) {
            int x;
            cin >> x;
            a.insert(x); // Insert x into the set
        } else if (query_type == 2) {
            int x;
            cin >> x;
            a.erase(x); // Remove x from the set
        } else if (query_type == 3) {
            cout << a.size() << endl; // Output the size of the set
        }
    }

    return 0;
}
```

This code correctly processes the queries as specified, ensuring that the operations on the set are efficient. Since the set operations `insert` and `erase` have an average time complexity of O(log n), this approach will handle the upper limits of the problem's constraints effectively.
