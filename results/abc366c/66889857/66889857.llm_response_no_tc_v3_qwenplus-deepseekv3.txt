The original code uses a `set<int>` to keep track of the distinct integers in the bag. However, this approach fails because a set cannot handle multiple occurrences of the same integer. For example, if you insert two balls with the integer `x`, the set will only store one occurrence of `x`. When you remove one `x`, the set will erase it entirely, even though there might still be another `x` in the bag.

To fix this, we need to maintain a count of how many times each integer appears in the bag. We can use a `map` or an array to keep track of these counts. Additionally, we need to maintain a separate count of the number of distinct integers, which we can increment when a new integer is added (count goes from 0 to 1) and decrement when an integer's count drops to 0.

Here is the corrected code:

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);
    
    int Q;
    cin >> Q;

    unordered_map<int, int> count;
    int distinct = 0;

    while (Q--) {
        int type;
        cin >> type;

        if (type == 1) {
            int x;
            cin >> x;
            count[x]++;
            if (count[x] == 1) {
                distinct++;
            }
        } else if (type == 2) {
            int x;
            cin >> x;
            count[x]--;
            if (count[x] == 0) {
                distinct--;
            }
        } else if (type == 3) {
            cout << distinct << '\n';
        }
    }
    return 0;
}
```

### Explanation:
1. **Data Structures**: We use an `unordered_map<int, int>` to keep track of the count of each integer in the bag. This allows efficient insertion, deletion, and lookup operations.
2. **Distinct Count**: We maintain a variable `distinct` that keeps track of the number of distinct integers in the bag. This is updated whenever the count of an integer changes from 0 to 1 (increment) or from 1 to 0 (decrement).
3. **Efficiency**: The operations are O(1) average time complexity for each query, making the solution efficient for the given constraints (Q up to 2 × 10^5).
4. **Edge Cases**: The solution correctly handles cases where multiple balls of the same integer are added and removed, ensuring the `distinct` count is accurate at all times.
5. **Output**: For each query of type 3, the current value of `distinct` is printed, which is the number of different integers in the bag at that moment.
