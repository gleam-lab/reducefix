To fix this code, we need to address the issue highlighted in the failing test case. The problem arises because the current implementation uses a `set` to track the distinct integers in the bag, which correctly handles the count of distinct integers but fails to account for the actual number of balls in the bag. Specifically, when a ball is removed, the code removes all instances of that integer from the set, which is incorrect if there are multiple balls with the same integer in the bag.

### Approach
1. **Track Count of Each Integer**: We need to maintain a count of how many balls there are for each integer in the bag.
2. **Track Distinct Integers**: We also need to keep track of the distinct integers present in the bag to efficiently answer the query of type 3 (count of distinct integers).
3. **Update Data Structures Appropriately**: When adding a ball (query type 1), we increment the count for that integer. If the count was zero before incrementing, we add the integer to the set of distinct integers. When removing a ball (query type 2), we decrement the count. If the count becomes zero after decrementing, we remove the integer from the set of distinct integers.

### Solution Code
```cpp
#include <bits/stdc++.h>
using namespace std;
#define rep(i, n) for (int i = 0; i < (int)(n); i++)
#define rep2(i, s, n) for (int i = (s); i < (int)(n); i++)

int main() {
    int Q; cin >> Q;
    unordered_map<int, int> count;
    set<int> distinct;

    rep(i, Q) {
        int query; cin >> query;
        if (query == 1) {
            int x; cin >> x;
            count[x]++;
            if (count[x] == 1) {
                distinct.insert(x);
            }
        } else if (query == 2) {
            int x; cin >> x;
            count[x]--;
            if (count[x] == 0) {
                distinct.erase(x);
            }
        } else if (query == 3) {
            cout << distinct.size() << endl;
        }
    }
}
```

### Explanation
1. **Data Structures**: We use an `unordered_map` called `count` to keep track of the number of balls for each integer. A `set` called `distinct` is used to maintain the distinct integers present in the bag.
2. **Handling Queries**:
   - **Type 1 (Add x)**: Increment the count of `x` in the map. If the count becomes 1 (meaning it was not present before), add `x` to the `distinct` set.
   - **Type 2 (Remove x)**: Decrement the count of `x` in the map. If the count reaches 0 (meaning no more balls with `x` are left), remove `x` from the `distinct` set.
   - **Type 3 (Print distinct count)**: The size of the `distinct` set gives the number of different integers in the bag, which is printed.
3. **Efficiency**: Both insertion and deletion operations in the `set` and `unordered_map` average O(1) time complexity, leading to an overall O(Q) solution, which is efficient given the constraints.
