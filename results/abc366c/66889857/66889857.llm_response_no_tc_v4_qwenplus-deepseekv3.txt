The original code uses a `set` to keep track of the different integers in the bag, which correctly handles the count of distinct integers (query type 3). However, it fails to account for multiple occurrences of the same integer. For example, if you add two balls with 'x' and then remove one, the set would erase 'x' entirely, which is incorrect because there's still one ball left with 'x'. 

To fix this, we need to maintain both the count of each integer (using a hash map or dictionary) and a set of integers that have at least one occurrence. The set should only remove an integer when its count drops to zero.

### Fixed Code
```cpp
#include <bits/stdc++.h>
using namespace std;
#define rep(i, n) for (int i = 0; i < (int)(n); i++)
#define rep2(i, s, n) for (int i = (s); i < (int)(n); i++)

int main() {
    int Q; cin >> Q;
    unordered_map<int, int> count;
    set<int> distinct;
    
    rep(i, Q) {
        int type; cin >> type;
        if (type == 1) {
            int x; cin >> x;
            if (count[x] == 0) {
                distinct.insert(x);
            }
            count[x]++;
        } else if (type == 2) {
            int x; cin >> x;
            count[x]--;
            if (count[x] == 0) {
                distinct.erase(x);
            }
        } else if (type == 3) {
            cout << distinct.size() << endl;
        }
    }
    return 0;
}
```

### Explanation
1. **Data Structures**: 
   - `unordered_map<int, int> count` keeps track of the number of balls for each integer `x`.
   - `set<int> distinct` maintains the integers that currently have at least one ball in the bag.
   
2. **Handling Queries**:
   - **Type 1 (`1 x`)**: Increment the count of `x`. If the count was zero before, add `x` to the `distinct` set.
   - **Type 2 (`2 x`)**: Decrement the count of `x`. If the count drops to zero, remove `x` from the `distinct` set.
   - **Type 3 (`3`)**: Output the size of the `distinct` set, which gives the number of different integers in the bag.

This approach ensures that we correctly handle multiple occurrences of the same integer and efficiently respond to each query in constant time on average.
