To fix this code, we need to ensure that we correctly account for the number of balls with each integer in the bag. The initial approach using a `set` only keeps track of the distinct integers but not their counts, which leads to incorrect results when multiple balls of the same integer are added and removed. 

### Approaches
1. **Tracking Counts**: Use a `map` or `unordered_map` to keep track of the count of each integer in the bag. This allows us to:
   - **Insert (`1 x`)**: Increment the count for `x`.
   - **Remove (`2 x`)**: Decrement the count for `x` and remove the entry if the count reaches zero.
   - **Query (`3`)**: The size of the map (number of distinct integers) is the answer.

### Solution Code
```cpp
#include <bits/stdc++.h>
using namespace std;
#define rep(i, n) for (int i = 0; i < (int)(n); i++)
#define rep2(i, s, n) for (int i = (s); i < (int)(n); i++)

int main() {
  int Q; cin >> Q;
  unordered_map<int, int> count;
  int distinct = 0;

  rep(i, Q) {
    int type; cin >> type;
    if (type == 1) {
      int x; cin >> x;
      if (count[x] == 0) {
        distinct++;
      }
      count[x]++;
    } else if (type == 2) {
      int x; cin >> x;
      count[x]--;
      if (count[x] == 0) {
        distinct--;
      }
    } else if (type == 3) {
      cout << distinct << endl;
    }
  }
}
```

### Explanation
1. **Data Structure**: We use an `unordered_map` to store counts of each integer. This allows efficient O(1) average time complexity for insertions, deletions, and lookups.
2. **Tracking Distinct Integers**: We maintain a variable `distinct` that keeps track of the number of distinct integers in the bag. This variable is:
   - **Incremented** when an integer's count transitions from 0 to 1 (new integer added).
   - **Decremented** when an integer's count transitions from 1 to 0 (last instance removed).
3. **Query Handling**: 
   - For **type 1**, we check if the integer is new (count was zero) before incrementing `distinct`.
   - For **type 2**, we decrement the count and check if it reaches zero to decrement `distinct`.
   - For **type 3**, we simply print the current value of `distinct`, which represents the number of distinct integers in the bag.

This approach ensures that we correctly handle all operations in O(1) average time per query, making it efficient and suitable for the given constraints.
