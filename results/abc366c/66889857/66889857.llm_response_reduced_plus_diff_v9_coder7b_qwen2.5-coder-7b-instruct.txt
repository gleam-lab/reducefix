To solve this problem efficiently while ensuring correct functionality, we need to use a combination of data structures to handle insertions, deletions, and querying distinct elements. Here's a step-by-step approach:

1. **Insertion**: We need to add elements to a container that can store unique elements efficiently.
2. **Deletion**: We need to remove elements from the same container efficiently.
3. **Querying Distinct Elements**: We need to determine the number of unique elements in the container efficiently.

Given these requirements, using a `std::set` or `std::unordered_set` would be appropriate. However, since the constraints involve up to \(2 \times 10^5\) operations, `std::set` might not be the most efficient due to its logarithmic time complexity for each operation. Therefore, using `std::unordered_set` will likely provide better performance.

Here is the corrected C++ code:

```cpp
#include <iostream>
#include <unordered_set>

int main() {
    int n;
    std::cin >> n;
    std::unordered_set<int> bag;

    for (int i = 0; i < n; ++i) {
        int op, x;
        std::cin >> op >> x;

        if (op == 1) {
            // Insert element x into the bag
            bag.insert(x);
        } else if (op == 2) {
            // Erase element x from the bag if it exists
            auto it = bag.find(x);
            if (it != bag.end()) {
                bag.erase(it);
            }
        } else if (op == 3) {
            // Print the number of distinct elements in the bag
            std::cout << bag.size() << std::endl;
        }
    }

    return 0;
}
```

### Explanation:
1. **Data Structure Choice**:
   - `std::unordered_set` provides average constant-time complexity for insertion (`O(1)`), deletion (`O(1)`), and lookup (`O(1)`).
   
2. **Operations**:
   - For `op == 1`, we simply insert the element `x` into the `unordered_set`.
   - For `op == 2`, we check if the element `x` exists in the `unordered_set`. If it does, we erase it.
   - For `op == 3`, we output the size of the `unordered_set`, which represents the number of distinct elements.

This solution ensures that all operations are performed efficiently, making it suitable for large inputs within the given constraints.
