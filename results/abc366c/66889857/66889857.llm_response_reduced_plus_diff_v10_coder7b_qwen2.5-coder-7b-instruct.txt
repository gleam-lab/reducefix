To solve the problem, we need to maintain a collection of unique elements efficiently. Given the operations involved (insert, delete, and count distinct elements), using a `std::set` is ideal because it automatically handles duplicates and provides efficient insertion, deletion, and counting of unique elements.

Here's the corrected and optimized C++ code:

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(0);
    
    int n; cin >> n;
    set<int> s;
    
    while(n--) {
        int type; cin >> type;
        if(type == 1) {
            int x; cin >> x;
            s.insert(x);
        } else if(type == 2) {
            int x; cin >> x;
            s.erase(s.find(x));
        } else if(type == 3) {
            cout << s.size() << '\n';
        }
    }
    
    return 0;
}
```

### Explanation:
1. **Data Structure**: We use a `std::set<int>` to store the elements. A `std::set` automatically maintains the elements in sorted order and ensures no duplicates.
2. **Operations**:
   - **Insert (`type == 1`)**: Insert the element `x` into the set.
   - **Delete (`type == 2`)**: Erase the element `x` from the set. We use `find` to locate the element first before erasing it.
   - **Count Unique Elements (`type == 3`)**: The size of the set directly gives us the number of unique elements.
3. **Efficiency**: 
   - Insertion and deletion operations in a `std::set` take \(O(\log N)\) time.
   - Counting unique elements takes \(O(1)\) time.
4. **Performance Optimization**: 
   - We disable synchronization between C and C++ streams to improve performance using `ios_base::sync_with_stdio(false)` and `cin.tie(0)`.

This solution ensures that all operations are handled correctly and efficiently within the given constraints.
