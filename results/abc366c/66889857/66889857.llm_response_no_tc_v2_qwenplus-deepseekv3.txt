For example, if you insert two balls with `x=5`, the `set` will still only count it once, and removing one `x=5` will remove it entirely from the set, which is incorrect because there was another ball with `x=5` still in the bag.

To fix this, we need to maintain a frequency count of each integer in the bag, and a separate count of distinct integers. The frequency count will help us track how many balls of each integer are present, and the distinct count will be updated whenever the frequency of an integer changes between zero and non-zero.

```cpp
using namespace std;
#define rep(i, n) for (int i = 0; i < (int)(n); i++)

  int Q; cin >> Q;
  unordered_map<int, int> freq;

  rep(i, Q) {

    if (type == 1) {

      if (freq[x] == 0) {
        distinct++;
      }
      freq[x]++;
    } else if (type == 2) {
      int x; cin >> x;
      freq[x]--;
      if (freq[x] == 0) {
        distinct--;
      }
    } else if (type == 3) {

    }
  }
}
```

### Explanation
1. **Frequency Map**: We use an `unordered_map<int, int>` to keep track of how many balls of each integer `x` are in the bag. This allows us to handle multiple balls with the same integer correctly.
2. **Distinct Count**: We maintain a separate variable `distinct` that counts the number of distinct integers in the bag. This variable is incremented when the frequency of an integer changes from 0 to 1 (indicating a new distinct integer) and decremented when the frequency changes from 1 to 0 (indicating no more balls of that integer are left).
3. **Efficiency**: The operations (insert, delete, and query) are all O(1) on average due to the use of the hash map, making the solution efficient for the given constraints.
4. **Output**: For each query of type 3, we simply print the current value of `distinct`, which gives the number of different integers in the bag at that moment.

This approach ensures correctness by accurately tracking the presence and count of each integer in the bag, while maintaining efficiency suitable for large input sizes.
