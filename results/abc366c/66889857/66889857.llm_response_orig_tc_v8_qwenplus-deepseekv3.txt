However, the problem requires handling multiple balls with the same integer value, which the original code doesn't account for since a `set` only keeps track of whether a value is present, not how many.

To fix this, we need to:
1. Use a `multiset` or a `map` to track the count of each integer in the bag
2. Maintain a separate count of distinct integers
3. Update the distinct count when adding/removing integers based on whether the count changes from/to zero

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);
    
    int Q;
    cin >> Q;
    
    unordered_map<int, int> count;
    int distinct = 0;
    
    while (Q--) {
        int type;
        cin >> type;
        
        if (type == 1) {
            int x;
            cin >> x;
            if (count[x] == 0) distinct++;
            count[x]++;
        } else if (type == 2) {
            int x;
            cin >> x;
            count[x]--;
            if (count[x] == 0) distinct--;
        } else if (type == 3) {
            cout << distinct << '\n';
        }
    }
}
```

1. Used `unordered_map` to track counts of each integer (faster than `map` for this problem)
2. Maintained a `distinct` counter that's incremented when a new integer is added (count goes from 0 to 1)
3. Decremented `distinct` when count of an integer reaches 0
4. Added `ios_base::sync_with_stdio(false)` and `cin.tie(nullptr)` for faster I/O
5. Used proper formatting with `'\n'` instead of `endl` for performance
