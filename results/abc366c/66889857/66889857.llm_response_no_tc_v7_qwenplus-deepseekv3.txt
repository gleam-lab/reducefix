The incorrect code provided uses a `set` to track the unique integers in the bag, which works correctly for determining the number of different integers (query type 3). However, it fails to handle cases where multiple balls with the same integer can be in the bag. Specifically, when removing a ball (query type 2), the code removes all instances of that integer from the set, which is incorrect if there are multiple balls with the same integer. We need to track the count of each integer in the bag to correctly handle additions and removals.

### Approaches
1. **Track Counts**: Use a dictionary (or hash map) to keep track of the count of each integer in the bag.
2. **Track Unique Integers**: Use a set to keep track of the unique integers currently in the bag. This allows for O(1) access to the number of unique integers when query type 3 is encountered.
3. **Update Carefully**: When adding or removing balls, update the counts in the dictionary. Only add or remove integers from the set when their count transitions between zero and one or vice versa.

### Solution Code
```cpp
#include <bits/stdc++.h>
using namespace std;
#define rep(i, n) for (int i = 0; i < (int)(n); i++)
#define rep2(i, s, n) for (int i = (s); i < (int)(n); i++)

int main() {
  ios_base::sync_with_stdio(false);
  cin.tie(nullptr);

  int Q; cin >> Q;
  unordered_map<int, int> count;
  set<int> unique;

  rep(i, Q) {
    int query_type; cin >> query_type;
    if (query_type == 1) {
      int x; cin >> x;
      count[x]++;
      if (count[x] == 1) {
        unique.insert(x);
      }
    } else if (query_type == 2) {
      int x; cin >> x;
      count[x]--;
      if (count[x] == 0) {
        unique.erase(x);
      }
    } else if (query_type == 3) {
      cout << unique.size() << '\n';
    }
  }
}
```

### Explanation
1. **Initialization**: We initialize a `count` map to keep track of how many times each integer appears in the bag and a `set` called `unique` to keep track of integers that are present at least once in the bag.
2. **Query Processing**: For each query:
   - **Type 1 (Add x)**: Increment the count of `x` in the map. If the count becomes 1, add `x` to the `unique` set.
   - **Type 2 (Remove x)**: Decrement the count of `x` in the map. If the count drops to 0, remove `x` from the `unique` set.
   - **Type 3 (Print Unique Count)**: The size of the `unique` set gives the number of different integers in the bag, which is printed directly.
3. **Efficiency**: Both the `unordered_map` and `set` operations are average O(1) per operation, making this approach efficient for the given constraints (Q up to 2×10^5).

This solution ensures that we correctly handle multiple instances of the same integer and efficiently provide the number of unique integers when queried.
