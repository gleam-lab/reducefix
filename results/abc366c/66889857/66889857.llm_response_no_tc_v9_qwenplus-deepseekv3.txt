The original code has a significant flaw: it uses a `set<int>` to keep track of the balls in the bag. While a set can efficiently track the number of distinct integers (which is needed for query type 3), it fails to correctly handle multiple occurrences of the same integer. 

For example, if you insert two balls with the integer `5`, the set will only store one `5`. When you remove one `5` (query type 2), the set will remove all occurrences of `5`, which is incorrect because there should still be one `5` left in the bag. 

To fix this, we need to track both the count of each integer and the distinct integers. We can use:
1. A `map` or `unordered_map` to keep track of the count of each integer.
2. A `set` to keep track of the distinct integers (only insert into the set when the count becomes 1, and remove from the set when the count becomes 0).

    unordered_map<int, int> count;
    set<int> distinct;

    rep(i, Q) {
        int query_type; cin >> query_type;
        if (query_type == 1) {

            if (count[x] == 0) {
                distinct.insert(x);
            }
            count[x]++;
        } else if (query_type == 2) {

            count[x]--;
            if (count[x] == 0) {

            }
        } else if (query_type == 3) {

        }
    }
    return 0;
}

### Explanation
1. **Data Structures**: 
   - `unordered_map<int, int> count` keeps track of how many times each integer `x` appears in the bag.
   - `set<int> distinct` keeps track of the distinct integers currently in the bag.
   
2. **Query Handling**:
   - **Type 1 (`1 x`)**:
     - If `x` is not already in the bag (`count[x] == 0`), add `x` to `distinct`.
     - Increment the count of `x`.
   - **Type 2 (`2 x`)**:
     - Decrement the count of `x`.
     - If the count of `x` drops to `0`, remove `x` from `distinct`.
   - **Type 3 (`3`)**:
     - Output the size of `distinct`, which gives the number of distinct integers in the bag.

This approach ensures that each operation is handled efficiently, with `O(1)` average time complexity for each operation (using `unordered_map` and `set`), making it suitable for large input sizes.
