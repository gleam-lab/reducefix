The issue with the provided code is that it reads the query type incorrectly. The input format uses string-like tokens where a query of type 1, 2, or 3 is given as:
- `1 x`
- `2 x`
- `3`

But in the current code, it reads the first integer (`x`) and checks if it's 1, 2, or 3. This works only if the query is exactly on one line — however, when reading `3`, there is **no following number**, but the code doesn't account for this.

So when a query is `3`, the code reads `3` into `x`, then tries to read another integer `y` for the `if(x == 1)` or `if(x == 2)` branches — which fails because no integer follows `3`.

This leads to undefined behavior (likely reading garbage or failing input), causing incorrect results.

### Fix
We must read the query type first, then conditionally read `x` only for queries of type 1 and 2.

---

```cpp
#include <bits/stdc++.h>
using namespace std;
#define rep(i, n) for (int i = 0; i < (int)(n); i++)

int main() {
    int Q;
    cin >> Q;

    set<int> bag;

    rep(i, Q) {
        int type;
        cin >> type;

        if (type == 1) {
            int x;
            cin >> x;
            bag.insert(x);
        } else if (type == 2) {
            int x;
            cin >> x;
            bag.erase(x);
        } else if (type == 3) {
            cout << bag.size() << '\n';
        }
    }

    return 0;
}
```
