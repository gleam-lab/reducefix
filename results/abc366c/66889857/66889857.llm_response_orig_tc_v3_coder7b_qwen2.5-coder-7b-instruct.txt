To address the issue and ensure the code correctly handles all queries while maintaining optimal performance, we need to focus on using efficient data structures. For this problem, using two sets seems appropriate:

- One set (`all_elements`) will store all elements in the bag.
- Another set (`distinct_elements`) will store only distinct elements, ensuring each element is counted only once.

This approach allows us to efficiently handle insertions, deletions, and counting unique elements. Here’s how you can implement it:

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(0);

    int n; cin >> n;
    set<int> all_elements;
    set<int> distinct_elements;

    for(int i = 0; i < n; ++i) {
        int type, x; cin >> type >> x;

        if(type == 1) {
            // Insert the element into both sets
            all_elements.insert(x);
            distinct_elements.insert(x);
        } else if(type == 2) {
            // Erase the element from both sets
            auto it = all_elements.find(x);
            if(it != end(all_elements)) {
                all_elements.erase(it);
                distinct_elements.erase(distinct_elements.find(x));
            }
        } else if(type == 3) {
            // Output the size of the distinct elements set
            cout << distinct_elements.size() << '\n';
        }
    }

    return 0;
}
```

### Explanation:
1. **Initialization**: We initialize two sets, `all_elements` to store all elements and `distinct_elements` to store unique elements.
2. **Processing Queries**:
   - For query type 1, we insert the element into both sets.
   - For query type 2, we remove the element from both sets if it exists.
   - For query type 3, we output the size of the `distinct_elements` set, which represents the number of unique elements in the bag.
   
### Complexity Analysis:
- **Insertion**: Both operations (inserting into `all_elements` and `distinct_elements`) take \(O(\log n)\) time.
- **Deletion**: Both operations (deleting from `all_elements` and `distinct_elements`) also take \(O(\log n)\) time.
- **Counting Unique Elements**: This operation takes \(O(1)\) time as we simply return the size of the `distinct_elements` set.

This solution ensures that all operations are handled efficiently and maintains the required functionality of the problem.
