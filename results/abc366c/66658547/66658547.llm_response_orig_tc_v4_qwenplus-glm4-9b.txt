
The problem requires managing a bag of balls with integers written on them, handling three types of operations: adding a ball, removing a ball, and counting the number of distinct integers in the bag. The provided code is inefficient because each insert and erase operation on the `std::set` data structure has a complexity of O(log n), leading to an overall complexity of O(Q log n), which can be too slow for large values of Q.

Below is an optimized version of the code using `std::unordered_set` which provides an average case complexity of O(1) for insertions, deletions, and lookups, making the overall complexity O(Q).

```cpp
#include <iostream>
#include <unordered_set>

int main() {
    std::unordered_set<int> bag;
    int q;
    std::cin >> q;

    while (q--) {
        int command;
        std::cin >> command;

        if (command == 1) {
            int x;
            std::cin >> x;
            bag.insert(x);
        } else if (command == 2) {
            int x;
            std::cin >> x;
            bag.erase(x);
        } else if (command == 3) {
            std::cout << bag.size() << "\n";
        }
    }

    return 0;
}
```

### Explanation:
1. **Data Structure**: We use `std::unordered_set` instead of `std::set`. This is because `std::unordered_set` offers average constant time complexity for insertion, deletion, and lookup operations.
2. **Input Handling**: We read the number of queries `q`. For each query, we determine the operation (`1`, `2`, or `3`) and execute it accordingly.
3. **Query Execution**:
   - For query `1 x`, we insert `x` into the `bag`.
   - For query `2 x`, we remove `x` from the `bag`.
   - For query `3`, we output the size of the `bag`, which represents the number of distinct integers in it.
4. **Efficiency**: Each operation (insert, erase, size retrieval) is O(1) on average, making the overall complexity O(Q). This is efficient enough given the constraints.
