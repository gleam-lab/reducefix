The problem requires handling three types of queries on a bag of balls with integers written on them. The initial code uses a `set` to keep track of the distinct integers in the bag, which correctly handles the count of distinct integers (query type 3). However, it fails to account for the fact that multiple balls can have the same integer, and simply inserting or erasing from the set doesn't track the actual number of balls with each integer. 

To fix this, we need a way to:
1. Track the count of each integer in the bag (using a `map` or an array for frequencies).
2. Only consider an integer as "present" in the distinct count if its count is greater than zero.
3. Adjust the count of distinct integers when an integer's count transitions between zero and non-zero.

Here's the corrected code:

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);
    
    int Q;
    cin >> Q;
    
    unordered_map<int, int> freq;
    int distinct = 0;
    
    while (Q--) {
        int type;
        cin >> type;
        
        if (type == 1) {
            int x;
            cin >> x;
            if (freq[x] == 0) {
                distinct++;
            }
            freq[x]++;
        } else if (type == 2) {
            int x;
            cin >> x;
            freq[x]--;
            if (freq[x] == 0) {
                distinct--;
            }
        } else if (type == 3) {
            cout << distinct << "\n";
        }
    }
    
    return 0;
}
```

### Explanation:
1. **Frequency Tracking**: We use an `unordered_map` (or a frequency array) to keep track of how many balls there are for each integer `x`.
2. **Distinct Count**: We maintain a `distinct` counter that is incremented when a new integer is added (its frequency goes from 0 to 1) and decremented when an integer's frequency drops to 0.
3. **Efficiency**: The operations are O(1) on average for each query, making the solution efficient for the given constraints (Q ≤ 2 × 10^5).
4. **Edge Cases**: Handles cases where multiple balls of the same integer are added and removed, ensuring the distinct count is accurate.

This approach ensures that the solution is both correct and efficient for the problem's constraints.
