The issue with the original code is that it uses a `set` to track the different integers in the bag, but it doesn't account for the multiplicity of each integer. For example, if there are multiple balls with the same integer `x`, inserting another `x` won't change the set, and removing one `x` will remove all instances of `x` from the set, which is incorrect.

To fix this, we need to track the count of each integer in the bag. We can use a `map` or `unordered_map` to keep track of how many balls there are for each integer. Here's the corrected approach:

1. Use a `map` to store the count of each integer.
2. Use a separate variable to keep track of the number of distinct integers (this will be the size of the map).
3. For `1 x`, increment the count of `x`. If the count was zero before incrementing, increment the distinct count.
4. For `2 x`, decrement the count of `x`. If the count becomes zero after decrementing, decrement the distinct count.
5. For `3`, simply output the distinct count.

Here's the fixed code:

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);
    
    unordered_map<int, int> count;
    int distinct = 0;
    int q;
    cin >> q;
    while (q--) {
        int type;
        cin >> type;
        if (type == 1) {
            int x;
            cin >> x;
            if (count[x] == 0) {
                distinct++;
            }
            count[x]++;
        } else if (type == 2) {
            int x;
            cin >> x;
            count[x]--;
            if (count[x] == 0) {
                distinct--;
            }
        } else if (type == 3) {
            cout << distinct << "\n";
        }
    }
    return 0;
}
```

### Explanation:
- **`unordered_map<int, int> count`**: This keeps track of how many balls there are for each integer `x`.
- **`int distinct`**: This variable keeps track of the number of distinct integers in the bag (i.e., the size of the map).
- **Query Handling**:
  - **`1 x`**: Increment the count for `x`. If it was zero before, it means `x` is a new integer, so increment `distinct`.
  - **`2 x`**: Decrement the count for `x`. If it becomes zero, it means `x` is no longer in the bag, so decrement `distinct`.
  - **`3`**: Output the current value of `distinct`.

This approach ensures that we correctly track the number of distinct integers in the bag while efficiently handling each query in constant time. The use of `unordered_map` ensures average O(1) time complexity for insertions and deletions, which is crucial given the constraint `Q ≤ 2 × 10^5`.
