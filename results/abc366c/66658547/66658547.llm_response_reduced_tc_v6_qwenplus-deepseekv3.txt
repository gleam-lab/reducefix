The issue with the original code is that it uses a `set` to track the unique integers in the bag. However, a `set` only keeps track of the presence or absence of an integer, not the count of how many times each integer appears. When removing a ball with a specific integer, the code erases it from the set, even if there are multiple balls with the same integer in the bag. This leads to incorrect counts of distinct integers when balls are removed.

To fix this, we need to use a data structure that can track the count of each integer in the bag. A `map` (or `unordered_map`) can be used to keep track of the counts of each integer. The number of distinct integers is then the size of the map (number of keys with a non-zero count).

### Fixed Code
```cpp
#include <bits/stdc++.h>
using namespace std;
using ll = long long;
constexpr ll MOD  = 1000000007;
constexpr int inf = 2000000000;
constexpr ll INF = 9200000000000000000;
int main(){
    unordered_map<int, int> count;
    int distinct = 0;
    int q; cin >> q;
    while(q--){
        int x; cin >> x;
        if(x == 1){
            int t; cin >> t;
            if(count[t] == 0){
                distinct++;
            }
            count[t]++;
        }
        else if(x == 2){
            int t; cin >> t;
            count[t]--;
            if(count[t] == 0){
                distinct--;
            }
        }
        else if(x == 3){
            cout << distinct << "\n";
        }
    }
}
```

### Explanation
1. **Data Structure**: We use an `unordered_map<int, int>` to keep track of the count of each integer in the bag. This allows O(1) average time complexity for insertions, deletions, and lookups.
2. **Distinct Count**: We maintain a variable `distinct` that keeps track of the number of distinct integers in the bag. This is updated whenever the count of an integer transitions between zero and non-zero.
3. **Queries**:
   - **Type 1 (`1 x`)**: We increment the count of `x`. If this was the first occurrence of `x`, we increment `distinct`.
   - **Type 2 (`2 x`)**: We decrement the count of `x`. If this was the last occurrence of `x`, we decrement `distinct`.
   - **Type 3 (`3`)**: We print the current value of `distinct`, which is the number of distinct integers in the bag.

This solution efficiently handles all operations in O(1) average time per query, making it suitable for large input sizes.
