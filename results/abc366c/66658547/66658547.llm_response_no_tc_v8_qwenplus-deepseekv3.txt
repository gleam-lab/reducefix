The initial approach uses a `set` to track the distinct integers in the bag, which correctly handles the count of different integers for query type 3. However, it fails to account for the fact that there can be multiple balls with the same integer in the bag. The `set` only keeps track of whether an integer is present or not, not how many balls have that integer. Therefore, removing a ball with integer `x` (query type 2) should only remove one occurrence of `x`, not all occurrences. If there are multiple balls with `x`, the bag should still contain the remaining balls with `x`.

To fix this, we need to use a data structure that can track the count of each integer in the bag. A `map` or `unordered_map` is suitable for this purpose, where the key is the integer `x` and the value is the count of balls with that integer. We also need to maintain a separate variable to track the number of distinct integers, which changes only when the count of an integer transitions between 0 and 1 or vice versa.

### Fixed Code
```cpp
#include <bits/stdc++.h>
using namespace std;
using ll = long long;
constexpr ll MOD  = 1000000007;
constexpr int inf = 2000000000;
constexpr ll INF = 9200000000000000000;

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);
    
    unordered_map<int, int> count_map;
    int distinct_count = 0;
    int q; cin >> q;
    
    while (q--) {
        int type; cin >> type;
        if (type == 1) {
            int x; cin >> x;
            if (count_map[x] == 0) {
                distinct_count++;
            }
            count_map[x]++;
        } else if (type == 2) {
            int x; cin >> x;
            if (count_map[x] == 1) {
                distinct_count--;
            }
            count_map[x]--;
        } else if (type == 3) {
            cout << distinct_count << "\n";
        }
    }
    
    return 0;
}
```

### Explanation
1. **Data Structures**: We use an `unordered_map` to keep track of the count of each integer in the bag. This allows O(1) average time complexity for insertions, deletions, and lookups.
2. **Tracking Distinct Count**: A variable `distinct_count` is maintained to keep track of the number of distinct integers currently in the bag. This is updated whenever the count of an integer changes between 0 and 1 (increment) or between 1 and 0 (decrement).
3. **Handling Queries**:
   - **Type 1 (Add x)**: Increment the count of `x` in the map. If this was the first occurrence of `x`, increment `distinct_count`.
   - **Type 2 (Remove x)**: Decrement the count of `x` in the map. If this was the last occurrence of `x`, decrement `distinct_count`.
   - **Type 3 (Print Distinct Count)**: Output the current value of `distinct_count`.
4. **Efficiency**: The operations for each query are O(1) on average, making the solution efficient for the given constraints (Q up to 2 × 10^5). The use of `unordered_map` ensures that the average case time complexity is optimal, though in the worst case it could be O(n), but this is mitigated by the problem constraints and typical inputs.
