
The given problem is about managing a bag of balls with integers written on them through a series of queries. The goal is to efficiently handle the queries, which include adding balls, removing balls, and counting the number of unique integers in the bag.

The provided solution incorrectly processes the input and has logical errors related to reading the variable `t` which represents the integer value written on the balls. Here is the corrected and optimized code:

```cpp
#include <iostream>
#include <set>
using namespace std;

int main() {
    int Q;
    cin >> Q;
    
    set<int> balls; // Using a set to store unique ball numbers
    
    for (int i = 0; i < Q; ++i) {
        int queryType;
        cin >> queryType;
        
        if (queryType == 1) {
            int ballNumber;
            cin >> ballNumber;
            balls.insert(ballNumber); // Insert new ball into the set
        } else if (queryType == 2) {
            int ballNumber;
            cin >> ballNumber;
            balls.erase(ballNumber); // Remove ball from the set
        } else if (queryType == 3) {
            cout << balls.size() << "\n"; // Output the number of unique balls
        }
    }
    
    return 0;
}
```

### Explanation of the Code

1. **Input Reading**: The number of queries `Q` is read first. Then, a set `balls` is used to store unique integers corresponding to the ball numbers.

2. **Processing Queries**:
   - For type `1 x`: Read the ball number `x` and insert it into the set `balls`.
   - For type `2 x`: Read the ball number `x` and remove it from the set `balls`.
   - For type `3`: Output the size of the set `balls`, which represents the number of unique integers (or balls).

3. **Efficiency**: The `set` data structure ensures that the insertion and deletion operations are efficient, typically O(log n), making the solution suitable for large numbers of queries.
