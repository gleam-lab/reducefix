The problem involves maintaining a bag of balls with integers written on them and processing three types of queries: adding a ball, removing a ball, and printing the number of distinct integers in the bag. The initial code uses a `set` to track the distinct integers, which correctly handles the distinct count but fails to account for multiple balls with the same integer. 

### Problem Analysis:
1. **Insertion (`1 x`)**: Adds a ball with integer `x` to the bag. If there are multiple balls with the same `x`, the count of `x` increases.
2. **Removal (`2 x`)**: Removes one ball with integer `x` from the bag. It's guaranteed that at least one such ball exists.
3. **Query (`3`)**: Prints the number of distinct integers in the bag. This is the size of the set of integers present in the bag.

### Issues in the Original Code:
- The original code uses a `set` to track distinct integers. While this correctly gives the count of distinct integers, it doesn't handle the multiplicity of each integer. For example, if you add `x` twice and remove `x` once, the `set` would incorrectly show `x` as absent because it doesn't track counts.

### Solution Approach:
- **Tracking Counts**: Use a `map` or `unordered_map` to keep track of the count of each integer in the bag. This allows us to accurately add and remove balls while maintaining the correct count.
- **Distinct Count**: Maintain a separate count of distinct integers. Each time the count of an integer changes from 0 to 1, increment the distinct count. Each time it changes from 1 to 0, decrement the distinct count.

### Solution Code:
```cpp
#include <bits/stdc++.h>
using namespace std;
using ll = long long;
constexpr ll MOD  = 1000000007;
constexpr int inf = 2000000000;
constexpr ll INF = 9200000000000000000;

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);
    
    unordered_map<int, int> count;
    int distinct = 0;
    int q; cin >> q;
    while (q--) {
        int type; cin >> type;
        if (type == 1) {
            int x; cin >> x;
            if (count[x] == 0) {
                distinct++;
            }
            count[x]++;
        } else if (type == 2) {
            int x; cin >> x;
            count[x]--;
            if (count[x] == 0) {
                distinct--;
            }
        } else if (type == 3) {
            cout << distinct << "\n";
        }
    }
    return 0;
}
```

### Explanation:
- **Initialization**: We use an `unordered_map` to count occurrences of each integer and a variable `distinct` to keep track of the number of distinct integers.
- **Insertion (`1 x`)**: For each insertion, if the integer `x` wasn't previously in the map (count was 0), we increment the `distinct` count. Then, we increment the count of `x`.
- **Removal (`2 x`)**: For each removal, we decrement the count of `x`. If this count reaches 0, we decrement the `distinct` count.
- **Query (`3`)**: Simply print the current value of `distinct`, which always reflects the number of distinct integers in the bag.

This approach efficiently handles up to 200,000 queries with integers up to 1,000,000, ensuring optimal performance with average O(1) operations for insertions, deletions, and queries.
