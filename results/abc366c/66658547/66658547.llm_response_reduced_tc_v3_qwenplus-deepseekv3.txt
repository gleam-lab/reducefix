To fix this code, we need to address the issue where the original solution incorrectly handles the removal of balls. Specifically, the problem arises because the original code uses a set to track the presence of distinct integers, but it doesn't account for multiple balls with the same integer. The set only keeps one instance of each integer, so when a ball is removed, the set loses track of any other balls with the same integer that might still be in the bag.

### Approaches
1. **Track Occurrences**: We need a way to track the count of each integer in the bag. This can be done using a dictionary (or a hash map) where the key is the integer, and the value is the count of how many times it appears in the bag.
2. **Maintain Distinct Count**: We also need to maintain the count of distinct integers in the bag. This can be done by incrementing the distinct count when the count of an integer changes from 0 to 1 and decrementing it when the count changes from 1 to 0.

### Solution Code
```cpp
#include <bits/stdc++.h>
using namespace std;
using ll = long long;
constexpr ll MOD  = 1000000007;
constexpr int inf = 2000000000;
constexpr ll INF = 9200000000000000000;

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);
    
    unordered_map<int, int> count;
    int distinct = 0;
    int q;
    cin >> q;
    while (q--) {
        int type;
        cin >> type;
        if (type == 1) {
            int x;
            cin >> x;
            if (count[x] == 0) {
                distinct++;
            }
            count[x]++;
        } else if (type == 2) {
            int x;
            cin >> x;
            if (count[x] == 1) {
                distinct--;
            }
            count[x]--;
        } else if (type == 3) {
            cout << distinct << "\n";
        }
    }
    
    return 0;
}
```

### Explanation
1. **Initialization**: We use an `unordered_map` to keep track of the count of each integer in the bag. The variable `distinct` keeps track of the number of distinct integers present in the bag.
2. **Query Processing**:
   - **Type 1 (Insert)**: When inserting a new integer, we check if it was previously not present (count is 0). If so, we increment the `distinct` count. Then, we increment the count of the integer in the map.
   - **Type 2 (Remove)**: When removing an integer, we first check if its count is 1. If so, we decrement the `distinct` count. Then, we decrement the count of the integer in the map.
   - **Type 3 (Print)**: For the print query, we simply output the current value of `distinct`, which represents the number of distinct integers in the bag.
3. **Efficiency**: This approach ensures that each query is processed in constant time on average, making the overall complexity O(Q) for Q queries, which is efficient enough for the given constraints. The use of an `unordered_map` allows for average O(1) time complexity for insertions, deletions, and lookups.
