To solve this problem efficiently, we need to maintain the counts of unique elements in the bag. Using a `std::set` allows us to keep track of unique elements and provides logarithmic time complexity for insertions and deletions. Here's the corrected and optimized version of your code:

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(0);

    set<int> s;
    int q; cin >> q;
    while (q--) {
        int x; cin >> x;
        if (x == 1) {
            int t; cin >> t;
            s.insert(t);
        } else if (x == 2) {
            int t; cin >> t;
            s.erase(s.find(t));
        } else {
            cout << s.size() << '\n';
        }
    }

    return 0;
}
```

### Explanation:
1. **Using `std::set`**: This ensures that all elements stored in the bag are unique and sorted. The operations (`insert`, `erase`) have logarithmic complexity, making them efficient for large inputs.
  
2. **Efficient Queries**:
   - For `Query Type 1`: We insert an element into the set.
   - For `Query Type 2`: We erase an element from the set.
   - For `Query Type 3`: We simply output the size of the set, which represents the number of unique elements in the bag.

This approach ensures that all constraints are handled correctly and efficiently. By using `std::set`, we avoid the need for manual management of duplicates and ensure that our solution is both correct and performant.
