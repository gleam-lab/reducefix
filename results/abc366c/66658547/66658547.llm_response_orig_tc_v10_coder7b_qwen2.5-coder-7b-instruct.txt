To solve this problem efficiently, we need to use a data structure that supports fast insertion, deletion, and size retrieval. The most suitable data structure for this task is a combination of a hash map and a set. The hash map will allow us to check the existence of an element in constant time, while the set will maintain the elements sorted and provide efficient operations for counting unique elements.

Here's the corrected C++ code:

```cpp
#include <iostream>
#include <set>
#include <unordered_map>

int main() {
    std::ios_base::sync_with_stdio(false);
    std::cin.tie(nullptr);

    std::unordered_map<int, bool> bag;
    std::set<int> unique_elements;

    int q;
    std::cin >> q;

    while (q--) {
        int x;
        std::cin >> x;

        if (x == 1) {
            int t;
            std::cin >> t;
            if (!bag[t]) {
                unique_elements.insert(t);
            }
            bag[t] = true;
        } else if (x == 2) {
            int t;
            std::cin >> t;
            if (bag[t]) {
                unique_elements.erase(unique_elements.find(t));
                bag[t] = false;
            }
        } else {
            std::cout << unique_elements.size() << '\n';
        }
    }

    return 0;
}
```

### Explanation:

1. **Data Structures**:
   - `std::unordered_map<int, bool>`: This map keeps track of whether each element is present in the bag (`true`) or not (`false`). This helps in checking the existence of an element in constant time.
   - `std::set<int>`: This set maintains all unique elements in the bag. It allows us to get the count of unique elements efficiently using its `size()` method.

2. **Operations**:
   - For query type `1`, read the value `t` and add it to the set if it's not already present. Update the map to mark the element as present.
   - For query type `2`, read the value `t` and remove it from the set if it's present. Update the map to mark the element as absent.
   - For query type `3`, simply output the size of the set, which represents the number of unique elements in the bag.

This approach ensures that all operations are performed efficiently, making it suitable for large inputs up to the maximum constraint provided.
