To solve this problem, we need to implement a data structure that supports three operations efficiently: adding an element, removing an element, and counting distinct elements. The most suitable data structure for these operations is a combination of a multiset and a map.

Here's the corrected C++ code:

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
  ios::sync_with_stdio(false);
  cin.tie(nullptr);

  int n; cin >> n;
  multiset<int> ms;
  unordered_map<int, int> countMap;

  while (n--) {
    int type; cin >> type;
    if (type == 1) {
      int x; cin >> x;
      if (countMap[x] == 0) {
        ms.insert(x);
      }
      ++countMap[x];
    } else if (type == 2) {
      int x; cin >> x;
      --countMap[x];
      if (countMap[x] == 0) {
        ms.erase(ms.find(x));
      }
    } else if (type == 3) {
      cout << ms.size() << '\n';
    }
  }

  return 0;
}
```

### Explanation:
1. **Multiset (`ms`)**: This will store all the elements in sorted order. It helps in maintaining the uniqueness of elements as well as providing efficient insertion and deletion operations.
   
2. **Unordered Map (`countMap`)**: This keeps track of how many times each element appears in the multiset. It helps in efficiently determining whether an element needs to be added or removed from the multiset.

### Operations:
- **Type 1**: Insert an element into the multiset only if it doesn't already exist in the multiset. Increment its count in the map.
- **Type 2**: Decrement the count of the element in the map. If the count becomes zero, remove the element from the multiset.
- **Type 3**: Output the size of the multiset, which represents the number of distinct elements.

This approach ensures that all operations are performed efficiently, making the solution suitable for large inputs up to \(2 \times 10^5\).
