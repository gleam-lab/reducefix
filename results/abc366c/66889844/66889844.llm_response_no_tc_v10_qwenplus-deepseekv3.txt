The problem requires handling three types of queries efficiently: adding a ball with a number, removing a ball with a specific number, and printing the count of distinct numbers in the bag. The initial code uses a `set` to track distinct numbers, which works for queries 1 and 3 but fails for query 2 because it removes all occurrences of the number, not just one ball. 

To fix this, we need to track the count of each number in the bag. We can use a `map` (or `unordered_map` for better performance) to maintain the frequency of each number. When removing a ball (query 2), we decrement the count and remove the number from the set only if the count reaches zero. This ensures that the set correctly represents the distinct numbers in the bag at any time.

#include <bits/stdc++.h>
using namespace std;
#define rep(i, n) for (int i = 0; i < (int)(n); i++)

    int Q;

    unordered_map<int, int> freq;
    set<int> distinct;

    rep(i, Q) {
        int query_type;
        cin >> query_type;
        
        if (query_type == 1) {

            freq[x]++;
            if (freq[x] == 1) {
                distinct.insert(x);
            }

            freq[x]--;

        }
    }

}

### Explanation:
1. **Data Structures**: 
   - `unordered_map<int, int> freq` keeps track of the count of each number in the bag.
   - `set<int> distinct` maintains the distinct numbers currently in the bag.

2. **Query Handling**:
   - **Query 1 (`1 x`)**:
     - Increment the count of `x` in `freq`.
     - If the count becomes 1 (indicating a new distinct number), add `x` to `distinct`.
   - **Query 2 (`2 x`)**:

     - If the count drops to 0 (indicating no more `x` in the bag), remove `x` from `distinct`.

This approach ensures that each operation is handled in logarithmic time (for `set` operations) and constant time (for `unordered_map` operations), making it efficient for large input sizes.
