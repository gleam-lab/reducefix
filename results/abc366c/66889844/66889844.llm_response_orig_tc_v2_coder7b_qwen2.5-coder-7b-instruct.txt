To solve this problem efficiently, we need to use a combination of a multiset and a set to keep track of the elements in the bag. The multiset will allow us to insert and remove elements in logarithmic time, and the set will help us determine the number of unique elements quickly.

Here's the corrected C++ code:

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n;
    cin >> n;
    multiset<int> bag;
    set<int> unique_elements;

    while (n--) {
        int type;
        cin >> type;
        
        if (type == 1) {
            int x;
            cin >> x;
            bag.insert(x);
            unique_elements.insert(x);
        } else if (type == 2) {
            int x;
            cin >> x;
            bag.erase(bag.find(x));
            unique_elements.erase(unique_elements.find(x));
        } else if (type == 3) {
            cout << unique_elements.size() << '\n';
        }
    }

    return 0;
}
```

### Explanation:
1. **Multiset (`bag`)**: This will store all the elements in the bag. We can insert and erase elements in \(O(\log N)\) time.
2. **Set (`unique_elements`)**: This will store only the unique elements. By using a set, we can easily get the count of unique elements in \(O(1)\) time.

### Key Points:
- **Insertion**: Insert the element into both the multiset and the set.
- **Deletion**: Erase the element from both the multiset and the set.
- **Query**: The size of the set gives the number of unique elements.

This approach ensures that all operations are efficient and meet the problem's constraints.
