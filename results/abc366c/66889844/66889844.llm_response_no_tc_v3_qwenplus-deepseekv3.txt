The problem requires handling three types of queries on a bag of balls, where each ball has an integer written on it. The operations involve adding a ball, removing a ball, and querying the count of distinct integers in the bag.

### Analysis of the Incorrect Code
The initial approach uses a `set<int>` to track the distinct integers in the bag. While a set is suitable for tracking distinct elements, it fails to account for multiple balls with the same integer. Specifically:
- **Query 1 (Add x):** The code inserts `x` into the set, which only keeps one instance of `x`, ignoring the count of how many balls have `x`.
- **Query 2 (Remove x):** The code removes `x` from the set, which works if there's only one ball with `x`. However, if there are multiple balls with `x`, removing one should not remove `x` from the set of distinct integers until the count reaches zero.

### Solution Approach
To correctly handle multiple balls with the same integer, we need:
1. **A frequency map (`unordered_map<int, int>`):** To track how many balls there are for each integer.
2. **A set or another structure to track distinct integers:** However, the frequency map can also serve this purpose by checking which integers have a count greater than zero.

The steps are:
1. For `1 x`, increment the count of `x` in the frequency map. If the count becomes 1, increment the distinct count.
2. For `2 x`, decrement the count of `x`. If the count becomes 0, decrement the distinct count.
3. For `3`, output the current distinct count.

### Fixed Code
```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);
    
    int Q;
    cin >> Q;
    
    unordered_map<int, int> freq;
    int distinct = 0;
    
    while (Q--) {
        int type;
        cin >> type;
        
        if (type == 1) {
            int x;
            cin >> x;
            if (freq[x] == 0) {
                distinct++;
            }
            freq[x]++;
        } else if (type == 2) {
            int x;
            cin >> x;
            freq[x]--;
            if (freq[x] == 0) {
                distinct--;
            }
        } else if (type == 3) {
            cout << distinct << '\n';
        }
    }
    
    return 0;
}
```

### Explanation
- **Frequency Map (`freq`):** Tracks how many balls there are for each integer `x`.
- **Distinct Count (`distinct`):** Maintains the count of integers with at least one ball in the bag.
- **Query Handling:**
  - **Type 1 (Add x):** Increments the count of `x`. If `x` was not present before (count was 0), increments the distinct count.
  - **Type 2 (Remove x):** Decrements the count of `x`. If the count drops to 0, decrements the distinct count.
  - **Type 3 (Query):** Outputs the current distinct count.

This approach efficiently handles each query in constant time on average, making it suitable for the constraints (up to 200,000 queries). The use of `unordered_map` ensures O(1) average time complexity for insertions, deletions, and lookups.
