```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    int Q;
    cin >> Q;
    
    multiset<int> balls;
    
    while (Q--) {
        int type;
        cin >> type;
        
        if (type == 1) {
            int x;
            cin >> x;
            balls.insert(x);
        } else if (type == 2) {
            int x;
            cin >> x;
            auto it = balls.find(x);
            if (it != balls.end()) {
                balls.erase(it);
            }
        } else if (type == 3) {
            set<int> unique_balls(balls.begin(), balls.end());
            cout << unique_balls.size() << endl;
        }
    }

    return 0;
}
```

### Explanation of Fixes:

1. **Data Structure Change**:
   - Original code used a `set<int>` which only stores unique values. This was incorrect because we can have multiple balls with the same number.
   - Changed to `multiset<int>` to correctly represent multiple occurrences.

2. **Query Type 3**:
   - In the original code, `a.size()` was used, where `a` was a `set`, meaning it counted only unique elements — but that was misleading since the multiset should track counts.
   - For query type 3, we construct a temporary `set<int>` from the `multiset` to count the number of distinct integers currently present in the bag.

3. **Efficiency**:
   - `multiset` supports O(log N) insertions, deletions, and lookups — suitable for up to 2×10⁵ operations.
   - The use of constructing a `set` for query type 3 is acceptable since it's only done when necessary and not on every operation.

4. **Correctness**:
   - Properly handles multiple balls with the same value.
   - Ensures that when a ball is removed, exactly one instance of that value is deleted.
   - Accurately reports the number of distinct integers upon request.
