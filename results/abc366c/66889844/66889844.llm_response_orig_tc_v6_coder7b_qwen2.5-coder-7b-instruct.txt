To solve this problem efficiently, we need to use a combination of data structures to handle insertions, deletions, and querying the count of distinct elements. The primary challenge is to ensure that operations are performed in \(O(\log n)\) time, which can be achieved using a balanced binary search tree like a `std::set`.

Here's the corrected and optimized C++ code:

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(0);

    int n; cin >> n;
    set<int> s;

    while (n--) {
        int type, x;
        cin >> type >> x;

        if (type == 1) {
            s.insert(x);
        } else if (type == 2) {
            s.erase(s.find(x));
        } else if (type == 3) {
            cout << s.size() << '\n';
        }
    }

    return 0;
}
```

### Explanation:
1. **Data Structure**: We use a `std::set` to store the elements. A `std::set` automatically maintains its elements in sorted order and provides efficient insertion and deletion operations in \(O(\log n)\) time.
   
2. **Operations**:
   - For type 1 queries (`1 x`), we simply insert `x` into the set.
   - For type 2 queries (`2 x`), we find the element `x` in the set and erase it.
   - For type 3 queries (`3`), we output the size of the set, which represents the number of unique elements currently stored.

This approach ensures that all operations are handled efficiently, meeting the problem's constraints. The use of `std::set` guarantees that each operation is performed in logarithmic time, making the solution both correct and performant.
