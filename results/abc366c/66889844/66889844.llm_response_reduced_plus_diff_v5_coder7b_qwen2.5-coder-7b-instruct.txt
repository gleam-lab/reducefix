To solve this problem efficiently, we need to use a data structure that allows us to add, remove, and count distinct elements quickly. A `std::unordered_set` is ideal for this purpose because it provides average O(1) time complexity for insertions, deletions, and lookups.

Here's the corrected code:

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
  ios_base::sync_with_stdio(false);
  cin.tie(nullptr);

  int n;
  cin >> n;
  unordered_set<int> s;

  while (n--) {
    int type, x;
    cin >> type >> x;
    if (type == 1) {
      s.insert(x);
    } else if (type == 2) {
      s.erase(s.find(x));
    } else {
      cout << s.size() << '\n';
    }
  }

  return 0;
}
```

### Explanation:
1. **Data Structure**: We use `std::unordered_set<int>` to store the elements. This data structure ensures that each element is unique and provides efficient operations for insertion, deletion, and size checking.
   
2. **Input Handling**: We read the number of queries, `n`, and then process each query in a loop.
   
3. **Query Processing**:
   - For query type `1`, we insert the value `x` into the set.
   - For query type `2`, we erase the value `x` from the set. Note that `find` is used to get an iterator to the element before erasing it.
   - For query type `3`, we output the size of the set, which represents the number of distinct elements.

This approach ensures that all operations are performed efficiently, meeting the constraints of the problem.
