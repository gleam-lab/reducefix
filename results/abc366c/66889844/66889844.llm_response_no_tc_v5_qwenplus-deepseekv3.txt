### Analysis of the Incorrect Code
The provided code uses a `set<int>` to keep track of the integers written on the balls in the bag. While a set is a good choice for maintaining unique elements, it fails to account for the possibility of multiple balls with the same integer in the bag. For example:
- When you insert a ball with integer `x` multiple times, the set will only keep one instance of `x`.
- When you remove a ball with integer `x`, it removes all instances of `x` from the set, not just one.

This leads to incorrect counting of the number of different integers when there are multiple balls with the same integer in the bag.

### Correct Approach
To accurately track the number of different integers, we need to:
1. **Count occurrences** of each integer in the bag. This can be done using a `map` or `unordered_map` where the key is the integer `x`, and the value is the count of how many times `x` appears in the bag.
2. **Maintain a separate count** of the number of unique integers. This count should be incremented when the count of an integer `x` goes from 0 to 1, and decremented when the count goes from 1 to 0.

### Solution Code
```cpp
#include <bits/stdc++.h>
using namespace std;
#define rep(i, n) for (int i = 0; i < (int)(n); i++)
#define rep2(i, s, n) for (int i = (s); i < (int)(n); i++)

int main() {
  int Q; cin >> Q;
  unordered_map<int, int> count;
  int unique_count = 0;
  
  rep(i, Q) {
    int query_type; cin >> query_type;
    if (query_type == 1) {
      int x; cin >> x;
      if (count[x] == 0) {
        unique_count++;
      }
      count[x]++;
    } else if (query_type == 2) {
      int x; cin >> x;
      if (count[x] == 1) {
        unique_count--;
      }
      count[x]--;
    } else if (query_type == 3) {
      cout << unique_count << endl;
    }
  }
}
```

### Explanation
1. **Initialization**: We use an `unordered_map<int, int>` named `count` to keep track of the number of balls for each integer `x`. The variable `unique_count` keeps track of the number of different integers present in the bag.
2. **Handling Queries**:
   - **Type 1 (`1 x`)**: For each insertion, we check if `x` is new (count is 0). If so, we increment `unique_count`. Then, we increment the count for `x`.
   - **Type 2 (`2 x`)**: For each removal, we decrement the count for `x`. If the count reaches 0, we decrement `unique_count` since `x` is no longer in the bag.
   - **Type 3 (`3`)**: This query simply outputs the current value of `unique_count`, which represents the number of different integers in the bag.
   
This approach ensures that we accurately track the number of unique integers even when multiple balls have the same integer, and it efficiently handles the constraints with O(1) average time complexity for each operation.
