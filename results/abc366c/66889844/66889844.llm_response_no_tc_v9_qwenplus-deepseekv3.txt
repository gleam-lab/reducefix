The initial approach uses a `set` to track the distinct integers in the bag, which correctly handles the count of different integers for query type 3. However, it fails to account for the fact that multiple balls with the same integer can be present in the bag. For instance, inserting two balls with `x=1` and then removing one should leave one ball with `x=1`, but the set would completely remove `1` after the first removal, leading to incorrect counts.

To fix this, we need to track the count of each integer in the bag. We can use a `map` (or `unordered_map` for better performance) to keep track of the counts. Here’s the corrected approach:
1. Use a `map` to store the count of each integer.
2. For query type 1, increment the count of `x`.
3. For query type 2, decrement the count of `x` and remove it from the map if the count reaches zero.
4. For query type 3, the size of the map (number of keys) gives the number of different integers.

Here’s the corrected code:

```cpp
#include <bits/stdc++.h>
using namespace std;
#define rep(i, n) for (int i = 0; i < (int)(n); i++)
#define rep2(i, s, n) for (int i = (s); i < (int)(n); i++)

int main() {
  int Q; cin >> Q;
  unordered_map<int, int> count;
  int distinct = 0;
  
  rep(i, Q) {
    int type; cin >> type;
    if (type == 1) {
      int x; cin >> x;
      if (count[x] == 0) distinct++;
      count[x]++;
    } else if (type == 2) {
      int x; cin >> x;
      count[x]--;
      if (count[x] == 0) distinct--;
    } else if (type == 3) {
      cout << distinct << endl;
    }
  }
}
```

### Explanation:
1. **Initialization**: We use an `unordered_map` to count occurrences of each integer and a variable `distinct` to keep track of the number of distinct integers.
2. **Query Handling**:
   - **Type 1 (Insert)**:
     - If the integer `x` is not already in the map (`count[x] == 0`), increment `distinct`.
     - Increment `count[x]`.
   - **Type 2 (Remove)**:
     - Decrement `count[x]`.
     - If `count[x]` becomes zero, decrement `distinct` (as `x` is no longer in the bag).
   - **Type 3 (Print Distinct Count)**:
     - Output the current value of `distinct`, which is the number of distinct integers in the bag.
3. **Efficiency**: Each operation is O(1) on average due to the use of `unordered_map`, making the solution efficient for the given constraints.
