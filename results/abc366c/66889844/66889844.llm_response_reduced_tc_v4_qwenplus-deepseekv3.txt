To fix this code, we need to handle the issue where the set is being modified incorrectly when processing queries of type `2`. The problem arises because the set only keeps track of unique integers, and when we remove an element, it might still have other occurrences of that integer in the bag. However, the problem guarantees that when a `2 x` query is given, there is at least one ball with the integer `x` in the bag. Therefore, we need to ensure that the count of each integer is tracked accurately.

### Approaches
1. **Track Counts of Each Integer**: Instead of using a set to track unique integers, we can use a map (or a hash map) to keep track of the count of each integer in the bag. This way, when we remove a ball, we decrement its count, and if the count reaches zero, we remove it from the map. The size of the map will give us the number of different integers in the bag at any time.
2. **Efficient Operations**: Using a map allows us to handle insertions and deletions in logarithmic time, which is efficient enough given the constraints (Q up to 2 × 10^5).

### Solution Code
```cpp
#include <bits/stdc++.h>
using namespace std;
#define rep(i, n) for (int i = 0; i < (int)(n); i++)
#define rep2(i, s, n) for (int i = (s); i < (int)(n); i++)

int main() {
  int Q; cin >> Q;
  unordered_map<int, int> count;
  int distinct = 0;
  
  rep(i, Q) {
    int query_type; cin >> query_type;
    if (query_type == 1) {
      int x; cin >> x;
      if (count[x] == 0) {
        distinct++;
      }
      count[x]++;
    } else if (query_type == 2) {
      int x; cin >> x;
      count[x]--;
      if (count[x] == 0) {
        distinct--;
      }
    } else if (query_type == 3) {
      cout << distinct << endl;
    }
  }
}
```

### Explanation
1. **Initialization**: We read the number of queries `Q` and initialize an `unordered_map` to keep track of the count of each integer and a variable `distinct` to track the number of unique integers.
2. **Processing Queries**:
   - **Insertion (Type 1)**: For each insertion query, we check if the integer `x` is new (count is zero). If it is, we increment the `distinct` counter. Then, we increment the count of `x` in the map.
   - **Deletion (Type 2)**: For each deletion query, we decrement the count of `x`. If the count reaches zero, we decrement the `distinct` counter.
   - **Query (Type 3)**: For each query of type 3, we simply print the current value of `distinct`, which represents the number of different integers in the bag at that time.
   
This approach ensures that we accurately maintain the count of each integer and efficiently determine the number of distinct integers whenever queried. The use of an `unordered_map` allows for average constant time operations, making the solution efficient for large input sizes.
