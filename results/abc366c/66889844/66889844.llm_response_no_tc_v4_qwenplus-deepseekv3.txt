### Solution Approach
To correctly handle the multiplicity of each integer, we need to:
1. Use a `map<int, int>` or `unordered_map<int, int>` to keep track of the count of each integer in the bag.
2. Use a separate variable to keep track of the number of distinct integers in the bag (the size of the map).
3. For each query:
   - **Type 1 (insert x)**:
     - Increment the count of `x` in the map.
     - If the count of `x` becomes 1 (it was not present before), increment the distinct count.
   - **Type 2 (remove x)**:
     - Decrement the count of `x` in the map.
     - If the count of `x` drops to 0, decrement the distinct count and remove `x` from the map.
   - **Type 3 (print distinct count)**:

This approach ensures that we correctly handle multiple instances of the same integer and efficiently update the distinct count.
