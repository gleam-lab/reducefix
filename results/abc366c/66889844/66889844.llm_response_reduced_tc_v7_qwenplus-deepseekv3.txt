To fix this code, we need to address the issue where the code incorrectly handles the removal of balls from the bag. The current implementation uses a `set<int>` to track the different integers written on the balls. However, a set only keeps one instance of each integer, which means it cannot accurately represent the scenario where multiple balls with the same integer are present in the bag. For example, if we insert a ball with integer `x` twice and then remove one, the set will lose track of `x` entirely, even though another ball with `x` might still be in the bag.

To solve this, we need to use a data structure that can keep track of the count of each integer in the bag. A `map` or `unordered_map` is suitable for this purpose, as it can store the count of each integer. The `map` will help us efficiently update the counts when balls are added or removed, and the size of the map will give the number of different integers in the bag at any point.

### Approaches
1. **Data Structure**: Use a `map<int, int>` to keep track of the count of each integer present in the bag. This allows us to efficiently add, remove, and query the counts.
2. **Handling Queries**:
   - For `1 x` (add operation), increment the count of `x` in the map. If the count becomes 1, this means a new integer is added to the bag.
   - For `2 x` (remove operation), decrement the count of `x` in the map. If the count becomes 0, remove the key `x` from the map.
   - For `3` (query operation), output the current size of the map, which gives the number of different integers in the bag.

### Solution Code
```cpp
#include <bits/stdc++.h>
using namespace std;
#define rep(i, n) for (int i = 0; i < (int)(n); i++)
#define rep2(i, s, n) for (int i = (s); i < (int)(n); i++)

int main() {
  int Q; cin >> Q;
  unordered_map<int, int> count_map;
  
  rep(i, Q){
    int type; cin >> type;
    if(type == 1){
      int x; cin >> x;
      count_map[x]++;
    } else if(type == 2){
      int x; cin >> x;
      if(--count_map[x] == 0){
        count_map.erase(x);
      }
    } else if(type == 3){
      cout << count_map.size() << endl;
    }
  }
}
```

### Explanation
- **Data Structure**: The `unordered_map<int, int>` (or `map<int, int>` for ordered keys) is used to map each integer `x` to its current count in the bag. This allows O(1) average time complexity for insertions, deletions, and updates.
- **Handling Queries**:
  - **Add Query (`1 x`)**: The count of `x` is incremented. If `x` was not previously in the map, it is added with a count of 1.
  - **Remove Query (`2 x`)**: The count of `x` is decremented. If the count drops to zero, the entry for `x` is removed from the map to reflect that there are no more balls with that integer in the bag.
  - **Query (`3`)**: The size of the map is printed, which corresponds to the number of different integers present in the bag at that moment.

This approach ensures that each operation is handled efficiently, and the solution correctly tracks the counts of each integer, providing accurate results for the query operations.
