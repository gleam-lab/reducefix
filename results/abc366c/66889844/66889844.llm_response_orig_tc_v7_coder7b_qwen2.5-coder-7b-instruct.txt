To solve this problem efficiently, we need to use a combination of a multiset and a map. The multiset will help us maintain the unique elements in sorted order, and the map will keep track of the count of each element.

Here's the corrected C++ code:

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);

    int n; cin >> n;
    multiset<int> mst;
    map<int, int> cnt;

    while(n--) {
        int type; cin >> type;
        if(type == 1) {
            int x; cin >> x;
            auto it = mst.find(x);
            if(it != end(mst)) {
                mst.erase(it);
                --cnt[x];
            }
            ++cnt[x];
            mst.insert(x);
        } else if(type == 2) {
            int x; cin >> x;
            ++cnt[x];
            mst.insert(x);
        } else if(type == 3) {
            cout << mst.size() << '\n';
        }
    }

    return 0;
}
```

### Explanation:

1. **Data Structures**:
   - `multiset<int> mst`: This will store all unique elements in sorted order.
   - `map<int, int> cnt`: This will keep track of the count of each element.

2. **Query Processing**:
   - **Type 1**: Insert a new element `x`. If `x` already exists in the multiset, remove it first. Then, insert `x` again and update its count in the map.
   - **Type 2**: Remove an existing element `x`. Increment its count in the map and re-insert it into the multiset.
   - **Type 3**: Print the size of the multiset, which represents the number of distinct elements.

This approach ensures that the operations are efficient and handle all edge cases correctly. The use of a multiset allows us to maintain the uniqueness and sorted order of elements, while the map helps us manage the counts effectively.
