The issue with the original code is that it uses a `set` to track the balls in the bag. While a `set` can efficiently manage unique elements, it doesn't account for the count of each type of ball. For example, if you insert multiple balls with the same integer `x`, the `set` will only store `x` once, and when you remove one `x`, it will remove the only occurrence, which is incorrect if there were multiple `x` balls.

To fix this, we need to maintain a frequency map (like a dictionary or hash map) that keeps track of the count of each integer `x` in the bag. This way, when we remove a ball with integer `x`, we decrement its count, and only remove `x` from the set of distinct integers when its count reaches zero.

### Fixed Code
```cpp
#include <iostream>
#include <unordered_map>
#include <unordered_set>
#include <vector>
#include <algorithm>
#include <string>
#include <climits>

using namespace std;

void Solve() {
    int Q;
    cin >> Q;
    unordered_map<int, int> freq;
    unordered_set<int> distinct;

    while (Q--) {
        int type;
        cin >> type;
        if (type == 1) {
            int x;
            cin >> x;
            freq[x]++;
            distinct.insert(x);
        } else if (type == 2) {
            int x;
            cin >> x;
            freq[x]--;
            if (freq[x] == 0) {
                distinct.erase(x);
            }
        } else {
            cout << distinct.size() << "\n";
        }
    }
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);
    
    Solve();
    
    return 0;
}
```

### Explanation
1. **Data Structures**:
   - `unordered_map<int, int> freq`: Tracks the count of each integer `x` in the bag.
   - `unordered_set<int> distinct`: Tracks the set of distinct integers in the bag.

2. **Operations**:
   - **Type 1 (`1 x`)**:
     - Increment the count of `x` in `freq`.
     - Add `x` to `distinct` if it wasn't already present.
   - **Type 2 (`2 x`)**:
     - Decrement the count of `x` in `freq`.
     - If the count of `x` reaches zero, remove `x` from `distinct`.
   - **Type 3 (`3`)**:
     - Output the size of `distinct`, which is the number of distinct integers in the bag.

This approach ensures that we correctly handle multiple occurrences of the same integer and efficiently manage the counts and distinct elements. The use of hash-based structures (`unordered_map` and `unordered_set`) ensures average O(1) time complexity for each operation, making the solution efficient for large input sizes.
