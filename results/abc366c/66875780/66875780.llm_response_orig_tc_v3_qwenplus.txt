The issue with the original code is that it uses a `set` to store ball values, but this approach doesn't handle duplicate balls correctly. When multiple balls with the same value are added, the set only stores one instance, so when we remove a ball, we might incorrectly remove the last instance even if there were multiple.

We need to track the count of each ball value separately. The correct approach is to use a map (or unordered_map) to maintain the frequency count of each integer, and then track the number of distinct integers.

{

    unordered_map<ll, ll> freq;
    ll distinct = 0;
    
    while (Q--)

        }

            ll tp;

            if (freq[tp] == 0) {
                distinct--;
            }
        }
        else
        {

        }
    }
}
