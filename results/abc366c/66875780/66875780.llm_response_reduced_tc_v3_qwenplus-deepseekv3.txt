To fix this code, we need to accurately track the count of each integer in the bag. The original code uses a set to keep track of the integers, which works for counting distinct integers but fails when multiple balls with the same integer are present. Specifically, when a ball is removed (`2 x`), the set erases all occurrences of `x`, not just one. This leads to incorrect counts when there are multiple balls with the same integer.

### Approaches
1. **Track Occurrences**: Use a dictionary (like `unordered_map` or `map`) to keep track of the count of each integer in the bag.
2. **Maintain Distinct Count**: Use a set to keep track of the distinct integers. However, since multiple balls can have the same integer, we need to ensure that the set only reflects whether an integer is present (count > 0).
3. **Update Counts**: For each operation:
   - `1 x`: Increment the count of `x`. If the count becomes 1, add `x` to the set.
   - `2 x`: Decrement the count of `x`. If the count becomes 0, remove `x` from the set.
   - `3`: Output the size of the set, which represents the number of distinct integers in the bag.

### Solution Code
```cpp
#include <iostream>
#include <unordered_map>
#include <unordered_set>

using namespace std;

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);
    cout.tie(nullptr);

    int Q;
    cin >> Q;
    unordered_map<int, int> count;
    unordered_set<int> distinct;

    while (Q--) {
        int type;
        cin >> type;
        if (type == 1) {
            int x;
            cin >> x;
            if (count[x] == 0) {
                distinct.insert(x);
            }
            count[x]++;
        } else if (type == 2) {
            int x;
            cin >> x;
            count[x]--;
            if (count[x] == 0) {
                distinct.erase(x);
            }
        } else if (type == 3) {
            cout << distinct.size() << "\n";
        }
    }

    return 0;
}
```

### Explanation
- **Tracking Occurrences**: The `unordered_map` `count` keeps track of how many times each integer `x` appears in the bag.
- **Maintaining Distinct Integers**: The `unordered_set` `distinct` keeps track of integers that have a count greater than zero. This ensures that only distinct integers are counted.
- **Operations Handling**:
  - **Insertion (`1 x`)**: Increment the count of `x`. If `x` was not previously in the bag (count was zero), add `x` to the set of distinct integers.
  - **Removal (`2 x`)**: Decrement the count of `x`. If the count drops to zero, remove `x` from the set of distinct integers.
  - **Query (`3`)**: The size of the set `distinct` gives the number of distinct integers in the bag, which is printed immediately.

This approach efficiently handles all operations in average O(1) time per query, making it suitable for large input sizes as specified in the problem constraints.
