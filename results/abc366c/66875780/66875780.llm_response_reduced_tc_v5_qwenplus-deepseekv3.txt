To fix this code, we need to address the issue where the code incorrectly handles the removal of balls from the bag. The original code uses a `set` to track the different integers written on the balls. However, a `set` only keeps track of unique elements, not the count of each element. When a ball is removed, the code simply erases the integer from the set, which is incorrect because there might still be other balls with the same integer in the bag. 

To correctly solve this problem, we need to maintain both the count of each integer and the number of distinct integers. We can use a `map` (or `unordered_map` for better performance) to keep track of the count of each integer. The number of distinct integers can be inferred from the size of the map when the count of an integer becomes zero (it should be removed from the map).

### Approaches
1. **Data Structures**: Use an `unordered_map` to keep track of the count of each integer. This allows O(1) average time complexity for insertions and deletions.
2. **Tracking Distinct Integers**: The size of the `unordered_map` will represent the number of distinct integers in the bag at any time.
3. **Handling Queries**:
   - For query type `1 x`, increment the count of `x` in the map.
   - For query type `2 x`, decrement the count of `x` in the map. If the count reaches zero, remove `x` from the map.
   - For query type `3`, output the size of the map, which gives the number of distinct integers.

### Solution Code
```cpp
#include <iostream>
#include <unordered_map>
using namespace std;

void Solve() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);
    cout.tie(nullptr);
    
    int Q;
    cin >> Q;
    unordered_map<int, int> count;
    int distinct = 0;
    
    while (Q--) {
        int type;
        cin >> type;
        if (type == 1) {
            int x;
            cin >> x;
            if (count[x] == 0) {
                distinct++;
            }
            count[x]++;
        } else if (type == 2) {
            int x;
            cin >> x;
            if (count[x] == 1) {
                distinct--;
            }
            count[x]--;
        } else {
            cout << distinct << '\n';
        }
    }
}

int main() {
    Solve();
    return 0;
}
```

### Explanation
- **Data Structures**: The `unordered_map` named `count` keeps track of how many times each integer `x` appears in the bag.
- **Tracking Distinct Integers**: The variable `distinct` is used to keep track of the number of distinct integers. It is incremented when a new integer is added (count transitions from 0 to 1) and decremented when an integer's count drops to zero (count transitions from 1 to 0).
- **Query Handling**:
  - **Type 1 (`1 x`)**: Adds a ball with integer `x`. If `x` wasn't in the bag before (count was zero), `distinct` is incremented.
  - **Type 2 (`2 x`)**: Removes a ball with integer `x`. If this was the last ball of `x` (count drops to zero), `distinct` is decremented.
  - **Type 3 (`3`)**: Outputs the current value of `distinct`, which is the number of distinct integers in the bag.

This approach efficiently handles each query in constant time on average, making it suitable for the given constraints.
