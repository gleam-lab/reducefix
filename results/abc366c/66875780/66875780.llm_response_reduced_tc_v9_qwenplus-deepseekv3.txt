To fix this code, we need to properly handle the counts of each integer in the bag. The initial approach used a set to track the distinct integers, which worked for insertion and deletion but failed to account for cases where multiple balls of the same integer are present in the bag. Specifically, when a ball is removed, it should only be removed from the set if there are no remaining balls of that integer left in the bag.

### Approaches
1. **Track Counts**: Use a dictionary (or `unordered_map`) to keep track of the count of each integer in the bag.
2. **Track Distinct Integers**: Use a set to keep track of the distinct integers currently in the bag. This helps in efficiently retrieving the number of distinct integers when query type 3 is encountered.
3. **Update Counts**: For insertion (query type 1), increment the count in the dictionary and add the integer to the set if its count was zero before insertion.
4. **Update Counts**: For deletion (query type 2), decrement the count in the dictionary and remove the integer from the set if its count becomes zero after deletion.

This approach ensures that both insertion and deletion operations are handled efficiently, and the set always contains exactly the distinct integers present in the bag at any moment.

### Solution Code
```cpp
#include <iostream>
#include <unordered_map>
#include <unordered_set>
#include <vector>

using namespace std;

void Solve() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);
    cout.tie(nullptr);

    int Q;
    cin >> Q;
    unordered_map<int, int> count;
    unordered_set<int> distinct;

    while (Q--) {
        int query_type;
        cin >> query_type;

        if (query_type == 1) {
            int x;
            cin >> x;
            count[x]++;
            if (count[x] == 1) {
                distinct.insert(x);
            }
        } else if (query_type == 2) {
            int x;
            cin >> x;
            count[x]--;
            if (count[x] == 0) {
                distinct.erase(x);
            }
        } else if (query_type == 3) {
            cout << distinct.size() << "\n";
        }
    }
}

int main() {
    Solve();
    return 0;
}
```

### Explanation
1. **Initialization**: We use `unordered_map<int, int> count` to keep track of how many balls of each integer are in the bag and `unordered_set<int> distinct` to keep track of the distinct integers.
2. **Handling Queries**:
   - **Type 1 (Insertion)**: For each insertion of integer `x`, increment its count in `count`. If the count becomes 1 (indicating it was not present before), add `x` to `distinct`.
   - **Type 2 (Deletion)**: For each deletion of integer `x`, decrement its count in `count`. If the count drops to 0 (indicating no more balls of `x` are left), remove `x` from `distinct`.
   - **Type 3 (Query Distinct Count)**: Output the size of `distinct`, which gives the number of distinct integers in the bag.
3. **Efficiency**: The operations on `unordered_map` and `unordered_set` average O(1) time complexity, making the solution efficient for the given constraints. This ensures the solution handles up to 200,000 queries efficiently.
