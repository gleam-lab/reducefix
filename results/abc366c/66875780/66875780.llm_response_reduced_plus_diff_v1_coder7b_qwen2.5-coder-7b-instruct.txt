The `std::unordered_set` will help us quickly insert and remove elements, ensuring efficient operations for adding and removing items. However, since we need to count unique elements, using `std::unordered_set` alone might not suffice because it does not maintain any order.
```cpp
#include <iostream>
#include <unordered_set>
#include <multiset>

using namespace std;

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(0);

    unordered_set<int> uniqueElements; // To store unique elements
    multiset<int> allElements; // To store all elements

    int q;
    cin >> q;
    while (q--) {
        int type;
        cin >> type;
        if (type == 1) {
            int value;
            cin >> value;
            uniqueElements.insert(value); // Insert into unique set
            allElements.insert(value); // Insert into all elements set
        } else if (type == 2) {
            int value;
            cin >> value;
            uniqueElements.erase(uniqueElements.find(value)); // Erase from unique set
            allElements.erase(allElements.find(value)); // Erase from all elements set
        } else {
            cout << uniqueElements.size() << '\n'; // Output size of unique set
        }
    }

    return 0;
}
```

### Explanation:
- **uniqueElements**: This is an `unordered_set` that stores only unique elements.
- **allElements**: This is a `multiset` that stores all elements, including duplicates.
  
When processing a `type 1` query, we add the element to both sets.
For a `type 2` query, we first erase the element from the `uniqueElements` set and then from the `allElements` set.
For a `type 3` query, we simply output the size of the `uniqueElements` set, which gives us the number of unique elements.

This approach ensures that we efficiently handle all types of queries and meet the problem constraints.
