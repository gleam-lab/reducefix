
### Problem Analysis

The problem requires managing a collection of integers stored in a bag where operations can be performed to add, remove, or query the number of unique integers in the bag. The operations are defined as follows:

1. Add an integer to the bag.
2. Remove an integer from the bag.
3. Report the number of unique integers in the bag.

Given the constraints, the solution must efficiently handle up to 200,000 operations and manage integers up to 1,000,000.

### Solution Strategy

To solve this problem, we can use a set data structure. A set inherently keeps only unique elements and provides average constant time complexity for insert, delete, and check operations. This makes it ideal for handling the operations defined in the problem:

- **Adding an integer** (`1 x`): Insert `x` into the set. If `x` is already in the set, the set remains unchanged.
- **Removing an integer** (`2 x`): Remove `x` from the set. The operation is guaranteed to succeed, hence no need to check if `x` is present.
- **Querying unique count** (`3`): Output the size of the set, which represents the count of unique integers.

### Optimized Solution

Here is the optimized C++ code using the above strategy:

```cpp
#include <iostream>
#include <set>

using namespace std;

int main() {
    int Q;
    cin >> Q;

    set<int> bag;

    while (Q--) {
        int query;
        cin >> query;

        if (query == 1) {
            int x;
            cin >> x;
            bag.insert(x);
        } else if (query == 2) {
            int x;
            cin >> x;
            bag.erase(x);
        } else if (query == 3) {
            cout << bag.size() << endl;
        }
    }

    return 0;
}
```

### Explanation

- We use a `set` named `bag` to store the integers.
- For each query:
  - If it's a type 1 query, we read an integer `x` and insert it into the set.
  - If it's a type 2 query, we read an integer `x` and remove it from the set.
  - If it's a type 3 query, we output the size of the set, which is the count of unique integers.
- This solution efficiently handles all operations within the constraints and provides the correct output for the problem.
