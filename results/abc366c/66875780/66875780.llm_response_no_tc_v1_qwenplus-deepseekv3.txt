The problem requires handling three types of queries on a bag of balls with integers written on them. The main issue with the provided code is that it uses a `set` to keep track of the distinct integers, which works for queries of type 1 and 3 but fails for type 2 when multiple balls with the same integer are present. The `set` only keeps track of the presence or absence of an integer, not the count of each integer. Hence, removing a ball with integer `x` (type 2 query) simply removes `x` from the set, even if there are multiple balls with `x` in the bag.

To fix this, we need to maintain a count of how many balls there are for each integer. We can use a `map` (or `unordered_map` for better performance) to keep track of the counts. Additionally, we'll maintain a separate count of distinct integers, which will be updated whenever the count of an integer transitions between zero and non-zero.

### Fixed Code
```cpp
#include <iostream>
#include <unordered_map>
#include <unordered_set>

using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);
    
    int Q;
    cin >> Q;
    
    unordered_map<int, int> count;
    int distinct = 0;
    
    while (Q--) {
        int type;
        cin >> type;
        
        if (type == 1) {
            int x;
            cin >> x;
            if (count[x] == 0) {
                distinct++;
            }
            count[x]++;
        } else if (type == 2) {
            int x;
            cin >> x;
            count[x]--;
            if (count[x] == 0) {
                distinct--;
            }
        } else if (type == 3) {
            cout << distinct << '\n';
        }
    }
    
    return 0;
}
```

### Explanation
1. **Data Structures**: 
   - `unordered_map<int, int> count` keeps track of the number of balls for each integer `x`.
   - `int distinct` maintains the count of distinct integers currently in the bag.

2. **Queries Handling**:
   - **Type 1 (`1 x`)**: Increment the count of `x`. If this was the first ball with `x`, increment `distinct`.
   - **Type 2 (`2 x`)**: Decrement the count of `x`. If this was the last ball with `x`, decrement `distinct`.
   - **Type 3 (`3`)**: Simply print the value of `distinct`, which is the number of distinct integers in the bag.

This approach ensures that all operations are handled in average O(1) time per query, making it efficient for the given constraints.
