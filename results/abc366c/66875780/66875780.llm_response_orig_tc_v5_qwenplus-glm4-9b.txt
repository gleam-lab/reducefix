- If the query type is `2 x`, it removes the integer `x` from the set using the `erase` method.
- If the query type is `3`, it prints the size of the set using the `size` method.

The problem with the code is that the `insert` and `erase` operations on a `set` can be costly if the set is large, because they require searching for the element in the set and then rebalancing the tree (if the set is implemented as a balanced binary search tree). This could lead to a time complexity of O(N log N) for `Q` insertions and deletions, which may be too slow for the given constraints.

To improve the performance, we can use a `set` but also take advantage of the fact that we only need to know the count of unique items, not their order or existence individually. If we use an efficient hash table data structure like `unordered_set`, we can achieve average-case constant time complexity for insertions and deletions (O(1)). #include <iostream>
#include <unordered_set>
#include <vector>

using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);
    cout.tie(nullptr);

    int Q;
    cin >> Q;
    unordered_set<int> s;
    vector<int> results;

    while (Q--) {
        int query;
        cin >> query;
        if (query == 1) {
            int x;
            cin >> x;
            s.insert(x);
        } else if (query == 2) {
            int x;
            cin >> x;
            s.erase(x);
        } else {
            results.push_back(s.size());
        }
    }

    for (int result : results) {
        cout << result << "\n";
    }

    return 0;
}
