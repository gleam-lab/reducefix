### Explanation
- **Data Structure Change**: The original code used a `std::set`, which provides logarithmic time complexity for insertion, deletion, and lookup. However, since we only need to count unique elements and do not require ordered access or duplicate entries, a `std::unordered_set` is more efficient as it offers average constant time complexity for these operations.
- **Algorithmic Correctness**: The logic remains the same: for type 1 queries, insert the element into the set; for type 2 queries, remove the element from the set; for type 3 queries, output the size of the set.
- **Efficiency**: Using `std::unordered_set` significantly improves the performance, especially for large inputs where the number of operations could reach up to \(2 \times 10^5\).
