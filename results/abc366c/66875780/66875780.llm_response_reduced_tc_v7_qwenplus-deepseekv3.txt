To fix this code, we need to correctly track the counts of each integer in the bag. The initial approach used a set to keep track of distinct integers, which works for the first type of query (inserting a ball) and the third type (counting distinct integers). However, it fails for the second type of query (removing a ball) because it removes the integer from the set entirely, even if there are multiple balls with the same integer in the bag. 

To handle this correctly, we should use a dictionary (like `std::unordered_map`) to keep track of the counts of each integer. When we remove a ball, we decrement its count, and only remove the integer from the set of distinct integers if the count reaches zero. Similarly, when we add a ball, we increment its count, and if the count was zero before, we add the integer to the set of distinct integers.

Here's the corrected code:

```cpp
#include <iostream>
#include <unordered_map>
#include <unordered_set>

using namespace std;

void Solve() {
    int Q;
    cin >> Q;
    unordered_map<int, int> count;
    unordered_set<int> distinct;

    while (Q--) {
        int type;
        cin >> type;
        
        if (type == 1) {
            int x;
            cin >> x;
            if (count[x] == 0) {
                distinct.insert(x);
            }
            count[x]++;
        } else if (type == 2) {
            int x;
            cin >> x;
            count[x]--;
            if (count[x] == 0) {
                distinct.erase(x);
            }
        } else if (type == 3) {
            cout << distinct.size() << "\n";
        }
    }
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);
    cout.tie(nullptr);

    int t = 1;
    while (t--) {
        Solve();
    }

    return 0;
}
```

### Explanation
1. **Data Structures**: We use an `unordered_map<int, int>` to keep track of the count of each integer and an `unordered_set<int>` to keep track of the distinct integers present in the bag.
2. **Query Handling**:
   - For `1 x`: We increment the count of `x` in the map. If the count was zero before, we add `x` to the set of distinct integers.
   - For `2 x`: We decrement the count of `x` in the map. If the count reaches zero, we remove `x` from the set of distinct integers.
   - For `3`: We simply output the size of the set of distinct integers.
3. **Efficiency**: Both insertion and deletion operations in `unordered_map` and `unordered_set` are average O(1), making the overall solution efficient with O(Q) time complexity, where Q is the number of queries. This meets the problem's constraints effectively.
