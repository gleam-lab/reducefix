To fix this code, we need to address the issue where the bag can contain multiple balls with the same integer, but the current implementation using a set only keeps track of unique integers without considering their counts. This causes incorrect behavior when adding and removing balls with the same integer.

### Approaches
1. **Track Counts of Each Integer**: Use a dictionary (like a `map` or `unordered_map`) to keep track of how many times each integer appears in the bag. This allows us to handle multiple balls with the same integer correctly.
2. **Maintain a Set of Unique Integers**: Use a set to efficiently keep track of the unique integers present in the bag. This set will be updated based on the counts from the dictionary.
3. **Process Queries Appropriately**:
   - For `1 x`: Increment the count of `x` in the dictionary. If the count becomes 1, add `x` to the set.
   - For `2 x`: Decrement the count of `x` in the dictionary. If the count becomes 0, remove `x` from the set.
   - For `3`: The size of the set gives the number of different integers in the bag.

### Solution Code
```cpp
#include <iostream>
#include <unordered_map>
#include <unordered_set>

using namespace std;

void Solve() {
    int Q;
    cin >> Q;
    unordered_map<int, int> count;
    unordered_set<int> unique_elements;
    
    while (Q--) {
        int query_type;
        cin >> query_type;
        
        if (query_type == 1) {
            int x;
            cin >> x;
            count[x]++;
            if (count[x] == 1) {
                unique_elements.insert(x);
            }
        } else if (query_type == 2) {
            int x;
            cin >> x;
            count[x]--;
            if (count[x] == 0) {
                unique_elements.erase(x);
            }
        } else if (query_type == 3) {
            cout << unique_elements.size() << "\n";
        }
    }
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);
    
    Solve();
    
    return 0;
}
```

### Explanation
1. **Initialization**: We initialize an `unordered_map` to keep track of the count of each integer in the bag and an `unordered_set` to store the unique integers present.
2. **Processing Queries**:
   - **Type 1 (`1 x`)**: We increment the count of `x` in the map. If this count becomes 1 (indicating a new unique integer), we add `x` to the set.
   - **Type 2 (`2 x`)**: We decrement the count of `x` in the map. If this count reaches 0 (indicating no more instances of `x`), we remove `x` from the set.
   - **Type 3 (`3`)**: The size of the set is printed, representing the number of unique integers in the bag.
3. **Efficiency**: Using `unordered_map` and `unordered_set` ensures average O(1) time complexity for insertions, deletions, and lookups, making the solution efficient for large input sizes. This approach handles all edge cases, including operations on multiple balls with the same integer.
