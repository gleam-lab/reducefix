The given code fails to account for the fact that the bag can contain multiple balls with the same integer. The use of a `set` only keeps track of unique integers, so when a ball is removed, it removes the integer entirely, even if there were multiple balls with that integer. 

To fix this, we need to maintain a count of how many balls there are for each integer. We can use a `map` or a `dictionary` to keep track of the counts of each integer. Additionally, we can maintain a separate count of the number of unique integers, which we will update whenever the count of an integer transitions between zero and non-zero.

### Approaches
1. **Data Structures**: Use a `map` (or `unordered_map` for better performance) to keep track of the counts of each integer in the bag. This allows efficient insertion, deletion, and count updates.
2. **Unique Count Tracking**: Maintain a variable (`unique_count`) that keeps track of the number of unique integers in the bag. This variable is adjusted whenever the count of any integer becomes zero (decrement `unique_count`) or transitions from zero to one (increment `unique_count`).
3. **Query Processing**: For each query:
   - **Type 1 (Insert x)**: Increment the count of `x`. If the count becomes 1, increment `unique_count`.
   - **Type 2 (Remove x)**: Decrement the count of `x`. If the count becomes 0, decrement `unique_count`.
   - **Type 3 (Print unique count)**: Output the value of `unique_count`.

### Solution Code
```cpp
#include<bits/stdc++.h>
using namespace std;

void solve() {
    int q;
    cin >> q;
    unordered_map<int, int> count_map;
    int unique_count = 0;
    for (int i = 0; i < q; i++) {
        int a;
        cin >> a;
        if (a == 1) {
            int x;
            cin >> x;
            count_map[x]++;
            if (count_map[x] == 1) {
                unique_count++;
            }
        } else if (a == 2) {
            int y;
            cin >> y;
            count_map[y]--;
            if (count_map[y] == 0) {
                unique_count--;
            }
        } else if (a == 3) {
            cout << unique_count << endl;
        }
    }
}

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(NULL);

#ifndef ONLINE_JUDGE

    freopen("output.txt", "w", stdout);
#endif

    int t = 1;
    //cin >> t;
    while (t--) {
        solve();
    }
    return 0;
}
```

### Explanation
- **Data Structures**: The `unordered_map<int, int> count_map` is used to store the counts of each integer in the bag. This allows O(1) average time complexity for insertions, deletions, and lookups.
- **Unique Count Tracking**: The variable `unique_count` is maintained to quickly answer queries of type 3 (number of unique integers). It is updated whenever adding a new integer (count becomes 1) or removing the last instance of an integer (count becomes 0).
- **Query Processing**: Each query is processed in O(1) average time:
  - **Insert (Type 1)**: Increment the count of `x`; if it was zero before, increment `unique_count`.
  - **Remove (Type 2)**: Decrement the count of `y`; if it becomes zero, decrement `unique_count`.
  - **Print (Type 3)**: Simply output the current value of `unique_count`.

This approach efficiently handles the constraints, ensuring optimal performance even for the upper limits of input size.
