The solution requires tracking the count of each integer in the bag. We can use a `map` or `unordered_map` to keep track of the counts of each integer. When adding a ball, we increment its count, and if the count becomes 1, we also add it to a set of distinct integers. When removing a ball, we decrement its count, and if the count becomes 0, we remove it from the set. For query type 3, we simply output the size of the set, which gives the number of distinct integers in the bag.

```cpp
#include <bits/stdc++.h>
using namespace std;

void solve() {
    int q;
    cin >> q;
    unordered_map<int, int> counts;
    set<int> distinct;
    
    while (q--) {
        int type;
        cin >> type;
        if (type == 1) {
            int x;
            cin >> x;
            counts[x]++;
            if (counts[x] == 1) {
                distinct.insert(x);
            }
        } else if (type == 2) {
            int x;
            cin >> x;
            counts[x]--;
            if (counts[x] == 0) {
                distinct.erase(x);
            }
        } else if (type == 3) {
            cout << distinct.size() << '\n';
        }
    }
}

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);
    
    solve();
    
    return 0;
}
```

2. **Operations**:
   - **Type 1 (Add x)**:

   - **Type 2 (Remove x)**:

     - If the count of `x` drops to 0, remove `x` from the `distinct` set.
   - **Type 3 (Count distinct)**:
     - Output the size of the `distinct` set, which gives the number of distinct integers in the bag.

3. **Efficiency**:
   - Using an `unordered_map` ensures average O(1) time for insertions and deletions.
