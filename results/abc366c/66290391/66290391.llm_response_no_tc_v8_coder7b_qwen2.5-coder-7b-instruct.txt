In your original code, you were using a `set`, which maintains elements in sorted order and allows logarithmic time complexity for insertion and deletion operations. However, for counting unique elements without needing them sorted, an `unordered_set` would be more efficient as it provides constant average-time complexity for insertions, deletions, and lookups. Additionally, I've added `ios_base::sync_with_stdio(false)` and `cin.tie(nullptr)` for faster input processing.
