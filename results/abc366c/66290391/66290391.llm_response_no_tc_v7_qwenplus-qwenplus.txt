The given code uses a `std::set` to track the integers in the bag. While this correctly handles uniqueness, it fails when trying to remove an element in the case where multiple copies of that element have been added. This is because `std::set` only stores one instance of each value.

To fix this, we need to track how many times each integer has been added (and removed). We can do this using a `std::unordered_map<int, int>` where the key is the number on the ball and the value is the count of such balls currently in the bag.

Here's the corrected C++ code:

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    int Q;
    cin >> Q;

    unordered_map<int, int> counts;
    int uniqueCount = 0;

    for (int i = 0; i < Q; ++i) {
        int type;
        cin >> type;

        if (type == 1) {
            int x;
            cin >> x;
            if (counts[x] == 0) {
                ++uniqueCount;
            }
            ++counts[x];
        } else if (type == 2) {
            int x;
            cin >> x;
            --counts[x];
            if (counts[x] == 0) {
                --uniqueCount;
            }
        } else if (type == 3) {
            cout << uniqueCount << endl;
        }
    }

    return 0;
}
```

### Explanation:
- `counts[x]` keeps track of how many times `x` has been added minus how many times it has been removed.
- When inserting (`type == 1`), if `counts[x] == 0`, we are adding a new distinct integer, so we increment `uniqueCount`.
- When removing (`type == 2`), if `counts[x]` becomes zero after decrementing, it means there are no more balls with `x` in the bag, so we decrement `uniqueCount`.
- For query `type == 3`, we just output `uniqueCount`.

This approach is efficient and meets all constraints, handling up to $2 \times 10^5$ operations quickly.
