The problem requires handling three types of queries on a bag of balls, where each ball has an integer written on it. The operations are: adding a ball, removing a ball, and printing the number of distinct integers in the bag. 

The initial code uses a `set` to track the distinct integers, which is correct for maintaining the count of distinct integers. However, the issue arises when there are multiple balls with the same integer. The `set` only keeps track of whether an integer is present or not, not how many times it appears. When removing a ball with integer `x`, the code removes `x` from the set, which is incorrect if there are still other balls with `x` in the bag. 

To fix this, we need to separately track the count of each integer in the bag. We can use a `map` or `unordered_map` to keep count of each integer. The `set` will then only contain integers that have a count greater than zero. 

Here’s the corrected code:

```cpp
#include <bits/stdc++.h>
using namespace std;
#define rep(i, n) for (int i = 0; i < (n); ++i)
using ll = long long;

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);
    int q;
    cin >> q;
    unordered_map<int, int> count;
    set<int> st;
    rep(i, q) {
        int type, x;
        cin >> type;
        if (type == 1) {
            cin >> x;
            count[x]++;
            if (count[x] == 1) {
                st.insert(x);
            }
        } else if (type == 2) {
            cin >> x;
            count[x]--;
            if (count[x] == 0) {
                st.erase(x);
            }
        } else if (type == 3) {
            cout << st.size() << '\n';
        }
    }
    return 0;
}
```

### Explanation:
1. **Data Structures**:
   - `unordered_map<int, int> count`: Tracks the number of balls for each integer `x`.
   - `set<int> st`: Tracks the distinct integers currently in the bag (those with `count[x] > 0`).

2. **Operations**:
   - **Type 1 (Add a ball)**:
     - Increment the count for `x`.
     - If the count becomes 1, add `x` to the set (since it's now present).
   - **Type 2 (Remove a ball)**:
     - Decrement the count for `x`.
     - If the count drops to 0, remove `x` from the set (since it's no longer present).
   - **Type 3 (Print distinct count)**:
     - Output the size of the set, which is the number of distinct integers in the bag.

This approach ensures that the set always reflects the current distinct integers accurately, even when multiple balls with the same integer are added or removed. The use of `unordered_map` and `set` ensures efficient operations with average time complexity of O(1) for insertions, deletions, and lookups.
