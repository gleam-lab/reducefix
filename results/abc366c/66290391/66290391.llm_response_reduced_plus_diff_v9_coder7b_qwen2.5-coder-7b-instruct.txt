To solve this problem, we need to efficiently handle multiple types of operations on a collection of unique integers. The operations involve adding, removing, and querying the count of distinct integers in the collection. Given the constraints, using a `std::set` is appropriate because it maintains the elements in sorted order and provides logarithmic time complexity for insertions, deletions, and lookups.

Here's the corrected C++ code:

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(0);

    int q;
    cin >> q;
    set<int> s;

    while (q--) {
        int type, x;
        cin >> type;
        if (type == 1) {
            cin >> x;
            s.insert(x);
        } else if (type == 2) {
            cin >> x;
            s.erase(s.find(x));
        } else if (type == 3) {
            cout << s.size() << '\n';
        }
    }

    return 0;
}
```

### Explanation:

1. **Reading Input**: We read the number of queries `q`.
2. **Using Set**: We use a `std::set<int>` to store the integers. Sets automatically maintain the elements in sorted order and provide efficient insertion, deletion, and size retrieval.
3. **Processing Queries**:
   - For type 1 (`1 x`), we insert `x` into the set.
   - For type 2 (`2 x`), we remove `x` from the set. Note that we use `s.find(x)` to get an iterator to `x`, then erase it using `s.erase()`.
   - For type 3 (`3`), we output the size of the set, which represents the number of distinct integers currently stored.

This approach ensures that all operations are performed efficiently, meeting the problem's constraints.
