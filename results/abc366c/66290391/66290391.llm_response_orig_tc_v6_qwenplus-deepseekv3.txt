Here's the fixed solution that uses a `map` to track counts of each integer along with a `set` to track the distinct integers:

```cpp
#include <bits/stdc++.h>
using namespace std;
#define rep(i, n) for (int i = 0; i < (n); ++i)

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);
    
    int q;
    cin >> q;
    unordered_map<int, int> count;
    set<int> distinct;
    
    rep(i, q) {
        int type, x;
        cin >> type;
        if (type == 1) {
            cin >> x;
            count[x]++;
            if (count[x] == 1) {
                distinct.insert(x);
            }
        } else if (type == 2) {
            cin >> x;
            count[x]--;
            if (count[x] == 0) {
                distinct.erase(x);
            }
        } else if (type == 3) {
            cout << distinct.size() << '\n';
        }
    }
}
```

Key changes made:
1. Used `unordered_map` to track the count of each integer in the bag
2. Used `set` to track the distinct integers present in the bag
3. When adding a ball (`type 1`), increment the count and add to the set if it's the first occurrence
4. When removing a ball (`type 2`), decrement the count and remove from the set if it was the last occurrence
5. For query (`type 3`), simply output the size of the set which represents the number of distinct integers
6. Added IO optimizations (`ios_base::sync_with_stdio(false)` and `cin.tie(nullptr)`) for better performance with large input
