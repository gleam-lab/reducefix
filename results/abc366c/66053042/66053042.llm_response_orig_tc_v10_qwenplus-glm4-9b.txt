
### Problem Analysis

The problem involves managing a collection of unique integers in a bag using a series of operations. We are given a number of queries, each of which either adds a new integer to the bag, removes an integer from the bag, or requests the count of unique integers currently in the bag.

### Solution Strategy

To efficiently manage the unique integers and support the operations, we can use a set data structure. This is because sets inherently store only unique elements and provide average O(1) time complexity for insertions, deletions, and lookups.

### Detailed Steps

1. **Initialization**: Use a set to store the integers.
2. **Processing Queries**:
   - For type `1 x`: Add integer `x` to the set if it's not already present.
   - For type `2 x`: Remove integer `x` from the set.
   - For type `3`: Output the size of the set, which represents the count of unique integers.

### Corrected Code

```cpp
#include <iostream>
#include <set>
using namespace std;

int main() {
    set<int> bag;
    int Q;
    cin >> Q;
    
    while (Q--) {
        int type;
        cin >> type;
        
        if (type == 1) {
            int x;
            cin >> x;
            bag.insert(x);
        } else if (type == 2) {
            int x;
            cin >> x;
            bag.erase(x);
        } else if (type == 3) {
            cout << bag.size() << endl;
        }
    }
    
    return 0;
}
```

### Explanation

- **Data Structure**: The set `bag` is used to store unique integers.
- **Query Processing**:
  - For `type 1`, the integer `x` is added to the set using `insert()`. If `x` is already in the set, it is not added again, ensuring that all elements in the set are unique.
  - For `type 2`, the integer `x` is removed from the set using `erase()`. Since the problem guarantees that `x` is always present in the set when this query is given, this operation will always succeed.
  - For `type 3`, the size of the set is printed, which corresponds to the number of unique integers in the bag.

This solution efficiently handles up to 200,000 operations as required by the problem constraints.
