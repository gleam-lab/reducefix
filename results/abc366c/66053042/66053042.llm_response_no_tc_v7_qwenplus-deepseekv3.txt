Here's the corrected code that ensures the map entry is removed when the count reaches zero, maintaining accurate counts of unique integers:

```cpp
#include <bits/stdc++.h>
using namespace std;

int main()
{
  map<int, int> m;
  int Q;
  cin >> Q;
  int n = 0;
  for (int i = 0; i < Q; i++)
  {
    int type;
    cin >> type;

    if (type == 1)
    {
      int x;
      cin >> x;
      if (m[x] == 0)
      {
        n++;
      }
      m[x]++;
    }
    else if (type == 2)
    {
      int x;
      cin >> x;
      m[x]--;
      if (m[x] == 0)
      {
        n--;
      }
    }
    else if (type == 3)
    {
      cout << n << endl;
    }
  }
}
```

Key changes made:
1. Simplified the `type == 1` case to always increment `m[x]` and only increment `n` if the previous count was zero.
2. In the `type == 2` case, after decrementing `m[x]`, check if it becomes zero and decrement `n` accordingly. The entry remains in the map with count zero, but this does not affect future operations because we check `m[x] == 0` in `type == 1` to increment `n` again if needed.

This ensures that the count of unique integers is always accurate, even if the same integer is added and removed multiple times. The time complexity remains efficient with O(log n) per operation for the map operations, which is acceptable given the constraints.
