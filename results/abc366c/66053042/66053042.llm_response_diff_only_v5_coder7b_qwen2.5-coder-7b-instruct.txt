```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
  unordered_set<int> s;

  int Q;
  cin >> Q;
  for (int i = 0; i < Q; i++) {
    int type;
    cin >> type;

    if (type == 1) {
      int x;
      cin >> x;
      s.insert(x); // Insert directly into set
    } else if (type == 2) {
      int x;
      cin >> x;
      s.erase(x); // Erase directly from set
    } else if (type == 3) {
      cout << s.size() << endl; // Output the size of the set
    }
  }

  return 0;
}
```

This solution uses an `unordered_set` instead of a `map`. This change addresses several issues:
1. **Efficiency**: The operations (`insert`, `erase`, `size`) on an `unordered_set` are generally faster than those on a `map` because `unordered_set` does not require storing frequency counts.
2. **Correctness**: Since we only need to count unique elements, an `unordered_set` is more appropriate as it automatically handles duplicates.
3. **Simplicity**: The code becomes simpler and easier to understand without the need for additional logic to manage frequencies.

By using an `unordered_set`, we ensure that each element is stored uniquely and efficiently, making the solution both correct and performant within the given constraints.
