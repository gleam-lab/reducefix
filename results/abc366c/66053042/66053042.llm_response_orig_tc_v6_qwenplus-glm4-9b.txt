### Problem Analysis
The problem involves managing a collection of integers stored in a "bag" and processing a series of queries that involve adding, removing, or querying the unique elements in the bag.

### Data Structures
- **map<int, int> m**: This is used to store the count of each integer in the bag.
- **int n**: This keeps track of the number of unique integers in the bag.

### Operations
1. **Adding an integer (Query type 1)**:
   - If the integer is already in the map, increment its count.
   - If it's not in the map, add it with a count of 1 and increment n.
2. **Removing an integer (Query type 2)**:
   - Decrement the count of the integer in the map.
   - If the count becomes zero, decrement n.
3. **Query unique count (Query type 3)**:
   - Output the current value of n.

### Issues in the Incorrect Code
- The method `m.contains(x)` is not a valid member function of `std::map`. The correct method is `m.find(x) != m.end()`.
- The code uses `n` to keep track of unique elements but does not reset it after processing all queries. This could lead to incorrect results if the program is run multiple times with the same queries.

### Corrected Code
```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    map<int, int> m;
    int n = 0;
    int Q, type, x;

    cin >> Q;
    while (Q--) {
        cin >> type;
        if (type == 1) {
            cin >> x;
            if (m.find(x) == m.end()) {
                m[x] = 1;
                n++;
            } else {
                m[x]++;
            }
        } else if (type == 2) {
            cin >> x;
            m[x]--;
            if (m[x] == 0) {
                m.erase(x);
                n--;
            }
        } else if (type == 3) {
            cout << n << endl;
        }
    }

    return 0;
}
```

### Explanation of Changes
- I replaced `m.contains(x)` with `m.find(x) != m.end()` to check for the presence of an element in the map.
- For query type 2, I used `m.erase(x)` to remove the element from the map if its count reaches zero, which helps in maintaining the accuracy of the unique count `n`.
- I've handled each query inside the loop to ensure that the program processes all queries sequentially and correctly maintains the state of the bag.
