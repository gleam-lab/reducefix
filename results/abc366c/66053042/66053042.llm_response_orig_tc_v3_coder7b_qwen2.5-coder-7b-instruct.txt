To solve this problem efficiently, we need to keep track of the unique numbers added to and removed from the bag. Using a `std::set` will help us maintain the uniqueness and allow us to easily count the distinct elements.

Here's the corrected C++ code:

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);

    set<int> s;
    int Q;
    cin >> Q;

    while (Q--) {
        int type;
        cin >> type;

        if (type == 1) {
            int x;
            cin >> x;
            s.insert(x);
        } else if (type == 2) {
            int x;
            cin >> x;
            s.erase(s.find(x));
        } else if (type == 3) {
            cout << s.size() << '\n';
        }
    }

    return 0;
}
```

### Explanation:
1. **Data Structure**: We use a `std::set<int>` to store the unique elements. The `set` automatically handles duplicates and keeps the elements sorted.
  
2. **Insert Operation** (`type == 1`):
   - Read the value `x`.
   - Insert `x` into the set using `s.insert(x)`.

3. **Remove Operation** (`type == 2`):
   - Read the value `x`.
   - Find the element `x` in the set using `s.find(x)`.
   - Erase the element from the set using `s.erase(...)`.

4. **Query Operation** (`type == 3`):
   - Print the size of the set using `cout << s.size() << '\n'`.

This approach ensures that each operation is efficient:
- `insert` and `erase` operations take \(O(\log n)\) time.
- `size` operation takes \(O(1)\) time.

Thus, the overall complexity is \(O(Q \log n)\), where \(Q\) is the number of queries and \(n\) is the number of unique elements in the bag. This should handle the maximum constraints effectively.
