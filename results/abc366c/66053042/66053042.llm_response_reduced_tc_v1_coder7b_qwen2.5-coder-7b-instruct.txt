```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
  map<int, int> m;
  set<int> s; // Use a set to keep track of unique elements

  int Q;
  cin >> Q;
  
  for (int i = 0; i < Q; i++) {
    int type;
    cin >> type;

    if (type == 1) {
      int x;
      cin >> x;

      if (m.find(x) != m.end()) {
        m[x]++;
      } else {
        m.emplace(x, 1);
        s.insert(x); // Insert into the set as well
      }
    } else if (type == 2) {
      int x;
      cin >> x;

      if (m[x] > 1) {
        m[x]--;
      } else {
        m.erase(x); // Remove from the map
        s.erase(x); // Remove from the set as well
      }
    } else if (type == 3) {
      cout << s.size() << endl; // Output the size of the set
    }
  }

  return 0;
}
```

### Explanation of Changes:
1. **Use of Set**: Instead of maintaining a separate counter (`n`), I used a set (`s`) to store all unique elements. This ensures that the size of the set directly gives us the number of unique elements.
2. **Updating Set**: Whenever an element is added to the map, it is also added to the set. When an element is removed from the map (and its count reaches zero), it is removed from both the map and the set.
3. **Output**: For query type `3`, instead of printing `n`, we print the size of the set (`s.size()`), which gives the number of unique elements.

This approach ensures that the solution is efficient and handles all edge cases correctly. The use of a set allows for O(log N) time complexity for insertions and deletions, making the overall solution efficient even for large inputs.
