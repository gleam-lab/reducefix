To fix this code, we need to ensure that when we remove a ball (query type 2), we correctly update the count of distinct integers in the bag. The issue in the original code arises when decrementing the count of a ball and checking if it reaches zero. The variable `n` (which tracks the number of distinct integers) is decremented too early, before the count actually reaches zero. 

### Approaches
1. **Tracking Counts and Distinct Elements**: Use a map to keep track of the counts of each integer in the bag. 
2. **Maintaining Distinct Counts**: Increment the distinct count `n` only when a new integer is added (count changes from 0 to 1). Decrement the distinct count `n` only when an integer's count drops from 1 to 0 during removal.

### Solution Code
```cpp
#include <bits/stdc++.h>
using namespace std;

int main()
{
  unordered_map<int, int> m;
  int Q;
  cin >> Q;
  int n = 0;
  for (int i = 0; i < Q; i++)
  {
    int type;
    cin >> type;

    if (type == 1)
    {
      int x;
      cin >> x;
      m[x]++;
      if (m[x] == 1) n++;
    }
    else if (type == 2)
    {
      int x;
      cin >> x;
      m[x]--;
      if (m[x] == 0) n--;
    }
    else if (type == 3)
    {
      cout << n << endl;
    }
  }
}
```

### Explanation
1. **Initialization**: We use an `unordered_map` to store the count of each integer present in the bag. The variable `n` keeps track of the number of distinct integers.
2. **Query Handling**:
   - **Type 1 (Add Ball)**: For each integer `x`, increment its count in the map. If the count becomes 1 (indicating it was not present before), increment `n`.
   - **Type 2 (Remove Ball)**: For each integer `x`, decrement its count in the map. If the count drops to 0 (indicating no more instances of `x` are left), decrement `n`.
   - **Type 3 (Print Distinct Count)**: Output the current value of `n`, which represents the number of distinct integers in the bag.
   
This approach ensures that we accurately maintain the count of distinct integers and efficiently respond to each query in constant time on average, leading to an overall time complexity of O(Q).
