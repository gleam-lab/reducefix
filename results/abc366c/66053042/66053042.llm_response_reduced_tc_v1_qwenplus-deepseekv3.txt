To fix this code, we need to address the issue where the count of distinct integers (`n`) is incorrectly decremented when the count of a particular integer `x` reaches zero. The problem arises because the code decrements `n` immediately when `m[x]` becomes zero, but it doesn't account for the fact that `m[x]` might still be in the map with a zero count, which should not affect the distinct count. Instead, we should remove the entry from the map when its count reaches zero to ensure that `n` accurately reflects the number of distinct integers in the bag.

### Approaches
1. **Data Structure**: Use a map (`std::map` or `std::unordered_map`) to keep track of the count of each integer in the bag.
2. **Handling Queries**:
   - For `1 x`: Increment the count of `x` in the map. If `x` was not previously in the map, increment the distinct count `n`.
   - For `2 x`: Decrement the count of `x` in the map. If the count reaches zero, remove the entry from the map and decrement the distinct count `n`.
   - For `3`: Simply print the current value of `n`, which represents the number of distinct integers in the bag.

### Solution Code
```cpp
#include <bits/stdc++.h>
using namespace std;

int main()
{
  unordered_map<int, int> m;
  int Q;
  cin >> Q;
  int n = 0;
  for (int i = 0; i < Q; i++)
  {
    int type;
    cin >> type;

    if (type == 1)
    {
      int x;
      cin >> x;
      if (m[x] == 0)
      {
        n++;
      }
      m[x]++;
    }
    else if (type == 2)
    {
      int x;
      cin >> x;
      m[x]--;
      if (m[x] == 0)
      {
        n--;
        m.erase(x);
      }
    }
    else if (type == 3)
    {
      cout << n << endl;
    }
  }
}
```

### Explanation
1. **Initialization**: We use an `unordered_map` to store the count of each integer in the bag for efficient access and updates. The variable `n` keeps track of the number of distinct integers.
2. **Query Handling**:
   - **Type 1 (`1 x`)**: If `x` is encountered for the first time (count was zero), we increment `n`. We then increase the count of `x` in the map.
   - **Type 2 (`2 x`)**: Decrease the count of `x` in the map. If the count becomes zero, we decrement `n` and remove `x` from the map to ensure `n` remains accurate.
   - **Type 3 (`3`)**: Output the current value of `n`, which represents the number of distinct integers in the bag at that moment.
3. **Efficiency**: The operations on the `unordered_map` (insertions, deletions, and lookups) are average-case O(1), making the solution efficient for large input sizes (up to 200,000 queries). The total time complexity is O(Q), which is optimal for the problem constraints.
