Here's the fixed code that properly handles the removal operation by only decrementing `n` when the count transitions from 1 to 0 (meaning we've removed the last instance of that number):

```cpp
using namespace std;

int main()
{
    map<int, int> m;
    int Q;
    cin >> Q;
    int n = 0;
    for (int i = 0; i < Q; i++)
    {
        int type;
        cin >> type;

        if (type == 1)
        {
            int x;
            cin >> x;
            if (m[x] == 0)
            {
                n++;
            }
            m[x]++;
        }
        else if (type == 2)
        {
            int x;
            cin >> x;
            if (m[x] == 1)
            {
                n--;
            }
            m[x]--;
        }
        else if (type == 3)
        {
            cout << n << endl;
        }
    }
}
```

1. Changed the condition for incrementing `n` in the insertion operation to check if the count was zero before insertion
2. Changed the condition for decrementing `n` in the removal operation to check if the count was one before removal
3. Simplified the logic by always updating the counts (m[x]++) and letting the conditions handle the distinct count

This ensures that we only modify `n` (the count of distinct integers) when we actually add a new unique number or remove the last instance of a number, which is the correct behavior for this problem.
