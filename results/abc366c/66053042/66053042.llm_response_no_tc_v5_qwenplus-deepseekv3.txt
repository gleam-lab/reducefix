```cpp
#include <bits/stdc++.h>
using namespace std;

int main()
{
  map<int, int> m;

  int n = 0;
  for (int i = 0; i < Q; i++)
  {
    int type;
    cin >> type;

    if (type == 1)
    {
      int x;
      cin >> x;
      if (m[x] == 0)
      {
        n++;
      }
      m[x]++;
    }
    else if (type == 2)
    {
      int x;
      cin >> x;
      m[x]--;
      if (m[x] == 0)
      {
        n--;
        m.erase(x);
      }
    }
    else if (type == 3)
    {
      cout << n << endl;
    }
  }
}
```

### Key Fixes:
1. **Initialization of `m[x]`**: In the type `1` query, the code now checks if `m[x]` is zero (i.e., the integer `x` was not present before) to increment `n`. This avoids double-counting when adding the same integer multiple times.
2. **Removing zero-count entries**: In the type `2` query, after decrementing the count of `x`, if the count reaches zero, the code removes the entry `x` from the map `m` and decrements `n`. This ensures that the count of distinct integers is accurate when queried next.

This approach ensures that the map only contains entries with positive counts, and the counter `n` accurately reflects the number of distinct integers in the bag at any point in time.
