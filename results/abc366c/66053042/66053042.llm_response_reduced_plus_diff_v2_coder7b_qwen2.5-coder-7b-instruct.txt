To solve this problem efficiently, we need to maintain a set of unique elements and perform operations like insertion, deletion, and querying the count of distinct elements. The use of a set ensures that each element is stored only once and allows for efficient insertion and deletion operations.

Here's the corrected C++ code:

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    unordered_set<int> s;
    int Q;
    cin >> Q;
    
    while (Q--) {
        int type;
        cin >> type;
        
        if (type == 1) {
            int x;
            cin >> x;
            s.insert(x);
        } else if (type == 2) {
            int x;
            cin >> x;
            s.erase(s.find(x));
        } else if (type == 3) {
            cout << s.size() << '\n';
        }
    }

    return 0;
}
```

### Explanation:
1. **Data Structure**: We use an `unordered_set<int>` to store the elements. An `unordered_set` automatically handles duplicate entries and provides average O(1) time complexity for insertions, deletions, and lookups.
  
2. **Operations**:
   - For `type 1`, we read an integer `x` and insert it into the set using `s.insert(x)`.
   - For `type 2`, we read an integer `x`, find it in the set using `s.find(x)`, and remove it using `s.erase(s.find(x))`. This ensures that we correctly handle the removal of elements.
   - For `type 3`, we output the size of the set using `cout << s.size() << '\n'`.

3. **Efficiency**: The operations are efficient due to the properties of `unordered_set`. Insertion, deletion, and lookup are all on average O(1), making the solution suitable for large inputs within the constraints.
