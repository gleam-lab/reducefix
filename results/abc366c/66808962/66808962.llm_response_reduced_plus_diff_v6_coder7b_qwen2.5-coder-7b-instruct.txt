To solve this problem, we need to efficiently handle three types of operations:
1. Inserting a ball with a specific value.
2. Removing a ball with a specific value.
3. Counting the number of distinct values currently in the bag.

A suitable data structure for these operations is a combination of a set and a map. The set will help us maintain the distinct values and allow for efficient insertion and deletion. The map will keep track of the count of each value, allowing us to quickly check and update the count.

Here's the corrected C++ code:

```cpp
#include <iostream>
#include <set>
#include <unordered_map>

using namespace std;

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);

    unordered_map<int, int> countMap;
    set<int> uniqueValues;

    int q;
    cin >> q;

    while (q--) {
        int type;
        cin >> type;

        if (type == 1) {
            int x;
            cin >> x;
            if (countMap[x] == 0) {
                uniqueValues.insert(x);
            }
            countMap[x]++;
        } else if (type == 2) {
            int x;
            cin >> x;
            countMap[x]--;
            if (countMap[x] == 0) {
                uniqueValues.erase(uniqueValues.find(x));
            }
        } else {
            cout << uniqueValues.size() << endl;
        }
    }

    return 0;
}
```

### Explanation:
1. **Data Structures**:
   - `unordered_map<int, int> countMap`: Keeps track of the count of each value.
   - `set<int> uniqueValues`: Stores all unique values in the bag, ensuring they are sorted and allows for efficient insertion and deletion.

2. **Operations**:
   - **Insert (`type == 1`)**: 
     - Read the value `x`.
     - If `x` is not already in `uniqueValues`, insert it.
     - Increment the count of `x` in `countMap`.

   - **Remove (`type == 2`)**:
     - Read the value `x`.
     - Decrement the count of `x` in `countMap`.
     - If the count becomes zero, remove `x` from `uniqueValues`.

   - **Count Distinct Values (`type == 3`)**:
     - Print the size of `uniqueValues`, which represents the number of distinct values in the bag.

This approach ensures that all operations are handled efficiently, with insertions and deletions from the set and updates in the map both being average O(log N) operations. This meets the requirement of handling up to \(2 \times 10^5\) queries within the time limit.
