To fix this code, we need to ensure that the count of distinct integers in the bag is accurately maintained. The issue in the original code arises when a ball is removed (type 2 query) and the count for that integer drops to zero. The original code decrements the `ans` variable (which tracks the number of distinct integers) when the count reaches zero, but it doesn't account for the fact that removing a ball might not necessarily reduce the count of distinct integers if there are still other balls with the same integer in the bag. However, in the given problem statement, it is guaranteed that when a type 2 query is given, the bag has at least one ball with the specified integer, so the count will always be at least 1 before the removal.

Here's the corrected code that ensures the `ans` variable is updated correctly:

```cpp
#include<iostream>
#include<map>
using namespace std;

int main(){
    map<int,int> m;
    int q;
    cin >> q;

    int ans = 0;
    for(int i = 0; i < q; i++){
        int n, x;
        cin >> n;

        if(n == 1){
            cin >> x;
            if(m[x] == 0) {
                ans++;
            }
            m[x]++;
        } else if(n == 2){
            cin >> x;
            m[x]--;
            if(m[x] == 0) {
                ans--;
            }
        } else {
            cout << ans << endl;
        }
    }
    return 0;
}
```

### Explanation
1. **Initialization**: We use a `map<int, int>` to keep track of the count of each integer in the bag and an integer `ans` to keep track of the number of distinct integers.
2. **Type 1 Query (`1 x`)**:
   - If the count of `x` is zero, it means this is a new integer being added to the bag, so we increment `ans`.
   - We then increment the count of `x` in the map.
3. **Type 2 Query (`2 x`)**:
   - We decrement the count of `x` in the map.
   - If the count of `x` drops to zero, it means there are no more balls with this integer in the bag, so we decrement `ans`.
4. **Type 3 Query (`3`)**:
   - Simply print the current value of `ans`, which represents the number of distinct integers in the bag.

This approach ensures that the count of distinct integers is accurately maintained throughout the operations, handling all edge cases and constraints efficiently.
