
To solve the problem, we can use a `set` instead of a `map` since the problem only requires us to keep track of the unique integers. Adding and removing elements from a `set` has an average time complexity of O(log n), which is efficient enough given the constraints.

Here's an improved version of the code using a `set`:

```cpp
#include <iostream>
#include <set>
using namespace std;

int main() {
    int q;
    cin >> q;
    set<int> s;

    for (int i = 0; i < q; i++) {
        int n;
        cin >> n;

        if (n == 1) {
            int x;
            cin >> x;
            s.insert(x);
        } else if (n == 2) {
            int x;
            cin >> x;
            s.erase(x);
        } else {
            cout << s.size() << endl;
        }
    }

    return 0;
}
```

Explanation:
- We use a `set<int>` named `s` to store unique integers.
- For each query:
  - If `n == 1`, we read the integer `x` and insert it into the set.
  - If `n == 2`, we read the integer `x` and erase it from the set.
  - If `n == 3`, we output the size of the set, which represents the number of unique integers in the bag.

This solution is efficient and meets the problem's requirements.
