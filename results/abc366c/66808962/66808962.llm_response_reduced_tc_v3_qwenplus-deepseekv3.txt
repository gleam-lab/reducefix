To fix this code, we need to ensure that the count of distinct integers is correctly updated when adding and removing balls. The issue in the original code arises when removing a ball: it decrements the count of distinct integers prematurely, before checking if the count of that integer has actually reached zero. This leads to incorrect tracking of the number of distinct integers.

```cpp
#include<iostream>
#include<map>
using namespace std;

int main(){
    map<int, int> m;
    int q;
    cin >> q;
    int ans = 0;
    
    for(int i = 0; i < q; i++){
        int n, x;
        cin >> n;
        
        if(n == 1){
            cin >> x;
            m[x]++;
            if(m[x] == 1) ans++; // Only increment if it's the first occurrence
        } else if(n == 2){
            cin >> x;
            m[x]--;
            if(m[x] == 0) ans--; // Only decrement if it's the last occurrence
        } else {
            cout << ans << endl;
        }
    }
    return 0;
}
```

### Explanation:
1. **Initialization**: We use a `map<int, int>` to keep track of the count of each integer in the bag. The variable `ans` keeps track of the number of distinct integers.
2. **Processing Queries**:
   - **Type 1 (`1 x`)**: Increment the count of `x` in the map. If the count becomes 1 (indicating this is the first occurrence of `x`), increment `ans`.
   - **Type 2 (`2 x`)**: Decrement the count of `x` in the map. If the count reaches 0 (indicating no more balls of `x` are left), decrement `ans`.
   - **Type 3 (`3`)**: Output the current value of `ans`, which represents the number of distinct integers in the bag.
3. **Correctness**: By updating `ans` only when the count of an integer transitions between 0 and 1 (or vice versa), we ensure that `ans` accurately reflects the number of distinct integers at any point.
