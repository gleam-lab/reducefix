To solve this problem efficiently, we need to keep track of the distinct numbers in the bag while processing the queries. The key operations are adding a number to the bag, removing a number from the bag, and counting the number of distinct numbers in the bag.

Given the constraints, using a set can help because sets automatically maintain the elements in sorted order and provide efficient insertions and deletions. Additionally, sets allow us to check for the existence of an element in constant time.

Here's the corrected C++ code:

```cpp
#include<iostream>
#include<set>
using namespace std;

int main(){
    set<int> s;
    int q;
    cin >> q;

    for(int i = 0; i < q; ++i){
        int n, x;
        cin >> n;

        if(n == 1){
            cin >> x;
            s.insert(x);
        } else if(n == 2){
            cin >> x;
            s.erase(s.find(x));
        } else {
            cout << s.size() << endl;
        }
    }

    return 0;
}
```

### Explanation:
1. **Set Initialization**: We initialize a set `s` to store the unique numbers in the bag.
2. **Query Processing**:
   - For `n == 1`, we read the number `x` and insert it into the set `s`. If `x` is already in the set, the insertion does nothing but ensures `x` remains in the set.
   - For `n == 2`, we read the number `x` and remove it from the set `s`. The `find` function locates the element in the set, and `erase` removes it.
   - For `n == 3`, we simply output the size of the set `s`, which represents the number of distinct integers in the bag.
3. **Efficiency**: 
   - Insertion (`insert`) and deletion (`erase`) operations on a set take \(O(\log k)\) time, where \(k\) is the current number of elements in the set.
   - Checking the size (`size`) operation takes \(O(1)\) time.

This approach ensures that all operations are performed efficiently and correctly, handling up to \(2 \times 10^5\) queries as required.
