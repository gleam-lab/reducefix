To fix this code, we need to address the issue where the count of distinct integers is incorrectly being decremented when the count of a specific integer reaches zero. The problem arises because the code decrements the `ans` variable whenever a ball's count reaches zero, even though the distinct count should only be decremented if the count was exactly one before the operation (i.e., the integer is no longer present in the bag).

### Approaches
1. **Use a Map to Track Counts**: We will use a map to keep track of the count of each integer in the bag.
2. **Track Distinct Integers**: We will maintain a variable `ans` that keeps track of the number of distinct integers in the bag.
   - For a `1 x` query, if `x` was not previously in the map (count was zero), increment `ans`.
   - For a `2 x` query, decrement the count of `x`. If the count reaches zero, decrement `ans` because `x` is no longer present in the bag.
3. **Output the Result**: For each `3` query, simply print the current value of `ans`.

### Solution Code
```cpp
#include<iostream>
#include<map>
using namespace std;

int main(){
    map<int,int> m;
    int q;
    cin >> q;

    int ans = 0;
    for(int i = 0; i < q; i++){
        int n, x;
        cin >> n;

        if(n == 1){
            cin >> x;
            if(m[x] == 0){
                ans++;
            }
            m[x]++;
        } else if(n == 2){
            cin >> x;
            m[x]--;
            if(m[x] == 0){
                ans--;
            }
        } else {
            cout << ans << endl;
        }
    }
    return 0;
}
```

### Explanation
- **Initialization**: We initialize a map `m` to keep track of the count of each integer and `ans` to zero to start with no distinct integers.
- **Processing Queries**:
  - For `1 x` queries, we check if the count of `x` is zero before incrementing. If it is, we increment `ans` because `x` is a new distinct integer in the bag. We then increment the count of `x` in the map.
  - For `2 x` queries, we decrement the count of `x` in the map. If the count reaches zero, we decrement `ans` because `x` is no longer present in the bag.
  - For `3` queries, we simply print the current value of `ans`, which represents the number of distinct integers in the bag.
This approach ensures that `ans` accurately reflects the count of distinct integers at all times, addressing the issue in the original code where `ans` was decremented incorrectly. The solution handles all edge cases and constraints efficiently.
