```cpp
#include<iostream>
#include<map>
using namespace std;

int main(){
    map<int,int> m;
    int q;
    cin >> q;

    int ans=0;
    for(int i=0;i<q;i++){
        int n, x;
        cin >> n;

        if(n==1){
            cin >> x;
            if(m[x] == 0) { // Only increment ans if it's the first occurrence of x
                ans++;
            }
            m[x]++;
        }else if(n==2){
            cin >> x;
            m[x]--;
            if(m[x] == 0) { // Only decrement ans if the count of x drops to zero
                ans--;
            }
        }else{
            cout << ans << endl;
        }
    }
    return 0;
}
```

### Explanation
1. **Initialization**: We use a `map<int, int>` to keep track of the count of each integer in the bag.
2. **Query Handling**:
   - **Type 1 (Add x)**: When adding a ball with integer `x`, we check if `x` was previously not in the map (count is 0). If so, we increment the unique count `ans`. We then increment the count of `x` in the map.
   - **Type 2 (Remove x)**: When removing a ball with integer `x`, we decrement the count of `x` in the map. If the count of `x` reaches zero, we decrement the unique count `ans`.
   - **Type 3 (Print unique count)**: Simply output the current value of `ans`, which maintains the count of different integers in the bag.
3. **Edge Cases**:
   - **Removing a ball that was never added**: As per the problem statement, this scenario is guaranteed not to occur.
   - **Adding a ball after removing all instances**: The count of `x` will correctly transition from 0 to 1, and `ans` will be adjusted accordingly.

This approach ensures that the unique count is accurately maintained throughout all operations, providing the correct output for all test cases.
