To solve this problem, we need to efficiently handle three operations on a collection of unique numbers:

1. **Add a number**: Insert a new number into the collection.
2. **Remove a number**: Remove a specific number from the collection.
3. **Count distinct numbers**: Determine how many unique numbers are currently in the collection.

Given the constraints, using a set is an ideal choice because sets automatically maintain their elements in sorted order and provide efficient insertion, deletion, and size queries.

Here's the corrected and optimized C++ code:

```cpp
#include<iostream>
#include<set>

int main(){
    std::ios_base::sync_with_stdio(false);
    std::cin.tie(NULL);

    std::set<int> s;
    int q;
    std::cin >> q;

    while(q--){
        int n, x;
        std::cin >> n;

        if(n == 1){
            std::cin >> x;
            s.insert(x); // Add element to the set
        } else if(n == 2){
            std::cin >> x;
            s.erase(s.find(x)); // Remove element from the set
        } else {
            std::cout << s.size() << "\n"; // Output the number of unique elements
        }
    }

    return 0;
}
```

### Explanation:
- **Set Usage**: We use a `std::set<int>` to store the numbers. Sets automatically ensure that all elements are unique and maintain them in sorted order.
  
- **Insert Operation (`n == 1`)**: 
  - We read the number `x`.
  - We insert `x` into the set `s`.

- **Delete Operation (`n == 2`)**:
  - We read the number `x`.
  - We find the position of `x` in the set using `s.find(x)` and remove it using `s.erase()`.

- **Query Operation (`n == 3`)**:
  - We simply output the size of the set `s`, which represents the number of unique elements.

### Complexity Analysis:
- **Insertion and Deletion**: Both operations take \(O(\log N)\) time where \(N\) is the number of elements in the set.
- **Size Query**: This operation takes \(O(1)\) time.

This approach ensures that the solution is both efficient and handles the constraints provided in the problem statement.
