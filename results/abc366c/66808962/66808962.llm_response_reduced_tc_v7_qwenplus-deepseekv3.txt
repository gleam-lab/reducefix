To fix this code, we need to address the issue where the count of different integers (ans) is incorrectly decremented when a ball's count reaches zero. The problem arises because the code decrements `ans` immediately when a ball's count drops to zero, but it doesn't account for the fact that the ball might still be in the bag if its count was more than one before the removal.

### Approaches
1. **Tracking Ball Counts**: Use a map to keep track of the count of each integer in the bag.
2. **Tracking Unique Integers**: Maintain a variable `ans` that keeps track of the number of different integers in the bag. This variable should be incremented when a new integer is added (count goes from 0 to 1) and decremented when an integer's count drops from 1 to 0 (meaning there are no more balls with that integer in the bag).

### Solution Code
```cpp
#include<iostream>
#include<map>
using namespace std;

int main() {
    map<int, int> m;
    int q;
    cin >> q;
    int ans = 0;

    for (int i = 0; i < q; i++) {
        int n, x;
        cin >> n;

        if (n == 1) {
            cin >> x;
            m[x]++;
            if (m[x] == 1) { // Only increment ans if it was zero before adding
                ans++;
            }
        } else if (n == 2) {
            cin >> x;
            m[x]--;
            if (m[x] == 0) { // Only decrement ans if it's now zero
                ans--;
            }
        } else {
            cout << ans << endl;
        }
    }

    return 0;
}
```

### Explanation
1. **Initialization**: We use a map `m` to keep track of how many balls of each integer `x` are present in the bag. The variable `ans` is initialized to zero and will be used to count the number of distinct integers in the bag.
2. **Handling Queries**:
   - **Insertion (`1 x`)**:
     - The count of `x` in the map is incremented.
     - If this count was zero before the increment (i.e., `m[x]` becomes 1), `ans` is incremented because a new distinct integer is added to the bag.
   - **Removal (`2 x`)**:
     - The count of `x` in the map is decremented.
     - If this count reaches zero after the decrement, `ans` is decremented because there are no more balls with the integer `x` in the bag.
   - **Query (`3`)**:
     - The current value of `ans` is printed, which represents the number of distinct integers in the bag at that moment.
3. **Efficiency**: The operations on the map (insertion, deletion, and lookup) are O(log N) per operation, and there are Q operations, making the overall complexity O(Q log N), which is efficient given the constraints.
