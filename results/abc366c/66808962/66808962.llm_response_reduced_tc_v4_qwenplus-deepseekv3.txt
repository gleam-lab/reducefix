To fix this code, we need to address the issue where the count of different integers (`ans`) is incorrectly decremented when a ball type is removed completely. Specifically, the problem arises in the `else if(n==2)` block where `ans` is decremented immediately after decrementing the count of `x` in the map, even though the count might not have reached zero yet. This leads to incorrect tracking of the number of different integers in the bag.

### Approaches
1. **Track Counts Correctly**: Use a dictionary (or map) to keep track of the counts of each integer in the bag.
2. **Update Distinct Count Properly**: Only update the count of distinct integers (`ans`) when the count of an integer transitions between zero and one, or vice versa. Specifically:
   - When adding a ball (`n==1`), increment `ans` only if the count of `x` was zero before insertion.
   - When removing a ball (`n==2`), decrement `ans` only if the count of `x` becomes zero after removal.

### Solution Code
```cpp
#include<iostream>
#include<map>
using namespace std;

int main() {
    map<int, int> m;
    int q;
    cin >> q;
    int ans = 0;

    for (int i = 0; i < q; i++) {
        int n, x;
        cin >> n;

        if (n == 1) {
            cin >> x;
            if (m[x] == 0) {
                ans++;
            }
            m[x]++;
        } else if (n == 2) {
            cin >> x;
            m[x]--;
            if (m[x] == 0) {
                ans--;
            }
        } else {
            cout << ans << endl;
        }
    }
    return 0;
}
```

### Explanation
- **Initialization**: A map `m` is used to keep track of the count of each integer in the bag. The variable `ans` keeps track of the number of distinct integers.
- **Query Processing**:
  - **Type 1 (`1 x`)**: When inserting a ball with integer `x`, if `x` was not present in the map (count was zero), we increment `ans`. Then, we increment the count of `x` in the map.
  - **Type 2 (`2 x`)**: When removing a ball with integer `x`, we decrement its count in the map. If the count reaches zero, we decrement `ans` because `x` is no longer in the bag.
  - **Type 3 (`3`)**: The current value of `ans` (number of distinct integers) is printed.
This ensures that `ans` accurately reflects the number of distinct integers in the bag at any point, leading to correct results for all query types.
