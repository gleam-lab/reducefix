```cpp
#include<iostream>
#include<map>
using namespace std;

int main(){
    map<int, int> m;
    int q;
    cin >> q;
    int distinct_count = 0;

    for(int i = 0; i < q; i++){
        int query_type, x;
        cin >> query_type;

        if(query_type == 1){
            cin >> x;
            if(m[x] == 0){
                distinct_count++;
            }
            m[x]++;
        }else if(query_type == 2){
            cin >> x;
            m[x]--;
            if(m[x] == 0){
                distinct_count--;
            }
        }else if(query_type == 3){
            cout << distinct_count << endl;
        }
    }
    return 0;
}
```

### Key Corrections:
1. **Initialization and Tracking**: The variable `distinct_count` is initialized to 0 and is used to keep track of the number of distinct integers in the bag.
2. **Adding Balls**: When a ball with integer `x` is added, if `x` was not previously in the map (i.e., `m[x] == 0`), `distinct_count` is incremented.
3. **Removing Balls**: When a ball with integer `x` is removed, the count for `x` is decremented. If this count reaches zero, `distinct_count` is decremented.
4. **Query Handling**: The third type of query simply prints the current value of `distinct_count`.

This approach ensures that the count of distinct integers is accurately maintained throughout all operations, addressing the issue in the original code where the count could be incorrectly decremented before checking the removal condition. The complexity remains efficient, with each operation being O(log N) due to the use of a map, which is suitable given the constraints.
