```cpp
#include<iostream>
#include<vector>
#include<string>
#include<set>
#include<algorithm>
#include<map>
#include<cmath>
#include<math.h>
#include<queue>
#include <iomanip> //setprecisionを使うため
using namespace std;
using ll = long long;
using ull = unsigned long long;

int main(){
    set<int> s; // Use set instead of map for better performance
    int q;
    cin >> q;

    for(int i=0;i<q;i++){
        int n,x;
        cin >> n;

        if(n==1){
            cin >> x;
            s.insert(x); // Insert the ball directly into the set
        }else if(n==2){
            cin >> x;
            s.erase(s.find(x)); // Erase the ball from the set
        }else{
            cout << s.size() << endl; // The size of the set represents the number of unique integers
        }
    }

    return 0;
}
```

### Explanation
- **Data Structure Change**: Changed from using a `map` to a `set`. A `set` automatically handles uniqueness and provides efficient insertion, deletion, and lookup operations, making it more suitable for this problem where we need to track unique elements efficiently.
- **Simplified Operations**: Removed the count mechanism and directly used the properties of a `set` to manage the presence of elements.
- **Efficiency**: This change improves the overall efficiency of the solution, especially as the number of queries increases. The time complexity for insertions and deletions in a `set` is O(log N), while using a `map` would require O(1) space but O(N log N) for these operations.
