The problem involves processing queries for a bag of balls with integers written on them. The operations include adding a ball, removing a ball, and querying the number of distinct integers in the bag. The initial approach uses a `set` to keep track of distinct integers, which is correct for counting distinct integers but fails to account for multiple occurrences of the same integer. 

### Issues in the Original Code
1. **Multiple Occurrences Handling**: The original code uses a `set` to track distinct integers. However, when multiple balls with the same integer are added, the `set` only stores the integer once. This means that removing a ball with the same integer will remove it entirely from the `set`, even if there are other balls with that integer still in the bag.
2. **Efficiency**: The original approach is efficient for distinct counts but incorrect for handling multiple occurrences of the same integer.

### Solution Approach
To correctly handle multiple occurrences of the same integer, we need to:
1. **Track Counts**: Use a `map` or a `dictionary` (like `unordered_map` in C++) to keep track of the count of each integer in the bag.
2. **Distinct Count**: Maintain a separate count of distinct integers. This count increases when a new integer is added (count goes from 0 to 1) and decreases when an integer's count drops to 0 (removed entirely).

### Fixed Code
```cpp
#include<bits/stdc++.h>

using namespace std;

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);

    int q;
    cin >> q;

    unordered_map<int, int> count;
    int distinct = 0;
    while (q--) {
        int type;
        cin >> type;
        if (type == 1) {
            int x;
            cin >> x;
            if (count[x] == 0) {
                distinct++;
            }
            count[x]++;
        } else if (type == 2) {
            int x;
            cin >> x;
            if (count[x] == 1) {
                distinct--;
            }
            count[x]--;
        } else {
            cout << distinct << '\n';
        }
    }
    return 0;
}
```

### Explanation
1. **Data Structures**: An `unordered_map` is used to count occurrences of each integer. This allows O(1) average time complexity for insertions, deletions, and lookups.
2. **Distinct Count Handling**: 
   - **Insertion (`1 x`)**: When adding a ball with integer `x`, if `x` was not previously in the map (count is 0), the distinct count is incremented. The count for `x` is then incremented.
   - **Removal (`2 x`)**: When removing a ball with integer `x`, if the count of `x` drops to 1 before removal, the distinct count is decremented (since this will be the last occurrence of `x`). The count for `x` is then decremented.
   - **Query (`3`)**: The current distinct count is printed, which is maintained dynamically during insertions and removals.
3. **Efficiency**: The operations are optimized to O(1) average time per query, making the solution efficient for large input sizes (Q up to 2×10^5). The use of `unordered_map` ensures fast access and updates, while `ios_base::sync_with_stdio(false)` and `cin.tie(nullptr)` optimize input/output speed for large datasets.
