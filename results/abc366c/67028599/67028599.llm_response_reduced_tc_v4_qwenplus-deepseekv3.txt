To fix this code, we need to accurately track the count of each integer in the bag, not just the presence or absence of the integer. The initial approach using a `set` only tracks whether an integer is present at least once, which fails when multiple balls with the same integer are added and removed. 

The correct approach involves using a `map` or `unordered_map` to keep track of the count of each integer. This allows us to:
1. Increment the count when a ball is added.
2. Decrement the count when a ball is removed, and remove the entry if the count reaches zero.
3. The size of the map (number of keys) gives the number of different integers in the bag at any time.

Here's the corrected code:

```cpp
#include <bits/stdc++.h>

using namespace std;

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);

    int q;
    cin >> q;

    unordered_map<int, int> count;
    int distinct = 0;

    while (q--) {
        int type;
        cin >> type;
        if (type == 1) {
            int x;
            cin >> x;
            if (count[x] == 0) {
                distinct++;
            }
            count[x]++;
        } else if (type == 2) {
            int x;
            cin >> x;
            count[x]--;
            if (count[x] == 0) {
                distinct--;
            }
        } else {
            cout << distinct << '\n';
        }
    }

    return 0;
}
```

### Explanation
1. **Data Structure**: We use an `unordered_map<int, int>` to keep track of the count of each integer in the bag. The key is the integer written on the ball, and the value is the count of such balls.
2. **Tracking Distinct Integers**: We maintain a variable `distinct` that keeps track of the number of distinct integers in the bag. This is more efficient than calculating the size of the map each time (though the map size could also be used, it's clearer to track it separately).
3. **Operations**:
   - **Insert (Type 1)**: When a ball with integer `x` is added, we increment its count in the map. If the count was zero before, we increment the `distinct` counter.
   - **Remove (Type 2)**: When a ball with integer `x` is removed, we decrement its count in the map. If the count reaches zero, we decrement the `distinct` counter.
   - **Query (Type 3)**: We simply output the current value of `distinct`, which represents the number of different integers in the bag.
4. **Efficiency**: The operations are O(1) on average for each query, making the overall complexity O(Q), which is optimal for the problem constraints.
