The original code correctly handles queries of type 1 and 3 by maintaining a set of unique integers. However, the problem arises with query type 2, where the code removes all occurrences of `x` from the set, but the problem requires removing only one ball with the integer `x`. If there are multiple balls with the same integer, the set will lose all of them at once, which is incorrect.

To fix this, we need to track the count of each integer in the bag. We can use a dictionary (like a `map` or `unordered_map`) to keep track of the counts of each integer. Additionally, we can maintain a set to keep track of the unique integers present in the bag. 

Here's the corrected code:

```cpp
#include<bits/stdc++.h>

using namespace std;

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(NULL);

    int q;
    cin >> q;

    unordered_map<int, int> count;
    set<int> unique;
    int type, x;
    while (q--) {
        cin >> type;
        if (type == 1) {
            cin >> x;
            count[x]++;
            if (count[x] == 1) {
                unique.insert(x);
            }
        } else if (type == 2) {
            cin >> x;
            count[x]--;
            if (count[x] == 0) {
                unique.erase(x);
            }
        } else {
            cout << unique.size() << '\n';
        }
    }

    return 0;
}
```

### Explanation:
1. **Data Structures Used**:
   - `unordered_map<int, int> count`: Keeps track of the count of each integer in the bag.
   - `set<int> unique`: Maintains the unique integers currently in the bag.

2. **Handling Queries**:
   - **Type 1 (`1 x`)**: Increment the count of `x` in the map. If the count becomes 1 (meaning it was just added), insert `x` into the set of unique integers.
   - **Type 2 (`2 x`)**: Decrement the count of `x`. If the count drops to 0, remove `x` from the set of unique integers.
   - **Type 3 (`3`)**: Simply output the size of the set of unique integers, which gives the number of different integers in the bag.

This approach ensures that we correctly handle multiple balls with the same integer and efficiently maintain the count of unique integers for query type 3. The use of `unordered_map` and `set` provides efficient operations with average time complexity of O(1) for insertions, deletions, and lookups.
