The original code uses a `set` to keep track of the distinct integers in the bag. While this approach correctly handles the distinct count (query type 3), it fails to account for multiple occurrences of the same integer. Specifically, when removing a ball with integer `x` (query type 2), the code removes all occurrences of `x` from the set, which is incorrect if there are multiple balls with the same integer. Instead, we need to track the count of each integer and only remove `x` from the set of distinct integers when its count drops to zero.

### Analysis
1. **Query Type 1 (`1 x`)**: Insert a ball with integer `x`. We need to increment the count of `x`. If the count was zero before this operation, we also add `x` to the set of distinct integers.
2. **Query Type 2 (`2 x`)**: Remove a ball with integer `x`. We decrement the count of `x`. If the count becomes zero after this operation, we remove `x` from the set of distinct integers.
3. **Query Type 3 (`3`)**: The size of the set of distinct integers is the answer.

### Solution Code
```cpp
#include<bits/stdc++.h>

using namespace std;

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);

    int q;
    cin >> q;

    unordered_map<int, int> count;
    set<int> distinct;
    int type, x;
    while (q--) {
        cin >> type;
        if (type == 1) {
            cin >> x;
            count[x]++;
            if (count[x] == 1) {
                distinct.insert(x);
            }
        } else if (type == 2) {
            cin >> x;
            count[x]--;
            if (count[x] == 0) {
                distinct.erase(x);
            }
        } else {
            cout << distinct.size() << '\n';
        }
    }

    return 0;
}
```

### Explanation
1. **Initialization**: We use an `unordered_map` to keep track of the count of each integer and a `set` to maintain the distinct integers.
2. **Query Handling**:
   - For `1 x`, increment the count of `x` in the map. If the count becomes 1, insert `x` into the set.
   - For `2 x`, decrement the count of `x` in the map. If the count drops to 0, remove `x` from the set.
   - For `3`, the size of the set gives the number of distinct integers in the bag, which is printed.
3. **Efficiency**: The operations on the `unordered_map` and `set` are average O(1) and O(log n) respectively, making the solution efficient for the given constraints. The use of `ios_base::sync_with_stdio(false)` and `cin.tie(nullptr)` speeds up input/output operations.
