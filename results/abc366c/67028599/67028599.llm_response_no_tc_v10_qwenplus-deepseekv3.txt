The issue with the original code is that it uses a `set` to keep track of the different integers, which correctly handles the uniqueness of the integers. However, the problem requires handling multiple balls with the same integer. The `set` only keeps one instance of each integer, so when you remove a ball with integer `x` using `h.erase(x)`, it removes all instances of `x`, which is incorrect if there were multiple balls with the same `x`. 

To fix this, we need to track the count of each integer in the bag. We can use a `map` or an array (since `x` is up to 10^6, an array is feasible) to keep the counts. Additionally, we need to track the number of distinct integers, which can be done by incrementing a counter when a count goes from 0 to 1 and decrementing when it goes from 1 to 0.

Here’s the corrected code:

```cpp
#include<bits/stdc++.h>
using namespace std;

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(NULL);

    int q;
    cin >> q;

    unordered_map<int, int> count;
    int distinct = 0;
    int type, x;
    while(q--) {
        cin >> type;
        if (type == 1) {
            cin >> x;
            if (count[x] == 0) {
                distinct++;
            }
            count[x]++;
        } else if (type == 2) {
            cin >> x;
            count[x]--;
            if (count[x] == 0) {
                distinct--;
            }
        } else {
            cout << distinct << '\n';
        }
    }

    return 0;
}
```

### Explanation:
1. **Data Structure**: We use an `unordered_map` to keep track of the count of each integer in the bag. This allows O(1) average time complexity for insertions, deletions, and lookups.
2. **Distinct Counter**: We maintain a `distinct` counter that keeps track of the number of distinct integers in the bag. This counter is incremented when a new integer is added (count transitions from 0 to 1) and decremented when an integer's count drops to 0.
3. **Efficiency**: The operations are O(1) on average per query, making the solution efficient for the given constraints (Q up to 2 × 10^5).
4. **Output**: For each type 3 query, we simply print the current value of `distinct`.

This solution correctly handles multiple balls with the same integer and efficiently answers the queries.
