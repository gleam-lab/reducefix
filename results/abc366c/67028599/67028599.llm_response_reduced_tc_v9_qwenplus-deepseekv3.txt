To fix this code, we need to properly handle the counting of balls in the bag, especially when there are multiple balls with the same integer. The initial approach using a `set` only tracks unique integers, but it doesn't account for the actual count of each integer. For example, if we have two balls with the same integer, the `set` will still only show one unique integer, but if we remove one, the count should still reflect that there's one left. 

To handle this correctly, we need to use a `multiset` or maintain a count of each integer separately. Here, we'll use a `map` to keep track of the count of each integer in the bag. This way, when we add a ball, we increment its count, and when we remove a ball, we decrement its count. The number of different integers is then the number of keys in the map that have a count greater than zero.

### Approaches
1. **Data Structure**: Use a `map` (or `unordered_map` for better performance) to count occurrences of each integer in the bag.
2. **Handling Queries**:
   - **Type 1 (Add x)**: Increment the count of `x` in the map.
   - **Type 2 (Remove x)**: Decrement the count of `x` in the map. If the count reaches zero, remove the entry from the map to maintain accurate counts.
   - **Type 3 (Print unique count)**: The number of unique integers is the size of the map (number of keys with counts > 0).

### Solution Code
```cpp
#include<bits/stdc++.h>

using namespace std;

int main() {
    int q;
    cin >> q;
    
    map<int, int> count;
    int unique_count = 0;
    
    while (q--) {
        int type;
        cin >> type;
        
        if (type == 1) {
            int x;
            cin >> x;
            if (count[x] == 0) {
                unique_count++;
            }
            count[x]++;
        } else if (type == 2) {
            int x;
            cin >> x;
            if (count[x] == 1) {
                unique_count--;
            }
            count[x]--;
            if (count[x] == 0) {
                count.erase(x);
            }
        } else if (type == 3) {
            cout << unique_count << endl;
        }
    }
    
    return 0;
}
```

### Explanation
- **Data Structure**: The `map<int, int> count` keeps track of how many times each integer `x` has been added to the bag. The key is the integer `x`, and the value is its count.
- **Type 1 Query (Add x)**: When adding a ball with integer `x`, we increment its count in the map. If this was the first occurrence, we also increment the `unique_count`.
- **Type 2 Query (Remove x)**: When removing a ball with integer `x`, we decrement its count in the map. If this was the last occurrence, we decrement the `unique_count` and remove the entry from the map to avoid counting zeros.
- **Type 3 Query (Print unique count)**: Simply output the `unique_count`, which is the number of distinct integers currently in the bag with counts greater than zero.

This approach ensures that we efficiently handle all operations in logarithmic time (due to the map operations) and correctly maintain the count of unique integers for each query.
