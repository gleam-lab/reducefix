The initial code provided by the user uses a `set` to keep track of the distinct integers in the bag. While this approach correctly handles queries of type 1 (insert) and type 3 (count distinct elements), it fails to correctly handle queries of type 2 (remove). Specifically, the code removes all occurrences of `x` from the set when a type 2 query is encountered, which is incorrect because the bag may still contain other balls with the integer `x` after the removal. 

To fix this, we need to track the count of each integer in the bag. We can use a `map` or `unordered_map` to keep track of the counts. When a type 1 query is encountered, we increment the count for `x`. When a type 2 query is encountered, we decrement the count for `x`. If the count reaches zero, we remove `x` from the set of distinct integers. For type 3 queries, we simply report the size of the set, which now accurately reflects the number of distinct integers in the bag.

Here is the corrected code:

```cpp
#include<bits/stdc++.h>

using namespace std;

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);

    int q;
    cin >> q;

    unordered_map<int, int> counts;
    set<int> distinct;
    
    while (q--) {
        int type;
        cin >> type;
        if (type == 1) {
            int x;
            cin >> x;
            if (counts[x]++ == 0) {
                distinct.insert(x);
            }
        } else if (type == 2) {
            int x;
            cin >> x;
            if (--counts[x] == 0) {
                distinct.erase(x);
            }
        } else {
            cout << distinct.size() << '\n';
        }
    }

    return 0;
}
```

### Explanation
1. **Data Structures**: 
   - `unordered_map<int, int> counts` keeps track of how many times each integer `x` appears in the bag.
   - `set<int> distinct` maintains the distinct integers currently in the bag.
   
2. **Handling Queries**:
   - **Type 1 (`1 x`)**:
     - Increment the count for `x` in the `counts` map.
     - If this was the first occurrence of `x`, add `x` to the `distinct` set.
   - **Type 2 (`2 x`)**:
     - Decrement the count for `x` in the `counts` map.
     - If the count for `x` drops to zero, remove `x` from the `distinct` set.
   - **Type 3 (`3`)**:
     - Output the size of the `distinct` set, which is the number of distinct integers in the bag.

3. **Efficiency**:
   - Each operation (insert, erase, count) on the `set` and `unordered_map` is O(1) average time, making the solution efficient for the given constraints.
   - The use of `ios_base::sync_with_stdio(false)` and `cin.tie(nullptr)` speeds up input and output operations, which is crucial for handling large input sizes efficiently.
