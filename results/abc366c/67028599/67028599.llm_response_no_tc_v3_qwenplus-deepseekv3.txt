The problem requires handling three types of queries on a bag of balls, where each ball has an integer written on it. The operations include adding a ball, removing a ball, and printing the number of distinct integers in the bag. The initial code uses a `set` to keep track of the distinct integers, which is correct for counting distinct integers, but it fails to account for the fact that multiple balls can have the same integer, and the `set` doesn't track the counts of each integer.

### Issues with the Original Code
1. **Tracking Counts**: The original code uses a `set` to store integers, which means it only keeps track of whether an integer is present or not, not how many times it appears. This leads to incorrect behavior when multiple balls with the same integer are added or removed.
2. **Removal Logic**: The code removes an integer from the `set` when a `2 x` query is encountered, but this is incorrect if there are multiple balls with the same integer. The integer should only be removed from the `set` when the count of that integer drops to zero.

### Solution Approach
To correctly handle the counts of each integer, we need:
1. **A Frequency Dictionary**: Use a dictionary (or hash map) to keep track of how many times each integer appears in the bag.
2. **A Set for Distinct Integers**: Maintain a `set` that contains only the integers that have a non-zero count in the frequency dictionary. This allows efficient querying of the number of distinct integers.
3. **Update Logic**: 
   - For `1 x` queries, increment the count of `x` in the frequency dictionary. If the count becomes 1, add `x` to the `set`.
   - For `2 x` queries, decrement the count of `x`. If the count drops to 0, remove `x` from the `set`.
   - For `3` queries, simply output the size of the `set`.

### Fixed Code
```cpp
#include<bits/stdc++.h>

using namespace std;

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(NULL);

    int q;
    cin >> q;

    unordered_map<int, int> freq;
    set<int> distinct;
    int type, x;
    while (q--) {
        cin >> type;
        if (type == 1) {
            cin >> x;
            freq[x]++;
            if (freq[x] == 1) {
                distinct.insert(x);
            }
        } else if (type == 2) {
            cin >> x;
            freq[x]--;
            if (freq[x] == 0) {
                distinct.erase(x);
            }
        } else {
            cout << distinct.size() << '\n';
        }
    }

    return 0;
}
```

### Explanation
- **Frequency Dictionary (`freq`)**: This keeps track of how many balls of each integer are currently in the bag.
- **Set (`distinct`)**: This set contains only those integers that have at least one ball in the bag. When the count of an integer in `freq` becomes zero, it is removed from `distinct`.
- **Efficiency**: 
  - Insertions and deletions in the `set` and `unordered_map` are average O(1) operations, making the solution efficient for the given constraints.
  - The use of `ios_base::sync_with_stdio(false)` and `cin.tie(NULL)` optimizes input/output operations for speed, which is crucial given the large input size (up to 200,000 queries).
