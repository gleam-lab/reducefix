1. **Modulo Prefix Sums**: Compute the prefix sums and take each modulo m. This transforms the problem into finding pairs where (prefix[t] - prefix[s]) % m == 0.
2. **Frequency Map**: Use a frequency map to count occurrences of each modulo value. For each modulo value, the number of valid pairs is the combination of counts of that value (i.e., count choose 2).

### Solution Code
```cpp
#include<bits/stdc++.h>
using namespace std;
int n, m;
int main(){
    cin >> n >> m;
    vector<int> nums(n);
    for (int i = 0; i < n; i++) {
        cin >> nums[i];
    }
    vector<int> prefix_sum(n + 1, 0);
    for (int i = 0; i < n; i++) {
        prefix_sum[i + 1] = (prefix_sum[i] + nums[i]) % m;
    }
    unordered_map<int, long long> mod_count;
    for (int i = 0; i <= n; i++) {
        mod_count[prefix_sum[i]]++;
    }
    long long count = 0;
    for (auto& pair : mod_count) {
        long long c = pair.second;
        count += c * (c - 1) / 2;
    }
    cout << count << endl;
    return 0;
}
```

### Explanation
1. **Prefix Sum Modulo**: We compute the prefix sums and take each modulo m. This helps in identifying segments where the sum is divisible by m.
2. **Frequency Map**: We use a hash map to count how many times each modulo value appears in the prefix sum array.
3. **Counting Pairs**: For each modulo value, the number of valid pairs is given by the combination formula `c choose 2`, where `c` is the count of that modulo value. This is because any two indices with the same modulo value will form a segment whose sum is divisible by m.
4. **Efficiency**: This approach runs in O(N) time for prefix sum calculation and O(N) time for counting modulo values, making it efficient for large inputs. The overall complexity is O(N), which is optimal for the problem constraints.
