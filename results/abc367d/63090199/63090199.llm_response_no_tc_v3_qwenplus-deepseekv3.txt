The original code is intended to count the number of pairs (s, t) where the sum of elements from s to t (wrapping around if s > t) is divisible by m. However, the algorithm is inefficient with a time complexity of O(n²), which is too slow for large n. The problem can be optimized using modular arithmetic and prefix sums.

### Approaches
1. **Prefix Sum with Modulo**: Compute the prefix sum modulo m for each position. This helps in tracking remainders which can be used to count pairs efficiently.
2. **Remainder Frequency Map**: Use a frequency map to count how many times each remainder occurs in the prefix sums. This allows us to compute the number of valid pairs in O(n) time by leveraging the formula where pairs of remainders that differ by zero (mod m) contribute to the count.

### Solution Code
```cpp
#include<bits/stdc++.h>
using namespace std;

int main() {
    int n, m;
    cin >> n >> m;
    vector<int> nums(n);
    for (int i = 0; i < n; ++i) {
        cin >> nums[i];
    }

    vector<int> prefix_mod(n + 1, 0);
    for (int i = 0; i < n; ++i) {
        prefix_mod[i + 1] = (prefix_mod[i] + nums[i]) % m;
    }

    unordered_map<int, int> mod_count;
    long long result = 0;
    for (int i = 0; i <= n; ++i) {
        result += mod_count[prefix_mod[i]];
        mod_count[prefix_mod[i]]++;
    }

    cout << result << endl;
    return 0;
}
```

### Explanation
1. **Prefix Modulo Array**: The prefix_mod array stores the cumulative sum of elements modulo m up to each index. This helps in tracking the remainders which can be used to find subarrays that sum to a multiple of m.
2. **Modulo Frequency Map**: The mod_count map keeps track of how many times each remainder has been encountered in the prefix_mod array. For each remainder encountered, the number of valid pairs increases by the count of that remainder previously seen.
3. **Efficient Counting**: By iterating through the prefix_mod array and updating the mod_count map, the solution efficiently counts all valid subarrays in O(n) time, which is optimal for this problem. This avoids the O(n²) complexity of the original nested loop approach, making it suitable for large input sizes.
