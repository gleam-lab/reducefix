### Issues in the Original Code:
1. **Efficiency**: The nested loops result in O(n²) time complexity, which is too slow for large n (e.g., n = 1e5).
2. **Correctness**: The logic for wrapping around (when s > t) is incorrect. The sum should be `prefixsum[n] - prefixsum[s] + prefixsum[t]`, but it doesn't account for the fact that the sum from s to t (wrapping around) should include nums[s-1] to nums[t-1] in reverse order.

### Optimized Approach:
1. **Modulo Properties**: Use the property that (a - b) % m = 0 implies a % m = b % m.
2. **Prefix Sum Modulo**: Compute prefix sums modulo m and count occurrences of each modulo value.
3. **Counting Pairs**: For each modulo value, the number of valid pairs is the combination of indices with the same modulo value.

### Fixed Code:
```cpp
#include<bits/stdc++.h>
using namespace std;

int main() {
    int n, m;
    cin >> n >> m;
    vector<int> nums(n);
    for (int i = 0; i < n; i++) {
        cin >> nums[i];
    }

    vector<int> prefix_mod(n + 1, 0);
    for (int i = 0; i < n; i++) {
        prefix_mod[i + 1] = (prefix_mod[i] + nums[i]) % m;
    }

    unordered_map<int, int> mod_count;
    for (int i = 0; i <= n; i++) {
        mod_count[prefix_mod[i]]++;
    }

    long long count = 0;
    for (auto& [mod, cnt] : mod_count) {
        count += (long long)cnt * (cnt - 1) / 2;
    }

    cout << count << endl;
    return 0;
}
```

### Explanation:
1. **Prefix Modulo Array**: `prefix_mod[i]` stores the sum of the first i elements modulo m. This helps in leveraging the modulo property to find pairs efficiently.
2. **Modulo Counting**: Using a hash map, we count how many times each modulo value appears in the `prefix_mod` array.
3. **Combination Calculation**: For each modulo value, the number of valid pairs is the combination of 2 indices from the count of that modulo value, i.e., `cnt * (cnt - 1) / 2`.
4. **Efficiency**: This approach runs in O(n) time, making it suitable for large inputs. The space complexity is O(n) due to the storage of prefix sums and hash map.

This solution correctly and efficiently counts all pairs (s, t) where the sum of elements from s to t (inclusive, wrapping around if necessary) is divisible by m.
