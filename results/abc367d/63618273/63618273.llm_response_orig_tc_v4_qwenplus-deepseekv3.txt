To fix this code, we need to address the core logic that calculates the number of subarrays whose sum modulo M equals zero. The original code had flaws in how it updated and queried the map for prefix sums, leading to incorrect counts. 

### Approaches
1. **Prefix Sum and Modulo**: We compute the prefix sums of the array and then take each prefix sum modulo M. This helps in identifying subarrays where the difference between two prefix sums modulo M is zero, indicating a subarray sum that is divisible by M.
2. **Hash Map for Counting**: We use a hash map to keep track of the frequencies of each modulo value encountered so far. For each current prefix modulo, we check the map for the count of previous prefix modulos that would make the subarray sum between them divisible by M.
3. **Initial Condition Handling**: We initialize the map with a zero count to handle the case where the prefix sum itself is divisible by M directly.

### Solution Code
```cpp
#include<bits/stdc++.h>

using namespace std;
using ll=long long;
typedef pair<int, int>PII;
const int N=2e5+10,mod=1e9+7;

void solve()
{
    int n, M;
    cin >> n >> M;
    vector<int> a(n);
    for(int i=0; i<n; i++){
        cin >> a[i];
    }
    vector<ll> prefix(n+1, 0);
    for(int i=0; i<n; i++){
        prefix[i+1] = prefix[i] + a[i];
    }
    map<ll, ll> mp;
    mp[0] = 1; // Initialize to handle subarrays starting from index 0
    ll ans = 0;
    for(int i=1; i<=n; i++){
        ll mod_val = (prefix[i] % M + M) % M;
        ans += mp[mod_val];
        mp[mod_val]++;
    }
    cout << ans << endl;
}

int main()
{
    cin.tie(0);
    ios::sync_with_stdio(0);
    ll t=1;
    while(t--){
        solve();
    }
    return 0;
}
```

### Explanation
1. **Prefix Sum Calculation**: The prefix sum array is constructed such that `prefix[i]` represents the sum of the elements from the start of the array up to the (i-1)th element. This allows us to compute the sum of any subarray in constant time.
2. **Modulo Handling**: For each prefix sum, we compute its modulo with M. This modulo value helps in identifying if the sum of a subarray (from some previous index to the current index) is divisible by M.
3. **Hash Map Usage**: A hash map is used to store the frequency of each modulo value encountered during the iteration. For each current prefix modulo, the number of previous prefix modulos that result in a zero modulo difference (indicating a valid subarray) is added to the answer.
4. **Initialization**: The map is initialized with `mp[0] = 1` to account for subarrays that start from the beginning of the array and are divisible by M by themselves.

This approach ensures that we efficiently count all valid subarrays in O(N) time with O(M) space complexity, leveraging the properties of prefix sums and modulo arithmetic.
