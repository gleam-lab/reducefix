To fix this code, we need to correctly count the number of subarrays whose sum modulo M is zero. The original code had logical errors in how it was using the map to track prefix sums and their modulo values. Here's the corrected approach:

### Approach
1. **Prefix Sum and Modulo Handling**: Compute the prefix sums of the array and for each prefix sum, compute its modulo with M. The key idea is that if two prefix sums have the same modulo value, the subarray between them will have a sum that is divisible by M.
2. **Frequency Map**: Use a hash map to keep track of the frequency of each modulo value encountered in the prefix sums. This allows us to efficiently count how many times each modulo value has been seen.
3. **Counting Valid Subarrays**: For each prefix sum modulo, if the same modulo value has been seen before, the number of valid subarrays increases by the frequency of that modulo value. We then update the frequency map accordingly.

### Solution Code
```cpp
#include<bits/stdc++.h>

using namespace std;
using ll=long long;
typedef pair<int, int>PII;
const int N=2e5+10,mod=1e9+7;

void solve()
{
    int n, M;
    cin>>n>>M;
    vector<int>a(n);
    for(int i=0; i<n; i++){
        cin>>a[i];
    }
    vector<ll>prefix(n+1, 0);
    for(int i=0; i<n; i++){
        prefix[i+1] = prefix[i] + a[i];
    }
    map<int, int>mp;
    mp[0] = 1;
    ll ans = 0;
    for(int i=1; i<=n; i++){
        int mod_val = (prefix[i] % M + M) % M;
        ans += mp[mod_val];
        mp[mod_val]++;
    }
    cout<<ans;
}

int main()
{
    cin.tie(0);
    ios::sync_with_stdio(0);
    ll t=1;
    while(t--){
        solve();
    }
    return 0;
}
```

### Explanation
1. **Prefix Sum Calculation**: The prefix sum array `prefix` is constructed where `prefix[i]` represents the sum of the first `i-1` elements of the array. This helps in efficiently calculating the sum of any subarray.
2. **Modulo Handling**: For each element in the prefix sum array, we compute its modulo with M. This modulo value is stored in `mod_val`.
3. **Frequency Map**: The map `mp` keeps track of how many times each modulo value has been encountered so far. Initially, `mp[0]` is set to 1 to handle the case where a prefix sum itself is divisible by M.
4. **Counting Subarrays**: For each `mod_val` in the prefix sum array, if this value has been seen before, it means there are subarrays ending at the current position whose sum is divisible by M. The count of such subarrays is incremented by the frequency of `mod_val` in the map. The frequency of `mod_val` is then incremented in the map for future checks.

This approach ensures that we efficiently count all valid subarrays in O(N) time with O(N) space complexity, leveraging the properties of prefix sums and modular arithmetic.
