### Correct Approach

2. **Hash Map for Counting**: Use a hash map to keep track of the frequency of each modulo result encountered in the prefix sum array. This allows us to count how many times a particular modulo result has been seen, which helps in finding subarrays where the sum is divisible by M.
3. **Initialization**: Initialize the map with a zero count to handle cases where the subarray starts from the beginning of the array.

#include<bits/stdc++.h>

using namespace std;
using ll=long long;
typedef pair<int, int>PII;
const int N=2e5+10,mod=1e9+7;

void solve()
{
    int n,M;

    vector<int>a(n+10);
    for(int i=1; i<=n; i++){
        cin>>a[i];
    }
    vector<ll>prefix(n+10,0);
    for(int i=1; i<=n; i++){
        prefix[i]=(prefix[i-1]+a[i])%M;
    }
    map<ll, ll>mp;
    mp[0]=1; // Initialize with prefix[0] = 0
    ll ans=0;
    for(int i=1; i<=n; i++){
        ans+=mp[prefix[i]];
        mp[prefix[i]]++;
    }
    cout<<ans;
}

int main()
{
    cin.tie(0);
    ios::sync_with_stdio(0);
    ll t=1;
    while(t--){
        solve();
    }
    return 0;

### Explanation
1. **Prefix Sum Calculation**: The prefix sum array `prefix` is computed where each element `prefix[i]` is the sum of the first `i` elements of the array modulo M. This helps in efficiently determining subarray sums modulo M.
2. **Hash Map Initialization**: The hash map `mp` is initialized with `mp[0] = 1` to account for subarrays that start from the beginning of the array and whose sum is divisible by M.
3. **Counting Valid Subarrays**: For each element in the prefix sum array, the code checks how many times the current modulo result has been seen before. Each occurrence indicates a subarray ending at the current index whose sum is divisible by M. The count is incremented by the frequency of the modulo result in the map, and the map is updated to include the current modulo result.

This approach ensures that all valid subarrays are counted efficiently, with a time complexity of O(N), making it suitable for large input sizes. The space complexity is O(M) due to the hash map, which is efficient given the constraints.
