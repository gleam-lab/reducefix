```cpp
#include<bits/stdc++.h>

using namespace std;
using ll=long long;
typedef pair<int, int>PII;
const int N=2e5+10,mod=1e9+7;

void solve()
{
    int n,M;
    cin>>n>>M;
    vector<int>a(n+10);
    for(int i=1; i<=n; i++){
        cin>>a[i];
    }
    vector<ll>s(n+10,0);
    for(int i=1; i<=n; i++){
        s[i]=(s[i-1]+a[i])%M;
    }
    map<int, ll>mp;
    mp[0]=1;  // empty prefix sum
    ll ans=0;
    for(int i=1; i<=n; i++){
        ans+=mp[s[i]];
        mp[s[i]]++;
    }
    cout<<ans;
}

int main()
{
    cin.tie(0);
    ios::sync_with_stdio(0);
    ll t=1;
    while(t--){
        solve();
    }
    return 0;
}
```

Key fixes:
1. Changed the prefix sum calculation to store modulo M values directly
2. Fixed the map counting logic - we count how many times each modulo has appeared before
3. Added initialization of mp[0] = 1 to account for subarrays starting from index 1
4. Changed the sum type to long long to prevent overflow
5. Removed the incorrect L calculation which wasn't needed
6. Simplified the counting logic to just track occurrences of each modulo value

The algorithm now correctly counts the number of subarrays whose sum is divisible by M using the prefix sum modulo technique, which is both efficient (O(n)) and correct for all cases.

The solution works by:
1. Computing prefix sums modulo M
2. Counting how many times each modulo value appears
3. For each position, the number of valid subarrays ending there is equal to the number of previous positions where the same modulo occurred
4. Adding these counts together gives the total number of valid subarrays
