To fix this code, we need to correctly count the number of subarrays where the sum modulo M equals zero. The original code had a logical error in how it was using the map to track prefix sums modulo M and how it was calculating the number of valid subarrays.

### Approaches
1. **Prefix Sum and Modulo**: Compute the prefix sums of the array and then take each prefix sum modulo M. This helps in finding subarrays where the sum modulo M is zero.
2. **Hash Map for Counting**: Use a hash map to keep track of the frequency of each modulo value encountered so far. For each current prefix sum modulo M, we check how many times the same modulo value has been seen before. This count gives the number of subarrays ending at the current index that have a sum modulo M equal to zero.
3. **Initial Condition**: Initialize the map with a zero modulo value count of 1 to handle the case where the subarray starts from the first element.

### Solution Code
```cpp
#include<bits/stdc++.h>

using namespace std;
using ll=long long;
typedef pair<int, int>PII;
const int N=2e5+10,mod=1e9+7;

void solve()
{
    int n, M;
    cin >> n >> M;
    vector<int> a(n);
    for(int i = 0; i < n; i++){
        cin >> a[i];
    }
    
    vector<ll> prefix(n + 1, 0);
    for(int i = 0; i < n; i++){
        prefix[i + 1] = prefix[i] + a[i];
    }
    
    map<ll, ll> mp;
    mp[0] = 1; // To account for subarrays starting from the first element
    ll ans = 0;
    
    for(int i = 1; i <= n; i++){
        ll mod_val = (prefix[i] % M + M) % M;
        ans += mp[mod_val];
        mp[mod_val]++;
    }
    
    cout << ans << endl;
}

int main()
{
    cin.tie(0);
    ios::sync_with_stdio(0);
    ll t = 1;
    while(t--){
        solve();
    }
    return 0;
}
```

### Explanation
1. **Prefix Sum Calculation**: The prefix sum array is computed to store cumulative sums up to each index in the original array. This allows efficient calculation of subarray sums.
2. **Modulo Handling**: For each prefix sum, we compute its modulo with M. The result is adjusted to be non-negative by adding M before taking modulo if necessary.
3. **Hash Map Usage**: The hash map keeps count of how many times each modulo value has been encountered. For each current modulo value, the number of previous occurrences is added to the answer, as each occurrence indicates a subarray ending at the current index with a sum divisible by M.
4. **Initialization**: The map is initialized with `mp[0] = 1` to handle cases where the subarray from the start of the array to the current index has a sum divisible by M.

This approach efficiently counts all valid subarrays in O(N) time with O(N) space complexity, leveraging prefix sums and modulo arithmetic to ensure correctness.
