To fix this code, we need to correctly count the number of subarrays where the sum of elements modulo M equals zero. The original code had logical errors in how it was tracking and counting these subarrays. 

### Approaches
1. **Prefix Sum Calculation**: Compute the prefix sums of the array to efficiently calculate the sum of any subarray.
2. **Modulo Tracking**: Use a hash map (or dictionary) to track the frequency of each modulo value encountered in the prefix sums.
3. **Counting Valid Subarrays**: For each prefix sum, compute its modulo with M. The number of valid subarrays ending at the current position is the count of how many times the same modulo value has been seen before. Additionally, we need to handle the case where the subarray from the start to the current position has a sum that is a multiple of M.

### Solution Code
```cpp
#include<bits/stdc++.h>

using namespace std;
using ll=long long;
typedef pair<int, int>PII;
const int N=2e5+10,mod=1e9+7;

void solve()
{
    int n, M;
    cin >> n >> M;
    vector<int> a(n);
    for(int i = 0; i < n; i++){
        cin >> a[i];
    }
    vector<ll> s(n+1);
    s[0] = 0;
    for(int i = 0; i < n; i++){
        s[i+1] = s[i] + a[i];
    }
    map<ll, int> mp;
    mp[0] = 1;
    ll ans = 0;
    for(int i = 1; i <= n; i++){
        ll mod_val = (s[i] % M + M) % M;
        ans += mp[mod_val];
        mp[mod_val]++;
    }
    cout << ans;
}

int main()
{
    cin.tie(0);
    ios::sync_with_stdio(0);
    ll t=1;
    while(t--){
        solve();
    }
    return 0;
}
```

### Explanation
1. **Prefix Sum Calculation**: The prefix sum array `s` is constructed where `s[i]` represents the sum of the first `i` elements of the array. This allows us to compute the sum of any subarray in constant time.
2. **Modulo Tracking**: A hash map `mp` is used to keep track of the frequency of each modulo value encountered in the prefix sums. The key idea is that if two prefix sums have the same modulo value, the subarray between their indices will have a sum that is a multiple of M.
3. **Counting Valid Subarrays**: For each prefix sum `s[i]`, we compute its modulo with M. The number of valid subarrays ending at position `i` is given by the count of how many times this modulo value has been seen before. This count is accumulated into `ans`. The modulo value is then incremented in the hash map to include the current prefix sum.
4. **Edge Cases**: The initial entry `mp[0] = 1` accounts for the case where the subarray starts from the beginning of the array, making the sum from index 0 to `i` a multiple of M.

This approach ensures that we efficiently count all valid subarrays in O(N) time with O(N) space complexity, leveraging prefix sums and hash map tracking for optimal performance.
