2. **Modulo Handling**: For each prefix sum, compute its modulo M value.
3. **Frequency Map**: Use a frequency map to keep track of how many times each modulo value has been encountered so far.
4. **Counting Valid Subarrays**: For each current prefix sum modulo M, if we have seen this value before, it means there exists a subarray between the previous occurrence(s) and the current index that sums to a multiple of M.

    ll ans=0;

        ans += mp[mod_val];
        mp[mod_val]++;

### Explanation:
1. **Prefix Sums**: The prefix sums array `s` is computed where `s[i]` is the sum of the first `i` elements of the array.
2. **Modulo Handling**: For each prefix sum, we compute its modulo M value, ensuring it is positive by adding M before taking modulo M if needed.
3. **Frequency Map**: The map `mp` keeps track of how many times each modulo value has been encountered. Initially, `mp[0]` is set to 1 to account for subarrays starting from the first element.
4. **Counting Valid Subarrays**: For each index `i`, if the current modulo value has been seen before, it means there are `mp[mod_val]` subarrays ending at `i` that sum to a multiple of M. We increment `ans` by this count and update the map.

This approach ensures that we efficiently count all valid subarrays in O(N) time with O(M) space complexity, which is optimal for the problem constraints.
