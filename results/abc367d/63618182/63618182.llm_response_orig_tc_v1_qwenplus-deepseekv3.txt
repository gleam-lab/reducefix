To fix this code, we need to correctly count the number of subarrays where the sum modulo M equals a specific value. The previous implementation had logic errors in how it was maintaining and updating the map of remainders.

### Approaches
1. **Prefix Sum and Modulo**: Compute the prefix sums of the array elements. For each prefix sum, compute its modulo with M to get the current remainder.
2. **Map Tracking**: Use a map to keep track of how many times each remainder has been encountered so far. This helps in efficiently calculating the number of valid subarrays ending at the current position.
3. **Valid Subarrays Count**: For each current remainder, if we have encountered a remainder that, when subtracted from the current remainder (adjusted with M), equals a target value (here, the total sum modulo M), we increment the count accordingly.

### Solution Code
```cpp
#include<bits/stdc++.h>

using namespace std;
using ll=long long;
typedef pair<int, int>PII;
const int N=2e5+10,mod=1e9+7;

void solve()
{
    int n,M;
    cin>>n>>M;
    vector<int>a(n+10);
    for(int i=1; i<=n; i++){
        cin>>a[i];
    }
    vector<ll>s(n+10);
    for(int i=1; i<=n; i++){
        s[i]=s[i-1]+a[i];
    }
    ll total=s[n];
    ll target=total%M;
    if(target<0) target+=M;
    
    map<ll, ll>mp;
    mp[0]=1;
    ll ans=0;
    for(int i=1; i<=n; i++){
        ll current=s[i]%M;
        if(current<0) current+=M;
        ll needed=(current-target+M)%M;
        ans+=mp[needed];
        mp[current]++;
    }
    cout<<ans;
}

int main()
{
    cin.tie(0);
    ios::sync_with_stdio(0);
    ll t=1;
    while(t--){
        solve();
    }
    return 0;
}
```

### Explanation
1. **Prefix Sum Calculation**: The prefix sum array `s` is computed where each element `s[i]` represents the sum of the first `i` elements of the array. This helps in efficiently calculating the sum of any subarray.
2. **Total Sum Modulo**: The total sum of the array is computed and its modulo with `M` is stored in `target`. This target value is what we need to find in the subarray sums modulo `M`.
3. **Map Initialization**: A map `mp` is initialized to keep track of the frequency of each remainder encountered during the prefix sum traversal. The initial value `mp[0] = 1` accounts for the sum of an empty subarray.
4. **Traversal and Counting**: For each element in the prefix sum array:
   - The current remainder `current` is computed as `s[i] % M`.
   - The needed remainder `needed` is calculated as `(current - target + M) % M` to ensure it is non-negative.
   - The count of subarrays ending at the current position that meet the target is incremented by the number of times the `needed` remainder has been encountered before, which is retrieved from the map `mp`.
   - The current remainder is then added to the map to keep track of its frequency for future calculations.
5. **Output**: The total count of such valid subarrays is printed as the result.

This approach efficiently counts the required subarrays in O(N) time with O(N) space complexity, leveraging prefix sums and a hash map for remainder tracking.
