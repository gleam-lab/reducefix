```cpp
#include<bits/stdc++.h>
using namespace std;
using ll=long long;
typedef pair<int, int>PII;
const int N=2e5+10,mod=1e9+7;

void solve()
{
    int n,M;
    cin>>n>>M;
    vector<ll>a(n+10),s(n+10);
    for(int i=1; i<=n; i++){
        cin>>a[i];
        s[i]=(s[i-1]+a[i])%M;
    }
    map<ll, int>mp;
    mp[0]++; // Initialize with one count for remainder 0
    ll ans=0;
    for(int i=1; i<=n; i++){
        ans=(ans+mp[s[i]])%mod;
        mp[s[i]]++;
    }
    cout<<ans<<endl;
}

int main()
{
    ios_base::sync_with_stdio(false);
    cin.tie(0);
    ll t=1;
    while(t--){
        solve();
    }
    return 0;
}
```

### Explanation of Fixes:
1. **Data Type for Prefix Sum Array**: Changed `vector<int>` to `vector<ll>` to handle large prefix sums without overflow.
2. **Modulo Operation in Prefix Sum Calculation**: Ensured modulo operation is applied correctly during prefix sum calculation.
3. **Initialization of Map**: Initialized the map with `{0: 1}` to account for the case where a subarray starting from the beginning has a sum that is already a multiple of `M`.
4. **Efficiency**: The solution maintains a time complexity of O(N) as it iterates through the array once and uses a hash map for constant-time lookups and updates.
5. **Output Formatting**: Added `endl` after printing the answer for proper formatting.
