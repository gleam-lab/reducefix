The problem involves counting the number of subarrays in an array where the sum of elements is divisible by a given integer `m`. The original code tries to achieve this by checking all possible subarrays, which results in a time complexity of O(N^2), making it inefficient for large inputs (as seen in the failing test case with N=23762).

### Analysis of the Problem
The key observation here is that for any subarray sum to be divisible by `m`, the remainder of the cumulative sum up to the end of the subarray must be the same as the remainder of the cumulative sum up to just before the start of the subarray. This can be efficiently tracked using a hash map to store the frequency of remainders encountered so far in the cumulative sum.

### Approach
1. **Cumulative Sum and Remainder Tracking**: We compute the cumulative sum modulo `m` for each position in the array. This helps in identifying subarrays where the difference in cumulative sums (mod `m`) is zero, indicating a divisible subarray.
2. **Hash Map for Remainder Frequencies**: We use a hash map to keep track of how many times each remainder has been encountered. For each current remainder, if it has been seen before, the number of subarrays ending at the current position with sum divisible by `m` increases by the count of that remainder.
3. **Initialization**: Initialize the hash map with a remainder of `0` to account for subarrays starting from the beginning of the array.

### Solution Code
```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    int n, m;
    cin >> n >> m;
    vector<int> arr(n);
    for (int i = 0; i < n; ++i) {
        cin >> arr[i];
    }

    long long ans = 0;
    vector<int> prefix_mod(n + 1, 0);
    unordered_map<int, int> mod_count;
    mod_count[0] = 1;

    for (int i = 1; i <= n; ++i) {
        prefix_mod[i] = (prefix_mod[i - 1] + arr[i - 1]) % m;
        if (prefix_mod[i] < 0) {
            prefix_mod[i] += m;
        }
        ans += mod_count[prefix_mod[i]];
        mod_count[prefix_mod[i]]++;
    }

    cout << ans << endl;
    return 0;
}
```

### Explanation
1. **Initialization**: We read the input values for `n` (number of elements) and `m` (divisor). The array `arr` stores the elements.
2. **Prefix Mod Array**: This array stores the cumulative sum modulo `m` up to each index. The size is `n+1` to include the initial state (sum of zero elements).
3. **Mod Count Map**: This map keeps track of how many times each remainder has been encountered during the traversal. It is initialized with `0` to handle subarrays starting from the first element.
4. **Traversal and Calculation**: For each element in the array, we compute the current cumulative sum modulo `m`. If the current remainder has been seen before, it means there are subarrays ending at the current index that are divisible by `m`. The count of such subarrays is added to `ans`, and the remainder's count in the map is incremented.
5. **Output**: The total count of such subarrays is printed.

This approach ensures that we efficiently count the subarrays in O(N) time, making it suitable for large input sizes.
