### Approaches
1. **BFS for Shortest Path Calculation**: First, perform a BFS from node 0 to calculate the shortest distances to all other nodes. This helps in knowing how far each node is from node 0.
2. **Check Reverse Path**: For each node that can directly reach node 0, check if it was reachable from node 0 in the BFS step. The shortest such cycle will be the sum of the shortest path from node 0 to this node plus 1 (for the edge back to node 0).
3. **Find Minimum Cycle Length**: Iterate through all nodes that have an edge to node 0 and find the one with the smallest distance from node 0. The answer is this distance plus 1 if such a node exists; otherwise, return -1.

### Solution Code
```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    int N, M;
    cin >> N >> M;
    vector<vector<int>> G(N);
    for (int i = 0; i < M; i++) {
        int u, v;
        cin >> u >> v;
        u--, v--;
        G[u].push_back(v);
    }

    vector<int> dist(N, -1);
    queue<int> que;
    que.push(0);
    dist[0] = 0;
    while (!que.empty()) {
        int pos = que.front();
        que.pop();
        for (int nex : G[pos]) {
            if (dist[nex] == -1) {
                dist[nex] = dist[pos] + 1;
                que.push(nex);
            }
        }
    }

    int ans = -1;
    for (int i = 0; i < N; i++) {
        if (dist[i] == -1) continue;
        for (int x : G[i]) {
            if (x == 0) {
                if (ans == -1 || dist[i] + 1 < ans) {
                    ans = dist[i] + 1;
                }
            }
        }
    }

    cout << ans << endl;
    return 0;
}
```

### Explanation
1. **Input Handling**: The code reads the number of nodes (N) and edges (M). It then constructs the adjacency list for the graph.
2. **BFS for Shortest Paths**: Using BFS from node 0, the code calculates the shortest distance from node 0 to all other nodes. This distance is stored in the `dist` array.
3. **Finding Minimum Cycle**: The code checks each node to see if it has an edge back to node 0. For each such node, it calculates the potential cycle length as the distance from node 0 to the node plus 1 (the edge back to node 0). The minimum of these values is taken as the answer.
4. **Output Result**: If no such node exists, the answer remains -1, indicating no cycle includes node 0. Otherwise, the shortest cycle length is printed.

This approach ensures that we efficiently find the shortest cycle involving node 0 using BFS and careful checking of reverse edges, adhering to the problem constraints and edge cases.
