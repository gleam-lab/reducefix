To fix this code, we need to ensure that the algorithm correctly identifies if there's a shortest cycle involving node 0 (the first node) in the directed graph. The original code attempts to find the shortest path from node 0 to any node that has a direct edge back to node 0, but it fails to handle cases where no such cycle exists.

### Approaches
1. **Graph Representation**: Represent the graph using adjacency lists.
2. **BFS for Shortest Path**: Use BFS to compute the shortest path from node 0 to all other nodes since BFS naturally finds the shortest path in an unweighted graph.
3. **Check for Cycles**: After computing the shortest paths, check each node to see if it has a direct edge back to node 0. If such a node is found, the length of the cycle would be the shortest path to this node plus 1 (the edge back to node 0).
4. **Output the Result**: If no such cycle is found, return -1. Otherwise, return the shortest cycle length found.

### Solution Code
```cpp
#include <bits/stdc++.h>
using namespace std;
const int INF = 1e9;

int main() {
    int N, M;
    cin >> N >> M;
    vector<vector<int>> G(N);
    for (int i = 0; i < M; i++) {
        int u, v;
        cin >> u >> v;
        u--, v--;
        G[u].push_back(v);
    }
    
    vector<int> dist(N, -1);
    queue<int> q;
    q.push(0);
    dist[0] = 0;
    
    while (!q.empty()) {
        int u = q.front();
        q.pop();
        for (int v : G[u]) {
            if (dist[v] == -1) {
                dist[v] = dist[u] + 1;
                q.push(v);
            }
        }
    }
    
    int min_cycle = INF;
    for (int u = 0; u < N; u++) {
        if (dist[u] != -1) {
            for (int v : G[u]) {
                if (v == 0) {
                    if (dist[u] + 1 < min_cycle) {
                        min_cycle = dist[u] + 1;
                    }
                }
            }
        }
    }
    
    if (min_cycle == INF) {
        cout << -1 << endl;
    } else {
        cout << min_cycle << endl;
    }
    
    return 0;
}
```

### Explanation
1. **Graph Representation**: The graph is stored using an adjacency list `G`, where each node `u` has a list of nodes `v` it can reach directly.
2. **BFS Initialization**: We initialize a distance vector `dist` to keep track of the shortest path distances from node 0, initially set to -1 (unvisited). Node 0 is enqueued with a distance of 0.
3. **BFS Execution**: The BFS processes each node `u` by visiting all its adjacent nodes `v`. If `v` hasn't been visited, its distance is set to `dist[u] + 1`, and it is enqueued for further processing.
4. **Cycle Detection**: After BFS completes, we check each node `u` to see if it has a direct edge back to node 0. If such an edge is found, the potential cycle length is `dist[u] + 1`. The smallest such cycle length is tracked.
5. **Result Output**: If no cycle is found (i.e., `min_cycle` remains `INF`), -1 is printed. Otherwise, the shortest cycle length is printed.

This approach efficiently computes the shortest cycle involving node 0 using BFS, ensuring correctness and optimal performance for large graphs.
