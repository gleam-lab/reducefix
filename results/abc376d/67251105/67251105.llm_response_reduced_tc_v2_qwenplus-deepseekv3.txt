To fix this code, we need to address the issue where the incorrect output is being generated for certain test cases. The problem involves finding the shortest distance from node 1 to node 0 in a directed graph, where node 0 is directly connected to some nodes. The incorrect code was returning 0 when it should have returned -1, indicating no valid path exists.

### Approaches

1. **Graph Representation**: The graph is represented using an adjacency list.
2. **Breadth-First Search (BFS)**:
   - We perform BFS starting from node 0 to find the shortest distance to node 1.
   - The BFS ensures that we explore all nodes level by level, marking their distances from the starting node.
3. **Check for Direct Connection**: After BFS, we check if node 1 has a direct edge to node 0. If such an edge exists, we return the distance from node 0 to node 1 plus 1 (to account for the direct edge). If no such path exists, we return -1.

### Solution Code

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    int N, M;
    cin >> N >> M;
    vector<vector<int>> G(N);
    for (int i = 0; i < M; i++) {
        int u, v;
        cin >> u >> v;
        u--, v--;
        G[u].push_back(v);
    }

    vector<int> dist(N, -1);
    queue<int> que;
    que.push(0);
    dist[0] = 0;
    while (!que.empty()) {
        int pos = que.front();
        que.pop();
        for (int nex : G[pos]) {
            if (dist[nex] == -1) {
                dist[nex] = dist[pos] + 1;
                que.push(nex);
            }
        }
    }

    bool found = false;
    for (int nex : G[1]) {
        if (nex == 0) {
            found = true;
            break;
        }
    }

    if (found && dist[1] != -1) {
        cout << dist[1] + 1 << endl;
    } else {
        cout << -1 << endl;
    }

    return 0;
}
```

### Explanation

1. **Graph Representation**: The graph is stored as an adjacency list where each node points to its neighbors.
2. **BFS Initialization**: We initialize the distance array `dist` with -1, indicating that nodes are initially unreachable. The BFS starts from node 0 (adjusted from 1-based to 0-based indexing), and its distance is set to 0.
3. **BFS Execution**: During BFS, for each node, we explore its neighbors. If a neighbor hasn't been visited (distance is -1), we set its distance to the current node's distance plus 1 and enqueue it.
4. **Checking for Direct Edge**: After BFS completes, we check if node 1 has a direct edge to node 0. If such an edge exists and node 1 was reachable from node 0, we output the distance from node 0 to node 1 plus 1 (the direct edge). If no such path exists, we output -1.

This approach ensures we correctly handle all edge cases and efficiently compute the shortest path or determine if no path exists using BFS, which operates in O(N + M) time complexity, suitable for large graphs.
