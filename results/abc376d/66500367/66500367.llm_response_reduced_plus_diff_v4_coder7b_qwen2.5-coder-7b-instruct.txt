The problem asks us to determine if there is a cycle in an undirected graph using BFS. If a cycle exists, we should return the length of the shortest cycle; otherwise, we should return -1.

Here’s a step-by-step approach to solve this:

1. **Graph Representation**: Use an adjacency list to represent the graph.
2. **Breadth-First Search (BFS)**: Perform BFS from each node to detect cycles. For each node, use a queue to explore its neighbors and keep track of distances from the start node.
3. **Cycle Detection**: During the BFS traversal, if you encounter a node that has already been visited and it is not the parent of the current node, then a cycle exists.

Let's implement this approach:

```cpp
#include <bits/stdc++.h>
using namespace std;

using ll = long long;
using pii = pair<int, int>;

const int N = 2e5 + 5;
vector<int> g[N];
int n, m, dep[N];

int bfs(int start) {
    queue<pii> q;
    q.push({start, 0});
    dep[start] = 1;
    while (!q.empty()) {
        auto [u, p] = q.front();
        q.pop();
        for (auto v : g[u]) {
            if (v == p) continue; // Skip the parent node
            if (dep[v]) { // Node already visited
                return dep[u] + dep[v]; // Return the length of the cycle
            }
            dep[v] = dep[u] + 1;
            q.push({v, u});
        }
    }
    return -1; // No cycle found
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);
    cout.tie(nullptr);

    cin >> n >> m;
    for (int i = 1; i <= m; i++) {
        int u, v;
        cin >> u >> v;
        g[u].push_back(v);
        g[v].push_back(u); // Since it's an undirected graph
    }

    int min_cycle = INT_MAX;
    bool found_cycle = false;

    for (int i = 1; i <= n; i++) {
        memset(dep, 0, sizeof(dep));
        int cycle_length = bfs(i);
        if (cycle_length != -1) {
            found_cycle = true;
            min_cycle = min(min_cycle, cycle_length);
        }
    }

    if (found_cycle) {
        cout << min_cycle << '\n';
    } else {
        cout << -1 << '\n';
    }

    return 0;
}
```

### Explanation:
1. **Graph Initialization**: We initialize the adjacency list `g` and read the number of nodes `n` and edges `m`.
2. **BFS Function**: The `bfs` function performs BFS starting from a given node `start`. It uses a queue to explore neighbors and keeps track of distances (`dep`). If a neighbor has already been visited and is not the parent of the current node, a cycle is detected.
3. **Main Logic**:
   - Initialize `min_cycle` to a large value and a boolean flag `found_cycle` to check if any cycle is found.
   - For each node, call the `bfs` function and update `min_cycle` if a shorter cycle is found.
   - Finally, print the minimum cycle length if a cycle was found, otherwise print -1.

This approach ensures that we correctly identify the shortest cycle in the graph, handling all edge cases as required.
