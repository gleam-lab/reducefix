To fix this code, we need to correctly determine if the graph contains a cycle. The current implementation incorrectly returns the depth of a node when it encounters a node that has already been visited, which is not the correct way to detect cycles in an undirected graph. 

### Approach
1. **Graph Representation**: The graph is represented using an adjacency list.
2. **BFS for Cycle Detection**: We use BFS to traverse the graph. For each node, we keep track of its parent to avoid revisiting the parent node. If we encounter a node that has been visited and it is not the parent of the current node, then a cycle exists.
3. **Cycle Length Calculation**: If a cycle is detected, its length is calculated as the sum of the depths of the two nodes involved in the cycle plus one (since both nodes are part of the cycle). If no cycle is found, we return -1.

### Solution Code
```cpp
#include <bits/stdc++.h>

using namespace std;
using ll = long long;
using pii = pair<int, int>;

const int N = 2e5 + 5;
vector<int> g[N];
int n, m, dep[N], parent[N];

int bfs() {
    queue<int> q;
    q.push(1);
    dep[1] = 1;
    parent[1] = -1;
    while (!q.empty()) {
        int u = q.front();
        q.pop();
        for (int v : g[u]) {
            if (v == parent[u]) continue;
            if (dep[v] == 0) {
                dep[v] = dep[u] + 1;
                parent[v] = u;
                q.push(v);
            } else {
                return dep[u] + dep[v] - 1;
            }
        }
    }
    return -1;
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);
    cout.tie(nullptr);

    cin >> n >> m;
    for (int i = 1; i <= m; i++) {
        int u, v;
        cin >> u >> v;
        g[u].push_back(v);
        g[v].push_back(u);
    }

    cout << bfs() << '\n';
    return 0;
}
```

### Explanation
1. **Graph Representation**: The graph is stored in an adjacency list `g` where each node points to its neighbors.
2. **BFS Initialization**: The BFS starts from node 1, with its depth set to 1 and parent initialized to -1 (indicating no parent).
3. **Cycle Detection**: For each node `u` dequeued, its neighbors `v` are checked. If a neighbor `v` has been visited (dep[v] != 0) and is not the parent of `u`, a cycle is detected. The cycle length is the sum of the depths of `u` and `v` minus one (since both depths include the common root path).
4. **Output**: If no cycle is found, BFS completes, and -1 is returned. Otherwise, the cycle length is returned.

This approach ensures correct cycle detection and length calculation in an undirected graph, meeting the problem requirements.
