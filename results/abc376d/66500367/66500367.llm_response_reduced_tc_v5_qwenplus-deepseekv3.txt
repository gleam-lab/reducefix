To fix this code, we need to properly detect whether the graph contains a cycle and return -1 if it does not, or the length of the cycle if it does. The original code incorrectly returns the depth of a node when it encounters a visited node, which does not necessarily indicate a cycle. 

The correct approach involves:
1. Using BFS to traverse the graph and detect cycles.
2. Keeping track of the parent node to avoid revisiting it.
3. If a node is visited that is not the parent, it indicates a cycle, and the sum of the depths of the current node and the encountered node plus one gives the cycle length.
4. If no cycle is found after BFS completes, return -1.

Here's the corrected code:

```cpp
#include <bits/stdc++.h>

using namespace std;
using ll = long long;
using pii = pair<int, int>;

const int N = 2e5 + 5;
vector<int> g[N];
int n, m, dep[N], parent[N];

int bfs() {
    queue<int> q;
    q.push(1);
    dep[1] = 1;
    parent[1] = -1;

    while (!q.empty()) {
        int u = q.front();
        q.pop();

        for (auto v : g[u]) {
            if (v == parent[u]) continue;
            if (dep[v] == 0) {
                dep[v] = dep[u] + 1;
                parent[v] = u;
                q.push(v);
            } else {
                return dep[u] + dep[v] - 1;
            }
        }
    }
    return -1;
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);
    cout.tie(nullptr);

    cin >> n >> m;
    for (int i = 1; i <= m; i++) {
        int u, v;
        cin >> u >> v;
        g[u].push_back(v);
        g[v].push_back(u);
    }

    cout << bfs() << '\n';
    return 0;
}
```

1. Added `parent[N]` array to keep track of the parent of each node during BFS.
2. Modified the BFS to properly handle undirected edges by adding both `u->v` and `v->u` to the adjacency list.
3. Adjusted the cycle detection logic to return the sum of the depths of the two nodes involved in the cycle minus one, which gives the length of the cycle.
