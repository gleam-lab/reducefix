After analyzing the code, I found several issues that need to be fixed:

1. The algorithm is trying to find the shortest cycle containing node 1, but the current DFS approach doesn't correctly handle this. DFS isn't suitable for finding shortest paths/cycles in unweighted graphs - BFS would be better.

2. The bitset `bj` is used for visited marking but isn't properly reset when backtracking, which can prevent finding the shortest path.

3. The initial distance for node 1 is incorrectly set to a large value (100000000) when it should be 0.

Here's the fixed code using BFS to find the shortest cycle containing node 1:

```cpp
#include <iostream>
#include <vector>
#include <queue>
#include <climits>
using namespace std;

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);
    
    int n, m;
    cin >> n >> m;
    
    vector<vector<int>> adj(n+1);
    for (int i = 0; i < m; ++i) {
        int u, v;
        cin >> u >> v;
        adj[u].push_back(v);
    }
    
    vector<int> dist(n+1, INT_MAX);
    queue<pair<int, int>> q; // {node, parent}
    
    // Start BFS from node 1
    dist[1] = 0;
    q.push({1, -1});
    
    int min_cycle = INT_MAX;
    
    while (!q.empty()) {
        auto [u, parent] = q.front();
        q.pop();
        
        for (int v : adj[u]) {
            if (v == 1 && u != parent) { // Found a cycle back to 1
                min_cycle = min(min_cycle, dist[u] + 1);
            }
            if (dist[v] > dist[u] + 1) {
                dist[v] = dist[u] + 1;
                q.push({v, u});
            }
        }
    }
    
    if (min_cycle != INT_MAX) {
        cout << min_cycle << '\n';
    } else {
        cout << "-1\n";
    }
    
    return 0;
}
```

Key improvements:
1. Uses BFS instead of DFS for shortest path finding
2. Properly tracks distances and updates them
3. Checks for cycles back to node 1 by looking for neighbors that are node 1 (while avoiding immediate backtracking)
4. More efficient and correct implementation overall
5. Uses standard input/output for better portability
6. Handles the case when no cycle exists (outputs -1)

The code now correctly finds the shortest cycle containing node 1 in an unweighted directed graph.
