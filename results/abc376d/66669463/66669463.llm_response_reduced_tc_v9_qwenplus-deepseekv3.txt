To fix this code, we need to correctly compute the shortest cycle that starts and ends at node 1. The original code uses a depth-first search (DFS) approach, which is not suitable for finding shortest paths in graphs with cycles. Instead, we should use a breadth-first search (BFS) approach, which is more appropriate for finding shortest paths in unweighted graphs.

### Approaches

1. **Graph Representation**: Represent the graph using an adjacency list.
2. **BFS for Shortest Path**: Use BFS to explore the graph starting from node 1. For each node, keep track of the shortest distance from node 1.
3. **Cycle Detection**: For node 1, if it is reachable from any of its neighbors, the cycle length will be the distance to that neighbor plus 1.
4. **Edge Cases**: Handle cases where there is no cycle involving node 1 by returning -1.

### Solution Code

```cpp
#include <algorithm>
#include <iostream>
#include <queue>
#include <vector>
#include <climits>
using namespace std;

int main() {
    int n, m;
    cin >> n >> m;
    vector<vector<int>> adj(n + 1);
    for (int i = 0; i < m; ++i) {
        int u, v;
        cin >> u >> v;
        adj[u].push_back(v);
    }

    vector<int> dist(n + 1, INT_MAX);
    queue<int> q;
    q.push(1);
    dist[1] = 0;
    int min_cycle = INT_MAX;

    while (!q.empty()) {
        int u = q.front();
        q.pop();
        for (int v : adj[u]) {
            if (v == 1) {
                min_cycle = min(min_cycle, dist[u] + 1);
            }
            if (dist[v] > dist[u] + 1) {
                dist[v] = dist[u] + 1;
                q.push(v);
            }
        }
    }

    if (min_cycle != INT_MAX) {
        cout << min_cycle << endl;
    } else {
        cout << -1 << endl;
    }

    return 0;
}
```

### Explanation

1. **Graph Representation**: The graph is stored using an adjacency list `adj`, where each node points to its neighbors.
2. **BFS Initialization**: Initialize the distance array `dist` with `INT_MAX` to represent initially unreachable nodes. The distance from node 1 to itself is set to 0.
3. **BFS Execution**: The BFS starts from node 1. For each node `u` dequeued, its neighbors `v` are examined:
   - If `v` is node 1, it means a cycle is found. The cycle length is `dist[u] + 1`.
   - If a shorter path to `v` is found (`dist[v] > dist[u] + 1`), update `dist[v]` and enqueue `v`.
4. **Result Handling**: After BFS completes, if a cycle was found (`min_cycle` is updated), print its length; otherwise, print -1.

This approach ensures that the shortest cycle involving node 1 is found efficiently using BFS, which is optimal for unweighted graphs.
