To fix this code, we need to address the issue of finding the shortest cycle starting and ending at node 1. The original code attempts to use DFS to explore paths, but it may not correctly handle all cases, especially when multiple paths exist or when cycles involve more nodes.

### Approaches
1. **Breadth-First Search (BFS)** is more suitable for unweighted shortest path problems. We can use BFS to explore the graph from node 1, keeping track of distances and checking for cycles back to node 1.
2. **Distance Tracking**: For each node, we maintain the shortest distance from node 1. If during the BFS we encounter node 1 again, we have found a cycle, and we can update the shortest cycle length if the current path is shorter.
3. **Visited Handling**: Ensure nodes are revisited only if a shorter path is found, which is crucial for correctness.

### Solution Code
```cpp
#include <iostream>
#include <vector>
#include <queue>
#include <climits>
using namespace std;

int main() {
    int n, m;
    cin >> n >> m;
    vector<vector<int>> adj(n + 1);
    for (int i = 0; i < m; ++i) {
        int u, v;
        cin >> u >> v;
        adj[u].push_back(v);
    }

    vector<int> dist(n + 1, INT_MAX);
    queue<int> q;
    dist[1] = 0;
    q.push(1);

    int min_cycle = INT_MAX;

    while (!q.empty()) {
        int u = q.front();
        q.pop();

        for (int v : adj[u]) {
            if (v == 1) {
                if (dist[u] + 1 < min_cycle) {
                    min_cycle = dist[u] + 1;
                }
            }
            if (dist[v] > dist[u] + 1) {
                dist[v] = dist[u] + 1;
                q.push(v);
            }
        }
    }

    if (min_cycle != INT_MAX) {
        cout << min_cycle << endl;
    } else {
        cout << -1 << endl;
    }

    return 0;
}
```

### Explanation
1. **Graph Representation**: The graph is represented using an adjacency list `adj`, where each node points to its neighbors.
2. **BFS Initialization**: We initialize the distance array `dist` with `INT_MAX` to represent initially unreachable nodes. The distance from node 1 to itself is set to 0, and node 1 is enqueued.
3. **BFS Execution**: For each node dequeued, we examine its neighbors. If a neighbor is node 1, we check if the path through the current node forms a shorter cycle. For other nodes, if a shorter path is found, we update their distance and enqueue them.
4. **Cycle Detection**: The shortest cycle length is kept in `min_cycle`. If no cycle is found (i.e., `min_cycle` remains `INT_MAX`), we output -1; otherwise, we output the shortest cycle length.

This approach ensures we efficiently find the shortest cycle involving node 1 by leveraging BFS, which inherently explores nodes level by level, thus guaranteeing the shortest path in an unweighted graph.
