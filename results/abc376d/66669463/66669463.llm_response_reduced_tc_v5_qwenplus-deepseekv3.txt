To fix this code, we need to address the issue that causes the incorrect output for the given test case. The problem involves finding the shortest cycle that includes node 1 in a directed graph. The original code attempts to use a depth-first search (DFS) approach, but it fails to correctly handle the detection of cycles, especially those involving node 1, leading to incorrect cycle lengths.

### Approaches

1. **Algorithm Selection**: 
   - Use Breadth-First Search (BFS) for each neighbor of node 1 to find the shortest cycle that includes node 1. This is because BFS is suitable for finding shortest paths in unweighted graphs.
   - For each neighbor `v` of node 1, perform a BFS to see if node 1 can be reached from `v`. The shortest cycle length will be `1 + the shortest path from v back to 1`.

2. **Complexity Analysis**:
   - The BFS runs in O(N + M) time for each neighbor of node 1. In the worst case, node 1 has O(N) neighbors, making the total complexity O(N*(N + M)). However, in practice, this is manageable if the graph is sparse.

3. **Edge Cases**:
   - Handle cases where there is no cycle involving node 1 by returning `-1`.
   - Ensure the graph is correctly represented with adjacency lists.

### Solution Code

```cpp
#include <iostream>
#include <vector>
#include <queue>
#include <climits>
using namespace std;

const int MAX_N = 200001;

vector<int> adj[MAX_N];
int dist[MAX_N];

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);

    int n, m;
    cin >> n >> m;

    for (int i = 0; i < m; ++i) {
        int u, v;
        cin >> u >> v;
        adj[u].push_back(v);
    }

    int min_cycle = INT_MAX;

    for (int v : adj[1]) {
        if (v == 1) {
            min_cycle = 1;
            continue;
        }

        fill(dist, dist + n + 1, -1);
        queue<int> q;
        q.push(v);
        dist[v] = 1;

        while (!q.empty()) {
            int u = q.front();
            q.pop();

            for (int next : adj[u]) {
                if (next == 1) {
                    if (dist[u] + 1 < min_cycle) {
                        min_cycle = dist[u] + 1;
                    }
                    break;
                }
                if (dist[next] == -1) {
                    dist[next] = dist[u] + 1;
                    q.push(next);
                }
            }
            if (min_cycle == 2) break; // Early exit if the minimal possible cycle is found
        }
        if (min_cycle == 2) break; // Early exit if the minimal possible cycle is found
    }

    if (min_cycle != INT_MAX) {
        cout << min_cycle << '\n';
    } else {
        cout << "-1\n";
    }

    return 0;
}
```

### Explanation

1. **Graph Representation**: The graph is represented using adjacency lists for efficient traversal.
2. **BFS for Shortest Path**: For each neighbor of node 1, a BFS is initiated to explore paths back to node 1. The distance from the neighbor to node 1 is recorded.
3. **Cycle Detection**: If during BFS, node 1 is reached from a neighbor, the cycle length is noted as `1 (the edge from 1 to neighbor) + distance from neighbor back to 1`.
4. **Early Termination**: The algorithm checks for the smallest possible cycle (length 2) early to optimize performance.
5. **Result Handling**: After checking all neighbors, the shortest cycle length is output, or `-1` if no cycle exists.

This approach ensures correctness and efficiency, handling all specified edge cases and constraints.
