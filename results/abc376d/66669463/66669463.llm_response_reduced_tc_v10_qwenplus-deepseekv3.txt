To fix this code, we need to address the issue related to finding the shortest cycle that includes node 1 in a directed graph. The original code attempts to use a depth-first search (DFS) approach, but it fails to correctly identify the shortest cycle due to improper handling of visited nodes and relaxation of distances.

### Approaches

1. **Graph Representation**: Represent the graph using an adjacency list for efficient traversal.
2. **BFS for Shortest Path**: Since BFS is more suitable for finding the shortest path in unweighted graphs, we can use a modified BFS approach to track the shortest cycle involving node 1.
3. **Tracking Distances**: Maintain a distance array to keep track of the shortest path distances from node 1 to all other nodes. Also, maintain a parent array to reconstruct paths.
4. **Cycle Detection**: For each node visited, check if it has a back edge to node 1. The shortest cycle will be the sum of the distance from node 1 to the current node plus 1 (the edge back to node 1).

### Solution Code

```cpp
#include <iostream>
#include <vector>
#include <queue>
#include <climits>
using namespace std;

const int MAXN = 200001;

vector<int> adj[MAXN];
int dist[MAXN];
int parent[MAXN];

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);

    int n, m;
    cin >> n >> m;

    for (int i = 1; i <= n; ++i) {
        adj[i].clear();
        dist[i] = INT_MAX;
        parent[i] = -1;
    }

    for (int i = 0; i < m; ++i) {
        int u, v;
        cin >> u >> v;
        adj[u].push_back(v);
    }

    queue<int> q;
    q.push(1);
    dist[1] = 0;
    int min_cycle = INT_MAX;

    while (!q.empty()) {
        int u = q.front();
        q.pop();

        for (int v : adj[u]) {
            if (v == 1) {
                if (dist[u] + 1 < min_cycle) {
                    min_cycle = dist[u] + 1;
                }
            }
            if (dist[v] > dist[u] + 1) {
                dist[v] = dist[u] + 1;
                parent[v] = u;
                q.push(v);
            }
        }
    }

    if (min_cycle != INT_MAX) {
        cout << min_cycle << '\n';
    } else {
        cout << "-1\n";
    }

    return 0;
}
```

### Explanation

1. **Graph Representation**: The graph is represented using an adjacency list where each node points to its neighbors.
2. **Initialization**: The distance array is initialized to `INT_MAX` (infinity) to represent initially unreachable nodes, and the parent array is initialized to -1 to denote no parent initially.
3. **BFS Setup**: The BFS starts from node 1 with its distance set to 0.
4. **BFS Execution**: For each node processed, its neighbors are checked. If a neighbor is node 1, it means a potential cycle is found. The cycle length is updated if a shorter path is found.
5. **Relaxation**: For other nodes, if a shorter path is found, their distance is updated, and they are added to the queue for further processing.
6. **Result**: After processing all nodes, if a cycle involving node 1 is found, the shortest cycle length is printed; otherwise, -1 is printed indicating no such cycle exists.

This approach ensures that we efficiently find the shortest cycle involving node 1 using BFS, which is optimal for unweighted graphs. The complexity is O(N + M), where N is the number of nodes and M is the number of edges, making it suitable for large inputs.
