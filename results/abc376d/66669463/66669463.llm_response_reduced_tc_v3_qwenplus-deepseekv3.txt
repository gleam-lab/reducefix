To fix this code, we need to correctly find the shortest cycle that starts and ends at node 1. The original code attempts to use a depth-first search (DFS) approach, but DFS is not suitable for finding shortest paths in unweighted graphs; breadth-first search (BFS) is more appropriate for this purpose.

### Approaches
1. **Graph Representation**: Represent the graph using an adjacency list.
2. **BFS for Shortest Path**: Use BFS to explore the graph level by level from node 1. This ensures that the first time we revisit node 1, we have found the shortest cycle.
3. **Tracking Distance**: Maintain a distance array to keep track of the shortest distance from node 1 to every other node. When we encounter node 1 again during BFS, the current distance plus one gives the length of the shortest cycle.

### Solution Code
```cpp
#include <iostream>
#include <vector>
#include <queue>
#include <climits>
using namespace std;

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);

    int n, m;
    cin >> n >> m;

    vector<vector<int>> adj(n + 1);
    for (int i = 0; i < m; ++i) {
        int u, v;
        cin >> u >> v;
        adj[u].push_back(v);
    }

    vector<int> dist(n + 1, INT_MAX);
    queue<int> q;
    dist[1] = 0;
    q.push(1);

    int shortest_cycle = INT_MAX;

    while (!q.empty()) {
        int u = q.front();
        q.pop();

        for (int v : adj[u]) {
            if (v == 1) {
                shortest_cycle = min(shortest_cycle, dist[u] + 1);
            }
            if (dist[v] > dist[u] + 1) {
                dist[v] = dist[u] + 1;
                q.push(v);
            }
        }
    }

    if (shortest_cycle != INT_MAX) {
        cout << shortest_cycle << '\n';
    } else {
        cout << "-1\n";
    }

    return 0;
}
```

### Explanation
1. **Graph Representation**: The graph is stored using an adjacency list `adj`, where each node points to a list of nodes it is connected to.
2. **BFS Initialization**: Initialize a queue with node 1, and set its distance to 0. All other nodes start with a distance of infinity (INT_MAX).
3. **BFS Execution**: For each node dequeued, explore its neighbors. If a neighbor is node 1, update the shortest cycle length if the current path provides a shorter cycle.
4. **Distance Update**: For other neighbors, if a shorter path is found, update their distance and enqueue them for further exploration.
5. **Result**: After BFS completes, if a cycle was found (shortest_cycle is not INT_MAX), print its length; otherwise, print -1.

This approach efficiently finds the shortest cycle starting and ending at node 1 using BFS, which is optimal for unweighted graphs.
