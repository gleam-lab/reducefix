To fix this code, we need to ensure that the algorithm correctly finds the shortest cycle starting and ending at node 1. The original code uses a depth-first search (DFS) approach, which is not suitable for finding shortest paths in unweighted graphs (BFS is more appropriate). 

### Analysis
1. **Problem Understanding**: The task is to find the shortest cycle that starts and ends at node 1. This means we need to find the shortest path from node 1 back to itself through any other nodes.
2. **Issue with DFS**: DFS explores as far as possible along each branch before backtracking, which doesn't guarantee the shortest path in an unweighted graph. BFS, on the other hand, explores all nodes at the present depth before moving on to nodes at the next depth level, making it suitable for shortest path problems in unweighted graphs.
3. **Correct Approach**: Use BFS to explore the graph from node 1, keeping track of visited nodes and distances. When we encounter node 1 again during BFS, we can determine the length of the cycle.

### Solution Code
```cpp
#include <iostream>
#include <vector>
#include <queue>
#include <climits>
using namespace std;

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);
    
    int n, m;
    cin >> n >> m;
    
    vector<vector<int>> adj(n + 1);
    for (int i = 0; i < m; ++i) {
        int u, v;
        cin >> u >> v;
        adj[u].push_back(v);
    }
    
    vector<int> dist(n + 1, INT_MAX);
    queue<int> q;
    q.push(1);
    dist[1] = 0;
    int min_cycle = INT_MAX;
    
    while (!q.empty()) {
        int u = q.front();
        q.pop();
        
        for (int v : adj[u]) {
            if (v == 1) {
                min_cycle = min(min_cycle, dist[u] + 1);
            }
            if (dist[v] > dist[u] + 1) {
                dist[v] = dist[u] + 1;
                q.push(v);
            }
        }
    }
    
    if (min_cycle != INT_MAX) {
        cout << min_cycle << '\n';
    } else {
        cout << "-1\n";
    }
    
    return 0;
}
```

### Explanation
1. **Input Handling**: The code reads the number of nodes `n` and edges `m`, followed by the edges themselves, constructing an adjacency list `adj`.
2. **BFS Initialization**: A queue `q` is initialized with node 1, and a distance vector `dist` is set up to keep track of the shortest distance from node 1 to all other nodes, initialized to `INT_MAX` (infinity) except for node 1, which is 0.
3. **BFS Execution**: For each node `u` dequeued, the code checks all its neighbors `v`. If `v` is node 1, it updates `min_cycle` with the current path length (`dist[u] + 1`). Otherwise, if a shorter path to `v` is found, it updates `dist[v]` and enqueues `v`.
4. **Result Output**: After BFS completes, if `min_cycle` was updated, it prints the shortest cycle length; otherwise, it prints `-1` indicating no cycle was found.

This approach ensures that the shortest cycle starting and ending at node 1 is found efficiently using BFS, which is optimal for unweighted graphs.
