To fix this code, we need to analyze the problem and the incorrect solution. The problem involves finding the shortest cycle that starts and ends at node 1 in a directed graph. The incorrect code attempts to use DFS (Depth First Search) to find this cycle, but it fails in some cases, as seen in the failing test case.

### Analysis
1. **Problem Understanding**: The goal is to find the shortest cycle that starts and ends at node 1 in a directed graph. This means we need to find the shortest path from node 1 back to itself.
2. **Incorrect Approach**: The original code uses DFS with a visited array to track visited nodes and attempts to find the shortest cycle by keeping track of the current path length. However, DFS is not suitable for finding shortest paths in unweighted graphs because it explores one path fully before backtracking, which can miss shorter paths.
3. **Correct Approach**: BFS (Breadth First Search) is more appropriate for finding shortest paths in unweighted graphs. We can use BFS to explore all possible paths from node 1 and keep track of the shortest cycle found.

### Solution Code
```cpp
#include<bits/stdc++.h>
#define ll long long
using namespace std;

ll n, m;
vector<int> adj[200005];
ll vis[200005];
ll dist[200005];

ll bfs() {
    queue<pair<int, int>> q;
    q.push({1, 0});
    dist[1] = 0;
    vis[1] = 1;
    ll mn = 200005;

    while (!q.empty()) {
        int u = q.front().first;
        int d = q.front().second;
        q.pop();

        for (int v : adj[u]) {
            if (v == 1) {
                mn = min(mn, (ll)d + 1);
                continue;
            }
            if (vis[v] == 0) {
                vis[v] = 1;
                dist[v] = d + 1;
                q.push({v, d + 1});
            }
        }
    }
    return mn == 200005 ? -1 : mn;
}

int main() {
    cin >> n >> m;
    for (int i = 1; i <= m; i++) {
        int u, v;
        cin >> u >> v;
        adj[u].push_back(v);
    }
    ll ans = bfs();
    cout << ans << endl;
    return 0;
}
```

### Explanation
1. **Input Handling**: The code reads the number of nodes `n` and edges `m`, followed by the edges themselves, which are stored in an adjacency list `adj`.
2. **BFS Initialization**: The BFS starts from node 1 with an initial distance of 0. The `vis` array keeps track of visited nodes to avoid revisiting and cycles, and the `dist` array stores the shortest distance from node 1 to each node.
3. **BFS Execution**: For each node `u` dequeued, the code checks all its neighbors `v`. If a neighbor `v` is node 1, it updates the minimum cycle length `mn` if the current path length plus 1 is smaller. Otherwise, if the neighbor hasn't been visited, it marks it as visited, sets its distance, and enqueues it.
4. **Result Output**: After BFS completes, the code checks if a cycle was found (`mn` was updated) and prints the result. If no cycle was found, it prints -1.

This approach ensures that the shortest cycle is found efficiently using BFS, which is optimal for unweighted graphs. The complexity is O(N + M), where N is the number of nodes and M is the number of edges, making it suitable for large inputs.
