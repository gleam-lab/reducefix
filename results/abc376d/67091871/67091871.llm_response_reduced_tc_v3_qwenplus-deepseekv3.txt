To fix this code, we need to address the issue that the current approach does not correctly find the shortest cycle involving node 1. The incorrect code attempts to use a depth-first search (DFS) to find cycles, but DFS is not suitable for finding the shortest path in an unweighted graph; instead, breadth-first search (BFS) is more appropriate for this purpose.

### Approaches
1. **Graph Representation**: Use an adjacency list to represent the graph.
2. **BFS for Shortest Cycle**: 
   - Perform a BFS starting from node 1 to find the shortest cycle that returns to node 1.
   - Track the distance from node 1 to all other nodes.
   - For each neighbor of node 1, if the neighbor has already been visited (i.e., has a distance assigned), the cycle length is the sum of the distances from node 1 to the neighbor plus 1 (the edge back to node 1).
3. **Edge Handling**: Ensure that all edges are considered, and handle bidirectional edges if necessary (though the problem seems to consider directed edges based on the input format).

### Solution Code
```cpp
#include<bits/stdc++.h>
#define ll long long
using namespace std;

int main(){
    ll n, m;
    cin >> n >> m;
    vector<vector<int>> adj(n + 1);
    for(int i = 0; i < m; i++){
        int u, v;
        cin >> u >> v;
        adj[u].push_back(v);
    }

    vector<int> dist(n + 1, -1);
    queue<int> q;
    q.push(1);
    dist[1] = 0;
    int min_cycle = INT_MAX;

    while(!q.empty()){
        int u = q.front();
        q.pop();
        for(int v : adj[u]){
            if(v == 1){
                min_cycle = min(min_cycle, dist[u] + 1);
            }
            if(dist[v] == -1){
                dist[v] = dist[u] + 1;
                q.push(v);
            }
        }
    }

    if(min_cycle == INT_MAX){
        cout << -1 << endl;
    } else {
        cout << min_cycle << endl;
    }
    return 0;
}
```

### Explanation
1. **Graph Representation**: The graph is represented using an adjacency list `adj`, where each node points to its neighbors.
2. **BFS Initialization**: We initialize a queue to perform BFS starting from node 1. The `dist` array keeps track of the shortest distance from node 1 to each node, initialized to -1 (indicating unvisited nodes).
3. **BFS Execution**: For each node processed, we check its neighbors. If a neighbor is node 1, it means we have found a cycle, and the cycle length is the current distance plus 1. If the neighbor hasn't been visited, we update its distance and enqueue it.
4. **Result Handling**: After BFS completes, if no cycle involving node 1 is found, we output -1. Otherwise, we output the shortest cycle length found.

This approach ensures that we efficiently find the shortest cycle involving node 1 using BFS, which is optimal for unweighted graphs. The complexity is O(N + M), where N is the number of nodes and M is the number of edges, making it suitable for large inputs.
