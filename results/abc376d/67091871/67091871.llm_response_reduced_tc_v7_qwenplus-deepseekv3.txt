I analyzed the incorrect code and the failing test case. The issue is that the code is trying to find the shortest cycle involving node 1 using DFS, but DFS isn't the right approach for shortest path problems - BFS would be more appropriate. Also, the visitation logic was flawed.

Here's the corrected code using BFS to find the shortest cycle containing node 1:

```cpp
#include<bits/stdc++.h>
#define ll long long
using namespace std;

int main(){
    ll n, m;
    cin >> n >> m;
    vector<vector<int>> adj(n+1);
    
    for(int i=0; i<m; i++){
        int u, v;
        cin >> u >> v;
        adj[u].push_back(v);
    }
    
    queue<pair<int,int>> q;
    vector<int> dist(n+1, -1);
    int min_cycle = INT_MAX;
    
    // Start BFS from node 1
    q.push({1, 0});
    dist[1] = 0;
    
    while(!q.empty()){
        auto [current, d] = q.front();
        q.pop();
        
        for(int neighbor : adj[current]){
            if(neighbor == 1){
                // Found a cycle back to node 1
                min_cycle = min(min_cycle, d + 1);
                continue;
            }
            if(dist[neighbor] == -1 || d + 1 < dist[neighbor]){
                dist[neighbor] = d + 1;
                q.push({neighbor, d + 1});
            }
        }
    }
    
    if(min_cycle == INT_MAX){
        cout << -1;
    }
    else{
        cout << min_cycle;
    }
    
    return 0;
}
```

Key improvements:
1. Replaced DFS with BFS which is better suited for shortest path problems
2. Properly initialized distance array and queue
3. Correctly handles cycles by checking for node 1 in neighbors
4. Only updates distances when a shorter path is found
5. Simplified the data structures and logic

This solution will correctly find the shortest cycle containing node 1 by:
- Starting BFS from node 1
- Whenever we find a neighbor that is node 1, we update the minimum cycle length
- Otherwise, we continue the BFS normally
- This ensures we find the shortest cycle in O(N + M) time complexity
