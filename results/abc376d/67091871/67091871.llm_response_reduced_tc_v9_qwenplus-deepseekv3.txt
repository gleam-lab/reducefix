To fix this code, we need to address the issue that the current implementation incorrectly handles cycles and does not properly track the shortest path back to node 1. The problem requires finding the shortest cycle that starts and ends at node 1, which is essentially the shortest path from node 1 back to itself through any directed path.

### Approaches

1. **Breadth-First Search (BFS)**:
   - BFS is suitable here because it naturally explores nodes level by level, which helps in finding the shortest path in an unweighted graph.
   - We start BFS from node 1 and keep track of the distance from node 1 to every other node.
   - For every neighbor of node 1, if the neighbor can reach node 1 directly or through other nodes, we calculate the total distance which forms a cycle (distance from node 1 to the neighbor + 1 to come back to node 1).
   - The minimum of all such cycles gives the answer.

2. **Handling Cycles**:
   - We need to ensure that we do not revisit nodes unnecessarily, which can lead to incorrect shortest path calculations.
   - Using a distance array initialized to a large value (infinity) helps in updating distances only when a shorter path is found.

### Solution Code

```cpp
#include<bits/stdc++.h>
#define ll long long
using namespace std;

int main() {
    ll n, m;
    cin >> n >> m;
    vector<vector<int>> adj(n + 1);
    for (int i = 0; i < m; ++i) {
        int u, v;
        cin >> u >> v;
        adj[u].push_back(v);
    }

    vector<ll> dist(n + 1, -1);
    queue<int> q;
    q.push(1);
    dist[1] = 0;
    ll min_cycle = LLONG_MAX;

    while (!q.empty()) {
        int u = q.front();
        q.pop();

        for (int v : adj[u]) {
            if (v == 1) {
                min_cycle = min(min_cycle, dist[u] + 1);
            }
            if (dist[v] == -1) {
                dist[v] = dist[u] + 1;
                q.push(v);
            }
        }
    }

    if (min_cycle != LLONG_MAX) {
        cout << min_cycle << endl;
    } else {
        cout << -1 << endl;
    }

    return 0;
}
```

### Explanation

1. **Graph Representation**:
   - The graph is represented using an adjacency list `adj`, where `adj[u]` contains all nodes `v` such that there is a directed edge from `u` to `v`.

2. **BFS Initialization**:
   - A queue `q` is initialized to start BFS from node 1. The distance array `dist` is initialized to -1 (indicating unvisited nodes), except for node 1 which is set to 0 (distance from itself).

3. **BFS Execution**:
   - For each node `u` dequeued, we examine all its neighbors `v`.
   - If a neighbor `v` is node 1 itself, it means we have found a cycle (path from 1 to `u` and back to 1). The cycle length is `dist[u] + 1`, and we update `min_cycle` if this length is shorter than the current value.
   - For other neighbors, if they haven't been visited (`dist[v] == -1`), we set their distance and enqueue them for further exploration.

4. **Result Output**:
   - After BFS completes, if `min_cycle` was updated (i.e., a cycle was found), its value is printed. Otherwise, -1 is printed indicating no cycle exists.

This approach ensures that we efficiently find the shortest cycle involving node 1 using BFS, which operates in O(N + M) time, where N is the number of nodes and M is the number of edges.
