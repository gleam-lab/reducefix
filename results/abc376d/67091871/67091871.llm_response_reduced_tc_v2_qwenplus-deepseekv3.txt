To fix this code, we need to correctly identify the shortest cycle that starts and ends at node 1. The original code attempts to use a DFS approach, but it has a flaw in how it checks for the cycle and calculates the path length. The key issue is that the DFS doesn't properly track the shortest path and can revisit nodes incorrectly, leading to longer path counts than necessary.

### Approaches
1. **Breadth-First Search (BFS)**: BFS is more suitable for finding the shortest path in unweighted graphs. We'll use BFS to explore all nodes level by level from node 1, keeping track of the distance from node 1.
2. **Cycle Detection**: For each neighbor of node 1, if that neighbor can reach node 1, the cycle length would be the distance from node 1 to the neighbor plus 1 (the edge back to node 1).
3. **Early Termination**: If we detect a cycle involving node 1 during the BFS, we can terminate early if we find the shortest possible cycle (the smallest cycle would be distance 2, but we look for the smallest actual cycle length).

### Solution Code
```cpp
#include<bits/stdc++.h>
#define ll long long
using namespace std;

int main(){
    ll n, m;
    cin >> n >> m;
    vector<vector<int>> adj(n + 1);
    for(int i = 0; i < m; i++){
        int u, v;
        cin >> u >> v;
        adj[u].push_back(v);
    }

    vector<int> dist(n + 1, -1);
    queue<int> q;
    q.push(1);
    dist[1] = 0;
    int min_cycle = INT_MAX;

    while(!q.empty()){
        int u = q.front();
        q.pop();
        for(int v : adj[u]){
            if(v == 1){
                min_cycle = min(min_cycle, dist[u] + 1);
            }
            if(dist[v] == -1){
                dist[v] = dist[u] + 1;
                q.push(v);
            }
        }
    }

    if(min_cycle != INT_MAX){
        cout << min_cycle;
    } else {
        cout << -1;
    }
    return 0;
}
```

### Explanation
1. **Graph Representation**: The graph is represented using an adjacency list `adj` where each node points to its neighbors.
2. **BFS Initialization**: We initialize a queue for BFS starting at node 1. The `dist` array keeps track of the distance from node 1 to all other nodes, initialized to -1 (unvisited) except for node 1 which is 0.
3. **BFS Execution**: For each node `u` dequeued, we explore its neighbors. If a neighbor `v` is node 1, we update `min_cycle` with the current path length (distance to `u` + 1). For other unvisited nodes, we set their distance and enqueue them.
4. **Result Output**: After BFS completes, if a cycle was found (`min_cycle` is not `INT_MAX`), we print the shortest cycle length; otherwise, we print -1 indicating no cycle.

This approach ensures that we efficiently find the shortest cycle involving node 1 using BFS, which is optimal for shortest path problems in unweighted graphs.
