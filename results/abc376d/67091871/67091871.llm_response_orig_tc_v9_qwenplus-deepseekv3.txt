After analyzing the incorrect code and the failing test case, I identified that the problem is to find the shortest cycle starting and ending at node 1 in a directed graph. The original code was using a DFS approach which isn't optimal for this problem since DFS doesn't naturally find the shortest path in an unweighted graph. 

Here's the fixed code using BFS (Breadth-First Search) which is more suitable for finding shortest paths in unweighted graphs:

```cpp
#include<bits/stdc++.h>
#define ll long long
using namespace std;

ll n, m;
vector<int> b[200005];
ll dist[200005];

ll bfs() {
    queue<int> q;
    memset(dist, -1, sizeof(dist));
    dist[1] = 0;
    q.push(1);
    
    ll min_cycle = 1e9;
    
    while (!q.empty()) {
        int u = q.front();
        q.pop();
        
        for (int v : b[u]) {
            if (v == 1) {
                min_cycle = min(min_cycle, dist[u] + 1);
            }
            if (dist[v] == -1) {
                dist[v] = dist[u] + 1;
                q.push(v);
            }
        }
    }
    
    return min_cycle == 1e9 ? -1 : min_cycle;
}

int main() {
    cin >> n >> m;
    for (int i = 1; i <= m; i++) {
        int u, v;
        cin >> u >> v;
        b[u].push_back(v);
    }
    
    ll result = bfs();
    cout << result << endl;
    
    return 0;
}
```

The key improvements in this solution are:

1. Using BFS instead of DFS to find the shortest cycle
2. Maintaining a distance array to track distances from node 1
3. When we encounter node 1 during the BFS (meaning we found a cycle back to node 1), we update the minimum cycle length
4. Proper initialization of the distance array with -1 (unvisited)
5. Correct handling of the case where no cycle exists (returning -1)

This approach guarantees that we find the shortest cycle containing node 1 (if one exists) in O(N + M) time complexity, which is optimal for this problem. The BFS naturally explores nodes level by level, ensuring the first time we find a path back to node 1 will be the shortest such path.
