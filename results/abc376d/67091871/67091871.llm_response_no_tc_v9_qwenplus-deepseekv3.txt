2. The condition `if(vis[b[x][i]]==0||b[x][i]==1)` is incorrect - it allows revisiting node 1 but not other nodes
3. There's no proper backtracking mechanism (unmarking visited nodes)
4. The initial visit to node 1 isn't properly handled
5. The minimum cycle detection logic is flawed

Here's the corrected version using BFS (more suitable for shortest path problems) with proper cycle detection:

```cpp
#include<bits/stdc++.h>
#define ll long long
using namespace std;

int main(){
    ll n, m;
    cin >> n >> m;
    vector<int> b[n+1];
    for(int i=1; i<=m; i++){
        int u, v;
        cin >> u >> v;
        b[u].push_back(v);
    }
    
    vector<int> dist(n+1, -1);
    queue<pair<int,int>> q; // {node, parent}
    q.push({1, -1});
    dist[1] = 0;
    int mn = INT_MAX;
    
    while(!q.empty()){
        auto [x, parent] = q.front();
        q.pop();
        
        for(int neighbor : b[x]){
            if(neighbor == 1 && x != 1){ // found a cycle back to 1
                mn = min(mn, dist[x] + 1);
            }
            if(dist[neighbor] == -1 && neighbor != 1){ // not visited and not starting node
                dist[neighbor] = dist[x] + 1;
                q.push({neighbor, x});
            }
        }
    }
    
    if(mn == INT_MAX){
        cout << -1;
    } else {
        cout << mn;
    }
    
    return 0;
}
```

Key improvements:
1. Uses BFS which is more appropriate for finding shortest paths
2. Properly tracks distances from node 1
3. Correctly detects cycles back to node 1
4. Handles the case when no cycle exists (outputs -1)
5. More efficient and won't get stuck in infinite loops
6. Uses proper data structures and initialization

The code now correctly finds the shortest cycle that starts and ends at node 1, or returns -1 if no such cycle exists.
