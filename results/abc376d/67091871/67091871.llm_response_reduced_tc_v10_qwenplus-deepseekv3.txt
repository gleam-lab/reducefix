To fix this code, we need to address the issue of finding the shortest cycle that starts and ends at node 1 in a directed graph. The initial approach uses a depth-first search (DFS) which is not optimal for finding the shortest path in an unweighted graph. Instead, a breadth-first search (BFS) is more suitable as it explores all nodes level by level, ensuring the first time we reach node 1 again will be via the shortest path.

### Approaches
1. **BFS for Shortest Path**: BFS is used because it naturally explores nodes in order of their distance from the starting node, ensuring the shortest path is found first.
2. **Handling Cycles**: Since we are looking for a cycle starting and ending at node 1, we start BFS from node 1 and whenever we encounter node 1 again during the traversal, we check if the current path length is the shortest.
3. **Visited Tracking**: We need to carefully manage the visited nodes to avoid reprocessing them unnecessarily while ensuring we do not miss any potential shorter cycles.

### Solution Code
```cpp
#include<bits/stdc++.h>
#define ll long long
using namespace std;

int main() {
    ll n, m;
    cin >> n >> m;
    vector<vector<int>> adj(n + 1);
    for (int i = 0; i < m; i++) {
        int u, v;
        cin >> u >> v;
        adj[u].push_back(v);
    }

    queue<pair<int, int>> q;
    vector<int> dist(n + 1, -1);
    q.push({1, 1});
    dist[1] = 0;
    int min_cycle = INT_MAX;

    while (!q.empty()) {
        auto [u, parent] = q.front();
        q.pop();
        for (int v : adj[u]) {
            if (v == 1 && u != 1) {
                if (min_cycle > dist[u] + 1) {
                    min_cycle = dist[u] + 1;
                }
            }
            if (dist[v] == -1) {
                dist[v] = dist[u] + 1;
                q.push({v, u});
            }
        }
    }

    if (min_cycle != INT_MAX) {
        cout << min_cycle << endl;
    } else {
        cout << -1 << endl;
    }

    return 0;
}
```

### Explanation
1. **Graph Construction**: The graph is built using an adjacency list where each node points to its neighbors.
2. **BFS Initialization**: The BFS starts from node 1 with an initial distance of 0. The queue stores pairs of the current node and its parent to avoid immediate backtracking.
3. **Cycle Detection**: During BFS, if node 1 is encountered again (and it's not directly from node 1 itself), the current path length (distance from node 1 to the current node plus one more step back to node 1) is checked to see if it's the shortest cycle found so far.
4. **Distance Tracking**: The distances from node 1 to all other nodes are tracked. If a node hasn't been visited before, its distance is updated, and it is added to the queue for further exploration.
5. **Result Output**: After BFS completes, if a cycle was found (i.e., `min_cycle` was updated), the shortest cycle length is printed; otherwise, -1 is printed indicating no cycle exists.

This approach ensures that the shortest cycle is found efficiently using BFS, which is optimal for unweighted graphs.
