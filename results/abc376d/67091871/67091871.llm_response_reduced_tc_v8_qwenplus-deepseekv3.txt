To fix this code, we need to ensure that the algorithm correctly finds the shortest cycle that includes node 1. The original code attempts to perform a DFS starting from node 1, keeping track of the path length and updating the minimum cycle length when it returns to node 1. However, this approach fails to correctly handle cycles that do not immediately return to node 1, leading to incorrect results.

### Approaches
1. **BFS for Shortest Path**: Instead of using DFS, which can go deep before exploring shorter paths, BFS is more suitable for finding the shortest cycle. This is because BFS explores all nodes at the current depth before moving on to nodes at the next depth level, ensuring that the first time we return to node 1, it is via the shortest path.
2. **Tracking Parent Nodes**: To avoid revisiting the parent node and incorrectly forming cycles, we need to track the parent of each node during the BFS traversal.
3. **Cycle Detection**: When we encounter node 1 during BFS (other than the starting node), we calculate the cycle length as the sum of the distances from the current node to node 1 plus 1 (the edge that closes the cycle).

### Solution Code
```cpp
#include<bits/stdc++.h>
#define ll long long
using namespace std;

int main() {
    ll n, m;
    cin >> n >> m;
    vector<vector<int>> adj(n + 1);
    for (int i = 0; i < m; ++i) {
        int u, v;
        cin >> u >> v;
        adj[u].push_back(v);
        adj[v].push_back(u); // Assuming undirected graph for cycle detection
    }

    queue<pair<int, int>> q; // {node, parent}
    vector<int> dist(n + 1, -1);
    q.push({1, -1});
    dist[1] = 0;
    int min_cycle = INT_MAX;

    while (!q.empty()) {
        auto [current, parent] = q.front();
        q.pop();
        for (int neighbor : adj[current]) {
            if (dist[neighbor] == -1) {
                dist[neighbor] = dist[current] + 1;
                q.push({neighbor, current});
            } else if (neighbor != parent && neighbor == 1) {
                min_cycle = min(min_cycle, dist[current] + 1);
            }
        }
    }

    if (min_cycle == INT_MAX) {
        cout << -1 << endl;
    } else {
        cout << min_cycle << endl;
    }

    return 0;
}
```

### Explanation
1. **Graph Representation**: The graph is represented using an adjacency list. Each node points to its neighboring nodes.
2. **BFS Initialization**: We start BFS from node 1, initializing its distance to 0. The queue stores pairs of nodes and their parent nodes to avoid revisiting the parent.
3. **Cycle Detection**: During BFS, if we encounter node 1 again (other than the starting node), we calculate the cycle length as the distance to the current node plus 1 (the edge back to node 1). The minimum of all such cycle lengths is recorded.
4. **Output**: If no cycle is found, we output -1. Otherwise, we output the shortest cycle length involving node 1.

This approach ensures that we efficiently find the shortest cycle involving node 1 using BFS, which naturally explores shorter paths first, thus guaranteeing correctness.
