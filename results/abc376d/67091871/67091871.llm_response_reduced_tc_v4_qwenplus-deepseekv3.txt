To fix this code, we need to address the issue of finding the shortest cycle that includes node 1. The original code attempts to use a DFS approach, but it fails to correctly track the shortest path back to node 1. Instead, we should use a BFS approach, which is more suitable for finding the shortest path in an unweighted graph.

### Approaches

1. **Problem Analysis**: The task is to find the shortest cycle that includes node 1. A cycle involving node 1 implies that we start at node 1, traverse some nodes, and return to node 1. The length of the cycle is the number of edges traversed.

2. **Intuition**: Using BFS from node 1, we can explore its neighbors level by level. If we encounter node 1 again during the BFS, the current level indicates the length of the shortest cycle. 

3. **Algorithm Selection**: BFS is chosen because it explores nodes level by level, ensuring that the first time we revisit node 1, we have found the shortest cycle.

4. **Complexity Analysis**: The BFS runs in O(N + M) time, where N is the number of nodes and M is the number of edges, which is efficient for the problem constraints.

### Solution Code
```cpp
#include<bits/stdc++.h>
#define ll long long
using namespace std;

int main() {
    ll n, m;
    cin >> n >> m;
    vector<vector<int>> adj(n + 1);
    for (int i = 0; i < m; i++) {
        int u, v;
        cin >> u >> v;
        adj[u].push_back(v);
    }
    
    queue<pair<int, int>> q;
    vector<int> dist(n + 1, -1);
    q.push({1, 1});
    dist[1] = 0;
    int min_cycle = -1;
    
    while (!q.empty()) {
        auto current = q.front();
        q.pop();
        int u = current.first;
        int parent = current.second;
        
        for (int v : adj[u]) {
            if (v == 1 && parent != 1) {
                if (min_cycle == -1 || dist[u] + 1 < min_cycle) {
                    min_cycle = dist[u] + 1;
                }
                continue;
            }
            if (dist[v] == -1) {
                dist[v] = dist[u] + 1;
                q.push({v, u});
            }
        }
    }
    
    cout << min_cycle << endl;
    return 0;
}
```

### Explanation

1. **Input Reading**: The code reads the number of nodes `n` and edges `m`. It constructs an adjacency list `adj` to represent the graph.

2. **BFS Initialization**: A queue `q` is initialized to start BFS from node 1. The `dist` array keeps track of the distance from node 1 to each node, initialized to -1 (indicating unvisited nodes). The distance to node 1 is set to 0.

3. **BFS Execution**: The BFS processes each node, exploring all its neighbors. If a neighbor is node 1 and the current node is not its direct parent, it calculates the cycle length as `dist[u] + 1` and updates `min_cycle` if this length is shorter than the current value.

4. **Cycle Detection**: The BFS ensures that the first time node 1 is revisited, the shortest cycle length is recorded. Subsequent revisits only update `min_cycle` if a shorter path is found.

5. **Output**: After processing all nodes, the shortest cycle length is printed. If no cycle is found, -1 is printed.

This approach efficiently finds the shortest cycle involving node 1 by leveraging BFS, ensuring optimal performance and correctness.
