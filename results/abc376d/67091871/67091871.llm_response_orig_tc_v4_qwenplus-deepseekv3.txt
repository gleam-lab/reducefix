The problem involves finding the shortest cycle in a directed graph starting and ending at node 1. The initial approach uses a DFS-based method, which might not be the most efficient or correct for this purpose, especially for large graphs.

### Approaches
1. **BFS for Shortest Path**: The BFS algorithm is more suitable for finding the shortest path in unweighted graphs. We can use BFS to explore the graph level by level from node 1, keeping track of the distance from node 1 to each node. If we encounter node 1 again during the BFS, we can determine the length of the cycle.
2. **Early Termination**: To optimize, we can terminate the BFS as soon as we find a cycle that includes node 1, ensuring we get the shortest cycle possible.

### Solution Code
```cpp
#include<bits/stdc++.h>
#define ll long long
using namespace std;

ll n, m;
vector<int> adj[200005];
ll dist[200005];

ll bfs() {
    queue<int> q;
    q.push(1);
    dist[1] = 0;
    ll min_cycle = LLONG_MAX;

    while (!q.empty()) {
        int u = q.front();
        q.pop();

        for (int v : adj[u]) {
            if (v == 1) {
                min_cycle = min(min_cycle, dist[u] + 1);
            }
            if (dist[v] == -1) {
                dist[v] = dist[u] + 1;
                q.push(v);
            }
        }
    }

    return (min_cycle == LLONG_MAX) ? -1 : min_cycle;
}

int main() {
    cin >> n >> m;
    for (int i = 1; i <= m; i++) {
        int u, v;
        cin >> u >> v;
        adj[u].push_back(v);
    }

    memset(dist, -1, sizeof(dist));
    ll result = bfs();
    cout << result << endl;

    return 0;
}
```

### Explanation
1. **Initialization**: We read the number of nodes `n` and edges `m`, then build the adjacency list for the graph.
2. **BFS Setup**: Initialize a queue for BFS and set the distance of node 1 to 0. All other nodes are initialized with a distance of -1 (indicating they haven't been visited yet).
3. **BFS Execution**: For each node dequeued, we explore its neighbors. If we encounter node 1 during the exploration, we update the minimum cycle length found. For other nodes, if they haven't been visited, we set their distance and enqueue them.
4. **Result Handling**: After BFS completes, if a cycle was found (indicated by `min_cycle` being updated), we print its length; otherwise, we print -1.

This approach ensures that we efficiently find the shortest cycle starting and ending at node 1 using BFS, which is optimal for unweighted graphs. The solution handles all edge cases, including graphs with no cycles involving node 1.
