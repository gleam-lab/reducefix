After analyzing the problem and the incorrect code, it appears we need to find the shortest cycle starting and ending at node 1 in a directed graph. The initial approach was using a DFS which isn't optimal for this shortest cycle problem. We should use BFS instead for finding shortest paths in unweighted graphs.

Here's the fixed code that uses BFS to find the shortest cycle:

```cpp
#include<bits/stdc++.h>
#define ll long long
using namespace std;

int main() {
    ll n, m;
    cin >> n >> m;
    vector<vector<int>> adj(n+1);
    for(int i=0; i<m; i++) {
        int u, v;
        cin >> u >> v;
        adj[u].push_back(v);
    }

    int min_cycle = INT_MAX;
    queue<pair<int, int>> q;
    vector<int> dist(n+1, -1);

    q.push({1, 0});
    dist[1] = 0;

    while(!q.empty()) {
        auto [u, d] = q.front();
        q.pop();

        for(int v : adj[u]) {
            if(v == 1) {
                min_cycle = min(min_cycle, d+1);
                continue;
            }
            if(dist[v] == -1 || d+1 < dist[v]) {
                dist[v] = d+1;
                q.push({v, d+1});
            }
        }
    }

    if(min_cycle == INT_MAX) {
        cout << -1 << endl;
    } else {
        cout << min_cycle << endl;
    }

    return 0;
}
```

Key changes made:
1. Replaced DFS with BFS since BFS is more suitable for finding shortest paths in unweighted graphs
2. Maintained a distance array to track the shortest path lengths
3. When we encounter node 1 during BFS traversal, we update the minimum cycle length
4. Removed unnecessary global variables and simplified the code structure
5. Used proper initializations and edge case handling

This approach ensures we find the shortest cycle containing node 1 in O(N + M) time complexity, which is optimal for this problem. The BFS guarantees that the first time we find a path back to node 1 will be the shortest possible cycle.
