To fix this code, we need to address the issue of finding the shortest cycle that starts and ends at node 1. The original code uses a depth-first search (DFS) approach, which is not suitable for finding the shortest path in an unweighted graph because DFS explores one path fully before backtracking, which can lead to longer paths being found first. Instead, we should use a breadth-first search (BFS) approach, which explores all neighbors at the present depth level before moving on to nodes at the next depth level, ensuring that the shortest path is found first.

### Approaches
1. **Breadth-First Search (BFS):** BFS is more appropriate for finding the shortest path in an unweighted graph. We will use BFS to explore all possible paths from node 1 back to node 1, keeping track of the shortest cycle found.
2. **Handling Cycles:** To detect cycles that start and end at node 1, we will start BFS from node 1 and check if we can return to node 1 through any of its neighbors. The first time we encounter node 1 again during BFS will give us the shortest cycle length.

### Solution Code
```cpp
#include<bits/stdc++.h>
#define ll long long
using namespace std;

int main() {
    ll n, m;
    cin >> n >> m;
    vector<vector<int>> adj(n + 1);
    for (int i = 0; i < m; i++) {
        int u, v;
        cin >> u >> v;
        adj[u].push_back(v);
    }

    queue<pair<int, int>> q; // {node, distance}
    vector<int> visited(n + 1, -1); // -1 means not visited
    q.push({1, 0});
    visited[1] = 0;
    int min_cycle = -1;

    while (!q.empty()) {
        auto current = q.front();
        q.pop();
        int node = current.first;
        int dist = current.second;

        for (int neighbor : adj[node]) {
            if (neighbor == 1) {
                if (min_cycle == -1 || dist + 1 < min_cycle) {
                    min_cycle = dist + 1;
                }
            }
            if (visited[neighbor] == -1) {
                visited[neighbor] = dist + 1;
                q.push({neighbor, dist + 1});
            }
        }
    }

    cout << min_cycle << endl;
    return 0;
}
```

### Explanation
1. **Graph Representation:** The graph is represented using an adjacency list `adj` where each node points to its neighbors.
2. **BFS Initialization:** We initialize a queue `q` to manage the BFS process, starting from node 1 with an initial distance of 0. The `visited` array keeps track of the distance from node 1 to each node.
3. **BFS Execution:** For each node dequeued, we check all its neighbors. If a neighbor is node 1, we update the shortest cycle length if the current path provides a shorter cycle. For other neighbors, if they haven't been visited, we mark their distance and enqueue them.
4. **Result:** After processing all nodes, the shortest cycle length found is printed. If no cycle is found, `-1` is printed.

This approach ensures that we efficiently find the shortest cycle starting and ending at node 1 using BFS, which is optimal for unweighted graphs.
