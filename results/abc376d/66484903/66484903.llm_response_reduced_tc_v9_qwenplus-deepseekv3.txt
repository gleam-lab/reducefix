To fix this code, we need to ensure that the BFS (Breadth-First Search) correctly finds the shortest path from node 1 back to itself through a cycle in the directed graph. The previous implementation had a logical flaw where it only checked if the next node was 1, which doesn't account for the shortest cycle properly. 

### Approaches
1. **BFS Initialization**: Start BFS from node 1 and keep track of the steps taken to reach each node.
2. **Cycle Detection**: For each node visited, if we encounter node 1 again, it means we've found a cycle. The length of the cycle is the current step count plus one.
3. **Tracking Visits**: Ensure nodes are not revisited to avoid redundant checks and infinite loops.
4. **Output the Result**: If a cycle is found, output the shortest cycle length; otherwise, output -1.

### Solution Code
```cpp
#include<bits/stdc++.h>
#define INF 0x3f3f3f3f3f3f3f3f
using namespace std;
typedef long long ll;
const int MAXN=2e5+10;
vector<ll> vec[MAXN];
queue<ll> q;
ll step[MAXN];
bool vis[MAXN];
int main(){
    ll n,m,u,v;
    scanf("%lld %lld",&n,&m);
    for(ll i=1;i<=m;i++){
        scanf("%lld %lld",&u,&v);
        vec[u].push_back(v);
    }
    q.push(1);
    vis[1] = true;
    step[1] = 0;
    while(!q.empty()){
        ll top=q.front();
        q.pop();
        for(ll i=0;i<vec[top].size();i++){
            ll to=vec[top][i];
            if(to == 1){
                printf("%lld",step[top] + 1);
                return 0;
            }
            if(!vis[to]){
                vis[to] = true;
                step[to] = step[top] + 1;
                q.push(to);
            }
        }
    }
    printf("-1");
    return 0;
}
```

### Explanation
1. **Initialization**: The BFS starts at node 1 with `step[1]` initialized to 0 and `vis[1]` marked as true.
2. **BFS Execution**: For each node dequeued, all its adjacent nodes are checked. If any adjacent node is 1, the current step count plus one (the edge back to 1) is returned immediately as the shortest cycle length.
3. **Visited Tracking**: Each node is marked as visited once it is enqueued to prevent re-processing and ensure each node is processed only once.
4. **Termination**: If the queue is exhausted without finding node 1 again, -1 is printed indicating no cycle exists back to node 1.

This approach ensures that the shortest cycle involving node 1 is found efficiently using BFS, which guarantees the shortest path in an unweighted graph.
