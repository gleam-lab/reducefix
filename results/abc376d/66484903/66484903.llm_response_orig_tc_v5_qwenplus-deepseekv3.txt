After analyzing the incorrect code and the failing test case, I identified that the issue lies in the BFS implementation. The original code was trying to find a cycle back to node 1, but it didn't properly handle the case where node 1 is directly connected to itself, which would make the shortest cycle length 1 (but since we need at least 2 edges, it's invalid). Also, the BFS was not correctly tracking visited nodes when checking for cycles.

Here's the fixed code that properly finds the shortest cycle containing node 1 in a directed graph:

```cpp
#include<bits/stdc++.h>
#define INF 0x3f3f3f3f3f3f3f3f
using namespace std;
typedef long long ll;
const int MAXN=2e5+10;
vector<ll> vec[MAXN];
ll dist[MAXN];
ll parent[MAXN];
ll min_cycle = INF;

void bfs(ll start) {
    queue<ll> q;
    q.push(start);
    dist[start] = 0;
    
    while(!q.empty()) {
        ll u = q.front();
        q.pop();
        
        for(ll v : vec[u]) {
            if(v == start) {
                min_cycle = min(min_cycle, dist[u] + 1);
            }
            if(dist[v] == -1) {
                dist[v] = dist[u] + 1;
                parent[v] = u;
                q.push(v);
            }
        }
    }
}

int main(){
    ll n,m,u,v;
    scanf("%lld %lld",&n,&m);
    for(ll i=1;i<=m;i++){
        scanf("%lld %lld",&u,&v);
        vec[u].push_back(v);
    }
    
    memset(dist, -1, sizeof(dist));
    memset(parent, -1, sizeof(parent));
    bfs(1);
    
    if(min_cycle != INF) {
        printf("%lld", min_cycle);
    } else {
        printf("-1");
    }
    return 0;
}
```

Key improvements made:
1. Properly implemented BFS with distance tracking from node 1
2. Added a check for when we find a node that connects back to node 1 (forming a cycle)
3. Keeps track of the minimum cycle length found
4. Uses -1 to indicate unvisited nodes instead of a separate visited array
5. Outputs -1 if no cycle containing node 1 is found

The algorithm now correctly finds the shortest cycle that includes node 1 by performing a BFS from node 1 and checking for any edges that lead back to node 1. The shortest such path found during the BFS gives the length of the shortest cycle containing node 1.

This solution has a time complexity of O(N + M) which is efficient for the given constraints (N ≤ 2×10^5). It avoids the issues in the original implementation that led to incorrect cycle length detection.
