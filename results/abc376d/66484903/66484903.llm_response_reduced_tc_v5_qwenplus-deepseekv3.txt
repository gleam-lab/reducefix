To fix this code, we need to ensure that the BFS (Breadth-First Search) correctly computes the shortest path from node 1 back to itself through the directed graph. The original code incorrectly sets the step count and does not handle the BFS queue properly, leading to an incorrect shortest path calculation.

### Approaches
1. **BFS Initialization**: Start BFS from node 1 to explore all reachable nodes.
2. **Visited Tracking**: Use a visited array to avoid revisiting nodes and unnecessary computations.
3. **Step Counting**: Maintain a step count array to record the shortest path from node 1 to each node.
4. **Early Termination**: If during the BFS we encounter node 1 again (indicating a cycle back to node 1), immediately return the step count plus one (for the edge back to node 1).
5. **No Cycle Handling**: If the BFS completes without finding node 1 again, return -1 indicating no such cycle exists.

### Solution Code
```cpp
#include<bits/stdc++.h>
#define INF 0x3f3f3f3f3f3f3f3f
using namespace std;
typedef long long ll;
const int MAXN=2e5+10;
vector<ll> vec[MAXN];
queue<ll> q;
ll step[MAXN];
bool vis[MAXN];
int main(){
    ll n,m,u,v;
    scanf("%lld %lld",&n,&m);
    for(ll i=1;i<=m;i++){
        scanf("%lld %lld",&u,&v);
        vec[u].push_back(v);
    }
    memset(step, -1, sizeof(step));
    q.push(1);
    step[1] = 0;
    vis[1] = true;
    while(!q.empty()){
        ll top=q.front();
        q.pop();
        for(ll i=0;i<vec[top].size();i++){
            ll to=vec[top][i];
            if(to == 1){
                printf("%lld", step[top] + 1);
                return 0;
            }
            if(!vis[to]){
                step[to] = step[top] + 1;
                vis[to] = true;
                q.push(to);
            }
        }
    }
    printf("-1");
    return 0;
}
```

### Explanation
1. **Initialization**: The BFS starts from node 1 with `step[1]` initialized to 0, marking it as visited.
2. **Queue Processing**: For each node dequeued, all its adjacent nodes are processed. If any adjacent node is node 1, the current step count (distance from node 1 to the current node) plus one (for the edge back to node 1) is printed immediately as the result.
3. **Visited Nodes**: Nodes are marked as visited once they are enqueued to prevent revisiting and redundant computations.
4. **No Cycle Case**: If the queue is exhausted without finding node 1 again, the output is -1 indicating no cycle back to node 1 exists.

This approach ensures that the shortest path back to node 1 is found efficiently using BFS, which is optimal for unweighted graphs. The solution handles all edge cases and avoids unnecessary computations through proper tracking of visited nodes.
