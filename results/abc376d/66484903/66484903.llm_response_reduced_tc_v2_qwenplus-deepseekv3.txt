To fix this code, we need to ensure that the BFS (Breadth-First Search) correctly finds the shortest path from node 1 back to itself through a directed cycle. The original code incorrectly marks nodes as visited immediately after they are popped from the queue, which can lead to missing shorter paths in some cases. Instead, nodes should be marked as visited as soon as they are discovered (i.e., when they are pushed into the queue) to prevent revisiting them and potentially finding longer paths.

### Approaches
1. **BFS Initialization**: Start the BFS from node 1.
2. **Visited Tracking**: Mark nodes as visited when they are added to the queue to avoid revisiting them.
3. **Cycle Detection**: If during the BFS traversal, we encounter node 1 again, we immediately return the current step count plus one (since this step represents the cycle length).
4. **Termination**: If the queue is exhausted without finding node 1, return -1 indicating no cycle exists.

### Solution Code
```cpp
#include<bits/stdc++.h>
#define INF 0x3f3f3f3f3f3f3f3f
using namespace std;
typedef long long ll;
const int MAXN=2e5+10;
vector<ll> vec[MAXN];
queue<ll> q;
ll step[MAXN];
bool vis[MAXN];
int main(){
    ll n,m,u,v;
    scanf("%lld %lld",&n,&m);
    for(ll i=1;i<=m;i++){
        scanf("%lld %lld",&u,&v);
        vec[u].push_back(v);
    }
    q.push(1);
    vis[1] = true;
    step[1] = 0;
    while(!q.empty()){
        ll top=q.front();
        q.pop();
        for(ll i=0;i<vec[top].size();i++){
            ll to=vec[top][i];
            if(to==1){
                printf("%lld",step[top]+1);
                return 0;
            }
            if(!vis[to]){
                vis[to] = true;
                step[to]=step[top]+1;
                q.push(to);
            }
        }
    }
    printf("-1");
    return 0;
}
```

### Explanation
1. **Initialization**: The BFS starts from node 1, which is marked as visited immediately with a step count of 0.
2. **Traversal**: For each node dequeued, we explore all its adjacent nodes. If an adjacent node is node 1, we return the step count of the current node plus one (forming the cycle).
3. **Visited Handling**: Nodes are marked as visited as soon as they are discovered to ensure they are not processed more than once, which helps in finding the shortest path.
4. **Termination**: If the BFS completes without finding node 1, it concludes there is no cycle and returns -1. This ensures the solution is both efficient and correct for all test cases.
