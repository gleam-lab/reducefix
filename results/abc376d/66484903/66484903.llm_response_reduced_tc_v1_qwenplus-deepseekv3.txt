To fix this code, we need to ensure that the BFS (Breadth-First Search) correctly finds the shortest path from node 1 back to itself. The original code incorrectly marks nodes as visited too early, which can lead to longer paths being considered before shorter ones. 

### Approaches
1. **BFS Initialization**: Start BFS from node 1, exploring all adjacent nodes.
2. **Visited Handling**: Mark nodes as visited when they are added to the queue to avoid revisiting them through longer paths.
3. **Cycle Detection**: If during BFS we encounter node 1 again (indicating a cycle), we return the current step count plus one (for the edge back to node 1).
4. **Termination Condition**: If the queue is exhausted without finding node 1, return -1 indicating no cycle exists.

### Solution Code
```cpp
#include<bits/stdc++.h>
#define INF 0x3f3f3f3f3f3f3f3f
using namespace std;
typedef long long ll;
const int MAXN=2e5+10;
vector<ll> vec[MAXN];
queue<ll> q;
ll step[MAXN];
bool vis[MAXN];
int main(){
    ll n,m,u,v;
    scanf("%lld %lld",&n,&m);
    for(ll i=1;i<=m;i++){
        scanf("%lld %lld",&u,&v);
        vec[u].push_back(v);
    }
    q.push(1);
    vis[1] = true; // Mark the starting node as visited
    while(!q.empty()){
        ll top=q.front();
        q.pop();
        for(ll i=0;i<vec[top].size();i++){
            ll to=vec[top][i];
            if(to == 1){
                printf("%lld",step[top]+1);
                return 0;
            }
            if(!vis[to]){
                vis[to] = true;
                step[to] = step[top] + 1;
                q.push(to);
            }
        }
    }
    printf("-1");
    return 0;
}
```

### Explanation
1. **Initialization**: The BFS starts at node 1, which is immediately marked as visited to prevent it from being revisited.
2. **Queue Processing**: For each node processed, all its adjacent nodes are checked. If any adjacent node is node 1, the current step count (from the parent node) plus one is printed as the result.
3. **Visited and Step Tracking**: Nodes are marked as visited when they are first discovered to ensure the shortest path is always found. The step count for each node is recorded based on its parent's step count plus one.
4. **Termination**: If the queue is empty and node 1 hasn't been revisited, the function outputs -1, indicating no cycle back to node 1 exists.

This approach ensures that the shortest cycle involving node 1 is found efficiently using BFS, which inherently explores nodes level by level, guaranteeing the shortest path in an unweighted graph.
