2. **Cycle Detection**: During BFS, if we encounter node 1 again, it means we have found a cycle. The length of this cycle is `step[top] + 1` where `top` is the current node being processed.
3. **Visited and Step Tracking**: Maintain a visited array to avoid revisiting nodes and a step array to keep track of the distance from node 1.
4. **Early Termination**: As soon as we find node 1 during BFS, we terminate early and return the current cycle length, ensuring the shortest cycle is found due to BFS properties.

typedef long long ll;
const int MAXN=2e5+10;
vector<ll> vec[MAXN];
queue<ll> q;
ll step[MAXN];
bool vis[MAXN];
int main(){
    ll n,m,u,v;
    scanf("%lld %lld",&n,&m);
    for(ll i=1;i<=m;i++){

        vec[u].push_back(v);
    }
    q.push(1);
    vis[1] = true;
    step[1] = 0;
    while(!q.empty()){
        ll top=q.front();
        q.pop();
        for(ll i=0;i<vec[top].size();i++){
            ll to=vec[top][i];
            if(to == 1){
                printf("%lld", step[top] + 1);
                return 0;
            }
            if(!vis[to]){
                vis[to] = true;
                step[to] = step[top] + 1;
                q.push(to);
            }
        }
    }
    printf("-1");
    return 0;
}

### Explanation
1. **Input Handling**: The input reads the number of nodes `n` and edges `m`, followed by `m` directed edges.
2. **BFS Initialization**: The BFS starts from node 1, which is marked as visited with a step count of 0.
3. **BFS Execution**: For each node dequeued, the algorithm checks all its neighbors. If a neighbor is node 1, it prints the current step count + 1 (the length of the cycle) and exits immediately.
4. **Visited and Step Updates**: Neighbors not yet visited are marked as visited, their step count is set to the current node's step count + 1, and they are enqueued for further processing.
5. **No Cycle Found**: If the queue is exhausted without finding node 1 again, the algorithm prints `-1`, indicating no cycle exists starting and ending at node 1.

This approach ensures that the shortest cycle is found efficiently using BFS, which inherently explores nodes level by level, thus guaranteeing the shortest path (or cycle) in an unweighted graph.
