The issue in the original code is that it marks nodes as visited too early, which can prevent the discovery of shorter paths. Specifically, in the failing case, the code prematurely marks nodes as visited, leading to a longer path being recorded.

### Approaches
1. **BFS for Shortest Path**: Use BFS to explore the graph layer by layer, ensuring that the first time we encounter node 1 again, it is via the shortest path.
2. **Visited Handling**: Instead of marking nodes as visited immediately when they are added to the queue, we should mark them as visited when they are dequeued. This allows for multiple paths to be considered until the shortest one is found.
3. **Initialization**: Initialize the step array with a large value (INF) to represent unvisited nodes, and set the step for node 1 to 0 initially.
4. **Cycle Detection**: If during BFS we revisit node 1, we immediately return the step count, which represents the shortest cycle length starting and ending at node 1.

```cpp
#include<bits/stdc++.h>
#define INF 0x3f3f3f3f3f3f3f3f
using namespace std;
typedef long long ll;
const int MAXN=2e5+10;
vector<ll> vec[MAXN];
queue<ll> q;
ll step[MAXN];
bool inQueue[MAXN];

int main(){
    ll n, m, u, v;
    scanf("%lld %lld", &n, &m);
    for(ll i=1; i<=m; i++){
        scanf("%lld %lld", &u, &v);
        vec[u].push_back(v);
    }
    
    memset(step, INF, sizeof(step));
    step[1] = 0;
    q.push(1);
    inQueue[1] = true;
    
    while(!q.empty()){
        ll top = q.front();
        q.pop();
        inQueue[top] = false;
        
        for(ll i=0; i<vec[top].size(); i++){
            ll to = vec[top][i];
            if(to == 1){
                if(step[top] + 1 < step[to]){
                    step[to] = step[top] + 1;
                    printf("%lld", step[to]);
                    return 0;
                }
            }
            if(step[top] + 1 < step[to]){
                step[to] = step[top] + 1;
                if(!inQueue[to]){
                    q.push(to);
                    inQueue[to] = true;
                }
            }
        }
    }
    
    printf("-1");
    return 0;
}
```

### Explanation
1. **Initialization**: The `step` array is initialized to INF, except for `step[1]` which is set to 0, indicating the starting point.
2. **BFS Setup**: Node 1 is added to the queue, and `inQueue` is marked to true to prevent redundant additions.
3. **BFS Execution**: For each node dequeued, we explore its neighbors. If we encounter node 1 as a neighbor, we check if the current path provides a shorter cycle (`step[top] + 1`). If so, we print the result and exit.
4. **Relaxation**: For other nodes, if a shorter path is found (`step[top] + 1 < step[to]`), the step count is updated, and the node is added to the queue if not already present.
5. **Termination**: If no cycle back to node 1 is found, `-1` is printed.

This approach ensures that the shortest cycle starting and ending at node 1 is found efficiently using BFS, with proper handling of visited nodes to avoid missing shorter paths.
