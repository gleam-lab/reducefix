To fix this code, we need to ensure that the BFS (Breadth-First Search) is correctly implemented to find the shortest path from node 1 back to itself in the given directed graph. The original code has a few issues: it doesn't correctly handle the BFS traversal, and it incorrectly checks for the condition `to == 1` without considering the initial step count properly.

### Approaches
1. **BFS Initialization**: Start BFS from node 1 because we are looking for the shortest cycle that starts and ends at node 1.
2. **Visited and Step Tracking**: Maintain a `visited` array to avoid revisiting nodes and a `step` array to keep track of the shortest distance from node 1 to each node.
3. **Cycle Detection**: During BFS, if we encounter node 1 again, immediately return the current step count plus one (since this forms a cycle of length `step[to] + 1`).
4. **Handling No Cycle**: If BFS completes without finding node 1 again, return -1 indicating no cycle exists.

### Solution Code
```cpp
#include<bits/stdc++.h>
#define INF 0x3f3f3f3f3f3f3f3f
using namespace std;
typedef long long ll;
const int MAXN=2e5+10;
vector<ll> vec[MAXN];
queue<ll> q;
ll step[MAXN];
bool vis[MAXN];
int main(){
    ll n, m, u, v;
    scanf("%lld %lld", &n, &m);
    for(ll i=1; i<=m; i++){
        scanf("%lld %lld", &u, &v);
        vec[u].push_back(v);
    }
    q.push(1);
    vis[1] = true;
    step[1] = 0;
    while(!q.empty()){
        ll top = q.front();
        q.pop();
        for(ll i=0; i<vec[top].size(); i++){
            ll to = vec[top][i];
            if(to == 1){
                printf("%lld", step[top] + 1);
                return 0;
            }
            if(!vis[to]){
                vis[to] = true;
                step[to] = step[top] + 1;
                q.push(to);
            }
        }
    }
    printf("-1");
    return 0;
}
```

### Explanation
1. **Initialization**: The BFS starts from node 1, marking it as visited and setting its step count to 0.
2. **Traversal**: For each node dequeued, its neighbors are checked. If a neighbor is node 1, the current step count plus one is printed (indicating the cycle length).
3. **Visited Handling**: Nodes are marked as visited when first encountered to ensure each node is processed only once, maintaining BFS efficiency.
4. **Termination**: If the queue is exhausted without finding node 1 again, -1 is printed, indicating no cycle exists from node 1 back to itself.

This approach ensures that the shortest cycle is found efficiently using BFS, which is optimal for unweighted graphs. The solution handles all edge cases, including those where no cycle exists.
