After analyzing the code and the failing test case, I identified that the problem requires finding the shortest cycle starting and ending at node 1 in a directed graph. The original code used BFS but didn't correctly handle cycles that don't immediately return to node 1. Here's the corrected solution:

```cpp
#include<bits/stdc++.h>
#define INF 0x3f3f3f3f3f3f3f3f
using namespace std;
typedef long long ll;
const int MAXN=2e5+10;
vector<ll> vec[MAXN];
ll dist[MAXN];

int main(){
    ll n,m,u,v;
    scanf("%lld %lld",&n,&m);
    for(ll i=1;i<=m;i++){
        scanf("%lld %lld",&u,&v);
        vec[u].push_back(v);
    }
    
    memset(dist, -1, sizeof(dist));
    queue<ll> q;
    q.push(1);
    dist[1] = 0;
    ll min_cycle = INF;
    
    while(!q.empty()){
        ll u = q.front();
        q.pop();
        
        for(ll v : vec[u]){
            if(v == 1){
                min_cycle = min(min_cycle, dist[u] + 1);
            }
            if(dist[v] == -1){
                dist[v] = dist[u] + 1;
                q.push(v);
            }
        }
    }
    
    if(min_cycle != INF){
        printf("%lld\n", min_cycle);
    } else {
        printf("-1\n");
    }
    return 0;
}
```

Key changes made:
1. Used BFS to calculate distances from node 1 to all other nodes
2. Tracked the minimum cycle length by checking when we find a direct edge back to node 1
3. Properly initialized distances and used a queue for BFS
4. Removed unnecessary data structures (vis array) since we can use distance for both visitation check and cycle detection

The algorithm now correctly finds the shortest directed cycle that starts and ends at node 1 by:
1. Performing BFS from node 1
2. For each node, when we find an edge back to node 1, we calculate the cycle length (distance to current node + 1)
3. Keeping track of the minimum such cycle length found

This solution has O(N + M) time complexity, which is efficient enough for the problem constraints.
