### Analysis of the Original Code
The original code attempts to find the shortest cycle containing node 1 in a directed graph using BFS. The main issues are:

1. **Cycle Detection Logic**: The code incorrectly checks for cycles by only looking at edges pointing back to node 1. This misses other potential cycles.

2. **Visited Marking**: Nodes are marked as visited too early (when they are popped from the queue), which can prevent finding shorter paths.

3. **Self-loop Handling**: The special case for node 1 pointing to itself is handled separately, but the general cycle detection is flawed.

### Corrected Approach
To correctly find the shortest cycle containing node 1:
1. Perform a BFS starting from node 1.
2. Track distances from node 1 to all other nodes.
3. For each edge `u -> v`, if `v` is node 1 and `u` is not node 1, we have a potential cycle of length `distance[u] + 1`.
4. Keep track of the minimum such cycle length found.

### Fixed Code
```cpp
#include<bits/stdc++.h>
#define INF 0x3f3f3f3f3f3f3f3f
using namespace std;
typedef long long ll;
const int MAXN=2e5+10;
vector<ll> vec[MAXN];
queue<ll> q;
ll dist[MAXN];
int main(){
    ll n,m,u,v;
    scanf("%lld %lld",&n,&m);
    for(ll i=1;i<=m;i++){
        scanf("%lld %lld",&u,&v);
        vec[u].push_back(v);
    }
    
    memset(dist, -1, sizeof(dist));
    dist[1] = 0;
    q.push(1);
    
    ll min_cycle = INF;
    
    while(!q.empty()){
        ll top = q.front();
        q.pop();
        
        for(ll i=0; i<vec[top].size(); i++){
            ll to = vec[top][i];
            
            if(to == 1){
                if(top != 1){
                    min_cycle = min(min_cycle, dist[top] + 1);
                } else {
                    min_cycle = 1;
                }
                continue;
            }
            
            if(dist[to] == -1){
                dist[to] = dist[top] + 1;
                q.push(to);
            }
        }
    }
    
    if(min_cycle != INF){
        printf("%lld", min_cycle);
    } else {
        printf("-1");
    }
    
    return 0;
}
```

### Key Improvements
1. **Proper Distance Initialization**: All distances are initialized to -1 (unvisited), with node 1's distance set to 0.
2. **Cycle Detection**: Correctly checks all edges pointing back to node 1 to find cycles.
3. **Early Termination**: The BFS continues to explore the graph fully to ensure the shortest cycle is found, but could be optimized further if needed.
4. **Self-loop Handling**: Special case for node 1 pointing to itself is handled properly.

This solution correctly finds the shortest cycle containing node 1 or returns -1 if no such cycle exists.
