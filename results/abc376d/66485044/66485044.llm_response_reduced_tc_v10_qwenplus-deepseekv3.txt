2. **Cycle Detection**: For each node visited, check if it has an edge back to node 1. If such an edge exists, the shortest cycle length is the current step count plus one.
3. **Visited Tracking**: Use a visited array to avoid revisiting nodes and to keep track of the shortest path distances from node 1.

### Solution Code
```cpp
#include<bits/stdc++.h>
#define INF 0x3f3f3f3f3f3f3f3f
using namespace std;
typedef long long ll;
const int MAXN=2e5+10;
vector<ll> vec[MAXN];
queue<ll> q;
ll step[MAXN];
bool vis[MAXN];
int main(){
    ll n, m, u, v;
    scanf("%lld %lld", &n, &m);
    for(ll i=1; i<=m; i++){
        scanf("%lld %lld", &u, &v);
        vec[u].push_back(v);
    }
    
    memset(step, -1, sizeof(step));
    q.push(1);
    step[1] = 0;
    vis[1] = true;
    
    ll min_cycle = INF;
    
    while(!q.empty()){
        ll top = q.front();
        q.pop();
        
        for(ll i=0; i<vec[top].size(); i++){
            ll to = vec[top][i];
            if(to == 1){
                if(step[top] + 1 < min_cycle){
                    min_cycle = step[top] + 1;
                }
            }
            if(!vis[to]){
                vis[to] = true;
                step[to] = step[top] + 1;
                q.push(to);
            }
        }
    }
    
    if(min_cycle != INF){
        printf("%lld\n", min_cycle);
    } else {
        printf("-1\n");
    }
    return 0;
}
```

1. **Initialization**: We initialize the BFS queue with node 1, set its step count to 0, and mark it as visited.
2. **BFS Execution**: For each node dequeued, we explore all its neighbors. If a neighbor is node 1, we update the shortest cycle length if the current path provides a shorter cycle.
3. **Visited and Step Tracking**: We ensure nodes are only enqueued once and their step counts are updated to reflect the shortest path from node 1.
4. **Result Output**: After BFS completes, if a cycle involving node 1 was found, the shortest cycle length is printed; otherwise, -1 is printed indicating no such cycle exists.

This approach efficiently explores all possible paths from node 1 and correctly identifies the shortest cycle involving node 1, ensuring optimal performance and correctness.
