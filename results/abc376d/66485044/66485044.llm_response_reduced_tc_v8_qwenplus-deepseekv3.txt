### Approaches
The correct approach involves:
1. **BFS Initialization**: Start from node 1 and explore its neighbors.
2. **Cycle Detection**: For each node visited, check if it leads back to node 1, which indicates a cycle.
3. **Shortest Path Tracking**: Track the shortest path from node 1 back to itself using BFS, ensuring the shortest cycle is found.

}

### Explanation
1. **Initialization**: The BFS is initialized from node 1 with `step[1]` set to 0. All other nodes start with `step` set to -1, indicating they are unvisited.
2. **BFS Execution**: For each node dequeued, its neighbors are checked. If a neighbor is node 1, the current `step` value plus one (indicating the edge back to node 1) is printed as the shortest cycle length.
3. **Cycle Detection**: If during BFS we revisit node 1, it means we have found a cycle, and the current path length is the shortest due to BFS properties.
4. **Termination**: If no cycle is found after BFS completes, "-1" is printed indicating no cycle involving node 1 exists.

This approach ensures the shortest cycle is found efficiently using BFS, which inherently explores nodes level by level, thus guaranteeing the shortest path when node 1 is revisited.
