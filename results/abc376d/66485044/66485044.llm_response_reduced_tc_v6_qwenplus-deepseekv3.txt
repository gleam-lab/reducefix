To fix this code, we need to correctly compute the shortest path from node 1 back to itself, forming a cycle. The original code attempts to do this using a BFS approach but fails to correctly handle the case where node 1 is directly connected to itself (which would form a cycle of length 1). Additionally, the BFS implementation does not correctly account for all possible paths that might form a cycle involving node 1.

### Approaches
1. **BFS for Shortest Path**: We use BFS to explore the graph starting from node 1. The goal is to find the shortest path that starts at node 1, visits some other nodes, and returns to node 1.
2. **Cycle Detection**: During the BFS, if we encounter node 1 again from any node other than itself, we have found a cycle. The length of the cycle will be the current step count plus one (the edge back to node 1).
3. **Handling Direct Self-loop**: If node 1 has a direct edge to itself, the answer is immediately 1, as this forms the shortest possible cycle.

### Solution Code

```cpp
#include<bits/stdc++.h>
#define INF 0x3f3f3f3f3f3f3f3f
using namespace std;
typedef long long ll;
const int MAXN=2e5+10;
vector<ll> vec[MAXN];
queue<ll> q;
ll step[MAXN];
bool vis[MAXN];

int main(){
    ll n, m, u, v;
    scanf("%lld %lld", &n, &m);
    for(ll i=1; i<=m; i++){
        scanf("%lld %lld", &u, &v);
        vec[u].push_back(v);
        // Check if there's a direct self-loop from 1 to 1
        if(u == 1 && v == 1){
            printf("1\n");
            return 0;
        }
    }
    fill(step, step + MAXN, INF);
    step[1] = 0;
    q.push(1);
    vis[1] = true;
    while(!q.empty()){
        ll top = q.front();
        q.pop();
        for(ll i=0; i<vec[top].size(); i++){
            ll to = vec[top][i];
            if(to == 1 && top != 1){
                printf("%lld\n", step[top] + 1);
                return 0;
            }
            if(!vis[to]){
                vis[to] = true;
                step[to] = step[top] + 1;
                q.push(to);
            }
        }
    }
    printf("-1\n");
    return 0;
}
```

### Explanation
1. **Initial Check for Self-loop**: The code first checks if there's a direct edge from node 1 to itself. If such an edge exists, it immediately returns 1, as this is the shortest possible cycle.
2. **BFS Initialization**: The BFS is initialized with node 1. The `step` array is used to keep track of the shortest path distances from node 1 to all other nodes, initialized to infinity (`INF`), except for node 1, which is set to 0.
3. **BFS Execution**: During the BFS traversal, for each node visited, the code checks if any of its neighbors is node 1 (except when the current node is node 1 itself). If such a neighbor is found, it means a cycle is detected, and the length of the cycle is `step[top] + 1`, which is then printed.
4. **Cycle Not Found**: If the BFS completes without finding any cycle back to node 1, the code prints `-1`, indicating no such cycle exists.

This approach ensures that the shortest cycle involving node 1 is found efficiently using BFS, which guarantees the shortest path in an unweighted graph. The solution handles all edge cases, including direct self-loops and cases where no cycle exists.
