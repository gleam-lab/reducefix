### Analysis of the Original Code
The original code attempts to find the shortest cycle involving node 1 in a directed graph using BFS. However, there are several issues:
1. **Visited Marking**: The code marks nodes as visited when they are popped from the queue, which is incorrect for BFS shortest path calculations. Nodes should be marked as visited when they are enqueued to avoid redundant processing.
2. **Cycle Detection Logic**: The logic for detecting cycles involving node 1 is flawed. It checks if `to == 1` but doesn't properly handle cases where node 1 is part of a cycle that isn't directly connected back to itself.
3. **Initialization**: The `step` array isn't initialized, which can lead to undefined behavior when accessed.
4. **Edge Cases**: The code doesn't handle cases where node 1 isn't part of any cycle.

### Fixed Code
The fixed code addresses these issues by:
1. Properly marking nodes as visited when they are enqueued.
2. Correctly initializing the `step` array.
3. Using BFS to find the shortest path back to node 1 from its neighbors, ensuring the shortest cycle is found.
4. Handling edge cases where no cycle exists.

```cpp
#include<bits/stdc++.h>
#define INF 0x3f3f3f3f3f3f3f3f
using namespace std;
typedef long long ll;
const int MAXN=2e5+10;
vector<ll> vec[MAXN];
queue<ll> q;
ll step[MAXN];
bool vis[MAXN];

int main(){
    ll n, m, u, v;
    scanf("%lld %lld", &n, &m);
    for(ll i = 1; i <= m; i++){
        scanf("%lld %lld", &u, &v);
        vec[u].push_back(v);
    }

    // Initialize step array with INF
    memset(step, 0x3f, sizeof(step));
    step[1] = 0;
    q.push(1);
    vis[1] = true;

    ll min_cycle = INF;

    while(!q.empty()){
        ll top = q.front();
        q.pop();
        for(ll i = 0; i < vec[top].size(); i++){
            ll to = vec[top][i];
            if(to == 1){
                min_cycle = min(min_cycle, step[top] + 1);
            }
            if(!vis[to]){
                vis[to] = true;
                step[to] = step[top] + 1;
                q.push(to);
            }
        }
    }

    if(min_cycle != INF){
        printf("%lld\n", min_cycle);
    } else {
        printf("-1\n");
    }
    return 0;
}
```

### Key Fixes
1. **Visited Marking**: Nodes are marked as visited when enqueued, ensuring no redundant processing.
2. **Cycle Detection**: The shortest cycle involving node 1 is tracked by checking if any neighbor of a node is node 1 and updating `min_cycle` accordingly.
3. **Initialization**: The `step` array is initialized to `INF` to correctly compute shortest paths.
4. **Edge Cases**: If no cycle is found, `-1` is printed. Otherwise, the length of the shortest cycle is printed.
