The original code had issues with how it handled the detection of the cycle and the calculation of steps. The corrected approach involves properly initializing the BFS, ensuring that the steps are correctly updated, and checking for the cycle in a systematic way.

### Approaches
1. **BFS Initialization**: Start the BFS from node 1, marking it as visited and initializing its step count to 0.
2. **Cycle Detection**: During the BFS traversal, if we encounter node 1 again from any other node, it means we've found a cycle. The step count at that node plus one gives the length of the cycle.
3. **Step Update**: For each adjacent node, if it hasn't been visited, update its step count and add it to the queue.
4. **Termination Condition**: If during the BFS, node 1 is revisited, return the step count immediately. If the queue is exhausted without finding such a cycle, return -1.

### Solution Code
```cpp
#include<bits/stdc++.h>
#define INF 0x3f3f3f3f3f3f3f3f
using namespace std;
typedef long long ll;
const int MAXN=2e5+10;
vector<ll> vec[MAXN];
queue<ll> q;
ll step[MAXN];
bool vis[MAXN];
int main(){
    ll n, m, u, v;
    scanf("%lld %lld", &n, &m);
    for(ll i = 1; i <= m; i++){
        scanf("%lld %lld", &u, &v);
        vec[u].push_back(v);
    }

    memset(vis, false, sizeof(vis));
    memset(step, -1, sizeof(step));
    q.push(1);
    step[1] = 0;
    vis[1] = true;

    while(!q.empty()){
        ll top = q.front();
        q.pop();
        for(ll i = 0; i < vec[top].size(); i++){
            ll to = vec[top][i];
            if(to == 1 && top != 1){
                printf("%lld", step[top] + 1);
                return 0;
            }
            if(!vis[to]){
                vis[to] = true;
                step[to] = step[top] + 1;
                q.push(to);
            }
        }
    }
    printf("-1");
    return 0;
}
```

### Explanation
1. **Initialization**: The BFS starts from node 1 with `step[1]` set to 0 and `vis[1]` marked as true.
2. **BFS Traversal**: For each node dequeued, its adjacent nodes are checked. If an adjacent node is node 1 and the current node is not node 1 itself, the cycle length (step count of current node + 1) is printed.
3. **Step Update**: For unvisited adjacent nodes, their step counts are updated, and they are enqueued for further processing.
4. **Cycle Not Found**: If the BFS completes without finding node 1 again, it means no cycle exists, so -1 is printed.

This approach ensures that the shortest cycle involving node 1 is found efficiently using BFS, which guarantees the shortest path in an unweighted graph.
