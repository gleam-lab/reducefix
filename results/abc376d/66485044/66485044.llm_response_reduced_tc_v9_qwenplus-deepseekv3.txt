To fix this code, we need to ensure that the shortest path from node 1 back to itself through any cycle is correctly calculated. The original code attempts to find such a path using a BFS approach but fails in some cases, as seen in the failing test case. The issue arises because the BFS does not properly account for all possible paths that might form a cycle back to node 1.

### Approaches
1. **BFS for Shortest Path**: We use BFS to explore the graph starting from node 1. BFS is suitable here because it naturally explores nodes level by level, ensuring that the first time we reach a node, it is via the shortest path.
2. **Cycle Detection**: We need to detect if there's a path from node 1 back to itself through any other nodes. The shortest such cycle will give the minimal steps required.
3. **Handling Direct Edges**: If there's a direct edge from node 1 to itself, the answer should be 1, but the problem might consider this as a trivial cycle. However, based on the failing test case, it seems the expected output is the shortest non-trivial cycle.
4. **Visited and Step Tracking**: We maintain a visited array to avoid revisiting nodes and a step array to keep track of the distance from the starting node (node 1).

### Solution Code
```cpp
#include<bits/stdc++.h>
#define INF 0x3f3f3f3f3f3f3f3f
using namespace std;
typedef long long ll;
const int MAXN=2e5+10;
vector<ll> vec[MAXN];
queue<ll> q;
ll step[MAXN];
bool vis[MAXN];
int main(){
    ll n,m,u,v;
    scanf("%lld %lld",&n,&m);
    for(ll i=1;i<=m;i++){
        scanf("%lld %lld",&u,&v);
        vec[u].push_back(v);
    }
    q.push(1);
    vis[1] = true;
    step[1] = 0;
    ll min_cycle = INF;
    while(!q.empty()){
        ll top=q.front();
        q.pop();
        for(ll i=0;i<vec[top].size();i++){
            ll to=vec[top][i];
            if(to == 1 && top != 1){
                min_cycle = min(min_cycle, step[top] + 1);
            }
            if(!vis[to]){
                vis[to] = true;
                step[to] = step[top] + 1;
                q.push(to);
            }
        }
    }
    if(min_cycle != INF){
        printf("%lld", min_cycle);
    } else {
        printf("-1");
    }
    return 0;
}
```

### Explanation
1. **Initialization**: We initialize the BFS queue with node 1 and mark it as visited. The step count for node 1 is set to 0.
2. **BFS Traversal**: For each node dequeued, we explore all its neighbors. If a neighbor is node 1 and the current node is not node 1, we update the minimum cycle length encountered.
3. **Tracking Visited Nodes**: To avoid cycles and unnecessary recomputations, nodes once visited are not processed again.
4. **Result Determination**: After BFS completes, if a cycle back to node 1 was found, the minimal cycle length is printed; otherwise, -1 is printed indicating no such cycle exists.

This approach ensures that we efficiently find the shortest cycle involving node 1 with a time complexity of O(N + M), where N is the number of nodes and M is the number of edges, which is optimal for this problem.
