To fix this code, we need to address the issue that the original DFS approach is not correctly finding the shortest cycle that includes node 1. The problem likely requires finding the shortest cycle starting and ending at node 1, which is essentially the shortest path from node 1 back to itself. DFS might not be the best approach for this because it can get stuck in long paths before finding shorter ones. Instead, a BFS approach would be more suitable as it explores all nodes level by level, ensuring the first time we return to node 1 gives the shortest cycle.

### Approaches
1. **BFS for Shortest Path**: Use BFS to explore the graph level by level from node 1. The moment we encounter node 1 again during the BFS, the current level (distance) gives the length of the shortest cycle.
2. **Visited Tracking**: Keep track of visited nodes to avoid re-processing them, ensuring efficiency. However, since we are looking for a cycle starting and ending at node 1, we need to handle node 1 specially by not marking it as visited initially.
3. **Early Termination**: As soon as we find node 1 during BFS (except the initial visit), we can terminate early and return the current distance.

### Solution Code
```cpp
#include <bits/stdc++.h>
#define int long long
using namespace std;

const int N = 2e5 + 5;

vector<int> g[N];
int dist[N];

signed main() {
    int n, m;
    cin >> n >> m;
    for (int i = 1; i <= m; i++) {
        int u, v;
        cin >> u >> v;
        g[u].push_back(v);
    }

    queue<int> q;
    fill(dist, dist + N, -1);
    q.push(1);
    dist[1] = 0;
    int ans = -1;

    while (!q.empty()) {
        int u = q.front();
        q.pop();
        for (int v : g[u]) {
            if (v == 1) {
                if (ans == -1 || dist[u] + 1 < ans) {
                    ans = dist[u] + 1;
                }
            }
            if (dist[v] == -1) {
                dist[v] = dist[u] + 1;
                q.push(v);
            }
        }
    }

    cout << ans << endl;
    return 0;
}
```

### Explanation
1. **Initialization**: The graph is represented using an adjacency list `g`, and `dist` array is initialized to -1 to indicate unvisited nodes.
2. **BFS Setup**: Start BFS from node 1 with `dist[1]` set to 0.
3. **BFS Execution**: For each node `u` dequeued, explore all its neighbors `v`. If `v` is node 1, update the answer `ans` with the current distance (dist[u] + 1) if it's shorter than the previously found cycle.
4. **Visited Handling**: For other nodes, if they haven't been visited (`dist[v] == -1`), update their distance and enqueue them.
5. **Result Output**: After BFS completes, output the shortest cycle length found (`ans`), or -1 if no cycle exists.

This approach ensures that we efficiently find the shortest cycle starting and ending at node 1 using BFS, which is optimal for shortest path problems in unweighted graphs.
