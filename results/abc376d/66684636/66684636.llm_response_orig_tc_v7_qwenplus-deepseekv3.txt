To fix this code, we need to address the issue that the original code uses a depth-first search (DFS) approach to find the shortest cycle starting and ending at node 1. However, DFS is not suitable for finding the shortest cycle in an unweighted graph because it explores paths in a depth-first manner, which may not guarantee the shortest path is found first. Instead, we should use a breadth-first search (BFS) approach, which is more appropriate for finding the shortest path in unweighted graphs.

### Approaches
1. **Problem Analysis**: The problem requires finding the shortest cycle that starts and ends at node 1 in a directed graph. The original DFS approach may not always find the shortest cycle because it doesn't explore all possible paths level by level.
2. **Intuition**: BFS is more suitable because it explores all nodes at the present depth level before moving on to nodes at the next depth level. This ensures that the first time we revisit node 1, we have found the shortest cycle.
3. **Algorithm Selection**: We will use BFS to explore the graph from node 1, keeping track of the distance from node 1 to each node. If we encounter node 1 during the BFS, we can immediately return the current distance plus one (forming the cycle).
4. **Complexity Analysis**: The BFS approach runs in O(N + M) time, where N is the number of nodes and M is the number of edges, which is efficient for the problem constraints.

### Solution Code
```cpp
#include <bits/stdc++.h>
#define int long long
using namespace std;

const int N = 2e5 + 5;
vector<int> g[N];
int n, m;

int findShortestCycle() {
    int ans = INT_MAX;
    vector<int> dist(n + 1, -1);
    queue<int> q;
    dist[1] = 0;
    q.push(1);
    
    while (!q.empty()) {
        int u = q.front();
        q.pop();
        for (int v : g[u]) {
            if (v == 1) {
                ans = min(ans, dist[u] + 1);
            }
            if (dist[v] == -1) {
                dist[v] = dist[u] + 1;
                q.push(v);
            }
        }
    }
    return ans == INT_MAX ? -1 : ans;
}

signed main() {
    cin >> n >> m;
    for (int i = 1; i <= m; i++) {
        int u, v;
        cin >> u >> v;
        g[u].push_back(v);
    }
    int ans = findShortestCycle();
    cout << ans << endl;
    return 0;
}
```

1. **Initialization**: The graph is represented using an adjacency list `g`. The `dist` array keeps track of the shortest distance from node 1 to each node, initialized to -1 (unvisited). 
2. **BFS Setup**: The BFS starts from node 1 with distance 0. Node 1 is enqueued to begin the BFS.
3. **BFS Execution**: For each node `u` dequeued, all its adjacent nodes `v` are checked. If `v` is node 1, a cycle is found, and the current distance plus one (to form the cycle) is compared with the current minimum cycle length. If `v` hasn't been visited, its distance is updated, and it is enqueued for further exploration.
4. **Result Handling**: After BFS completes, if a cycle was found (i.e., `ans` was updated from `INT_MAX`), the shortest cycle length is printed; otherwise, -1 is printed indicating no cycle exists.

This approach ensures that the shortest cycle starting and ending at node 1 is found efficiently using BFS, which is optimal for unweighted graphs.
