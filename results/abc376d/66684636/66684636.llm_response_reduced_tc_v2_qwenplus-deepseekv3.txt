However, DFS is not suitable for finding shortest paths in unweighted graphs; breadth-first search (BFS) is more appropriate for this task. The DFS approach can miss shorter cycles because it explores paths deeply before exploring all possible neighbors at the current level.

To fix this, we should use BFS to find the shortest cycle that includes node 1. Here's the corrected code:

#include <bits/stdc++.h>
#define int long long
using namespace std;

const int N = 2e5 + 5;
const int INF = 1e18;

vector<int> g[N];
int dist[N];
int n, m, ans = INF;

void bfs() {
    queue<pair<int, int>> q;
    q.push({1, -1});
    dist[1] = 0;

    while (!q.empty()) {
        auto [u, parent] = q.front();
        q.pop();

        for (int v : g[u]) {
            if (v == parent) continue;
            if (dist[v] == -1) {
                dist[v] = dist[u] + 1;
                q.push({v, u});
            } else {
                ans = min(ans, dist[u] + dist[v] + 1);
            }
        }
    }
}

signed main() {
    cin >> n >> m;
    for (int i = 1; i <= m; i++) {
        int u, v;
        cin >> u >> v;
        g[u].push_back(v);
        g[v].push_back(u);
    }

    memset(dist, -1, sizeof dist);
    bfs();

    if (ans != INF) cout << ans;
    else cout << -1;
    return 0;
}

### Explanation:
1. **BFS Initialization**: We initialize the BFS from node 1 with a distance of 0. The queue stores pairs of nodes and their parent to avoid revisiting the immediate parent.
2. **Distance Tracking**: The `dist` array keeps track of the shortest distance from node 1 to each other node.
3. **Cycle Detection**: If we encounter a node `v` that has already been visited (i.e., `dist[v] != -1`), it means we've found a cycle. The length of this cycle is `dist[u] + dist[v] + 1`, which we compare with the current shortest cycle length.
4. **Output**: After BFS completes, if a cycle was found (`ans != INF`), we print its length; otherwise, we print -1.
