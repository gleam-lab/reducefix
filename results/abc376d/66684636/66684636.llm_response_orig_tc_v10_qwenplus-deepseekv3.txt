To fix this code, we need to correctly identify the shortest cycle that starts and ends at node 1. The original code uses a DFS approach which may not be the most efficient or correct method for finding the shortest cycle, especially in large graphs. Instead, we should use a BFS approach which is more suitable for finding shortest paths in unweighted graphs.

### Approaches
1. **BFS for Shortest Cycle**: We will perform a BFS starting from node 1, keeping track of the distance from node 1 to all other nodes. If we encounter node 1 again during the BFS, we can calculate the cycle length as the current distance plus 1.
2. **Adjacency List**: We will use an adjacency list to represent the graph, which is efficient for traversal operations.
3. **Distance Tracking**: We will maintain an array to track the shortest distance from node 1 to each node. This helps in efficiently determining the cycle length when we revisit node 1.

### Solution Code
```cpp
#include <bits/stdc++.h>
#define int long long
using namespace std;

const int N = 2e5 + 5;
const int INF = 1e18;

vector<int> g[N];
int dist[N];
int n, m;

int findShortestCycle() {
    int ans = INF;
    for (int u = 1; u <= n; ++u) {
        for (int v : g[u]) {
            if (v == 1) {
                ans = 1;
                break;
            }
        }
        if (ans == 1) break;
    }
    if (ans == 1) return ans;

    queue<int> q;
    fill(dist, dist + n + 1, INF);
    dist[1] = 0;
    q.push(1);

    while (!q.empty()) {
        int u = q.front();
        q.pop();
        for (int v : g[u]) {
            if (v == 1) {
                ans = min(ans, dist[u] + 1);
            }
            if (dist[v] == INF) {
                dist[v] = dist[u] + 1;
                q.push(v);
            }
        }
    }

    return ans == INF ? -1 : ans;
}

signed main() {
    cin >> n >> m;
    for (int i = 1; i <= m; i++) {
        int u, v;
        cin >> u >> v;
        g[u].push_back(v);
    }
    int ans = findShortestCycle();
    cout << ans << endl;
    return 0;
}
```

### Explanation
1. **Initial Check for Immediate Cycle**: The loop checks if there is a direct edge from node 1 to itself, which forms a cycle of length 1. If found, it immediately returns 1 as the shortest cycle length.
2. **BFS Initialization**: The BFS starts from node 1, initializing the distance of node 1 to 0 and all other nodes to infinity. The queue is initialized with node 1.
3. **BFS Traversal**: For each node dequeued, the algorithm checks all its neighbors. If a neighbor is node 1, it updates the shortest cycle length found so far. Otherwise, if the neighbor hasn't been visited, it sets the neighbor's distance and enqueues it.
4. **Result Handling**: After BFS completes, if no cycle was found (indicated by `ans` remaining `INF`), the function returns -1. Otherwise, it returns the shortest cycle length found.

This approach ensures that the shortest cycle starting and ending at node 1 is found efficiently using BFS, which is optimal for unweighted graphs.
