To fix this code, we need to ensure that it correctly finds the shortest cycle starting and ending at node 1 in the given directed graph. The original code uses a depth-first search (DFS) approach, which may not be the most efficient or correct way to find the shortest cycle in a directed graph. Instead, we can use a breadth-first search (BFS) approach, which is more suitable for finding shortest paths in unweighted graphs.

### Approaches
1. **BFS for Shortest Cycle Detection**: We will perform a BFS starting from node 1 to explore all possible paths. For each node, we will keep track of the distance from node 1. If we encounter node 1 again during the BFS, we can determine the length of the cycle by adding 1 to the current distance.
2. **Early Termination**: The BFS will terminate as soon as it finds a cycle involving node 1, ensuring we get the shortest possible cycle early.
3. **Distance Tracking**: We maintain a distance array to keep track of the shortest distance from node 1 to each node. This helps in efficiently determining the cycle length when we revisit node 1.

### Solution Code
```cpp
#include <bits/stdc++.h>
#define int long long
using namespace std;

const int N = 2e5 + 5;
const int INF = 1e18;

vector<int> g[N];
int dist[N];
int n, m, ans = INF;

void bfs(int start) {
    fill(dist, dist + n + 1, INF);
    queue<int> q;
    q.push(start);
    dist[start] = 0;

    while (!q.empty()) {
        int u = q.front();
        q.pop();

        for (int v : g[u]) {
            if (v == start) {
                ans = min(ans, dist[u] + 1);
            }
            if (dist[v] > dist[u] + 1) {
                dist[v] = dist[u] + 1;
                q.push(v);
            }
        }
    }
}

signed main() {
    cin >> n >> m;
    for (int i = 1; i <= m; i++) {
        int u, v;
        cin >> u >> v;
        g[u].push_back(v);
    }
    bfs(1);
    if (ans != INF) {
        cout << ans;
    } else {
        cout << -1;
    }
    return 0;
}
```

### Explanation
1. **Initialization**: We initialize the distance array `dist` with infinity (INF), indicating that initially, all nodes are unreachable from node 1.
2. **BFS Setup**: We start BFS from node 1, setting its distance to 0. The BFS explores each neighbor of the current node.
3. **Cycle Detection**: During BFS, if we encounter node 1 again (i.e., a neighbor of the current node is node 1), we update the shortest cycle length `ans` with the current distance plus 1 (the edge back to node 1).
4. **Distance Update**: For each node, if a shorter path is found (via the current node), we update its distance and enqueue it for further exploration.
5. **Result Output**: After BFS completes, if a cycle was found (`ans` is not INF), we print the shortest cycle length; otherwise, we print -1 indicating no cycle exists.

This approach efficiently finds the shortest cycle involving node 1 using BFS, ensuring optimal performance with a time complexity of O(N + M), where N is the number of nodes and M is the number of edges.
