To solve this problem, we need to find the shortest path from node 1 to node `n` in an undirected graph where each node can have multiple edges connecting it to other nodes. The goal is to minimize the number of steps taken to reach node `n`.

Here's the corrected version of the C++ code:

```cpp
#include <bits/stdc++.h>
#define int long long
using namespace std;

const int N = 2e5 + 5;

bool vis[N];
vector<int> g[N];
int n, m, ans = INT_MAX;

void bfs() {
    queue<pair<int, int>> q;
    q.push({1, 0});
    vis[1] = true;

    while (!q.empty()) {
        auto [u, cnt] = q.front();
        q.pop();

        if (u == n) {
            ans = min(ans, cnt);
            continue;
        }

        for (int v : g[u]) {
            if (!vis[v]) {
                vis[v] = true;
                q.push({v, cnt + 1});
            }
        }
    }
}

signed main() {
    cin >> n >> m;
    for (int i = 1; i <= m; i++) {
        int u, v;
        cin >> u >> v;
        g[u].push_back(v);
        g[v].push_back(u); // Since the graph is undirected
    }
    
    memset(vis, false, sizeof(vis));
    bfs();
    
    if (ans != INT_MAX) cout << ans << endl;
    else cout << -1 << endl;
    
    return 0;
}
```

### Explanation of Changes:
1. **BFS Algorithm**: We use Breadth-First Search (BFS) instead of Depth-First Search (DFS). BFS is more suitable for finding the shortest path in an unweighted graph because it explores all neighbors at the present depth level before moving on to nodes at the next depth level.

2. **Initialization**: We initialize the distance array with `INT_MAX` to represent that initially, no node is reachable except the starting node (node 1), which has a distance of 0.

3. **Queue Usage**: We use a queue to keep track of the current node and its distance from the start node. This allows us to explore the graph layer by layer.

4. **Edge Handling**: Since the graph is undirected, we add both `(u, v)` and `(v, u)` edges to the adjacency list.

5. **Result**: If we find a path to node `n`, we update the answer with the minimum distance found. If no path is found, we output `-1`.

This approach ensures that we efficiently find the shortest path using BFS, which runs in \(O(V + E)\) time complexity, where \(V\) is the number of vertices and \(E\) is the number of edges.
