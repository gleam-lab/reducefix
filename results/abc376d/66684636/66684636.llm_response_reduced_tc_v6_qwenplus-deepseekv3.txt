To fix this code, we need to address the issue that the current DFS approach is not correctly finding the shortest cycle involving node 1. The problem requires finding the shortest cycle that starts and ends at node 1, meaning we need to explore all possible paths from node 1 back to itself and keep track of the shortest path length.

### Analysis
1. **Current Approach**: The code uses a DFS approach to explore paths starting from node 1. If it finds a path back to node 1, it updates the minimum cycle length (`ans`). However, DFS does not inherently find the shortest path in an unweighted graph; it explores paths in a depth-first manner, which can lead to suboptimal results.
2. **Correct Approach**: BFS is more suitable for finding the shortest path in an unweighted graph. We can use BFS to explore all nodes level by level from node 1, keeping track of the distance from node 1. The first time we encounter node 1 again (other than the initial visit), we can return the current distance as the shortest cycle length.

### Solution Code
```cpp
#include <bits/stdc++.h>
#define int long long
using namespace std;

const int N = 2e5 + 5;

vector<int> g[N];
int n, m, ans = 1e18;

void bfs() {
    queue<pair<int, int>> q;
    vector<int> dist(n + 1, -1);
    q.push({1, 0});
    dist[1] = 0;
    
    while (!q.empty()) {
        auto [u, d] = q.front();
        q.pop();
        
        for (int v : g[u]) {
            if (v == 1 && d + 1 < ans) {
                ans = d + 1;
                continue;
            }
            if (dist[v] == -1) {
                dist[v] = d + 1;
                q.push({v, d + 1});
            }
        }
    }
}

signed main() {
    cin >> n >> m;
    for (int i = 1; i <= m; i++) {
        int u, v;
        cin >> u >> v;
        g[u].push_back(v);
    }
    bfs();
    if (ans != (int)1e18) cout << ans;
    else cout << -1;
    return 0;
}
```

### Explanation
1. **BFS Initialization**: We initialize a queue to manage the BFS traversal and a distance array to keep track of the shortest distance from node 1 to each other node.
2. **BFS Traversal**: For each node dequeued, we explore its neighbors. If a neighbor is node 1, we update the shortest cycle length (`ans`) if the current path length (`d + 1`) is shorter than the previously found cycle length.
3. **Distance Update**: For other nodes, if they haven't been visited yet (distance is -1), we update their distance and enqueue them for further exploration.
4. **Result Output**: After BFS completes, if a cycle involving node 1 was found, we print the shortest cycle length; otherwise, we print -1.

This approach ensures that we efficiently find the shortest cycle involving node 1 using BFS, which is optimal for shortest path problems in unweighted graphs.
