Instead, we should use BFS (Breadth-First Search) for finding the shortest path in unweighted graphs, which inherently finds the shortest path due to its level-order traversal.

### Approaches
1. **Problem Analysis**: The task is to find the shortest cycle that starts and ends at node 1. This can be rephrased as finding the shortest path from node 1 back to itself, visiting at least one other node in between.
2. **Insight**: Using BFS, we can explore nodes level by level, which ensures that the first time we revisit node 1, we have found the shortest cycle.
3. **Algorithm Selection**: BFS is suitable here because it explores all nodes at the present depth before moving on to nodes at the next depth level, ensuring the shortest path is found first.
4. **Complexity Analysis**: BFS runs in O(V + E) time, where V is the number of vertices and E is the number of edges, which is efficient for the problem constraints.

### Solution Code
```cpp
#include <bits/stdc++.h>
#define int long long
using namespace std;

const int N = 2e5 + 5;
const int INF = 1e18;

vector<int> g[N];
int dist[N];
int n, m;

signed main() {
    cin >> n >> m;
    for (int i = 1; i <= m; i++) {
        int u, v;
        cin >> u >> v;
        g[u].push_back(v);
    }

    int ans = INF;
    for (int start_node : g[1]) {
        queue<int> q;
        fill(dist, dist + N, INF);
        dist[start_node] = 1;
        q.push(start_node);

        while (!q.empty()) {
            int u = q.front();
            q.pop();

            for (int v : g[u]) {
                if (v == 1) {
                    ans = min(ans, dist[u] + 1);
                    continue;
                }
                if (dist[v] > dist[u] + 1) {
                    dist[v] = dist[u] + 1;
                    q.push(v);
                }
            }
        }
    }

    if (ans != INF) {
        cout << ans << endl;
    } else {
        cout << -1 << endl;
    }
    return 0;
}
```

1. **Initialization**: We initialize the graph adjacency list and read the input values for nodes and edges.
2. **BFS for Shortest Cycle Detection**: For each neighbor of node 1, we perform a BFS to find the shortest path back to node 1. The distance from the neighbor to node 1 is tracked, and whenever node 1 is reached, the current path length is compared to the minimum found so far.
3. **Result Handling**: After processing all neighbors, if a valid cycle is found (ans is not INF), we print the shortest cycle length; otherwise, we print -1 indicating no cycle exists.

This approach ensures that we efficiently find the shortest cycle starting and ending at node 1 using BFS, which is optimal for unweighted shortest path problems.
