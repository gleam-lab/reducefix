The original code uses a DFS approach which may not be optimal for finding shortest cycles. Here's the corrected solution using BFS which is more suitable for shortest path problems:

```cpp
#include <bits/stdc++.h>
#define int long long
using namespace std;

const int N = 2e5 + 5;
const int INF = 1e18;

vector<int> g[N];
int dist[N];
int n, m, ans = INF;

void bfs() {
    queue<int> q;
    q.push(1);
    dist[1] = 0;
    
    while (!q.empty()) {
        int u = q.front();
        q.pop();
        
        for (int v : g[u]) {
            if (v == 1) {
                ans = min(ans, dist[u] + 1);
                continue;
            }
            if (dist[v] > dist[u] + 1) {
                dist[v] = dist[u] + 1;
                q.push(v);
            }
        }
    }
}

signed main() {
    cin >> n >> m;
    for (int i = 1; i <= m; i++) {
        int u, v;
        cin >> u >> v;
        g[u].push_back(v);
    }
    
    fill(dist, dist + n + 1, INF);
    bfs();
    
    if (ans != INF) cout << ans;
    else cout << -1;
    
    return 0;
}
```

Key improvements:
1. Changed from DFS to BFS since we need the shortest cycle
2. Added distance array to track shortest paths
3. Initialize distances to INF
4. When we encounter node 1 during traversal, we update the answer with the current cycle length
5. More efficient for shortest path problems in unweighted graphs (O(N + M) complexity)

This should correctly find the shortest cycle starting and ending at node 1, passing all test cases including the one where the expected output was 3.

The BFS approach is more appropriate here because:
- It explores nodes level by level from the source
- The first time we find a path back to node 1 will be the shortest cycle
- It avoids the potential infinite recursion that DFS might encounter in cyclic graphs
- More efficient than DFS for shortest path problems in unweighted graphs
