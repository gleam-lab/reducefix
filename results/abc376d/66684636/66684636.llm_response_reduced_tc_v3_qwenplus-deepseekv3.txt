To fix this code, we need to address the issue where the DFS approach fails to find the shortest cycle involving node 1. The original code attempts to use DFS to find the shortest cycle, but DFS is not suitable for finding the shortest path in an unweighted graph because it does not explore all paths in a breadth-first manner. Instead, BFS (Breadth-First Search) is more appropriate for finding the shortest path in an unweighted graph.

### Approaches
1. **BFS for Shortest Cycle Detection**: Instead of using DFS, we should use BFS to explore the graph level by level from node 1. This ensures that the first time we encounter node 1 again, we have found the shortest cycle.
2. **Tracking Parent Nodes**: To avoid revisiting the immediate parent node and getting stuck in an infinite loop, we need to keep track of the parent node during BFS traversal.
3. **Early Termination**: The BFS can terminate early once a cycle is detected, as it guarantees the shortest path due to the nature of BFS.

### Solution Code
```cpp
#include <bits/stdc++.h>
#define int long long
using namespace std;

const int N = 2e5 + 5;

vector<int> g[N];
int n, m;

int bfs() {
    vector<int> dist(n + 1, -1);
    queue<pair<int, int>> q; // {node, parent}
    q.push({1, -1});
    dist[1] = 0;
    int ans = 1e18;

    while (!q.empty()) {
        auto [u, parent] = q.front();
        q.pop();

        for (int v : g[u]) {
            if (v == parent) continue;
            if (dist[v] == -1) {
                dist[v] = dist[u] + 1;
                q.push({v, u});
            } else {
                ans = min(ans, dist[u] + dist[v] + 1);
            }
        }
    }
    return ans;
}

signed main() {
    cin >> n >> m;
    for (int i = 1; i <= m; i++) {
        int u, v;
        cin >> u >> v;
        g[u].push_back(v);
        g[v].push_back(u);
    }
    int ans = bfs();
    if (ans != (int)1e18) cout << ans;
    else cout << -1;
    return 0;
}
```

### Explanation
1. **BFS Initialization**: The BFS starts from node 1 with an initial distance of 0. The queue stores pairs of nodes and their parent nodes to avoid backtracking.
2. **Distance Tracking**: A distance array keeps track of the shortest distance from node 1 to each node. If a node is revisited, it means a cycle is detected, and the sum of the distances from node 1 to the current node and the neighbor node plus one (the edge between them) gives the cycle length.
3. **Cycle Detection**: The shortest cycle length is updated whenever a cycle is detected during the BFS traversal. The BFS ensures the first detected cycle is the shortest.
4. **Output**: If a cycle is found, the shortest cycle length is printed; otherwise, -1 is printed indicating no cycle involving node 1.

This approach efficiently finds the shortest cycle involving node 1 using BFS, which is optimal for unweighted graphs and ensures correctness across all test cases.
