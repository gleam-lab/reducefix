To fix this code, we need to address the issue that the original DFS approach incorrectly finds the shortest cycle involving node 1. The problem requires finding the shortest cycle that starts and ends at node 1, not just any cycle that includes node 1. The original code counts the number of nodes in the path, but the cycle length should be the number of edges, which is one less than the number of nodes in the cycle.

### Approaches
1. **BFS for Shortest Path**: 
   - Use BFS to explore nodes level by level from node 1.
   - Track the parent of each node to avoid revisiting the immediate parent.
   - If we encounter node 1 again during BFS, it means we've found a cycle. The level at which node 1 is revisited gives the length of the cycle.

2. **Algorithm**:
   - Initialize a queue for BFS starting from node 1.
   - Keep a distance array to track the shortest path distances from node 1.
   - For each neighbor of the current node, if it hasn't been visited, update its distance and enqueue it.
   - If the neighbor is node 1 and it's not the immediate parent, update the minimum cycle length found.
   - The shortest cycle length is the minimum value found during BFS.

### Solution Code
```cpp
#include <bits/stdc++.h>
#define int long long
using namespace std;

const int N = 2e5 + 5;
const int INF = 1e18;

vector<int> g[N];
int dist[N];
int n, m, ans = INF;

void bfs() {
    queue<int> q;
    q.push(1);
    dist[1] = 0;

    while (!q.empty()) {
        int u = q.front();
        q.pop();

        for (int v : g[u]) {
            if (v == 1 && u != 1) {
                ans = min(ans, dist[u] + 1);
            }
            if (dist[v] == INF) {
                dist[v] = dist[u] + 1;
                q.push(v);
            }
        }
    }
}

signed main() {
    cin >> n >> m;
    for (int i = 1; i <= m; i++) {
        int u, v;
        cin >> u >> v;
        g[u].push_back(v);
    }

    fill(dist, dist + N, INF);
    bfs();

    if (ans != INF) {
        cout << ans;
    } else {
        cout << -1;
    }
    return 0;
}
```

### Explanation
- **Initialization**: The graph is built using adjacency lists, and the distances are initialized to infinity except for node 1, which starts at distance 0.
- **BFS Execution**: The BFS starts from node 1. For each node, its neighbors are checked. If node 1 is encountered again (and it's not the immediate parent), the cycle length is updated if it's shorter than the previously found one.
- **Result Handling**: After BFS completes, if a cycle was found (ans is not infinity), the shortest cycle length is printed; otherwise, -1 is printed indicating no cycle was found.

This approach efficiently finds the shortest cycle involving node 1 by leveraging BFS, which inherently explores nodes in order of their distance from the start node, ensuring the first cycle found is the shortest.
