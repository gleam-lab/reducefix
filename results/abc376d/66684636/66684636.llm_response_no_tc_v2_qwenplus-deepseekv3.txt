The original code attempts to find the shortest cycle starting and ending at node 1 in a directed graph using DFS. However, it has several issues:

2. The condition `if (v != 1 && vis[v])` incorrectly skips node 1 even when it's the target node we want to complete the cycle with.
3. DFS is not the most efficient way to find shortest cycles in unweighted graphs - BFS would be better.

Here's the fixed version using BFS which is more appropriate for finding shortest paths in unweighted graphs:

```cpp
#include <bits/stdc++.h>
#define int long long
using namespace std;

const int N = 2e5 + 5;
const int INF = 1e18;

vector<int> g[N];
int n, m;

int find_min_cycle() {
    int min_cycle = INF;
    vector<int> dist(n + 1, INF);
    queue<pair<int, int>> q;
    
    dist[1] = 0;
    q.push({1, -1}); // {current node, parent node}
    
    while (!q.empty()) {
        auto [u, parent] = q.front();
        q.pop();
        
        for (int v : g[u]) {
            if (v == 1 && u != parent) {
                min_cycle = min(min_cycle, dist[u] + 1);
            }
            if (dist[v] > dist[u] + 1) {
                dist[v] = dist[u] + 1;
                q.push({v, u});
            }
        }
    }
    
    return min_cycle == INF ? -1 : min_cycle;
}

signed main() {
    cin >> n >> m;
    for (int i = 1; i <= m; i++) {
        int u, v;
        cin >> u >> v;
        g[u].push_back(v);
    }
    
    int ans = find_min_cycle();
    cout << ans << endl;
    
    return 0;
}
```

Key improvements:
1. Uses BFS instead of DFS for shortest path finding
2. Properly tracks distances and updates them
3. Checks for cycles back to node 1 specifically
4. Handles the case where no cycle exists by returning -1
5. More efficient and correct for finding shortest cycles in unweighted graphs

The BFS approach ensures we find the shortest cycle (if one exists) by exploring nodes level by level, and the distance tracking prevents unnecessary revisits while still allowing cycle detection.
