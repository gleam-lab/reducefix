To fix this code, we need to address the issue of finding the shortest cycle in a directed graph that starts and ends at node 1. The original code uses a depth-first search (DFS) approach, which is not suitable for finding shortest paths in graphs with cycles due to its nature of exploring paths deeply before backtracking. Instead, we should use a breadth-first search (BFS) approach, which is more appropriate for finding shortest paths in unweighted graphs.

### Approaches
1. **Problem Analysis**: The task is to find the shortest cycle that starts and ends at node 1 in a directed graph. If no such cycle exists, we should return -1.
2. **Algorithm Selection**: BFS is optimal for finding the shortest path in unweighted graphs. We will perform a BFS starting from node 1, keeping track of visited nodes and the distance from node 1. If we revisit node 1 during the BFS, we can determine the length of the cycle.
3. **Complexity Analysis**: The BFS approach runs in O(N + M) time, where N is the number of nodes and M is the number of edges, which is efficient for the problem constraints.

### Solution Code
```cpp
#include <bits/stdc++.h>
#define int long long
using namespace std;

const int N = 2e5 + 5;
const int INF = 1e18;

vector<int> g[N];
int dist[N];
int n, m;

signed main() {
    cin >> n >> m;
    for (int i = 1; i <= m; i++) {
        int u, v;
        cin >> u >> v;
        g[u].push_back(v);
    }

    int ans = INF;
    queue<pair<int, int>> q;
    q.push({1, 0});
    dist[1] = 0;
    fill(dist, dist + N, INF);
    dist[1] = 0;

    while (!q.empty()) {
        auto [u, d] = q.front();
        q.pop();

        for (int v : g[u]) {
            if (v == 1) {
                ans = min(ans, d + 1);
            }
            if (dist[v] > d + 1) {
                dist[v] = d + 1;
                q.push({v, d + 1});
            }
        }
    }

    if (ans != INF) {
        cout << ans << endl;
    } else {
        cout << -1 << endl;
    }

    return 0;
}
```

### Explanation
1. **Initialization**: We read the number of nodes (n) and edges (m). We also initialize the adjacency list `g` to store the graph.
2. **Graph Construction**: We populate the adjacency list with the given edges.
3. **BFS Setup**: We initialize a queue to perform BFS, starting from node 1. The `dist` array is used to keep track of the shortest distance from node 1 to each other node, initialized to infinity.
4. **BFS Execution**: For each node dequeued, we check its neighbors. If a neighbor is node 1, we update the shortest cycle length found. Otherwise, we update the distance for the neighbor and enqueue it if a shorter path is found.
5. **Result Handling**: After BFS completes, if a cycle was found (ans is not infinity), we print the shortest cycle length. Otherwise, we print -1 indicating no cycle was found.

This approach ensures that we efficiently find the shortest cycle starting and ending at node 1 using BFS, which is optimal for shortest path problems in unweighted graphs.
