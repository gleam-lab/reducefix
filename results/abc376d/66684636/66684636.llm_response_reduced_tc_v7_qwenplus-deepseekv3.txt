To fix this code, we need to address the issue of finding the shortest cycle that includes node 1 in a directed graph. The original code uses a depth-first search (DFS) approach, which is not efficient for finding the shortest cycle in a graph because DFS explores all possible paths and can get stuck in long paths before finding the shortest one. Instead, a breadth-first search (BFS) approach is more suitable for finding the shortest path in an unweighted graph, which can be adapted to find the shortest cycle involving node 1.

### Approaches
1. **BFS for Shortest Path**: 
   - We'll use BFS to explore the graph starting from node 1. BFS explores nodes level by level, ensuring that the first time we revisit node 1, we've found the shortest possible cycle.
   - We maintain a distance array to keep track of the shortest distance from node 1 to all other nodes.
   - During BFS, if we encounter node 1 again (i.e., we find a cycle back to node 1), we update the shortest cycle length if the current path is shorter.

### Solution Code
```cpp
#include <bits/stdc++.h>
#define int long long
using namespace std;

const int N = 2e5 + 5;
const int INF = 1e18;

vector<int> g[N];
int dist[N];
int n, m, ans = INF;

void bfs() {
    queue<int> q;
    q.push(1);
    dist[1] = 0;
    
    while (!q.empty()) {
        int u = q.front();
        q.pop();
        
        for (int v : g[u]) {
            if (v == 1) {
                ans = min(ans, dist[u] + 1);
            }
            if (dist[v] == INF) {
                dist[v] = dist[u] + 1;
                q.push(v);
            }
        }
    }
}

signed main() {
    cin >> n >> m;
    for (int i = 1; i <= n; i++) {
        dist[i] = INF;
    }
    for (int i = 1; i <= m; i++) {
        int u, v;
        cin >> u >> v;
        g[u].push_back(v);
    }
    bfs();
    if (ans != INF) {
        cout << ans;
    } else {
        cout << -1;
    }
    return 0;
}
```

### Explanation
1. **Initialization**: 
   - We start by initializing the distance array `dist` with infinity (`INF`), indicating that initially, no nodes are reachable from node 1. The distance from node 1 to itself is set to 0.
2. **BFS Execution**: 
   - We use a queue to process nodes level by level. For each node `u` dequeued, we examine all its neighbors `v`.
   - If a neighbor `v` is node 1, it means we've found a cycle, and we update the shortest cycle length (`ans`) if the current path length (`dist[u] + 1`) is shorter.
   - If `v` hasn't been visited yet (i.e., `dist[v]` is still `INF`), we set its distance to `dist[u] + 1` and enqueue it for further processing.
3. **Output**: 
   - After BFS completes, if `ans` has been updated (i.e., a cycle was found), we print the shortest cycle length. Otherwise, we print `-1`, indicating no cycle involving node 1 was found.

This approach ensures that we efficiently find the shortest cycle involving node 1 by leveraging the properties of BFS, which guarantees the shortest path in an unweighted graph. The complexity is O(n + m), which is optimal for this problem.
