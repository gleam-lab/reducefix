The original code attempts to find the shortest cycle starting and ending at node 1 in a directed graph using DFS, but it has several issues:
1. It resets `vis[u]` too late, preventing proper cycle detection
2. It doesn't handle the case where node 1 has no outgoing edges
3. The DFS approach is inefficient for finding shortest cycles in general

Here's the fixed code using BFS (Breadth-First Search) which is more suitable for finding shortest paths:
```cpp
#include <bits/stdc++.h>
#define int long long
using namespace std;

const int N = 2e5 + 5;
const int INF = 1e18;

vector<int> g[N];
int dist[N];
int n, m;

signed main()
{
    cin >> n >> m;
    for (int i = 1; i <= m; i++)
    {
        int u, v;
        cin >> u >> v;
        g[u].push_back(v);
    }

    int ans = INF;
    queue<pair<int, int>> q;
    
    for (int v : g[1])
    {
        q.push({v, 1});
    }

    while (!q.empty())
    {
        auto [u, d] = q.front();
        q.pop();
        
        if (u == 1)
        {
            ans = d;
            break;
        }
        
        if (d >= ans) continue;
        
        for (int v : g[u])
        {
            if (d + 1 < ans)
            {
                q.push({v, d + 1});
            }
        }
    }

    if (ans != INF) cout << ans;
    else cout << -1;
    
    return 0;
}
```

Key improvements:
1. Uses BFS instead of DFS for shortest path finding
2. Properly tracks distance from node 1
3. Immediately checks if we return to node 1
4. Early termination when shortest cycle is found
5. Handles the case where no cycle exists
6. More efficient time complexity (O(N + M)) compared to the original DFS approach

The BFS approach is more appropriate because it explores nodes in order of their distance from the starting node, guaranteeing that the first time we return to node 1 will be via the shortest cycle.
