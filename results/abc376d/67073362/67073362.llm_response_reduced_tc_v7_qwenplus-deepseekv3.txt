To fix this code, we need to correctly find the shortest cycle that starts and ends at node 1 in the given directed graph. The original code uses a depth-first search (DFS) approach, which is not suitable for finding the shortest path in an unweighted graph. Instead, we should use a breadth-first search (BFS) approach, which is more appropriate for finding the shortest path in unweighted graphs.

### Approaches
1. **Graph Representation**: Represent the graph using an adjacency list.
2. **BFS for Shortest Path**: Perform a BFS starting from node 1 to explore all nodes level by level. For each node, keep track of the distance from node 1.
3. **Check for Cycles**: During the BFS, if we encounter node 1 again (other than the initial node), it means we have found a cycle. The length of this cycle will be the current distance plus one.
4. **Output the Result**: The shortest cycle found during the BFS will be the answer. If no cycle is found, output -1.

### Solution Code
```cpp
#include<iostream>
#include<vector>
#include<queue>
#include<climits>
using namespace std;

const int N = 2e5 + 10;
const int INF = INT_MAX;

vector<int> adj[N];
int dist[N];

void solve() {
    int n, m;
    cin >> n >> m;
    
    for (int i = 0; i < m; i++) {
        int u, v;
        cin >> u >> v;
        adj[u].push_back(v);
    }
    
    fill(dist, dist + n + 1, INF);
    queue<int> q;
    q.push(1);
    dist[1] = 0;
    
    int min_cycle = INF;
    
    while (!q.empty()) {
        int u = q.front();
        q.pop();
        
        for (int v : adj[u]) {
            if (v == 1) {
                min_cycle = min(min_cycle, dist[u] + 1);
            }
            if (dist[v] > dist[u] + 1) {
                dist[v] = dist[u] + 1;
                q.push(v);
            }
        }
    }
    
    if (min_cycle != INF) {
        cout << min_cycle << endl;
    } else {
        cout << -1 << endl;
    }
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);
    
    int T = 1;
    // cin >> T;
    while (T--) {
        solve();
    }
    return 0;
}
```

### Explanation
1. **Graph Representation**: The graph is represented using an adjacency list `adj`, where each node points to its neighbors.
2. **BFS Initialization**: Initialize a queue for BFS and set the initial distance of node 1 to 0. All other nodes' distances are set to infinity.
3. **BFS Execution**: For each node dequeued, check all its neighbors. If a neighbor is node 1, update the minimum cycle length. Otherwise, if a neighbor's distance can be improved, update its distance and enqueue it.
4. **Result Output**: After BFS completes, if a cycle (path back to node 1) was found, output the shortest cycle length. Otherwise, output -1 indicating no cycle exists.

This approach ensures that we efficiently find the shortest cycle starting and ending at node 1 using BFS, which is optimal for unweighted graphs. The time complexity is O(N + M), where N is the number of nodes and M is the number of edges.
