### Analysis of the Problem
The problem requires finding the shortest cycle in a directed graph that starts and ends at node 1. The initial code uses a DFS approach to traverse the graph and attempts to find the shortest cycle by keeping track of the depth when revisiting node 1. However, this approach fails to correctly identify the shortest cycle because DFS does not naturally find the shortest path in an unweighted graph—BFS is more appropriate for this purpose.

### Issues in the Original Code
1. **DFS Approach**: DFS explores as far as possible along each branch before backtracking, which doesn't guarantee finding the shortest cycle first.
2. **Cycle Detection Logic**: The code only checks if it revisits node 1, but doesn't efficiently track the shortest path back to node 1.
3. **Global Minimum Update**: The minimum cycle length is updated whenever node 1 is revisited, but this doesn't ensure the shortest path is found first.

### Solution Approach
To find the shortest cycle starting and ending at node 1, we should use BFS (Breadth-First Search) because it explores all nodes at the present depth level before moving on to nodes at the next depth level. This property ensures that the first time we revisit node 1, we have found the shortest cycle.

1. **BFS Initialization**: Start BFS from node 1, keeping track of the distance from node 1 to all other nodes.
2. **Cycle Detection**: For each neighbor of a node, if the neighbor is node 1 and the current node is not node 1 itself, update the minimum cycle length.
3. **Early Termination**: If we find a cycle back to node 1 during BFS, we can terminate early since BFS guarantees the first found cycle is the shortest.

### Final Corrected Code
```cpp
#include<iostream>
#include<queue>
#include<vector>
#include<climits>
using namespace std;

const int N = 2e5 + 10;
vector<int> adj[N];
int dist[N];

void solve() {
    int n, m;
    cin >> n >> m;
    for (int i = 0; i < m; ++i) {
        int u, v;
        cin >> u >> v;
        adj[u].push_back(v);
    }

    fill(dist, dist + n + 1, INT_MAX);
    queue<int> q;
    q.push(1);
    dist[1] = 0;
    int min_cycle = INT_MAX;

    while (!q.empty()) {
        int u = q.front();
        q.pop();

        for (int v : adj[u]) {
            if (v == 1) {
                min_cycle = min(min_cycle, dist[u] + 1);
            }
            if (dist[v] > dist[u] + 1) {
                dist[v] = dist[u] + 1;
                q.push(v);
            }
        }
    }

    if (min_cycle == INT_MAX) {
        cout << -1 << endl;
    } else {
        cout << min_cycle << endl;
    }
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int T = 1;
    while (T--) {
        solve();
    }

    return 0;
}
```

### Explanation
1. **Initialization**: The adjacency list `adj` stores the directed graph, and `dist` array keeps track of the shortest distance from node 1 to each node.
2. **BFS Setup**: The BFS starts from node 1 with `dist[1] = 0`. The queue `q` manages the nodes to be visited.
3. **Cycle Detection**: For each node `u`, if a neighbor `v` is node 1, the cycle length is `dist[u] + 1`. This value updates `min_cycle` if it's shorter than the current minimum.
4. **Distance Update**: If a shorter path to node `v` is found via `u`, update `dist[v]` and enqueue `v`.
5. **Result Output**: After BFS completes, if no cycle is found (i.e., `min_cycle` remains `INT_MAX`), output `-1`. Otherwise, output the shortest cycle length.

This approach ensures the shortest cycle is found efficiently with a time complexity of O(N + M), where N is the number of nodes and M is the number of edges.
