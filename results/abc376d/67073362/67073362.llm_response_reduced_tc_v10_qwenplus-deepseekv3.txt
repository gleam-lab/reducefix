To fix this code, we need to address the issue of finding the shortest cycle that starts and ends at node 1. The original code uses a DFS approach, which is not suitable for finding the shortest cycle in a graph efficiently. Instead, we should use a BFS approach, which is more appropriate for finding shortest paths in unweighted graphs.

### Approaches
1. **Graph Representation**: Represent the graph using an adjacency list.
2. **BFS for Shortest Cycle**: Perform a BFS from node 1 to find the shortest cycle that returns to node 1. The BFS will explore nodes level by level, ensuring that the first time we return to node 1, we have found the shortest cycle.
3. **Distance Tracking**: Maintain a distance array to keep track of the shortest distance from node 1 to each node. This helps in detecting cycles efficiently.

### Solution Code
```cpp
#include<iostream>
#include<vector>
#include<queue>
#include<climits>
using namespace std;

const int N = 2e5 + 10;
vector<int> adj[N];
int dist[N];

void solve() {
    int n, m;
    cin >> n >> m;
    for (int i = 0; i < m; ++i) {
        int u, v;
        cin >> u >> v;
        adj[u].push_back(v);
    }
    
    queue<int> q;
    fill(dist, dist + n + 1, INT_MAX);
    dist[1] = 0;
    q.push(1);
    int min_cycle = INT_MAX;
    
    while (!q.empty()) {
        int u = q.front();
        q.pop();
        
        for (int v : adj[u]) {
            if (v == 1) {
                min_cycle = min(min_cycle, dist[u] + 1);
            }
            if (dist[v] > dist[u] + 1) {
                dist[v] = dist[u] + 1;
                q.push(v);
            }
        }
    }
    
    if (min_cycle == INT_MAX) {
        cout << -1 << endl;
    } else {
        cout << min_cycle << endl;
    }
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);
    
    int T = 1;
    // cin >> T;
    while (T--) {
        solve();
    }
    return 0;
}
```

### Explanation
1. **Graph Representation**: The graph is read from input and stored in an adjacency list `adj`.
2. **BFS Initialization**: A queue is initialized with node 1, and the distance array `dist` is filled with `INT_MAX` except for node 1, which is set to 0.
3. **BFS Execution**: For each node `u` dequeued, the algorithm checks all adjacent nodes `v`. If `v` is node 1, it updates the shortest cycle length. Otherwise, if a shorter path to `v` is found, the distance is updated, and `v` is enqueued.
4. **Result Output**: After BFS completes, the shortest cycle length is output if found; otherwise, `-1` is printed indicating no cycle back to node 1 exists.

This approach ensures that the shortest cycle is found efficiently using BFS, which is optimal for unweighted graphs. The complexity is O(N + M), where N is the number of nodes and M is the number of edges, making it suitable for large graphs.
