To fix this code, we need to ensure that we correctly find the shortest cycle starting and ending at node 1. The initial approach used a depth-first search (DFS) with a global minimum to track the shortest cycle, but it failed to correctly handle cycles and revisit nodes. 

### Approaches
1. **Breadth-First Search (BFS)** is more suitable for finding the shortest path in an unweighted graph. We can modify BFS to find the shortest cycle starting and ending at node 1.
2. **Algorithm**:
   - Start BFS from node 1, keeping track of the distance from node 1 to each other node.
   - For each neighbor of node 1, if it has a path back to node 1, the cycle length is `distance[neighbor] + 1`.
   - The minimum of all such cycle lengths gives the shortest cycle.

### Solution Code
```cpp
#include <iostream>
#include <vector>
#include <queue>
#include <climits>
using namespace std;

const int N = 2e5 + 10;
vector<int> adj[N];
int dist[N];

void solve() {
    int n, m;
    cin >> n >> m;
    for (int i = 0; i < m; ++i) {
        int u, v;
        cin >> u >> v;
        adj[u].push_back(v);
    }

    int min_cycle = INT_MAX;
    queue<int> q;
    q.push(1);
    fill(dist, dist + n + 1, -1);
    dist[1] = 0;

    while (!q.empty()) {
        int u = q.front();
        q.pop();
        for (int v : adj[u]) {
            if (v == 1) {
                min_cycle = min(min_cycle, dist[u] + 1);
            }
            if (dist[v] == -1) {
                dist[v] = dist[u] + 1;
                q.push(v);
            }
        }
    }

    if (min_cycle == INT_MAX) {
        cout << -1 << endl;
    } else {
        cout << min_cycle << endl;
    }
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int T = 1;
    // cin >> T;
    while (T--) {
        solve();
    }
    return 0;
}
```

### Explanation
1. **Initialization**: The adjacency list `adj` is used to store the graph, and the `dist` array keeps track of the shortest distance from node 1 to each node.
2. **BFS Setup**: Start BFS from node 1, initializing its distance to 0 and marking other nodes as unvisited (`-1`).
3. **Cycle Detection**: For each node `u` processed in BFS, check its neighbors:
   - If a neighbor is node 1, update the minimum cycle length if the current path through `u` provides a shorter cycle.
   - For other neighbors, if they are unvisited, set their distance and enqueue them.
4. **Result**: After BFS completes, if no cycle was found (i.e., `min_cycle` remains `INT_MAX`), output `-1`; otherwise, output the shortest cycle length.

This approach ensures that we efficiently find the shortest cycle involving node 1 using BFS, which is optimal for unweighted graphs.
