To fix this code, we need to address the issue in the algorithm that computes the shortest cycle starting and ending at node 1. The original code uses a depth-first search (DFS) approach, which is not suitable for finding the shortest cycle in a directed graph efficiently. Instead, we should use a breadth-first search (BFS) approach, which is more appropriate for finding shortest paths in unweighted graphs.

### Approaches
1. **Problem Analysis**: The problem requires finding the shortest cycle that starts and ends at node 1 in a directed graph. If no such cycle exists, the output should be -1.
2. **Intuition**: BFS is suitable because it explores all nodes level by level, ensuring that the first time we revisit node 1, we have found the shortest cycle.
3. **Algorithm**: 
   - Use BFS to traverse the graph from node 1, keeping track of the distance from node 1 to each node.
   - For each neighbor of the current node, if the neighbor is node 1, update the shortest cycle length if the current path length plus 1 is smaller.
   - If the neighbor hasn't been visited, mark it as visited and enqueue it with an incremented distance.
4. **Complexity**: The BFS runs in O(N + M) time, where N is the number of nodes and M is the number of edges, which is efficient for the given constraints.

### Solution Code
```cpp
#include<iostream>
#include<vector>
#include<queue>
#include<climits>
using namespace std;

const int INF = INT_MAX;

void solve() {
    int n, m;
    cin >> n >> m;
    vector<vector<int>> adj(n + 1);
    for (int i = 0; i < m; ++i) {
        int u, v;
        cin >> u >> v;
        adj[u].push_back(v);
    }

    int min_cycle = INF;
    queue<pair<int, int>> q; // node, distance from node 1
    vector<int> dist(n + 1, -1);
    q.push({1, 0});
    dist[1] = 0;

    while (!q.empty()) {
        auto [u, d] = q.front();
        q.pop();
        for (int v : adj[u]) {
            if (v == 1) {
                if (d + 1 < min_cycle) {
                    min_cycle = d + 1;
                }
            }
            if (dist[v] == -1) {
                dist[v] = d + 1;
                q.push({v, d + 1});
            }
        }
    }

    if (min_cycle == INF) {
        cout << -1 << endl;
    } else {
        cout << min_cycle << endl;
    }
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);
    solve();
    return 0;
}
```

### Explanation
1. **Input Handling**: The code reads the number of nodes (n) and edges (m), then constructs the adjacency list for the graph.
2. **BFS Initialization**: A queue is initialized to start BFS from node 1, with the initial distance set to 0.
3. **BFS Execution**: For each node dequeued, the code checks all its neighbors. If a neighbor is node 1, the current path length plus 1 is considered as a potential cycle length and the minimum is updated. For other nodes, if they haven't been visited, they are enqueued with an incremented distance.
4. **Result Output**: After BFS completes, if no cycle is found (min_cycle remains INF), it outputs -1; otherwise, it outputs the shortest cycle length found.

This approach efficiently finds the shortest cycle using BFS, ensuring optimal performance and correctness for all test cases.
