After analyzing the code, I found several issues that need to be fixed:

1. The DFS approach is incorrect for finding the shortest cycle containing node 1 because DFS doesn't naturally find shortest paths. BFS would be more appropriate.
2. The global variable `mi` is initialized but not reset for multiple test cases.
3. The visited array `g` is not properly managed - it's used both for cycle detection and visited tracking in a confusing way.
4. The cycle detection logic is flawed - it only checks if it can return to node 1, but doesn't properly track visited nodes.

Here's the fixed code using BFS for finding the shortest cycle containing node 1:

```cpp
#include<iostream>
#include<vector>
#include<queue>
#include<climits>
using namespace std;

const int N = 2e5 + 10;
const int INF = INT_MAX;

vector<int> adj[N];

void solve() {
    int n, m;
    cin >> n >> m;
    
    for (int i = 0; i < m; i++) {
        int u, v;
        cin >> u >> v;
        adj[u].push_back(v);
    }
    
    int min_cycle = INF;
    queue<pair<int, int>> q;
    vector<int> dist(n + 1, -1);
    
    q.push({1, 0});
    dist[1] = 0;
    
    while (!q.empty()) {
        auto [u, d] = q.front();
        q.pop();
        
        for (int v : adj[u]) {
            if (v == 1) {
                min_cycle = min(min_cycle, d + 1);
                continue;
            }
            if (dist[v] == -1) {
                dist[v] = d + 1;
                q.push({v, d + 1});
            }
        }
    }
    
    if (min_cycle == INF) {
        cout << -1 << endl;
    } else {
        cout << min_cycle << endl;
    }
    
    // Clean up for next test case
    for (int i = 1; i <= n; i++) {
        adj[i].clear();
    }
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);
    
    int T = 1;
    // cin >> T;
    while (T--) {
        solve();
    }
    
    return 0;
}
```

Key improvements:
1. Uses BFS instead of DFS to find the shortest cycle
2. Properly manages distance tracking with a distance array
3. Clears adjacency lists between test cases
4. Simplified code structure without unnecessary macros
5. Properly handles the case when no cycle exists
6. More efficient as BFS naturally finds shortest paths in unweighted graphs

The algorithm now correctly finds the shortest cycle containing node 1 by exploring all paths from node 1 and checking when we can return to it. The BFS ensures we find the shortest such cycle.
