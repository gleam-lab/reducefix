To fix this code, we need to address the issue of finding the shortest cycle that includes node 1. The original code uses a depth-first search (DFS) approach, which is not suitable for finding the shortest path in an unweighted graph. Instead, we should use a breadth-first search (BFS) approach, which is more efficient for finding the shortest path in such scenarios.

### Approaches
1. **Graph Representation**: Represent the graph using an adjacency list.
2. **BFS for Shortest Path**: Use BFS to explore the graph starting from node 1. BFS naturally explores nodes level by level, ensuring that the first time we encounter node 1 again, we have found the shortest cycle.
3. **Tracking Visited Nodes**: Maintain a visited array to keep track of the nodes we have already visited to avoid revisiting them unnecessarily.
4. **Parent Tracking**: Track the parent of each node to ensure we do not consider the immediate back edge (from a node to its parent) as a cycle.

### Solution Code
```cpp
#include <iostream>
#include <vector>
#include <queue>
#include <climits>
using namespace std;

const int N = 2e5 + 10;
const int INF = INT_MAX;

vector<int> adj[N];
int dist[N];
int parent[N];

void bfs(int start, int n) {
    queue<int> q;
    for (int i = 1; i <= n; ++i) {
        dist[i] = -1;
        parent[i] = -1;
    }
    dist[start] = 0;
    q.push(start);

    while (!q.empty()) {
        int u = q.front();
        q.pop();

        for (int v : adj[u]) {
            if (dist[v] == -1) {
                dist[v] = dist[u] + 1;
                parent[v] = u;
                q.push(v);
            } else if (v == start && parent[u] != v) {
                // Found a cycle back to the start node
                cout << dist[u] + 1 << endl;
                return;
            }
        }
    }
    cout << -1 << endl;
}

void solve() {
    int n, m;
    cin >> n >> m;
    for (int i = 0; i < m; ++i) {
        int u, v;
        cin >> u >> v;
        adj[u].push_back(v);
    }
    bfs(1, n);
}

int main() {
    ios::sync_with_stdio(0);
    cin.tie(0);
    cout.tie(0);

    int T = 1;
    while (T--) {
        solve();
    }
    return 0;
}
```

### Explanation
1. **Graph Representation**: The graph is represented using an adjacency list `adj`, where each node points to its neighbors.
2. **BFS Initialization**: The BFS starts from node 1. The `dist` array keeps track of the shortest distance from node 1 to each node, and the `parent` array records the parent of each node in the BFS tree to avoid considering trivial cycles (immediate back edges).
3. **BFS Execution**: For each node dequeued, its neighbors are examined. If a neighbor hasn't been visited, it is enqueued with an updated distance. If the neighbor is the start node (node 1) and it's not the immediate parent, a cycle is detected, and the cycle length is printed.
4. **Cycle Detection**: The cycle length is determined by the distance from node 1 to the current node plus one (the edge back to node 1). If no cycle is found, `-1` is printed.

This approach ensures that the shortest cycle involving node 1 is found efficiently using BFS, which is optimal for unweighted graphs.
