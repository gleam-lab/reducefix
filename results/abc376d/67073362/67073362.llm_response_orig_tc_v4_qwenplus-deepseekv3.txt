Based on the error analysis and problem description, the issue appears to be with finding the shortest cycle starting and ending at node 1. The original code uses a DFS approach which may not correctly find the shortest cycle. We need to implement a BFS approach instead, as BFS is more suitable for finding shortest paths in unweighted graphs.

Here's the fixed code:

```cpp
#include<iostream>
#include<queue>
#include<vector>
#include<climits>
using namespace std;

const int N = 2e5 + 10;
vector<int> adj[N];
int dist[N];

void solve() {
    int n, m;
    cin >> n >> m;
    
    for (int i = 0; i < m; i++) {
        int u, v;
        cin >> u >> v;
        adj[u].push_back(v);
    }
    
    int min_cycle = INT_MAX;
    
    for (int start : adj[1]) {
        queue<int> q;
        fill(dist, dist + n + 1, -1);
        dist[start] = 1;
        q.push(start);
        
        while (!q.empty()) {
            int u = q.front();
            q.pop();
            
            for (int v : adj[u]) {
                if (v == 1) {
                    min_cycle = min(min_cycle, dist[u] + 1);
                    break;
                }
                if (dist[v] == -1) {
                    dist[v] = dist[u] + 1;
                    q.push(v);
                }
            }
        }
    }
    
    if (min_cycle == INT_MAX) {
        cout << -1 << endl;
    } else {
        cout << min_cycle << endl;
    }
}

int main() {
    ios::sync_with_stdio(0);
    cin.tie(0);
    
    int T = 1;
    while (T--) {
        solve();
    }
    return 0;
}
```

Key changes made:
1. Replaced DFS with BFS since BFS naturally finds shortest paths in unweighted graphs
2. For each neighbor of node 1, run BFS to find the shortest path back to node 1
3. Keep track of the minimum cycle length found
4. Used proper initialization and distance tracking
5. Simplified the code structure and removed unnecessary components

This approach ensures we correctly find the shortest cycle containing node 1 by exploring all possible paths from its neighbors back to itself using BFS, which guarantees the shortest path will be found first.

The time complexity is O(N + M) for each BFS, and we perform this for each neighbor of node 1, making the overall complexity O(K*(N + M)) where K is the number of neighbors of node 1. In the worst case, this is O(N*M), but in practice it's much better since most nodes won't be neighbors of node 1.
