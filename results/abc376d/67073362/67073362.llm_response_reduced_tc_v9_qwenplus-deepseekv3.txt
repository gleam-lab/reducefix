To fix this code, we need to address the issue where the current implementation incorrectly calculates the shortest cycle involving node 1. The problem requires finding the shortest cycle that starts and ends at node 1, but the current code does not correctly track visited nodes and their depths, leading to incorrect results.

### Approaches
1. **Graph Representation**: Use an adjacency list to represent the graph.
2. **BFS for Shortest Path**: Perform a BFS starting from node 1 to find the shortest cycle. BFS is suitable here because it explores all nodes at the present depth before moving on to nodes at the next depth level, ensuring the shortest path is found first.
3. **Tracking Visits and Depths**: Maintain a distance array to keep track of the shortest distance from node 1 to each node. If during BFS traversal, we encounter node 1 again, we check if the current path length plus one (to complete the cycle) is shorter than the previously found minimum.

### Solution Code
```cpp
#include<iostream>
#include<queue>
#include<vector>
#include<climits>
using namespace std;

const int N = 2e5 + 10;
vector<int> adj[N];
int dist[N];

void solve() {
    int n, m;
    cin >> n >> m;
    for (int i = 0; i < m; ++i) {
        int u, v;
        cin >> u >> v;
        adj[u].push_back(v);
    }
    
    int min_cycle = INT_MAX;
    queue<pair<int, int>> q; // {node, distance from node 1}
    q.push({1, 0});
    dist[1] = 0;
    
    while (!q.empty()) {
        auto [u, d] = q.front();
        q.pop();
        
        for (int v : adj[u]) {
            if (v == 1) {
                min_cycle = min(min_cycle, d + 1);
            }
            if (dist[v] == -1) {
                dist[v] = d + 1;
                q.push({v, d + 1});
            }
        }
    }
    
    if (min_cycle == INT_MAX) {
        cout << -1 << endl;
    } else {
        cout << min_cycle << endl;
    }
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);
    
    int T = 1;
    // cin >> T;
    while (T--) {
        fill(dist, dist + N, -1);
        solve();
    }
    return 0;
}
```

### Explanation
1. **Graph Representation**: The graph is represented using an adjacency list `adj` where each node points to its neighbors.
2. **BFS Initialization**: The BFS starts from node 1 with an initial distance of 0. The distance array `dist` is initialized to -1 to indicate unvisited nodes, except for node 1 which is set to 0.
3. **BFS Execution**: For each node `u` dequeued, all its neighbors `v` are checked. If a neighbor `v` is node 1, the current path length plus one gives a cycle length, which is compared to the current minimum cycle length. If `v` is unvisited, it is enqueued with an incremented distance.
4. **Result Output**: After BFS completes, if a cycle was found (i.e., `min_cycle` is not `INT_MAX`), the shortest cycle length is printed; otherwise, -1 is printed indicating no cycle exists involving node 1.

This approach ensures that we efficiently find the shortest cycle involving node 1 using BFS, which guarantees the shortest path in an unweighted graph. The solution handles all edge cases and large input sizes within feasible time complexity.
