To fix this code, we need to address the incorrect output produced when finding the shortest cycle starting and ending at node 1. The original code uses a depth-first search (DFS) approach, which is not suitable for finding the shortest path in an unweighted graph. Instead, we should use a breadth-first search (BFS) approach because BFS naturally explores nodes level by level, ensuring the shortest path is found first.

### Approaches
1. **Graph Representation**: Represent the graph using an adjacency list.
2. **BFS for Shortest Cycle**: Use BFS to explore all possible paths from node 1 back to node 1. By keeping track of the visited nodes and the current depth, we can find the shortest cycle.
3. **Cycle Detection**: During BFS, if node 1 is revisited, update the minimum cycle length if the current path length is shorter than the previously found minimum.

### Solution Code
```cpp
#include<iostream>
#include<queue>
#include<vector>
#include<climits>
using namespace std;

const int N = 2e5 + 10;
vector<int> adj[N];
int min_cycle = INT_MAX;

void bfs(int start, int n) {
    vector<int> dist(n + 1, -1);
    queue<int> q;
    q.push(start);
    dist[start] = 0;

    while (!q.empty()) {
        int u = q.front();
        q.pop();

        for (int v : adj[u]) {
            if (v == start) {
                min_cycle = min(min_cycle, dist[u] + 1);
                continue;
            }
            if (dist[v] == -1) {
                dist[v] = dist[u] + 1;
                q.push(v);
            }
        }
    }
}

void solve() {
    int n, m;
    cin >> n >> m;
    for (int i = 0; i < m; ++i) {
        int u, v;
        cin >> u >> v;
        adj[u].push_back(v);
    }
    bfs(1, n);
    if (min_cycle == INT_MAX) {
        cout << -1 << endl;
    } else {
        cout << min_cycle << endl;
    }
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);
    solve();
    return 0;
}
```

### Explanation
1. **Graph Representation**: The graph is stored in an adjacency list `adj` where each node points to its neighbors.
2. **BFS Initialization**: The BFS starts from node 1 with an initial distance of 0. A queue is used to manage the nodes to be visited next.
3. **Distance Tracking**: A `dist` array keeps track of the shortest distance from node 1 to each node. Nodes are marked as visited by setting their distance.
4. **Cycle Detection**: During BFS, if we encounter node 1 again (indicating a cycle), we update `min_cycle` with the current path length if it's shorter than the existing value.
5. **Result Output**: After BFS completes, if no cycle was found (`min_cycle` remains `INT_MAX`), output -1. Otherwise, output the shortest cycle length.

This approach ensures that the shortest cycle is found efficiently using BFS, which is optimal for unweighted graphs. The time complexity is O(N + M), where N is the number of nodes and M is the number of edges, making it suitable for large graphs.
