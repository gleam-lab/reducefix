To fix this code, we need to address the problem of finding the shortest cycle that starts and ends at node 1 in a directed graph. The initial approach used a depth-first search (DFS) to track the depth and detect cycles involving node 1, but it failed to correctly identify the shortest cycle in some cases, as seen in the failing test case.

### Approaches
1. **Breadth-First Search (BFS)**:
   - BFS is more suitable for finding the shortest path in unweighted graphs. We can use BFS to explore the graph level by level from node 1, keeping track of the distance from node 1 to each node.
   - For each neighbor of the current node, if the neighbor is node 1 itself and the current distance plus 1 forms a cycle, we update the shortest cycle length if this new cycle is shorter.
   - The BFS ensures that the first time we encounter node 1 again gives us the shortest cycle length.

### Solution Code
```cpp
#include <iostream>
#include <vector>
#include <queue>
#include <climits>
using namespace std;

const int N = 2e5 + 10;
const int INF = INT_MAX;

vector<int> adj[N];
int dist[N];

void solve() {
    int n, m;
    cin >> n >> m;
    for (int i = 0; i < m; ++i) {
        int u, v;
        cin >> u >> v;
        adj[u].push_back(v);
    }
    
    fill(dist, dist + n + 1, INF);
    queue<int> q;
    q.push(1);
    dist[1] = 0;
    int min_cycle = INF;
    
    while (!q.empty()) {
        int u = q.front();
        q.pop();
        
        for (int v : adj[u]) {
            if (v == 1) {
                min_cycle = min(min_cycle, dist[u] + 1);
            }
            if (dist[v] == INF) {
                dist[v] = dist[u] + 1;
                q.push(v);
            }
        }
    }
    
    if (min_cycle == INF) {
        cout << -1 << endl;
    } else {
        cout << min_cycle << endl;
    }
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);
    
    solve();
    
    return 0;
}
```

### Explanation
1. **Initialization**:
   - The adjacency list `adj` stores the directed edges of the graph.
   - The `dist` array keeps track of the shortest distance from node 1 to each node, initialized to infinity (`INF`).

2. **BFS Setup**:
   - Start BFS from node 1, setting `dist[1]` to 0.
   - Use a queue to manage nodes level by level.

3. **Cycle Detection**:
   - For each node `u` dequeued, examine all its neighbors `v`.
   - If `v` is node 1, a cycle is detected. The cycle length is `dist[u] + 1`, and we update `min_cycle` if this length is shorter than the current value.
   - For other nodes, if they haven't been visited yet (`dist[v] == INF`), update their distance and enqueue them.

4. **Result Output**:
   - If no cycle involving node 1 is found (`min_cycle` remains `INF`), output `-1`.
   - Otherwise, output the shortest cycle length found.

This approach efficiently finds the shortest cycle starting and ending at node 1 using BFS, ensuring correctness and optimal performance for the given problem constraints.
