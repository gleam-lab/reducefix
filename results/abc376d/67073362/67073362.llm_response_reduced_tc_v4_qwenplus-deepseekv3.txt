To fix this code, we need to address the issue in finding the shortest cycle that includes node 1. The original code attempts to use a DFS approach, which is not suitable for finding the shortest cycle because DFS explores as far as possible along each branch before backtracking, which means it doesn't necessarily find the shortest path.

Instead, we should use a BFS approach to find the shortest cycle involving node 1. BFS is more appropriate for shortest path problems in unweighted graphs because it explores all nodes at the present depth level before moving on to nodes at the next depth level.

### Approaches
1. **Graph Representation**: Represent the graph using adjacency lists.
2. **BFS for Shortest Cycle**: For each node connected to node 1, perform a BFS to find the shortest path back to node 1. The sum of the path from node 1 to the neighbor and back is the length of the cycle.
3. **Check for Cycles**: Ensure that the cycle exists by checking if a path back to node 1 is found during BFS.

### Solution Code
```cpp
#include <iostream>
#include <vector>
#include <queue>
#include <climits>
using namespace std;

const int INF = INT_MAX;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n, m;
    cin >> n >> m;
    vector<vector<int>> adj(n + 1);
    for (int i = 0; i < m; ++i) {
        int u, v;
        cin >> u >> v;
        adj[u].push_back(v);
    }

    int min_cycle = INF;
    for (int neighbor : adj[1]) {
        queue<int> q;
        vector<int> dist(n + 1, INF);
        dist[neighbor] = 1;
        q.push(neighbor);

        while (!q.empty()) {
            int u = q.front();
            q.pop();
            for (int v : adj[u]) {
                if (v == 1) {
                    min_cycle = min(min_cycle, dist[u] + 1);
                    break;
                }
                if (dist[v] == INF) {
                    dist[v] = dist[u] + 1;
                    q.push(v);
                }
            }
        }
    }

    cout << (min_cycle == INF ? -1 : min_cycle) << endl;
    return 0;
}
```

### Explanation
1. **Graph Representation**: The graph is stored as an adjacency list `adj`, where each node points to its neighbors.
2. **BFS Initialization**: For each neighbor of node 1, we initialize a BFS to explore paths starting from that neighbor.
3. **Distance Tracking**: We maintain a distance array `dist` to keep track of the shortest path from the neighbor to other nodes. The distance from the neighbor to itself is set to 1.
4. **Cycle Detection**: During BFS, if we encounter node 1, we update `min_cycle` with the shortest path length found so far.
5. **Result Output**: After processing all neighbors, if no cycle is found (`min_cycle` remains `INF`), we output `-1`. Otherwise, we output the shortest cycle length.

This approach ensures that we efficiently find the shortest cycle involving node 1 using BFS, which is optimal for shortest path problems in unweighted graphs.
