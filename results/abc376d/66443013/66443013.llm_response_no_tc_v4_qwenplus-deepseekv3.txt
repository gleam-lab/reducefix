1. The main problem is finding the shortest cycle containing node 1 in a directed graph.
2. Your current DFS approach doesn't properly track visited nodes during the search, leading to incorrect cycle detection.
3. The `seen` map is never reset, which prevents proper backtracking.
4. The logic for detecting cycles is flawed - it only checks if we return to node 1, but doesn't properly handle other cases.

Here's the corrected version using BFS, which is more suitable for finding shortest paths:

```cpp
#include<bits/stdc++.h>
using namespace std;
using ll=long long;

int main() {
    ll n, m;
    cin >> n >> m;
    vector<vector<ll>> graph(n+1);
    for(int i=0; i<m; i++) {
        ll a, b;
        cin >> a >> b;
        graph[a].push_back(b);
    }
    
    ll ans = -1;
    queue<pair<ll, ll>> q; // {current node, distance}
    vector<ll> dist(n+1, -1);
    
    // Start BFS from node 1's neighbors
    for(auto neighbor : graph[1]) {
        q.push({neighbor, 1});
        dist[neighbor] = 1;
    }
    
    while(!q.empty()) {
        auto [current, d] = q.front();
        q.pop();
        
        for(auto next : graph[current]) {
            if(next == 1) {
                if(ans == -1 || d+1 < ans) {
                    ans = d+1;
                }
                continue;
            }
            if(dist[next] == -1 || d+1 < dist[next]) {
                dist[next] = d+1;
                q.push({next, d+1});
            }
        }
    }
    
    cout << ans << endl;
    return 0;
}
```

Key improvements:
1. Uses BFS instead of DFS for finding shortest paths
2. Properly tracks distances to each node
3. Specifically checks for returning to node 1 to detect cycles
4. More efficient and correct cycle detection
5. Better memory usage with vectors instead of maps
6. Handles all edge cases properly

The algorithm now:
1. Starts BFS from all neighbors of node 1
2. For each node visited, records the distance from node 1
3. If we find a path back to node 1, updates the shortest cycle length
4. Continues until all possible paths are explored
5. Outputs the shortest cycle length or -1 if no cycle exists
