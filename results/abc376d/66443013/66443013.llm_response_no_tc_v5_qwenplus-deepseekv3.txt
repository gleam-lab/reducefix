The original code attempts to find the shortest cycle starting and ending at node 1 in a directed graph using DFS. However, it has several issues:

2. The condition `if(x!=1&&seen[x])continue` prevents proper exploration of all paths.
3. The base case for node 1 is not correctly handled for all possible cycles.
4. The code doesn't properly handle cases where multiple edges exist between nodes.

Here's the corrected version that uses BFS (more suitable for shortest path problems) with proper tracking of visited nodes:

```cpp
#include<bits/stdc++.h>
using namespace std;
using ll=long long;

int main(){
    ll n, m;
    cin >> n >> m;
    map<ll, vector<ll>> graph;
    
    for(int i=0; i<m; i++){
        ll a, b;
        cin >> a >> b;
        graph[a].push_back(b);
    }

    ll ans = -1;
    queue<pair<ll, ll>> q; // {current node, distance}
    vector<ll> dist(n+1, -1); // distance from node 1
    
    // Start BFS from node 1
    q.push({1, 0});
    dist[1] = 0;
    
    while(!q.empty()){
        auto [current, d] = q.front();
        q.pop();
        
        for(auto neighbor : graph[current]){
            if(neighbor == 1){
                // Found a cycle back to node 1
                if(ans == -1 || d+1 < ans){
                    ans = d+1;
                }
                continue;
            }
            if(dist[neighbor] == -1 || d+1 < dist[neighbor]){
                dist[neighbor] = d+1;
                q.push({neighbor, d+1});
            }
        }
    }
    
    cout << ans << endl;
    return 0;
}
```

Key improvements:
1. Uses BFS which is more appropriate for shortest path problems
2. Properly tracks distances from node 1
3. Correctly handles cycles back to node 1
4. More efficient with O(n + m) time complexity
5. Handles cases where no cycle exists by returning -1
6. Uses proper data structures (vector instead of map for distance tracking)

The code now correctly finds the shortest cycle starting and ending at node 1, or returns -1 if no such cycle exists.
