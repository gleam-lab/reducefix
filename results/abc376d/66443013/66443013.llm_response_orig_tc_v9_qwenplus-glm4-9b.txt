
The problem involves finding the minimum number of additional edges required to make all nodes in a graph have degree exactly 1, starting from a given node. #include<bits/stdc++.h>
using namespace std;
using ll=long long;
map<ll,vector<ll>> graph;
map<ll,ll> degree;
const ll inf=1LL<<60;
ll ans=inf;

void dfs(ll n){
    // Mark the node as visited
    seen[n]=true;

    for(auto x:graph[n]){
        if(seen[x]) continue; // Skip if already visited
        if(x == 1) {
            ans = min(ans,degree[n]+1);
        } else {
            degree[x] += degree[n]; // Update the degree of the child node
            dfs(x);
        }
    }
}

int main(){
    ll n,m;
    cin>>n>>m;
    for(int i=0; i<m; i++){
        ll a,b;
        cin>>a>>b;
        graph[a].push_back(b);
        graph[b].push_back(a);
        degree[a]++;
        degree[b]++;
    }
    // Start DFS from node 1 (ensure it's present in the graph)
    if(graph.count(1)) dfs(1);
    // If no valid answer found, set to -1
    if(ans == inf) ans = -1;
    cout<<ans<<endl;
}

1. **Degree Map:** Instead of counting the edges using seen, I've introduced a degree map that keeps track of the degree (number of edges connected to a node) for each node. This will help in updating the degree of child nodes efficiently.

2. **DFS Function:** The DFS function now only recursively calls for nodes that haven't been visited. This prevents unnecessary recursive calls and revisits.

3. **Edge Cases Handling:** The code checks if node 1 (starting node) exists in the graph before performing DFS. This ensures that the DFS has a starting point and handles cases where the graph might not start from node 1.
