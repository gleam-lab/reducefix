To fix this code, we need to address the issue of finding the shortest cycle that starts and ends at node 1 in a directed graph. The original code attempts to use a depth-first search (DFS) approach, but it fails to correctly handle all cases, particularly when the shortest cycle involves more than one edge.

### Approaches
1. **Breadth-First Search (BFS)**: BFS is more suitable for finding the shortest path in unweighted graphs. We can use BFS to explore nodes level by level from node 1, keeping track of the distance from node 1 to each node. If we encounter node 1 again during the BFS traversal, we can determine the length of the cycle.
2. **Early Termination**: Since we are only interested in the shortest cycle, we can terminate the BFS as soon as we find a cycle involving node 1.

### Solution Code
```cpp
#include<bits/stdc++.h>
using namespace std;
using ll=long long;

int main() {
    ll n, m;
    cin >> n >> m;
    vector<vector<ll>> graph(n + 1);
    for (int i = 0; i < m; i++) {
        ll a, b;
        cin >> a >> b;
        graph[a].push_back(b);
    }

    ll ans = -1;
    queue<pair<ll, ll>> q; // {node, distance}
    vector<ll> dist(n + 1, -1);
    q.push({1, 0});
    dist[1] = 0;

    while (!q.empty()) {
        auto [current, d] = q.front();
        q.pop();

        for (auto neighbor : graph[current]) {
            if (neighbor == 1) {
                if (ans == -1 || d + 1 < ans) {
                    ans = d + 1;
                }
                continue;
            }
            if (dist[neighbor] == -1) {
                dist[neighbor] = d + 1;
                q.push({neighbor, d + 1});
            }
        }
    }

    cout << ans << endl;
    return 0;
}
```

### Explanation
1. **Input Handling**: The code reads the number of nodes (n) and edges (m), then constructs the adjacency list for the graph.
2. **BFS Initialization**: A queue is initialized to start BFS from node 1 with an initial distance of 0. The distances from node 1 to all other nodes are initialized to -1 (indicating unvisited nodes), with node 1's distance set to 0.
3. **BFS Execution**: For each node dequeued, the code checks its neighbors. If a neighbor is node 1, it updates the shortest cycle length if the current path length plus one is shorter than the previously found cycle length. For other neighbors, if they haven't been visited, their distance is updated, and they are enqueued.
4. **Output**: After BFS completes, the shortest cycle length is printed. If no cycle is found, -1 is printed.

This approach ensures that we efficiently find the shortest cycle involving node 1 using BFS, which is optimal for unweighted graphs. The complexity is O(N + M), where N is the number of nodes and M is the number of edges.
