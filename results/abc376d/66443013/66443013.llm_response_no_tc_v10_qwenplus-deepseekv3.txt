I analyzed the code and found several issues that need to be fixed:

1. The main problem is finding the shortest cycle that starts and ends at node 1. The current approach using DFS doesn't properly track visited nodes and can get stuck in infinite loops or miss valid cycles.

2. The `seen` map isn't properly reset when backtracking, which is necessary to explore all possible paths.

3. The condition `if(x!=1&&seen[x])continue` is too restrictive - it prevents revisiting nodes even when it might form a valid cycle.

Here's the corrected version using BFS (Breadth-First Search) which is more suitable for finding shortest paths:

```cpp
#include<bits/stdc++.h>
using namespace std;
using ll=long long;

int main() {
    ll n, m;
    cin >> n >> m;
    map<ll, vector<ll>> graph;
    
    for(int i = 0; i < m; i++) {
        ll a, b;
        cin >> a >> b;
        graph[a].push_back(b);
    }
    
    ll ans = -1;
    queue<pair<ll, ll>> q; // {current node, distance}
    q.push({1, 0});
    
    while(!q.empty()) {
        auto [current, dist] = q.front();
        q.pop();
        
        for(auto neighbor : graph[current]) {
            if(neighbor == 1) {
                if(dist + 1 >= 2) { // Need at least 2 edges to form a cycle
                    if(ans == -1 || dist + 1 < ans) {
                        ans = dist + 1;
                    }
                }
                continue;
            }
            if(dist + 1 < n) { // Prevent infinite loops, longest possible simple path is n-1
                q.push({neighbor, dist + 1});
            }
        }
    }
    
    cout << ans << endl;
    return 0;
}
```

Key improvements:
1. Uses BFS instead of DFS to naturally find the shortest path/cycle
2. Properly handles the cycle detection by checking when we return to node 1
3. Includes a check to ensure the cycle has at least length 2 (dist + 1 >= 2)
4. Prevents infinite loops by limiting the search depth (dist + 1 < n)
5. More efficient and correct overall approach

The fixed code will now correctly find the shortest cycle starting and ending at node 1, or return -1 if no such cycle exists.
