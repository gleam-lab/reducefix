After analyzing the code, I identified several issues:

1. The main problem is to find the shortest cycle containing node 1, but the current approach doesn't correctly handle cycles and can get stuck in infinite loops.
2. The seen/unseen tracking is incorrect - we shouldn't mark nodes as seen permanently since we need to explore all possible paths.
3. The cycle detection logic is flawed - it only checks if we return to node 1 without properly considering all possible cycles.
4. The graph is directed, which wasn't being properly accounted for in the cycle detection.

Here's the corrected code using BFS for finding the shortest cycle containing node 1:

```cpp
#include<bits/stdc++.h>
using namespace std;
using ll=long long;

int main() {
    ll n, m;
    cin >> n >> m;
    vector<vector<ll>> graph(n+1);
    for(int i=0; i<m; i++) {
        ll a, b;
        cin >> a >> b;
        graph[a].push_back(b);
    }

    ll min_cycle = LLONG_MAX;
    queue<pair<ll, ll>> q; // {current node, distance}
    vector<ll> dist(n+1, -1);
    
    // Start BFS from node 1's neighbors
    for(auto neighbor : graph[1]) {
        q.push({neighbor, 1});
        dist[neighbor] = 1;
    }

    while(!q.empty()) {
        auto [current, d] = q.front();
        q.pop();

        for(auto next : graph[current]) {
            if(next == 1) {
                min_cycle = min(min_cycle, d+1);
                continue;
            }
            if(dist[next] == -1 || d+1 < dist[next]) {
                dist[next] = d+1;
                q.push({next, d+1});
            }
        }
    }

    if(min_cycle != LLONG_MAX) {
        cout << min_cycle << endl;
    } else {
        cout << -1 << endl;
    }
    return 0;
}
```

Key improvements:
1. Uses BFS instead of DFS which is more suitable for shortest path problems
2. Properly tracks distances to nodes
3. Efficiently checks for cycles by looking for the shortest path back to node 1
4. Handles directed edges correctly
5. Uses proper initialization and termination conditions

The algorithm now correctly finds the shortest cycle containing node 1 by performing a BFS from its neighbors and checking for paths that return to node 1. The shortest such path plus 1 (for the edge back to node 1) gives the shortest cycle length.
