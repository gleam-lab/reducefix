Here's a corrected version of your code. The issue in your original code was that you were not resetting the `seen` map after each DFS call. This led to incorrect results when the DFS explored multiple paths. Additionally, I've added a check to ensure that we only consider cycles that start and end at vertex 1. Here is the corrected code:

```cpp
#include <bits/stdc++.h>
using namespace std;

vector<vector<int>> graph;
vector<bool> seen;

const int INF = 1e9;

void dfs(int node, int parent, int depth, int &min_edges) {
    if (node == 1 && depth > 0) { // Check if we're back at vertex 1
        min_edges = min(min_edges, depth);
        return;
    }
    
    seen[node] = true;
    
    for (int neighbor : graph[node]) {
        if (neighbor != parent && !seen[neighbor]) {
            dfs(neighbor, node, depth + 1, min_edges);
        }
    }
    
    seen[node] = false; // Reset seen status to allow other paths
}

int main() {
    int n, m;
    cin >> n >> m;
    graph.resize(n + 1);
    seen.resize(n + 1, false);
    
    for (int i = 0; i < m; ++i) {
        int a, b;
        cin >> a >> b;
        graph[a].push_back(b);
    }
    
    int min_edges = INF;
    dfs(1, -1, 0, min_edges); // Start DFS from vertex 1
    
    if (min_edges == INF) {
        cout << "-1" << endl;
    } else {
        cout << min_edges << endl;
    }
    
    return 0;
}
```

### Explanation:
1. **Graph Representation**: Using an adjacency list (`graph`) to store the directed edges.
2. **DFS Function**:
   - We pass `parent` to avoid revisiting the immediate predecessor.
   - We reset `seen[node]` to `false` after exploring all neighbors to allow other paths.
   - We update `min_edges` when we detect a cycle that starts and ends at vertex 1.
3. **Main Function**:
   - Initialize the graph and seen array.
   - Call DFS starting from vertex 1.
   - Print the result based on whether a valid cycle was found.

This approach ensures that we correctly identify the minimum length of cycles containing vertex 1 and handle the constraints efficiently.
