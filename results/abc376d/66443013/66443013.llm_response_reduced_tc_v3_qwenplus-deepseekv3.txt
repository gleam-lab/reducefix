2. **Queue Initialization**: Start BFS from node 1, keeping track of the current path length.
3. **Visited Tracking**: Use a visited array or map to keep track of visited nodes to avoid revisiting them unnecessarily, but ensure that we can revisit node 1 to detect cycles.

### Solution Code
```cpp
#include<bits/stdc++.h>
using namespace std;
using ll=long long;

int main(){
    ll n, m;
    cin >> n >> m;
    map<ll, vector<ll>> graph;
    for(int i = 0; i < m; ++i) {
        ll a, b;
        cin >> a >> b;
        graph[a].push_back(b);
    }
    
    ll ans = -1;
    queue<pair<ll, ll>> q; // {current node, distance}
    q.push({1, 0});
    map<ll, ll> dist;
    dist[1] = 0;
    
    while(!q.empty()) {
        auto [current, d] = q.front();
        q.pop();
        
        for(auto neighbor : graph[current]) {
            if(neighbor == 1) {
                if(ans == -1 || d + 1 < ans) {
                    ans = d + 1;
                }
                continue;
            }
            if(!dist.count(neighbor) || d + 1 < dist[neighbor]) {
                dist[neighbor] = d + 1;
                q.push({neighbor, d + 1});
            }
        }
    }
    
    cout << ans << endl;
    return 0;
}
```

### Explanation
1. **Graph Construction**: The graph is constructed using an adjacency list where each node points to its neighbors.
2. **BFS Initialization**: The BFS starts from node 1 with an initial distance of 0.
3. **Cycle Detection**: During BFS, whenever node 1 is encountered again, the current distance plus one (forming a cycle) is compared with the current shortest cycle length stored in `ans`.
4. **Distance Tracking**: The distances of nodes from node 1 are tracked to avoid revisiting nodes with longer paths, ensuring efficiency.
5. **Output the Result**: After BFS completes, the shortest cycle length is printed. If no cycle is found, `-1` is printed.

This approach ensures that we efficiently find the shortest cycle involving node 1 using BFS, which is optimal for unweighted graphs. The complexity is O(N + M), where N is the number of nodes and M is the number of edges, making it suitable for large graphs.
