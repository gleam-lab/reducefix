The original code attempts to find the shortest cycle that starts and ends at node 1 in a directed graph using DFS. However, it has several issues:
1. The `seen` map is not properly reset during backtracking, which prevents exploring all possible paths.
2. The condition `if(x!=1&&seen[x])continue;` incorrectly skips nodes that have been visited, even if they might lead to a shorter cycle.
3. The cycle length calculation is incorrect when the start node is revisited.

Here's the fixed code that uses BFS (more suitable for shortest path problems) to find the shortest cycle starting and ending at node 1:

```cpp
#include<bits/stdc++.h>
using namespace std;
using ll=long long;

int main() {
    ll n, m;
    cin >> n >> m;
    map<ll, vector<ll>> graph;
    for (int i = 0; i < m; i++) {
        ll a, b;
        cin >> a >> b;
        graph[a].push_back(b);
    }

    ll ans = -1;
    queue<pair<ll, ll>> q; // {current node, distance}
    q.push({1, 0});
    map<ll, ll> dist;
    dist[1] = 0;

    while (!q.empty()) {
        auto [node, d] = q.front();
        q.pop();

        for (auto neighbor : graph[node]) {
            if (neighbor == 1) {
                if (d + 1 >= 2) { // Cycle must have at least 2 edges
                    if (ans == -1 || d + 1 < ans) {
                        ans = d + 1;
                    }
                }
                continue;
            }
            if (!dist.count(neighbor) || d + 1 < dist[neighbor]) {
                dist[neighbor] = d + 1;
                q.push({neighbor, d + 1});
            }
        }
    }

    cout << ans << endl;
    return 0;
}
```

Key improvements:
1. Uses BFS which naturally finds shortest paths in unweighted graphs
2. Properly tracks distances to each node
3. Checks for cycles when returning to node 1 (ensuring cycle length ≥ 2)
4. More efficient and correct handling of visited nodes
5. Better structure and readability

The BFS approach is more suitable here because:
- It guarantees the first time we find a cycle back to node 1 will be the shortest one
- It avoids the complexities of backtracking needed in DFS
- It's generally more efficient for shortest path problems in unweighted graphs
