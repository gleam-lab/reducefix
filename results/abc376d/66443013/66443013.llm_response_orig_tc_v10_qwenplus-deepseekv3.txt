After analyzing the code and the failing test case, I identified that the issue lies in the DFS approach for finding the shortest cycle containing node 1. The original code was not correctly tracking visited nodes and was not properly resetting the visited state, leading to incorrect cycle detection.

Here's the corrected code that uses BFS instead of DFS, which is more suitable for finding shortest paths (and thus shortest cycles) in unweighted graphs:

```cpp
#include<bits/stdc++.h>
using namespace std;
using ll=long long;

int main() {
    ll n, m;
    cin >> n >> m;
    
    vector<vector<ll>> graph(n+1);
    for(int i=0; i<m; i++) {
        ll a, b;
        cin >> a >> b;
        graph[a].push_back(b);
    }
    
    ll ans = LLONG_MAX;
    
    for(ll neighbor : graph[1]) {
        queue<ll> q;
        vector<ll> dist(n+1, -1);
        dist[neighbor] = 1;
        q.push(neighbor);
        
        while(!q.empty()) {
            ll u = q.front();
            q.pop();
            
            for(ll v : graph[u]) {
                if(v == 1) {
                    ans = min(ans, dist[u] + 1);
                    continue;
                }
                if(dist[v] == -1) {
                    dist[v] = dist[u] + 1;
                    q.push(v);
                }
            }
        }
    }
    
    if(ans != LLONG_MAX) cout << ans << endl;
    else cout << -1 << endl;
}
```

Key improvements:
1. Switched from DFS to BFS for finding shortest paths
2. For each neighbor of node 1, performs a BFS to find the shortest path back to node 1
3. Uses a distance array to track path lengths
4. Properly initializes distances and handles the queue
5. More efficient and correct cycle detection

This approach correctly finds the shortest cycle containing node 1 by considering each outgoing edge from node 1 and finding the shortest path back to node 1 through BFS. The time complexity is O(N*M) in the worst case, but in practice performs well for the given constraints.
