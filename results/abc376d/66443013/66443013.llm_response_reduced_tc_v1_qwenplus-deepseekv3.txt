The problem involves finding the shortest cycle that starts and ends at node 1 in a directed graph. The original code uses a DFS approach but fails to correctly track the shortest cycle because it doesn't properly handle revisiting nodes and doesn't explore all possible paths optimally.

### Analysis of the Failing Case
In the given input, the shortest cycle starting and ending at node 1 is `1 -> 2 -> 4 -> 5 -> 1` with length 4. The original code incorrectly outputs 6, suggesting it's finding a longer cycle instead of the shortest one.

### Solution Approach
To find the shortest cycle:
1. **Use BFS** since it naturally explores nodes level by level, making it efficient for finding shortest paths.
2. **Track distances** from node 1 to all other nodes.
3. **For each neighbor of node 1**, check if there's a path back to node 1. The shortest cycle will be the sum of the path from node 1 to the neighbor plus the path back.

### Fixed Code
```cpp
#include<bits/stdc++.h>
using namespace std;
using ll=long long;

int main() {
    ll n, m;
    cin >> n >> m;
    vector<vector<ll>> graph(n + 1);
    for (ll i = 0; i < m; ++i) {
        ll a, b;
        cin >> a >> b;
        graph[a].push_back(b);
    }

    ll ans = -1;
    for (ll neighbor : graph[1]) {
        queue<ll> q;
        vector<ll> dist(n + 1, -1);
        dist[neighbor] = 1;
        q.push(neighbor);
        while (!q.empty()) {
            ll u = q.front();
            q.pop();
            for (ll v : graph[u]) {
                if (v == 1) {
                    ll cycle_length = dist[u] + 1;
                    if (ans == -1 || cycle_length < ans) {
                        ans = cycle_length;
                    }
                }
                if (dist[v] == -1) {
                    dist[v] = dist[u] + 1;
                    q.push(v);
                }
            }
        }
    }

    cout << ans << endl;
    return 0;
}
```

### Explanation
1. **Graph Construction**: The graph is built using adjacency lists from the input.
2. **BFS for Shortest Path**: For each neighbor of node 1, BFS is used to explore paths starting from that neighbor. The distance from the neighbor to each node is recorded.
3. **Cycle Detection**: If during BFS we encounter node 1 again, the cycle length is calculated as the distance to the current node plus 1 (to return to node 1). The shortest such cycle is tracked.
4. **Output**: The shortest cycle length is printed; if no cycle exists, `-1` is printed.

This approach ensures we efficiently find the shortest cycle by leveraging BFS's property of exploring nodes in order of increasing distance, guaranteeing optimality for shortest path problems.
