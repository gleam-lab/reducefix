To fix this code, we need to address the issue of finding the shortest cycle that starts and ends at node 1. The original code attempts to perform a depth-first search (DFS) from node 1, keeping track of the path length and updating the answer whenever it returns to node 1. However, the DFS approach is not optimal for finding the shortest cycle in an unweighted graph because it explores all possible paths before finding the shortest one. Instead, a breadth-first search (BFS) is more suitable for this purpose as it explores nodes level by level, ensuring that the first time we return to node 1, we have found the shortest cycle.

### Approaches
1. **BFS for Shortest Cycle Detection**: We use BFS to explore the graph from node 1, keeping track of the distance from node 1 to each node. If we encounter node 1 again during the BFS, the sum of the distances from node 1 to the current node and back to node 1 gives the length of the cycle.
2. **Handling Cycles**: The BFS ensures that the first time we find a cycle involving node 1, it is the shortest possible cycle. We start BFS from each neighbor of node 1 to check for cycles that include node 1 directly.

### Solution Code
```cpp
#include<bits/stdc++.h>
using namespace std;
using ll=long long;

int main() {
    ll n, m;
    cin >> n >> m;
    map<ll, vector<ll>> graph;
    for (int i = 0; i < m; i++) {
        ll a, b;
        cin >> a >> b;
        graph[a].push_back(b);
    }

    ll ans = LLONG_MAX;
    queue<pair<ll, ll>> q; // {current node, distance from start}

    for (auto neighbor : graph[1]) {
        q.push({neighbor, 1});
        map<ll, ll> dist;
        dist[neighbor] = 1;
        while (!q.empty()) {
            auto [current, d] = q.front();
            q.pop();
            for (auto next : graph[current]) {
                if (next == 1) {
                    ans = min(ans, d + 1);
                    continue;
                }
                if (dist.find(next) == dist.end()) {
                    dist[next] = d + 1;
                    q.push({next, d + 1});
                }
            }
        }
    }

    if (ans != LLONG_MAX) {
        cout << ans << endl;
    } else {
        cout << -1 << endl;
    }
    return 0;
}
```

### Explanation
1. **Graph Construction**: The graph is constructed using an adjacency list where each node points to its neighbors.
2. **BFS Initialization**: For each neighbor of node 1, we initialize a BFS queue. The queue stores pairs of nodes and their distances from the initial neighbor.
3. **BFS Execution**: While processing each node, if we encounter node 1 again, we update the answer with the current cycle length (distance + 1). This ensures we find the shortest cycle because BFS explores nodes in order of increasing distance.
4. **Result Handling**: After processing all neighbors, if a cycle was found (ans is updated), we print the shortest cycle length; otherwise, we print -1 indicating no cycle exists involving node 1.

This approach efficiently finds the shortest cycle by leveraging BFS properties, ensuring optimal performance and correctness.
