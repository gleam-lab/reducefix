To fix this code, we need to ensure that we correctly find the minimum cycle length in the directed graph starting and ending at node 1. The original code uses a depth-first search (DFS) approach but fails to correctly track the minimum cycle length because it doesn't properly reset the `seen` markers after exploring paths, leading to incorrect cycle detection.

### Approaches
1. **Graph Representation**: Use an adjacency list to represent the graph.
2. **BFS for Shortest Cycle Detection**: Since BFS naturally explores nodes level by level, it is suitable for finding the shortest path. We can modify BFS to detect the shortest cycle starting and ending at node 1.
3. **Tracking Path Lengths**: For each node, we keep track of the distance from node 1. If we encounter node 1 again during BFS with a distance greater than 1, we update the minimum cycle length.

### Solution Code
```cpp
#include<bits/stdc++.h>
using namespace std;
using ll=long long;
const ll inf=1LL<<60;

int main() {
    ll n, m;
    cin >> n >> m;
    vector<vector<ll>> graph(n+1);
    for (int i = 0; i < m; ++i) {
        ll a, b;
        cin >> a >> b;
        graph[a].push_back(b);
    }
    
    ll ans = inf;
    queue<pair<ll, ll>> q; // {node, distance}
    vector<ll> dist(n+1, inf);
    q.push({1, 0});
    dist[1] = 0;
    
    while (!q.empty()) {
        auto [current, d] = q.front();
        q.pop();
        
        for (auto neighbor : graph[current]) {
            if (neighbor == 1) {
                if (d + 1 > 1) {
                    ans = min(ans, d + 1);
                }
                continue;
            }
            if (dist[neighbor] > d + 1) {
                dist[neighbor] = d + 1;
                q.push({neighbor, d + 1});
            }
        }
    }
    
    if (ans != inf) {
        cout << ans << endl;
    } else {
        cout << "-1" << endl;
    }
    return 0;
}
```

### Explanation
1. **Graph Representation**: The graph is represented using an adjacency list where each node points to its neighbors.
2. **BFS Initialization**: We initialize a queue to perform BFS starting from node 1 with an initial distance of 0.
3. **Distance Tracking**: A distance array keeps track of the shortest distance from node 1 to each node. This helps avoid revisiting nodes unnecessarily and ensures we always take the shortest path.
4. **Cycle Detection**: During BFS, if we encounter node 1 again (indicating a cycle), we check if the distance is greater than 1 (to ensure it's a valid cycle) and update the minimum cycle length accordingly.
5. **Result Output**: After processing all nodes, if a cycle starting and ending at node 1 is found, the minimum cycle length is printed; otherwise, -1 is printed indicating no such cycle exists.

This approach ensures we efficiently find the shortest cycle using BFS, which is optimal for shortest path problems in unweighted graphs.
