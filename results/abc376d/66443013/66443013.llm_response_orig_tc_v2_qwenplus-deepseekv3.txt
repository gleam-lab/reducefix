Based on the error analysis, the initial code was finding a cycle length of 8 when the expected answer was 3. This suggests the depth-first search (DFS) approach wasn't correctly identifying the shortest cycle containing node 1. 

Here's the corrected solution using BFS (Breadth-First Search) which is more suitable for finding shortest paths in unweighted graphs:

```cpp
#include<bits/stdc++.h>
using namespace std;
using ll=long long;

const ll inf=1LL<<60;

int main() {
    ll n, m;
    cin >> n >> m;
    
    vector<vector<ll>> graph(n+1);
    for(ll i=0; i<m; i++) {
        ll a, b;
        cin >> a >> b;
        graph[a].push_back(b);
    }
    
    ll ans = inf;
    
    for(ll start : graph[1]) {
        queue<pair<ll, ll>> q;
        vector<ll> dist(n+1, -1);
        q.push({start, 1});
        dist[start] = 1;
        
        while(!q.empty()) {
            auto [node, d] = q.front();
            q.pop();
            
            if(node == 1) {
                ans = min(ans, d);
                break;
            }
            
            for(ll neighbor : graph[node]) {
                if(dist[neighbor] == -1) {
                    dist[neighbor] = d + 1;
                    q.push({neighbor, d + 1});
                }
            }
        }
    }
    
    if(ans != inf) cout << ans << endl;
    else cout << -1 << endl;
    
    return 0;
}
```

This solution:
1. Uses BFS instead of DFS to find the shortest cycle containing node 1
2. For each neighbor of node 1, performs BFS to find the shortest path back to node 1
3. Keeps track of the minimum cycle length found
4. Uses a distance array to avoid revisiting nodes and to track path lengths

The BFS approach is more appropriate here because:
- It naturally finds shortest paths in unweighted graphs
- It avoids the potential for infinite recursion that DFS might have in cyclic graphs
- The time complexity is O(N + M) per BFS, and since we do this for each neighbor of node 1, the overall complexity is O(K*(N + M)) where K is the number of neighbors of node 1

This should correctly identify the shortest cycle containing node 1 for all test cases, including the one where the expected output was 3.
