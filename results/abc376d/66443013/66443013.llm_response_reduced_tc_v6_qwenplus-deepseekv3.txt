To fix this code, we need to address the issue of finding the shortest cycle that starts and ends at node 1. The original code attempts to perform a DFS to find such a cycle, but it fails to correctly track the shortest path due to improper handling of visited nodes and cycle detection.

### Approaches
1. **Breadth-First Search (BFS)**: BFS is more suitable for finding the shortest path in an unweighted graph. We can use BFS to explore the graph level by level from node 1, keeping track of the distance from node 1 to each node.
2. **Cycle Detection**: For each neighbor of node 1, if we encounter node 1 during BFS, it indicates a cycle. The shortest cycle will be the smallest distance encountered plus one (for the edge back to node 1).
3. **Visited Tracking**: We need to ensure that we do not revisit nodes in a way that could lead to incorrect shortest path calculations. The BFS inherently ensures that the first time we visit a node is via the shortest path.

### Solution Code
```cpp
#include<bits/stdc++.h>
using namespace std;
using ll=long long;

int main() {
    ll n, m;
    cin >> n >> m;
    map<ll, vector<ll>> graph;
    for (int i = 0; i < m; i++) {
        ll a, b;
        cin >> a >> b;
        graph[a].push_back(b);
    }

    ll ans = -1;
    queue<pair<ll, ll>> q; // {node, distance}
    q.push({1, 0});
    map<ll, ll> dist;
    dist[1] = 0;

    while (!q.empty()) {
        auto [current, d] = q.front();
        q.pop();

        for (auto neighbor : graph[current]) {
            if (neighbor == 1) {
                if (d + 1 >= 2) { // Ensure it's a cycle with at least 2 edges
                    if (ans == -1 || d + 1 < ans) {
                        ans = d + 1;
                    }
                }
                continue;
            }
            if (!dist.count(neighbor) || d + 1 < dist[neighbor]) {
                dist[neighbor] = d + 1;
                q.push({neighbor, d + 1});
            }
        }
    }

    cout << ans << endl;
    return 0;
}
```

### Explanation
1. **Graph Construction**: The graph is built using an adjacency list where each node points to its neighbors.
2. **BFS Initialization**: The BFS starts from node 1 with an initial distance of 0. The queue stores pairs of nodes and their current distance from node 1.
3. **Cycle Detection**: For each neighbor of the current node, if the neighbor is node 1, it checks if the cycle length (distance + 1) is at least 2 and updates the shortest cycle length found.
4. **Distance Tracking**: The BFS ensures that each node is visited with the shortest distance from node 1. If a neighbor hasn't been visited or can be reached with a shorter path, it updates the distance and enqueues the neighbor.
5. **Output**: The shortest cycle length is printed. If no cycle is found, -1 is printed.

This approach efficiently finds the shortest cycle starting and ending at node 1 by leveraging BFS, ensuring optimal performance and correctness.
