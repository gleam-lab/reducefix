To fix this code, we need to address the issue that the original implementation was not correctly finding the shortest cycle that includes node 1. The problem essentially requires finding the shortest cycle that starts and ends at node 1, which can be rephrased as finding the shortest path from node 1 back to itself that passes through at least one other node.

### Approaches
1. **Breadth-First Search (BFS)**: We can use BFS to explore all nodes level by level from node 1. For each node encountered, we keep track of the distance from node 1. If we encounter node 1 again during the BFS, the sum of the current distance and 1 (for the edge back to node 1) gives the cycle length. The shortest such cycle is our answer.
2. **Handling Cycles**: The BFS approach ensures that the first time we revisit node 1, we have found the shortest possible cycle involving it because BFS explores nodes in order of increasing distance from the start node.

### Solution Code
```cpp
#include <bits/stdc++.h>
using namespace std;
using ll = long long;

int main() {
    ll n, m;
    cin >> n >> m;
    map<ll, vector<ll>> graph;
    for (int i = 0; i < m; ++i) {
        ll a, b;
        cin >> a >> b;
        graph[a].push_back(b);
    }

    ll ans = -1;
    queue<pair<ll, ll>> q; // {node, distance}
    q.push({1, 0});
    map<ll, ll> dist;
    dist[1] = 0;

    while (!q.empty()) {
        auto [current, d] = q.front();
        q.pop();

        for (auto neighbor : graph[current]) {
            if (neighbor == 1) {
                if (d + 1 >= 2) { // Ensure it's a cycle (not a self-loop)
                    if (ans == -1 || d + 1 < ans) {
                        ans = d + 1;
                    }
                }
                continue;
            }
            if (!dist.count(neighbor) || d + 1 < dist[neighbor]) {
                dist[neighbor] = d + 1;
                q.push({neighbor, d + 1});
            }
        }
    }

    cout << ans << endl;
    return 0;
}
```

### Explanation
1. **Graph Construction**: The graph is built using an adjacency list where each node points to its neighbors.
2. **BFS Initialization**: We start BFS from node 1 with an initial distance of 0.
3. **Cycle Detection**: During BFS, whenever we encounter node 1 again, we check if the path length (distance + 1) is at least 2 (to ensure it's a valid cycle) and update the shortest cycle length found.
4. **Distance Tracking**: For other nodes, we track the shortest distance from node 1. If a node is revisited with a shorter path, we update its distance and continue BFS.
5. **Output Result**: After BFS completes, the shortest cycle length involving node 1 is printed. If no such cycle exists, -1 is printed.

This approach efficiently finds the shortest cycle involving node 1 using BFS, ensuring optimal performance with a time complexity of O(N + M), where N is the number of nodes and M is the number of edges.
