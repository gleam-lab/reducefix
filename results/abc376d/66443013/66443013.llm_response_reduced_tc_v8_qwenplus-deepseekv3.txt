### Approaches

1. **Problem Analysis**: The task is to find the shortest cycle that starts and ends at node 1 in a directed graph. If no such cycle exists, the output should be -1. 

2. **Intuition**: The shortest cycle can be found by performing a BFS (Breadth-First Search) from node 1 to its neighbors and checking if we can return to node 1 in the shortest path possible. BFS is suitable here because it explores all nodes at the present depth level before moving on to nodes at the next depth level, which ensures the shortest path in an unweighted graph.

3. **Algorithm Selection**: 
   - **BFS for Shortest Path**: Using BFS, we can explore each node level by level. When we encounter node 1 during the BFS from node 1's neighbors, the current level (distance) plus one gives the length of the cycle.
   - **Early Termination**: The BFS can terminate early as soon as we find the first occurrence of node 1, which would represent the shortest cycle.

4. **Complexity Analysis**: 
   - **Time Complexity**: O(N + M), where N is the number of nodes and M is the number of edges. Each node and edge is processed once.
   - **Space Complexity**: O(N + M) for storing the graph and the BFS queue.

### Solution Code

```cpp
#include<bits/stdc++.h>
using namespace std;
using ll=long long;

int main() {
    ll n, m;
    cin >> n >> m;
    map<ll, vector<ll>> graph;
    for (int i = 0; i < m; i++) {
        ll a, b;
        cin >> a >> b;
        graph[a].push_back(b);
    }

    ll ans = -1;
    queue<pair<ll, ll>> q;
    q.push({1, 0});

    while (!q.empty()) {
        auto current = q.front();
        q.pop();
        ll node = current.first;
        ll dist = current.second;

        for (auto neighbor : graph[node]) {
            if (neighbor == 1) {
                if (dist + 1 < ans || ans == -1) {
                    ans = dist + 1;
                }
                continue;
            }
            if (dist + 1 >= ans && ans != -1) continue;
            q.push({neighbor, dist + 1});
        }
    }

    cout << ans << endl;
    return 0;
}
```

### Explanation

1. **Initialization**: The graph is built using an adjacency list where each node points to its neighbors. The BFS starts from node 1 with an initial distance of 0.

2. **BFS Execution**: For each node dequeued, its neighbors are checked. If a neighbor is node 1, the current distance plus one (forming a cycle) is compared with the current shortest cycle length (ans). If it's shorter, ans is updated.

3. **Early Termination**: The BFS skips processing further neighbors if the current path length (dist + 1) already exceeds the found shortest cycle length (ans), optimizing the search.

4. **Output**: After processing all possible nodes, the shortest cycle length (ans) is printed. If no cycle is found, ans remains -1, indicating no cycle exists starting and ending at node 1.

This approach ensures that we efficiently find the shortest cycle by leveraging BFS properties, providing optimal performance and correctness.
