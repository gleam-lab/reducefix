The problem requires finding the shortest cycle that starts and ends at node 1 in a directed graph. The original code attempts to do this with a DFS approach but fails to correctly identify the shortest cycle in some cases, as shown in the failing test case.

### Analysis of the Original Code
1. **DFS Approach**: The original code uses a DFS to traverse the graph from node 1, keeping track of the current path length (`hen`). When it encounters node 1 again, it updates the answer if the path length is greater than 1 (to avoid trivial cycles like 1 -> 1).
2. **Issue**: The DFS approach doesn't guarantee finding the shortest cycle first. It might explore longer paths before shorter ones, leading to incorrect results. Additionally, the `seen` map is not properly reset after exploring a path, which could prevent finding valid cycles.

### Correct Approach
To find the shortest cycle starting and ending at node 1 in a directed graph, a BFS approach is more suitable. BFS explores nodes level by level, ensuring that the first time we return to node 1, it's via the shortest path. Here's the plan:
1. **BFS Initialization**: Start BFS from node 1, keeping track of the distance from node 1 to each node.
2. **Cycle Detection**: For each neighbor of node 1, if during BFS we encounter node 1 again, update the shortest cycle length if the current path length is minimal.
3. **Early Termination**: Since BFS explores nodes in order of increasing distance, the first cycle found will be the shortest.

### Solution Code
```cpp
#include<bits/stdc++.h>
using namespace std;
using ll=long long;

int main() {
    ll n, m;
    cin >> n >> m;
    map<ll, vector<ll>> graph;
    for (int i = 0; i < m; i++) {
        ll a, b;
        cin >> a >> b;
        graph[a].push_back(b);
    }

    ll ans = -1;
    queue<pair<ll, ll>> q; // {current node, distance from node 1}
    q.push({1, 0});
    map<ll, ll> dist; // distance from node 1
    dist[1] = 0;

    while (!q.empty()) {
        auto [current, d] = q.front();
        q.pop();
        for (auto neighbor : graph[current]) {
            if (neighbor == 1) {
                if (d + 1 >= 1) {
                    if (ans == -1 || d + 1 < ans) {
                        ans = d + 1;
                    }
                }
                continue;
            }
            if (dist.find(neighbor) == dist.end()) {
                dist[neighbor] = d + 1;
                q.push({neighbor, d + 1});
            }
        }
    }

    cout << ans << endl;
    return 0;
}
```

### Explanation
1. **Graph Construction**: The graph is built using an adjacency list where each node points to its neighbors.
2. **BFS Initialization**: The BFS starts from node 1 with an initial distance of 0.
3. **Cycle Detection**: During BFS, if we encounter node 1 as a neighbor of the current node, we check if the path length (current distance + 1) forms a valid cycle (length >= 1) and update the answer if it's shorter than the current best.
4. **Distance Tracking**: The `dist` map keeps track of the shortest distance from node 1 to each node, ensuring we only process each node once and in the shortest path order.
5. **Output**: The shortest cycle length is printed, or -1 if no cycle exists.

This approach ensures that we efficiently find the shortest cycle starting and ending at node 1 using BFS, which is optimal for shortest path problems in unweighted graphs.
