### Approaches
1. **Breadth-First Search (BFS):** We can use BFS to explore the graph starting from node 1. BFS is suitable here because it explores nodes level by level, which means the first time we encounter node 1 again, we would have found the shortest cycle.
2. **Tracking Path Length:** During BFS, we keep track of the path length from the starting node (node 1) to each node. If we encounter node 1 again during the BFS, we record the length of the cycle (current path length + 1).
3. **Early Termination:** Once we find the first cycle involving node 1, we can terminate early because BFS guarantees that this is the shortest cycle.

### Solution Code
```cpp
#include <bits/stdc++.h>
using namespace std;
using ll = long long;

int main() {
    ll n, m;
    cin >> n >> m;
    map<ll, vector<ll>> graph;
    for (int i = 0; i < m; ++i) {
        ll a, b;
        cin >> a >> b;
        graph[a].push_back(b);
    }

    ll ans = -1;
    queue<pair<ll, ll>> q; // {node, distance}
    q.push({1, 0});
    map<ll, ll> dist;
    dist[1] = 0;

    while (!q.empty()) {
        auto current = q.front();
        q.pop();
        ll u = current.first;
        ll d = current.second;

        for (auto v : graph[u]) {
            if (v == 1) {
                if (d + 1 >= 1) {
                    if (ans == -1 || d + 1 < ans) {
                        ans = d + 1;
                    }
                }
                continue;
            }
            if (!dist.count(v) || d + 1 < dist[v]) {
                dist[v] = d + 1;
                q.push({v, d + 1});
            }
        }
    }

    cout << ans << endl;
    return 0;
}
```

### Explanation
1. **Graph Construction:** The graph is built using an adjacency list where each node points to its neighbors.
2. **BFS Initialization:** We initialize the BFS queue with node 1 and a distance of 0.
3. **BFS Execution:** For each node dequeued, we examine its neighbors. If a neighbor is node 1, we update the answer with the current distance + 1 (the cycle length). For other nodes, if they haven't been visited or a shorter path is found, we update their distance and enqueue them.
4. **Result Output:** After BFS completes, the shortest cycle length is printed. If no cycle is found, -1 is printed.

This approach ensures that we efficiently find the shortest cycle starting and ending at node 1 using BFS, which guarantees the shortest path in an unweighted graph.
