2. **Binary Search for Efficiency**: For each enemy, use binary search to determine the minimum number of attacks (T) needed to reduce their health to 0 or below. This involves checking whether a given T would suffice by counting the number of special attacks (every 3rd attack) and normal attacks.

        long long low = 0;

        long long T = 0;
        while (low <= high) {
            long long mid = (low + high) / 2;
            long long special = mid / 3;
            long long normal = mid - special;
            long long damage = normal * 1 + special * 3;
            if (damage >= hi) {

### Explanation
1. **Binary Search for Minimal Attacks**: For each enemy, we perform a binary search to find the smallest number of attacks (T) such that the total damage (sum of normal and special attacks) is at least the enemy's health. The binary search efficiently narrows down the possible values of T.
2. **Damage Calculation**: The damage from T attacks is computed as `normal * 1 + special * 3`, where `special` is the number of attacks that are multiples of 3 (i.e., every 3rd attack), and `normal` is the remaining attacks.
