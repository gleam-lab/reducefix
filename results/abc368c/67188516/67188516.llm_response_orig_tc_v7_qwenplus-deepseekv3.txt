To solve this problem, we need to simulate the process of attacking enemies until all their health points (HP) are reduced to zero or below. The key observation is that the attack pattern changes every 3 turns: every 3rd turn, the attack deals 3 damage, while other turns deal 1 damage. 

### Approach
1. **Understanding the Attack Pattern**: The attacks cycle every 3 turns. For each enemy, we can decompose the total damage into cycles of 3 turns. Each cycle of 3 turns deals 5 damage (1 + 1 + 3). The remaining damage after these full cycles can be handled by checking the position within the next partial cycle.
2. **Efficient Calculation**: For each enemy, calculate the number of full 3-turn cycles (each dealing 5 damage) and the remaining HP. Then, determine how many additional turns are needed to reduce the remaining HP based on the position within the current cycle.
3. **Tracking the Current Turn Position**: As we process each enemy, we need to keep track of the current position in the 3-turn cycle (0, 1, or 2) because the damage dealt in the next turn depends on this position.
4. **Handling Overlapping Damage**: The total time taken is the sum of the turns required for each enemy, adjusted for the overlapping turns where multiple enemies might be attacked in the same turn if their HP is zero.

### Solution Code
```cpp
#include <iostream>
#include <vector>

using namespace std;

int main() {
    int N;
    cin >> N;
    vector<long long> H(N);
    for (int i = 0; i < N; ++i) {
        cin >> H[i];
    }

    long long T = 0;
    long long pos = 0; // 0, 1, 2 indicating the position in the 3-turn cycle

    for (long long h : H) {
        long long full_cycles = h / 5;
        T += full_cycles * 3;
        long long remainder = h % 5;

        if (remainder == 0) continue;

        if (pos == 0) {
            T += 1;
            pos = 1;
        } else if (pos == 1) {
            if (remainder <= 1) {
                T += 1;
                pos = 2;
            } else {
                T += 2;
                pos = 0;
            }
        } else { // pos == 2
            if (remainder <= 3) {
                T += 1;
                pos = 0;
            } else {
                T += 2;
                pos = 1;
            }
        }
    }

    cout << T << endl;
    return 0;
}
```

### Explanation
1. **Initialization and Input Handling**: The code reads the number of enemies and their respective health points.
2. **Tracking Cycles and Remainder**: For each enemy, the total damage is divided into full cycles of 3 turns (5 damage per cycle) and a remainder. The full cycles contribute 3 turns each, and the remainder is processed based on the current position in the 3-turn cycle.
3. **Position Handling**: The position (`pos`) tracks where we are in the 3-turn cycle (0, 1, or 2). Depending on the remainder and the current position, the code calculates the additional turns needed:
   - **Position 0**: The next attack is 1 damage. If the remainder is exactly 1, only one turn is needed. Otherwise, more turns might be required.
   - **Position 1**: The next attack is 1 damage. If the remainder is 1, one turn suffices. If the remainder is 2, two turns are needed (1 + 1), but if the remainder is more, it might require additional turns.
   - **Position 2**: The next attack is 3 damage. If the remainder is up to 3, one turn suffices. Otherwise, two turns are needed (3 + 1).
4. **Output**: The total turns (`T`) accumulated for all enemies is printed.

This approach efficiently calculates the total turns by leveraging the periodic nature of the attack pattern, ensuring optimal performance even for large inputs.
