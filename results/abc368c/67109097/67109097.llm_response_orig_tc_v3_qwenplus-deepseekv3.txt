### Approaches
1. **Simulation of Attacks**: We need to simulate the attacks step by step, considering the frontmost enemy with health greater than zero at each step.
2. **Attack Pattern**: 
   - For each attack step (T), if T is a multiple of 3, the attack reduces the frontmost enemy's health by 3; otherwise, it reduces by 1.
   - After each attack, if the frontmost enemy's health drops to zero or below, it is removed from the queue, and the next enemy becomes the frontmost.
3. **Efficiency**: Given the constraints, a brute-force simulation would be too slow. Instead, we can calculate the number of attacks required for each enemy by considering the periodic nature of the attacks (every 3 steps, the attack is stronger).

#include <bits/stdc++.h>

using namespace std;

int main() {
    int N;

    vector<long long> H(N);
    for (int i = 0; i < N; i++) cin >> H[i];
    
    long long T = 0;
    for (int i = 0; i < N; i++) {
        long long h = H[i];
        if (i > 0) {
            long long prev_T = T;
            long long damage_from_prev = prev_T / 3;
            h -= damage_from_prev;
            long long remainder_T = prev_T % 3;
            if (remainder_T >= 1) h -= 1;
            if (remainder_T >= 2) h -= 1;
            if (h <= 0) continue;

        long long cycles = h / 5;
        T += cycles * 3;
        h -= cycles * 5;
        long long steps = 0;
        long long damage = 0;
        while (damage < h) {

            if ((T + steps) % 3 == 0) damage += 3;
            else damage += 1;
