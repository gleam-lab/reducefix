To solve this problem correctly, we need to simulate the attack pattern efficiently.

### Key Observations:

- Each enemy must be attacked until its health drops to 0 or less.
- You attack the current frontmost enemy (in order), and the damage depends on whether `T` is a multiple of 3:
  - If `T % 3 == 0`, you deal **3** damage.
  - Otherwise, you deal **1** damage.
- This creates a repeating pattern where every 3 attacks results in a total of 5 damage: two attacks of 1 and one of 3.

### Strategy:

Instead of simulating each attack step-by-step (which would be too slow for large N and H[i]), we can precompute how many full cycles of 3 attacks are needed (`full_cycles`) and handle any remaining damage with a while loop.

Each full cycle (3 attacks) does 5 damage. So:
```cpp
full_cycles = (H[i] + 4) / 5;
T += full_cycles * 3;
```

This ensures that even if there's a remainder, we account for all required attacks.

### Why Your Code Was Incorrect:

Your code used integer division (`H[i] / 5`) which truncates any remainder. This led to undercounting when the health wasn't perfectly divisible by 5.

We fix this by using ceiling division `(H[i] + 4) / 5`.

### ✅ Fixed C++ Code:
```cpp
#include <bits/stdc++.h>

using namespace std;

int main() {
    int N;
    cin >> N;
    vector<long long> H(N);
    for (int i = 0; i < N; ++i) {
        cin >> H[i];
    }

    long long T = 0;
    for (int i = 0; i < N; ++i) {
        long long full_cycles = (H[i] + 4) / 5;
        T += full_cycles * 3;
    }

    cout << T << endl;
    return 0;
}
```

---

### Explanation:

- `(H[i] + 4) / 5` computes how many full 5-damage cycles are needed.
- Each such cycle takes 3 attacks → multiply by 3 to get total time added.
- No need to simulate each attack individually — just calculate the total number mathematically.

### Time Complexity:
- O(N) — Efficient even for maximum constraints.
