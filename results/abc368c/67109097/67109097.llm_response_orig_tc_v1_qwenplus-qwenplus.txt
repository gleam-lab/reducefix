To solve this problem efficiently, we need to simulate the attack pattern on enemies while minimizing redundant computations.

Key Observations:
- Each enemy must be attacked until its health is ≤ 0.
- You can only attack one enemy at a time (the frontmost one with positive health).
- Attacks alternate: normal attack (−1) except every third second which deals −3 damage.

The naive approach of simulating each attack step-by-step would be too slow for large inputs. Instead, we use an optimized greedy method:

### Efficient Strategy:
For each enemy:
1. Simulate how many full **cycles** of attacks are needed where every 3rd attack is powerful (−3), and others are weak (−1).
2. Track how much damage is dealt in full cycles.
3. Handle leftover damage after full cycles using individual steps.

This leads to an efficient O(N) solution by computing optimal T per enemy while maintaining the order of attacks.

Here's the fixed C++ code:

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int N;
    cin >> N;
    vector<long long> H(N);
    for (int i = 0; i < N; ++i) {
        cin >> H[i];
    }

    long long T = 0;

    for (int i = 0; i < N; ++i) {
        long long h = H[i];

        // Full cycles: each cycle has 3 attacks (deals 5 damage total: 1+1+3)
        long long cycles = h / 5;
        T += cycles * 3;
        h -= cycles * 5;

        if (h > 0) {
            // Apply remaining attacks manually
            T++;
            if (T % 3 == 0) h -= 3;
            else h--;

            if (h > 0) {
                T++;
                if (T % 3 == 0) h -= 3;
                else h--;
            }
            if (h > 0) {
                T++;
                if (T % 3 == 0) h -= 3;
                else h--;
            }
        }
    }

    cout << T << endl;
    return 0;
}
```
