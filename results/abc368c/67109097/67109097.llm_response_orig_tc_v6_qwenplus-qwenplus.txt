To solve this problem correctly, we need to simulate the attack strategy in an optimized way. The naive simulation of attacking each enemy one-by-one and decrementing their health based on time `T` being a multiple of 3 is too slow for large inputs.

---

### **Key Insight**

Each attack reduces an enemy's health by:
- **1** if `T % 3 != 0`
- **3** if `T % 3 == 0`

So over a sequence of attacks (say 3), the total damage dealt is:
- 1st attack: -1
- 2nd attack: -1
- 3rd attack: -3  
→ Total = -5 damage over 3 attacks

This pattern repeats every 3 steps, dealing exactly **5 damage per group of 3 attacks**.

We can calculate how many full groups of 3 attacks are needed to reduce an enemy's health to ≤ 0, and then handle any remaining attacks.

But crucially, since enemies are lined up and you must defeat them in order, the timing of when you hit each enemy matters — because the global time `T` increases with every attack.

---

### **Strategy**

We will process each enemy in order and determine how much time `T` is spent attacking it. Since attacks are only applied to the current enemy until its health is ≤ 0, we can compute the number of full cycles (`cycles`) where 3 attacks do 5 damage, and then simulate any leftover attacks.

However, instead of simulating all individual attacks (which would be too slow), we can use binary search or mathematical computation to determine how many attacks (i.e., increments of T) are required to finish off each enemy given the timing rules.

---

### **Efficient Computation**

For each enemy:
1. Use binary search to find the minimum value of `T` such that the cumulative damage applied to the enemy (based on the timing rules) equals or exceeds its health.
2. After computing the necessary `T` for the current enemy, update the global `T`.

The key is to simulate how many of those `T` values correspond to "multiple of 3" and how many don't, and compute the total damage accordingly.

---

### ✅ Final Optimized C++ Code

```cpp
#include <bits/stdc++.h>
using namespace std;

// Calculates the minimum T such that total damage >= h
long long get_required_T(long long h) {
    long long low = 0, high = 3 * h; // upper bound estimation
    while (low < high) {
        long long mid = (low + high) / 2;
        long long three_damage = mid / 3;
        long long one_damage = mid - three_damage;
        long long total_damage = three_damage * 3 + one_damage * 1;
        if (total_damage >= h)
            high = mid;
        else
            low = mid + 1;
    }
    return low;
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(0);

    int N;
    cin >> N;
    vector<long long> H(N);
    for (int i = 0; i < N; ++i)
        cin >> H[i];

    long long T = 0;
    for (int i = 0; i < N; ++i) {
        long long t_needed = get_required_T(H[i]);
        T += t_needed;
    }

    cout << T << endl;
    return 0;
}
```

### ✅ Explanation

- We define a helper function `get_required_T(h)` which uses binary search to find the minimal `T` such that the cumulative damage (according to the rules) reaches at least `h`.
- For each enemy, we call this function and add the result to the global `T`.
- This approach is efficient and works within constraints (`O(N log H)` where `H` is up to `1e9`, and `N` is up to `2e5`).
