### Approaches

1. **Simulating the Attacks**: For each enemy, we need to simulate the attacks until their health drops to zero or below. The attack pattern alternates between dealing 1 damage (for non-multiple of 3 steps) and 3 damage (for steps that are multiples of 3).
2. **Optimal Calculation**: Direct simulation for each enemy step-by-step would be too slow for large H_i (up to 10^9). Instead, we can calculate the number of full cycles (3 steps: 1, 1, 3 damage) and then handle the remaining damage in the partial cycle.
3. **Efficiency**: For each enemy, compute how many full cycles (each cycle of 3 steps deals 5 damage) are needed, then process the remaining health with individual steps (1 or 3 damage based on the step count).

#include <bits/stdc++.h>
using namespace std;
#define rep(i, n) for (int i = 0; i < (n); ++i)
using ll = long long;

ll compute_steps(ll T, ll H) {
    ll left = 0, right = 3 * H;
    ll ans = right;
    while (left <= right) {
        ll mid = (left + right) / 2;
        ll k = (T + mid) / 3 - T / 3;
        ll damage = mid + 2 * k;
        if (damage >= H) {
            ans = mid;
            right = mid - 1;
        } else {
            left = mid + 1;
        }
    }
    return ans;
}

int main() {
    int n;

    vector<ll> H(n);
    rep(i, n) cin >> H[i];
    ll T = 0;
    for (ll h : H) {
        ll s = compute_steps(T, h);
        T += s;

Explanation:

- For each enemy, we perform a binary search to find the minimal number of steps `s` such that the total damage dealt to the enemy is at least its health `h`. The damage is calculated as `s + 2*k`, where `k` is the number of steps within the current step range that are multiples of 3 (computed as `k = (T + s)/3 - T/3`).

- The binary search efficiently narrows down the possible values of `s` between `0` and `3*h` (since worst case, each of the steps could deal at least 1 damage, but `3*h` steps would certainly suffice).

- Once the minimal `s` is found, it is added to the total step count `T`, and the process repeats for the next enemy.

This approach ensures that we handle each enemy in logarithmic time relative to their health, making the overall complexity O(N log H_max), which is efficient for the given constraints.
