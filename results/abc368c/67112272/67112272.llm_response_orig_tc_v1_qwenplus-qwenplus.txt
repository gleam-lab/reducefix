To solve this problem optimally and correctly, we need to simulate the attack mechanism efficiently.

### Key Observations:
- Each enemy must be reduced to 0 or less.
- You attack enemies **in order**, one at a time.
- Every second `T` increases by 1.
- If `T % 3 == 0`, the attack does **3 damage**; otherwise, it does **1 damage**.

This means for each enemy, we must calculate how many attacks are needed (i.e., how much `T` increases) until its health drops to ≤ 0. The twist is that every third attack deals 3 damage instead of 1.

### Strategy:
Each group of 3 attacks (e.g., T=1,2,3) results in a total of **5 damage**: 1 + 1 + 3 = 5.
So, if an enemy has `h` health:
- Compute how many full groups of 3 attacks are needed: `q = h / 5`
- Remaining damage after `q` groups: `r = h % 5`
- Time taken for `q` full groups: `3 * q`
- Add extra time based on remainder `r`:
  - If `r <= 2`: add `r` seconds (each does 1 damage)
  - If `r >= 3`: add 3 seconds (two 1-damage attacks and one 3-damage)

This logic correctly accumulates `T` as you process each enemy in order.

### Final C++ Code:

```cpp
#include <bits/stdc++.h>
using namespace std;

#define rep(i, n) for (long long i = 0; i < (n); ++i)
using ll = long long;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    ll n;
    cin >> n;
    vector<ll> h(n);
    for (ll i = 0; i < n; ++i) {
        cin >> h[i];
    }

    ll ans = 0;
    for (ll health : h) {
        ll q = health / 5;
        ll r = health % 5;
        ans += q * 3;
        if (r > 0) {
            if (r >= 3) ans += 3;
            else ans += r;
        }
    }

    cout << ans << "\n";
}
```
