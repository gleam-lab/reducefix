### Approaches
1. **Understanding the Attack Pattern**: The attack sequence is such that for each time step T:
   - If T is a multiple of 3 (i.e., T % 3 == 0), the frontmost enemy's health decreases by 3.
   - Otherwise, the health decreases by 1.
   - Each attack step increases T by 1, and only the frontmost enemy with health > 0 is attacked until all enemies' health drops to ≤ 0.

2. **Efficient Calculation**: For each enemy, we need to determine how many attack steps are required to reduce its health to ≤ 0. This involves:
   - Recognizing that every 3rd attack deals 3 damage, and other attacks deal 1 damage.
   - The total attacks needed for an enemy with health H can be derived by finding the number of 3-damage attacks (`x`) and the remaining 1-damage attacks (`y`), such that \(3x + y \geq H\) and \(x\) is maximized under the constraint that \(3x\) is part of the attack sequence.

3. **Mathematical Formulation**: For each health value H:
   - The minimum number of attacks required is such that \(x\) is the number of 3-damage attacks and \(y\) is the number of 1-damage attacks. The total attacks \(T\) for this enemy will be \(x + y\), but we need to ensure that the sequence of attacks (every 3rd attack is 3-damage) is respected.

4. **Optimization**: To avoid a per-enemy loop (which would be too slow for large H), we can compute the required attacks mathematically:
   - Let \(x\) be the number of 3-damage attacks. Each 3-damage attack occurs at positions \(3, 6, 9, \ldots\). The total damage dealt is \(3x\).
   - The remaining health \(H - 3x\) must be dealt with 1-damage attacks. The total attacks for this part is \(H - 3x\).
   - However, the 3-damage attacks must be spaced every 3 attacks, so the total attacks \(T\) must satisfy \(T \geq 3x\) and \(T \geq x + (H - 3x)\).
   - Solving this, we find \(x \leq \frac{H}{3}\) and \(T = \max(3x, x + (H - 3x)) = \max(3x, H - 2x)\).

5. **Binary Search**: For each enemy, perform a binary search to find the optimal \(x\) (number of 3-damage attacks) that minimizes the total attacks \(T\).

### Solution Code
```cpp
#include <bits/stdc++.h>
using namespace std;
#define rep(i, n) for (int i = 0; i < (n); ++i)
using ll = long long;

ll calculate_attacks(ll h) {
    ll low = 0, high = h / 3;
    ll min_attacks = LLONG_MAX;
    while (low <= high) {
        ll mid = (low + high) / 2;
        ll remaining = h - 3 * mid;
        if (remaining < 0) {
            high = mid - 1;
            continue;
        }
        ll attacks = mid + remaining;
        if (3 * mid >= attacks) {
            min_attacks = min(min_attacks, 3 * mid);
            high = mid - 1;
        } else {
            low = mid + 1;
        }
    }
    return min_attacks;
}

int main() {
    int n;
    cin >> n;
    vector<ll> h(n);
    rep(i, n) cin >> h[i];
    ll ans = 0;
    ll cumulative_attacks = 0;
    rep(i, n) {
        ll required_attacks = calculate_attacks(h[i]);
        if (cumulative_attacks >= required_attacks) {
            cumulative_attacks++;
        } else {
            cumulative_attacks = required_attacks;
        }
        ans = cumulative_attacks;
    }
    cout << ans << endl;
    return 0;
}
```

### Explanation
1. **Binary Search for Optimal Attacks**: The function `calculate_attacks` uses binary search to determine the minimum number of attacks needed to reduce an enemy's health to ≤ 0. It checks for the optimal number of 3-damage attacks (`mid`) and ensures that the sequence of attacks (every 3rd attack being a 3-damage attack) is respected.
2. **Cumulative Attacks Handling**: The main loop processes each enemy's health, updating the cumulative attack count (`cumulative_attacks`). If the previous cumulative attacks suffice to kill the current enemy, it increments the count; otherwise, it sets the count to the required attacks for the current enemy.
3. **Efficiency**: The binary search approach ensures each enemy's health is processed in \(O(\log H_i)\) time, making the overall complexity \(O(N \log H_{max})\), which is efficient for the given constraints (up to \(2 \times 10^5\) enemies and \(10^9\) health each).
