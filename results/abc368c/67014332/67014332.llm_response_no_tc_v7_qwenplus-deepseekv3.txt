### Approaches
1. **Understanding the Game Mechanics**: The player attacks the frontmost enemy every turn. The attack strength varies based on whether the turn count `T` is a multiple of 3:
   - If `T % 3 == 0`, the attack reduces the enemy's health by 3.
   - Otherwise, the attack reduces the health by 1.
2. **Efficient Calculation**: For each enemy, we need to determine how many turns it takes to reduce its health to zero or below, considering the alternating attack pattern. 
   - The key insight is that every 3 turns there is a stronger attack (3 damage) followed by two weaker attacks (1 damage each). This pattern can be treated as a cycle.
   - For each enemy, calculate the number of full cycles (3 turns) and the remaining turns needed after these cycles.
3. **Handling Remaining Health**: After accounting for full cycles, the remaining health can be dealt with by simulating the remaining turns:
   - If the remaining health is 1 or 2, it takes 1 or 2 turns respectively.
   - If the remaining health is 3, it takes 1 turn (since the next attack is a strong attack).

### Solution Code
```cpp
#include<bits/stdc++.h>
using namespace std;
using ll = long long;

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);
    
    ll n;
    cin >> n;
    vector<ll> H(n);
    for (ll &h : H) cin >> h;
    
    ll ans = 0;
    for (ll h : H) {
        ll cycles = h / 5;
        ans += cycles * 3;
        ll rem = h % 5;
        
        if (rem > 0) {
            if (ans % 3 == 2) {
                // Next turn is a strong attack (T+1 is a multiple of 3)
                if (rem <= 3) {
                    ans += 1;
                } else {
                    ans += 2;
                }
            } else if (ans % 3 == 1) {
                // Next turn is a weak attack (T+1 is not a multiple of 3)
                if (rem <= 1) {
                    ans += 1;
                } else {
                    ans += 2;
                }
            } else { // ans % 3 == 0
                // Current turn is a strong attack if T is exact multiple of 3
                if (rem <= 3) {
                    ans += 1;
                } else {
                    ans += 2;
                }
            }
        }
    }
    
    cout << ans << "\n";
    return 0;
}
```

### Explanation
1. **Initialization**: The code reads the number of enemies `n` and their health values `H`.
2. **Cycle Calculation**: For each enemy, it calculates the number of full 3-turn cycles (`cycles`) that can be executed. Each cycle reduces the health by 5 (3 + 1 + 1 damage over 3 turns) and contributes 3 turns to the total count `ans`.
3. **Remaining Health Handling**: The remaining health after full cycles (`rem`) is handled based on the current turn count `ans`:
   - If `ans % 3 == 2`, the next attack will be a strong attack (3 damage). If the remaining health is 3 or less, it takes 1 more turn; otherwise, 2 turns.
   - If `ans % 3 == 1`, the next attack will be a weak attack (1 damage). If the remaining health is 1, it takes 1 more turn; otherwise, 2 turns.
   - If `ans % 3 == 0`, the current attack is a strong attack (3 damage). If the remaining health is 3 or less, it takes 1 more turn; otherwise, 2 turns.
4. **Output**: The total turns `ans` is printed once all enemies' health is reduced to zero or below.

This approach ensures that the attacks are simulated efficiently, handling each enemy's health in logarithmic time relative to the health values, making it suitable for large input sizes.
