```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    long double cost = 0;
    int n;
    cin >> n;

    vector<pair<long double, long double>> points(n);
    for(int i = 0; i < n; ++i) {
        cin >> points[i].first >> points[i].second;
    }

    // Add the origin as the first point
    points.push_back({0, 0});

    for(int i = 1; i <= n; ++i) {
        long double dx = points[i].first - points[i - 1].first;
        long double dy = points[i].second - points[i - 1].second;
        cost += sqrtl(dx * dx + dy * dy);
    }

    cout << setprecision(10) << fixed << cost << '\n';

    return 0;
}
```

### Explanation of Changes
1. **Efficiency**: The original code was inefficient due to repeated calculations. By storing all points including the origin, we can iterate once through the list to calculate the total distance.
2. **Standard Input/Output Handling**: Improved efficiency by disabling synchronization between C and C++ I/O streams and tying cin to nullptr.
3. **Fixed Precision**: Ensured that the output is formatted with 10 decimal places to match the required precision.
4. **Edge Cases**: The solution handles edge cases such as an empty input array or a single element array gracefully without hardcoding any special cases.
