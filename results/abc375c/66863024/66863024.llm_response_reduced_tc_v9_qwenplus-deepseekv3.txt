The problem states that the output should have an absolute or relative error of at most \(10^{-6}\), so we need to print the result with enough precision to meet this requirement.

### Approaches
1. **Input Reading**: Read the number of points \(N\) and then each of the \(N\) points.
2. **Distance Calculation**: Calculate the Euclidean distance between consecutive points, starting from the origin (0, 0) to the first point, then between each subsequent point, and finally back from the last point to the origin.
3. **Precision Handling**: Ensure the output is printed with sufficient precision (at least 15 decimal places) to meet the problem's accuracy requirements.

### Solution Code
```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    long double cost = 0;
    int N;
    long double nowX = 0, nowY = 0;
    long double nextX, nextY;

    cin >> N;

    for (int i = 0; i < N; i++) {
        cin >> nextX >> nextY;
        long double dx = nowX - nextX;
        long double dy = nowY - nextY;
        cost += sqrtl(dx * dx + dy * dy);
        nowX = nextX;
        nowY = nextY;
    }

    cost += sqrtl(nowX * nowX + nowY * nowY);

    cout << fixed << setprecision(15) << cost << endl;

    return 0;
}
```

### Explanation
1. **Input Handling**: The code reads the number of points \(N\) and then each point \((X_i, Y_i)\) in sequence.
2. **Distance Calculation**: For each point, it calculates the Euclidean distance from the current position to the next point. The initial current position is the origin (0, 0). After processing all points, it adds the distance from the last point back to the origin.
3. **Precision Handling**: The output is printed using `fixed` and `setprecision(15)` to ensure that the result is displayed with 15 decimal places, which meets the required precision for all test cases, including hidden ones. This ensures the output is accurate within the problem's specified error tolerance.
