```cpp
#include <iostream>
#include <cmath>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(0);
    
    int n;
    cin >> n;
    
    long double x = 0, y = 0, total_cost = 0;
    
    for(int i = 0; i < n; ++i) {
        long double nx, ny;
        cin >> nx >> ny;
        
        total_cost += sqrt((x - nx) * (x - nx) + (y - ny) * (y - ny));
        
        x = nx;
        y = ny;
    }
    
    total_cost += sqrt(x * x + y * y);
    
    printf("%.10Lf\n", total_cost);
    
    return 0;
}
```

Explanation of Fixes:
1. **Efficiency**: The original code had a redundant calculation of `nextX` and `nextY`. It was not needed as the coordinates were already being updated in each iteration.
2. **Precision**: The use of `long double` ensures higher precision which is crucial for large inputs where the difference between expected and actual results can be very small.
3. **I/O Optimization**: Used `ios::sync_with_stdio(false);` and `cin.tie(0);` to speed up I/O operations.
4. **Output Formatting**: Used `printf` with `%.10Lf` to print the result with exactly 10 decimal places, matching the problem's requirement.
