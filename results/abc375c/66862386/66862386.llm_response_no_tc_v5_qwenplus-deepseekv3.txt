3. **Algorithm**: The algorithm itself is straightforward and correct in terms of the sequence of steps: iterating through each point, calculating the distance from the current position to the next point, updating the current position, and finally calculating the distance back to the origin. The issue lies in the data types used for the coordinates.

### Fixes
1. **Use `long long` for Coordinates**: To prevent overflow in the squared differences, the coordinates should be stored in a type that can handle large numbers, such as `long long`.
2. **Direct Calculation in `long double`**: Ensure that the squared differences are calculated in `long double` to maintain precision throughout the computations.

using namespace std;

int main() {
    long double ans = 0;
    int N;
    long long nowX = 0, nowY = 0;
    long long nextX, nextY;

    cin >> N;

    for (int i = 0; i < N; i++) {
        cin >> nextX >> nextY;
        long double dx = nowX - nextX;
        long double dy = nowY - nextY;
        ans += sqrtl(dx * dx + dy * dy);
        nowX = nextX;
        nowY = nextY;
    }

    long double dx = nowX;
    long double dy = nowY;
    ans += sqrtl(dx * dx + dy * dy);

    cout << fixed << setprecision(10) << ans << endl;

    return 0;
}
