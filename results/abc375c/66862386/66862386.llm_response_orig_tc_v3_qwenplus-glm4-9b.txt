
It appears that your code is almost correct, but there is a critical error in the calculation of the cost for each move and the final return to the origin.

The problem requires calculating the Euclidean distance between two points on a 2D plane, which can be found using the formula:

$$\text{Distance} = \sqrt{(x_2 - x_1)^2 + (y_2 - y_1)^2}$$

Your code is attempting to do this, but it has several issues:

1. The calculation of the cost for each move is done inside a loop. However, it is only updating the `nowX` and `nowY` values after adding the cost to `ans`. Since `cost` is recalculated in each iteration without actually updating `nowX` and `nowY`, the distance is not accumulating correctly.

2. The final return to the origin is not correctly calculated because `cost` is calculated after the loop ends but before updating `nowX` and `nowY`.

Here is the corrected version of your code:

```cpp
#include <iostream>
#include <cmath>
#include <vector>

using namespace std;

int main() {
    long double totalCost = 0.0;
    int nowX = 0, nowY = 0;
    
    int N;
    cin >> N;
    vector<int> X(N), Y(N);

    for (int i = 0; i < N; ++i) {
        cin >> X[i] >> Y[i];
    }

    for (int i = 0; i < N; ++i) {
        int nextX = X[i], nextY = Y[i];
        long double distance = sqrt(pow(nextX - nowX, 2) + pow(nextY - nowY, 2));
        totalCost += distance;
        nowX = nextX;
        nowY = nextY;
    }

    // Return to the origin
    long double distanceToOrigin = sqrt(pow(nowX, 2) + pow(nowY, 2));
    totalCost += distanceToOrigin;

    cout << fixed << setprecision(10) << totalCost << endl;

    return 0;
}
```

Some notes about the changes:

1. I used a vector to store the coordinates since we will not use them after reading them from the input.

2. I changed the type of `totalCost` to `long double` to have more precision when dealing with floating-point operations.

3. I now calculate the distance between the current position and the next point before updating the current positions.

4. After the loop, I add the distance from the last point to the origin to `totalCost`.

5. I also changed the precision of the output to meet the problem's requirements.
