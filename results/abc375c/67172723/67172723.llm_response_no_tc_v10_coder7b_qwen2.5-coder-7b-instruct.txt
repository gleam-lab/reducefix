```cpp
#include <bits/stdc++.h>
using namespace std;

int main(){
  int n;
  cin >> n;
  
  int px=0, py=0;
  double ans=0;
  int x, y;
  for(int i=0; i<n; i++){
    cin >> x >> y;
    ans += hypot(x-px, y-py);
    px = x;
    py = y;
  }
  
  ans += hypot(px, py); // Correctly add the distance back to the origin after visiting all points
  printf("%.10f\n", ans);
  return 0;
}
```

In the original code, there was an issue where the final step of returning to the origin was not correctly calculated. The variables `x` and `y` were used instead of the last coordinates visited (`px` and `py`). This has been corrected in the revised code. Additionally, the use of `hypot` function ensures that the calculation of distances is accurate even with large numbers, adhering to the problem's constraints.
