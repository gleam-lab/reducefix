```cpp
#include <bits/stdc++.h>
using namespace std;

int main(){
  int n;
  cin >> n;
  
  int px=0, py=0;
  double ans=0;
  int x, y;
  for(int i=0; i<n; i++){
    cin >> x >> y;
    ans += hypot(x-px, y-py);
    px = x;
    py = y;
  }
  
  ans += hypot(px, py);
  printf("%.10f\n", ans);
  return 0;
}
```

**Explanation of Fixes:**
1. **Initialization of `x` and `y`:** In the original code, `x` and `y` were declared but not initialized before being used to calculate the distance from the previous point to the first point after the loop. This led to incorrect calculations since the initial position was incorrectly set to `(x, y)` instead of `(0, 0)`.
2. **Updating `px` and `py`:** After each iteration where the new point `(x, y)` is processed, it should update the previous point coordinates (`px`, `py`) to `(x, y)` respectively. This ensures that the next iteration calculates the distance correctly between consecutive points.
3. **Final Distance Calculation:** The final distance calculation should use `px` and `py` which now hold the last visited point's coordinates, ensuring the return trip is calculated accurately from the last point back to the origin.
