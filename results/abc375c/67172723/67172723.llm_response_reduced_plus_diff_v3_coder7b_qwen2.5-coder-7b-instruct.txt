To solve this problem efficiently, we need to calculate the total distance Takahashi travels when starting at the origin, visiting each of the N points in sequence, and returning to the origin. The key is to compute the Euclidean distance between consecutive points and sum them up.

Here's the corrected and optimized C++ code that addresses all the issues mentioned:

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false); cin.tie(0); cout.tie(0);

    int n;
    cin >> n;

    vector<pair<long long, long long>> points(n);
    for (int i = 0; i < n; ++i) {
        cin >> points[i].first >> points[i].second;
    }

    // Function to calculate Euclidean distance between two points
    auto dist = [](const pair<long long, long long>& p1, const pair<long long, long long>& p2) {
        long long dx = p1.first - p2.first;
        long long dy = p1.second - p2.second;
        return sqrt(dx * dx + dy * dy);
    };

    double total_distance = 0;
    
    // Calculate the distance from origin to the first point
    total_distance += dist({0, 0}, points[0]);
    
    // Calculate the distance between consecutive points
    for (int i = 0; i < n - 1; ++i) {
        total_distance += dist(points[i], points[i + 1]);
    }
    
    // Calculate the distance from the last point back to the origin
    total_distance += dist(points[n - 1], {0, 0});

    printf("%.10f\n", total_distance);
    return 0;
}
```

### Explanation:
1. **Input Handling**: We read the number of points `n` and store their coordinates in a vector of pairs.
2. **Distance Calculation**: We define a lambda function `dist` to calculate the Euclidean distance between two points.
3. **Total Distance Calculation**:
   - Start by adding the distance from the origin `(0, 0)` to the first point.
   - Then, loop through the points to add the distances between consecutive points.
   - Finally, add the distance from the last point back to the origin.
4. **Output**: Print the total distance with 10 decimal places.
