```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
  int n;
  cin >> n;
  
  long long px = 0, py = 0;
  double ans = 0;
  long long x, y;
  for(int i = 0; i < n; i++) {
    cin >> x >> y;
    ans += sqrt((double)((x - px) * (x - px) + (y - py) * (y - py)));
    px = x;
    py = y;
  }
  
  ans += sqrt((double)(px * px + py * py));
  printf("%.10f\n", ans);
  return 0;
}
```

Explanation of changes made:
1. Changed `int` to `long long` for variables `px`, `py`, `x`, and `y`. This ensures that we can handle large inputs without overflow.
2. Casted `(x - px)` and `(y - py)` to `double` before squaring them in the distance calculation. This avoids integer overflow issues with large coordinates.
3. The final step calculates the distance back to the origin using `sqrt(px * px + py * py)` instead of `hypot(x, y)`. This is because `x` and `y` have been updated to the last visited point's coordinates, not the original origin.
