#define rep(i, n) for (int i = 0; i < (int)(n); i++)

// Function to calculate Euclidean distance between two points
double Euclid(int x1, int x2, int y1, int y2 ){
  double A = sqrt(pow(x1-x2, 2) + pow(y1- y2, 2)); // Corrected function to handle integer inputs properly
  return A;
}

int main() {
  int n; cin >> n;
  double ans = 0; 
  
  vector<int> x(n), y(n);
  rep(i,n) cin >> x[i] >> y[i];
  
  rep(i,n){
    ans += Euclid(x[i],x[(i+1)%n],y[i],y[(i+1)%n]); // Modified loop to correctly handle the last element with modulo operation
  }
  
  cout << fixed << setprecision(12) << ans << endl;
}

1. **Euclid Function**: The function now correctly calculates the Euclidean distance using `pow` instead of `(x1-x2)*(x1-x2)` and `(y1- y2)*(y1-y2)` which were causing issues with integer overflow. This ensures that it can handle large input values within the constraints.

2. **Loop Modification**: The loop now uses modulo operation `(i+1)%n` to correctly connect the last point back to the first point, ensuring the path forms a closed loop as required by the problem statement.

These changes address the core issues in the original code, making it both correct and efficient while adhering to the constraints provided.
