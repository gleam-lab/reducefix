```cpp
#include <iostream>
#include <cmath>
#include <vector>

using namespace std;

int main() {
    int n;
    cin >> n;

    vector<pair<int, int>> points(n);
    for(int i = 0; i < n; ++i) {
        cin >> points[i].first >> points[i].second;
    }

    double cost = 0;
    for(int i = 1; i <= n; ++i) {
        cost += sqrt(pow(points[i % n].first - points[(i - 1) % n].first, 2) 
                    + pow(points[i % n].second - points[(i - 1) % n].second, 2));
    }

    cout << cost << endl;

    return 0;
}
```

### Explanation:
1. **Input Handling**: The input is read into a vector of pairs representing the coordinates of each point.
2. **Cost Calculation**: A loop iterates through the points, calculating the Euclidean distance between consecutive points (and wrapping around to the first point after the last). This is done using the modulo operator (`%`) to handle the circular nature of the path.
3. **Output**: The total cost is printed out.

This solution ensures that:
- It handles all edge cases properly (e.g., empty input, single element).
- It uses standard C++20 features.
- The time complexity is O(N), which is efficient enough for the given constraints.
