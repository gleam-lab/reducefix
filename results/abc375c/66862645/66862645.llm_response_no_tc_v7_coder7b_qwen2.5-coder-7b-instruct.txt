```cpp
#include <bits/stdc++.h>
using namespace std;

int main(){
    long double cost = 0;
    long double ans = 0;
    int N;
    int nowX = 0, nowY = 0;
    int nextX, nextY;

    cin >> N;

    for(int i = 0; i < N; i++) {
        cin >> nextX >> nextY;

        cost = sqrt(pow(nextX - nowX, 2.0) + pow(nextY - nowY, 2.0));

        ans += cost;

        nowX = nextX;
        nowY = nextY;
    }

    cost = sqrt(pow(nowX, 2.0) + pow(nowY, 2.0));
    ans += cost;

    cout << fixed << setprecision(10) << ans << endl;

    return 0;
}
```

### Explanation of Changes
1. **Variable `operation` renamed to `N`:** The variable name was misleading and did not accurately represent what it stored. Renaming it to `N` makes it clearer that it represents the number of points.
2. **Removed unnecessary loop:** The original code had an extra loop iteration that was not needed. This has been removed.
3. **Corrected calculation of `cost`:** The original code used `powl` instead of `pow`, which could lead to precision issues. Using `pow` ensures that we are working with floating-point arithmetic correctly.
4. **Added `fixed` and `setprecision(10)` for output formatting:** This ensures that the output is formatted to match the required precision, avoiding any rounding errors that might occur during output.
5. **Directly calculate the final cost to return to the origin:** Instead of calculating the distance twice, we can directly calculate the cost to return to the origin after visiting all points.
