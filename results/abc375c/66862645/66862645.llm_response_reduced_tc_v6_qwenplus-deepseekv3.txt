To fix this code, we need to ensure that the total cost includes the distance from the origin to the first point, the distances between consecutive points in the order they are given, and finally the distance from the last point back to the origin. The original code missed accumulating the distances between consecutive points, only calculating the distance from the last point back to the origin.

### Approaches
1. **Initialization**: Start from the origin (0, 0).
2. **First Movement**: Calculate the distance from the origin to the first point (X1, Y1).
3. **Intermediate Movements**: For each subsequent point, calculate the distance from the current point to the next point and add it to the total cost.
4. **Return to Origin**: After visiting all points, calculate the distance from the last point back to the origin and add it to the total cost.
5. **Output**: Print the total cost with sufficient precision.

### Solution Code
```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);

    long double total_cost = 0;
    int N;
    cin >> N;
    int prevX = 0, prevY = 0;

    for (int i = 0; i < N; ++i) {
        int X, Y;
        cin >> X >> Y;
        long double dx = X - prevX;
        long double dy = Y - prevY;
        total_cost += sqrtl(dx * dx + dy * dy);
        prevX = X;
        prevY = Y;
    }

    // Add the distance from the last point back to the origin
    long double dx = 0 - prevX;
    long double dy = 0 - prevY;
    total_cost += sqrtl(dx * dx + dy * dy);

    cout << fixed << setprecision(15) << total_cost << endl;

    return 0;
}
```

### Explanation
1. **Initialization**: The variables `total_cost` is initialized to 0 to accumulate the total distance. `prevX` and `prevY` start at 0, representing the origin.
2. **Reading Input**: The number of points `N` is read first. For each point, the coordinates `(X, Y)` are read.
3. **Distance Calculation**: For each point, the distance from the previous point `(prevX, prevY)` to the current point `(X, Y)` is calculated using the Euclidean distance formula and added to `total_cost`. The `prevX` and `prevY` are updated to the current point for the next iteration.
4. **Return to Origin**: After processing all points, the distance from the last point back to the origin `(0, 0)` is calculated and added to `total_cost`.
5. **Output**: The total cost is printed with a precision of 15 decimal places to ensure accuracy as per the problem requirements.

This approach efficiently calculates the total travel distance by sequentially processing each point and accumulating the distances, ensuring correctness for all test cases.
