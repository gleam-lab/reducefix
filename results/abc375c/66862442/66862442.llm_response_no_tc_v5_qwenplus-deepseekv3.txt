The issue with the original code is that it attempts to compute the total distance as the sum of squared distances (dx^2 + dy^2) without taking the square roots of each individual segment. This leads to incorrect results because the actual cost is the sum of the square roots of these squared distances.

### Approaches
1. **Correct Calculation of Distances**: The cost between two points (a, b) and (c, d) should be calculated as the Euclidean distance, which is the square root of ((a - c)^2 + (b - d)^2).
2. **Handling Floating-Point Precision**: Since the problem allows for floating-point outputs, we need to ensure that the intermediate calculations are done with sufficient precision. Using `double` should be sufficient for the given constraints.
3. **Efficiency**: The solution involves a straightforward loop through each point and summing the distances, which is O(N) and efficient enough for the given constraints (N ≤ 2 × 10^5).

### Solution Code
```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int N;
    cin >> N;

    long long curX = 0, curY = 0;
    double total = 0.0;

    for (int i = 0; i < N; ++i) {
        long long x, y;
        cin >> x >> y;

        long long dx = curX - x;
        long long dy = curY - y;
        total += sqrt(dx * dx + dy * dy);

        curX = x; curY = y;
    }
    total += sqrt(curX * curX + curY * curY); // Distance back to origin

    cout << fixed << setprecision(12) << total << '\n';
    return 0;
}
```

### Explanation
1. **Initialization**: The variables `curX` and `curY` are initialized to 0, representing the starting point at the origin. The `total` variable is initialized to 0.0 to accumulate the total distance as a floating-point number.
2. **Loop Through Points**: For each point (x, y) in the input, the code calculates the Euclidean distance from the current position (curX, curY) to (x, y) using the formula `sqrt(dx * dx + dy * dy)`, where `dx` and `dy` are the differences in the x and y coordinates, respectively. This distance is added to `total`.
3. **Return to Origin**: After visiting all points, the code calculates the distance from the last point back to the origin (0, 0) and adds it to `total`.
4. **Output**: The total distance is printed with a precision of 12 decimal places, ensuring the output meets the problem's accuracy requirements.

This approach ensures that the distance calculations are accurate and efficient, handling the large input size within acceptable time limits.
