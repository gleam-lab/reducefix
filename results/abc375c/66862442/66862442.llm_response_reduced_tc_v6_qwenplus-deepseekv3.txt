To fix this code, we need to address the main issue: the code calculates the sum of squared distances instead of the actual Euclidean distances (which require taking the square root). The original code accumulates the squared distances, which is incorrect because the problem requires the sum of the actual distances, including the final distance back to the origin.

### Approaches
1. **Calculate Euclidean Distances:** For each segment of the path (from the current point to the next point, and finally back to the origin), compute the Euclidean distance using the formula \(\sqrt{(a - c)^2 + (b - d)^2}\).
2. **Accumulate Distances:** Sum all these distances to get the total cost.
3. **Precision Handling:** Since the output requires high precision (up to \(10^{-6}\) relative or absolute error), we need to ensure that the square roots are calculated with sufficient precision and that the final sum is printed with enough decimal places.

### Solution Code
```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int N;
    cin >> N;

    long long curX = 0, curY = 0;
    double total = 0.0;

    for (int i = 0; i < N; ++i) {
        long long x, y;
        cin >> x >> y;

        long long dx = curX - x;
        long long dy = curY - y;
        total += sqrt(dx * dx + dy * dy);

        curX = x;
        curY = y;
    }
    total += sqrt(curX * curX + curY * curY);

    cout << fixed << setprecision(15) << total << '\n';
}
```

### Explanation
1. **Initialization:** The variables `curX` and `curY` are initialized to 0, representing the origin. `total` is initialized to 0.0 to accumulate the distances.
2. **Reading Input:** For each point \((X_i, Y_i)\), the code calculates the Euclidean distance from the current point \((curX, curY)\) to \((X_i, Y_i)\) using the formula \(\sqrt{(curX - X_i)^2 + (curY - Y_i)^2}\). This distance is added to `total`.
3. **Updating Position:** After processing each point, the current position \((curX, curY)\) is updated to the next point.
4. **Returning to Origin:** After visiting all points, the distance from the last point back to the origin is calculated and added to `total`.
5. **Output:** The total distance is printed with a precision of 15 decimal places to meet the required accuracy constraints.

This approach ensures that the total cost is accurately calculated by summing the actual Euclidean distances between consecutive points and the return trip to the origin, providing the correct result without any precision loss.
