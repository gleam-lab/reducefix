To solve this problem correctly, we need to ensure that we calculate the Euclidean distance between each pair of consecutive points and also account for the distance from the last point back to the origin. The total cost should be computed accurately as a floating-point number rather than an integer sum of squares. We will use the `long double` type for calculations to handle the precision required.

Here's the corrected and optimized C++ code:

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int N;
    cin >> N;

    long double curX = 0, curY = 0;
    long double total = 0;          // Use long double for high precision

    for (int i = 0; i < N; ++i) {
        long long x, y;
        cin >> x >> y;

        long double dx = curX - x;
        long double dy = curY - y;
        total += sqrtl(dx * dx + dy * dy);

        curX = x; 
        curY = y;
    }
    total += sqrtl(curX * curX + curY * curY); // Return to origin

    cout << fixed << setprecision(15) << total << '\n'; // Output with high precision
}
```

### Explanation:
1. **Data Type**: Changed the variable types from `__int128` and `long long` to `long double` to support higher precision required for the square root and addition operations.
2. **Precision Handling**: Used `fixed` and `setprecision(15)` in the output stream to ensure the result is printed with up to 15 decimal places.
3. **Algorithm**: The algorithm remains straightforward, calculating the Euclidean distance between each consecutive pair of points and summing these distances. Finally, it adds the distance from the last point back to the origin.

This approach ensures that the solution is both accurate and efficient, meeting all the problem requirements.
