To fix this code, we need to address the primary issue: the incorrect calculation of the Euclidean distance. The original code sums the squares of the distances but fails to take the square root at each step, which is necessary for the correct total cost. The Euclidean distance between two points \((a, b)\) and \((c, d)\) is \(\sqrt{(a - c)^2 + (b - d)^2}\), and the total cost is the sum of these distances for each consecutive point, including the return to the origin.

### Approaches
1. **Initialization**: Start from the origin \((0, 0)\).
2. **Iterate through each point**: For each point \((X_i, Y_i)\), calculate the Euclidean distance from the current position to this point.
3. **Sum the distances**: Add each calculated distance to a running total.
4. **Return to the origin**: After visiting all points, calculate the distance from the last point back to the origin and add it to the total.
5. **Output the result**: Print the total distance with sufficient precision to meet the problem's error tolerance.

### Solution Code
```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int N;
    cin >> N;

    double total = 0.0;
    long long curX = 0, curY = 0;

    for (int i = 0; i < N; ++i) {
        long long x, y;
        cin >> x >> y;

        long long dx = curX - x;
        long long dy = curY - y;
        total += sqrt(dx * dx + dy * dy);

        curX = x;
        curY = y;
    }

    total += sqrt(curX * curX + curY * curY);

    cout << fixed << setprecision(15) << total << '\n';

    return 0;
}
```

### Explanation
1. **Initialization**: The variables `total` is initialized to `0.0` to accumulate the total distance as a floating-point number. `curX` and `curY` are set to `0`, representing the initial position at the origin.
2. **Loop through Points**: For each point \((X_i, Y_i)\) read from input, the code calculates the differences in coordinates (`dx` and `dy`) from the current position to the point. The Euclidean distance is computed using `sqrt(dx*dx + dy*dy)` and added to `total`.
3. **Update Position**: After processing each point, the current position is updated to \((X_i, Y_i)\).
4. **Return to Origin**: After processing all points, the distance from the last point back to the origin is computed and added to `total`.
5. **Output**: The total distance is printed with a precision of 15 decimal places to ensure the output meets the required accuracy constraints.

This approach correctly calculates the total travel distance by summing the Euclidean distances between consecutive points and the return to the origin, ensuring both algorithmic correctness and precision in the output.
