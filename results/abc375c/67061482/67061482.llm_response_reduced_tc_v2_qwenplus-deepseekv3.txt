### Approaches
1. **Input Reading and Initialization**: Read the number of points \( N \) and initialize the starting position at the origin \((0, 0)\).
2. **Distance Calculation**: For each point, calculate the Euclidean distance from the current position to the next point and accumulate this distance.
3. **Return to Origin**: After visiting all points, calculate the distance from the last point back to the origin and add it to the accumulated distance.
4. **Output**: Print the total distance with sufficient precision using `std::fixed` and `std::setprecision` to ensure at least 15 decimal places, which should cover the precision required by the problem.

### Solution Code
```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    double a = 0, b = 0, c, d, total_cost = 0;
    int N;
    cin >> N;
    for (int i = 0; i < N; i++) {
        cin >> c >> d;
        total_cost += sqrt((a - c) * (a - c) + (b - d) * (b - d));
        a = c;
        b = d;
    }
    total_cost += sqrt(a * a + b * b);
    cout << fixed << setprecision(15) << total_cost << endl;
    return 0;
}
```

### Explanation
- **Initialization**: The starting point \((a, b)\) is initialized to \((0, 0)\) and the total cost (`total_cost`) is initialized to 0.
- **Loop Through Points**: For each point \((c, d)\) read from the input, the Euclidean distance from the current position \((a, b)\) to \((c, d)\) is calculated and added to `total_cost`. The current position is then updated to \((c, d)\).
- **Return to Origin**: After processing all points, the distance from the last point back to the origin \((0, 0)\) is calculated and added to `total_cost`.
- **Output**: The total cost is printed with a precision of 15 decimal places using `std::fixed` and `std::setprecision(15)` to ensure the output matches the expected precision.

This approach ensures that the total distance is calculated accurately and printed with the required precision, addressing the issue seen in the original code where the output was not precise enough.
