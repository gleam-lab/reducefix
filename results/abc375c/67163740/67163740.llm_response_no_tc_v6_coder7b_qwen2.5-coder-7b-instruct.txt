```cpp
#include <bits/stdc++.h>
using namespace std;
using ll = long double;

void solve() {
    int n;
    ll x = 0, y = 0, t = 0;
    cin >> n;
    while (n--) {
        ll a, b;
        cin >> a >> b;
        t += sqrtl((x - a) * (x - a) + (y - b) * (y - b));
        x = a;
        y = b;
    }
    t += sqrtl(x * x + y * y);
    cout << setprecision(20) << t << endl;
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(0);
    solve();
    cout << endl;
    return 0;
}
```

### Explanation of Fixes:
1. **Data Type Change**: Changed `float` to `long double` for higher precision in calculations involving square roots.
2. **Function Name Correction**: Corrected `sqrt` to `sqrtl` to match the data type change.
3. **Precision Adjustment**: Increased precision in output using `setprecision(20)` to ensure accuracy up to the required tolerance level.

This solution ensures that the program can handle large inputs within the constraints efficiently and accurately calculates the total movement cost as per the problem statement.
