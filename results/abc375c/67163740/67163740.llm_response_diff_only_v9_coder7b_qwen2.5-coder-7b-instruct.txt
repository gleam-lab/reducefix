```cpp
#include <bits/stdc++.h>
using namespace std;
using ll = long long;

void solve() {
    int n;
    double x = 0, y = 0, t = 0; // Change float to double for higher precision
    cin >> n;
    while (n--) {
        double a, b;
        cin >> a >> b;
        t += sqrt((x - a) * (x - a) + (y - b) * (y - b)); // Calculate distance correctly
        x = a;
        y = b;
    }
    t += sqrt(x * x + y * y); // Add the final distance back to the origin
    cout << setprecision(20) << t << endl;
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(0);
    solve();
    cout << endl;
    return 0;
}
```

**Explanation**: The original code had issues with type mismatch between `float` and `double`, which could lead to precision loss. By changing all instances of `float` to `double`, we ensure that the calculations maintain higher accuracy, especially important given the large constraints on the problem's inputs. Additionally, the calculation of distances and the addition of the final distance back to the origin were done correctly, ensuring the solution adheres to the problem requirements.
