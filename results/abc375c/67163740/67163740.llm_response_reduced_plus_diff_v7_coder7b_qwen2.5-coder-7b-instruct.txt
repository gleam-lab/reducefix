```cpp
#include <bits/stdc++.h>
using namespace std;
using ld = long double;

void solve() {
    int n;
    ld x = 0, y = 0, t = 0;
    cin >> n;
    while (n--) {
        ld a, b;
        cin >> a >> b;
        t += sqrtl((x - a) * (x - a) + (y - b) * (y - b));
        x = a;
        y = b;
    }
    t += sqrtl(x * x + y * y);
    cout << setprecision(20) << fixed << t << endl;
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(0);
    solve();
    cout << endl;
    return 0;
}
```

1. **Data Type Change**: Changed all instances of `float` to `ld` (long double) to ensure higher precision.
2. **Function Name Change**: Renamed `sqrt` to `sqrtl` to use the long double version of the square root function.
3. **Output Precision**: Used `setprecision(20)` with `fixed` to ensure the output has up to 20 decimal places, which should be sufficient for the problem constraints.

This change ensures that the calculations are performed with high precision, avoiding issues related to floating-point arithmetic errors.
