To solve this problem efficiently, we need to determine which squares on the grid are safe for placing a new knight (piece). A knight moves in an "L" shape: two squares in one direction and one square perpendicular to that direction. Therefore, we need to check each square on the board and see if there's any other knight that could attack it.

Given the constraints, a brute-force approach would be too slow, so we need a more efficient solution. We can use a set to store the positions of the knights and then iterate through each square to check if it is under attack.

Here's the corrected and optimized C++ code:

```cpp
#include <iostream>
#include <vector>
#include <set>
#include <utility>

using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    i32 N;
    cin >> N;

    i32 piece_num;
    cin >> piece_num;

    set<pair<i32, i32>> anti_dup;

    FOR_U(_, 0, piece_num, 1) {
        i32 a, b;
        cin >> a >> b;
        anti_dup.insert({a, b});

        FOR(i, -2, 3) {
            FOR(j, -2, 3) {
                if (abs(i) + abs(j) == 3) {
                    i32 nx = a + i, ny = b + j;
                    if (nx >= 1 && nx <= N && ny >= 1 && ny <= N) {
                        anti_dup.insert({nx, ny});
                    }
                }
            }
        }
    }

    i128 safe_squares = static_cast<i128>(N * N) - static_cast<i128>(anti_dup.size());

    cout << safe_squares << endl;

    return 0;
}
```

### Explanation:
1. **Reading Input**: We read `N` (the size of the grid) and `M` (the number of knights).
2. **Storing Positions**: We use a set to store the positions of the knights.
3. **Marking Attacked Squares**: For each knight, we mark all the squares it can attack by iterating over the "L" shape movements. If the new position is within bounds, we add it to the set.
4. **Calculating Safe Squares**: The total number of safe squares is the total number of squares minus the number of attacked squares.
5. **Output**: We print the number of safe squares.

This approach ensures that we efficiently handle the constraints and correctly identify all safe squares on the board.
