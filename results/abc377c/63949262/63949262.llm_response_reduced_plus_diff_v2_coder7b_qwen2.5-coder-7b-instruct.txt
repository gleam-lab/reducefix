To solve this problem efficiently, we need to determine which squares on the grid can be safely occupied by a new knight move without being captured by any existing knights. Here's a step-by-step approach to achieve this:

1. **Understand the Knight Moves**: A knight moves in an L-shape: two steps in one direction and then one step perpendicular. This means for each existing knight at position `(x, y)`, there are exactly 8 possible positions that it can attack.

2. **Mark Attacked Squares**: For each existing knight, mark the squares it can attack as invalid for placing a new knight.

3. **Count Valid Squares**: Count the total number of valid squares that are not marked as attacked.

Here's the corrected C++ code implementing the above logic:

```cpp
#include <iostream>
#include <vector>
#include <set>

using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    long long N, M;
    cin >> N >> M;

    set<pair<long long, long long>> attacks;
    set<pair<long long, long long>> pieces;

    // Directions for knight moves
    vector<pair<int, int>> directions = {
        {2, 1}, {2, -1}, {-2, 1}, {-2, -1},
        {1, 2}, {1, -2}, {-1, 2}, {-1, -2}
    };

    for (long long i = 0; i < M; ++i) {
        long long x, y;
        cin >> x >> y;
        pieces.insert({x, y});

        // Mark all squares that can be attacked by this knight
        for (const auto& dir : directions) {
            long long nx = x + dir.first;
            long long ny = y + dir.second;
            if (nx >= 1 && nx <= N && ny >= 1 && ny <= N) {
                attacks.insert({nx, ny});
            }
        }
    }

    // Calculate the number of valid squares
    long long valid_squares = N * N - attacks.size();

    cout << valid_squares << endl;

    return 0;
}
```

### Explanation:

1. **Input Reading**: We read `N` (the size of the grid) and `M` (the number of existing knights). Then, we read the positions of these knights into a set called `pieces`.

2. **Directions for Knight Moves**: We define the 8 possible directions a knight can move using a vector of pairs.

3. **Attacking Squares**: For each knight, we calculate all the squares it can attack and add them to the `attacks` set. We ensure that we do not go out of bounds by checking the limits of the grid.

4. **Valid Squares Calculation**: The number of valid squares is calculated by subtracting the size of the `attacks` set from the total number of squares (`N * N`).

This solution ensures that we correctly identify and count all valid squares where a new knight can be placed without being captured by any existing knights. It handles all the constraints and edge cases effectively.
