To solve this problem, we need to determine how many empty squares in an \( N \times N \) grid can have a knight placed on them without being captured by any existing knights. The key observation here is that a knight moves in an "L" shape: two steps in one direction and one step perpendicular to that.

Given the constraints, a naive approach would involve checking each empty square individually to see if placing a knight there would result in any captures. However, this approach would be inefficient due to its quadratic time complexity.

Instead, we can use a more efficient approach by leveraging the properties of knight moves. Specifically, we can mark all squares that can be captured by any existing knight and then count the remaining empty squares.

Here's the revised C++ code:

```cpp
#include <iostream>
#include <vector>
#include <set>

using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    // Read input
    int N;
    cin >> N;
    int M;
    cin >> M;

    // Set to store positions of existing knights
    set<pair<int, int>> knight_positions;

    // Read positions of existing knights
    for (int i = 0; i < M; ++i) {
        int x, y;
        cin >> x >> y;
        knight_positions.insert({x, y});
    }

    // Directions for knight moves
    vector<pair<int, int>> knight_moves = {
        {2, 1}, {2, -1}, {-2, 1}, {-2, -1},
        {1, 2}, {1, -2}, {-1, 2}, {-1, -2}
    };

    // Set to store positions that can be captured by any knight
    set<pair<int, int>> capture_positions;

    // Mark all positions that can be captured by any knight
    for (const auto& pos : knight_positions) {
        int x = pos.first;
        int y = pos.second;
        for (const auto& move : knight_moves) {
            int nx = x + move.first;
            int ny = y + move.second;
            if (nx >= 1 && nx <= N && ny >= 1 && ny <= N) {
                capture_positions.insert({nx, ny});
            }
        }
    }

    // Calculate the number of empty squares
    int empty_squares = N * N - capture_positions.size();

    // Output the result
    cout << empty_squares << endl;

    return 0;
}
```

### Explanation
1. **Reading Input**: We read the dimensions \( N \) and the number of existing knights \( M \). Then, we read the positions of these knights into a set `knight_positions` for quick lookup.

2. **Knight Moves**: We define the possible knight moves in the `knight_moves` vector.

3. **Marking Capture Positions**: For each existing knight position, we check all possible knight moves. If a move results in a valid position within the grid, we add it to the `capture_positions` set.

4. **Counting Empty Squares**: The number of empty squares is simply the total number of squares minus the number of squares that can be captured by any knight.

5. **Output**: Finally, we output the number of empty squares.

This approach ensures that we efficiently determine the number of empty squares that can be safely occupied by a knight without being captured. It handles all given constraints and edge cases effectively.
