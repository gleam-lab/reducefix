To solve this problem efficiently, we need to ensure that each empty square does not lie within the capturing range of any existing knight piece. The key observation here is that a knight can move in an "L" shape: two squares in one direction and one square perpendicular to that direction.

Given this, we need to check each square on the board to see if it can be reached by any existing knight. If it can't, then it's a valid position for our new piece. To do this efficiently, we can use a breadth-first search (BFS) approach starting from each knight's position to mark all reachable positions.
