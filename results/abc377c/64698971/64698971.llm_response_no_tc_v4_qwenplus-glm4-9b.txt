```cpp
#include <iostream>
#include <vector>
#include <set>

using namespace std;

const vector<vector<pair<int, int>>> vic = {
    {{0, 0}}, {{2, 1}}, {{1, 2}}, {{-1, 2}}, {{-2, 1}}, {{-2, -1}},
    {{-1, -2}}, {{1, -2}}, {{2, -1}}
};

int main() {
    int N, M;
    cin >> N >> M;

    set<pair<int, int>> valid_cells;
    for (int i = 0; i < M; ++i) {
        int a, b;
        cin >> a >> b;
        a--; b--;
        for (auto &vec : vic) {
            int nx = a + vec[0].first;
            int ny = b + vec[0].second;
            if (nx >= 0 && nx < N && ny >= 0 && ny < N) {
                valid_cells.insert({nx, ny});
            }
        }
    }

    int N2 = N * N;
    cout << N2 - valid_cells.size();
    return 0;
}
```

### Explanation

1. **Data Structures**: I used `set` to store the valid cells because it allows for efficient insertion and lookup.

2. **Looping through pieces**: For each piece, we calculate the potential cells that can capture the piece using the `vic` array which contains the relative positions that a piece can capture.

3. **Validating positions**: I added a check to make sure the calculated `nx` and `ny` are within the grid bounds (`0 <= nx < N` and `0 <= ny < N`).

4. **Output**: Finally, the number of valid cells where a piece can be placed is calculated by subtracting the number of valid cells determined by the existing pieces from the total number of cells (`N^2`).

### Notes

- The code ensures that we don't count any cell more than once, even if it can be captured by multiple pieces.
- The use of `set` ensures that all valid cells are unique.
- The complexity is approximately `O(M * 8 * N)` which is efficient given the constraints (`M <= 200,000` and `N <= 1,000,000,000`).
