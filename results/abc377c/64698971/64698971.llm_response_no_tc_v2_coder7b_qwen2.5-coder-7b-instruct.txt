The issue with the original code lies in its approach to determining which squares cannot be used because they are adjacent to another piece. The current implementation uses a brute-force method to check all possible moves from each piece's position, but this approach is inefficient for large grids due to its high time complexity.

To optimize the solution, we need to identify the squares that are already occupied or threaten other squares based on their movement patterns. Here’s how we can do it:

1. Identify the positions of all pieces.
2. For each piece, determine the squares it threatens using the given movement rules.
3. Mark these threatened squares in a data structure to keep track of them.
4. Calculate the total number of empty squares that are not threatened.

Here's the corrected and optimized C++ code:

#define rep(i, n) for (int i = 0; i < (int)(n); i++)
#define repi(i, a, b) for (int i = (int)(a); i < (int)(b); i++)

vector<pair<int,int>> movements = {{2, 1}, {1, 2}, {-1, 2}, {-2, 1}, {-2, -1}, {-1, -2}, {1, -2}, {2, -1}};

bool isValid(int x, int y, int N) {
    return x >= 0 && x < N && y >= 0 && y < N;
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int N, M;
    cin >> N >> M;

    set<pair<int, int>> occupied;
    set<pair<int, int>> threatened;

    rep(i, M) {
        int a, b;
        cin >> a >> b;
        a--, b--;

        occupied.insert({a, b});

        for (auto& move : movements) {
            int new_a = a + move.first;
            int new_b = b + move.second;

            while (isValid(new_a, new_b, N)) {
                if (!occupied.count({new_a, new_b})) {
                    threatened.insert({new_a, new_b});
                }
                new_a += move.first;
                new_b += move.second;
            }
        }
    }

    ll total_squares = N * N;
    ll usable_squares = total_squares - threatened.size();

    cout << usable_squares << endl;

    return 0;
}

### Explanation:
1. **Movement Patterns**: We define the movement patterns in the `movements` array.
2. **Validation Function**: The `isValid` function checks if a given position is within the grid bounds.
3. **Occupied and Threatened Sets**: We use two sets, `occupied` to store the positions of existing pieces and `threatened` to store the positions that are threatened by existing pieces.
4. **Marking Threatened Squares**: For each piece, we calculate the positions it threatens by moving according to the defined patterns until we encounter an occupied square or go out of bounds. We then mark these threatened positions in the `threatened` set.
5. **Calculating Usable Squares**: Finally, we calculate the number of usable squares by subtracting the size of the `threatened` set from the total number of squares.

This approach ensures that we efficiently determine the number of usable squares without checking every possible move for each piece.
