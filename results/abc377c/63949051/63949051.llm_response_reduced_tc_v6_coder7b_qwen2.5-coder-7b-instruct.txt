To solve this problem efficiently within the given constraints, we need to determine how many empty squares on an \(N \times N\) grid remain after placing \(M\) knight pieces. Each knight can attack up to 8 other squares around it. Therefore, we need to mark these attacked squares as unusable.

Given the large size of \(N\) (up to \(10^9\)), a direct approach would be too slow. Instead, we use a more efficient method:

1. **Marking Attacked Squares**: For each knight, mark all the squares it can attack.
2. **Counting Unusable Squares**: Count the number of unique squares marked as attacked.
3. **Calculating Usable Squares**: Subtract the count of unusable squares from the total number of squares (\(N^2\)).

To handle the large size of \(N\) and ensure efficiency, we use a hash set to store the coordinates of the attacked squares. This helps in quickly checking and marking squares without running into performance issues due to large arrays.

Here's the corrected and optimized C++ code:

```cpp
#include <iostream>
#include <vector>
#include <string>
#include <set>
#include <algorithm>

using namespace std;

#define ll long long

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    ll N, M;
    cin >> N >> M;

    set<pair<ll, ll>> attackedSquares;

    vector<pair<ll, ll>> moves = {
        {2, 1}, {1, 2}, {-1, 2}, {-2, 1},
        {-2, -1}, {-1, -2}, {1, -2}, {2, -1}
    };

    for (ll i = 0; i < M; ++i) {
        ll a, b;
        cin >> a >> b;

        for (const auto& move : moves) {
            ll x = a + move.first;
            ll y = b + move.second;
            if (x >= 1 && x <= N && y >= 1 && y <= N) {
                attackedSquares.insert({x, y});
            }
        }
    }

    ll usableSquares = N * N - attackedSquares.size();
    cout << usableSquares << endl;

    return 0;
}
```

### Explanation
1. **Initialization**: We initialize `N` and `M`, which represent the dimensions of the grid and the number of knights, respectively.
2. **Set for Attacked Squares**: We use a `set` to store the coordinates of squares that are attacked by any knight.
3. **Knight Moves**: We define the 8 possible moves a knight can make using a `vector` of pairs.
4. **Processing Knights**: For each knight, we calculate the coordinates of all squares it can attack and add them to the `attackedSquares` set.
5. **Counting Usable Squares**: The number of usable squares is calculated as \(N^2\) minus the size of the `attackedSquares` set.
6. **Output**: Finally, we print the number of usable squares.

This solution ensures that we efficiently handle the constraints and correctly compute the number of usable squares.
