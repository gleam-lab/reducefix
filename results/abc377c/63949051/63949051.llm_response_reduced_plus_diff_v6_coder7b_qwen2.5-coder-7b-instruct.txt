To solve this problem, we need to determine how many empty squares on an \(N \times N\) grid can be occupied without capturing any existing pieces. Each piece can capture other pieces based on specific rules involving knight moves.

The key steps to solve this problem are:

1. **Identify Capturing Squares**: For each piece, identify the squares it can capture based on its position and the knight move rules.
2. **Use a Set for Efficient Lookup**: Store these capturing squares in a set to ensure efficient lookup and avoid counting duplicates.
3. **Calculate Valid Squares**: Subtract the number of capturing squares from the total number of squares to get the valid squares where new pieces can be placed.

Here's the corrected and optimized C++ code to achieve this:

```cpp
#include <iostream>
#include <vector>
#include <set>
#include <utility>

using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    i32 N;
    cin >> N;

    i32 piece_num;
    cin >> piece_num;

    vector<pair<i32, i32>> pieces(piece_num);
    for (auto &piece : pieces) {
        cin >> piece.first >> piece.second;
    }

    set<pair<i32, i32>> anti_dup;

    FOR_R(p, pieces) {
        int x = p.first, y = p.second;

        if (x + 2 <= N && y + 1 <= N) {
            anti_dup.insert({x + 2, y + 1});
        }
        if (x + 1 <= N && y + 2 <= N) {
            anti_dup.insert({x + 1, y + 2});
        }
        if (x - 1 >= 1 && y + 2 <= N) {
            anti_dup.insert({x - 1, y + 2});
        }
        if (x - 2 >= 1 && y + 1 <= N) {
            anti_dup.insert({x - 2, y + 1});
        }
        if (x - 2 >= 1 && y - 1 >= 1) {
            anti_dup.insert({x - 2, y - 1});
        }
        if (x - 1 >= 1 && y - 2 >= 1) {
            anti_dup.insert({x - 1, y - 2});
        }
        if (x + 1 <= N && y - 2 >= 1) {
            anti_dup.insert({x + 1, y - 2});
        }
        if (x + 2 <= N && y - 1 >= 1) {
            anti_dup.insert({x + 2, y - 1});
        }
    }

    out<i128>((i128)pow(N, 2) - (i128)anti_dup.size());
    return 0;
}
```

### Explanation:
1. **Input Handling**: Read \(N\) and \(M\), followed by the positions of the \(M\) pieces.
2. **Set for Capturing Squares**: Use a set `anti_dup` to store all squares that are within the knight move range of any existing piece.
3. **Knight Moves**: For each piece, calculate all possible squares it can capture using the knight move rules and insert them into the set.
4. **Valid Squares Calculation**: The number of valid squares is \(N^2\) minus the size of the set `anti_dup`.

This approach ensures that we efficiently handle the constraints and correctly count the number of valid squares where no piece can be captured.
