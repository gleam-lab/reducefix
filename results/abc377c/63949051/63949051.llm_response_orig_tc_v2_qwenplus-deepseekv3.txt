2. **Tracking Threatened Squares**: For each piece, compute all squares that it can potentially threaten (8 possible knight moves). Add these squares to a set to avoid duplicates.
3. **Counting Safe Squares**: Subtract the total number of threatened squares (including the original pieces) from the total grid squares \(N^2\) to get the number of safe squares.
4. **Large Integer Handling**: Use `long long` (or `i64`) to handle large values and prevent overflow.

### Solution Code
```cpp
#include <iostream>
#include <vector>
#include <set>
#include <utility>

using namespace std;

using i64 = long long;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    i64 N, M;
    cin >> N >> M;

    set<pair<i64, i64>> threatened;

    for (i64 k = 0; k < M; ++k) {
        i64 a, b;
        cin >> a >> b;

        threatened.insert({a, b});

        // All possible knight moves
        vector<pair<i64, i64>> moves = {
            {a + 2, b + 1},
            {a + 1, b + 2},
            {a - 1, b + 2},
            {a - 2, b + 1},
            {a - 2, b - 1},
            {a - 1, b - 2},
            {a + 1, b - 2},
            {a + 2, b - 1}
        };

        for (const auto& move : moves) {
            i64 i = move.first;
            i64 j = move.second;
            if (i >= 1 && i <= N && j >= 1 && j <= N) {
                threatened.insert({i, j});
            }
        }
    }

    i64 total_squares = N * N;
    i64 unsafe_squares = threatened.size();
    i64 safe_squares = total_squares - unsafe_squares;

    cout << safe_squares << '\n';

    return 0;
}
```

### Explanation
1. **Input Handling**: The code reads the grid size \(N\) (up to \(10^9\)) and the number of pieces \(M\) (up to \(2 \times 10^5\)).
2. **Tracking Threatened Squares**: For each piece at position \((a, b)\), the code checks all 8 possible knight moves. Each valid move (within the grid bounds) is added to a set to ensure uniqueness.
3. **Counting Safe Squares**: The total number of squares in the grid is \(N^2\). The number of unsafe squares is the size of the set containing all threatened squares (including the original pieces). The safe squares are then computed by subtracting the unsafe squares from the total squares.
4. **Large Integer Handling**: The use of `i64` (a 64-bit integer) ensures that all calculations, especially \(N^2\), are performed correctly without overflow, even for the maximum constraints.

This approach efficiently computes the number of safe squares by leveraging set operations to avoid duplicates and ensures accurate results for large inputs.
