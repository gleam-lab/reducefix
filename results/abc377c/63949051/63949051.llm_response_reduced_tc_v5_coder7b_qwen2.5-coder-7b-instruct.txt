To solve this problem efficiently, we need to determine how many empty squares exist that do not conflict with the movement pattern of a knight. The key insight here is to identify which squares a knight can attack from a given position and then count the number of squares that are not attacked by any of the existing knights.

Here's the corrected and optimized C++ code:

```cpp
#include <iostream>
#include <vector>
#include <set>
#include <utility>

using namespace std;

const vector<pair<int, int>> knight_moves = {
    {2, 1}, {1, 2}, {-1, 2}, {-2, 1},
    {-2, -1}, {-1, -2}, {1, -2}, {2, -1}
};

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    long long N;
    cin >> N;

    long long M;
    cin >> M;

    set<pair<long long, long long>> pieces;
    for (long long i = 0; i < M; ++i) {
        long long a, b;
        cin >> a >> b;
        pieces.insert({a, b});
    }

    long long total_empty_squares = N * N;
    long long occupied_squares = 0;

    for (const auto& piece : pieces) {
        for (const auto& move : knight_moves) {
            long long new_x = piece.first + move.first;
            long long new_y = piece.second + move.second;
            if (new_x >= 1 && new_x <= N && new_y >= 1 && new_y <= N) {
                occupied_squares++;
            }
        }
    }

    // Subtract each piece itself as it is also counted in the moves
    occupied_squares -= M;

    out(total_empty_squares - occupied_squares);

    return 0;
}
```

### Explanation:
1. **Knight Moves**: We define the possible moves a knight can make using a predefined list of pairs.
2. **Input Handling**: We read the board size `N` and the number of pieces `M`. Then, we read the positions of the pieces into a set to ensure uniqueness and fast lookup.
3. **Counting Occupied Squares**: For each piece, we calculate the squares it can attack by iterating over its possible moves. We check if these moves stay within the bounds of the board.
4. **Adjusting for Overcounting**: Each piece is counted multiple times (once for each move), so we subtract `M` from the total count of occupied squares.
5. **Output**: Finally, we output the total number of empty squares minus the number of occupied squares.

This approach ensures that we correctly count the number of valid empty squares in \(O(M \times 8)\) time complexity, which is efficient enough given the constraints.
