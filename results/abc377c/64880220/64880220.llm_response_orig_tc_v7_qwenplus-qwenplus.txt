To solve this problem efficiently given the constraints (`N` up to 10⁹ and `M` up to 2×10⁵), we must avoid brute-force grid traversal.

### Key Observations:
1. Each placed piece can threaten **up to 8** specific positions (like a knight's move in chess).
2. We need to:
   - Track all positions that are occupied or threatened.
   - Subtract these from the total number of cells `N*N`.

### Strategy:
- Use a `std::unordered_set` (or `std::set`) to store blocked positions (occupied + threatened).
- For each input piece, add its own position and all 8 possible threatening positions to the set if they're within bounds.
- At the end, subtract the size of the set from `N * N` to get the number of valid positions.

This approach ensures:
- Time complexity: O(M)
- Space complexity: O(M)

Here is the corrected and optimized code:

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);

    long long N;
    int M;
    cin >> N >> M;

    unordered_set<long long> blocked;
    // 8 directions a piece can attack (knight-style moves)
    vector<pair<int, int>> dirs = {
        {2, 1}, {1, 2}, {-1, 2}, {-2, 1},
        {-2, -1}, {-1, -2}, {1, -2}, {2, -1}
    };

    for (int i = 0; i < M; ++i) {
        int x, y;
        cin >> x >> y;

        // Block current position
        blocked.insert((long long)x * N + y);

        // Block all threatened positions
        for (auto [dx, dy] : dirs) {
            int nx = x + dx;
            int ny = y + dy;
            if (nx >= 1 && nx <= N && ny >= 1 && ny <= N) {
                blocked.insert((long long)nx * N + ny);
            }
        }
    }

    cout << N * N - (long long)blocked.size() << '\n';
}
```

### Explanation:
- We use `(long long)x * N + y` as a unique key for each cell. This avoids using pairs in the hash set which could be slower.
- The maximum value of `x * N + y` will be less than `10^18`, which fits comfortably into a 64-bit integer.
