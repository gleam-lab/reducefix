To solve this problem efficiently, we need to determine how many empty squares can be occupied without being captured by any existing pieces. The key observation here is that each piece can capture other pieces based on specific relative positions. We need to identify all such positions and mark them accordingly.

Here's a step-by-step approach:

1. **Input Parsing**: Read the grid dimensions `N` and the number of pieces `M`. Then, read the positions of the pieces.

2. **Marking Capture Zones**: For each piece, mark the squares that it can capture. This involves checking all 8 possible directions around the piece.

3. **Counting Empty Squares**: Count the number of empty squares that are not marked as capture zones.

Here's the corrected and optimized C++ code:

```cpp
#include <iostream>
#include <set>
#include <unordered_set>
#include <algorithm>
using namespace std;

int main() {
    long long N, M;
    cin >> N >> M;
    
    // Set to store positions of pieces
    unordered_set<long long> pieces;
    
    // Directions a piece can move to capture another piece
    int dir[8][2] = {{2, 1}, {1, 2}, {-1, 2}, {-2, 1}, {-2, -1}, {-1, -2}, {1, -2}, {2, -1}};
    
    // Mark all squares that can be captured by any piece
    for (long long i = 0; i < M; ++i) {
        long long x, y;
        cin >> x >> y;
        pieces.insert((x - 1) * N + y);
        
        for (auto &d : dir) {
            long long nx = x + d[0];
            long long ny = y + d[1];
            
            if (nx > 0 && nx <= N && ny > 0 && ny <= N) {
                pieces.insert((nx - 1) * N + ny);
            }
        }
    }
    
    // Total number of empty squares
    long long total_squares = N * N;
    long long empty_squares = total_squares - pieces.size();
    
    cout << empty_squares << endl;
    
    return 0;
}
```

### Explanation:

1. **Input Reading**:
   - We read `N` and `M`.
   - We use an unordered set `pieces` to keep track of the positions of all pieces.

2. **Direction Array**:
   - We define an array `dir` containing the relative positions that a piece can move to capture another piece.

3. **Marking Capture Zones**:
   - For each piece, we add its position to the `pieces` set.
   - We then iterate over the 8 possible directions and check if the new position `(nx, ny)` is within bounds. If it is, we add it to the `pieces` set.

4. **Counting Empty Squares**:
   - We calculate the total number of squares (`total_squares`).
   - We subtract the size of the `pieces` set (which represents the number of squares that can be captured) from `total_squares` to get the number of empty squares (`empty_squares`).

This solution ensures that we correctly identify all squares that can be captured and count the remaining empty squares efficiently. The time complexity is O(M), which is efficient enough given the constraints.
