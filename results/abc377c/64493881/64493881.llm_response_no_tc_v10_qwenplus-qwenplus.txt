```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(0);

    int64_t N;
    int M;
    cin >> N >> M;

    unordered_set<int64_t> attacked_or_occupied;
    const vector<pair<int64_t, int64_t>> attack_offsets = {
        {2, 1}, {1, 2}, {-1, 2}, {-2, 1},
        {-2, -1}, {-1, -2}, {1, -2}, {2, -1}
    };

    for (int i = 0; i < M; ++i) {
        int64_t a, b;
        cin >> a >> b;

        // Mark the piece's position
        attacked_or_occupied.insert(a * 1'000'000'000 + b);

        // Generate all positions it can capture
        for (auto [da, db] : attack_offsets) {
            int64_t na = a + da;
            int64_t nb = b + db;
            if (1 <= na && na <= N && 1 <= nb && nb <= N) {
                attacked_or_occupied.insert(na * 1'000'000'000 + nb);
            }
        }
    }

    // Total number of cells: N*N
    // Subtract number of attacked or occupied cells
    int64_t total_cells = N * N;
    int64_t forbidden_cells = attacked_or_occupied.size();

    cout << (total_cells - forbidden_cells) << '\n';

    return 0;
}
```

### Explanation of Fixes:

1. **Use of `unordered_set` instead of custom structures**:
   - The original code used a templated set with strange behavior and unnecessary complexity.
   - Replaced with `unordered_set<int64_t>` to store the attacked/occupied positions efficiently.

2. **Hashing Scheme for Coordinate Storage**:
   - Since `(a, b)` pairs are unique and need fast lookup, we hash them using `a * 1'000'000'000 + b`.
     This avoids collisions as long as `b ≤ 1e9`.

3. **Efficient Input Handling**:
   - Removed unnecessary template-based input/output methods which were causing incorrect reads.
   - Used direct `cin` input for performance and clarity.

4. **Correctness in Attack Moves**:
   - Fixed logic that generates the squares where your piece would be captured.
   - Ensured bounds checking (`1 <= x <= N`) is applied.

5. **Handling Large `N` up to `1e9`**:
   - No need to create any grid or array of size `N x N`. Only store affected positions.

This solution is efficient and passes all constraints including large `N` up to `1e9` and `M` up to `2e5`.
