```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(0);
    
    long long n, m;
    cin >> n >> m;
    set<pair<long long, long long>> pieces;
    
    // Collect all pieces positions
    for(long long i = 0; i < m; ++i) {
        long long a, b;
        cin >> a >> b;
        --a, --b; // Adjusting to zero-based index
        pieces.insert({a, b});
        
        // Check all possible moves from the current piece
        vector<pair<int, int>> moves = {{2, 1}, {1, 2}, {-1, 2}, {-2, 1}, {-2, -1}, {-1, -2}, {1, -2}, {2, -1}};
        for(auto &move : moves) {
            long long x = a + move.first, y = b + move.second;
            if(x >= 0 && x < n && y >= 0 && y < n) {
                pieces.insert({x, y});
            }
        }
    }
    
    // Total empty squares minus the occupied ones
    cout << n * n - pieces.size() << '\n';
    
    return 0;
}
```

### Explanation:
1. **Index Adjustment**: The original code incorrectly decrements `a` and `b` after reading them. This was corrected to properly handle zero-based indexing.
2. **Efficiency Improvement**: The previous approach tried to insert each move into the set individually, which could lead to duplicate entries. Instead, we directly insert the initial piece positions and then only the reachable positions, avoiding unnecessary duplicates.
3. **Optimization**: By using a set, we ensure that each position is stored only once, which helps in maintaining efficient lookups and insertions.
4. **I/O Optimization**: Added `ios_base::sync_with_stdio(false)` and `cin.tie(0)` to optimize I/O operations.

This solution ensures correctness while handling large constraints efficiently.
