### Approaches
1. **Marking Unsafe Positions**: For each existing piece, we need to mark all positions that can be attacked by it. These positions are determined by the knight's move in chess (all 8 possible L-shaped moves). 
2. **Handling Boundaries**: Ensure that the marked attack positions do not go outside the grid boundaries (1 ≤ i, j ≤ N).
3. **Counting Safe Positions**: The safe positions are the total grid squares (N²) minus the number of positions that are either occupied by existing pieces or can be attacked by any existing piece.

### Solution Code
```cpp
#include <bits/stdc++.h>
using namespace std;
using ll = long long;

int main() {
    ll N, M;
    cin >> N >> M;
    set<pair<ll, ll>> unsafe;
    vector<pair<ll, ll>> pieces(M);

    for (int i = 0; i < M; ++i) {
        ll a, b;
        cin >> a >> b;
        pieces[i] = {a, b};
        unsafe.insert({a, b});
    }

    int dx[] = {2, 1, -1, -2, -2, -1, 1, 2};
    int dy[] = {1, 2, 2, 1, -1, -2, -2, -1};

    for (auto [a, b] : pieces) {
        for (int k = 0; k < 8; ++k) {
            ll ni = a + dx[k];
            ll nj = b + dy[k];
            if (ni >= 1 && ni <= N && nj >= 1 && nj <= N) {
                unsafe.insert({ni, nj});
            }
        }
    }

    cout << N * N - unsafe.size() << endl;
    return 0;
}
```

### Explanation
1. **Input Handling**: The code reads the grid size `N` and the number of existing pieces `M`. It then reads the positions of these pieces.
2. **Unsafe Positions**: The existing pieces' positions are stored in a set `unsafe` to ensure uniqueness.
3. **Knight Moves Calculation**: For each existing piece, the code checks all 8 possible knight moves. For each move, if the resulting position is within the grid bounds (1 ≤ i, j ≤ N), it is added to the `unsafe` set.
4. **Safe Positions Calculation**: The total number of safe positions is calculated by subtracting the size of the `unsafe` set (which includes both existing pieces and their attackable positions) from the total grid squares (N²).

This approach efficiently marks all unsafe positions and calculates the safe positions by leveraging set operations to avoid duplicates, ensuring correctness and optimal performance.
