### Approaches
1. **Problem Analysis**: The problem requires counting the number of squares on an N x N chessboard where a new piece can be placed without being captured by any existing pieces. The existing pieces can capture pieces placed in specific positions relative to them (like a knight in chess).
2. **Key Insight**: For each existing piece, the forbidden positions are the 8 possible knight moves from that piece. We need to collect all such forbidden positions, ensure they are within the bounds of the chessboard, and then count them once.
3. **Optimization**: Using a set to store the forbidden positions ensures that duplicates are automatically handled, and we only count each forbidden position once. The total valid positions is then \( N^2 - \text{(number of forbidden positions)} - M \), where M is the number of existing pieces (since those squares are already occupied and shouldn't be counted).
4. **Handling Large Values**: Given that N can be as large as \( 10^9 \), we need to ensure that our calculations handle such large values without overflow. Using `long long` for calculations is essential.

### Solution Code

```cpp
#include <bits/stdc++.h>
using namespace std;

#define ll long long

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(NULL);

    ll N; int M;
    cin >> N >> M;

    set<pair<ll, ll>> forbidden;
    vector<pair<int, int>> moves = {
        {2, 1}, {1, 2}, {-1, 2}, {-2, 1},
        {-2, -1}, {-1, -2}, {1, -2}, {2, -1}
    };

    for (int i = 0; i < M; ++i) {
        ll a, b;
        cin >> a >> b;
        a--; b--;

        for (auto [dx, dy] : moves) {
            ll x = a + dx;
            ll y = b + dy;
            if (x >= 0 && x < N && y >= 0 && y < N) {
                forbidden.insert({x, y});
            }
        }
    }

    ll total = N * N;
    ll forbidden_count = forbidden.size();
    ll answer = total - forbidden_count - M;

    // Check if any existing pieces are in forbidden positions
    // Since existing pieces are already occupied, they are not in forbidden set
    // So, answer is total squares - (forbidden squares + existing squares)
    // Existing squares are already not in forbidden set as per problem statement
    cout << answer << endl;

    return 0;
}
```

### Explanation
1. **Initialization**: The code initializes the size of the chessboard (N) and the number of existing pieces (M). It also prepares a set to store forbidden positions and a vector of possible knight moves.
2. **Processing Each Piece**: For each existing piece, the code calculates all 8 possible knight moves from its position. Each calculated position is checked to ensure it lies within the bounds of the chessboard before being added to the set of forbidden positions.
3. **Calculating Valid Positions**: The total number of squares on the chessboard is \( N^2 \). The number of forbidden positions is the size of the set, which automatically handles duplicates. The answer is computed as \( \text{total squares} - \text{forbidden positions} - \text{existing pieces} \), ensuring that already occupied squares are not counted.
4. **Output**: The result is printed, which is the count of squares where the new piece can be placed safely.

This approach efficiently handles the constraints and ensures correct results even for very large values of N by leveraging set operations to avoid duplicate forbidden positions and careful arithmetic to prevent overflow.
