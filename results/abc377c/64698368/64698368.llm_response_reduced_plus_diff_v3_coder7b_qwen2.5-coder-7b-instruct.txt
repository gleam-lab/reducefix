To solve this problem, we need to determine which empty squares on the board can be occupied without being captured by any existing pieces. Each piece can capture pieces located at specific relative positions:

- (2, 1)
- (1, 2)
- (-1, 2)
- (-2, 1)
- (-2, -1)
- (-1, -2)
- (1, -2)
- (2, -1)

Given these constraints, we can iterate through each square on the board and check if placing a piece there would result in it being captured. If not, we count it as a valid position.

Here's the corrected C++ code that addresses all the issues mentioned:

```cpp
#include <iostream>
#include <vector>
#include <set>
#include <tuple>

using namespace std;

int main() {
    long long N, M;
    cin >> N >> M;

    // Directions in which a piece can capture another piece
    vector<array<long long, 2>> directions = {
        {2, 1}, {1, 2}, {-1, 2}, {-2, 1},
        {-2, -1}, {-1, -2}, {1, -2}, {2, -1}
    };

    // Set to store the positions of the pieces
    set<array<long long, 2>> pieces;

    // Read the positions of the pieces
    for (long long i = 0; i < M; ++i) {
        long long x, y;
        cin >> x >> y;
        --x, --y; // Convert 1-based index to 0-based index
        pieces.insert({x, y});
    }

    // Count the number of valid positions
    long long valid_positions = 0;
    for (long long i = 0; i < N; ++i) {
        for (long long j = 0; j < N; ++j) {
            bool is_valid = true;
            for (auto& dir : directions) {
                long long nx = i + dir[0];
                long long ny = j + dir[1];
                if (nx >= 0 && nx < N && ny >= 0 && ny < N) {
                    if (pieces.count({nx, ny})) {
                        is_valid = false;
                        break;
                    }
                }
            }
            if (is_valid) {
                ++valid_positions;
            }
        }
    }

    cout << valid_positions << endl;

    return 0;
}
```

### Explanation:
1. **Reading Input**: The program reads the dimensions `N` and `M`, followed by the positions of `M` pieces.
2. **Direction Array**: An array of pairs representing the relative positions where a piece can capture another piece.
3. **Piece Positions**: A set to store the positions of the pieces for quick lookup.
4. **Validation Loop**: For each square `(i, j)` on the board, it checks if placing a piece there would result in it being captured by any existing pieces. It iterates over the possible capture directions and checks if any of those positions contain a piece.
5. **Counting Valid Positions**: If a square is not captured by any piece, it increments the count of valid positions.
6. **Output**: Finally, the program outputs the total number of valid positions.

This approach ensures that all constraints are handled correctly and efficiently. The use of a set for storing piece positions allows for O(log N) insertions and lookups, making the solution scalable even for large inputs.
