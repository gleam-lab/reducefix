3. **Handling Large Values**: The total number of squares is N^2, which can be up to 1e18, so we need to use `long long` to avoid overflow when calculating the result.

```cpp
#include <bits/stdc++.h>
using namespace std;

typedef long long ll;

int main() {
    int N, M;
    cin >> N >> M;

    set<pair<int, int>> attacked;
    vector<pair<int, int>> moves = {{2, 1}, {1, 2}, {-1, 2}, {-2, 1}, {-2, -1}, {-1, -2}, {1, -2}, {2, -1}};

    for (int i = 0; i < M; i++) {
        int a, b;
        cin >> a >> b;
        a--; b--; // Convert to 0-based index
        for (auto [dx, dy] : moves) {
            int nx = a + dx;
            int ny = b + dy;
            if (nx >= 0 && nx < N && ny >= 0 && ny < N) {
                attacked.insert({nx, ny});
            }
        }
    }

    ll total_squares = (ll)N * N;
    ll safe_squares = total_squares - attacked.size() - M; // Subtract M because the pieces themselves are occupied
    cout << safe_squares << endl;
    return 0;
}
```

### Key Fixes:
1. **Removed `{0,0}` from moves**: This ensures we only consider the actual attack positions around each piece.
2. **Used `long long` for total squares**: To handle the large value of N^2 which can be up to 1e18.
3. **Subtracted M from safe squares**: The positions where the existing pieces are placed are already occupied and cannot be used, so they should not be counted as safe squares.
4. **Efficient use of set**: The set `attacked` stores all unique positions under attack, ensuring no double counting.

This code now correctly computes the number of safe squares where a new piece can be placed without being captured by any existing pieces.
