### Approaches

1. **Problem Analysis**: The task is to count the number of empty squares on an N x N grid where a new piece can be placed without being captured by any of the M existing pieces. Each existing piece can capture pieces in up to 8 specific positions relative to it (like a knight in chess).

2. **Key Insight**: For each existing piece, we need to mark its position and the 8 positions it can capture. The total number of unsafe squares is the union of all these marked squares. The answer is then the total number of squares (N x N) minus the number of unsafe squares, adjusted for overlaps.

3. **Optimization**: Given the constraints (N can be up to 1e9 and M up to 2e5), we cannot process each square individually. Instead, we use a hash set to keep track of all unsafe squares (both the existing pieces and their attack positions) to efficiently handle the large grid size.

### Solution Code

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    long long N, M;
    cin >> N >> M;
    unordered_set<long long> unsafe;
    vector<int> dx = {2, 2, -2, -2, 1, 1, -1, -1};
    vector<int> dy = {1, -1, 1, -1, 2, -2, 2, -2};

    for (int i = 0; i < M; ++i) {
        long long a, b;
        cin >> a >> b;
        unsafe.insert((a - 1) * N + (b - 1)); // Using 0-based index for easier calculation

        for (int j = 0; j < 8; ++j) {
            long long x = a + dx[j];
            long long y = b + dy[j];
            if (x >= 1 && x <= N && y >= 1 && y <= N) {
                unsafe.insert((x - 1) * N + (y - 1));
            }
        }
    }

    cout << N * N - unsafe.size() << endl;
    return 0;
}
```

### Explanation

1. **Input Handling**: The code reads the grid size N and the number of existing pieces M.
2. **Tracking Unsafe Squares**: An unordered set `unsafe` is used to store all squares that are either occupied by existing pieces or can be captured by them.
3. **Checking Capture Positions**: For each existing piece at (a, b), the code checks all 8 possible positions it can capture. Each valid position (within the grid bounds) is added to the `unsafe` set.
4. **Result Calculation**: The total safe squares is the total number of squares (N x N) minus the number of unsafe squares in the set.

This approach efficiently handles the large grid size by using a hash set to avoid duplicate entries and ensures correct results by considering all possible unsafe squares. The solution is optimized to run within the problem constraints, providing the correct count of safe squares for placement.
