To solve this problem, we need to determine how many empty squares on an \(N \times N\) grid can be placed on without being captured by any existing pieces. Each piece can capture another piece if they are positioned according to specific rules.

The key observation here is that each piece can capture up to 8 other pieces based on the given rules. Therefore, each piece effectively "blocks" a set of positions around itself. To find the maximum number of empty squares available, we need to count these blocked positions and subtract them from the total number of squares.

Here's a step-by-step approach to solving the problem:

1. **Initialize Data Structures**: We'll use a set to keep track of all positions that are blocked by existing pieces.
2. **Iterate Over Existing Pieces**: For each piece, calculate the positions it can capture and add them to the set of blocked positions.
3. **Count Unblocked Positions**: Subtract the number of blocked positions from the total number of squares to get the number of unblocked positions.

Let's implement this in C++:

```cpp
#include <bits/stdc++.h>
using namespace std;

const int MAXN = 1e9 + 10;
const int DIRS[8][2] = {{2, 1}, {1, 2}, {-1, 2}, {-2, 1}, {-2, -1}, {-1, -2}, {1, -2}, {2, -1}};

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    long long n, m;
    cin >> n >> m;

    unordered_set<long long> blocked;
    for (long long i = 0; i < m; ++i) {
        long long a, b;
        cin >> a >> b;
        long long idx = (a - 1) * n + b;
        
        // Check all directions
        for (int d = 0; d < 8; ++d) {
            int dx = DIRS[d][0], dy = DIRS[d][1];
            long long x = a + dx, y = b + dy;
            if (x >= 1 && x <= n && y >= 1 && y <= n) {
                blocked.insert((x - 1) * n + y);
            }
        }
    }

    // Calculate the number of unblocked positions
    long long total_squares = n * n;
    long long unblocked_squares = total_squares - blocked.size();

    cout << unblocked_squares << endl;

    return 0;
}
```

### Explanation:

1. **Initialization**:
   - `MAXN` is a large constant to handle the upper limit of \(N\).
   - `DIRS` array contains the relative positions that a piece can capture.
   - `blocked` is a set to store the indices of blocked positions.

2. **Reading Input**:
   - Read \(N\) and \(M\).
   - For each piece, read its position \((a, b)\).

3. **Marking Blocked Positions**:
   - Convert the 2D position \((a, b)\) to a 1D index using `(a - 1) * n + b`.
   - For each direction defined in `DIRS`, calculate the new position \((x, y)\).
   - If the new position is within bounds, mark it as blocked by adding its index to the `blocked` set.

4. **Calculating Unblocked Positions**:
   - The total number of squares is \(N \times N\).
   - Subtract the size of the `blocked` set from the total number of squares to get the number of unblocked positions.
   - Print the result.

This solution ensures that we correctly identify and count all positions that are blocked by existing pieces, allowing us to determine the number of valid positions for placing new pieces.
