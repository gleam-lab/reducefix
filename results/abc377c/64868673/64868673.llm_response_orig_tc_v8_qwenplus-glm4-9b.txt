
Given the problem, we need to determine the number of empty squares on a grid of size \(N \times N\) where a new piece can be placed without being captured by existing pieces. The capturing conditions are based on the L-shape directions. We can solve this problem efficiently by marking the cells that are under threat due to each captured piece and then subtracting these from the total number of cells.

### Solution Steps:

1. Initialize a grid `ex` of size \(N \times N\) to represent the grid where each cell is initially empty.
2. For each captured piece, mark the cells that can be captured by this piece in the grid `ex` and count them.
3. The cells which are marked as captured for at least one piece are the cells where the new piece cannot be placed.
4. The answer will be the total number of cells minus the number of cells that are marked as captured by existing pieces.

To implement this, we can use a 2D difference array technique to efficiently mark the cells in a grid.

### Implementation:

```cpp
#include <iostream>
#include <vector>

using namespace std;

int main() {
    long long n, m;
    cin >> n >> m;

    vector<vector<int>> grid(n, vector<int>(n, 0));
    long long threat_count = 0;

    // Directions for capturing pieces
    vector<pair<int, int>> directions = {
        {2, 1}, {1, 2}, {-1, 2}, {-2, 1}, {-2, -1}, {-1, -2}, {1, -2}, {2, -1}
    };

    for (long long i = 0; i < m; ++i) {
        long long a, b;
        cin >> a >> b;
        a--; b--;  // Convert to 0-based index

        // For each piece, mark the cells it can capture
        for (const auto& dir : directions) {
            long long x = a + dir.first;
            long long y = b + dir.second;
            // Ensure the coordinates are within bounds
            if (x >= 0 && x < n && y >= 0 && y < n) {
                grid[x][y] = 1; // Mark as threatened
                threat_count++;
            }
        }
    }

    // Calculate the number of empty cells not under threat
    long long result = n * n - threat_count;
    cout << result << endl;

    return 0;
}
```

This solution efficiently marks the grid with the cells that are threatened by the existing pieces and calculates the number of non-threatened cells, which is the answer.
