
The problem requires finding the minimum value of the expression \((\max_{i \in S} A_i) \times (\sum_{i \in S} B_i)\) for a subset \(S\) of size \(K\) from sequences \(A\) and \(B\).

Initial observations:
- The maximum element in \(S\) should be as small as possible to minimize the expression.
- The sum \( \sum_{i \in S} B_i \) should be as large as possible for a fixed maximum element.

To achieve this:
1. Sort \(A\) in ascending order. This allows us to control the maximum element by simply taking the first \(K\) elements.
2. Sort \(B\) in descending order. This maximizes the sum \( \sum_{i \in S} B_i \) for the smallest maximum element in \(S\).

The solution can be broken down as follows:
1. For each test case, read \(N\) and \(K\), and then read sequences \(A\) and \(B\).
2. Create an array \(A'\) by sorting \(A\) in ascending order and an array \(B'\) by sorting \(B\) in descending order.
3. Iterate over the first \(K\) elements of \(A'\) and compute the product of the corresponding elements in \(B'\).
4. The minimum product across these iterations gives the answer for the test case.

However, the given solution uses a priority queue and a stack to compute the answer, which is not necessary. Here's how to simplify the solution:

```cpp
#include <iostream>
#include <vector>
#include <algorithm>

using namespace std;

int main() {
    int T;
    cin >> T;
    while (T--) {
        int N, K;
        cin >> N >> K;
        vector<int> A(N), B(N);
        for (int i = 0; i < N; i++) cin >> A[i];
        for (int i = 0; i < N; i++) cin >> B[i];

        sort(A.begin(), A.end()); // Sort A ascending
        sort(B.begin(), B.end(), greater<int>()); // Sort B descending

        long long ans = LLONG_MAX;
        for (int i = 0; i < K; i++) {
            ans = min(ans, (long long)A[i] * B[i]);
        }

        cout << ans << endl;
    }
    return 0;
}
```

This solution sorts \(A\) and \(B\) as mentioned and then simply computes the \(K\)-smallest products of corresponding elements from \(A\) and \(B\). This approach is more intuitive and avoids the overhead of the priority queue and stack, providing an efficient solution to the problem.
