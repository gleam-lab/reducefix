To solve this problem efficiently, we need to minimize the value of:

$$
\left(\max_{i \in S} A_i\right) \times \left(\sum_{i \in S} B_i\right)
$$

Where $ S $ is a subset of size $ K $ from indices $ 1 $ to $ N $. Given that both $ N $ and $ T $ can be large (up to $ 2 \times 10^5 $), we must ensure our solution runs in acceptable time.

---

### **Key Observations:**

- We can sort all elements by their `A[i]` values. This way, for any index `i`, all previous indices have smaller or equal `A[i]`.
- For each possible maximum element in the subset (sorted by A), we want to pick the top `K` elements with the smallest `A[i]` but largest `B[i]` values.
- To maintain the sum of top `K` B values efficiently as we iterate through the sorted list, we use a **min-heap**:
  - Add `B[i]` to the heap.
  - If the heap has more than `K` elements, remove the smallest one.
  - At each step, compute the product of current max `A[i]` and the sum of kept `B[i]`.

This approach ensures an efficient solution with complexity $ O(N \log K) $ per test case.

---

### ✅ Fixed C++ Code:
```cpp
#include <iostream>
#include <vector>
#include <queue>
#include <algorithm>

using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int T;
    cin >> T;

    while (T--) {
        int N, K;
        cin >> N >> K;

        vector<pair<int, int>> A(N);
        for (int i = 0; i < N; ++i) {
            cin >> A[i].first;
            A[i].second = i;
        }

        vector<int> B(N);
        for (int i = 0; i < N; ++i) {
            cin >> B[i];
        }

        // Sort by A[i]
        sort(A.begin(), A.end());

        long long sum_b = 0;
        priority_queue<int, vector<int>, greater<>> min_heap;
        long long result = 1e18;  // Large initial value

        for (int i = 0; i < N; ++i) {
            int a_val = A[i].first;
            int b_val = B[A[i].second];

            min_heap.push(b_val);
            sum_b += b_val;

            if (min_heap.size() > K) {
                sum_b -= min_heap.top();
                min_heap.pop();
            }

            if (min_heap.size() == K) {
                result = min(result, sum_b * a_val);
            }
        }

        cout << result << '\n';
    }

    return 0;
}
```

---

### 💡 Explanation:

- We sort the pairs `(A[i], i)` so we process elements in increasing order of `A[i]`.
- As we go through each element, we add its `B[i]` to a min-heap to keep track of the `K` largest `B[i]` values seen so far.
- If the heap grows beyond size `K`, we remove the smallest `B[i]` to keep only the top `K` ones.
- Whenever the heap has exactly `K` elements, we calculate the product `current_A * sum_of_B` and update the minimum result.

---

### ⚙️ Time Complexity:

- Sorting takes $ O(N \log N) $
- Each heap operation is $ O(\log K) $
- Total per test case: $ O(N \log N + N \log K) $
- Overall across all test cases: $ O(\text{sum over all test cases}(N \log N)) $, which is efficient given constraints.
