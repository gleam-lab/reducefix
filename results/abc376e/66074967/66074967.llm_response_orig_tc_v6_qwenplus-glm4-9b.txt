
### Problem Analysis

Given two sequences A and B of length N, and a number K, the task is to find a subset S of size K that minimizes the expression:

\[ \left(\max_{i \in S} A_i\right) \times \left(\sum_{i \in S} B_i\right) \]

This problem can be approached by considering the relationship between the maximum value in the subset and the sum of the corresponding B values.

### Solution Strategy

1. **Sorting and Indexing**: Start by sorting the sequence A in descending order while keeping track of the original indices. This helps in quickly determining the maximum value for any subset size K.
2. **Prefix Sum Array**: Construct a prefix sum array of sequence B. This allows quick calculation of the sum of any subsequence.
3. **Sliding Window Technique**: Use a sliding window approach to maintain the sum of the smallest K elements in sequence B (since we are interested in minimizing the product). This is crucial for efficient computation as the window slides over the sequence.
4. **Maximizing the Product**: For each possible maximum value in A (considered as the maximum of the subset S), compute the product with the sum of the K smallest values of B. Track the minimum of these products.

### Optimized Code

```cpp
#include <iostream>
#include <vector>
#include <algorithm>
using namespace std;

const int N = 200005;

int n, k;
int a[N];
int b[N];
int b_sum[N];
int a_index[N];

void solve() {
    cin >> n >> k;
    long long sum_k_elements = 0;
    priority_queue<int, vector<int>, greater<int>> q;

    for (int i = 0; i < n; ++i) {
        cin >> a[i];
        a_index[i] = i;
        cin >> b[i];
        b_sum[i] = b[i];
    }

    // Sort indices based on A values in descending order
    sort(a_index, a_index + n, [&](int i, int j) {
        return a[i] > a[j];
    });

    // Calculate initial sum of the first k elements in B
    for (int i = 0; i < k; ++i) {
        q.push(b[a_index[i]]);
        sum_k_elements += b[a_index[i]];
    }

    long long min_product = (long long)sum_k_elements * a[a_index[0]];

    // Sliding window over the sorted indices
    for (int i = k; i < n; ++i) {
        sum_k_elements -= q.top();
        q.pop();
        sum_k_elements += b[a_index[i]];
        q.push(b[a_index[i]]);
        min_product = min(min_product, (long long)sum_k_elements * a[a_index[i]]);
    }

    cout << min_product << endl;
}

int main() {
    int T;
    cin >> T;
    while (T--) {
        solve();
    }
    return 0;
}
```

This solution efficiently computes the minimum product for each test case using sorting, a priority queue for maintaining the sum of the smallest K B values, and a sliding window technique. This approach ensures that the solution is optimal and can handle the upper limits of the problem constraints effectively.
