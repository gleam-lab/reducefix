To solve this problem efficiently, we need to minimize the expression:

$$
\left(\max_{i \in S} A_i\right) \times \left(\sum_{i \in S} B_i\right)
$$

for subsets $ S \subseteq \{1, 2, ..., N\} $ of size exactly $ K $.

### Key Observations:
1. We are given multiple test cases.
2. For each test case, we need to choose a subset $ S $ of size $ K $ such that the product of the maximum A-value in the subset and the sum of corresponding B-values is minimized.
3. Brute-force checking all combinations would be too slow due to large $ N $ (up to 200,000).
4. **Strategy**:
   - Sort elements by `A[i]`.
   - As we move through sorted `A`, maintain a max-heap of `B` values for the current best `K` elements.
   - At each step, keep track of the top `K` smallest `A` values and compute the minimal value of `max_A * sum_B`.

### Efficient Approach:
1. Sort all pairs `(A[i], B[i])` by `A[i]`.
2. Use a min-heap to keep track of the `K` largest `B[i]` values among the first `i` elements.
3. Maintain a running total of those `K` `B[i]` values.
4. For each `i` from `K` to `N`, consider using `A[i]` as the max value and the sum of the `K` smallest `B[i]` values so far.
5. Update the answer accordingly.

Here's the corrected and efficient implementation:

```cpp
#include <iostream>
#include <vector>
#include <queue>
#include <algorithm>

using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(0);

    int T;
    cin >> T;

    while (T--) {
        int N, K;
        cin >> N >> K;

        vector<pair<int, int>> A(N);
        for (int i = 0; i < N; ++i) {
            cin >> A[i].first;
        }
        for (int i = 0; i < N; ++i) {
            cin >> A[i].second;
        }

        // Sort by A value
        sort(A.begin(), A.end());

        // Min heap to store B values of size up to K
        priority_queue<int, vector<int>, greater<int>> pq;
        long long sum_b = 0;
        long long result = 1e18; // Large initial value

        // Initialize with first K-1 B values (we will add one more at each step)
        for (int i = 0; i < K - 1; ++i) {
            sum_b += A[i].second;
            pq.push(A[i].second);
        }

        // Now iterate from index K-1 to N-1
        for (int i = K - 1; i < N; ++i) {
            sum_b += A[i].second;
            pq.push(A[i].second);

            // If heap size exceeds K, remove the smallest element
            if (pq.size() > K) {
                sum_b -= pq.top();
                pq.pop();
            }

            // Only calculate when we have exactly K elements
            if (pq.size() == K) {
                result = min(result, 1LL * A[i].first * sum_b);
            }
        }

        cout << result << "\n";
    }

    return 0;
}
```

### Explanation:
- Sorting ensures that we always use the current `A[i]` as the maximum in the subset, since we process in increasing order of `A`.
- The min-heap ensures we always pick the `K` smallest `B` values among the first `i` elements.
- This gives an efficient solution with time complexity **O(N log K)** per test case.

This code handles edge cases correctly (like K=1), avoids overflow issues by using `long long`, and works within constraints efficiently.
