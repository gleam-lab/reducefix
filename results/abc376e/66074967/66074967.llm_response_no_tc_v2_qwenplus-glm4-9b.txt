
```cpp
#include <iostream>
#include <vector>
#include <algorithm>
using namespace std;

const int MAX_N = 2e5 + 9;

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);
    
    int T;
    cin >> T;
    
    while (T--) {
        int N, K;
        cin >> N >> K;
        
        vector<int> A(N), B(N);
        
        for (int i = 0; i < N; ++i) {
            cin >> A[i];
        }
        
        for (int i = 0; i < N; ++i) {
            cin >> B[i];
        }
        
        // Pair A and B values and sort by A descending
        vector<pair<int, int>> AB(N);
        for (int i = 0; i < N; ++i) {
            AB[i] = {A[i], B[i]};
        }
        sort(AB.begin(), AB.end(), [](const pair<int, int>& a, const pair<int, int>& b) {
            return a.first > b.first;
        });
        
        long long sum = 0, maxB = 0, result = LLONG_MAX;
        
        // Iterate over sorted A to find the minimum result
        for (int i = 0; i < N; ++i) {
            if (i >= K - 1) { // Ensure we only consider K elements
                sum += AB[i].second;
                maxB = max(maxB, AB[i].second);
            }
            
            if (i >= K - 1) { // Once we have considered K elements, start calculating results
                result = min(result, maxB * sum);
            }
        }
        
        cout << result << '\n';
    }
    
    return 0;
}
```

### Explanation:
1. **Input and Pairing**: The code starts by reading the number of test cases `T`. For each test case, it reads `N` and `K`, followed by `N` integers for `A` and `N` integers for `B`. The values of `A` and `B` are then paired together, sorted by the values in `A` in descending order.
2. **Calculation**: We then iterate over the sorted pairs and maintain a running sum of the `B` values and the maximum `B` value among the last `K` elements. We calculate the result by taking the minimum of the current result and the product of the maximum `B` and the sum of `B` values among the last `K` elements.
3. **Output**: After processing all test cases, the result for each is printed.

This approach ensures that we are considering only the last `K` elements of `A` in an efficient manner and that the calculations are done in a straightforward manner, leading to a correct solution.
