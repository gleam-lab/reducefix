To solve this problem efficiently, we need to find the minimum value of the expression:

$$
\left(\max_{i \in S} A_i\right) \times \left(\sum_{i \in S} B_i\right)
$$

for a subset `S` of size `K` from the given arrays `A` and `B`.

---

### **Understanding the Problem**
- We're given two arrays: `A` (positive integers), and `B` (positive integers).
- We are to choose exactly `K` elements such that the product of:
  - the **maximum** `A[i]` among those selected, and
  - the **sum** of `B[i]` for those selected
  is **minimized**.

This is a classic optimization problem where brute force (`O(N^K)`) is not feasible due to constraints. So we need an efficient approach.

---

### **Key Insight**
If we fix the maximum `A[i] = x`, then we should select the `K-1` smallest `B[i]` values among all elements where `A[i] <= x`. This ensures the sum of `B` is minimized for a fixed max `A`.

So, the optimal strategy:
1. Sort the pairs `(A[i], B[i])` by `A[i]`.
2. Iterate through each element assuming it's the maximum `A[i]` in the subset.
3. Maintain a **max-heap** of size `K`, storing the `K` smallest `B[i]` values so far.
4. For each candidate `A[i]`, compute the current product `A[i] * sum(Bs)` and update the minimum.

---

### **Why Use a Max-Heap?**
We use a max-heap of size `K` to always maintain the **smallest K elements** of `B` (by removing the largest among them when needed). This allows us to keep track of the minimal sum efficiently.

---

### **Complexity**
- Sorting takes `O(N log N)`
- Each heap operation is `O(log K)`
- Total time per test case: `O(N log N + N log K)` which is acceptable within constraints.

---

Here is the **corrected C++ code**:

```cpp
#include <bits/stdc++.h>
using namespace std;

#define int long long
#define PII pair<int, int>

int INF = 1e18;

signed main() {
    ios::sync_with_stdio(0);
    cin.tie(0); cout.tie(0);

    int T;
    cin >> T;

    while (T--) {
        int N, K;
        cin >> N >> K;

        vector<PII> AB(N);
        for (int i = 0; i < N; ++i)
            cin >> AB[i].first;
        for (int i = 0; i < N; ++i)
            cin >> AB[i].second;

        // Sort by A[i]
        sort(AB.begin(), AB.end());

        priority_queue<int> pq; // max-heap to keep track of smallest K B[i]
        int sum_B = 0;
        int res = INF;

        for (int i = 0; i < N; ++i) {
            int curr_A = AB[i].first;
            int curr_B = AB[i].second;

            if (pq.size() >= K) {
                // If heap has more than K elements, remove the largest B
                sum_B -= pq.top();
                pq.pop();
            }

            pq.push(curr_B);
            sum_B += curr_B;

            if (pq.size() == K) {
                res = min(res, curr_A * sum_B);
            }
        }

        cout << res << endl;
    }

    return 0;
}
```

This solution:
- Sorts the array based on `A`
- Uses a max-heap to maintain the `K` smallest `B` values
- Efficiently computes the required minimum product
- Handles large inputs efficiently with total complexity `O(N log N + N log K)`
