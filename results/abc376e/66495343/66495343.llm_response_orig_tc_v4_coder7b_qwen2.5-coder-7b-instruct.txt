To solve this problem efficiently, we need to find the minimum value of the expression \((\max_{i \in S} A_i) \times (\sum_{i \in S} B_i)\) where \(S\) is a subset of \(\{1, 2, \ldots, N\}\) of size \(K\).

### Key Observations
1. **Maximize Sum**: To minimize the product, we want to maximize the sum of \(B_i\) while keeping the maximum value of \(A_i\) as small as possible.
2. **Greedy Approach**: We can use a greedy approach where we sort the arrays \(A\) and \(B\) based on their values and then select the top \(K\) elements from both arrays such that the sum of \(B_i\) is maximized and the maximum value of \(A_i\) is minimized.

### Steps
1. Sort the pairs \((A_i, B_i)\) based on \(A_i\) in ascending order.
2. Select the top \(K\) pairs from the sorted list.
3. Calculate the sum of the corresponding \(B_i\) values.
4. Multiply the maximum \(A_i\) value among these pairs with the sum of \(B_i\) values.

#include <bits/stdc++.h>
using namespace std;
#define int long long
#define PII pair<int, int>
#define fi first
#define se second
#define arr array<int, 3>

const int N = 2e5 + 10, INF = 0x3f3f3f3f, mod = 1e9 + 7;

int a[N], b[N];
PII sa[N];

signed main() {
    ios::sync_with_stdio(0);
    cin.tie(0), cout.tie(0);

    int t;
    cin >> t;

    while (t--) {
        int n, k;
        cin >> n >> k;

        for (int i = 1; i <= n; i++) 
            cin >> a[i];
        
        for (int i = 1; i <= n; i++) 
            cin >> b[i];
            
        for (int i = 1; i <= n; i++) 
            sa[i] = {a[i], b[i]};
        
        sort(sa + 1, sa + 1 + n);

        int res = INF, sum = 0;
        priority_queue<int> pq;

        for (int i = 1; i <= n; i++) {
            if (i <= k) {
                sum += sa[i].se;
                pq.push(-sa[i].se);
                continue;
            }

            int na = sa[i].fi;
            sum += sa[i].se;
            pq.push(-sa[i].se);

            if (pq.size() > k) {
                int big = -pq.top();
                pq.pop();
                sum -= big;
            }

            res = min(res, na * sum);
        }

        cout << res << endl;
    }

    return 0;
}

### Explanation
1. **Sorting**: We sort the pairs \((A_i, B_i)\) based on \(A_i\) in ascending order.
2. **Priority Queue**: We use a priority queue (min-heap) to keep track of the smallest sums of \(B_i\) values. This helps us maintain the correct sum of \(B_i\) values for subsets of size \(K\).
3. **Calculation**: For each element, we update the sum and the priority queue accordingly. If the size of the priority queue exceeds \(K\), we remove the largest sum to ensure we only consider the top \(K\) elements.
4. **Result**: Finally, we calculate the product of the maximum \(A_i\) value and the current sum of \(B_i\) values and take the minimum across all valid subsets.

This approach ensures that we efficiently find the minimum value of the given expression within the constraints.
