To solve this problem efficiently, we need to find the minimum value of the expression \((\max_{i \in S} A_i) \times (\sum_{i \in S} B_i)\) for a subset \(S\) of size \(K\). We can approach this using a combination of sorting and a greedy strategy.

### Key Observations:
1. **Sorting**: By sorting the arrays \(A\) and \(B\) in descending order, we can easily manage the selection process.
2. **Greedy Selection**: We will select the largest elements from \(A\) and corresponding smallest elements from \(B\) to minimize the product. This ensures that the maximum element in \(A\) is paired with the smallest elements in \(B\) to achieve the minimum possible product.

### Steps:
1. Combine the arrays \(A\) and \(B\) into pairs.
2. Sort these pairs based on the value of \(A\) in descending order.
3. Select the top \(K\) elements from the sorted list.
4. Calculate the required product for the selected elements.

### Implementation:
Here's the corrected and optimized C++ code:

```cpp
#include <bits/stdc++.h>
using namespace std;

typedef pair<long long, long long> pii;

int main() {
    ios::sync_with_stdio(0);
    cin.tie(0);
    cout.tie(0);

    int t;
    cin >> t;
    while (t--) {
        int n, k;
        cin >> n >> k;

        vector<pii> ab(n);
        for (int i = 0; i < n; ++i) {
            cin >> ab[i].first >> ab[i].second;
        }

        // Sort by A in descending order
        sort(ab.begin(), ab.end(), [&](const pii &a, const pii &b) {
            return a.first > b.first;
        });

        long long result = 0;
        long long sum_b = 0;
        priority_queue<long long> pq;

        for (int i = 0; i < k; ++i) {
            sum_b += ab[i].second;
            pq.push(-ab[i].second); // Using negative to simulate min heap
        }

        result = ab[k - 1].first * sum_b;

        for (int i = k; i < n; ++i) {
            sum_b -= pq.top();
            pq.pop();

            sum_b += ab[i].second;
            pq.push(-ab[i].second);

            result = min(result, ab[i].first * sum_b);
        }

        cout << result << '\n';
    }

    return 0;
}
```

### Explanation:
1. **Pairing and Sorting**: We create pairs of elements from \(A\) and \(B\) and sort them by the values in \(A\) in descending order.
2. **Initial Calculation**: We initially take the top \(K\) elements from the sorted list, calculate their sum of \(B\) values, and compute the initial product.
3. **Sliding Window**: For the remaining elements, we maintain a sliding window of size \(K\) by removing the smallest element added previously and adding the next element from the sorted list. This helps in maintaining the minimum possible product.
4. **Result Update**: During each step, we update the result if the new product is smaller.

This approach ensures that we efficiently find the minimum product while handling the constraints effectively.
