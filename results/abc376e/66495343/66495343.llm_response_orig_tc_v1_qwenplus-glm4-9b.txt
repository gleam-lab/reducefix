
The problem aims to find the minimum possible value of the expression:

\[
\left(\max_{i \in S} A_i\right) \times \left(\sum_{i \in S} B_i\right)
\]

where \( S \) is a subset of \(\{1, 2, \dots, N\}\) of size \( K \), and \( A \) and \( B \) are sequences of length \( N \).

### Analysis

To solve this problem, we need to consider two main factors:
1. **Maximize \( A_i \)**: We want to maximize the maximum value of \( A \) in the subset \( S \).
2. **Minimize the sum of \( B_i \)**: We want to minimize the sum of \( B \) in the same subset \( S \).

A naive approach would be to try all combinations of subsets \( S \) of size \( K \), but this is computationally infeasible due to the constraints.

### Optimized Approach

1. **Sorting**: We can start by sorting the pairs \( (A_i, B_i) \) by \( A_i \) in descending order. This allows us to consider the largest \( A_i \) values first, which is our first factor to maximize.

2. **Sliding Window**: For each largest \( A_i \) (starting from the largest since we sorted them), we calculate the sum of \( B \) values for the smallest \( K \) values (or fewer, if \( N < K \)). We do this using a sliding window approach:
   - Calculate the sum for the first \( K \) values.
   - Slide the window by removing the first element and adding the next one, and update the sum and the minimum result accordingly.

3. **Result Calculation**: The result for each test case is the minimum value obtained from the above sliding window process for each \( A_i \).

### Implementation

```cpp
#include <iostream>
#include <vector>
#include <algorithm>

using namespace std;

typedef long long ll;

int main() {
    int T;
    cin >> T;
    
    while (T--) {
        int N, K;
        cin >> N >> K;
        
        vector<pair<ll, ll>> arr(N);
        
        for (int i = 0; i < N; ++i) {
            cin >> arr[i].first;
            arr[i].second = i;
        }
        
        for (int i = 0; i < N; ++i) {
            cin >> arr[i].second;
        }
        
        sort(arr.rbegin(), arr.rend()); // Sort by A in descending order
        
        ll min_result = LLONG_MAX;
        
        // Process each possible choice of maximum element in the subset
        for (int i = 0; i < min(K, N); ++i) {
            ll sum_B = 0;
            // Sum the first i+1 smallest B values
            for (int j = 0; j <= i; ++j) {
                sum_B += arr[j].second;
            }
            // Calculate the result for this subset
            min_result = min(min_result, arr[i].first * sum_B);
        }
        
        cout << min_result << endl;
    }
    
    return 0;
}
```

This solution sorts the pairs by \( A \) first, then iterates over each possible choice of the largest element in the subset, calculating the sum of the smallest \( K \) elements for each choice and keeping track of the minimum result. This approach is efficient given the constraints and avoids the need for a brute-force solution.
