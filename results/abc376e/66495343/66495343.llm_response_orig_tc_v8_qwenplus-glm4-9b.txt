
The problem statement asks for the minimum possible value of the expression:

\[ \left(\max_{i \in S} A_i\right) \times \left(\sum_{i \in S} B_i\right) \]

where \( S \) is a subset of \(\{1, 2, \dots, N\}\) of size \( K \), and \( A \) and \( B \) are sequences of length \( N \).

### Corrected Code Explanation

To solve this problem, the key idea is to leverage sorting and priority queues to efficiently compute the required values for the expression.

```cpp
#include <bits/stdc++.h>
using namespace std;
#define int long long
#define PII pair<int, int>

int t;
int n, k;
vector<PII> arr;

int32_t main() {
    ios_base::sync_with_stdio(0);
    cin.tie(0);
    cout.tie(0);

    cin >> t;
    while (t--) {
        cin >> n >> k;

        arr.assign(n + 1, {0, 0});  // Assign default values
        for (int i = 1; i <= n; i++) {
            cin >> arr[i].first;  // A_i
            cin >> arr[i].second; // B_i
        }

        // Sort by A_i in descending order
        sort(arr.begin() + 1, arr.begin() + n + 1, greater<PII>());

        // Priority queue to track sum of B_i for the top (k-1) elements
        priority_queue<int> pq;
        int res = LLONG_MAX, sum = 0;

        // Calculate initial sum for the first (k-1) elements
        for (int i = 1; i <= k - 1; i++) {
            sum += arr[i].second;
            pq.push(arr[i].second);
        }

        // Compute minimum value for expression
        for (int i = k; i <= n; i++) {
            sum += arr[i].second;
            pq.push(arr[i].second);

            if (pq.size() > k - 1) {
                int big = pq.top();
                pq.pop();
                sum -= big;
            }

            res = min(res, arr[i].first * sum);
        }

        cout << res << '\n';
    }

    return 0;
}
```

### Explanation

1. **Input Reading**: We read the number of test cases \( T \) and for each test case, read \( N \), \( K \), and the sequences \( A \) and \( B \).
2. **Sorting**: We sort the pairs \( \{A_i, B_i\} \) such that \( A_i \) is in descending order. This allows us to find the maximum \( A_i \) for the top \( K \) elements efficiently.
3. **Priority Queue**: We use a priority queue to maintain the sum of \( B_i \) values for the top \( K-1 \) elements. This allows us to efficiently compute the sum when the next element \( A_i \) is considered.
4. **Computation**: For each element from \( k \) to \( N \), we compute the value of the expression and update the minimum result found so far.

This solution efficiently handles the constraints and gives the correct output for the given problem.
