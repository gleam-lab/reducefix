To solve this problem efficiently and correctly, we must identify the minimum possible value of:

$$
\left(\max_{i \in S} A_i\right) \times \left(\sum_{i \in S} B_i\right)
$$

Where:
- $ S $ is a subset of indices of size exactly $ K $
- We're given two arrays $ A $ and $ B $ of length $ N $
- $ K \leq N $

---

### Key Observations

1. **Sorting by A**: Since the expression involves `max(A[i])`, sorting elements by `A` allows us to process them in increasing order.
2. **Greedy + Heap**: As we iterate through each element (sorted by A), we can maintain a max-heap of size `K` to always keep track of the top `K` elements with the smallest `B[i]` values (since we want to minimize the sum). This way, for every index `i`, we can consider it as the maximum A-value in the subset and compute the product.

---

### Algorithm Outline

1. Sort all pairs `(A[i], B[i])` by `A[i]`.
2. Iterate from `i = 0` to `N - 1`:
   - Maintain a priority queue (min-heap) of size `K` storing `B[i]`.
   - Keep a running total of the current `K` `B[i]` values.
   - Once the heap has `K` elements, compute:  
     `A[i] * sum_of_Bs_in_heap`
   - Update the result with the minimum value found so far.

This ensures an efficient solution of time complexity **O(N log K)** per test case.

### Corrected C++ Code

```cpp
#include <bits/stdc++.h>
using namespace std;

#define int long long

signed main() {
    ios::sync_with_stdio(false);
    cin.tie(0);

    int T;
    cin >> T;

    while (T--) {
        int N, K;
        cin >> N >> K;

        vector<pair<int, int>> AB(N);
        for (int i = 0; i < N; ++i) {
            cin >> AB[i].first;
        }
        for (int i = 0; i < N; ++i) {
            cin >> AB[i].second;
        }

        // Sort by A[i]
        sort(AB.begin(), AB.end());

        // Min-heap to store B values
        priority_queue<int, vector<int>, greater<int>> pq;
        int sum_b = 0;
        int res = numeric_limits<int>::max();

        for (int i = 0; i < N; ++i) {
            int a = AB[i].first;
            int b = AB[i].second;

            if (pq.size() < K) {
                sum_b += b;
                pq.push(b);
            } else {
                if (b > pq.top()) {
                    sum_b -= pq.top();
                    pq.pop();
                    sum_b += b;
                    pq.push(b);
                }
            }

            if (pq.size() == K) {
                res = min(res, a * sum_b);
            }
        }

        cout << res << "\n";
    }

    return 0;
}
```

### Why the Original Code Was Wrong

The original code used a `priority_queue<int>` (a max-heap), but was trying to remove the largest elements to keep only the smallest `K`. However, that logic was flawed — particularly in cases where small `B[i]` values were incorrectly removed or kept, leading to incorrect sums.

By switching to a **min-heap** and maintaining the **smallest `K` B-values**, we ensure the correct greedy strategy is applied.
