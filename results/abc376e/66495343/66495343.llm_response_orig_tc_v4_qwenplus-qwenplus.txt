To solve this problem efficiently, we need to find the minimum value of the expression:

$$
\left(\max_{i \in S} A_i\right) \times \left(\sum_{i \in S} B_i\right)
$$

for a subset `S` of size `K` from a list of pairs `(A[i], B[i])`.

### Key Observations:
1. We want to pick K elements such that the maximum A value among them multiplied by the sum of their corresponding B values is **minimized**.
2. A greedy approach won't suffice here due to dependencies between the max and the sum.
3. Sorting the elements by `A[i]` helps because as we process larger `A[i]`, we know any subset including it will have it as the max.

### Strategy:
- Sort all pairs by `A[i]`.
- Use a **min-heap** (priority queue) to maintain the smallest `B[i]` values for the current window of K elements.
- As we iterate through the sorted list:
  - Maintain a running sum of the top-K smallest B values using the heap.
  - At each step after the first `K-1` elements, compute the candidate result: `A[i] * (sum of selected B values)`.
- Keep track of the minimum result found.

This ensures an efficient solution with time complexity **O(N log K)** per test case.

Here's the corrected and optimized code:

```cpp
#include <bits/stdc++.h>
using namespace std;

#define int long long
#define PII pair<int, int>
#define fi first
#define se second
#define arr array<int, 3>

signed main() {
    ios::sync_with_stdio(0);
    cin.tie(0), cout.tie(0);

    int t;
    cin >> t;

    while (t--) {
        int n, k;
        cin >> n >> k;

        vector<int> a(n + 1), b(n + 1);
        vector<PII> items(n + 1);

        for (int i = 1; i <= n; ++i) {
            cin >> a[i];
        }
        for (int i = 1; i <= n; ++i) {
            cin >> b[i];
        }

        // Pair up A and B arrays and sort by A
        for (int i = 1; i <= n; ++i) {
            items[i] = {a[i], b[i]};
        }
        sort(items.begin() + 1, items.begin() + n + 1); // Sort by A

        int res = numeric_limits<int>::max(), sum = 0;
        priority_queue<int, vector<int>, greater<int>> pq; // Min-heap to keep track of smallest K B-values

        for (int i = 1; i <= n; ++i) {
            if (i <= k) {
                sum += items[i].se;
                pq.push(items[i].se);
            } else {
                // Maintain only K elements in the heap
                if (pq.size() > k - 1) {
                    sum -= pq.top();
                    pq.pop();
                }
                sum += items[i].se;
                pq.push(items[i].se);
            }

            if (pq.size() == k) {
                res = min(res, items[i].fi * sum);
            }
        }

        cout << res << "\n";
    }

    return 0;
}
```

### Why This Works:
- Sorting ensures that when we consider item `i`, its `A[i]` is the current max.
- The min-heap allows us to always retain the smallest `K` values of `B[i]` efficiently.
- This guarantees correctness and efficiency even under tight constraints.
