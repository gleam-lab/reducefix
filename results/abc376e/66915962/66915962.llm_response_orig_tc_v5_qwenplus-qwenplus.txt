To solve this problem efficiently, we need to minimize the value of:

$$
\left(\max_{i \in S} A_i\right) \times \left(\sum_{i \in S} B_i\right)
$$

Where $S$ is any subset of size $K$ from the indices $\{1, 2, ..., N\}$.

---

### **Key Observations:**

1. **Sorting by A:** Since we're trying to find a subset of size $K$, and the maximum $A_i$ in that subset will be multiplied by the sum of corresponding $B_i$, it makes sense to sort all elements by $A_i$. This allows us to consider candidates in increasing order of $A_i$, which ensures that when we fix an element as the max $A_i$, all previous $K-1$ elements have smaller or equal $A_i$.

2. **Using a Min-Heap for B:** We want to maintain the smallest $K$ elements seen so far based on their $B_i$ values. If we encounter a new element with a higher $B_i$, replacing the smallest $B_i$ can potentially improve the total sum. Using a min-heap helps manage this efficiently.

3. **Greedy + Heap Strategy:**
   - Sort all pairs $(A_i, B_i)$ by $A_i$.
   - Use a min-heap to track the $K$ largest $B_i$'s (by maintaining the top K elements).
   - For each position $i$ starting from index $K-1$ to $N-1$, keep updating the max $A_i$ and the current sum of $B_i$'s.
   - At each step, calculate the product $\text{A}[i] \times \text{sum of selected B's}$ and update the minimum result.

This approach runs in $O(N \log K)$ per test case due to sorting and heap operations.

### ✅ **Corrected & Optimized C++ Code:**

```cpp
#include <bits/stdc++.h>
using namespace std;

using ll = long long;
using P = pair<int, int>;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(0);

    int T;
    cin >> T;

    while (T--) {
        int N, K;
        cin >> N >> K;

        vector<P> AB(N);
        for (int i = 0; i < N; ++i) {
            cin >> AB[i].first; // A_i
        }
        for (int i = 0; i < N; ++i) {
            cin >> AB[i].second; // B_i
        }

        // Sort by A_i ascending
        sort(AB.begin(), AB.end());

        priority_queue<int, vector<int>, greater<>> min_heap;
        ll sum_b = 0;

        // Initialize the first K elements
        for (int i = 0; i < K; ++i) {
            sum_b += AB[i].second;
            min_heap.push(AB[i].second);
        }

        ll ans = 1LL * AB[K - 1].first * sum_b;

        // Try including better combinations
        for (int i = K; i < N; ++i) {
            if (AB[i].second > min_heap.top()) {
                sum_b -= min_heap.top();
                min_heap.pop();
                sum_b += AB[i].second;
                min_heap.push(AB[i].second);
            }
            ans = min(ans, 1LL * AB[i].first * sum_b);
        }

        cout << ans << '\n';
    }

    return 0;
}
```

- The original code used `set<int>` to simulate a heap but had logical errors in removing the max instead of the smallest $B_i$.
- We now use a proper **min-heap** to always retain the $K$ elements with the largest $B_i$ values.
- Sorting ensures we process elements in increasing $A_i$ order, so the current $A[i]$ is always the maximum among the selected $K$.
- Time complexity remains efficient: $O(N \log K)$ per test case, acceptable under constraints.
