The initial approach sorts the pairs \((A_i, B_i)\) by \( A_i \) and then tries to maintain the top \( K \) \( B_i \) values using a set, but this approach is flawed because it doesn't efficiently track the smallest sum of \( B_i \) for the current \( A_i \).

However, this approach is incorrect because it doesn't efficiently maintain the smallest possible sum of \( B_i \) values for the current \( A_i \). The set only ensures that the largest \( B_i \) values are removed, but this doesn't necessarily minimize the sum.

3. **Updating the Sum and Calculating the Product**:

### Correct Approach
1. **Sorting by \( A_i \)**:
   - Sort the pairs \((A_i, B_i)\) by \( A_i \) in ascending order. This allows us to fix \( A_i \) as the maximum \( A \) in the subset \( S \) when considering the first \( i \) elements.

2. **Maintaining the Minimum Sum of \( B_i \)**:
   - Use a max-heap (priority queue) to keep track of the largest \( B_i \) values in the current window of \( K \) elements. The sum of \( B_i \) is adjusted by removing the largest \( B_i \) when a new \( B_i \) is added if it increases the sum.

3. **Calculating the Product**:
   - For each \( i \) from \( K \) to \( N \), the product \( A_i \times \text{sum}(B_i \text{ in current window}) \) is computed, and the minimum product is tracked.

#include<bits/stdc++.h>
using namespace std;
using ll = long long;
using VI = vector<int>;
using P = pair<int, int>;
constexpr int INF = 1001001001;
constexpr ll LINF = 1001001001001001001ll;
#define rep(i, n) for (ll i = 0; i < (ll)(n); i++)
#define rep2(i, k, n) for (ll i = k; i < (ll)(n); i++)
#define OUT(x) cout << #x << ":" << x << endl

int main(){
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);
    int t;
    cin >> t;
    while (t--) {
        int n, k;
        cin >> n >> k;
        vector<P> p(n);
        rep(i, n) cin >> p[i].first;
        rep(i, n) cin >> p[i].second;
        sort(p.begin(), p.end());
        vector<ll> a(n), b(n);
        rep(i, n) {
            a[i] = p[i].first;
            b[i] = p[i].second;
        }
        ll ans = LINF;
        ll sumB = 0;
        priority_queue<ll, vector<ll>, greater<ll>> minHeap;
        rep(i, k-1) {
            minHeap.push(b[i]);
            sumB += b[i];
        }
        rep2(i, k-1, n) {
            if (minHeap.size() < k) {
                minHeap.push(b[i]);
                sumB += b[i];
            } else if (b[i] > minHeap.top()) {
                sumB += b[i] - minHeap.top();
                minHeap.pop();
                minHeap.push(b[i]);
            }
            if (minHeap.size() == k) {
                ans = min(ans, a[i] * sumB);
            }
        }
        cout << ans << '\n';
    }
    return 0;
}

1. **Sorting the Pairs**:

2. **Heap Management**:
   - A min-heap is used to maintain the \( K \) smallest \( B_i \) values encountered so far. This ensures that the sum of \( B_i \) is minimized for the current \( A_i \).

3. **Sum Adjustment**:
   - As we iterate through the sorted list, we adjust the heap to include the current \( B_i \) if it is smaller than the largest element in the heap, thus maintaining the smallest sum possible.

4. **Product Calculation**:
   - For each \( i \), the product of \( A_i \) (the current maximum) and the sum of the \( B_i \) values in the heap is computed, and the minimum product is tracked.

This approach efficiently ensures that we find the minimal product for each test case while adhering to the constraints.
