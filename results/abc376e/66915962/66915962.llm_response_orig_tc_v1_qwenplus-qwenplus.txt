To solve this problem efficiently and correctly, we must compute the minimum possible value of:

$$
\left(\max_{i \in S} A_i\right) \times \left(\sum_{i \in S} B_i\right)
$$

Where `S` is a subset of size `K` chosen from indices `{1..N}`.

---

### **Key Observations**

- We need to select exactly `K` elements.
- For any selection, the cost is determined by:
  - The maximum `A[i]` in the selected set (`max_A`)
  - The sum of all `B[i]` in the selected set (`sum_B`)
- Our goal is to minimize `max_A * sum_B`.

---

### **Strategy**

We sort the pairs `(A[i], B[i])` in increasing order of `A`. This allows us to fix the current element as the maximum `A[i]`, and maintain a subset of `K` elements including this one with the largest `K` values of `B`.

Here's how we proceed:

1. Sort the array of pairs `(A[i], B[i])` by `A[i]`.
2. Maintain a multiset (or priority queue) of the `K` largest `B[i]` values seen so far.
3. As we iterate through the sorted list, for each index `i`, we:
   - Add `B[i]` to the multiset.
   - If the multiset has more than `K` elements, remove the smallest one.
   - Once we have at least `K` elements, calculate `A[i] * sum_of_selected_B` and update the minimum result.

This ensures that for every candidate max_A (sorted), we're considering only the best K elements based on B.

Time Complexity: **O(N log N)** per test case due to sorting and multiset operations.

---

### ✅ Corrected Code

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);

    int T;
    cin >> T;

    while (T--) {
        int N, K;
        cin >> N >> K;

        vector<pair<int, int>> items(N);
        for (int i = 0; i < N; ++i) {
            cin >> items[i].first;
        }
        for (int i = 0; i < N; ++i) {
            cin >> items[i].second;
        }

        // Sort by A[i]
        sort(items.begin(), items.end());

        multiset<int> b_values;
        long long sumB = 0;
        long long result = numeric_limits<long long>::max();

        for (int i = 0; i < N; ++i) {
            int a = items[i].first;
            int b = items[i].second;

            b_values.insert(b);
            sumB += b;

            if (b_values.size() > K) {
                // Remove the smallest B in the set
                auto it = b_values.begin();
                sumB -= *it;
                b_values.erase(it);
            }

            if (b_values.size() == K) {
                result = min(result, a * sumB);
            }
        }

        cout << result << '\n';
    }

    return 0;
}
```

---

### ✅ Why This Works

- Sorting by `A[i]` allows us to treat each `A[i]` as a potential `max_A`.
- Using a multiset maintains the top `K` largest `B[i]` values.
- Efficiently updates `sumB` by removing the smallest `B[i]` when needed.
- Ensures we always compute `max_A * sum_B` with exactly `K` elements.

---

### 📌 Edge Cases Handled
- `K == N`: Only one possible subset.
- `K == 1`: Try all single elements.
- Large input sizes within constraints.
