
The problem requires finding the minimum value of the expression 

\[
\left(\max_{i \in S} A_i\right) \times \left(\sum_{i \in S} B_i\right)
\]

where \( S \) is a subset of \(\{1, 2, \dots, N\}\) of size \( K \).

### Solution Strategy

1. **Sort the Arrays**: First, we need to sort the arrays \( A \) and \( B \) based on their elements such that \( A_i \) and \( B_i \) are in increasing order.
2. **Initial Setup**: Let's consider the first \( K \) elements for \( S \). The maximum of these elements is \( A_k \), and the sum of the corresponding elements of \( B \) is \( \sum_{i=1}^{k} B_i \).
3. **Sliding Window Technique**: To efficiently find the minimum value of the expression, we can use a sliding window of size \( K \) from the sorted array and compute the value of the expression.
4. **Tracking Maximum Value**: As we slide the window, we need to keep track of the maximum value of \( A \) in the window and adjust the sum of \( B \) accordingly.

### Optimized Code

```cpp
#include <iostream>
#include <vector>
#include <algorithm>
using namespace std;
#define int long long

int32_t main() {
    int t;
    cin >> t;
    while (t--) {
        int n, k;
        cin >> n >> k;
        vector<int> A(n), B(n);
        for (int i = 0; i < n; i++) {
            cin >> A[i];
        }
        for (int i = 0; i < n; i++) {
            cin >> B[i];
        }
        
        // Pairing A and B and sorting by A
        vector<pair<int, int>> ab(n);
        for (int i = 0; i < n; i++) {
            ab[i] = {A[i], B[i]};
        }
        sort(ab.begin(), ab.end());
        
        // Initial window
        long long sumB = 0, maxA = ab[0].first;
        for (int i = 1; i <= k; i++) {
            sumB += ab[i - 1].second;
        }
        
        long long ans = maxA * sumB;
        
        // Sliding window
        for (int i = k; i < n; i++) {
            sumB += ab[i].second - ab[i - k].second;
            maxA = ab[i - k + 1].first;
            ans = min(ans, maxA * sumB);
        }
        
        cout << ans << '\n';
    }
    return 0;
}
```

### Explanation

1. **Input**: We read the number of test cases \( t \).
2. **Input Reading**: For each test case, we read \( n \), \( k \), and arrays \( A \) and \( B \).
3. **Pairing and Sorting**: We pair the elements of \( A \) and \( B \) and sort them based on \( A \).
4. **Initial Window**: We compute the sum of \( B \) elements for the first \( k \) and set the maximum \( A \) to \( A_k \).
5. **Sliding Window**: We slide the window from \( k \) to \( n \), updating the sum of \( B \) and the maximum \( A \). We keep track of the minimum value of the expression during the sliding.
6. **Output**: We print the minimum value for each test case.

This solution ensures that we consider all possible subsets of size \( K \) efficiently using sorting and a sliding window approach.
