To solve this problem efficiently, we need to find the minimum value of the expression:

$$
\left(\max_{i \in S} A_i\right) \times \left(\sum_{i \in S} B_i\right)
$$

Where $S$ is a subset of size $K$ from indices $1$ to $N$, and we're given multiple test cases.

---

### **Key Observations**

- For each test case:
  - We want to choose exactly `K` elements such that the product of the maximum A-value in the subset and the sum of their B-values is minimized.
  - Brute-force checking all combinations would be too slow (combinatorial explosion).
  
- **Greedy + Priority Queue Strategy**:
  - Sort the pairs by A-value.
  - Iterate through the list while maintaining a max heap of size `K`.
  - At each step, we consider the current element as the max A-value (`A[i]`) and maintain the sum of the smallest `K-1` B-values among the previous `K` elements.
  - Use a priority queue (min-heap for B-values) to keep track of the smallest `K` values.
  - This ensures an efficient runtime of $O(N \log N)$ per test case.

---

### **Corrected C++ Code**

```cpp
#include <bits/stdc++.h>
using namespace std;

#define int long long

signed main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int T;
    cin >> T;

    while (T--) {
        int N, K;
        cin >> N >> K;

        vector<int> A(N), B(N);
        for (int i = 0; i < N; ++i) cin >> A[i];
        for (int i = 0; i < N; ++i) cin >> B[i];

        vector<pair<int, int>> AB(N);
        for (int i = 0; i < N; ++i) {
            AB[i] = {A[i], B[i]};
        }

        // Sort by A[i] ascending
        sort(AB.begin(), AB.end());

        // Min-heap to store B values of selected K elements
        priority_queue<int, vector<int>, greater<>> pq;
        int sum_b = 0;
        int res = numeric_limits<int>::max();

        for (int i = 0; i < N; ++i) {
            if (pq.size() >= K) {
                // Remove the smallest B value to make room
                sum_b -= pq.top();
                pq.pop();
            }

            pq.push(AB[i].second);
            sum_b += AB[i].second;

            if (pq.size() == K) {
                // The current A is the largest in the window due to sorting
                res = min(res, AB[i].first * sum_b);
            }
        }

        cout << res << "\n";
    }

    return 0;
}
```

---

### **Explanation**

- **Sorting**: Sorting by A ensures that as we process items, any group of K elements has the current item's A-value as the maximum.
- **Min Heap**: Maintains the K smallest B-values so far. If more than K are present, remove the smallest one.
- **Sum Management**: Efficiently tracks the sum of B-values in the current window.
- **Result Update**: Only compute the product when the window has K elements.

This approach guarantees:
- **Efficiency**: Each operation on the heap is $O(\log K)$, and we do it N times.
- **Accuracy**: We always have the correct combination of K elements with minimum cost.

---

### **Edge Cases Handled**
- K = 1: Just pick the minimum A[i]*B[i]
- All elements have same A or B values
- Large input sizes within constraints
