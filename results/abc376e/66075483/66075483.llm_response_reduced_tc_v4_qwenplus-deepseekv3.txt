2. **Prefix Sum and Priority Queue**: For each element at position i (after sorting), the maximum A value in any subset that includes A[i] will be A[i]. We then need to find the sum of the smallest K-1 B values from the first i-1 elements (since A[i] is the maximum, we take K-1 elements from the previous elements to form a subset of size K). This involves maintaining a max-heap to efficiently track the largest K-1 B values up to each point, and using a prefix sum to quickly compute the sum of these values.

#include <iostream>
#include <vector>
#include <queue>
#include <algorithm>
using namespace std;
#define int long long
const int inf = 1e18;

signed main() {
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);
    
    int t;
    cin >> t;
    while (t--) {
        int n, k;
        cin >> n >> k;
        vector<pair<int, int>> a(n);
        vector<int> b(n);
        
        for (int i = 0; i < n; ++i) {
            cin >> a[i].first;
            a[i].second = i;
        }
        for (int i = 0; i < n; ++i) {
            cin >> b[i];
        }
        
        sort(a.begin(), a.end());
        vector<int> sorted_b(n);
        for (int i = 0; i < n; ++i) {
            sorted_b[i] = b[a[i].second];
        }
        
        priority_queue<int> pq;
        int sum = 0;
        int ans = inf;
        
        for (int i = 0; i < n; ++i) {
            int current_a = a[i].first;
            int current_b = sorted_b[i];
            if (pq.size() < k - 1) {
                sum += current_b;
                pq.push(current_b);
            } else if (!pq.empty() && current_b < pq.top()) {
                sum -= pq.top();
                pq.pop();
                sum += current_b;
                pq.push(current_b);
            }
            
            if (pq.size() == k - 1) {
                ans = min(ans, current_a * (sum + current_b));
            }
        }
        
        cout << ans << '\n';
    }
    return 0;
}

### Explanation
1. **Sorting**: The elements are sorted based on their A values. This allows us to process each element in increasing order of A, ensuring that when we consider A[i], it is the largest element in any subset formed up to that point.
2. **Priority Queue**: A max-heap is used to keep track of the largest B values encountered so far. For each element, if the heap has fewer than K-1 elements, we simply add the current B value. Otherwise, if the current B value is smaller than the largest element in the heap, we replace the largest element in the heap with the current B value. This ensures that the heap always contains the smallest K-1 B values up to the current point.
3. **Sum Calculation**: The sum of the B values in the heap (which are the smallest K-1 B values up to the current element) is maintained. For each element, the potential subset includes this element and the K-1 elements in the heap. The product of the current A value (which is the max in the subset) and the sum of the B values (sum of heap elements plus the current B value) is computed and the minimum value is tracked.
4. **Edge Handling**: Special cases, such as K=1, are naturally handled since the heap will be empty, and the sum will only include the current B value, multiplied by its A value (which is the max in the subset of size 1).

This approach efficiently computes the desired minimum value by leveraging sorting and a priority queue to manage the subset selection, ensuring optimal performance with O(N log K) time complexity per test case.
