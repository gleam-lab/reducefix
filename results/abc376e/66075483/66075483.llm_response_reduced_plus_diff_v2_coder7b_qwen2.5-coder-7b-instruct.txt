To solve this problem efficiently, we need to minimize the expression \((\max_i A_i) \times (\sum_j B_j)\) for a subset \(S\) of size \(K\). The key insight here is that we want to maximize \(\min(A_i)\) because it will naturally lead to the smallest product when paired with the sum of \(B_i\).

Here's a step-by-step approach to achieve this:

1. **Sort Arrays**: Sort array \(A\) based on its values.
2. **Select Elements**: Select the first \(K\) elements from the sorted array \(A\) since they will have the smallest values. These correspond to the indices that will give us the minimum \(\max(A_i)\).
3. **Calculate Sum**: Calculate the sum of the corresponding elements in array \(B\) for these selected indices.
4. **Compute Result**: Compute the result as the product of the maximum element in the selected subset and the sum of the corresponding elements in \(B\).

This approach ensures that we always pick the smallest possible \(\max(A_i)\) and thus minimize the overall product.

Here is the corrected C++ code implementing this logic:

```cpp
#include<iostream>
#include<vector>
#include<algorithm>
using namespace std;

typedef pair<int, int> pii;

int main() {
    ios_base::sync_with_stdio(false); cin.tie(NULL);

    int t;
    cin >> t;
    while (t--) {
        int n, k;
        cin >> n >> k;
        
        vector<int> A(n), B(n);
        for (int i = 0; i < n; ++i) cin >> A[i];
        for (int i = 0; i < n; ++i) cin >> B[i];

        // Create pairs of (A[i], i)
        vector<pii> AB(n);
        for (int i = 0; i < n; ++i) AB[i] = {A[i], i};

        // Sort the pairs by A[i]
        sort(AB.begin(), AB.end());

        // Select the first K elements
        int max_A = 0, sum_B = 0;
        for (int i = 0; i < k; ++i) {
            max_A = max(max_A, A[AB[i].second]);
            sum_B += B[AB[i].second];
        }

        // Calculate the result
        cout << max_A * sum_B << '\n';
    }

    return 0;
}
```

### Explanation:
1. **Reading Input**: We read the number of test cases and process each one individually.
2. **Sorting**: We create pairs of elements from arrays \(A\) and \(B\) and sort them based on the values in \(A\).
3. **Selection and Calculation**: We select the first \(K\) elements from the sorted list, compute the maximum value among them, and sum the corresponding values from \(B\).
4. **Output**: We output the computed result.

This solution has a time complexity of \(O(N \log N)\) due to the sorting step, which is efficient enough given the constraints.
