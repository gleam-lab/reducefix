To solve this problem efficiently, we need to minimize the expression \((\max_i A_i) \times (\sum_j B_j)\) where \(j\) is in the subset \(S\) of size \(K\). We can achieve this by selecting the largest \(K\) elements from \(A\) and the smallest \(K\) elements from \(B\).

Here's how we can approach it:

1. **Sort Arrays**: Sort array \(A\) in descending order and array \(B\) in ascending order.
2. **Calculate Result**: Multiply the maximum element from \(A\) with the sum of the smallest \(K\) elements from \(B\).

This approach ensures that we are minimizing the product by pairing the largest possible \(A_i\) with the smallest possible \(B_j\).

Let's implement this in C++:

```cpp
#include <bits/stdc++.h>
#define int long long
using namespace std;

signed main() {
    ios::sync_with_stdio(false);
    cin.tie(0);

    int t; cin >> t;
    while(t--) {
        int n, k; cin >> n >> k;
        vector<int> a(n), b(n);
        
        // Read input arrays
        for(int i = 0; i < n; ++i) cin >> a[i];
        for(int i = 0; i < n; ++i) cin >> b[i];
        
        // Sort A in descending order
        sort(a.begin(), a.end(), greater<int>());
        
        // Sort B in ascending order
        sort(b.begin(), b.end());
        
        // Calculate the result
        int max_a = a[k-1];  // Get the Kth largest element in A
        int sum_b = 0;      // Sum of the K smallest elements in B
        
        for(int i = 0; i < k; ++i) sum_b += b[i];
        
        // Output the result
        cout << max_a * sum_b << '\n';
    }
    
    return 0;
}
```

### Explanation:
1. **Sorting**:
   - We sort `a` in descending order to get the largest elements easily.
   - We sort `b` in ascending order to get the smallest elements easily.

2. **Calculating the Product**:
   - The maximum element in the top \(K\) elements of `a` is stored in `max_a`.
   - The sum of the smallest \(K\) elements of `b` is calculated and stored in `sum_b`.

3. **Result**:
   - The result is computed as `max_a * sum_b`, which gives us the minimum possible value of the required expression.

This solution has a time complexity of \(O(N \log N)\) due to the sorting steps, which is efficient enough given the constraints.
