To solve this problem efficiently, we need to minimize the expression \((\max_i A_i) \times (\sum_i B_i)\) for a subset \(S\) of size \(K\). One effective approach is to use a greedy strategy combined with sorting.

### Steps to Solve the Problem:

1. **Sort Arrays**: Sort both arrays \(A\) and \(B\) in descending order.
2. **Select Elements**: To minimize the product, select the largest element from \(A\) and pair it with the smallest sum of corresponding elements from \(B\).
3. **Use Min-Heap**: Maintain a min-heap to keep track of the sums of pairs involving the largest elements from \(A\).

### Detailed Explanation:

1. **Sorting**:
   - Sort array \(A\) in descending order to get the largest elements first.
   - Sort array \(B\) in descending order to get the smallest sums first.

2. **Using Min-Heap**:
   - Initialize a priority queue (min-heap) to store the sums of pairs.
   - Start by adding the smallest sums of pairs involving the largest elements from \(A\).

3. **Iterate and Update**:
   - For each element in \(A\), compute the sum with the current top of the heap and push it back into the heap.
   - Pop the top element from the heap to ensure we always consider the smallest sum.

4. **Result Calculation**:
   - After processing all elements, the top element of the heap will give us the minimum possible value of the expression.

### Implementation:

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(0);

    int t; cin >> t;
    while(t--) {
        int n, k; cin >> n >> k;
        vector<int> A(n), B(n);
        for(int i = 0; i < n; ++i) cin >> A[i];
        for(int i = 0; i < n; ++i) cin >> B[i];

        // Sort A in descending order
        sort(A.begin(), A.end(), greater<int>());
        // Sort B in descending order
        sort(B.begin(), B.end());

        // Min-heap to store the sums of pairs
        priority_queue<int, vector<int>, greater<int>> pq;

        // Initialize the heap with the smallest sums
        for(int i = 0; i < k; ++i) {
            pq.push(A[i] * B[n-k+i]);
        }

        // Iterate through the remaining elements
        for(int i = k; i < n; ++i) {
            int curr_max = pq.top();
            pq.pop();
            pq.push(curr_max + A[i] * B[n-k+i]);
        }

        // The top element of the heap is the result
        cout << pq.top() << '\n';
    }
    return 0;
}
```

### Explanation of the Code:

1. **Input Reading**: Read the number of test cases and process each one.
2. **Sorting**: Sort arrays \(A\) and \(B\) in descending order.
3. **Min-Heap Initialization**: Initialize a min-heap with the smallest sums of pairs involving the largest elements from \(A\).
4. **Processing Remaining Elements**: For each remaining element in \(A\), update the heap to keep track of the minimum possible value.
5. **Output Result**: The top element of the heap gives the minimum value of the expression.

This approach ensures that we efficiently find the minimum value using sorting and a min-heap, making it suitable for large inputs up to the given constraints.
