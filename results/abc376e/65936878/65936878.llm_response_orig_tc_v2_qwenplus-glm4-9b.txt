
The given problem is a combinatorial optimization problem where we need to minimize a specific expression involving the maximum of a subset and the sum of the corresponding elements of another sequence. The problem's constraints allow for a brute-force approach to be infeasible, so a more efficient approach is required.

### Problem Analysis
Given two sequences \( A \) and \( B \), and a subset size \( K \), we need to find a subset \( S \) of size \( K \) such that the expression:
\[ \left(\max_{i \in S} A_i\right) \times \left(\sum_{i \in S} B_i\right) \]
is minimized.

### Solution Strategy
1. **Sorting**: Start by sorting the sequences \( A \) and \( B \) in descending order. This allows us to efficiently access the largest elements and their corresponding sums.
2. **Sliding Window**: Use a sliding window approach to maintain a sum of the last \( K \) elements of \( B \) and update the result accordingly.
3. **Efficient Update**: When the window slides, efficiently update the maximum value from \( A \) and the sum of \( B \) within the window.

### Detailed Steps
1. **Input Reading**: Read the number of test cases. For each test case, read \( N \), \( K \), \( A \), and \( B \).
2. **Sorting**: Sort \( A \) and \( B \) in descending order.
3. **Sliding Window Initialization**:
   - Initialize a variable to keep track of the sum of the last \( K \) elements of \( B \). Start it with the sum of the first \( K \) elements.
   - Initialize a variable to keep track of the maximum value of \( A \) in the first \( K \) elements.
4. **Sliding Window Execution**:
   - Slide the window from the start to the end of the array.
   - For each step, update the sum and maximum value.
   - Calculate the potential result for the current window and update the minimum result if the current one is smaller.
5. **Output**: After processing all windows for a test case, output the minimum result.

### C++ Code Implementation
```cpp
#include <iostream>
#include <vector>
#include <algorithm>

using namespace std;

int main() {
    int t;
    cin >> t;
    
    while (t--) {
        int n, k;
        cin >> n >> k;
        vector<int> A(n), B(n);
        
        for (int i = 0; i < n; i++) {
            cin >> A[i];
        }
        for (int i = 0; i < n; i++) {
            cin >> B[i];
        }
        
        sort(A.begin(), A.end(), greater<int>());
        sort(B.begin(), B.end(), greater<int>());
        
        long long sumB = 0;
        long long maxA = A[0];
        long long result = A[0] * sumB; // Start with the first element
        
        for (int i = 1; i <= n - k; i++) {
            sumB += B[i + k - 1] - B[i - 1];
            maxA = max(maxA, A[i + k - 1]);
            result = min(result, maxA * sumB);
        }
        
        cout << result << endl;
    }
    
    return 0;
}
```

This solution efficiently computes the minimum value of the expression by using a sliding window technique, ensuring that the time complexity per test case is \( O(N \log N) \) due to sorting, which is feasible given the problem constraints.
