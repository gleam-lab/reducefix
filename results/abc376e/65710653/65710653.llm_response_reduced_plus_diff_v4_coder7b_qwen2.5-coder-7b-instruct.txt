To solve this problem efficiently, we need to find the minimum value of the expression \((\max_{i \in S} A_i) \times (\sum_{i \in S} B_i)\) for a subset \(S\) of size \(K\) from two arrays \(A\) and \(B\). The key insight here is to use a greedy approach combined with sorting.

### Steps to Solve:
1. **Sort Arrays**: Sort array \(A\) in descending order and array \(B\) in ascending order.
2. **Select Elements**: Select the first \(K\) elements from the sorted \(A\) and the last \(K\) elements from the sorted \(B\).
3. **Calculate the Result**: Compute the product of the maximum element from the selected \(A\) and the sum of the selected elements from \(B\).

This approach ensures that we are maximizing the product by combining the largest possible values from \(A\) with the smallest possible values from \(B\), which helps in minimizing the overall product.

### Implementation:
Below is the corrected and optimized C++ code that follows the above steps:

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);

    int t;
    cin >> t;
    while (t--) {
        int n, k;
        cin >> n >> k;
        vector<int> a(n), b(n);
        
        // Read input arrays
        for (int i = 0; i < n; ++i) cin >> a[i];
        for (int i = 0; i < n; ++i) cin >> b[i];

        // Sort A in descending order
        sort(a.begin(), a.end(), greater<int>());
        // Sort B in ascending order
        sort(b.begin(), b.end());

        long long result = 0;
        // Calculate the result as described
        for (int i = 0; i < k; ++i) {
            result += 1LL * a[i] * b[n - k + i];
        }

        cout << result << '\n';
    }
    return 0;
}
```

### Explanation:
- **Sorting**: We sort \(A\) in descending order to get the largest elements and \(B\) in ascending order to get the smallest elements.
- **Product Calculation**: We iterate through the first \(K\) elements of \(A\) and the last \(K\) elements of \(B\), calculating the product and accumulating it into `result`.

### Complexity:
- **Time Complexity**: Sorting both arrays takes \(O(N \log N)\), and iterating through \(K\) elements is \(O(K)\). Thus, the overall time complexity is dominated by the sorting step, making it \(O(N \log N)\).
- **Space Complexity**: The space complexity is \(O(N)\) due to storing the input arrays.
