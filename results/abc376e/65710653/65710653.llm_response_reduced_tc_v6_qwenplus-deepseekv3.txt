### Problem Analysis
The problem requires finding a subset S of size K from the given sequences A and B such that the product of the maximum A_i in S and the sum of B_i in S is minimized. 

### Key Insights
1. **Sorting by A_i**: Since the maximum A_i in S contributes directly to the product, it's beneficial to consider subsets where the maximum A_i is as small as possible. Sorting the pairs (A_i, B_i) by A_i allows us to systematically consider subsets where the maximum A_i is controlled.
2. **Sliding Window and Priority Queue**: For each possible maximum A_i (after sorting), we need to select the K-1 smallest B_i values from the elements with A_i less than or equal to the current A_i. This ensures the sum of B_i is minimized for that particular maximum A_i. A max-heap (or a multiset that allows efficient access to the largest elements) can be used to maintain the K-1 smallest B_i values.

### Solution Approach
1. **Sorting**: Sort the pairs (A_i, B_i) based on A_i in ascending order.
2. **Sliding Window with Heap**: Iterate through the sorted list, maintaining a max-heap of the K smallest B_i values encountered so far. For each element at position i (0-based), the current A_i is the potential maximum in the subset S. The sum of the K smallest B_i values up to this point (including B_i) is calculated, and the product with A_i is considered for the minimum value.

### Solution Code
```cpp
#include <bits/stdc++.h>
using namespace std;
using ll = long long;
using pint = pair<int, int>;
using pll = pair<ll, ll>;

#define rep(i, a, b) for (ll i = (a); i < (b); i++)
#define rrep(i, a, b) for (ll i = (a); i >= (b); i--)
#define all(a) (a).begin(), (a).end()

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);
    
    int T;
    cin >> T;
    while (T--) {
        int N, K;
        cin >> N >> K;
        vector<pll> pairs(N);
        rep(i, 0, N) cin >> pairs[i].first;
        rep(i, 0, N) cin >> pairs[i].second;
        
        sort(all(pairs));
        
        priority_queue<ll> max_heap;
        ll sum = 0;
        ll ans = LLONG_MAX;
        
        rep(i, 0, N) {
            ll current_A = pairs[i].first;
            ll current_B = pairs[i].second;
            
            if (max_heap.size() < K) {
                max_heap.push(current_B);
                sum += current_B;
                if (max_heap.size() == K) {
                    ans = min(ans, sum * current_A);
                }
            } else {
                if (current_B < max_heap.top()) {
                    sum -= max_heap.top();
                    max_heap.pop();
                    max_heap.push(current_B);
                    sum += current_B;
                    ans = min(ans, sum * current_A);
                }
            }
        }
        
        cout << ans << '\n';
    }
    
    return 0;
}
```

### Explanation
1. **Sorting**: The pairs (A_i, B_i) are sorted by A_i to ensure we process elements in increasing order of A_i, which helps in controlling the maximum A_i in any subset S.
2. **Heap Management**: A max-heap is used to keep track of the K smallest B_i values encountered so far. This allows efficient access to the largest element in the heap, which can be replaced if a smaller B_i is found, maintaining the sum of the smallest K B_i values.
3. **Product Calculation**: For each element, after ensuring the heap contains exactly K elements, the product of the current A_i (which is the maximum in the subset due to sorting) and the sum of B_i values in the heap is computed. The minimum of these products across all elements is the answer.

This approach efficiently balances the trade-off between minimizing the maximum A_i and the sum of B_i, leveraging sorting and heap operations to achieve optimal performance.
