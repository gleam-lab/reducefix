To address the problem, we need to minimize the expression \((\max_i A_i) \times (\sum_i B_i)\) for a subset \(S\) of size \(K\). We can achieve this by selecting the largest \(K\) elements from \(A\) and their corresponding smallest \(K\) elements from \(B\). This approach ensures that the product is minimized because multiplying large values from \(A\) with small values from \(B\) will result in a smaller overall product compared to other combinations.

Here's the corrected C++ code:

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);

    int t;
    cin >> t;
    while (t--) {
        int n, k;
        cin >> n >> k;
        
        vector<int> a(n), b(n);
        for (int i = 0; i < n; ++i) cin >> a[i];
        for (int i = 0; i < n; ++i) cin >> b[i];
        
        sort(a.begin(), a.end());
        sort(b.begin(), b.end());
        
        long long ans = 0;
        for (int i = 0; i < k; ++i) {
            ans += (long long)a[n - 1 - i] * b[i];
        }
        
        cout << ans << '\n';
    }

    return 0;
}
```

### Explanation:
1. **Input Reading**: We read the number of test cases `t` and then for each test case, we read `n`, `k`, followed by arrays `a` and `b`.
2. **Sorting**: We sort both arrays `a` and `b`. Sorting helps us easily pick the largest elements from `a` and the smallest elements from `b`.
3. **Calculation**: We calculate the desired product by iterating through the first `k` elements of the sorted array `b` and multiplying them with the last `k` elements of the sorted array `a`.
4. **Output**: Finally, we output the calculated product for each test case.

This solution ensures that we are always pairing the largest element from `a` with the smallest element from `b`, which minimizes the product as required. The time complexity of this solution is \(O(n \log n)\) due to the sorting step, making it efficient enough for the given constraints.
