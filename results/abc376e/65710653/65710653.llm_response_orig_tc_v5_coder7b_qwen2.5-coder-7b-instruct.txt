Given \(K\) elements, one optimal strategy is to always include the largest \(K\) elements from both sequences \(A\) and \(B\), because this will naturally result in the smallest possible product due to the properties of multiplication.

Here's how you can implement this in C++:

```cpp
#include <iostream>
#include <vector>
#include <algorithm>

using namespace std;

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);

    int t;
    cin >> t;

    while (t--) {
        int n, k;
        cin >> n >> k;

        vector<int> a(n), b(n);
        for (int i = 0; i < n; ++i) {
            cin >> a[i];
        }
        for (int i = 0; i < n; ++i) {
            cin >> b[i];
        }

        // Sort arrays
        sort(a.begin(), a.end());
        sort(b.begin(), b.end());

        // Select the last k elements from both arrays
        int max_a = a[n - k];
        int sum_b = 0;
        for (int i = n - k; i < n; ++i) {
            sum_b += b[i];
        }

        // Calculate the result
        long long result = static_cast<long long>(max_a) * sum_b;
        cout << result << '\n';
    }

    return 0;
}
```

### Explanation:
1. **Sorting**: We sort both arrays \(A\) and \(B\).
2. **Selecting Elements**: We select the last \(k\) elements from both sorted arrays.
   - For array \(A\), these elements represent the largest values.
   - For array \(B\), these elements also represent the largest values.
3. **Calculating Result**: The product of the maximum value from \(A\) and the sum of values from \(B\) gives us the desired minimum product.

This approach ensures that we are always using the largest available values, which minimizes the overall product. The time complexity of this solution is \(O(N \log N)\) due to the sorting step, which is efficient enough given the problem constraints.
