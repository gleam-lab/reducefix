### Approaches
1. **Sorting and Priority Queue**:
   - First, sort the pairs (A_i, B_i) based on the values of A_i. This allows us to efficiently consider subsets where the maximum A_i is in a specific range.
   - For each possible maximum A_i in the sorted list, consider the subset of size K that includes this A_i and the K-1 smallest B_j values from the elements with A_j <= A_i. This ensures that the sum of B_i is minimized for each possible maximum A_i.
   - Use a priority queue (max heap) to keep track of the largest B_i values encountered so far. This helps in efficiently maintaining the sum of the smallest K-1 B_i values when combined with the current A_i.

### Solution Code

#include<bits/stdc++.h>
using namespace std;

using ll = long long;
const ll inf = 1LL << 60;

void solve() {
    int n, k; cin >> n >> k;
    vector<pair<ll, ll>> ab(n);
    for (int i = 0; i < n; i++) cin >> ab[i].first;
    for (int i = 0; i < n; i++) cin >> ab[i].second;
    sort(ab.begin(), ab.end());
    
    if (k == 1) {
        ll ans = inf;
        for (int i = 0; i < n; i++) {
            ans = min(ans, ab[i].first * ab[i].second);
        }
        cout << ans << '\n';
        return;
    }
    
    priority_queue<ll> q;
    ll sum = 0;
    for (int i = 0; i < k - 1; i++) {
        q.push(ab[i].second);
        sum += ab[i].second;
    }
    
    ll ans = inf;
    for (int i = k - 1; i < n; i++) {
        auto [a, b] = ab[i];
        ans = min(ans, a * (sum + b));
        if (!q.empty() && q.top() > b) {
            sum -= q.top();
            q.pop();
            sum += b;
            q.push(b);
        }
    }
    cout << ans << '\n';
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);
    int t; cin >> t;
    while (t--) solve();
    return 0;
}

### Explanation
- **Sorting**: The pairs (A_i, B_i) are sorted based on A_i to facilitate processing elements in order of increasing A_i.
- **Handling K=1**: For the special case where K=1, the solution simply iterates through all elements to find the minimum product of A_i and B_i.
- **Priority Queue**: For K > 1, a max heap is used to maintain the largest K-1 B_i values encountered so far. This allows efficient calculation of the sum of the smallest K-1 B_i values when combined with the current A_i (which is the maximum in the subset).
- **Updating the Sum**: As we iterate through the sorted list, we update the sum of B_i values by replacing the largest value in the heap with the current B_i if it is smaller, ensuring the sum remains minimized for each potential maximum A_i.
- **Result Calculation**: The result for each test case is the minimum value obtained by multiplying the current maximum A_i with the sum of the corresponding B_i values in the subset.

This approach ensures that we efficiently compute the desired minimum value for each test case while adhering to the constraints and providing optimal performance.
