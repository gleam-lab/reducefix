### Approaches

1. **Sorting by A_i**: First, we sort the pairs \(A_i\) and \(B_i\) in ascending order based on \(A_i\). This allows us to efficiently consider subsets where the maximum \(A_i\) is fixed at each step.
2. **Priority Queue for B_i**: We use a priority queue (max-heap) to maintain the top \(K-1\) largest \(B_i\) values encountered so far. This helps in efficiently calculating the sum of the largest \(B_i\) values when combined with the current \(A_i\).
3. **Sliding Window Technique**: For each element \(i\) from \(K-1\) to \(N-1\), we calculate the sum of the largest \(K-1\) \(B_i\) values seen so far and add the current \(B_i\). The product of this sum with the current \(A_i\) gives a candidate value for the minimum expression. We keep track of the minimum such value encountered during the iteration.

### Solution Code
```cpp
#include<bits/stdc++.h>
using namespace std;

using ll = long long;
const ll inf = 1LL << 60;

void solve() {
    int n, k; cin >> n >> k;
    vector<pair<ll, ll>> ab(n);
    for (int i = 0; i < n; ++i) cin >> ab[i].first;
    for (int i = 0; i < n; ++i) cin >> ab[i].second;
    sort(ab.begin(), ab.end());

    if (k == 1) {
        ll ans = inf;
        for (int i = 0; i < n; ++i) {
            ans = min(ans, ab[i].first * ab[i].second);
        }
        cout << ans << '\n';
        return;
    }

    priority_queue<ll, vector<ll>, greater<ll>> min_heap;
    ll sum = 0;

    for (int i = 0; i < k - 1; ++i) {
        min_heap.push(ab[i].second);
        sum += ab[i].second;
    }

    ll ans = inf;
    for (int i = k - 1; i < n; ++i) {
        ll current_sum = sum + ab[i].second;
        ans = min(ans, ab[i].first * current_sum);

        if (!min_heap.empty() && ab[i].second > min_heap.top()) {
            sum -= min_heap.top();
            sum += ab[i].second;
            min_heap.pop();
            min_heap.push(ab[i].second);
        }
    }

    cout << ans << '\n';
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);
    int t; cin >> t;
    while (t--) solve();
    return 0;
}
```

1. **Input Handling and Initialization**: The input is read and stored in a vector of pairs, where each pair contains \(A_i\) and \(B_i\). The pairs are then sorted by \(A_i\) to facilitate processing.
2. **Special Case Handling for \(K = 1\)**: If the subset size \(K\) is 1, the solution is simply the minimum product of \(A_i\) and \(B_i\) across all elements.
3. **Priority Queue Setup**: A min-heap is used to keep track of the largest \(B_i\) values encountered so far. The sum of these values is maintained to efficiently compute the sum of \(B_i\) values when combined with the current \(A_i\).
4. **Sliding Window Technique**: For each element from position \(K-1\) to \(N-1\), the current sum of the largest \(K-1\) \(B_i\) values plus the current \(B_i\) is computed. The product of this sum with the current \(A_i\) gives a candidate value for the minimum expression. The heap is adjusted to ensure it always contains the largest \(K-1\) \(B_i\) values up to the current position.
5. **Output the Result**: The minimum value encountered during the iteration is printed as the result for each test case.
