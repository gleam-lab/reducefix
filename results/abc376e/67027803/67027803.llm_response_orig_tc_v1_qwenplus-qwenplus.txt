To solve this problem efficiently, we need to find the minimum value of the expression:

$$
\left(\max_{i \in S} A_i\right) \times \left(\sum_{i \in S} B_i\right)
$$

for all subsets `S` of size `K`, where each element in the subset is defined by a pair `(A[i], B[i])`.

---

### **Key Observations:**

1. We're given `T` test cases.
2. For each test case:
   - We are given two arrays `A` and `B` of length `N`.
   - We need to choose a subset `S` of size `K` from these pairs.
   - Compute the product of the maximum `A[i]` in `S` and the sum of `B[i]` for all `i ∈ S`.
   - Our goal is to minimize this value over all such subsets.

---

### **Efficient Strategy (O(N log N)):**

1. Sort the pairs `(A[i], B[i])` in increasing order of `A[i]`.
2. Iterate through the sorted list, maintaining a max-heap of size `K-1` for the largest `K-1` values of `B[i]` seen so far.
3. At each index `i >= K-1`, consider using `A[i]` as the max in the current subset and combine it with the top `K-1` `B[i]` values.
4. Use a max-heap and maintain a running sum of those top `K-1` `B[i]` values.
5. Update the result if the current combination gives a better (smaller) result.

This ensures that for every possible candidate `A[i]` as the max in a valid subset, we try to minimize the sum of `B[i]` values by combining it with the best `K-1` elements with the highest `B[i]`.

### **Corrected C++ Code:**

```cpp
#include <bits/stdc++.h>
using namespace std;

using ll = long long;
const ll INF = 1e18;

void solve() {
    int n, k; cin >> n >> k;
    vector<pair<ll, ll>> ab(n);
    for (int i = 0; i < n; ++i) cin >> ab[i].first;
    for (int i = 0; i < n; ++i) cin >> ab[i].second;

    // Sort by A in ascending order
    sort(ab.begin(), ab.end());

    priority_queue<ll> max_heap;
    ll b_sum = 0;
    ll res = INF;

    // Initialize heap with first K-1 elements' B values
    for (int i = 0; i < k - 1; ++i) {
        max_heap.push(ab[i].second);
        b_sum += ab[i].second;
    }

    // Try every element from index K-1 onward as the max A
    for (int i = k - 1; i < n; ++i) {
        auto [a, b] = ab[i];
        b_sum += b;
        max_heap.push(b);

        // Maintain only top K-1 B values
        while (max_heap.size() > k - 1) {
            b_sum -= max_heap.top();
            max_heap.pop();
        }

        // Calculate current cost
        res = min(res, a * b_sum);

        // Restore the heap for next iteration
        max_heap.push(b);
        b_sum -= b;
    }

    cout << res << '\n';
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int t; cin >> t;
    while (t--) {
        solve();
    }
}
```

### **Explanation:**

- Sorting ensures that when we pick an element's `A[i]` as the max, all earlier elements can be considered for `B[i]` without violating the condition.
- The max-heap helps maintain the largest `K-1` values of `B[i]` efficiently.
- This approach guarantees we explore the best combinations while staying within acceptable time complexity.
