To solve this problem efficiently and correctly, we must carefully manage how we select the K elements that minimize:

$$
\left(\max_{i \in S} A_i\right) \times \left(\sum_{i \in S} B_i\right)
$$

### Key Observations:
1. We are given two arrays `A` and `B`, both of size N.
2. We must choose a subset `S` of size K such that the above expression is minimized.
3. The maximum `A[i]` in the subset multiplies the sum of corresponding `B[i]`.
4. A brute-force approach (trying all subsets) is infeasible due to constraints (`N` up to 2e5).

### Efficient Strategy:
- **Sort** the pairs `(A[i], B[i])` by increasing `A[i]`. This allows us to fix the maximum value of `A[i]` in the subset (as we process them in order), and then focus on minimizing the sum of selected `B[i]`.
- Use a **min-heap (priority queue)** to maintain the smallest `K-1` values of `B[i]` seen so far. As we iterate through the array starting from index `K-1`, we keep the top `K-1` smallest `B[i]` values to combine with the current element's `B[i]`.

This gives an efficient greedy solution with time complexity `O(N log N)` per test case due to sorting and heap operations.

### Fixed Code (Corrected Edge Cases and Logic):

```cpp
#include <bits/stdc++.h>
using namespace std;

using ll = long long;
const ll INF = 1e18;

void solve() {
    int n, k; cin >> n >> k;
    vector<pair<ll, ll>> ab(n);
    for (int i = 0; i < n; ++i) cin >> ab[i].first;
    for (int i = 0; i < n; ++i) cin >> ab[i].second;

    sort(ab.begin(), ab.end());

    if (k == 1) {
        ll ans = INF;
        for (auto [a, b] : ab)
            ans = min(ans, a * b);
        cout << ans << '\n';
        return;
    }

    priority_queue<ll> max_heap; // max-heap to keep track of the largest among the smallest K-1 B values
    ll sum_b = 0;
    for (int i = 0; i < k - 1; ++i) {
        sum_b += ab[i].second;
        max_heap.push(ab[i].second);
    }

    ll result = INF;
    for (int i = k - 1; i < n; ++i) {
        ll curr_a = ab[i].first;
        ll curr_b = ab[i].second;

        // Try forming a subset with current A[i] as the max A and selecting smallest K-1 B's
        result = min(result, (sum_b + curr_b) * curr_a);

        // Maintain only the K-1 smallest B's so far
        if (!max_heap.empty() && max_heap.top() > curr_b) {
            sum_b -= max_heap.top();
            sum_b += curr_b;
            max_heap.pop();
            max_heap.push(curr_b);
        }
    }

    cout << result << '\n';
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int t; cin >> t;
    while (t--) {
        solve();
    }

    return 0;
}
```

### Explanation:
- We sort the pairs by `A[i]` because we want to consider each possible candidate for the max `A[i]` in increasing order.
- For each such `A[i]`, we try to pair it with the smallest `K-1` values of `B[i]` encountered so far.
- The use of a **max-heap** helps in maintaining the smallest `K-1` `B[i]` values — allowing replacement of the largest one if a smaller new `B[i]` is found.

This approach ensures correctness even in edge cases like duplicate values or when all `B[i]` values are large/small.
