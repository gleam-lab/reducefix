
### Problem Analysis

The problem requires finding the minimum value of the expression \((\max_{i \in S} A_i) \times (\sum_{i \in S} B_i)\) for a given subset \(S\) of indices from sequences \(A\) and \(B\). The subset \(S\) has a size \(K\). Given the constraints, a direct approach that evaluates all possible subsets of size \(K\) would be computationally infeasible due to the high potential number of combinations.

### Solution Strategy

The key to solving this problem efficiently lies in understanding how the maximum value of \(A\) and the sum of \(B\) over a subset \(S\) can be minimized. The strategy involves:

1. **Sorting and Pairing**: Start by pairing elements from \(A\) and \(B\) and sort these pairs in descending order based on the product of the elements in each pair. This helps in quickly identifying the most promising candidates for the maximum value of \(A\) in the subset \(S\).

2. **Iterative Selection**: Use an iterative approach to select the top \(K\) elements from the sorted list. For each selection, calculate the potential minimum value of the expression and update the answer accordingly.

3. **Efficient Sum Calculation**: As the subset \(S\) grows, efficiently calculate the sum of \(B\) for the current subset without recalculating from scratch. This can be achieved using a running sum or a similar technique.

### Implementation Details

Here's a step-by-step breakdown of the implementation:

1. **Input Reading**: Read the input values for \(N\), \(K\), \(A\), and \(B\).

2. **Pairing and Sorting**: Pair each element from \(A\) with the corresponding element from \(B\) and store these pairs. Sort the pairs in descending order based on their product.

3. **Subset Selection**:
   - Initialize variables for the answer and the running sum of \(B\).
   - Iterate through the sorted pairs, adding each pair to the subset \(S\) and updating the running sum of \(B\).
   - After each addition, calculate the potential minimum value of the expression using the current maximum value of \(A\) from the subset and the running sum of \(B\).
   - Update the answer if the current potential minimum is lower than the previously recorded answer.

4. **Output**: After processing all pairs for a test case, print the minimum value obtained.

This approach ensures that we efficiently find the minimum value by leveraging sorting and an iterative subset evaluation process.

```cpp
#include <iostream>
#include <vector>
#include <algorithm>
#include <numeric>

using namespace std;

int main() {
    int T;
    cin >> T;
    while (T--) {
        int N, K;
        cin >> N >> K;
        vector<pair<long long, long long>> AB(N);
        for (int i = 0; i < N; i++) {
            cin >> AB[i].first >> AB[i].second;
        }
        
        // Sort pairs by their product in descending order
        sort(AB.begin(), AB.end(), [](const pair<long long, long long>& a, const pair<long long, long long>& b) {
            return a.first * a.second > b.first * b.second;
        });
        
        long long min_value = LLONG_MAX;
        long long sum_B = 0;
        
        // Evaluate subsets
        for (int i = 0; i < N; i++) {
            if (N - i - 1 >= K - 1) {
                sum_B += AB[i].second;
                long long max_A = AB[i].first;
                long long current_value = max_A * sum_B;
                min_value = min(min_value, current_value);
            }
        }
        
        cout << min_value << '\n';
    }
    return 0;
}
```

This refined solution efficiently calculates the minimum value for the given problem by focusing on the most promising candidates for the maximum value of \(A\) and using an efficient subset evaluation strategy.
