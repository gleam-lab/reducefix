To address the failing cases, we need to rethink our approach to solving this problem efficiently. Let's break down the requirements and constraints:

1. **Objective**: Find the minimum value of the expression \((\max_i A_i) \times (\sum_j B_j)\) where \(S\) is a subset of \([1, N]\) with size \(K\).
2. **Constraints**:
   - \(1 \leq T \leq 2 \times 10^5\)
   - \(1 \leq K \leq N \leq 2 \times 10^5\)
   - \(1 \leq A_i, B_i \leq 10^6\)
   - Sum of \(N\) over all test cases ≤ \(2 \times 10^5\)

Given these constraints, a brute-force approach that tries all subsets of size \(K\) would be too slow. Instead, we can use a greedy approach combined with sorting and a sliding window technique.

### Steps to Solve:
1. **Sort the Arrays**: Sort both `A` and `B` arrays based on their values.
2. **Sliding Window Approach**: Use two pointers to maintain a sliding window of size \(K\) on the sorted array `A`. For each position of the window, calculate the sum of corresponding elements in `B` and update the result accordingly.

This approach ensures that we efficiently find the minimum value without explicitly generating all subsets.

### Implementation:
```cpp
#include <bits/stdc++.h>
using namespace std;

using ll = long long;
const int INF = 1e9 + 10;
const ll inf = 1LL << 60;

void solve() {
  int n, k; cin >> n >> k;
  vector<int> a(n), b(n);
  for (int i = 0; i < n; i++) cin >> a[i];
  for (int i = 0; i < n; i++) cin >> b[i];

  // Sort A and B based on their values
  sort(a.begin(), a.end());
  sort(b.begin(), b.end());

  // Initialize variables for the sliding window
  ll sum_b = 0;
  for (int i = 0; i < k; i++) sum_b += b[i];

  // Initial answer
  ll ans = (ll)a[k - 1] * sum_b;

  // Slide the window across the array
  for (int i = k; i < n; i++) {
    sum_b += b[i] - b[i - k];
    ans = min(ans, (ll)a[i] * sum_b);
  }

  cout << ans << '\n';
}

int main() {
  ios::sync_with_stdio(false);
  cin.tie(nullptr);
  int t = 1; cin >> t;
  while (t--) solve();
}
```

### Explanation:
1. **Sorting**: We sort both `A` and `B` to facilitate the sliding window approach.
2. **Initial Calculation**: We initialize the sum of the first \(K\) elements of `B`.
3. **Sliding Window**: We slide the window from left to right, updating the sum of `B` and recalculating the result using the current maximum element of `A`.

This approach ensures that we efficiently find the minimum value without generating all subsets, making it suitable for large inputs as specified in the constraints.
