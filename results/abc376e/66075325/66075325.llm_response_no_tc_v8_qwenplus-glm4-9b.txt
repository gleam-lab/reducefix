
```cpp
#include <iostream>
#include <vector>
#include <algorithm>
using namespace std;

#define MAXN 200010
long long A[MAXN], B[MAXN];
vector<pair<long long, long long>> V;

int main() {
    int T, N, K;
    cin >> T;
    while (T--) {
        cin >> N >> K;
        V.clear();

        for (int i = 0; i < N; i++) {
            cin >> A[i];
        }
        for (int i = 0; i < N; i++) {
            cin >> B[i];
            V.emplace_back(B[i], A[i]);
        }

        sort(V.begin(), V.end());

        long long ans = LLONG_MAX;
        long long sumB = 0, sumA = 0;
        int count = 0;

        // Traverse from the end to find the minimum possible value
        for (int i = N - 1; i >= 0; i--) {
            sumB += V[i].first;
            sumA += V[i].second;
            count++;

            // If we have more than K elements, we can start removing
            if (count > K) {
                sumA -= V[i + K].second;
                sumB -= V[i + K].first;
                count--;
            }

            // If we have exactly K elements, we can calculate the current answer
            if (count == K) {
                ans = min(ans, sumB * V[i].second);
            }
        }

        cout << ans << endl;
    }

    return 0;
}
```

### Explanation:
1. **Input and Initialization**: We read the number of test cases `T`. For each test case, we read `N` and `K`, and then the arrays `A` and `B`. We store the pairs `(B[i], A[i])` in a vector `V` and sort it based on `B[i]`.

2. **Processing Each Test Case**:
   - We initialize `ans` to `LLONG_MAX`, `sumB` to 0, `sumA` to 0, and `count` to 0.
   - We iterate from the end of the sorted vector `V` to the start. For each element, we add its `B` value to `sumB` and its `A` value to `sumA`.
   - If `count` exceeds `K`, we remove the `B` and `A` values of the element that is `K` positions ahead in the vector and decrement `count`.
   - If `count` equals `K`, we calculate the product of `sumB` and the `A` value of the current element, and update `ans` if this product is smaller than the current `ans`.
   - This process ensures that we always consider the smallest `B` values while maintaining a size of `K` or more in our subset.

3. **Output**: After processing all elements for a test case, we output the value of `ans`.
