To address this problem efficiently, we need to focus on finding an optimal way to select a subset of indices such that the product of the maximum element in the subset and the sum of corresponding elements in another array is minimized.             b[i].second = i;
        }

        // Sort arrays based on A[i] and B[i]
        sort(a.begin(), a.end());
        sort(b.begin(), b.end());

        // Priority queue to keep track of current sums
        priority_queue<long long, vector<long long>, greater<long long>> pq;
        long long totalSum = 0;

        // Push first K elements into the priority queue
        for (int i = 0; i < k; ++i) {
            pq.push(b[a[i].second].first);
            totalSum += b[a[i].second].first;
        }

        long long ans = INF;
        // Calculate initial answer
        ans = min(ans, (totalSum + b[a[k].second].first) * a[k].first);

        // Iterate through remaining elements
        for (int i = k; i < n; ++i) {
            totalSum += b[a[i].second].first - pq.top();
            pq.pop();
            pq.push(b[a[i].second].first);
            ans = min(ans, (totalSum + b[a[i].second].first) * a[i].first);
        }

### Explanation:
1. **Sorting**: We sort both `a` and `b` based on their respective first elements (`A_i` and `B_i`). This allows us to efficiently pick the smallest elements from both arrays.

2. **Priority Queue**: We use a priority queue to keep track of the sums of selected elements. Initially, we push the first `K` elements from the sorted `b` array into the priority queue and calculate the initial sum.

3. **Sliding Window**: As we iterate through the remaining elements, we update the sum by removing the smallest element currently in the queue and adding the next smallest element from `b`. This ensures we always have the smallest possible sum for the chosen subset.

4. **Minimum Calculation**: We calculate the product for each iteration and keep track of the minimum value.

This approach ensures that we efficiently find the minimum product with a time complexity of \(O(N \log N)\) due to sorting and priority queue operations.
