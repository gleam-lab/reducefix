2. **Priority Queue for Sum Calculation**: For each element \( A_i \), if it is the maximum in the subset \( S \), the other \( K-1 \) elements must be chosen from elements with \( A_j \leq A_i \). To minimize the sum of \( B \) values, we should select the smallest \( K-1 \) \( B \) values from these elements.
3. **Efficient Sum Maintenance**: We use a max-heap (priority queue) to maintain the largest \( K-1 \) \( B \) values among the elements processed so far. This allows us to efficiently compute the sum of the smallest \( K \) \( B \) values (including \( B_i \)) when \( A_i \) is the maximum.

            a[i].second = i;
        }
        vector<int> b(n);
        for (int i = 0; i < n; ++i) {
            cin >> b[i];
        }
        sort(a.begin(), a.end());
        vector<int> sorted_b(n);
        for (int i = 0; i < n; ++i) {
            sorted_b[i] = b[a[i].second];
        }
        priority_queue<int> pq;
        int sum = 0;
        int ans = inf;
        for (int i = 0; i < n; ++i) {
            int current_sum = sum + sorted_b[i];
            if (pq.size() == k - 1) {
                ans = min(ans, a[i].first * current_sum);
            }
            pq.push(sorted_b[i]);
            sum += sorted_b[i];
            if (pq.size() > k - 1) {
                sum -= pq.top();
                pq.pop();
            }
        }
        cout << ans << '\n';
    }

}

1. **Sorting and Pairing**: The elements of array \( A \) are sorted, and the corresponding \( B \) values are rearranged to maintain their original pairing with \( A \).
2. **Priority Queue for Sum Calculation**: As we iterate through the sorted \( A \) array, we maintain a max-heap of the \( B \) values. This heap helps us keep track of the smallest \( K-1 \) \( B \) values processed so far (since we pop the largest values when the heap size exceeds \( K-1 \)).
3. **Sum Maintenance**: For each element \( A_i \), if it is considered as the maximum in the subset \( S \), the sum of \( B \) values in \( S \) is the sum of \( B_i \) and the sum of the smallest \( K-1 \) \( B \) values from elements before \( i \). This sum is computed efficiently using the heap.
4. **Edge Cases Handling**: The algorithm correctly handles cases where \( N = 1 \) or \( K = 1 \) by ensuring that the sum and heap operations are appropriately managed, avoiding incorrect initializations or overflow issues.
