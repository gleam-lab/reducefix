### Approach
1. **Sort the Elements**: First, we sort the pairs \((A_i, B_i)\) based on the values of \(A_i\) in ascending order. This allows us to consider each \(A_i\) as the potential maximum \(A\) value in the subset \(S\).
2. **Precompute Prefix Sums**: For each element \(A_i\) (after sorting), we consider it as the maximum in the subset \(S\). The subset \(S\) must include \(A_i\) and \(K-1\) other elements with \(A_j \leq A_i\). To minimize the sum \(\sum B_i\) for these \(K\) elements, we should select the \(K-1\) smallest \(B_j\) values among the first \(i-1\) elements.
3. **Use a Priority Queue**: To efficiently keep track of the largest \(B_j\) values among the first \(i-1\) elements, we use a max-heap (priority queue). This allows us to maintain the sum of the smallest \(K-1\) \(B_j\) values by removing the largest ones if the heap size exceeds \(K-1\).

#include <iostream>
#include <vector>
#include <queue>
#include <algorithm>
using namespace std;

const int inf = 1e18;

signed main() {
    ios_base::sync_with_stdio(false);
    cin.tie(NULL);
    int t;
    cin >> t;
    while (t--) {
        int n, k;
        cin >> n >> k;
        vector<pair<int, int>> a(n);
        for (int i = 0; i < n; ++i) {
            cin >> a[i].first;
        }
        for (int i = 0; i < n; ++i) {
            cin >> a[i].second;
        }
        sort(a.begin(), a.end());
        
        priority_queue<int> pq;
        int sum = 0;
        int ans = inf;
        
        for (int i = 0; i < n; ++i) {
            int current = a[i].first;
            int current_sum = sum + a[i].second;
            if (pq.size() == k - 1) {
                ans = min(ans, current * current_sum);
            }
            pq.push(a[i].second);
            sum += a[i].second;
            if (pq.size() >= k) {
                sum -= pq.top();
                pq.pop();
            }
        }
        cout << ans << '\n';
    }
    return 0;
}

1. **Sorting**: The pairs \((A_i, B_i)\) are sorted by \(A_i\) to process each element in ascending order. This allows us to consider each \(A_i\) as the maximum in the subset \(S\) of size \(K\).

3. **Result Calculation**: For each \(A_i\), the potential result is computed as \(A_i \times (\sum B_i)\), where the sum includes \(B_i\) and the smallest \(K-1\) \(B_j\) values from the previous elements. The minimum of these products across all \(A_i\) is the answer for each test case.
