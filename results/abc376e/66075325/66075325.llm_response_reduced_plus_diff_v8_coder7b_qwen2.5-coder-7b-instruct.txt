To solve this problem efficiently, we need to minimize the expression \((\max_i A_i) \times (\sum_i B_i)\) for a subset \(S\) of size \(K\). We can achieve this by selecting the largest \(K\) elements from array \(A\) and pairing them with the smallest \(K\) elements from array \(B\). This approach ensures that the product is minimized because it balances the maximum element from \(A\) with the smallest corresponding sum from \(B\).

Here's the corrected and optimized C++ code:

```cpp
#include<iostream>
#include<vector>
#include<algorithm>
using namespace std;

int main() {
    ios_base::sync_with_stdio(false); cin.tie(0); cout.tie(0);

    int t;
    cin >> t;
    while (t--) {
        int n, k;
        cin >> n >> k;
        
        vector<int> a(n), b(n);
        for (int i = 0; i < n; ++i) cin >> a[i];
        for (int i = 0; i < n; ++i) cin >> b[i];

        // Sort arrays
        sort(a.begin(), a.end());
        sort(b.begin(), b.end());

        // Calculate the result
        long long res = 0;
        for (int i = 0; i < k; ++i) {
            res += static_cast<long long>(a[n - 1 - i]) * b[i];
        }

        cout << res << '\n';
    }

    return 0;
}
```

### Explanation:
1. **Input Reading**: Read the number of test cases \(T\). For each test case, read \(N\) and \(K\), followed by the arrays \(A\) and \(B\).
2. **Sorting**: Sort both arrays \(A\) and \(B\). Sorting takes \(O(N \log N)\) time.
3. **Result Calculation**: Iterate through the first \(K\) elements of the sorted array \(A\) and the last \(K\) elements of the sorted array \(B\). Multiply the corresponding elements and accumulate the result.
4. **Output**: Print the result for each test case.

This approach ensures that we are always pairing the largest element from \(A\) with the smallest element from \(B\), thus minimizing the overall product. The time complexity is dominated by the sorting step, making it efficient enough for the given constraints.
