To solve this problem efficiently, we must find the minimum possible value of the expression:

$$
\left(\max_{i \in S} A_i\right) \times \left(\sum_{i \in S} B_i\right)
$$

Where `S` is a subset of size `K` from the given sequences `A` and `B`.

---

### Key Observations:
1. We want to select K elements such that the **maximum A** in the subset multiplied by the **sum of B** in the subset is minimized.
2. A brute-force approach is infeasible due to constraints (`N up to 2e5`, sum over all test cases).
3. The optimal strategy involves sorting the pairs `(A_i, B_i)` in increasing order of `A_i`. Then, for each index `i` (from `K-1` to `N-1`), consider `A[i]` as the max A-value in the subset and try to pick the K smallest `B` values among the first `i` items.

---

### Efficient Strategy:
- Sort the array of `(A_i, B_i)` based on `A_i`.
- Use a **min-heap** to maintain the top K smallest B-values while iterating through the sorted list.
- At each step:
  - Add current `B[i]` to the heap.
  - If heap size exceeds `K`, remove the largest `B` so far (to keep only K smallest ones).
  - When heap has exactly `K` elements, calculate: `A[i] * sum_of_heap` and update the answer.

This ensures an overall time complexity of **O(N log K)** per test case — acceptable under given constraints.

---

### Fixed & Optimized Code:
```cpp
#include <iostream>
#include <vector>
#include <queue>
#include <algorithm>

using namespace std;

#define int long long

signed main() {
    ios::sync_with_stdio(false);
    cin.tie(0);
    cout.tie(0);

    int T;
    cin >> T;

    while (T--) {
        int N, K;
        cin >> N >> K;

        vector<pair<int, int>> items(N);
        for (int i = 0; i < N; ++i) {
            cin >> items[i].first; // A_i
        }
        for (int i = 0; i < N; ++i) {
            cin >> items[i].second; // B_i
        }

        // Sort by A_i ascending
        sort(items.begin(), items.end());

        priority_queue<int> max_heap; // To maintain K smallest B's (as max heap of K smallest)
        int sum_b = 0;
        int ans = 1LL << 60; // Large number

        for (int i = 0; i < N; ++i) {
            int a = items[i].first;
            int b = items[i].second;

            sum_b += b;
            max_heap.push(b);

            // Maintain heap size <= K
            if (max_heap.size() > K) {
                sum_b -= max_heap.top();
                max_heap.pop();
            }

            // Once we have K elements, compute cost
            if (max_heap.size() == K) {
                ans = min(ans, a * sum_b);
            }
        }

        cout << ans << "\n";
    }

    return 0;
}
```

---

### Why This Works:
- Sorting by `A` ensures we always treat the current `A[i]` as the maximum in the potential subset.
- Using a max-heap helps us maintain the **K smallest B values** seen so far — crucial for minimizing the total sum.
- Each operation inside the loop is `O(log K)`, leading to efficient execution.

---

### Edge Cases Handled:
- K = 1 → Just take the minimum of all `A[i] * B[i]`
- All `A` or `B` values same → Still handled correctly
- Input with large values → Uses 64-bit integers
