### Analysis of the Original Code
1. **Sorting and Pairing**: The original code sorts the pairs \((A_i, B_i)\) based on \(A_i\) and then processes them. However, the sorting comparator is incorrect as it doesn't handle ties properly and doesn't ensure a consistent order.

Specifically, it doesn't correctly maintain the sum of the \(K\) smallest \(B_i\) values for each possible maximum \(A_i\).

1. **Sort by \(A_i\)**: First, sort the elements in increasing order of \(A_i\). This allows us to consider each \(A_i\) as the potential maximum in the subset \(S\).
2. **Maintain a Priority Queue for \(B_i\)**: For each element \(i\) (starting from the \(K\)-th element), we need to consider the \(K\) elements with the smallest \(B_i\) values up to \(i\).
