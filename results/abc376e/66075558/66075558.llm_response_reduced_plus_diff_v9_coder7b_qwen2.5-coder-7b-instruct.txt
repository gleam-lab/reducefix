To solve this problem efficiently, we need to find the minimum value of the expression \((\max_{i \in S} A_i) \times (\sum_{i \in S} B_i)\) for a subset \(S\) of size \(K\). Given the constraints, a brute-force approach is not feasible due to its exponential time complexity. Instead, we can use a greedy algorithm combined with sorting and a min-heap to achieve an efficient solution.

Here's the corrected C++ code that addresses the given requirements:

```cpp
#include <iostream>
#include <vector>
#include <queue>
#include <algorithm>
using namespace std;

#define int long long

struct Node {
    int a, b;
    Node(int _a, int _b) : a(_a), b(_b) {}
};

bool cmp(const Node &a, const Node &b) {
    if (a.a == b.a) return a.b > b.b;
    return a.a < b.a;
}

int main() {
    ios::sync_with_stdio(false); cin.tie(nullptr);
    int t;
    cin >> t;
    while (t--) {
        int n, k;
        cin >> n >> k;
        vector<Node> a(n);
        for (int i = 0; i < n; ++i) {
            cin >> a[i].a;
        }
        for (int i = 0; i < n; ++i) {
            cin >> a[i].b;
        }
        
        // Sort nodes by a in ascending order, and by b in descending order for same a
        sort(a.begin(), a.end(), cmp);
        
        // Priority queue to maintain the sum of B values
        priority_queue<int, vector<int>, greater<int>> pq;
        int sum_b = 0;
        
        // Initialize the first K elements
        for (int i = 0; i < k; ++i) {
            pq.push(a[i].b);
            sum_b += a[i].b;
        }
        
        // Result will store the minimum value
        int result = sum_b * a[k - 1].a;
        
        // Try replacing the smallest element in the current set with the next largest one
        for (int i = k; i < n; ++i) {
            sum_b -= pq.top();
            pq.pop();
            pq.push(a[i].b);
            sum_b += a[i].b;
            result = min(result, sum_b * a[i].a);
        }
        
        cout << result << '\n';
    }
    return 0;
}
```

### Explanation
1. **Sorting**: We first sort the pairs `(A[i], B[i])` by `A[i]` in ascending order. If two pairs have the same `A[i]`, we sort them by `B[i]` in descending order. This ensures that we prioritize larger `B[i]` values for smaller `A[i]` values.

2. **Min-Heap**: We use a min-heap to keep track of the sum of the `B[i]` values for the current subset of size `K`. Initially, we push the first `K` `B[i]` values into the heap and calculate their sum.

3. **Greedy Approach**: For each subsequent element, we replace the smallest `B[i]` value in the heap with the current `B[i]` value. We update the sum and recalculate the result. This ensures that we always consider the best possible combination of `B[i]` values for each `A[i]`.

4. **Result Calculation**: We keep track of the minimum result obtained during the process.

This approach ensures that we find the minimum value of the expression in \(O(N \log K)\) time complexity, which is efficient enough given the constraints.
