The initial approach has several issues, primarily with the logic used to select the subset of size K and calculate the minimum expression value. Here's the corrected approach:

### Correct Approach
1. **Sort Elements by A_i**: First, we sort all elements in non-decreasing order of A_i to systematically consider the maximum A_i in any subset.
2. **Use a Min-Heap for Efficient Sum Calculation**: For each element as the potential maximum A_i in the subset, we maintain the largest K-1 B_i values from the previous elements. This allows us to efficiently compute the sum of the top K-1 B_i values plus the current B_i, which together form the sum of the subset where the current A_i is the maximum.
3. **Calculate the Minimum Expression**: For each element in the sorted list (from index K to N), we calculate the expression value (A_i multiplied by the sum of the selected B_i values) and keep track of the minimum value encountered.

### Solution Code
```cpp
#include <iostream>
#include <vector>
#include <queue>
#include <algorithm>
using namespace std;
#define int long long
const int inf = 1e18;

signed main() {
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);
    int T;
    cin >> T;
    while (T--) {
        int N, K;
        cin >> N >> K;
        vector<pair<int, int>> elements(N);
        for (int i = 0; i < N; ++i) {
            cin >> elements[i].first;
        }
        for (int i = 0; i < N; ++i) {
            cin >> elements[i].second;
        }
        
        sort(elements.begin(), elements.end());
        
        priority_queue<int, vector<int>, greater<int>> min_heap;
        int sum_B = 0;
        int answer = inf;
        
        for (int i = N - 1; i >= 0; --i) {
            int current_A = elements[i].first;
            int current_B = elements[i].second;
            
            sum_B += current_B;
            min_heap.push(current_B);
            
            if (min_heap.size() > K) {
                sum_B -= min_heap.top();
                min_heap.pop();
            }
            
            if (min_heap.size() == K) {
                answer = min(answer, current_A * sum_B);
            }
        }
        
        cout << answer << '\n';
    }
    return 0;
}
```

### Explanation
1. **Sorting**: The elements are sorted based on their A_i values in non-decreasing order. This allows us to consider each element in turn as the maximum A_i in a subset of size K.
2. **Min-Heap**: A min-heap is used to keep track of the largest K-1 B_i values encountered so far. As we iterate from the largest A_i to the smallest, we add each B_i to the heap and maintain the sum of the top K elements.
3. **Efficiency**: The use of a min-heap ensures that we efficiently manage the sum of the top B_i values, allowing us to compute the required expression in O(N log K) time per test case, which is optimal given the constraints.
4. **Edge Cases**: The solution handles cases where K = 1 or K = N by naturally integrating them into the general approach, ensuring correctness across all scenarios.
