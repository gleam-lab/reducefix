
To solve the problem, we need to find the minimum possible value of the expression:

\[
\left(\max\_{i \in S} A\_i\right) \times \left(\sum\_{i \in S} B\_i\right).
\]

Given the constraints, the solution needs to be efficient. The naive approach would involve checking all possible subsets of size \( K \), which is computationally infeasible due to the factorial growth of the number of subsets.

### Strategy

To achieve an efficient solution, we can leverage the observation that to minimize the product, we should minimize the maximum value in the subset \( S \) and maximize the sum of the subset \( S \). This suggests sorting the elements and using a sliding window or a greedy strategy to maintain the minimum maximum element and maximize the sum.

1. **Pairing and Sorting**: First, pair the elements \( A \) and \( B \) together and sort them based on \( A \) in ascending order. This ensures that when we choose the maximum \( A \) for a subset, it comes from the last \( K \) elements of the sorted list.

2. **Prefix Sum**: Compute the prefix sum of the sorted list of \( B \). This allows us to quickly calculate the sum of any subset of size \( K \).

3. **Sliding Window**: Use two pointers to implement a sliding window that maintains the sum of the last \( K \) elements of \( B \). Slide the window from the end of the list towards the beginning.

4. **Calculate the Expression**: For each position of the sliding window, calculate the expression with the current maximum \( A \) and the current sum of \( B \) elements. Keep track of the minimum value found.

### Implementation

```cpp
#include <iostream>
#include <vector>
#include <algorithm>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(0);
    cout.tie(0);
    
    int t;
    cin >> t;
    while (t--) {
        int n, k;
        cin >> n >> k;
        vector<pair<int, int>> v(n);
        
        for (int i = 0; i < n; ++i) {
            cin >> v[i].first >> v[i].second;
        }
        
        sort(v.begin(), v.end());
        
        vector<int> prefixSum(n + 1, 0);
        for (int i = 0; i < n; ++i) {
            prefixSum[i + 1] = prefixSum[i] + v[i].second;
        }
        
        double ans = 1e18;
        int j = n - k;
        for (int i = j; i >= 0; --i) {
            double sum = (double)(prefixSum[n] - prefixSum[j]);
            double maxA = (double)v[i].first;
            ans = min(ans, maxA * sum);
            j--;
        }
        
        cout << fixed << setprecision(10) << ans << '\n';
    }
    
    return 0;
}
```

This solution sorts the pairs, computes the prefix sum, and then uses a sliding window to find the minimum possible value of the expression. The time complexity is dominated by the sorting step, which is \( O(N \log N) \). This is efficient enough given the constraints.
