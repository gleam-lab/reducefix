To solve this problem efficiently, we need to minimize the expression \((\max_{i \in S} A_i) \times (\sum_{i \in S} B_i)\) for a subset \(S\) of size \(K\). 

One efficient way to approach this problem is to use a greedy strategy combined with sorting. Here's how you can do it:

1. **Sort the Arrays**: Sort array \(A\) in descending order and array \(B\) in ascending order.
2. **Select Elements**: Select the first \(K\) elements from the sorted \(A\) and last \(K\) elements from the sorted \(B\).
3. **Calculate the Result**: Compute the product of the maximum element in \(A\) and the sum of the selected elements in \(B\).

This approach ensures that we are balancing between the maximum value in \(A\) and the total sum in \(B\) in an optimal manner.

Here's the corrected C++ code implementing this logic:

```cpp
#include <bits/stdc++.h>
using namespace std;

const int maxN = 2e5 + 5;
int t, n, k, a[maxN], b[maxN];

int main() {
    ios_base::sync_with_stdio(0);
    cin.tie(0);
    cout.tie(0);

    cin >> t;
    while (t--) {
        cin >> n >> k;
        vector<pair<int, int>> pairs(n);

        // Read input into pairs
        for (int i = 0; i < n; ++i) {
            cin >> a[i];
            pairs[i] = {a[i], b[i]};
        }

        // Sort pairs based on A in descending order and B in ascending order
        sort(pairs.begin(), pairs.end(), [&](const pair<int, int>& p1, const pair<int, int>& p2) {
            if (p1.first == p2.first) return p1.second < p2.second;
            return p1.first > p2.first;
        });

        long long maxA = pairs[k - 1].first;
        long long sumB = 0;
        
        // Calculate the sum of the smallest K elements in B
        for (int i = 0; i < k; ++i) {
            sumB += pairs[n - 1 - i].second;
        }

        // Output the result
        cout << maxA * sumB << "\n";
    }
    return 0;
}
```

### Explanation:
1. **Sorting**: We sort the pairs of `(A_i, B_i)` such that `A` is in descending order and `B` is in ascending order.
2. **Selection**: We select the largest `K` elements from `A` and the smallest `K` elements from `B`.
3. **Calculation**: We compute the product of the maximum element in `A` and the sum of the selected elements in `B`.

This approach ensures that we get the minimum possible value of the expression for any subset of size `K`. The time complexity of this solution is \(O(N \log N)\), which is efficient given the constraints.
