To solve this problem efficiently, we need to minimize the expression \((\max\_{i \in S} A\_i) \times (\sum\_{i \in S} B\_i)\) where \(S\) is a subset of \([1, N]\) with size \(K\). This can be approached using a greedy algorithm combined with sorting and a min-heap (priority queue).

Here's a step-by-step plan:
1. Pair each element \(A_i\) with its corresponding element \(B_i\) and store these pairs.
2. Sort these pairs primarily by \(A_i\) in descending order. If two pairs have the same \(A_i\), sort them by \(B_i\) in ascending order.
3. Use a min-heap to keep track of the smallest sums of \(B_i\) values.
4. Iterate through the sorted list and maintain a heap of size \(K\). For each pair, calculate the product and update the result accordingly.

This approach ensures that we always consider the largest \(A_i\) first and balance it with the smallest \(B_i\)s to minimize the overall product.

Here is the corrected and optimized C++ code:

```cpp
#include <bits/stdc++.h>
using namespace std;

const int maxN = 2e5 + 5;
typedef long long ll;

struct Node {
    int a, b;
};

bool cmp(const Node &x, const Node &y) {
    if (x.a != y.a) return x.a > y.a;
    return x.b < y.b;
}

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);

    int t;
    cin >> t;
    while (t--) {
        int n, k;
        cin >> n >> k;
        
        vector<Node> v(n);
        for (int i = 0; i < n; ++i) {
            cin >> v[i].a >> v[i].b;
        }

        sort(v.begin(), v.end(), cmp);

        priority_queue<int, vector<int>, greater<int>> pq;
        ll sum = 0, ans = LLONG_MAX;

        for (int i = 0; i < n; ++i) {
            pq.push(v[i].b);
            sum += v[i].b;

            if (pq.size() > k) {
                sum -= pq.top();
                pq.pop();
            }

            if (pq.size() == k) {
                ans = min(ans, static_cast<ll>(v[i].a) * sum);
            }
        }

        cout << ans << "\n";
    }

    return 0;
}
```

### Explanation:
1. **Struct Definition**: We define a `Node` structure to hold pairs of \(A_i\) and \(B_i\).
2. **Sorting**: We sort the nodes first by \(A_i\) in descending order. If two nodes have the same \(A_i\), we sort them by \(B_i\) in ascending order.
3. **Min-Heap**: We use a min-heap (`priority_queue`) to keep track of the smallest sums of \(B_i\) values.
4. **Iterate and Update**: As we iterate through the sorted list, we add each \(B_i\) to the sum and push it into the heap. If the heap size exceeds \(K\), we remove the smallest element from the heap and subtract it from the sum.
5. **Result Calculation**: Whenever the heap size equals \(K\), we calculate the product and update the result if it's smaller than the current minimum.

This approach ensures that we efficiently find the minimum value of the required expression within the given constraints.
