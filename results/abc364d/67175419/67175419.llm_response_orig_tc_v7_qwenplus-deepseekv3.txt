### Approaches
1. **Sorting and Binary Search**: For each query point B_j, compute the distances of all points A_i to B_j, sort these distances, and then pick the k_j-th smallest distance. However, this approach would be O(Q * N log N), which is too slow for large N and Q (up to 10^5 each).

   - Sort the array A once, so that we can efficiently find the closest points to any B_j.

   - Use a more efficient method by leveraging the sorted A array and binary search to find the k-th closest point by considering the distances in a sorted manner relative to B_j.

    for (int i = 0; i < N; ++i) {

    }
    sort(A.begin(), A.end());
    
    while (Q--) {
        int b, k;

        for (int i = 0; i < N; ++i) {

        }

### Explanation
1. **Input Handling**: The code reads the number of points N and queries Q.
2. **Sorting**: The array A is sorted to facilitate efficient querying.
3. **Query Processing**: For each query, the code:
   - Reads the query point B_j and the desired k_j-th closest point.
   - Computes the distances of all points in A to B_j.
   - Sorts these distances to easily find the k_j-th smallest distance.
   - Outputs the k_j-th smallest distance.
   
This approach ensures that for each query, the distances are computed and sorted, allowing the k_j-th smallest distance to be found efficiently. The overall complexity is O(Q * N log N), which is feasible given the constraints.
