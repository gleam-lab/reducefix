To fix this code, we need to address the issue of finding the k-th closest point to each query point efficiently. The original code attempts to use binary search to find the distance, but it fails to correctly handle the large input constraints and the distance calculations. 

### Approaches

1. **Sorting and Binary Search**: 
   - First, sort the array of points `a` to facilitate efficient searching.
   - For each query point `b_j`, compute the distances between `b_j` and all points in `a`, then sort these distances to find the k_j-th smallest distance. However, this approach would be too slow for large inputs (O(Q * N log N)).

2. **Optimized Distance Calculation with Binary Search**:
   - Instead of computing all distances for each query, we can use binary search to find the k-th smallest distance. The idea is to perform a binary search on the possible distance values (from 0 to the maximum possible distance) and, for each mid-value, count how many points in `a` are within `mid` distance from `b_j`. If the count is at least `k_j`, we adjust the search range to find the smallest such distance.

3. **Efficient Counting with Lower and Upper Bounds**:
   - For each query point `b_j`, after sorting the array `a`, we can use binary search (lower and upper bounds) to count how many points are within a certain distance from `b_j`. This reduces the counting step to O(log N) per binary search step.

### Solution Code

```cpp
#include<bits/stdc++.h>
using namespace std;
typedef long long ll;

int main() {
    ios::sync_with_stdio(0);
    cin.tie(0);
    cout.tie(0);
    
    int N, Q;
    cin >> N >> Q;
    vector<ll> a(N);
    for (int i = 0; i < N; ++i) {
        cin >> a[i];
    }
    sort(a.begin(), a.end());
    
    while (Q--) {
        ll b;
        int k;
        cin >> b >> k;
        
        ll low = 0, high = 2e18;
        ll answer = high;
        
        auto count = [&](ll d) -> int {
            ll left = b - d;
            ll right = b + d;
            int l = lower_bound(a.begin(), a.end(), left) - a.begin();
            int r = upper_bound(a.begin(), a.end(), right) - a.begin();
            return r - l;
        };
        
        while (low <= high) {
            ll mid = low + (high - low) / 2;
            int cnt = count(mid);
            if (cnt >= k) {
                answer = mid;
                high = mid - 1;
            } else {
                low = mid + 1;
            }
        }
        
        cout << answer << '\n';
    }
    
    return 0;
}
```

### Explanation

1. **Sorting the Array**: The array `a` is sorted to allow efficient binary search operations during the distance calculations.
2. **Binary Search for Each Query**: For each query point `b_j` and integer `k_j`, we perform a binary search on the possible distances from `b_j` to find the smallest distance `d` such that at least `k_j` points in `a` are within `d` units of `b_j`.
3. **Counting Points within Distance**: The `count` function uses `lower_bound` and `upper_bound` to efficiently count the number of points in the sorted array `a` that lie within the range `[b_j - d, b_j + d]`. This count determines whether to adjust the binary search range.
4. **Efficiency**: The binary search operates in O(log(max_distance)) per query, and each counting step is O(log N). With Q queries, the total complexity is O(Q log(max_distance) log N), which is efficient for the given constraints (N and Q up to 10^5). 

This approach ensures that the solution is both correct and efficient, handling all edge cases and large input sizes appropriately.
