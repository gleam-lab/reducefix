To fix this code, we need to correctly implement the algorithm to find the k-th closest point from a set of points A to each query point B. The original code attempts to use binary search to find the k-th smallest distance, but it has several issues, including incorrect logic in the `check` function and inefficient handling of the binary search bounds.

### Approaches
1. **Sorting the Points**: First, we sort the array of points `A` to facilitate efficient distance calculations.
2. **Binary Search for Distance**: For each query point `B_j`, we perform a binary search on possible distances to find the smallest distance `d` such that there are at least `k_j` points in `A` within distance `d` from `B_j`.
3. **Checking Validity**: For a given distance `d` during the binary search, we count the number of points in `A` that lie within the interval `[B_j - d, B_j + d]` using binary search (lower_bound and upper_bound). If the count is at least `k_j`, we adjust the search range to find the smallest such `d`.

### Solution Code
```cpp
#include<bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(0);
    cin.tie(0);
    cout.tie(0);
    
    int N, Q;
    cin >> N >> Q;
    vector<int> a(N);
    for (int i = 0; i < N; ++i) {
        cin >> a[i];
    }
    sort(a.begin(), a.end());
    
    while (Q--) {
        int b, k;
        cin >> b >> k;
        
        int low = 0, high = 2e8;
        int answer = 0;
        while (low <= high) {
            int mid = low + (high - low) / 2;
            
            auto left = lower_bound(a.begin(), a.end(), b - mid);
            auto right = upper_bound(a.begin(), a.end(), b + mid);
            int count = right - left;
            
            if (count >= k) {
                answer = mid;
                high = mid - 1;
            } else {
                low = mid + 1;
            }
        }
        cout << answer << "\n";
    }
    
    return 0;
}
```

### Explanation
1. **Sorting**: The array `a` is sorted to allow the use of binary search operations for efficient range queries.
2. **Binary Search Setup**: For each query `(b, k)`, we initialize the binary search range with `low` set to 0 and `high` set to a sufficiently large value (200 million) to cover all possible distances.
3. **Binary Search Execution**: During each iteration of the binary search:
   - **Mid Calculation**: The midpoint `mid` of the current search range is calculated.
   - **Range Check**: Using `lower_bound` and `upper_bound`, we find the number of points in `a` that lie within the range `[b - mid, b + mid]`.
   - **Adjust Search Range**: If the count of points within the range is at least `k`, we adjust the `high` to search for a smaller distance. Otherwise, we adjust the `low` to search for a larger distance.
4. **Output**: The smallest distance `mid` that satisfies the condition (having at least `k` points within the range) is output for each query.

This approach ensures that we efficiently and correctly find the k-th smallest distance from each query point to the set of points in `A`, leveraging sorting and binary search to handle the large input sizes within constraints.
