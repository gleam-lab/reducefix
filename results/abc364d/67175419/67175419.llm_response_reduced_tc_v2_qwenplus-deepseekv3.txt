To fix this code, we need to address the fundamental issue in the original approach, which is the incorrect handling of the binary search for the k-th closest distance. The original code attempts to perform a binary search on the distance, but it doesn't correctly account for the sorted distances and the k-th smallest distance. 

### Approaches
1. **Sort the Array**: First, sort the array of points `A` to facilitate efficient searching.
2. **Binary Search for Distance**: For each query point `B_j`, perform a binary search to find the smallest distance `d` such that there are at least `k_j` points in `A` within distance `d` from `B_j`.
3. **Check Function**: The check function should count the number of points in `A` that lie within distance `d` of `B_j` using binary search (lower_bound and upper_bound).
4. **Binary Search Adjustment**: Adjust the binary search bounds to find the exact distance that corresponds to the k-th smallest distance.

### Solution Code
```cpp
#include <bits/stdc++.h>
using namespace std;
using ll = long long;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int N, Q;
    cin >> N >> Q;
    vector<ll> A(N);
    for (int i = 0; i < N; ++i) {
        cin >> A[i];
    }
    sort(A.begin(), A.end());

    while (Q--) {
        ll b, k;
        cin >> b >> k;
        ll low = 0, high = 2e18;
        ll answer = 0;
        while (low <= high) {
            ll mid = low + (high - low) / 2;
            auto left = lower_bound(A.begin(), A.end(), b - mid);
            auto right = upper_bound(A.begin(), A.end(), b + mid);
            ll count = right - left;
            if (count >= k) {
                answer = mid;
                high = mid - 1;
            } else {
                low = mid + 1;
            }
        }
        cout << answer << '\n';
    }

    return 0;
}
```

### Explanation
1. **Sorting the Array**: The array `A` is sorted to allow binary search operations, which are crucial for efficiently finding the range of elements within a certain distance.
2. **Binary Search for Distance**: For each query, we perform a binary search on the possible distances from the query point `B_j`. The goal is to find the smallest distance `d` such that at least `k_j` points in `A` lie within the range `[B_j - d, B_j + d]`.
3. **Check Function**: During each iteration of the binary search, we use `lower_bound` and `upper_bound` to count the number of points in `A` that fall within the current distance `mid` from `B_j`. If the count is at least `k_j`, we adjust the search range to find a smaller distance; otherwise, we search for a larger distance.
4. **Output the Result**: The smallest valid distance found during the binary search is printed for each query, which corresponds to the k-th smallest distance from `B_j` to the points in `A`.

This approach ensures that each query is processed in logarithmic time relative to the range of possible distances, making the solution efficient and scalable for large input sizes.
