
### Problem Analysis

The problem involves finding the distance between a specific point \( X \) and a list of points \( B \) on a number line, where \( X \) is chosen from a group of points \( A \). For each point \( B_j \), we need to determine the \( k_j \)-th closest point from \( A \) and calculate the distance between them.

### Solution Strategy

1. **Sorting the Points**: Start by sorting the array \( A \) based on the coordinates. This allows us to efficiently find the \( k \)-th closest point using binary search.

2. **Binary Search for Closest Point**: For each point \( B_j \), use binary search on the sorted array \( A \) to find the \( k_j \)-th closest point. The binary search will help in narrowing down the potential candidates for \( X \) efficiently.

3. **Calculate Distance**: Once the closest point \( X \) is identified, calculate the distance between \( X \) and \( B_j \).

### Corrected Code

```cpp
#include<bits/stdc++.h>
using namespace std;
const int N = 2e5 + 5;
int n, q;
int a[N], b[Q], k[Q];

int check(int mid) {
    int count = 0;
    for (int i = 1; i <= n; i++) {
        if (abs(a[i] - b[mid]) <= abs(a[i] - k[mid])) count++;
    }
    return count >= k[mid];
}

int main() {
    ios::sync_with_stdio(0), cin.tie(0), cout.tie(0);
    cin >> n >> q;
    for (int i = 1; i <= n; i++) cin >> a[i];
    sort(a + 1, a + n + 1);
    for (int i = 0; i < q; i++) {
        cin >> b[i] >> k[i];
        // Binary search for the k-th closest point
        int l = 1, r = n;
        while (l < r) {
            int mid = (l + r) / 2;
            if (check(mid)) l = mid + 1;
            else r = mid;
        }
        // Output the distance between the k-th closest point and B[i]
        cout << abs(a[l] - b[i]) << "\n";
    }
    return 0;
}
```

### Explanation of Key Changes

1. **Function `check`**: This function checks if there are at least \( k_j \) points from \( A \) that are closer to \( B_j \) than the point at `mid`. It iterates through all points in \( A \) and counts how many are closer to \( B_j \) than the point at `mid`.

2. **Binary Search**: We use a binary search to find the smallest index `l` such that there are at least \( k_j \) points from \( A \) that are closer to \( B_j \) than the points at indices less than or equal to `l`.

3. **Distance Calculation**: After finding the closest point \( X \) using binary search, the distance between \( X \) and \( B_j \) is calculated and printed.

This solution efficiently handles the constraints and should work within the given limits.
