```cpp
#include<bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(0);
    cin.tie(0);
    cout.tie(0);
    
    int N, Q;
    cin >> N >> Q;
    vector<long long> a(N);
    for (int i = 0; i < N; ++i) {
        cin >> a[i];
    }
    sort(a.begin(), a.end());
    
    while (Q--) {
        long long b;
        int k;
        cin >> b >> k;
        vector<long long> distances(N);
        for (int i = 0; i < N; ++i) {
            distances[i] = abs(a[i] - b);
        }
        nth_element(distances.begin(), distances.begin() + k - 1, distances.end());
        cout << distances[k - 1] << '\n';
    }
    
    return 0;
}
```

### Explanation
1. **Input Handling**: The input is read with `N` and `Q` representing the number of points in set A and the number of queries, respectively.
2. **Sorting Points**: The points in set A are sorted to facilitate distance calculations.
3. **Processing Queries**: For each query, the distance from the query point B_j to each point in A is computed as the absolute difference. These distances are then processed using `nth_element` to efficiently find the k-th smallest distance without fully sorting the array, which optimizes the solution to O(N) per query on average.
4. **Output**: The k-th smallest distance is printed for each query.

This approach ensures that the solution is efficient and correct, handling all edge cases and constraints appropriately. The use of `nth_element` reduces the time complexity from O(N log N) per query to O(N) on average, making the solution feasible for large input sizes.
