To fix this code, we need to address the main issue, which is efficiently finding the k-th closest point from a set of given points (A) to each query point (B). The original code attempts to use binary search on the distance, but it has several flaws, including incorrect bounds and inefficient checks.

### Approaches
1. **Sorting and Binary Search**: 
   - First, sort the array of A points. This allows us to efficiently find the closest points using binary search.
   - For each query point B_j, the k-th closest point can be found by considering the distances to all points in A. The sorted array helps in narrowing down the search space.
   - We use binary search to determine the smallest distance `d` such that there are at least `k` points in A within distance `d` from B_j. The k-th closest distance will be the smallest such `d`.

2. **Two-Pointers Technique**:
   - For each B_j, after sorting the A array, we can use binary search to find the position where B_j would fit in the sorted A array.
   - Then, we expand outwards (both left and right) from this position to collect the closest points in order until we find the k-th closest point.

### Solution Code
```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);
    int N, Q;
    cin >> N >> Q;
    vector<int> A(N);
    for (int i = 0; i < N; ++i) {
        cin >> A[i];
    }
    sort(A.begin(), A.end());
    while (Q--) {
        int b, k;
        cin >> b >> k;
        int left = 0, right = 0;
        auto it = lower_bound(A.begin(), A.end(), b);
        if (it != A.begin()) {
            left = distance(A.begin(), it) - 1;
        } else {
            left = -1;
        }
        right = distance(A.begin(), it);
        vector<int> distances;
        while (distances.size() < k && (left >= 0 || right < N)) {
            if (left >= 0 && right < N) {
                int dist_left = abs(A[left] - b);
                int dist_right = abs(A[right] - b);
                if (dist_left <= dist_right) {
                    distances.push_back(dist_left);
                    --left;
                } else {
                    distances.push_back(dist_right);
                    ++right;
                }
            } else if (left >= 0) {
                distances.push_back(abs(A[left] - b));
                --left;
            } else {
                distances.push_back(abs(A[right] - b));
                ++right;
            }
        }
        cout << distances[k - 1] << '\n';
    }
    return 0;
}
```

### Explanation
1. **Sorting**: The array `A` is sorted to facilitate efficient searching and distance calculations.
2. **Binary Search for Initial Position**: For each query point `b`, the `lower_bound` function is used to find the first position in `A` where the element is not less than `b`.
3. **Two Pointers Technique**: Two pointers (`left` and `right`) are used to expand outward from the initial position found by `lower_bound`. This helps in collecting distances in increasing order from `b`.
4. **Distance Collection**: Distances from `b` to the elements at `left` and `right` pointers are computed and added to a list until we have at least `k` distances. The k-th smallest distance is then printed for each query.

This approach ensures that we efficiently find the k-th closest point for each query by leveraging the sorted order and expanding our search space smartly, avoiding unnecessary computations.
