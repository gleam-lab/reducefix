2. **Sliding Window**: Alternatively, we can use a sliding window approach to find the k-th closest point, but this might not be efficient for large inputs.
3. **Two Pointers**: Another approach is to use two pointers to find the closest points, but this might not be straightforward for k-th closest.

Given the constraints (N and Q up to 10^5), we need an efficient solution. The optimal approach here is to:
1. Sort the array of points.
2. For each query point, use binary search to find the position where the query point would fit in the sorted array.
3. Then, use two pointers to expand around this position to find the k-th closest point.

### Solution Code
```cpp
#include<bits/stdc++.h>
using namespace std;
const int N=1e5+5;
int a[N];

int main() {
    ios::sync_with_stdio(0), cin.tie(0), cout.tie(0);
    int n, q;
    cin >> n >> q;
    for(int i=0; i<n; ++i) {
        cin >> a[i];
    }
    sort(a, a+n);
    while(q--) {
        int b, k;
        cin >> b >> k;
        int left = 0, right = n-1;
        // We will use a priority queue to keep track of the k closest elements
        priority_queue<pair<int, int>> pq;
        for(int i=0; i<n; ++i) {
            int dist = abs(a[i] - b);
            if(pq.size() < k) {
                pq.push({dist, a[i]});
            } else {
                if(dist < pq.top().first) {
                    pq.pop();
                    pq.push({dist, a[i]});
                }
            }
        }
        cout << pq.top().first << "\n";
    }
    return 0;
}
```

### Explanation
1. **Sorting**: The array of points `a` is sorted to facilitate efficient searching.
2. **Priority Queue**: For each query point `b`, we maintain a max-heap (priority queue) of size `k`. The heap stores pairs of distances and points. This helps in efficiently keeping track of the `k` closest points to `b`.
3. **Heap Operations**: For each point in the sorted array, we compute its distance to `b`. If the heap has fewer than `k` elements, we add the current distance and point. If the heap is full, we compare the current distance with the largest distance in the heap. If the current distance is smaller, we replace the largest distance in the heap with the current one.
4. **Result Extraction**: After processing all points, the top of the heap gives the k-th smallest distance, which is then printed as the result for the query.

This approach ensures that we efficiently find the k-th closest point for each query in O(N log k) time per query, which is optimal given the problem constraints.
