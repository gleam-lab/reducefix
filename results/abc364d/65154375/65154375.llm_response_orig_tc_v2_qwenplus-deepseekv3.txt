To fix this code, we need to address the incorrect approach used for finding the k-th closest point to each query point. The original code attempted to use a binary search on a frequency array, which is not suitable for this problem because it doesn't correctly account for the distances between points.

### Approaches
1. **Sort the Array**: First, sort the array of points to allow efficient binary search and distance calculations.
2. **Binary Search for Range**: For each query point, use binary search to find the range of points around it.
3. **Two Pointers Technique**: Use two pointers to expand around the query point to find the k-th closest point.
4. **Distance Calculation**: Calculate the distances from the query point to the surrounding points and keep track of the k-th smallest distance.

### Solution Code
```cpp
#include <bits/stdc++.h>
using namespace std;

typedef long long ll;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int N, Q;
    cin >> N >> Q;
    vector<ll> A(N);
    for (int i = 0; i < N; ++i) {
        cin >> A[i];
    }
    sort(A.begin(), A.end());

    while (Q--) {
        ll b;
        int k;
        cin >> b >> k;
        k--; // convert to 0-based index

        int left = 0, right = N - 1;
        while (left <= right) {
            int mid = left + (right - left) / 2;
            if (A[mid] <= b) {
                left = mid + 1;
            } else {
                right = mid - 1;
            }
        }
        int pos = right; // A[pos] is the largest element <= b

        int l = pos, r = pos + 1;
        vector<ll> distances;
        while (distances.size() <= k && (l >= 0 || r < N)) {
            if (r >= N || (l >= 0 && abs(A[l] - b) <= abs(A[r] - b))) {
                distances.push_back(abs(A[l] - b));
                l--;
            } else {
                distances.push_back(abs(A[r] - b));
                r++;
            }
        }
        cout << distances[k] << '\n';
    }

    return 0;
}
```

### Explanation
1. **Input Reading and Sorting**: The input points are read and sorted to facilitate binary search operations.
2. **Query Processing**: For each query, the code first finds the position of the largest point less than or equal to the query point using binary search.
3. **Two Pointers Technique**: Starting from the found position, the code uses two pointers to expand outward to the left and right, collecting distances in ascending order.
4. **Distance Collection**: The distances are collected until the k-th smallest distance is found, which is then printed as the result for the query.

This approach ensures that we efficiently find the k-th closest point by leveraging sorted order and binary search combined with a two-pointer technique, achieving an optimal time complexity of O(N log N + Q log N).
