The problem requires finding the k-th closest point from a set of points A to a query point B for each query. The original code attempted to use a binary search approach with a frequency array, which is not suitable for this problem.

### Approaches
The correct approach involves:
1. Sorting the array of points A to facilitate binary search operations.
2. For each query point B, we need to determine the k-th closest point in A to B. This can be efficiently done by:
   - Finding the position in the sorted array where B would fit.
   - Expanding outwards from this position to collect the k closest points using a two-pointer technique or by considering ranges around B.
3. Once the k-th closest point is identified, the distance between B and this point is computed and stored.

### Solution Code
```cpp
#include<bits/stdc++.h>
using namespace std;
typedef long long ll;

int main() {
    ios::sync_with_stdio(0), cin.tie(0), cout.tie(0);
    int N, Q;
    cin >> N >> Q;
    vector<ll> A(N);
    for (int i = 0; i < N; ++i) {
        cin >> A[i];
    }
    sort(A.begin(), A.end());
    
    while (Q--) {
        ll b;
        int k;
        cin >> b >> k;
        int left = 0, right = N - 1;
        while (left <= right) {
            int mid = left + (right - left) / 2;
            if (A[mid] < b) {
                left = mid + 1;
            } else {
                right = mid - 1;
            }
        }
        int pos = left;
        int low = pos - 1, high = pos;
        vector<ll> distances;
        while (distances.size() < k && (low >= 0 || high < N)) {
            if (low >= 0 && high < N) {
                if (abs(A[low] - b) <= abs(A[high] - b)) {
                    distances.push_back(abs(A[low] - b));
                    low--;
                } else {
                    distances.push_back(abs(A[high] - b));
                    high++;
                }
            } else if (low >= 0) {
                distances.push_back(abs(A[low] - b));
                low--;
            } else {
                distances.push_back(abs(A[high] - b));
                high++;
            }
        }
        cout << distances[k - 1] << '\n';
    }
    return 0;
}
```

### Explanation
1. **Input Handling**: The input is read, including the number of points N and the number of queries Q. The points A are stored in a vector and sorted to allow binary search operations.
2. **Query Processing**: For each query, the point B and the integer k are read. The position of B in the sorted array A is found using binary search to determine where B would be inserted.
3. **Finding k-th Closest Point**: Using two pointers, one moving left from the insertion position and the other moving right, the closest points to B are collected in a vector. The vector is populated until it contains k elements, ensuring the closest points are considered first.
4. **Output**: The k-th element in the distances vector is printed, representing the k-th smallest distance from B to the points in A.

This approach efficiently handles each query in logarithmic time relative to the size of A, making it suitable for large input sizes as specified in the problem constraints.
