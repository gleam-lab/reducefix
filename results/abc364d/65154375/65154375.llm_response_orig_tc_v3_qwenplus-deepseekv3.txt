### Approaches

2. **Binary Search for Closest Points**: For each query point `b[j]`, use binary search to find the position in the sorted array where `b[j]` would fit. This helps in determining the closest points around `b[j]`.
3. **Two Pointers Technique**: After finding the initial position, use two pointers to expand outward from this position, comparing the distances of points to `b[j]` to collect the k closest points.
4. **Sorting and Selection**: Collect the distances of the k closest points, sort them, and then select the k-th smallest distance.

```cpp
#include <bits/stdc++.h>
using namespace std;

typedef long long ll;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);
    
    int N, Q;
    cin >> N >> Q;
    vector<ll> a(N);
    for (int i = 0; i < N; ++i) {
        cin >> a[i];
    }
    sort(a.begin(), a.end());
    
    while (Q--) {
        ll b;
        int k;
        cin >> b >> k;
        
        int left = 0, right = N - 1;
        int pos = lower_bound(a.begin(), a.end(), b) - a.begin();
        
        vector<ll> distances;
        int l = pos - 1, r = pos;
        for (int i = 0; i < k; ++i) {
            if (l >= 0 && (r >= N || abs(a[l] - b) <= abs(a[r] - b))) {
                distances.push_back(abs(a[l] - b));
                --l;
            } else {
                distances.push_back(abs(a[r] - b));
                ++r;
            }
        }
        
        nth_element(distances.begin(), distances.begin() + k - 1, distances.end());
        cout << distances[k - 1] << '\n';
    }
    
    return 0;
}
```

### Explanation
1. **Sorting the Array**: The array `a` is sorted to allow efficient binary search operations, which helps in quickly locating the position of the query point `b` in the sorted array.
2. **Binary Search**: For each query point `b`, we use `lower_bound` to find the first position in the sorted array where `b` could be inserted without disrupting the order. This gives us a starting point to search for the closest elements.
3. **Two Pointers Technique**: Using two pointers, one moving left from the found position and the other moving right, we compare the distances of the elements at these pointers to `b` and collect the k closest distances. This ensures that we efficiently gather the closest points without unnecessary comparisons.
4. **Nth Element Selection**: The collected distances are partially sorted using `nth_element` to find the k-th smallest distance, which is then printed as the result for the query. This approach ensures optimal performance with an average time complexity of O(N log N) for sorting and O(Q * k) for query processing, which is efficient given the constraints.
