2. **Binary search for the k-th closest point**: For each query, we can use binary search to find the range of points around B_j that contain the k-th closest point. This involves:

   - Sorting these distances.
   - Selecting the k-th smallest distance.

### Solution Code
```cpp
#include <bits/stdc++.h>
using namespace std;

typedef long long ll;

int main() {
    ios::sync_with_stdio(0);
    cin.tie(0);
    
    int N, Q;
    cin >> N >> Q;
    vector<ll> A(N);
    for (int i = 0; i < N; ++i) {
        cin >> A[i];
    }
    sort(A.begin(), A.end());
    
    while (Q--) {
        ll b, k;
        cin >> b >> k;
        vector<ll> distances(N);
        for (int i = 0; i < N; ++i) {
            distances[i] = abs(A[i] - b);
        }
        nth_element(distances.begin(), distances.begin() + k - 1, distances.end());
        cout << distances[k - 1] << '\n';
    }
    
    return 0;
}
```

### Explanation
1. **Input Handling**: The code reads the number of A points (N) and the number of queries (Q). It then reads the coordinates of the A points into a vector and sorts them.
2. **Processing Queries**: For each query, it reads the coordinate of the B point (b) and the desired k-th closest point (k).
3. **Distance Calculation**: The code calculates the absolute distances between each A point and the B point.
4. **Finding the k-th Closest Point**: Using the `nth_element` function, it efficiently finds the k-th smallest distance in the distances vector. This function partially sorts the vector such that the element at the (k-1)th position is the k-th smallest element.
5. **Output**: The result for each query is the k-th smallest distance, which is printed.

This approach ensures that each query is processed efficiently with a time complexity of O(N) per query, which is feasible given the constraints. The sorting step for the A points ensures that the distances are calculated in a consistent manner, and the use of `nth_element` optimizes the selection of the k-th smallest distance.
