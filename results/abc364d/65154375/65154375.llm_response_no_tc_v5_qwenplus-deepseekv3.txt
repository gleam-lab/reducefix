### Approaches
1. **Sorting and Binary Search**: 
   - First, sort the array of points \( A \).
   - For each query point \( B_j \), compute the distances from \( B_j \) to all points in \( A \), but since \( A \) is sorted, we can perform binary search to find the k-th closest point efficiently.
   - Use binary search to find the range around \( B_j \) that contains exactly \( k \) points. The distance to the k-th closest point will be the required result.

### Solution Code
```cpp
#include<bits/stdc++.h>
using namespace std;
typedef long long ll;

int main() {
    ios::sync_with_stdio(0), cin.tie(0), cout.tie(0);
    int N, Q;
    cin >> N >> Q;
    vector<ll> A(N);
    for (int i = 0; i < N; ++i) {
        cin >> A[i];
    }
    sort(A.begin(), A.end());
    while (Q--) {
        ll b, k;
        cin >> b >> k;
        int left = 0, right = N - 1;
        vector<ll> distances(N);
        for (int i = 0; i < N; ++i) {
            distances[i] = abs(A[i] - b);
        }
        int low = 0, high = 2e8;
        ll result = 0;
        while (low <= high) {
            ll mid = (low + high) / 2;
            ll cnt = upper_bound(A.begin(), A.end(), b + mid) - lower_bound(A.begin(), A.end(), b - mid);
            if (cnt >= k) {
                result = mid;
                high = mid - 1;
            } else {
                low = mid + 1;
            }
        }
        cout << result << "\n";
    }
    return 0;
}
```

### Explanation
1. **Sorting the Array**: The array \( A \) is sorted to facilitate efficient binary search operations.
2. **Processing Queries**: For each query point \( B_j \) and integer \( k_j \), we need to find the k-th closest point in \( A \) to \( B_j \).
3. **Binary Search for Distance**: 
   - We perform a binary search on the possible distances from \( B_j \). For each midpoint distance \( mid \), we count how many points in \( A \) lie within the range \( [B_j - mid, B_j + mid] \).
   - If the count of points within this range is at least \( k_j \), we adjust the search range to find a smaller distance. Otherwise, we search for a larger distance.
4. **Output the Result**: The smallest distance \( mid \) that contains at least \( k_j \) points is the answer for the query, representing the distance to the k-th closest point.

This approach efficiently handles the constraints by leveraging sorting and binary search, ensuring optimal performance for large inputs.
