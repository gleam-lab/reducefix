This solution works as follows:

1. We read the coordinates of points \(A_i\) into a vector `coords` and sort it.
2. For each query:
   - We find the position of point \(B_j\) in the sorted list using `lower_bound`.
   - If the position is exactly \(k_j - 1\), then the point \(X\) is already at \(B_j\), so the distance is 0.
   - If the position is less than \(k_j - 1\), then the point \(X\) is the next closest point after \(B_j\), so we calculate the distance to this point.
   - Otherwise, the point \(X\) is one of the previous points, so we calculate the distance to this point.

This approach ensures that we efficiently find the \(k_j\)-th closest point for each query without sorting distances repeatedly. The time complexity is \(O((N + Q) \log N)\), which is efficient given the constraints.
