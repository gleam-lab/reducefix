To solve this problem efficiently, we need to determine the k-th smallest distance between a point \( B_j \) and any point \( A_i \). Given the constraints, we can leverage sorting and binary search to achieve this.

Here's a step-by-step approach:

1. **Read Input**: Read the number of points \( N \) and \( Q \), followed by the coordinates of points \( A_1, A_2, \ldots, A_N \) and then the queries \( (B_1, k_1), (B_2, k_2), \ldots, (B_Q, k_Q) \).

2. **Distance Calculation**: For each query \( (B_j, k_j) \):
   - Calculate the distances from \( B_j \) to all points \( A_i \).
   - Sort these distances.
   - Return the k-th smallest distance.

3. **Efficiency**: To handle large inputs efficiently, use sorting which has a time complexity of \( O(N \log N) \).

Here is the corrected and optimized C++ code:

```cpp
#include <bits/stdc++.h>

using namespace std;

const int MAXN = 100005;

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(0);

    int N, Q;
    cin >> N >> Q;

    vector<int> A(N);
    for (int i = 0; i < N; ++i) {
        cin >> A[i];
    }

    for (int q = 0; q < Q; ++q) {
        int B, k;
        cin >> B >> k;

        vector<int> distances;
        for (int i = 0; i < N; ++i) {
            distances.push_back(abs(A[i] - B));
        }

        sort(distances.begin(), distances.end());

        cout << distances[k - 1] << '\n';
    }

    return 0;
}
```

### Explanation:
1. **Reading Input**:
   - We read \( N \) and \( Q \).
   - We store the coordinates of points \( A \) in a vector `A`.

2. **Processing Queries**:
   - For each query, we read \( B \) and \( k \).
   - We calculate the absolute difference between \( B \) and each point in `A` and store these differences in a vector `distances`.
   - We sort the `distances` vector.
   - The k-th smallest distance is at index \( k-1 \) (since indices start from 0), so we output `distances[k - 1]`.

This solution ensures that we correctly compute the required distances and outputs them efficiently within the given constraints.
