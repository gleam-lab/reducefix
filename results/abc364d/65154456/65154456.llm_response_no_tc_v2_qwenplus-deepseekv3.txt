To fix this code, we need to address several issues. The original approach attempted to use a binary search on a frequency array to find the k-th closest point, but this method is flawed because it doesn't correctly account for the distances between points. Here's a corrected approach that sorts the array of A points and uses binary search to efficiently find the k-th closest point for each query.

### Approaches
1. **Sorting and Binary Search**: 
   - First, sort the array of A points. This allows us to efficiently find the closest points using binary search.
   - For each query point B, we can determine the range of points in A that are closest to B by considering both the left and right sides relative to B's position in the sorted array.
   - We use a priority queue to keep track of the closest points, merging distances from left and right directions until we find the k-th closest point.

### Solution Code
```cpp
#include <bits/stdc++.h>
using namespace std;

typedef long long ll;

int main() {
    ios::sync_with_stdio(0);
    cin.tie(0);

    int N, Q;
    cin >> N >> Q;
    vector<ll> A(N);
    for (int i = 0; i < N; ++i) {
        cin >> A[i];
    }
    sort(A.begin(), A.end());

    while (Q--) {
        ll b;
        int k;
        cin >> b >> k;
        k--; // convert to 0-based index

        int left = lower_bound(A.begin(), A.end(), b) - A.begin() - 1;
        int right = left + 1;
        priority_queue<pair<ll, int>, vector<pair<ll, int>>, greater<pair<ll, int>>> pq;

        if (left >= 0) {
            pq.push({abs(A[left] - b), left});
        }
        if (right < N) {
            pq.push({abs(A[right] - b), right});
        }

        ll ans = 0;
        while (k--) {
            auto current = pq.top();
            pq.pop();
            int idx = current.second;
            if (idx > 0 && A[idx - 1] < A[idx]) {
                pq.push({abs(A[idx - 1] - b), idx - 1});
            }
            if (idx < N - 1 && A[idx + 1] > A[idx]) {
                pq.push({abs(A[idx + 1] - b), idx + 1});
            }
        }
        ans = pq.top().first;
        cout << ans << '\n';
    }

    return 0;
}
```

### Explanation
1. **Sorting**: The array `A` is sorted to facilitate efficient binary search operations.
2. **Binary Search for Initial Position**: For each query point `b`, we find its potential position in the sorted array using `lower_bound`, which gives us the first element not less than `b`.
3. **Priority Queue for Closest Points**: We use a min-heap (priority queue) to keep track of the closest points. Initially, we add the left and right neighbors of the found position (if they exist).
4. **Expanding the Search**: For each of the `k-1` iterations, we pop the closest point from the heap and add its adjacent points (if they exist and haven't been added already). This ensures we always have the next closest points ready in the heap.
5. **Result Extraction**: After processing `k-1` points, the top of the heap will be the k-th closest point's distance to `b`, which we then print.

This approach efficiently narrows down the closest points using a combination of sorting and a priority queue, ensuring both correctness and performance within the problem constraints.
