3. **Binary Search for k-th Closest Element**: For each query, use binary search to find the k-th closest point in the sorted array `a` to the query point `b_j`. The distance is calculated as the absolute difference between the query point and the found point.

### Solution Code
```cpp
#include <bits/stdc++.h>
using namespace std;

typedef long long ll;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int N, Q;
    cin >> N >> Q;
    vector<ll> a(N);
    for (int i = 0; i < N; ++i) {
        cin >> a[i];
    }
    sort(a.begin(), a.end());

    while (Q--) {
        ll b;
        int k;
        cin >> b >> k;
        int left = 0, right = N - 1;
        while (left < right) {
            int mid = (left + right) / 2;
            if (a[mid] < b) {
                left = mid + 1;
            } else {
                right = mid;
            }
        }
        int pos = left;
        int low = max(0, pos - k);
        int high = min(N - 1, pos + k - 1);
        vector<ll> distances;
        for (int i = low; i <= high; ++i) {
            distances.push_back(abs(a[i] - b));
        }
        sort(distances.begin(), distances.end());
        cout << distances[k - 1] << '\n';
    }

    return 0;
}
```

### Explanation
1. **Input Handling**: The input is read for the number of points `N` and queries `Q`. The points `a_i` are stored in a vector.
2. **Sorting**: The vector `a` is sorted to facilitate binary search operations, which are crucial for efficiently finding the k-th closest point.
3. **Binary Search for Initial Position**: For each query point `b_j`, we perform a binary search to find the position in the sorted array where the point would be inserted to maintain order. This helps narrow down the range of points that could be the k-th closest.
4. **Distance Calculation**: We then consider a window around the found position to collect distances of nearby points, sort these distances, and pick the k-th smallest distance, which gives the required answer.

This approach ensures that we efficiently handle each query in logarithmic time relative to the number of points, making the solution scalable for large inputs as specified in the problem constraints.
