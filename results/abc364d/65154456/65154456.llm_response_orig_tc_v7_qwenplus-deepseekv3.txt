```cpp
#include <bits/stdc++.h>
using namespace std;

typedef long long ll;

int main() {
    ios::sync_with_stdio(0);
    cin.tie(0);

    int N, Q;
    cin >> N >> Q;
    vector<ll> A(N);
    for (int i = 0; i < N; ++i) {
        cin >> A[i];
    }
    sort(A.begin(), A.end());

    while (Q--) {
        ll b, k;
        cin >> b >> k;
        k--; // convert to 0-based index

        // Binary search to find the k-th closest element
        int left = 0, right = N - 1;
        while (left <= right) {
            int mid = left + (right - left) / 2;
            ll d = abs(A[mid] - b);

            // Count the number of elements with distance <= d
            auto lower = lower_bound(A.begin(), A.end(), b - d);
            auto upper = upper_bound(A.begin(), A.end(), b + d);
            int cnt = distance(lower, upper);

            if (cnt > k) {
                // The k-th element is within this distance
                right = mid - 1;
            } else {
                // Need to increase the distance
                left = mid + 1;
            }
        }

        // The k-th closest element is A[left]
        ll res = abs(A[left] - b);
        cout << res << "\n";
    }

    return 0;
}
```

### Explanation:
1. **Input Handling**: The code reads the number of points \( N \) and the number of queries \( Q \). It then reads the coordinates of the points \( A \) and sorts them for efficient processing.
2. **Query Processing**: For each query, it reads the point \( B \) and the rank \( k \). The rank is adjusted to 0-based indexing for easier calculation.
3. **Binary Search**: The binary search is used to determine the k-th closest point to \( B \). The search is based on the distance from \( B \). For each mid-point in the binary search, it calculates the number of points within the current distance \( d \) from \( B \) using `lower_bound` and `upper_bound`.
4. **Adjusting Search Range**: If the count of points within distance \( d \) is greater than \( k \), it narrows the search to the left half; otherwise, it searches the right half.
5. **Output**: The distance of the k-th closest point from \( B \) is printed for each query.

This approach ensures that the solution is efficient and handles the large input constraints effectively. The sorting step is \( O(N \log N) \), and each query is processed in \( O(\log N) \) time due to the binary search, making the overall complexity \( O((N + Q) \log N) \).
