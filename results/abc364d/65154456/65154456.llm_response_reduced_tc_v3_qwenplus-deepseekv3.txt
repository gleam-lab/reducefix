To fix this code, we need to address the problem of finding the k-th closest point from a set of N points to a given query point. The original approach attempted to use a binary search on a frequency array, but this is incorrect and inefficient for the given problem. Instead, we should sort the array of points and use binary search to find the k-th closest point efficiently.

### Approaches
1. **Sort the Array**: The first step is to sort the array of points `a_i` to facilitate binary search operations.
2. **Binary Search for k-th Closest Point**: For each query point `b_j`, we need to determine the range of distances around `b_j` that contain the k-th closest point. This involves:
   - Using binary search to find the smallest distance `d` such that there are at least `k_j` points within `d` distance from `b_j`.
   - The distance `d` is found by checking the number of points in the range `[b_j - d, b_j + d]` via binary search in the sorted array.

### Solution Code
```cpp
#include<bits/stdc++.h>
using namespace std;
typedef long long ll;

int main() {
    ios::sync_with_stdio(0);
    cin.tie(0);
    cout.tie(0);
    
    int N, Q;
    cin >> N >> Q;
    vector<ll> A(N);
    for(int i = 0; i < N; ++i) {
        cin >> A[i];
    }
    sort(A.begin(), A.end());
    
    while(Q--) {
        ll b, k;
        cin >> b >> k;
        auto cmp = [&](ll val) {
            return val < b;
        };
        int pos = lower_bound(A.begin(), A.end(), b) - A.begin();
        
        int left = pos - 1, right = pos;
        vector<ll> distances;
        for(int i = 0; i < k; ++i) {
            if(left >= 0 && right < N) {
                if(b - A[left] <= A[right] - b) {
                    distances.push_back(b - A[left]);
                    left--;
                } else {
                    distances.push_back(A[right] - b);
                    right++;
                }
            } else if(left >= 0) {
                distances.push_back(b - A[left]);
                left--;
            } else if(right < N) {
                distances.push_back(A[right] - b);
                right++;
            }
        }
        cout << distances[k-1] << '\n';
    }
    
    return 0;
}
```

### Explanation
1. **Sorting the Array**: The array `A` of points is sorted to allow efficient binary search operations.
2. **Processing Queries**: For each query, we find the position of the query point `b_j` in the sorted array using `lower_bound`.
3. **Finding k-th Closest Point**: We then use a two-pointer approach to expand around the position of `b_j` in the sorted array, collecting the distances of the closest points. The pointers move left and right from the initial position, comparing the distances of the adjacent points to `b_j` and collecting the smallest distances first.
4. **Output the Result**: The k-th smallest distance from the collected distances is printed for each query.

This approach ensures that each query is processed in O(k) time after sorting, which is efficient given the constraints. The sorting step is O(N log N), and the query processing is O(Q * k), which overall is acceptable for the problem constraints.
