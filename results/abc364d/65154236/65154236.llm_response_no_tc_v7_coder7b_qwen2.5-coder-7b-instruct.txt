Here's the corrected version of your code. The primary issues were with how you were handling the distances and sorting them. Instead of using a segment tree, which was unnecessary for this problem, we can use a priority queue (min-heap) to efficiently find the k-th smallest distance.

```cpp
#include <bits/stdc++.h>
using namespace std;

vector<int> a, b, k;

int main() {
    ios_base::sync_with_stdio(false); cin.tie(NULL);

    int n, q;
    cin >> n >> q;
    
    a.resize(n + 1);
    b.resize(q + 1);
    k.resize(q + 1);

    for(int i = 1; i <= n; ++i)
        cin >> a[i];

    for(int i = 1; i <= q; ++i) {
        cin >> b[i] >> k[i];
    }

    for(int i = 1; i <= q; ++i) {
        priority_queue<int, vector<int>, greater<int>> pq;
        for(int j = 1; j <= n; ++j) {
            int dist = abs(a[j] - b[i]);
            pq.push(dist);
            if(pq.size() > k[i]) 
                pq.pop();
        }
        cout << pq.top() << '\n';
    }

    return 0;
}
```

### Explanation:
1. **Input Reading**: We read `n` and `q`, followed by the coordinates of points `A` and the queries for points `B` and their respective `k_j`.
2. **Priority Queue Usage**: For each query, we initialize a min-heap (`priority_queue`) to store the distances. As we calculate the distance from each point `A_i` to the current point `B_j`, we push it into the heap. If the size of the heap exceeds `k_j`, we pop the largest element to ensure that only the k-th smallest distances remain.
3. **Output**: After processing all points `A` for the current query, the top element of the heap will be the k-th smallest distance, which we print.

This approach ensures that we efficiently find the k-th smallest distance for each query within the constraints.
