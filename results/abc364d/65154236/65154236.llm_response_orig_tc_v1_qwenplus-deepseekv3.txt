```cpp
#include<bits/stdc++.h>
using namespace std;
typedef long long ll;

int main() {
    ios::sync_with_stdio(0), cin.tie(0), cout.tie(0);
    int N, Q;
    cin >> N >> Q;
    vector<ll> A(N);
    for (int i = 0; i < N; ++i) {
        cin >> A[i];
    }
    sort(A.begin(), A.end());
    
    while (Q--) {
        ll b, k;
        cin >> b >> k;
        k--; // convert to 0-based index
        
        // Binary search on the answer
        ll left = 0, right = 2e14;
        ll answer = 0;
        while (left <= right) {
            ll mid = (left + right) / 2;
            
            // Count how many points are <= mid distance from b
            auto lower = lower_bound(A.begin(), A.end(), b - mid);
            auto upper = upper_bound(A.begin(), A.end(), b + mid);
            ll count = distance(lower, upper);
            
            if (count > k) {
                answer = mid;
                right = mid - 1;
            } else {
                left = mid + 1;
            }
        }
        cout << answer << '\n';
    }
    return 0;
}
```

### Explanation:
1. **Input Handling**: We read the number of points (N) and queries (Q), followed by the coordinates of the A points.
2. **Sorting**: The A points are sorted to enable efficient binary search operations.
3. **Query Processing**: For each query:
   - We perform a binary search on the possible distances to find the k-th smallest distance.
   - For each mid value in the binary search, we count how many A points are within ±mid distance from the query point b using `lower_bound` and `upper_bound`.
   - If there are more than k points within this distance, we search for a smaller distance; otherwise, we search for a larger distance.
4. **Output**: For each query, we print the found distance.

This approach ensures O(N log N) preprocessing time (for sorting) and O(log(max_distance) * log N) per query time, which is efficient enough for the given constraints.
