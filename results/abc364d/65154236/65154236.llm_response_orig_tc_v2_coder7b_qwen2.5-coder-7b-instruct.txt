To address the problem efficiently, we need to find the k-th smallest distance from each point B_j to any point A_i. This can be done using a combination of sorting and binary search. Here's the corrected and optimized C++ code:

```cpp
#include <bits/stdc++.h>

#define endl '\n'
#define lowbit(x) (x & (-x))

using namespace std;

typedef long long ll;

const int MAXN = 100005;
int N, Q;
vector<int> A, B, K;

vector<int> dists;

bool check(int mid, int k) {
    int cnt = 0;
    for (int d : dists) {
        if (d <= mid) cnt++;
        if (cnt >= k) return true;
    }
    return false;
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    cin >> N >> Q;
    A.resize(N);
    B.resize(Q);
    K.resize(Q);

    for (int i = 0; i < N; ++i) cin >> A[i];
    for (int i = 0; i < Q; ++i) cin >> B[i] >> K[i];

    sort(A.begin(), A.end());

    for (int b : B) {
        dists.clear();
        for (int a : A) {
            dists.push_back(abs(a - b));
        }

        int left = 0, right = 2e8;
        while (left < right) {
            int mid = (left + right) / 2;
            if (check(mid, K[i])) {
                right = mid;
            } else {
                left = mid + 1;
            }
        }
        cout << left << endl;
    }

    return 0;
}
```

### Explanation of Changes:
1. **Sorting**: The distances from each point `B_j` to all points `A_i` are sorted for each `B_j`.
2. **Binary Search**: Binary search is used to find the k-th smallest distance. The `check` function counts how many distances are less than or equal to a given threshold (`mid`). If this count is at least `k`, it means the threshold is large enough, so we adjust the right boundary of the binary search.
3. **Efficiency**: Sorting takes \(O(N \log N)\) time, and for each query, binary search takes \(O(\log D)\) time, where \(D\) is the maximum possible distance. Given the constraints, this approach ensures efficient computation even for large inputs.
