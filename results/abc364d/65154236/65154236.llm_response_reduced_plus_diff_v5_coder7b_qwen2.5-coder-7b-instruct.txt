To solve this problem efficiently, we need to determine the k-th smallest distance from each point \( B_j \) to any of the points \( A_i \). Given the constraints, a brute-force approach would be too slow, so we need a more efficient method.

One effective way to achieve this is to use a priority queue to keep track of the distances. Here's a step-by-step plan:

1. **Read Input**: Read the coordinates of points \( A \) and \( B \), and the values of \( k_j \).
2. **Sort Distances**: For each \( B_j \), compute all distances to points \( A_i \) and store them in a min-heap (priority queue).
3. **Extract k-th Smallest Distance**: Extract the k-th smallest element from the heap.

Here's the corrected C++ code implementing this approach:

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n, q;
    cin >> n >> q;

    vector<int> a(n);
    for (int i = 0; i < n; ++i) {
        cin >> a[i];
    }

    vector<int> results(q);
    for (int qi = 0; qi < q; ++qi) {
        int b, k;
        cin >> b >> k;
        
        // Priority queue to store distances
        priority_queue<int, vector<int>, greater<int>> pq;
        
        for (int ai : a) {
            int dist = abs(b - ai);
            pq.push(dist);
            if (pq.size() > k) {
                pq.pop();
            }
        }
        
        // The top of the heap is the k-th smallest distance
        results[qi] = pq.top();
    }

    for (int result : results) {
        cout << result << '\n';
    }

    return 0;
}
```

### Explanation:
1. **Reading Input**: We read the number of points \( N \) and \( Q \), followed by the coordinates of points \( A \) and then pairs of \( B \) and \( k \).
2. **Priority Queue**: For each \( B_j \), we initialize a min-heap to store distances. As we calculate distances to each point \( A_i \), we push them into the heap. If the size of the heap exceeds \( k \), we remove the largest element to ensure only the k smallest distances remain.
3. **Result Extraction**: After processing all points \( A \) for a given \( B_j \), the root of the heap will be the k-th smallest distance. We store this result.
4. **Output**: Finally, we print all the results.

This approach ensures that we efficiently find the k-th smallest distance using a priority queue, making it suitable for large inputs up to the specified constraints.
