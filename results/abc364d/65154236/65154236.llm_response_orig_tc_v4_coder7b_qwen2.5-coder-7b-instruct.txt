To solve this problem efficiently, we need to find the k-th closest point from a list of N points to a given query point B_j. The brute-force approach of sorting all distances would be too slow for large inputs, so we can use a more efficient method.

One effective approach is to use a priority queue (min-heap) to keep track of the smallest k distances. For each query point B_j, we will iterate through the list of points A_i, calculate the distance to B_j, and maintain a min-heap of size k. This way, after processing all points, the top element of the heap will be the k-th closest point.

Here's the corrected code:

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    int n, q;
    cin >> n >> q;
    
    vector<int> a(n);
    for(int i = 0; i < n; ++i) {
        cin >> a[i];
    }

    while(q--) {
        int b, k;
        cin >> b >> k;
        
        priority_queue<pair<int, int>, vector<pair<int, int>>, greater<pair<int, int>>> pq;
        
        for(int i = 0; i < n; ++i) {
            int dist = abs(a[i] - b);
            if(pq.size() < k) {
                pq.push({dist, a[i]});
            } else if(dist < pq.top().first) {
                pq.pop();
                pq.push({dist, a[i]});
            }
        }
        
        cout << pq.top().first << endl;
    }
    
    return 0;
}
```

### Explanation:
1. **Input Reading**: We read the number of points `n` and queries `q`. Then, we read the coordinates of the points `A`.
2. **Query Processing**:
   - For each query, we initialize a min-heap `pq` to store pairs of distances and corresponding points.
   - For each point `A[i]`, we calculate its distance to the query point `B`.
   - If the heap size is less than `k`, we push the current distance and point into the heap.
   - If the current distance is smaller than the largest distance in the heap (`pq.top().first`), we remove the largest distance and push the new one.
3. **Output**: After processing all points for a query, the root of the heap (`pq.top()`) contains the k-th closest point, and we output its distance.

This approach ensures that we only keep the k smallest distances at any time, making it efficient with a time complexity of \(O(Q \cdot N \log k)\). This is suitable for the given constraints since \(N\) and \(Q\) can be up to \(10^5\).
