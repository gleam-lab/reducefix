### Approaches
1. **Sorting**: First, sort the array of points `a_i` to facilitate efficient distance calculations and binary searches.
2. **Binary Search for Closest Points**: For each query point `b_j`, we can determine the k-th closest point by considering the distances from `b_j` to all points in the sorted array. This involves:
   - Calculating the absolute differences (distances) between `b_j` and each point in the sorted array.
   - Using binary search to find the k-th smallest distance efficiently.

```cpp
#include <bits/stdc++.h>
using namespace std;
typedef long long ll;

void solve() {
    int N, Q;
    cin >> N >> Q;
    vector<ll> A(N);
    for (int i = 0; i < N; ++i) {
        cin >> A[i];
    }
    sort(A.begin(), A.end());
    
    while (Q--) {
        ll b;
        int k;
        cin >> b >> k;
        
        int left = 0, right = N - 1;
        while (left < right) {
            int mid = (left + right) / 2;
            if (A[mid] < b) {
                left = mid + 1;
            } else {
                right = mid;
            }
        }
        int pos = left;
        
        int low = 0, high = 2e8;
        ll res = 0;
        while (low <= high) {
            ll mid = (low + high) / 2;
            int l = lower_bound(A.begin(), A.end(), b - mid) - A.begin();
            int r = upper_bound(A.begin(), A.end(), b + mid) - A.begin();
            if (r - l >= k) {
                res = mid;
                high = mid - 1;
            } else {
                low = mid + 1;
            }
        }
        cout << res << endl;
    }
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);
    solve();
    return 0;
}
```

### Explanation
1. **Sorting the Array**: The array `A` is sorted to allow efficient binary search operations.
2. **Processing Queries**: For each query, we first find the position in the sorted array `A` where the query point `b` would fit. This helps in narrowing down the range of points around `b`.
3. **Binary Search for k-th Closest Distance**: We perform a binary search on possible distances from `b`. For each candidate distance `mid`, we count how many points in `A` lie within the range `[b - mid, b + mid]`. If the count is at least `k`, we adjust the search range to find a smaller distance; otherwise, we look for a larger distance.
4. **Output the Result**: The smallest distance `mid` that contains at least `k` points is the answer for the query, printed as the result.

This approach ensures that we efficiently handle each query in logarithmic time relative to the number of points, making it suitable for large input sizes as specified in the problem constraints.
