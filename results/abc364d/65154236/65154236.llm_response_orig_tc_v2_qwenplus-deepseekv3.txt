To fix this code, we need to address the underlying issue in the approach used to solve the problem. The original code attempted to use a binary search on a frequency array, which is not suitable for this problem because the distances between points can vary widely, and the binary search approach doesn't correctly account for the k-th closest distance.

### Approach
1. **Problem Analysis**: The problem requires finding the k-th closest point from a set of points A to each query point B. The distance is measured as the absolute difference between the coordinates of the points.
2. **Intuition**: For each query point B_j, we need to compute the distances from all points in A to B_j, sort these distances, and then select the k_j-th smallest distance.
3. **Algorithm Selection**: 
   - **Sorting and Binary Search**: For each query, sort the points in A based on their distance to B_j and pick the k_j-th element. However, this approach would be O(Q * N log N), which is too slow for large N and Q (up to 10^5 each).
   - **Pre-sorting and Efficient Query Handling**: Instead, we can pre-sort the array A. For each query B_j, we can find the k_j-th closest element by using binary search around B_j in the sorted array A. This approach reduces the complexity to O(N log N) for sorting and O(Q log N) for queries, which is feasible given the constraints.
4. **Solution**: 
   - Sort the array A to facilitate efficient querying.
   - For each query point B_j, use binary search to find the position of B_j in the sorted array A. Then, use two pointers to expand outward from this position to find the k_j closest elements, comparing left and right distances to B_j at each step.
   - The k_j-th smallest distance will be the maximum distance among the k_j closest elements found.

### Solution Code
```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int N, Q;
    cin >> N >> Q;
    vector<int> A(N);
    for (int i = 0; i < N; ++i) {
        cin >> A[i];
    }
    sort(A.begin(), A.end());

    while (Q--) {
        int b, k;
        cin >> b >> k;
        int left = 0, right = N - 1;
        vector<int> distances;
        priority_queue<pair<int, int>> max_heap;

        auto it = lower_bound(A.begin(), A.end(), b);
        int idx = it - A.begin();

        int l = idx - 1, r = idx;
        for (int i = 0; i < k; ++i) {
            if (l >= 0 && r < N) {
                if (abs(A[l] - b) <= abs(A[r] - b)) {
                    max_heap.push({abs(A[l] - b), l});
                    --l;
                } else {
                    max_heap.push({abs(A[r] - b), r});
                    ++r;
                }
            } else if (l >= 0) {
                max_heap.push({abs(A[l] - b), l});
                --l;
            } else if (r < N) {
                max_heap.push({abs(A[r] - b), r});
                ++r;
            }
        }

        cout << max_heap.top().first << '\n';
    }

    return 0;
}
```

### Explanation
1. **Input Handling**: The input is read, including the number of points N and the number of queries Q, followed by the coordinates of the points in A and the queries (B_j, k_j).
2. **Sorting**: The array A is sorted to facilitate efficient binary search operations.
3. **Query Processing**: For each query:
   - The position of B_j in the sorted array A is found using `lower_bound`.
   - Two pointers (l and r) are initialized to the positions just before and after the found position of B_j.
   - A max-heap is used to keep track of the k smallest distances encountered as we expand the pointers outward. The heap ensures that the largest of these k distances is at the top.
   - The top element of the heap is the k-th smallest distance, which is printed for each query.
This approach efficiently handles each query in logarithmic time relative to the size of A, making it suitable for large input sizes.
