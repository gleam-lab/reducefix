The original code attempted to use a binary search on the distance, but it didn't correctly handle the k-th closest point calculation. Instead, we should sort the array of points and use binary search to find the k-th closest point for each query efficiently.

### Approaches
1. **Sorting and Binary Search**: 
   - First, sort the array of points A. This allows us to efficiently find the k-th closest point to any query point B_j.
   - For each query point B_j, use binary search to find the range of points in the sorted array A that are closest to B_j. The k-th closest point will be within this range.
   - Calculate the distances of these points to B_j and sort them to find the k-th smallest distance.

2. **Efficient Distance Calculation**:
   - For each query, determine the range of points in the sorted array A that are potential candidates for the k-th closest point.
   - Calculate the distances of these candidate points to B_j, sort them, and pick the k-th smallest distance.

### Solution Code
```cpp
#include<bits/stdc++.h>
using namespace std;
typedef long long ll;

void solve() {
    int N, Q;
    cin >> N >> Q;
    vector<ll> A(N);
    for(int i = 0; i < N; ++i) {
        cin >> A[i];
    }
    sort(A.begin(), A.end());
    
    while(Q--) {
        ll b;
        int k;
        cin >> b >> k;
        k--; // converting to 0-based index
        
        int left = 0, right = N;
        while(left < right) {
            int mid = left + (right - left) / 2;
            if(A[mid] < b) {
                left = mid + 1;
            } else {
                right = mid;
            }
        }
        int pos = left;
        
        vector<ll> distances;
        int l = max(0, pos - k - 1);
        int r = min(N - 1, pos + k + 1);
        for(int i = l; i <= r; ++i) {
            distances.push_back(abs(A[i] - b));
        }
        sort(distances.begin(), distances.end());
        cout << distances[k] << endl;
    }
}

int main() {
    ios::sync_with_stdio(0);
    cin.tie(0);
    solve();
    return 0;
}
```

### Explanation
1. **Sorting**: The array A is sorted to facilitate efficient binary search operations.
2. **Binary Search for Position**: For each query point B_j, we find the position in the sorted array A where B_j would fit. This helps in identifying the nearest points around B_j.
3. **Distance Calculation**: We consider a window around the found position to collect candidate points. The distances of these points to B_j are computed, sorted, and the k-th smallest distance is selected.
4. **Efficiency**: Sorting the array initially allows us to use binary search, making each query operation O(log N) for position finding and O(k log k) for sorting the distances, where k is a small window around the position. This ensures the solution handles the constraints efficiently.
