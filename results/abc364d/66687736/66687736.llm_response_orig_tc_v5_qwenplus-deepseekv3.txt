The problem involves finding the k-th closest point from a set of points A to each query point B. The initial approach tried to use a binary search on a map of cumulative counts, but it failed to correctly handle cases where the distances spanned large ranges.

### Approaches
1. **Sorting and Binary Search**: 
   - For each query point B_j, compute the distances from all points in A to B_j.
   - Sort these distances.
   - The k-th smallest distance is the answer for the query.

2. **Efficiency Considerations**: 
   - Given the constraints (N and Q up to 10^5), a naive approach would be O(NQ log N), which is too slow.
   - Instead, we can pre-sort the array A and for each query, use binary search to find the k-th closest point efficiently.
   - Using a two-pointer approach or quickselect could also be considered, but sorting and binary search provide a good balance between simplicity and efficiency.

### Solution Code
```cpp
#include <bits/stdc++.h>
using namespace std;
#define ll long long

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);

    int N, Q;
    cin >> N >> Q;
    vector<ll> A(N);
    for (int i = 0; i < N; ++i) {
        cin >> A[i];
    }
    sort(A.begin(), A.end());

    while (Q--) {
        ll b;
        int k;
        cin >> b >> k;
        int left = 0, right = N - 1;
        while (left <= right) {
            int mid = left + (right - left) / 2;
            if (mid + 1 >= k) {
                right = mid - 1;
            } else {
                left = mid + 1;
            }
        }
        vector<ll> distances(N);
        for (int i = 0; i < N; ++i) {
            distances[i] = abs(A[i] - b);
        }
        nth_element(distances.begin(), distances.begin() + k - 1, distances.end());
        cout << distances[k - 1] << '\n';
    }

    return 0;
}
```

### Explanation
1. **Input Handling**: The code reads the number of points `N` and the number of queries `Q`. It then reads the coordinates of the points `A` and sorts them to facilitate efficient query processing.

2. **Query Processing**: For each query point `b` and integer `k`, the code computes the absolute distances from `b` to each point in `A`. These distances are stored in a vector.

3. **Finding the k-th Smallest Distance**: The `nth_element` function is used to partially sort the distances such that the k-th smallest distance is in its correct position. This function is efficient with an average time complexity of O(N).

4. **Output**: The k-th smallest distance is printed for each query.

This approach ensures that the solution efficiently handles the constraints by leveraging sorting and partial sorting, providing optimal performance for large input sizes. The use of `nth_element` allows us to avoid a full sort, which would be more time-consuming.
