### Explanation
1. **Sorting the Array**: The array `a` is sorted to facilitate efficient distance computation and searching.
2. **Processing Queries**: For each query, the point `b` and the integer `k` are read. The distances from `b` to each point in the sorted array `a` are computed and stored in a temporary vector `distances`.
3. **Finding the k-th Closest Distance**: The `nth_element` function is used to partially sort the `distances` vector such that the element at the (k-1)-th position is the k-th smallest distance. This function is efficient for this purpose as it does not fully sort the array, which is optimal for large input sizes.
4. **Output the Result**: The k-th smallest distance is printed for each query.
