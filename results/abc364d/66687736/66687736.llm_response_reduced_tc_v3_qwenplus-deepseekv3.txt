To fix this code, we need to address the issue of finding the k-th closest point from a set of points A to a query point B efficiently. The original approach attempted to use a binary search on a map of sorted distances, which is not directly applicable here. Instead, we should sort the array A and then use binary search to find the k-th closest point for each query.

### Approaches
1. **Sorting the Array**: First, we sort the array A. This allows us to efficiently find the closest points using binary search.
2. **Binary Search for Each Query**: For each query point B_j, we perform a binary search to find the range of points in A that are closest to B_j. Depending on the value of k_j, we then determine the k-th closest point.
3. **Handling Distances**: The distance between B_j and the k-th closest point in A can be calculated by considering both sides of B_j in the sorted array A. We use a two-pointer approach or binary search to efficiently find the required distances.

### Solution Code
```cpp
#include <bits/stdc++.h>
using namespace std;
#define ll long long
#define rep(i,n) for(int i=0;i<(int)n;i++)

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);
    
    int N, Q;
    cin >> N >> Q;
    vector<ll> A(N);
    rep(i, N) cin >> A[i];
    
    sort(A.begin(), A.end());
    
    rep(_, Q) {
        ll b, k;
        cin >> b >> k;
        
        int left = lower_bound(A.begin(), A.end(), b) - A.begin();
        int right = left;
        vector<ll> distances;
        
        int i = left - 1, j = left;
        while (distances.size() < k && (i >= 0 || j < N)) {
            if (i >= 0 && j < N) {
                if (abs(A[i] - b) <= abs(A[j] - b)) {
                    distances.push_back(abs(A[i] - b));
                    i--;
                } else {
                    distances.push_back(abs(A[j] - b));
                    j++;
                }
            } else if (i >= 0) {
                distances.push_back(abs(A[i] - b));
                i--;
            } else {
                distances.push_back(abs(A[j] - b));
                j++;
            }
        }
        
        cout << distances[k - 1] << '\n';
    }
    
    return 0;
}
```

### Explanation
1. **Sorting**: The array A is sorted to facilitate efficient binary search operations and to easily access the closest points.
2. **Query Processing**: For each query (B_j, k_j), we find the position in the sorted array A where B_j would fit. This helps in determining the closest points around B_j.
3. **Two Pointers Technique**: Using two pointers, one moving left and the other moving right from the initial position of B_j in the sorted array, we collect the distances of the closest points. The distances are collected in ascending order of their absolute difference from B_j.
4. **Output**: The k-th distance in the collected list is then output for each query. This approach ensures that we efficiently find the k-th closest point without resorting to brute force methods, making the solution optimal for large input sizes.
