To solve this problem efficiently, we need to find the k-th closest point to each given point on the number line. Given the constraints, a direct approach would be too slow, so we need an optimized solution.

Here's a step-by-step breakdown of the solution:

1. **Sort Distances**: For each point \( B_j \), calculate the distances to all points \( A_i \). Store these distances in a sorted list.
2. **Binary Search**: Use binary search to find the k-th smallest distance in the sorted list.

Let's implement this in C++:

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);
    
    int n, q;
    cin >> n >> q;
    
    vector<int> a(n);
    for (int i = 0; i < n; ++i) {
        cin >> a[i];
    }
    
    for (int query = 0; query < q; ++query) {
        int b, k;
        cin >> b >> k;
        
        // Calculate distances from B to all points A
        vector<int> distances;
        for (int ai : a) {
            distances.push_back(abs(ai - b));
        }
        
        // Sort the distances
        sort(distances.begin(), distances.end());
        
        // Output the k-th smallest distance
        cout << distances[k - 1] << '\n';
    }
    
    return 0;
}
```

### Explanation:

1. **Input Reading**:
   - Read the number of points `N` and queries `Q`.
   - Read the coordinates of points `A`.

2. **Processing Each Query**:
   - For each query, read the coordinate `B` and the index `k`.
   - Calculate the absolute difference between `B` and each point `A`, storing these differences in a vector `distances`.
   - Sort the `distances` vector.
   - The k-th smallest distance is at index `k-1` in the sorted vector (since indices start at 0).

This solution ensures that we handle up to \(10^5\) points and queries efficiently within the given constraints. The sorting step takes \(O(N \log N)\) time, which is feasible given the maximum value of \(N\).
