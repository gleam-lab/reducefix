To fix this code, we need to address the issue causing incorrect outputs in certain test cases. The problem involves finding the k-th closest point from a set of points A to each query point B. The incorrect approach used binary search on the distances, which may not correctly handle cases where multiple points have the same distance. 

Here’s the corrected approach:
1. Sort the array A to facilitate efficient distance calculations.
2. For each query point B, compute the distances of all points in A to B.
3. Use a priority queue (max-heap) to keep track of the k smallest distances.
4. The top of the heap will then give the k-th smallest distance.

This approach ensures correctness by explicitly computing and comparing distances for each query point.

### Fixed Code
```cpp
#include <bits/stdc++.h>
using namespace std;
#define ll long long
#define rep(i, n) for(int i = 0; i < (int)n; i++)

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);
    
    int N, Q;
    cin >> N >> Q;
    vector<ll> A(N);
    rep(i, N) cin >> A[i];
    sort(A.begin(), A.end());
    
    while (Q--) {
        ll b;
        int k;
        cin >> b >> k;
        
        int left = 0, right = N - 1;
        while (left <= right) {
            int mid = left + (right - left) / 2;
            if (A[mid] <= b) {
                left = mid + 1;
            } else {
                right = mid - 1;
            }
        }
        
        int i = right;
        int j = i + 1;
        priority_queue<ll> max_heap;
        
        while (i >= 0 && j < N) {
            if (b - A[i] <= A[j] - b) {
                max_heap.push(b - A[i]);
                i--;
            } else {
                max_heap.push(A[j] - b);
                j++;
            }
            if (max_heap.size() == k) break;
        }
        
        while (i >= 0 && max_heap.size() < k) {
            max_heap.push(b - A[i]);
            i--;
        }
        
        while (j < N && max_heap.size() < k) {
            max_heap.push(A[j] - b);
            j++;
        }
        
        cout << max_heap.top() << '\n';
    }
    
    return 0;
}
```

### Explanation
1. **Sorting**: The array `A` is sorted to allow efficient binary search and distance calculations.
2. **Binary Search**: For each query point `b`, we find the position in the sorted array where `b` would fit using binary search. This helps in dividing the array into left and right parts relative to `b`.
3. **Priority Queue**: A max-heap is used to maintain the `k` smallest distances. By comparing distances from the left and right parts of the array relative to `b`, we populate the heap with the smallest distances.
4. **Handling Remaining Elements**: If there are remaining elements on either side of `b` after initial comparisons, they are added to the heap until it contains `k` elements.
5. **Output**: The top element of the heap gives the k-th smallest distance, which is printed for each query.

This approach ensures that we efficiently and correctly find the k-th smallest distance for each query point, handling all edge cases and large input sizes within acceptable time complexity.
