To solve this problem efficiently, we need to find the k-th closest point for each query in optimal time. The key idea is to use a combination of sorting and binary search. Here's a step-by-step breakdown of the approach:

1. **Sort Points**: First, sort all the points \(A_i\) on the number line.
2. **Binary Search**: For each query, perform a binary search to find the k-th smallest distance.

Here’s how you can implement this in C++:

```cpp
#include <vector>
#include <iostream>
#include <algorithm>

using m_int = long long;
using std::cin;
using std::cout;
using std::vector;

bool check(const vector<m_int>& sorted_points, const m_int& target, const m_int& k, const m_int& b) {
    int count = 0;
    for (const auto& point : sorted_points) {
        if (std::abs(point - b) <= target) {
            ++count;
        }
    }
    return count >= k;
}

int main() {
    m_int n, q; cin >> n >> q;
    vector<m_int> a(n);
    
    for (m_int i = 0; i < n; ++i) {
        cin >> a[i];
    }
    std::sort(a.begin(), a.end());
    
    while (q--) {
        m_int b, k; cin >> b >> k;
        
        // Binary search for the minimum distance
        m_int left = 0, right = 2 * 1e8; // Upper bound based on constraints
        while (left < right) {
            m_int mid = (left + right) / 2;
            if (check(a, mid, k, b)) {
                right = mid;
            } else {
                left = mid + 1;
            }
        }
        
        cout << left << "\n";
    }

    return 0;
}
```

### Explanation

1. **Sorting**:
   - We first sort the points \(A_i\) because sorting helps us to efficiently find the k-th smallest distance using binary search.

2. **Binary Search**:
   - For each query, we perform a binary search over possible distances.
   - The `check` function counts how many points within the current distance `target` from point `B_j`.
   - If the count is at least `k`, it means the distance is valid, so we try smaller distances (move `right`).
   - Otherwise, we increase the distance (move `left`).

3. **Efficiency**:
   - Sorting takes \(O(N \log N)\) time.
   - Each query performs a binary search with \(O(\log D)\) operations, where \(D\) is the maximum possible distance (which is \(2 \times 10^8\) here).

This solution ensures that we handle all edge cases correctly and efficiently, making it suitable for large inputs up to the specified limits.
