```cpp
#include <vector>
#include <iostream>
#include <algorithm>

using m_int = long long;
using std::cin;
using std::cout;
using std::vector;

int main() {
    m_int n, q;
    cin >> n >> q;
    vector<m_int> a(n);

    for (m_int i = 0; i < n; ++i) {
        cin >> a[i];
    }

    while (q--) {
        m_int b, k;
        cin >> b >> k;

        // Calculate distances from point B_j to all points A_i
        vector<m_int> distances;
        for (const auto& ai : a) {
            distances.push_back(abs(ai - b));
        }

        // Sort the distances in ascending order
        std::sort(distances.begin(), distances.end());

        // The k-th closest point will have the k-th smallest distance
        cout << distances[k - 1] << "\n";
    }
}
```

### Explanation of Fixes
1. **Direct Distance Calculation**: Instead of using binary search to find the k-th smallest distance, we directly calculate the distances from point \( B_j \) to all points \( A_i \), sort them, and then pick the k-th smallest distance.
2. **Efficiency**: This approach ensures that the solution remains efficient even for large values of \( N \) and \( Q \). Sorting the distances takes \( O(N \log N) \) time, which is feasible within the constraints.
3. **Correctness**: This method guarantees that we correctly identify the k-th closest point and compute its distance accurately.
