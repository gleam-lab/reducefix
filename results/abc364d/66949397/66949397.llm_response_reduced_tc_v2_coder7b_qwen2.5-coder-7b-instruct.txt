To solve this problem correctly, we need to efficiently find the k-th smallest distance from a point \( B_j \) to any of the points \( A_i \). The brute-force approach of sorting distances for each query would be too slow due to its high time complexity. Instead, we can use a more efficient method involving binary search and counting.

```cpp
#include <vector>
#include <iostream>
#include <algorithm>

using m_int = long long;
using std::cin;
using std::cout;
using std::vector;

int main() {
    m_int n, q;
    cin >> n >> q;
    vector<m_int> a(n);

    for (m_int i = 0; i < n; ++i) {
        cin >> a[i];
    }
    std::sort(a.begin(), a.end());

    while (q--) {
        m_int b, k;
        cin >> b >> k;

        auto count_within_range = [&](m_int max_distance) {
            m_int count = 0;
            for (m_int ai : a) {
                if (std::abs(ai - b) <= max_distance) {
                    ++count;
                }
            }
            return count;
        };

        m_int left = 0, right = 2 * 1e8 + 1;
        while (left < right) {
            m_int mid = (left + right) / 2;
            if (count_within_range(mid) >= k) {
                right = mid;
            } else {
                left = mid + 1;
            }
        }

        cout << left << "\n";
    }

    return 0;
}
```

### Explanation
1. **Reading Input**: We read the number of points \( N \) and \( Q \), followed by the coordinates of points \( A \) and the queries for points \( B \) and their corresponding \( k \)-values.

2. **Sorting Points \( A \)**: We sort the points \( A \) to facilitate efficient range counting.

3. **Binary Search for k-th Smallest Distance**:
   - For each query, we perform a binary search on the possible distances.
   - The `count_within_range` function counts how many points \( A \) are within a certain distance from \( B \).
   - We adjust our search range based on whether the count is greater than or equal to \( k \).

4. **Output**: We output the k-th smallest distance for each query.

This approach ensures that we handle large inputs efficiently and correctly find the k-th smallest distance for each query.
