To solve this problem efficiently, we need to determine the k-th smallest distance from a set of points \(A\) to a specific point \(B\). The brute-force approach would involve calculating all distances and sorting them, which is not efficient enough for large inputs (\(O(N \log N)\)). Instead, we can use a more optimized approach by maintaining a min-heap to keep track of the k-th smallest distances.

Here's the corrected and optimized C++ code:

```cpp
#include <iostream>
#include <vector>
#include <queue>
#include <cmath>

using namespace std;

typedef long long ll;

struct DistanceInfo {
    int index;
    ll distance;
};

struct CompareDistance {
    bool operator()(const DistanceInfo& a, const DistanceInfo& b) {
        return a.distance > b.distance;
    }
};

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);

    int n, q;
    cin >> n >> q;
    
    vector<ll> a(n);
    for(int i = 0; i < n; ++i) {
        cin >> a[i];
    }

    while(q--) {
        ll b, k;
        cin >> b >> k;
        
        priority_queue<DistanceInfo, vector<DistanceInfo>, CompareDistance> pq;
        
        for(int i = 0; i < n; ++i) {
            ll dist = abs(a[i] - b);
            if(pq.size() < k) {
                pq.push({i, dist});
            } else if(dist < pq.top().distance) {
                pq.pop();
                pq.push({i, dist});
            }
        }
        
        cout << pq.top().distance << '\n';
    }

    return 0;
}
```

### Explanation:

1. **Structures**:
   - `DistanceInfo`: Stores the index and distance from a point to `B`.
   - `CompareDistance`: Custom comparator for the priority queue to maintain a min-heap based on distance.

2. **Main Logic**:
   - For each query, initialize a min-heap with size `k`.
   - Iterate through all points `A` and calculate the distance to `B`. If the heap is smaller than `k`, push the new distance. Otherwise, compare the new distance with the current maximum distance in the heap (the root), and replace it if the new distance is smaller.
   - After processing all points, the root of the heap will be the k-th smallest distance.

This approach ensures that we only maintain the k-th smallest distances, making the solution efficient even for large inputs.
