### Solution Code
```cpp
#include <vector>
#include <iostream>
#include <algorithm>

using m_int = long long;
using std::cin;
using std::cout;
using std::vector;

int main() {
    m_int n, q;
    cin >> n >> q;
    vector<m_int> a(n);
    
    for (m_int i = 0; i < n; ++i) {
        cin >> a[i];
    }
    std::sort(a.begin(), a.end());

    while (q--) {
        m_int b, k;
        cin >> b >> k;

        auto f = [&](m_int x) -> bool {
            auto it_l = std::lower_bound(a.begin(), a.end(), b - x);
            auto it_r = std::upper_bound(a.begin(), a.end(), b + x);
            return (it_r - it_l) >= k;
        };

        m_int l = 0;
        m_int r = 2e18 + 1;
        m_int answer = r;

        while (l <= r) {
            m_int mid = l + (r - l) / 2;
            if (f(mid)) {
                answer = mid;
                r = mid - 1;
            } else {
                l = mid + 1;
            }
        }
        cout << answer << "\n";
    }
    return 0;
}
```

### Explanation
1. **Sorting the Array**: The array `a` is sorted to allow efficient range queries using binary search functions (`std::lower_bound` and `std::upper_bound`).
2. **Binary Search Setup**: For each query, we initialize the binary search bounds. The lower bound `l` is set to 0, and the upper bound `r` is set to a sufficiently large value (2e18 + 1) to cover the maximum possible distance in the input constraints.
3. **Binary Search Execution**: For each midpoint `mid` in the current search range, we check if there are at least `k` points within the distance `mid` from `b`. This is done by counting the number of points in the interval `[b - mid, b + mid]` using `std::lower_bound` and `std::upper_bound`.
4. **Adjusting Bounds**: If the count of points within the distance `mid` is at least `k`, we update the answer to `mid` and search for a smaller distance in the left half. Otherwise, we search in the right half for a larger distance.
5. **Output the Result**: After completing the binary search, the smallest distance `answer` that satisfies the condition is printed for each query.

This approach ensures that we efficiently and correctly find the k-th smallest distance for each query in logarithmic time relative to the maximum possible distance, leveraging the sorted nature of the array for quick range queries.
