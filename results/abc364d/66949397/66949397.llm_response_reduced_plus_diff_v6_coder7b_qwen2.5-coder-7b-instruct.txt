To solve this problem, we need to efficiently find the k-th closest point for each query. Here's a step-by-step approach to achieve this:

1. **Sort Points**: First, sort all points A_i.
2. **Binary Search for k-th Closest Point**: For each query, use binary search to find the k-th closest point.

Here's the corrected C++ code implementing this approach:

```cpp
#include <vector>
#include <iostream>
#include <algorithm>

using m_int = long long;
using std::cin;
using std::cout;
using std::vector;

int main() {
    m_int n, q;
    cin >> n >> q;
    vector<m_int> a(n);
    
    for (m_int i = 0; i < n; ++i) {
        cin >> a[i];
    }
    std::sort(a.begin(), a.end());
    
    while (q--) {
        m_int b, k;
        cin >> b >> k;
        
        // Binary search to find the k-th closest point
        auto it = std::lower_bound(a.begin(), a.end(), b);
        if (it == a.begin()) {
            cout << (*it - b) * (k - 1) << "\n";
        } else if (it == a.end()) {
            cout << (b - *(it - 1)) * k << "\n";
        } else {
            m_int left_dist = b - *(it - 1);
            m_int right_dist = *it - b;
            m_int total_points = n;
            
            if (left_dist <= right_dist) {
                m_int idx = std::max(0LL, k - 1);
                cout << left_dist * idx << "\n";
            } else {
                m_int idx = std::min(total_points, k);
                cout << right_dist * idx << "\n";
            }
        }
    }
    
    return 0;
}
```

### Explanation:
1. **Sorting Points**: We first sort all points `A` to facilitate the binary search.
2. **Binary Search for k-th Closest Point**:
   - For each query, we use `std::lower_bound` to find the position of `b` in the sorted array `A`.
   - If `b` is less than or equal to the smallest element, the k-th closest point is at index `k-1`.
   - If `b` is greater than or equal to the largest element, the k-th closest point is at index `k`.
   - Otherwise, we calculate the distances to the nearest points and determine the k-th closest point based on these distances.

This approach ensures that we efficiently find the k-th closest point for each query with a time complexity of \(O(\log N)\) per query, making it suitable for large inputs up to \(N = 10^5\).
