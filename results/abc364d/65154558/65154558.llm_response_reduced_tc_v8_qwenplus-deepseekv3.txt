   - **Binary Search for Closest Points**: For each query point B_j, perform a binary search to find the position where B_j would fit in the sorted array. This helps in determining the closest points to B_j.
   - **Finding the k-th Closest Point**: Once the position is found, expand outwards to find the k_j-th closest point, considering the distances from B_j to the points in the sorted array.

   - For each query, binary search operations take O(log N) time, and finding the k-th closest point takes O(1) time after sorting.

#include <bits/stdc++.h>
using namespace std;
typedef long long ll;

int main() {

    cin.tie(nullptr);
    
    int N, Q;
    cin >> N >> Q;
    vector<ll> A(N);
    for (int i = 0; i < N; ++i) {
        cin >> A[i];
    }
    sort(A.begin(), A.end());
    
    while (Q--) {
        ll b, k;
        cin >> b >> k;
        
        int left = lower_bound(A.begin(), A.end(), b) - A.begin();
        int right = left;
        vector<ll> distances;
        
        int l = left - 1, r = left;
        for (int i = 0; i < k && (l >= 0 || r < N); ++i) {
            if (l >= 0 && r < N) {
                if (b - A[l] <= A[r] - b) {
                    distances.push_back(b - A[l]);
                    --l;
                } else {
                    distances.push_back(A[r] - b);
                    ++r;
                }
            } else if (l >= 0) {
                distances.push_back(b - A[l]);
                --l;
            } else {

    }
    
    return 0;
}

1. **Sorting**: The array `A` is sorted to allow efficient binary search operations.
2. **Binary Search**: For each query point `b`, we find the position where `b` would fit in the sorted array using `lower_bound`.
3. **Finding k-th Closest Points**: We maintain two pointers, `l` and `r`, starting from the position found by `lower_bound`. We move these pointers outward to collect the closest points to `b` by comparing the distances from `b` to `A[l]` and `A[r]`.
4. **Distance Collection**: The distances of the closest points are collected in a vector until we have `k` distances.
5. **Output**: The k-th distance in the collected vector is printed for each query.
