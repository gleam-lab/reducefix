To address the problem correctly, we need to find the k-th closest point to a given point \( B_j \) among the points \( A_i \). The key steps are:

1. Calculate the distances between all points \( A_i \) and \( B_j \).
2. Sort these distances.
3. Select the k-th smallest distance.

Given the large constraints (\( N, Q \leq 10^5 \)), using a simple sorting approach directly would not be efficient. Instead, we can use a more optimized method such as a priority queue (min-heap) to efficiently find the k-th smallest element.

Here's the corrected code:

```cpp
#include<bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(0), cin.tie(0), cout.tie(0);

    int n, q;
    cin >> n >> q;

    vector<int> a(n);
    for(int i = 0; i < n; ++i) {
        cin >> a[i];
    }

    for(int qi = 0; qi < q; ++qi) {
        int bj, kj;
        cin >> bj >> kj;

        // Priority queue to store distances
        priority_queue<int, vector<int>, greater<int>> pq;

        for(int ai : a) {
            int dist = abs(ai - bj);
            pq.push(dist);
            if(pq.size() > kj) {
                pq.pop(); // Keep only the k smallest distances
            }
        }

        // The top of the heap is the k-th smallest distance
        cout << pq.top() << '\n';
    }

    return 0;
}
```

### Explanation:
1. **Reading Input**: We read the number of points \( N \) and queries \( Q \). Then, we read the coordinates of the points \( A_i \).
2. **Processing Queries**:
   - For each query, we read the coordinate of point \( B_j \) and the value of \( k_j \).
   - We use a min-heap (priority queue) to keep track of the k smallest distances.
   - As we iterate through the points \( A_i \), we calculate the distance to \( B_j \) and push it into the heap.
   - If the size of the heap exceeds \( k_j \), we pop the largest element to ensure the heap contains only the k smallest distances.
3. **Output**: After processing all points, the top of the heap will contain the k-th smallest distance, which we print.

This approach ensures that we efficiently find the k-th smallest distance using a priority queue, making it suitable for large inputs.
