To address the problem of finding the k-th closest point for each query efficiently, we can use a combination of sorting and binary search techniques. Here's a step-by-step approach to solving this problem:

1. **Sort Points**: We will sort all points along with their indices.
2. **Binary Search**: For each query, we will perform a binary search to find the k-th smallest distance.

Here is the corrected and optimized C++ code:

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false); cin.tie(0); cout.tie(0);

    int n, q;
    cin >> n >> q;

    vector<int> a(n);
    for (int i = 0; i < n; ++i) {
        cin >> a[i];
    }

    vector<pair<int, int>> sorted_a(n);
    for (int i = 0; i < n; ++i) {
        sorted_a[i] = {a[i], i};
    }

    sort(sorted_a.begin(), sorted_a.end());

    vector<int> ans(q);
    for (int qi = 0; qi < q; ++qi) {
        int b, k;
        cin >> b >> k;

        int left = 0, right = n - 1;
        while (left < right) {
            int mid = left + (right - left) / 2;
            int dist = abs(sorted_a[mid].first - b);
            if (dist == sorted_a[k - 1].first) {
                ans[qi] = dist;
                break;
            } else if (dist < sorted_a[k - 1].first) {
                left = mid + 1;
            } else {
                right = mid;
            }
        }
    }

    for (int i = 0; i < q; ++i) {
        cout << ans[i] << '\n';
    }

    return 0;
}
```

### Explanation:
1. **Reading Input**:
   - Read `n` and `q`.
   - Read the coordinates of the points `A`.

2. **Sorting Points**:
   - Store the coordinates along with their original indices in a vector of pairs.
   - Sort this vector based on the coordinates.

3. **Processing Queries**:
   - For each query, read `b` and `k`.
   - Perform a binary search to find the position where the k-th smallest distance would be located.
   - Since the array is sorted, we can directly compare distances to determine the correct index.

4. **Output**:
   - Print the result for each query.

This solution ensures that we handle the constraints efficiently and correctly, providing the k-th closest distance for each query. The overall time complexity is \(O((N + Q) \log N)\), which is efficient given the problem constraints.
