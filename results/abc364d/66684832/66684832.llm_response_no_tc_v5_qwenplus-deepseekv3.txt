#include <iostream>
#include <algorithm>
#include <vector>
using namespace std;
#define int long long

const int N = 1e5 + 5;
int a[N];

int find_kth_distance(int n, int x, int k) {
    int left = 0;
    int right = 2e8; // Maximum possible distance considering the constraints
    int answer = right;

    while (left <= right) {
        int mid = left + (right - left) / 2;
        int count = 0;

        // Use lower_bound and upper_bound to find the range [x - mid, x + mid]
        int lower = x - mid;
        int upper = x + mid;
        int l = lower_bound(a, a + n, lower) - a;
        int r = upper_bound(a, a + n, upper) - a;

        count = r - l;

        if (count >= k) {
            answer = mid;
            right = mid - 1;
        } else {
            left = mid + 1;
        }
    }

    return answer;
}

### Explanation
1. **Sorting**: The array `a` is sorted to allow efficient range queries using binary search functions like `lower_bound` and `upper_bound`.
2. **Binary Search**: For each query point `x`, the binary search is performed over possible distances from `x` to find the smallest distance `mid` such that at least `k` points are within `mid` distance from `x`.
3. **Range Counting**: The `lower_bound` function finds the first point in `a` that is not less than `x - mid`, and `upper_bound` finds the first point greater than `x + mid`. The difference between these two positions gives the count of points within the range `[x - mid, x + mid]`.
4. **Adjusting Search Range**: If the count of points within the current distance `mid` is at least `k`, the search continues with a smaller distance; otherwise, it searches larger distances. The smallest valid distance found is stored as the answer for the query.

This approach ensures that each query is answered in logarithmic time relative to the maximum distance, making it efficient for large input sizes as specified in the problem constraints.
