To solve this problem efficiently, we need to find the k-th closest point to a given point in a list of points. This involves sorting distances and finding the k-th smallest element. Given the constraints, we can use a more efficient approach than brute-force sorting for every query.

Here's a step-by-step breakdown of the solution:

1. **Sort Points**: We first sort all points \(A_i\) by their coordinates.
2. **Distance Calculation**: For each query, calculate the distance between the given point \(B_j\) and all points \(A_i\).
3. **K-th Smallest Element**: Using a priority queue (min-heap), we maintain the k smallest distances. This ensures that we only keep track of the relevant points without sorting all distances.

Let's implement this approach:

```cpp
#include <iostream>
#include <vector>
#include <queue>
#include <algorithm>

using namespace std;

struct Point {
    int coord, index;
};

// Comparator for sorting points based on their coordinate
bool comparePoints(const Point& a, const Point& b) {
    return a.coord < b.coord;
}

// Comparator for the min-heap
struct CompareDist {
    bool operator()(const pair<int, int>& a, const pair<int, int>& b) {
        return a.first > b.first;
    }
};

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n, q;
    cin >> n >> q;

    vector<Point> points(n);
    for (int i = 0; i < n; ++i) {
        cin >> points[i].coord;
        points[i].index = i;
    }

    // Sort points by their coordinate
    sort(points.begin(), points.end(), comparePoints);

    while (q--) {
        int x, k;
        cin >> x >> k;

        // Min-heap to store k smallest distances
        priority_queue<pair<int, int>, vector<pair<int, int>>, CompareDist> pq;

        for (int i = 0; i < n; ++i) {
            int dist = abs(points[i].coord - x);
            pq.push({dist, points[i].index});

            // If heap size exceeds k, remove the largest distance
            if (pq.size() > k) {
                pq.pop();
            }
        }

        // The top of the heap is the k-th smallest distance
        cout << pq.top().first << endl;
    }

    return 0;
}
```

### Explanation:
1. **Point Structure**: We define a `Point` structure to store the coordinate and index of each point.
2. **Sorting**: We sort the points based on their coordinates using a custom comparator.
3. **Priority Queue**: For each query, we use a min-heap to keep track of the k smallest distances. If the heap size exceeds k, we remove the largest distance to ensure we always have the k smallest distances.
4. **Output**: The top of the heap gives us the k-th smallest distance, which we print.

This approach ensures that we handle the constraints efficiently, with a time complexity of \(O(N \log N)\) for sorting and \(O(Q \cdot N \log k)\) for processing each query, making it suitable for large inputs.
