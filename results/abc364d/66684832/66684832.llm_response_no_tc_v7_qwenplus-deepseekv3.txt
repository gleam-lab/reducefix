   - For each query point `B_j`, compute the distances from `B_j` to all points in `a`. However, computing distances for each query individually would be too slow for large `N` and `Q`.
   - Instead, for each query, we can use binary search to find the k-th smallest distance. This involves:
     - Using a binary search to guess the distance `dx`.
     - For each guess, count how many points are within `dx` distance from `B_j` using `lower_bound` and `upper_bound`.
     - Adjust the binary search range based on whether the count is sufficient to include the k-th closest point.

#include <vector>

const int N = 1e5 + 10;
int a[N];

int find_kth_distance(int x, int k, int n) {
    vector<int> distances(n);
    for (int i = 0; i < n; ++i) {
        distances[i] = abs(a[i] - x);
    }
    nth_element(distances.begin(), distances.begin() + k - 1, distances.end());
    return distances[k - 1];
}

1. **Input Handling**: The code reads the number of points `N` and queries `Q`, followed by the coordinates of the points `a_1` to `a_N`.
