To solve this problem efficiently, we need to find the k-th closest point for each query in a way that minimizes computational overhead. Here's a step-by-step approach to achieve this:

1. **Sort Points**: Initially, sort all points \(A_i\) and \(B_j\).
2. **Binary Search**: For each query, use binary search to find the k-th closest point efficiently.

Here’s how you can implement it in C++:

```cpp
#include <iostream>
#include <vector>
#include <algorithm>

using namespace std;

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);

    int n, q;
    cin >> n >> q;

    vector<int> A(n), B(q);
    for (int i = 0; i < n; ++i) cin >> A[i];
    for (int i = 0; i < q; ++i) cin >> B[i];

    // Sort points A and B
    sort(A.begin(), A.end());
    sort(B.begin(), B.end());

    for (int j = 0; j < q; ++j) {
        int x = B[j];
        int k;
        cin >> k;

        // Binary search to find the k-th closest point
        auto it = lower_bound(A.begin(), A.end(), x);
        int idx = it - A.begin();

        // Calculate distances to the left and right of the current index
        int left_dist = idx > 0 ? abs(A[idx - 1] - x) : INT_MAX;
        int right_dist = idx < n ? abs(A[idx] - x) : INT_MAX;

        // Find the k-th smallest distance
        if (k == 1) {
            cout << min(left_dist, right_dist) << endl;
        } else {
            k--;
            int dist1 = (idx > 0) ? abs(A[max(0, idx - k)] - x) : INT_MAX;
            int dist2 = (idx < n) ? abs(A[min(n - 1, idx + k)] - x) : INT_MAX;
            cout << min(dist1, dist2) << endl;
        }
    }

    return 0;
}
```

### Explanation:

1. **Sorting**:
   - We first sort both arrays `A` and `B`.

2. **Binary Search**:
   - For each query, we find the position of `x` in array `A` using `lower_bound`.
   - We calculate the distances to the left (`left_dist`) and right (`right_dist`) of this position.
   
3. **Handling Queries**:
   - If `k == 1`, the k-th closest point is either directly above or below `x`.
   - For larger `k`, we consider the `k-th` closest points on both sides and take the minimum.

This approach ensures that each query runs in \(O(\log N)\) time due to the binary search, making it efficient even for large inputs up to \(N = 10^5\).
