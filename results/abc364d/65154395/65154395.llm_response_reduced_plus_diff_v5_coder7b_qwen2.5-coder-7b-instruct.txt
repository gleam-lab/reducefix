To solve this problem efficiently, we need to find the k-th smallest distance for each query. Given the constraints, using a direct approach like sorting all distances and finding the k-th smallest value would not be feasible due to its high time complexity. Instead, we can use a more efficient method involving sorting and binary search.

Here's a step-by-step breakdown of the solution:

1. **Read Input**: Read the number of points \( N \) and \( Q \). Then read the coordinates of the points \( A_i \) and the queries \( (B_j, k_j) \).
2. **Sort Points**: Sort the coordinates of points \( A_i \).
3. **Binary Search for k-th Smallest Distance**:
   - For each query \( (B_j, k_j) \), perform a binary search over the range of possible distances.
   - For each midpoint in the binary search, count how many points have a distance less than or equal to the midpoint.
   - Adjust the binary search range based on whether the count matches \( k_j \).

Here is the corrected and optimized C++ code:

```cpp
#include <bits/stdc++.h>
using namespace std;

vector<int> A;

bool check(int mid, int k) {
    int cnt = 0;
    for (int a : A) {
        if (abs(a - mid) <= mid) {
            cnt++;
        }
    }
    return cnt >= k;
}

int main() {
    int N, Q;
    cin >> N >> Q;
    
    A.resize(N);
    for (int i = 0; i < N; ++i) {
        cin >> A[i];
    }
    
    sort(A.begin(), A.end());
    
    for (int j = 0; j < Q; ++j) {
        int B, k;
        cin >> B >> k;
        
        int left = 0, right = 2 * 1e8;
        while (left < right) {
            int mid = left + (right - left) / 2;
            if (check(B, mid)) {
                right = mid;
            } else {
                left = mid + 1;
            }
        }
        
        cout << left << endl;
    }
    
    return 0;
}
```

### Explanation:
1. **Reading Input**: We read the number of points \( N \) and \( Q \). We then read the coordinates of the points \( A_i \) into a vector `A`.
2. **Sorting**: We sort the vector `A` to facilitate the binary search.
3. **Binary Search Function**: The function `check(mid, k)` checks if there are at least `k` points within a distance of `mid` from the point `B`. It iterates through all points and counts those that satisfy this condition.
4. **Main Loop**: For each query, we initialize the binary search with `left = 0` and `right = 2 * 1e8` (since the maximum possible distance is \( 2 \times 10^8 \)). We adjust the search range based on whether the count of valid points is greater than or equal to \( k \).

This approach ensures that we efficiently find the k-th smallest distance for each query using binary search, making it suitable for large inputs.
