The correct approach involves:

2. **Using binary search** to find the range of distances around B_j that contains the k-th closest point.
3. **Calculating the distances** for each point in A relative to B_j and then sorting these distances to find the k-th smallest distance.

### Solution Code
```cpp
#include <bits/stdc++.h>
using namespace std;
typedef long long ll;

int main() {
    ios::sync_with_stdio(0);
    cin.tie(0);

    int N, Q;
    cin >> N >> Q;
    vector<ll> A(N);
    for (int i = 0; i < N; ++i) {
        cin >> A[i];
    }
    sort(A.begin(), A.end());

    while (Q--) {
        ll b;
        int k;
        cin >> b >> k;
        --k; // convert to 0-based index

        int left = 0, right = N - 1;
        while (left <= right) {
            int mid = left + (right - left) / 2;
            if (A[mid] <= b) {
                left = mid + 1;
            } else {
                right = mid - 1;
            }
        }
        int pos = right;

        vector<ll> distances(N);
        for (int i = 0; i < N; ++i) {
            distances[i] = abs(A[i] - b);
        }
        sort(distances.begin(), distances.end());

        cout << distances[k] << '\n';
    }

    return 0;
}
```

### Explanation
1. **Input Handling**: The code reads the number of points `N` and the number of queries `Q`. It then reads the coordinates of the points `A` and sorts them to facilitate efficient searching.
2. **Query Processing**: For each query, the code reads the coordinate `b` of the point `B_j` and the index `k` indicating the k-th closest point.
3. **Binary Search**: The binary search is used to find the position in the sorted array `A` where the elements are less than or equal to `b`. This helps in narrowing down the candidates for the closest points.
4. **Distance Calculation**: The code calculates the absolute distances between each point in `A` and the query point `b`, stores these distances in an array, and sorts this array to easily find the k-th smallest distance.
5. **Output**: The k-th smallest distance is printed for each query.

This approach ensures that the solution is efficient and correct, handling all edge cases and constraints properly. The sorting of the array `A` allows for efficient distance calculations and the subsequent sorting of distances ensures the correct k-th smallest distance is found. The overall complexity is dominated by the sorting steps, making it O(N log N) per query, which is feasible given the constraints.
