### Approaches
1. **Sort and Binary Search**: For each query point, compute the distances to all points in the list, sort these distances, and then pick the k-th smallest distance. However, sorting for each query would lead to O(Q*N log N) time complexity, which is not efficient for large N and Q (up to 10^5 each).
2. **Pre-sort and Binary Search**: To optimize, we can pre-sort the list of points. For each query point, use binary search to find the closest points and then determine the k-th smallest distance by considering the sorted list of points around the query point. This approach reduces the time complexity to O(N log N + Q log N) due to the sorting step and binary searches for each query.

        vector<ll> temp_dist;
        
        // Collect distances around the query point
        int l = left - 1, r = left;
        while(count < k && (l >= 0 || r < N)){
            ll dist_l = (l >= 0) ? abs(a[l] - b) : LLONG_MAX;
            ll dist_r = (r < N) ? abs(a[r] - b) : LLONG_MAX;
            if(dist_l <= dist_r){
                temp_dist.push_back(dist_l);
                l--;
            } else {
                temp_dist.push_back(dist_r);
                r++;
            }
            count++;
        }

    }

}
