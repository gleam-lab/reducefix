2. **Binary Search for Closest Points**: For each query point `b_j`, perform a binary search to find the range of points around `b_j` that are the k_j-th closest. The distance can then be derived by checking the sorted distances from `b_j`.

### Solution Code

```cpp
#include <bits/stdc++.h>
using namespace std;

typedef long long ll;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int N, Q;
    cin >> N >> Q;
    vector<ll> a(N);
    for (int i = 0; i < N; ++i) {
        cin >> a[i];
    }
    sort(a.begin(), a.end());

    while (Q--) {
        ll b_j;
        int k_j;
        cin >> b_j >> k_j;

        // Using binary search to find the k_j-th closest element
        // The idea is to find the smallest distance d such that there are at least k_j elements in a within distance d of b_j
        ll left = 0;
        ll right = 2e14; // A sufficiently large upper bound for the distance

        while (left < right) {
            ll mid = (left + right) / 2;

            // Count the number of elements in a that are within distance mid of b_j
            auto lower = lower_bound(a.begin(), a.end(), b_j - mid);
            auto upper = upper_bound(a.begin(), a.end(), b_j + mid);

            int count = upper - lower;

            if (count >= k_j) {
                right = mid;
            } else {
                left = mid + 1;
            }
        }

        cout << left << '\n';
    }

    return 0;
}
```

### Explanation

1. **Sorting**: The array `a` is sorted to allow efficient range queries and binary search operations.
2. **Binary Search for Each Query**: For each query `(b_j, k_j)`, we perform a binary search to find the smallest distance `d` such that there are at least `k_j` elements in `a` within the range `[b_j - d, b_j + d]`.
3. **Counting Elements**: For each midpoint `mid` during the binary search, we use `lower_bound` and `upper_bound` to count how many elements in `a` fall within the range `[b_j - mid, b_j + mid]`. This count helps determine whether to adjust the search range left or right.
4. **Output**: The smallest distance `d` that satisfies the condition is printed for each query.

This approach ensures efficiency and correctness, handling the constraints effectively. The binary search operates in logarithmic time, and the sorting step ensures that the range queries are efficient.
