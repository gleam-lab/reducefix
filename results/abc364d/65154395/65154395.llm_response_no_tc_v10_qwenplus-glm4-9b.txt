
```cpp
#include <iostream>
#include <vector>
#include <algorithm>

using namespace std;

int main() {
    int n, q;
    cin >> n >> q;

    vector<int> a(n + 1);
    for (int i = 0; i < n; ++i) {
        cin >> a[i + 1];
    }

    for (int j = 0; j < q; ++j) {
        int b, k;
        cin >> b >> k;
        // Using a vector to store distances from A_i to B_j
        vector<int> distances(n + 1);
        for (int i = 1; i <= n; ++i) {
            distances[i] = abs(a[i] - b);
        }

        // Sorting distances to find the k-th closest point
        sort(distances.begin(), distances.end());

        // Output the distance of the k-th closest point
        cout << distances[k - 1] << endl;
    }

    return 0;
}
```

### Explanation:

1. **Input Reading**: The program starts by reading the number of points \(N\) and queries \(Q\). Then, it reads the coordinates of the \(N\) points \(A_i\).

2. **Query Processing**: For each query, the program reads the point \(B_j\) and the index \(k_j\). It calculates the distances from each point \(A_i\) to \(B_j\) and stores them in a vector.

3. **Sorting**: The vector of distances is sorted in ascending order to easily find the \(k_j\)-th closest point.

4. **Output**: The \(k_j\)-th closest distance is printed as the result for each query.

This solution is straightforward and correct. It leverages the efficiency of sorting to handle the problem within the constraints. The time complexity for each query is dominated by the sorting step, which is \(O(N \log N)\). Given the constraints, this is efficient enough.
