   - Using binary search to find the position where `b_j` would fit in the sorted array.

3. **Distance Calculation**: Calculate the distance between the k-th closest point and `b_j` for each query.

### Solution Code
```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(0);
    cin.tie(0);
    cout.tie(0);
    
    int N, Q;
    cin >> N >> Q;
    vector<long long> A(N);
    for (int i = 0; i < N; ++i) {
        cin >> A[i];
    }
    sort(A.begin(), A.end());
    
    while (Q--) {
        long long b;
        int k;
        cin >> b >> k;
        
        int left = 0, right = N - 1;
        while (left < right) {
            int mid = (left + right) / 2;
            if (A[mid] < b) {
                left = mid + 1;
            } else {
                right = mid;
            }
        }
        
        int pos = left;
        vector<long long> distances;
        int i = pos, j = pos - 1;
        while (distances.size() < N) {
            if (i < N && (j < 0 || abs(A[i] - b) < abs(A[j] - b))) {
                distances.push_back(abs(A[i] - b));
                i++;
            } else if (j >= 0) {
                distances.push_back(abs(A[j] - b));
                j--;
            } else {
                distances.push_back(abs(A[i] - b));
                i++;
            }
        }
        
        cout << distances[k - 1] << '\n';
    }
    
    return 0;
}
```

### Explanation
1. **Sorting the Array**: The array `A` is sorted to allow efficient binary searches. This step ensures that we can quickly locate the position of `b_j` in the sorted array.
2. **Binary Search for Position**: For each query, we use binary search to find the position `pos` where `b_j` would fit in the sorted array. This helps in starting our search for the k closest points around `b_j`.
3. **Finding k-th Closest Point**: We compare the distances of points around `pos` to `b_j` by expanding outward. The closest points are collected in a list `distances`.
4. **Output the Result**: The k-th closest distance is found at the `k-1` index of the `distances` list (since indexing starts at 0), and this distance is printed for each query.

This approach efficiently handles the problem by leveraging sorting and binary search, ensuring optimal performance even for large input sizes.
