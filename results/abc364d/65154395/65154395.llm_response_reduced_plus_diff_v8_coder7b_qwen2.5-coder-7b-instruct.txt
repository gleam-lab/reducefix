To address the problem of finding the k-th closest point to a given point on a number line efficiently, we need to leverage sorting and binary search techniques. The goal is to determine the k-th smallest distance from the set of points \(A_i\) to a point \(B_j\). Here's how you can achieve this:

### Key Steps:
1. **Read Input**: Read the number of points \(N\) and \(Q\), followed by the coordinates of points \(A_i\) and pairs of \(B_j\) and \(k_j\).
2. **Sort Distances**: For each \(B_j\), compute the distances to all points \(A_i\) and sort these distances.
3. **Find k-th Closest Distance**: Retrieve the k-th smallest distance from the sorted list for each query.

### Implementation:
We'll use a map to store the distances and then retrieve the k-th element using the properties of the map. This ensures efficient sorting and access.

Here's the corrected and optimized C++ code:

```cpp
#include <bits/stdc++.h>

using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int N, Q;
    cin >> N >> Q;

    vector<int> A(N);
    for (int i = 0; i < N; ++i) {
        cin >> A[i];
    }

    for (int q = 0; q < Q; ++q) {
        int B, k;
        cin >> B >> k;

        // Map to store distances from B to all points A
        map<int, int> dist_count;
        
        for (int a : A) {
            int d = abs(a - B);
            dist_count[d]++;
        }

        int count = 0;
        for (auto& [dist, freq] : dist_count) {
            count += freq;
            if (count >= k) {
                cout << dist << endl;
                break;
            }
        }
    }

    return 0;
}
```

### Explanation:
1. **Reading Input**:
   - We read \(N\) and \(Q\).
   - We store the coordinates of points \(A_i\) in a vector `A`.

2. **Processing Each Query**:
   - For each query, we read \(B\) and \(k\).
   - We use a map `dist_count` to store the frequency of each distance from \(B\) to any point in \(A\).
   
3. **Finding k-th Closest Distance**:
   - We iterate through the map to accumulate the frequencies of distances until we reach or exceed \(k\).
   - The first distance that makes the accumulated count greater than or equal to \(k\) is the k-th smallest distance, which we print.

This approach ensures that we handle the constraints efficiently and correctly, providing the correct k-th smallest distance for each query.
