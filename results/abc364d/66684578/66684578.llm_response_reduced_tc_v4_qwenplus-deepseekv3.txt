```cpp
#include <bits/stdc++.h>
using namespace std;
#define ll long long
#define rep(i, n) for(int i = 0; i < (int)(n); i++)

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);
    int N, Q;
    cin >> N >> Q;
    vector<ll> A(N);
    rep(i, N) cin >> A[i];
    sort(A.begin(), A.end());
    
    rep(i, Q) {
        ll b, k;
        cin >> b >> k;
        
        ll low = 0, high = 2e18;
        ll answer = 0;
        
        while (low <= high) {
            ll mid = low + (high - low) / 2;
            auto left = lower_bound(A.begin(), A.end(), b - mid);
            auto right = upper_bound(A.begin(), A.end(), b + mid);
            ll count = distance(left, right);
            
            if (count >= k) {
                answer = mid;
                high = mid - 1;
            } else {
                low = mid + 1;
            }
        }
        
        cout << answer << '\n';
    }
    
    return 0;
}
```

### Explanation:
1. **Sorting**: The array `A` is sorted to allow binary search operations which are crucial for efficiency.
2. **Binary Search for Distance**: For each query point `b` and its k-th closest point, we perform a binary search on the possible distances `d`. The goal is to find the smallest `d` such that there are at least `k` points in `A` within the range `[b - d, b + d]`.
3. **Counting Points within Range**: Using `lower_bound` and `upper_bound`, we efficiently count the number of points in `A` that fall within the current range `[b - d, b + d]` during each binary search step.
4. **Adjusting Search Range**: The binary search adjusts the range based on whether the current midpoint provides enough points (`count >= k`) or not, narrowing down to the correct distance `d`.

This approach ensures that the solution is efficient and handles all edge cases correctly, including large values and the constraints provided. The complexity is O(N log N) for sorting and O(Q log N) for each query, making it suitable for large inputs as specified in the problem.
