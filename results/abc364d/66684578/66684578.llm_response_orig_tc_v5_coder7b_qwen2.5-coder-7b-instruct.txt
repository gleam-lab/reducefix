To solve this problem efficiently, we need to find the k-th closest point for each query. Given the constraints, a straightforward approach of sorting distances for each query would be too slow. Instead, we can use a combination of sorting and binary search to achieve the desired result in an efficient manner.

Here's the step-by-step plan:

1. **Sort Points**: First, sort all the points \(A_i\) and \(B_j\) by their x-coordinates.
2. **Binary Search**: For each query, use binary search to find the k-th closest point efficiently.

Let's implement this approach in C++:

```cpp
#include <bits/stdc++.h>
using namespace std;

#define ll long long
#define rep(i, n) for (int i = 0; i < (int)(n); ++i)

vector<int> A, B;

// Function to perform binary search to find the k-th closest point
int binarySearch(int target, int k) {
    int left = 0, right = A.size() - 1;
    int ans = INT_MAX;
    
    while (left <= right) {
        int mid = left + (right - left) / 2;
        int dist = abs(A[mid] - target);
        
        if (dist < ans) {
            ans = dist;
        }
        
        if (dist == ans) {
            --k;
        }
        
        if (k == 0) break;
        
        if (dist > ans) {
            left = mid + 1;
        } else {
            right = mid - 1;
        }
    }
    
    return ans;
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);
    
    int n, q;
    cin >> n >> q;
    
    A.resize(n);
    B.resize(q);
    
    rep(i, n) cin >> A[i];
    rep(j, q) {
        cin >> B[j].first >> B[j].second;
    }
    
    sort(A.begin(), A.end());
    
    for (auto& [b, k] : B) {
        int ans = binarySearch(b, k);
        cout << ans << '\n';
    }
    
    return 0;
}
```

### Explanation:

1. **Reading Input**: We read the number of points `N` and queries `Q`, followed by the coordinates of points `A` and pairs `(B, k)` for each query.
2. **Sorting Points**: The points `A` are sorted to facilitate binary search.
3. **Binary Search Function**:
   - For each query, we use binary search to find the k-th closest point.
   - We maintain a variable `ans` to store the current minimum distance found.
   - We adjust the `k` value based on whether the current distance matches the minimum distance.
   - Depending on the comparison of distances, we either move the left or right pointer to narrow down the search range.
4. **Output**: For each query, we output the k-th closest distance.

This approach ensures that we handle large inputs efficiently by leveraging sorting and binary search, making it suitable for the given constraints.
