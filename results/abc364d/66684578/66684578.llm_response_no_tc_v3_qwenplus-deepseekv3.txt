### Approaches
1. **Sorting and Binary Search**:
   - First, sort the array `A` to facilitate efficient binary search operations.
   - For each query point `B_j`, compute the distances from `B_j` to all points in `A`, sort these distances, and then pick the k-th smallest distance.
   - To optimize, we can use binary search to find the correct distance range where the k-th closest point lies.

        int left = 0, right = 1e8 * 2;
        int answer = 0;
        while (left <= right) {
            int mid = left + (right - left) / 2;
            auto lower = lower_bound(a.begin(), a.end(), b - mid);
            auto upper = upper_bound(a.begin(), a.end(), b + mid);
            int count = upper - lower;
            if (count >= k) {
                answer = mid;
                right = mid - 1;
            } else {
                left = mid + 1;
            }
        }
        cout << answer << endl;
    }
    
    return 0;
}

### Explanation
1. **Sorting**:
   - The array `A` is sorted to allow binary search operations, which are crucial for efficiently finding the range of points within a certain distance from `B_j`.

2. **Binary Search for Each Query**:
   - For each query point `B_j`, we perform a binary search on the possible distance values (`mid`).
   - Using `lower_bound` and `upper_bound`, we determine how many points in `A` lie within the range `[B_j - mid, B_j + mid]`.
   - If the count of such points is at least `k_j`, we adjust our search range to find a smaller possible distance. Otherwise, we look for a larger distance.

3. **Output**:
   - For each query, the smallest distance `mid` that includes at least `k_j` points is printed. This distance represents the k-th closest distance from `B_j` to the points in `A`.

This approach ensures that each query is processed in logarithmic time relative to the maximum possible distance, making it efficient for large input sizes as specified in the problem constraints.
