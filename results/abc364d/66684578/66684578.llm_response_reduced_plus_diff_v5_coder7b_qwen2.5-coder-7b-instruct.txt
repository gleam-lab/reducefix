To solve this problem efficiently, we need to find the k-th smallest distance between points \(A_i\) and \(B_j\) for each query. Given the constraints, using a brute-force approach would be too slow. Instead, we can use a more efficient method involving sorting and binary search.

Here's the step-by-step approach:

1. **Sort Points**: First, sort all points \(A_i\) and \(B_j\).
2. **Binary Search**: For each query, perform a binary search to find the k-th smallest distance.

Let's implement this in C++:

```cpp
#include <iostream>
#include <vector>
#include <algorithm>
#include <cmath>

using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);
    
    int n, q;
    cin >> n >> q;
    
    vector<int> a(n), b(q);
    for (int i = 0; i < n; ++i) cin >> a[i];
    for (int i = 0; i < q; ++i) cin >> b[i];
    
    // Sort the points
    sort(a.begin(), a.end());
    sort(b.begin(), b.end());
    
    for (int i = 0; i < q; ++i) {
        int target = b[i];
        int k = i + 1; // Convert to 0-based index
        
        // Binary search for the k-th smallest distance
        int left = 0, right = 2 * 1e8; // Upper bound for the maximum distance
        while (left < right) {
            int mid = left + (right - left) / 2;
            int count = 0;
            
            // Count how many points are within distance mid of target
            for (int x : a) {
                if (abs(x - target) <= mid) {
                    count++;
                }
            }
            
            if (count >= k) {
                right = mid;
            } else {
                left = mid + 1;
            }
        }
        
        cout << left << endl;
    }
    
    return 0;
}
```

### Explanation:
1. **Sorting**: We first sort both arrays `a` and `b` to facilitate the binary search.
2. **Binary Search**: For each query, we perform a binary search on the possible distances. The search range is from 0 to \(2 \times 10^8\) because the maximum possible distance between any two points in the given range is \(2 \times 10^8\).
3. **Counting Valid Points**: For each midpoint in our binary search, we count how many points in `a` are within a distance of `mid` from `target`.
4. **Adjusting Search Range**: If the count is at least `k`, it means the k-th smallest distance is less than or equal to `mid`, so we adjust the upper bound. Otherwise, we adjust the lower bound.

This approach ensures that we efficiently find the k-th smallest distance for each query in \(O((n + q) \log n)\) time complexity, which is suitable given the constraints.
