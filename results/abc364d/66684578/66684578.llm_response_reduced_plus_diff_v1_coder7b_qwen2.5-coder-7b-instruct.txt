To solve this problem efficiently, we need to find the k-th closest point from a list of points `A` to a target point `B`. The naive approach would involve calculating all distances and sorting them, which would be inefficient for large inputs. Instead, we can use a more efficient method leveraging heaps (priority queues).

Here's the corrected C++ code:

```cpp
#include <bits/stdc++.h>
using namespace std;

typedef pair<int, int> pii;

int main() {
    ios::sync_with_stdio(false); cin.tie(0); cout.tie(0);

    int n, q;
    cin >> n >> q;

    vector<int> a(n);
    for (int i = 0; i < n; ++i) {
        cin >> a[i];
    }

    while (q--) {
        int b, k;
        cin >> b >> k;

        priority_queue<pii, vector<pii>, greater<pii>> min_heap;

        for (int ai : a) {
            int dist = abs(ai - b);
            min_heap.push({dist, ai});
        }

        for (int i = 1; i < k; ++i) {
            min_heap.pop();
        }

        cout << min_heap.top().first << '\n';
    }

    return 0;
}
```

### Explanation:
1. **Input Reading**: We read the number of points `N`, `Q`, and then the coordinates of points `A`.
2. **Priority Queue Initialization**: For each query, we initialize a min-heap to store pairs of distances and corresponding points.
3. **Distance Calculation and Heap Insertion**: For each point `ai` in `A`, we calculate its distance to `b` and push it into the heap.
4. **Finding the k-th Closest Point**: We pop the smallest element from the heap `k-1` times to reach the k-th smallest element.
5. **Output**: Finally, we output the distance of the k-th closest point.

This approach ensures that we efficiently find the k-th closest point using a min-heap, achieving a time complexity of \(O(Q \cdot N \log N)\), which is suitable for the given constraints.
