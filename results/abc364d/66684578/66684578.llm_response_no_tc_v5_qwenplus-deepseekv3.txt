To fix this code, we need to address the incorrect approach used for finding the k-th closest point to each query point B_j. The original code attempts to use a binary search on a map of cumulative counts, which is not the correct method for solving this problem. Instead, we should sort the array of points A_i and perform binary searches to find the k-th closest point for each query.

### Approaches
1. **Sorting the Array**: First, we sort the array of points A_i to facilitate efficient binary search operations.
2. **Binary Search for Each Query**: For each query point B_j, we:
   - Use binary search to find the position of B_j in the sorted array A.
   - Expand outward from this position to find the k-th closest point, considering both left and right neighbors in the sorted array.
   - Use a priority queue (min-heap) to efficiently track the closest points during the expansion.

### Solution Code
```cpp
#include <bits/stdc++.h>
using namespace std;
#define ll long long
#define rep(i, n) for(int i = 0; i < (int)(n); i++)
#define all(v) v.begin(), v.end()

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);
    
    int N, Q;
    cin >> N >> Q;
    vector<ll> A(N);
    rep(i, N) cin >> A[i];
    
    sort(all(A));
    
    while (Q--) {
        ll b, k;
        cin >> b >> k;
        
        int idx = lower_bound(all(A), b) - A.begin();
        
        int left = idx - 1;
        int right = idx;
        priority_queue<pair<ll, int>, vector<pair<ll, int>>, greater<pair<ll, int>>> pq;
        
        if (idx < N) {
            pq.push({abs(A[right] - b), right});
            right++;
        }
        if (left >= 0) {
            pq.push({abs(A[left] - b), left});
            left--;
        }
        
        ll ans = 0;
        while (k--) {
            auto top = pq.top();
            pq.pop();
            ans = top.first;
            
            int pos = top.second;
            if (pos > left + 1 && right < N) {
                pq.push({abs(A[right] - b), right});
                right++;
            } else if (pos < right - 1 && left >= 0) {
                pq.push({abs(A[left] - b), left});
                left--;
            }
        }
        
        cout << ans << '\n';
    }
    
    return 0;
}
```

### Explanation
1. **Sorting**: The array `A` is sorted to allow efficient binary search operations, which helps in quickly locating the vicinity of each query point B_j.
2. **Binary Search**: For each query point B_j, we use `lower_bound` to find the position where B_j would fit in the sorted array. This gives us the starting point for finding the closest points.
3. **Priority Queue (Min-Heap)**: We use a min-heap to keep track of the closest points to B_j. The heap is initialized with the immediate neighbors of B_j's position in the sorted array.
4. **Expanding Outward**: We expand our search to include points to the left and right of the initial position, adding their distances to the heap. The heap ensures that we always process the closest points first.
5. **Extracting the k-th Closest**: By repeatedly extracting the smallest distance from the heap, we find the k-th closest point after `k` iterations. The distance of this point from B_j is our answer for the query.

This approach efficiently handles each query in logarithmic time relative to the size of the array, making it suitable for the given constraints.
