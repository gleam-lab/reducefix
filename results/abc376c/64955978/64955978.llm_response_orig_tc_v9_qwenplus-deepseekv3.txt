The goal is to find the smallest such x if it exists, otherwise return -1.

### Approach
1. **Sorting**: First, sort both the toys and the boxes in non-decreasing order. This helps in efficiently matching toys to boxes.
2. **Greedy Matching**: Use a greedy approach to match the largest toys to the largest boxes. The largest toy that cannot be matched to any of the existing boxes will need to be placed in the newly purchased box of size x. Thus, x should be at least the size of this toy.
3. **Check Feasibility**: After matching as many toys as possible to the existing boxes, check if the remaining toys can be placed in the newly purchased box. The size of this box must be at least the size of the largest remaining toy, and there should be exactly one such toy left (since we can only have one new box).
