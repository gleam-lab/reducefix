To solve this problem, we need to determine the smallest box size \( x \) that can accommodate all the toys without violating any constraints. The key observation here is that for each toy, we need to find a box whose size is at least as large as the toy's size. 

Here’s how you can approach solving the problem:

1. **Sort the Arrays**: 
   - Sort the array of toy sizes \( A \) in ascending order.
   - Sort the array of box sizes \( B \) in descending order. This allows us to efficiently match the largest toys with the largest available boxes.

2. **Greedy Matching**:
   - Iterate through the sorted toy sizes.
   - For each toy, try to place it in the smallest box among the available boxes that can fit the toy.
   - If we cannot place a toy in any of the remaining boxes, then it means we need a larger box, and we should output `-1`.

Let's implement this approach in C++:

```cpp
#include <bits/stdc++.h>
#define ll long long
using namespace std;

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);

    int N;
    cin >> N;

    vector<ll> A(N), B(N - 1);
    for (int i = 0; i < N; ++i) cin >> A[i];
    for (int i = 0; i < N - 1; ++i) cin >> B[i];

    // Sort A in ascending order
    sort(A.begin(), A.end());

    // Sort B in descending order
    sort(B.rbegin(), B.rend());

    // Initialize index for B
    int j = 0;

    for (int i = 0; i < N; ++i) {
        if (j == N - 1 || A[i] > B[j]) {
            // No suitable box found for the current toy
            cout << -1 << endl;
            return 0;
        }
        while (j < N - 1 && A[i] <= B[j]) {
            ++j;
        }
    }

    // The smallest box size needed is the last box used
    cout << B.back() << endl;

    return 0;
}
```

### Explanation:

1. **Sorting**:
   - `sort(A.begin(), A.end());` sorts the toy sizes in ascending order.
   - `sort(B.rbegin(), B.rend());` sorts the box sizes in descending order.

2. **Iterating and Matching**:
   - We iterate through each toy size.
   - For each toy, we check if we can place it in the current box (smallest available box that fits the toy).
   - If we cannot place the toy in the current box, we increment the box index and continue checking the next box.
   - If we exhaust all boxes before placing all toys, we output `-1`.

This solution ensures that we always use the smallest possible box that can fit each toy, thus minimizing the cost. The time complexity is \(O(N \log N)\) due to the sorting operations, which is efficient enough for the given constraints.
