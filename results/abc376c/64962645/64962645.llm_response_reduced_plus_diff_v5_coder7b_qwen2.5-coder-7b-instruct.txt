To solve this problem, we need to determine if there exists a box size \( x \) such that we can place each toy in a separate box without violating the size constraint. The key observation here is that we need to ensure that for each toy, there is at least one box available that can accommodate it.

Here's a step-by-step approach to solve the problem:

1. **Sort the Arrays**: We start by sorting both arrays `va` (toy sizes) and `vb` (box sizes). This helps us efficiently check the feasibility of placing each toy in a box.

2. **Check Feasibility**: We iterate through the sorted array of toy sizes (`va`). For each toy, we check if there exists a box in the sorted array of box sizes (`vb`) that is at least as large as the current toy. If we find such a box, we mark it as used and move to the next toy. If we cannot find a suitable box for any toy, we conclude that it is impossible to place all toys in separate boxes.

3. **Determine Minimum Box Size**: If we successfully place all toys, we need to find the smallest box size that was used. This will be our answer.

Here is the corrected and optimized C++ code:

```cpp
#include <iostream>
#include <vector>
#include <algorithm>

int main() {
    int n;
    std::cin >> n;
    std::vector<int> va(n), vb(n);
    
    // Read toy sizes
    for (int i = 0; i < n; ++i)
        std::cin >> va[i];
    
    // Read box sizes
    for (int i = 0; i < n - 1; ++i)
        std::cin >> vb[i];
    
    // Sort toy sizes and box sizes
    std::sort(va.begin(), va.end());
    std::sort(vb.begin(), vb.end());
    
    // Check if it's possible to place all toys in separate boxes
    bool possible = true;
    for (int i = 0; i < n && possible; ++i) {
        auto it = std::lower_bound(vb.begin(), vb.end(), va[i]);
        if (it == vb.end()) {
            possible = false;
        } else {
            vb.erase(it); // Mark the box as used
        }
    }
    
    // Determine the result
    if (!possible) {
        std::cout << "-1" << std::endl;
    } else {
        std::cout << va[n - 1] << std::endl;
    }
    
    return 0;
}
```

### Explanation:
1. **Sorting**: Both `va` and `vb` are sorted to facilitate efficient checking using binary search.
2. **Placement Check**: For each toy, we use `std::lower_bound` to find the first box that is at least as large as the toy. If such a box is found, it is marked as used by erasing it from `vb`.
3. **Result Determination**: If we successfully place all toys, the largest toy size (`va[n - 1]`) is printed. If not, `-1` is printed.

This solution ensures that we handle all edge cases and constraints effectively, providing a correct and efficient way to determine the minimum box size required.
