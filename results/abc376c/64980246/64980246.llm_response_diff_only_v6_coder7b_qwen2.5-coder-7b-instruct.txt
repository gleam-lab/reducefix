```cpp
#include <bits/stdc++.h>
using namespace std;

#define ll long long

void solve() {
    int n;
    cin >> n;
    vector<int> a(n), b(n - 1);
    for (auto &tt : a)
        cin >> tt;
    for (auto &tt : b)
        cin >> tt;
    
    // Sort toys in ascending order
    sort(a.begin(), a.end());
    // Sort boxes in descending order
    sort(b.rbegin(), b.rend());

    int x = -1;
    int j = 0;
    for (int i = 0; i < n; ++i) {
        // Check if the current toy can fit into any remaining box
        if (j < n - 1 && a[i] <= b[j]) {
            ++j;
        } else {
            // If we cannot place the current toy in any box, we need a new box
            x = a[i];
            break;
        }
    }

    cout << x << endl;
}

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);

    int t = 1;
    // cin >> t;
    while (t--) {
        solve();
    }
    return 0;
}
```

### Explanation of Changes
1. **Sorting Order**: The toys were initially sorted in descending order, which was incorrect because we need to place smaller toys first. Changed to ascending order for toys.
2. **Box Sorting**: The boxes were sorted in descending order, which means the largest box would always be used first. Changed to descending order to ensure we use the smallest possible box when necessary.
3. **Placement Logic**: Added logic to check if the current toy can fit into any of the remaining boxes. If not, increment `x` to the size of the current toy and break out of the loop.
4. **Efficiency**: Ensured the solution remains efficient by avoiding unnecessary comparisons once a suitable box is found for a toy.

This approach ensures that the smallest possible box is used, adhering to the problem constraints and requirements.
