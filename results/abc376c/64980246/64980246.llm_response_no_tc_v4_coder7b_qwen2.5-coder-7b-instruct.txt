```cpp
#include <bits/stdc++.h>
using namespace std;

#define ll long long

void solve()
{
    int n;
    cin >> n;
    vector<int> a(n), b(n - 1);
    for (auto &tt : a)
        cin >> tt;
    for (auto &tt : b)
        cin >> tt;
    sort(a.begin(), a.end());
    sort(b.begin(), b.end());
    int x = -1;
    for (int i = 0, j = 0; i < n && j < n - 1; ++i)
    {
        if (a[i] <= b[j])
        {
            if (x == -1)
                x = b[j] + 1;
            else
            {
                cout << -1 << endl;
                return;
            }
        }
        else
            ++j;
    }
    cout << x << endl;
}

int main()
{
    std::ios::sync_with_stdio(false);
    std::cin.tie(nullptr);

    int t = 1;
    // cin >> t;
    while (t--)
    {
        solve();
    }
    return 0;
}
```

### Explanation of Fixes:
1. **Sorting Direction**: The original code sorted arrays `a` and `b` in descending order, which was incorrect because it did not align with the problem's requirement of placing toys into boxes where the box size is at least as large as the toy size. The corrected code sorts both arrays in ascending order.
2. **Condition Check**: The condition inside the loop should check if `a[i] <= b[j]`, meaning if the current toy can fit into the current box or not. If it cannot fit, we need to purchase a new box.
3. **Box Size Calculation**: When a new box needs to be purchased, it should be just larger than the largest toy that doesn't fit into the current box, hence `x = b[j] + 1`.
4. **Edge Case Handling**: If at any point a toy does not fit into any available box, the function prints `-1`. This handles scenarios where the largest toy is too big to fit even after purchasing the largest available box.
