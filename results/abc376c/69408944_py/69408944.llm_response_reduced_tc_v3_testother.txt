```python
import bisect

N = int(input())
A = sorted(map(int, input().split()), reverse=True)  # おもちゃのサイズ降順
B = sorted(map(int, input().split()))  # 箱のサイズ昇順

# 各箱は1回しか使えない。大きいおもちゃから順に、入れられる最大の箱 or 新しい箱を使うか決める。
# 貪欲法: 大きいおもちゃに対して、入れられる最小の箱（でも大きすぎず無駄にならない）ではなく、
# 実際には「今使える箱の中で、これ以上大きくないが入る最小の箱」を選ぶべきだが、
# ここでは逆に、大きいおもちゃから見て、使える箱があれば使う（できるだけ小さな適切な箱を使いたいが、
# 今回は既存の箱をうまく使い切れるか？という問題）

# 正しい戦略:
# 1. すべてのおもちゃを既存のN-1個の箱に入れるのは不可能（箱が1つ足りない）ので、1つは新しく買う箱に入る。
# 2. 新しい箱に入るおもちゃを仮決めして、残りのN-1個のおもちゃがN-1個の箱に入るようにマッチングできるか？
#    → そのためには、残りのおもちゃと箱で、すべての玩具が対応する箱に収まるような割り当てがあるか判定。

# 貪欲に：残りの玩具を降順、箱を昇順にして、それぞれ貪欲に割り当て（大きい玩具に小さい箱を割り当てる戦略はNG）
# 正しい貪欲: 残った玩具を昇順、箱を昇順にして、各玩具に対して入れられる最小の箱を割り当てる（二分探索）

# アプローチ:
# 新しい箱に入る玩具を全探索するのはO(N)、それぞれで貪欲O(N log N) → 全体O(N^2 log N) → N=2e5でTLE

# 別のアプローチ:
# まず、すべての玩具とすべての箱を使って、どのくらいまでカバーできるか？
# 「新しい箱に入れるべき玩具」は、貪欲に「どうやっても既存の箱に入らないもの」または「仕方なく外す」もの。

# 正しい方法（公式解法風）:
# 1. すべての玩具 A をソート、箱 B をソート。
# 2. まず、N-1 個の箱で N 個の玩具のうち N-1 個を入れることを考える。
#    → 最大で何個入れられるか？ではなく、「必ず1つ余らせる」という制約。
# 3. 貪欲マッチング: 箱 B を昇順、玩具 A を昇順にして、各箱に収まる最小の玩具を割り当てる（貪欲）。
#    → これで最大何個の玩具が箱に入れられるか？答えは高々 N-1 個。
#    → もし N-1 個以上入れられたら、残り1個を新しい箱に入れる。そのとき新しい箱のサイズはその残った玩具のサイズ以上。
#    → そして、x の最小値は「取り除くべき玩具のサイズの最小値」？いや、取り除く玩具を工夫して、最も小さい x を実現。

# 具体的な戦略:
# - まず、全ての箱 B を使って、できるだけ多くの玩具を入れる（最大マッチング）。
#   昇順の箱に対して、昇順の玩具を貪欲に割り当て。
# - このとき、最大 k 個の玩具が箱に入るので、k >= N-1 であれば、残り1個を新しい箱に入れる。
#   つまり、N-1 個以上入れられれば OK。
# - しかし、正確には「ちょうど N-1 個」入れたい。なので、最大マッチングが N-1 個以上なら、少なくとも1つの選び方が存在。
# - そして、新しい箱に入れる玩具として、どの玩具を残すか？→ その中でサイズが最小になるように選びたい。
#   なぜなら、x はその玩具のサイズ以上でよいので、最小の x は「取り除いても残りが全部箱に入るような玩具のサイズの最小値」。

# アルゴリズム:
# 1. A を昇順、B を昇順。
# 2. 貪欲マッチング: B の各箱に対して、まだ使ってない玩具でサイズが B[i] 以下で最小のものを選ぶ。
#    → 実装上は、A を走査しながらポインタを進めるか、二分探索で。
# 3. このマッチングで、最大何個入れられるか？→ count
#    - count < N-1 → 箱が足りないので、新しい箱をどんなに大きくしても無理 → -1
# 4. count >= N-1 → ある意味OKだが、正確には、N-1 個以上入れられるということは、N-1 個の選び方は存在。
#    - ここで、どの玩具を取り除けばよいか？
#    - 取り除く候補: すべての玩具について、「それを除いたときに残り N-1 個が B の箱に全部入りさえすればよい」
#    - しかし、全探索 O(N * N) は重い。

# 効率的な方法:
# - まず、全体で最大何個マッチできるか？→ total_match
# - total_match < N-1 → -1
# - total_match >= N-1 → ある。
#   さらに、どの玩具を取り除いても残りがマッチできるわけではない。
#   → そこで、貪欲マッチングの結果、どの玩具がマッチしたか記録。
#   - マッチングで使われなかった玩具は、当然新しい箱に入れる候補。
#   - また、マッチングで使われた玩具の中でも、他の組み合わせでカバーできるなら、それを外して新しい箱に入れるのもあり。
#   → しかし、最適戦略: 新しい箱に入れる玩具を v としたとき、残り N-1 個の玩具が箱 B に全部入るか？

# 更に良い方法（典型的な解法）:
# - 新しい箱のサイズ x を固定すると、x 以上の玩具は新しい箱に入れる必要がある（ただし1つまで）。
#   → しかし x を全探索できない。

# 別解（よくある類題より）:
# - まず、貪欲に「B の箱だけで最大いくつ玩具が詰められるか？」を求める。
#   A.sort(), B.sort()
#   j = 0
#   matched = []
#   for i in range(N-1):  # 各箱
#       while j < N and A[j] <= B[i]:
#           j += 1
#       if j == 0: continue
#       # A[0..j-1] の中でまだ使ってない最小のインデックスを選ぶ
#       → 実は、貪欲に「今入れられる最小のまだ使ってない玩具」を選ぶのが最適。
#
# 正しい貪欲マッチング（最大マッチング）:
#   A.sort()
#   B.sort()
#   use = [False] * N
#   count = 0
#   for b in B:
#       # b の箱に、A[i] <= b で、まだ使ってない最小の i を探す
#       for i in range(N):
#           if not use[i] and A[i] <= b:
#               use[i] = True
#               count += 1
#               break
#   → O(N^2) で TLE

# 高速化: 二分探索 or ヒープ
# 方法: 
#   - A を昇順、B を昇順
#   - まだ使ってない玩具をヒープやリストで管理
#   - 各箱 b に対して、b 以下の玩具の中で最小のものを取る（貪欲）

# 実装:
#   toys = sorted(A)
#   boxes = sorted(B)
#   from heapq import heapify, heappop, heappush
#   available = []  # ヒープ（最小値を取り出す）
#   index = 0
#   count = 0
#   for b in boxes:
#       # b 以下の玩具をすべてヒープに追加
#       while index < N and toys[index] <= b:
#           heappush(available, toys[index])
#           index += 1
#       if available:
#           heappop(available)  # 最小のものを入れる（無駄が少ない）
#           count += 1
#       # else: 無視
#   if count < N-1:
#       print(-1)
#   else:
#       # ここで、x の最小値は？
#       # 新しい箱に入れる玩具は、上のマッチングで「ヒープに残ったもの」または「追加されなかったもの」
#       # しかし、上のマッチングは最大マッチングではない？→ 実はこれは正しい貪欲（各箱に最小の入れられる玩具を入れる）で最大マッチングが得られる。
#       # このマッチングで使われなかった玩具の集合: 
#       #   - ヒープに残ったもの（toys[index:] かつ available 内）
#       #   - そもそも b に追加されなかったもの（toys[index:]）
#       # つまり、使われなかった玩具 = (available 内のもの) + (toys[index:])
#       # それらの最小値が x の最小値？
#       # いや、そうではない。なぜなら、別のマッチングでは、別の玩具が残る可能性がある。
#       # 例えば、大きい玩具をあえて残して、小さいのを箱に入れる方が x を小さくできる？
#       # → 新しい箱のサイズ x は、そこにしまう玩具のサイズ以上であればよい。
#       #   したがって、x の最小値は「残しても残りの N-1 個が全部箱に入るような玩具のサイズの最小値」
#       # つまり、すべての玩具 v について、「v を除いたとき、残りの N-1 個の玩具が箱 B に全部入る」かどうかを判定し、そのような v の中で A[v] の最小値を取る。
#
#       # しかし、全探索 O(N^2) は厳しい。
#
# さらに高度なテクニック:
# - 上の貪欲マッチングで得られた「最大マッチング数」が M とする。
# - M >= N-1 ならば、N-1 個のマッチングは可能。
# - ここで、どの玩具を取り除いても残りがマッチできるとは限らない。
# - 逆に、取り除ける玩具の条件: その玩具が「必須」でない場合。
#   → 必須な玩具: すべての最大マッチングに含まれる玩具。
#
# しかし、必須性の判定は難しい。

# 別解（想定解？）:
# - 新しい箱に入れる玩具を v とする。
# - 残りの玩具をソート、箱をソートして、貪欲マッチング（上で述べたヒープを使った方法）で全部入りさえすればよい。
# - v を全探索 O(N)、それぞれ O(N log N) → O(N^2 log N) → TLE

# 高速化のアイデア:
# - 全体の最大マッチングを計算。
# - それが N-1 未満 → -1
# - それが N-1 以上 → ある。
# - さらに、各玩具 v について、「v を除いたときに最大マッチングが N-1 以上」かどうか？
#   → でも N-1 個の箱しかないから、最大 N-1 個まで。つまり「N-1 個のマッチングが存在」＝「完全マッチング」
#
# そこで、全体で最大マッチングが K >= N-1 ならば、K >= N-1 なので、N-1 個のマッチングは存在。
# そして、どの玩具を除いても残りの N-1 個が箱に入るのは無理？→ いや、全体のマッチングで使われなかった玩具を除けば、残りは全部箱に入っている（N-1 個以上マッチしているなら、少なくとも1つの完全マッチングが存在し、そのとき使われなかった玩具を除けばよい）。
#
# 具体的に:
#   - 上の貪欲で、マッチング数が M。
#   - M < N-1 → -1
#   - M >= N-1 → 新しい箱に入れる玩具として、貪欲マッチングで「使われなかった玩具」のどれかを選べばよい。
#     なぜなら、そのとき残りはすべて箱に入っている（M>=N-1 なので、少なくとも N-1 個のマッチングがあり、使われなかった玩具を除けば、残り N-1 個は箱に入っている）。
#   - したがって、x の最小値は「使われなかった玩具のサイズの最小値」
#
# しかし、貪欲マッチングで使われなかった玩具の集合は？
#   - 上のヒープ法では、マッチングで実際に使った玩具は pop された。
#   - 使われなかった玩具 = (ヒープに残ったもの) + (最後まで追加されなかったもの: toys[index:])
#   - つまり、set = set(toys[:index]) - マッチングで使った分？いや、ヒープには toys[0:index] のうち、まだ使われてないものが入っている。
#   - ヒープに残っているものが「使われなかった玩具」の一部。
#   - そして、toys[index:] はどの箱にも追加されなかった（すべての箱 b に対して、toys[i] > b）ので、これらも使われていない。
#
# したがって、使われなかった玩具 = list(available) + toys[index:]
# そして、その最小値が答え？
#
# 例: N=2, A=[999993332, 999997752], B=[662]
#   toys = sorted(A) = [999993332, 999997752]
#   boxes = [662]
#   index=0
#   b=662: while index<2 and toys[index]<=662 -> 999993332<=662? False → 追加なし
#   available は空 → マッチング0個
#   count = 0 < N-1=1 → 条件不成立 → print(-1)
#
# → これが期待される挙動！

# 修正案:
#   - ヒープ法で最大マッチング数を求める。
#   - マッチング数 < N-1 → -1
#   - そうでなければ、使われなかった玩具の最小値を x として出力。

# しかし、使われなかった玩具の最小値は？
#   - available: ヒープに残っているもの（まだ使ってないで、どこかの箱に追加されたが使われなかったもの）
#   - leftover: toys[index:] （どの箱にも追加されなかったもの）
#   - 使われなかった玩具の集合 = set(available) ∪ set(leftover)
#   - min_value = min(min(available), min(leftover)) if both exist

# ただし、available はヒープなので、min は available[0] だが、leftover は別途 min を取る。

# でも、available と leftover は disjoint か？ yes.

# 実装:

N = int(input())
A = list(map(int, input().split()))
B = list(map(int, input().split()))

import heapq

toys = sorted(A)
boxes = sorted(B)

# ヒープを使った貪欲マッチング
available = []  # 最小ヒープ: まだ使ってないで、現在の箱までに追加された玩具
index = 0
count = 0

for b in boxes:
    # b 以下の玩具をすべて available に追加
    while index < N and toys[index] <= b:
        heapq.heappush(available, toys[index])
        index += 1
    # 使える最小の玩具を選ぶ
    if available:
        heapq.heappop(available)
        count += 1

# マッチング数が N-1 未満なら不可能
if count < N - 1:
    print(-1)
else:
    # 使われなかった玩具の候補
    # available に残っているもの + toys[index:] (追加されなかったもの)
    candidates = []
    candidates.extend(available)  # ヒープに残ったもの
    candidates.extend(toys[index:])  # 追加されなかったもの
    # 使われなかった玩具が1個以上あるはず（N - count >= 1）
    x = min(candidates)
    print(x)
```
