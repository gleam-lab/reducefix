To solve this problem, we need to determine if there exists a box size \( x \) such that all toys can be placed in separate boxes without violating any constraints. The key observation here is that the smallest possible box size \( x \) must be at least as large as the largest toy size because any smaller box would not accommodate the largest toy.

Given this insight, the solution involves sorting both the toy sizes and the box sizes, then checking if the largest toy can fit into the largest available box. If it does, then all toys can be accommodated with the remaining boxes.

Here's the corrected C++ code:

```cpp
#include <iostream>
#include <vector>
#include <algorithm>

using namespace std;

int main() {
    int N;
    cin >> N;
    
    vector<int> A(N);
    vector<int> B(N - 1);
    
    for (int i = 0; i < N; i++) cin >> A[i];
    for (int i = 0; i < N - 1; i++) cin >> B[i];
    
    // Sort toy sizes and box sizes
    sort(A.begin(), A.end());
    sort(B.begin(), B.end());
    
    // Check if the largest toy can fit into the largest available box
    if (A[N - 1] <= B[N - 2]) {
        cout << B[N - 2] << endl;
    } else {
        cout << "-1" << endl;
    }
    
    return 0;
}
```

### Explanation:
1. **Sorting**: We first sort the array of toy sizes (`A`) and the array of box sizes (`B`).
2. **Checking Feasibility**: We check if the largest toy (`A[N-1]`) can fit into the second-largest box (`B[N-2]`). This ensures that all toys can be placed in separate boxes.
3. **Output**: If the condition is met, we output the second-largest box size. Otherwise, we output `-1`.

This approach ensures that we efficiently determine the minimum box size required, adhering to the problem constraints and requirements.
