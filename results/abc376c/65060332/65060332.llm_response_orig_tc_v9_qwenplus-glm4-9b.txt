### Proposed Solution:
To solve the problem correctly, we can follow these steps:
1. Sort the toy sizes in ascending order.
2. Sort the existing box sizes in ascending order.
3. Start with the smallest unplaced toy and try to place it in the smallest available box. If the toy cannot be placed (i.e., the box size is smaller than the toy size), then a new box of sufficient size is needed.
4. If more than one toy cannot be placed in an existing box, output `-1` since there isn't a single box size that can hold all toys.
5. If all toys can be placed in existing boxes, output `-1`.

def find_min_box_size(n, toy_sizes, box_sizes):
    toy_sizes.sort()
    box_sizes.sort()

    # Try to place each toy in the smallest box of sufficient size
    for i in range(n):
        if toy_sizes[i] > box_sizes[i]:
            # If the current toy cannot be placed in the current box, a new box is needed
            return toy_sizes[i]
    
    # If all toys can be placed in existing boxes, no new box is needed
    return -1

# Read input
n = int(input())
toy_sizes = list(map(int, input().split()))
box_sizes = list(map(int, input().split()))

# Find the minimum box size
result = find_min_box_size(n, toy_sizes, box_sizes)
