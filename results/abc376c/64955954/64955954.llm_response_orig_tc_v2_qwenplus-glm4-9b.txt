4. For each toy size, it checks if the current box size is less than the toy size.
5. If it finds a box size that is less than the current toy size and also less than the previous toy size, it prints `-1` and exits.
6. If the loop ends without finding such a box, it assumes the largest toy size as the answer and prints it.

### Failing Case Analysis:

1. First, sort the toy sizes (`va`) in ascending order.

4. If we reach a point where all box sizes and all previous toy sizes are less than the current toy size, the current toy size will be the smallest box size needed if we don't have any other options.

#include <iostream>
#include <vector>
#include <algorithm>
using namespace std;
