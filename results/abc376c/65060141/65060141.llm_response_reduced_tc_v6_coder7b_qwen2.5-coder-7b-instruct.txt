#include<bits/stdc++.h>

const ll N = 2e5 + 5;
ll a[N], b[N];

int main() {

    cin >> n;
    
    // Read toy sizes
    for(ll i = 1; i <= n; i++) {
        cin >> a[i];
    }
    
    // Read box sizes
    for(ll i = 1; i < n; i++) {
        cin >> b[i];
    }

    // Sort toy sizes in descending order
    sort(a + 1, a + n + 1, greater<ll>());
    
    // Sort box sizes in ascending order
    sort(b + 1, b + n);

    ll ans = 0;
    for(ll i = 1, j = 1; i <= n; i++) {
        // If we have enough boxes left
        if(j < n) {
            // If current toy fits in the next box
            if(b[j] >= a[i]) {
                j++;  // Move to the next box
            } else {
                ans = max(ans, a[i]);  // Update the answer with current toy size
            }
        } else {
            ans = max(ans, a[i]);  // If no more boxes, update answer with current toy size
        }
    }

    cout << ans << endl;  // Print the result
    return 0;
}

This solution works as follows:

1. **Reading Input**: The program reads the number of toys and their respective sizes, followed by the sizes of the boxes.

2. **Sorting**: 
   - The toy sizes are sorted in descending order to ensure that we try to place the largest toy first.
   - The box sizes are sorted in ascending order to use the smallest available box first.

3. **Placement Logic**:
   - We iterate through each toy and try to place it in the next available box.
   - If the current toy fits in the next box, we move to the next box.
   - If it doesn't fit, we keep track of the largest toy size that couldn't fit using the current box.

4. **Result**: The largest toy size that couldn't fit using the current box is printed as the result.

This approach ensures that we use the smallest possible box while ensuring all toys can be stored without violating any conditions. The time complexity is dominated by the sorting steps, which is \(O(N \log N)\).
