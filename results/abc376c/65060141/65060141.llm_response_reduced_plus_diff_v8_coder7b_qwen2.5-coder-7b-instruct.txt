To solve this problem, we need to determine the smallest box size `x` such that all toys can be placed in separate boxes where each toy fits in a box whose size is at least as large as the toy's size. The key observation here is that if we sort both the sizes of the toys and the sizes of the boxes, we can greedily assign each toy to the smallest available box that can fit it.

Here’s how you can implement this logic:

1. Sort the array of toy sizes (`A`) and the array of box sizes (`B`).
2. Initialize two pointers, one for the toys and one for the boxes.
3. Iterate through the sorted arrays using these pointers and try to place each toy in the smallest available box.
4. If a toy cannot be placed in any remaining box, return `-1`.
5. If all toys can be placed successfully, return the size of the last used box.

Let's write the C++ code based on this approach:

```cpp
#include <bits/stdc++.h>
#define ll long long
using namespace std;

const ll N = 2e5 + 5;
ll a[N], b[N];

int main() {
    ll n, cnt = 0, ans = 0, mx = 0;
    cin >> n;
    
    for (ll i = 1; i <= n; ++i) {
        cin >> a[i];
    }
    for (ll i = 1; i < n; ++i) {
        cin >> b[i];
    }

    // Sort the toy sizes and box sizes
    sort(a + 1, a + n + 1);
    sort(b + 1, b + n);

    ll i = 1, j = 1;
    while (i <= n) {
        if (j <= n - 1 && a[i] <= b[j]) {
            i++;
            j++;
        } else {
            if (j == n) {
                cout << -1 << endl;
                return 0;
            }
            j++;
        }
    }

    cout << b[n - 1] << endl;
    return 0;
}
```

### Explanation:
1. **Input Reading**: We read the number of toys `n` and then the sizes of the toys and boxes.
2. **Sorting**: Both the toy sizes (`A`) and the box sizes (`B`) are sorted.
3. **Greedy Placement**: We use two pointers `i` and `j`. Pointer `i` iterates over the toys, and pointer `j` iterates over the boxes. For each toy, we try to place it in the smallest available box that can fit it.
4. **Edge Cases Handling**: If at any point we run out of boxes to place a toy, we output `-1`.
5. **Result**: If all toys are placed successfully, the size of the last used box is printed.

This solution ensures that we always use the smallest possible box for each toy, minimizing the total cost. The time complexity of this solution is \(O(N \log N)\) due to the sorting step, which is efficient enough for the given constraints.
