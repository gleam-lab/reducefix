### Solution Code
```cpp
#include <vector>
#include <iostream>
#include <algorithm>
#include <cmath>
#include <string>
#include <numeric>

using namespace std;

#define INF (long long)1e18

int main() {
    int n;
    long long dp0, dp1, x, temp;
    dp0 = 0;
    dp1 = -INF;
    cin >> n;

    for (int i = 0; i < n; i++) {
        cin >> x;
        temp = dp0;
        dp0 = max(dp0, dp1);
        dp1 = temp + x;
    }

    cout << max(dp0, dp1) << endl;
    return 0;
}
```

### Explanation
1. **Initialization**: `dp0` is initialized to 0 (no elements taken yet), and `dp1` is initialized to a very small value (`-INF`) to represent an invalid state where no elements have been taken yet.
2. **Reading Input**: For each element in the input sequence:
   - `temp` stores the value of `dp0` before it is updated.
   - `dp0` is updated to the maximum of its current value and `dp1`, representing the best sum achievable without taking the current element.
   - `dp1` is updated to `temp + x`, representing the best sum achievable by taking the current element (which requires that the previous element was not taken).
3. **Output**: The maximum value between `dp0` and `dp1` is printed, representing the maximum sum achievable under the given constraints.

This approach ensures that no two consecutive elements are selected, which matches the expected behavior for problems of this nature.
