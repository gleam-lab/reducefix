### Approaches
1. **Dynamic Programming (DP) Setup**: We will use a DP table where `dp[i][0]` represents the maximum sum up to the i-th element when the last operation was not a doubling, and `dp[i][1]` represents the maximum sum when the last operation was a doubling.
2. **Initial Conditions**: The initial conditions should correctly handle the first element. If we take the first element without any prior doubling, it's just the element itself. If we consider doubling the first element, it's actually not possible since there's no previous operation, so this state should remain invalid.
3. **Transitions**:
   - For `dp[i][0]` (not doubling the i-th element), it can come from either `dp[i-1][0]` or `dp[i-1][1]` (previous state could be either).
   - For `dp[i][1]` (doubling the i-th element), it must come from `dp[i-1][0]` (since doubling can only follow a non-doubling operation).
4. **Result**: The maximum value between the last state (either doubling or not) will be the answer.

#include <bits/stdc++.h>

using namespace std;
using ll = long long;
using vi = vector<ll>;
#define all(v) v.begin(), v.end()
#define rall(v) v.rbegin(), v.rend()
#define fast ios_base::sync_with_stdio(0), cin.tie(0), cout.tie(0);

void solve() {
    int n;
    cin >> n;
    vi v(n);
    for (ll &x : v) cin >> x;
    
    if (n == 1) {
        cout << v[0] << '\n';
        return;
    }
    
    vector<vector<ll>> dp(n, vector<ll>(2, 0));
    dp[0][0] = v[0];
    dp[0][1] = -1e18; // cannot double first element
    
    for (int i = 1; i < n; ++i) {
        dp[i][0] = max(dp[i-1][0], dp[i-1][1]) + v[i];
        dp[i][1] = dp[i-1][0] + 2 * v[i];
    }
    
    cout << max(dp[n-1][0], dp[n-1][1]) << '\n';
}

int main() {
    fast;
    int t = 1;
    while (t--) solve();
    return 0;
}

### Explanation
- **Initialization**: The DP table is initialized for the first element. The first element can be taken as is (state 0) but cannot be doubled (state 1 is set to negative infinity to indicate invalid state).
- **Transitions**: For each subsequent element, the DP states are updated:
  - **State 0 (not doubling)** is derived by taking the maximum of the previous states (0 or 1) and adding the current element.
  - **State 1 (doubling)** is derived by taking the previous state 0 (since doubling follows a non-doubling operation) and adding twice the current element.
- **Result**: The result is the maximum value between the last states (0 or 1) of the DP table, which gives the maximum sum achievable under the given constraints.

This approach efficiently computes the solution in O(N) time with O(N) space, handling all edge cases and constraints correctly.
