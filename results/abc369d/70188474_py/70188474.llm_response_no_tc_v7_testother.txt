```python
n = int(input())
a = list(map(int, input().split()))

# dp[i][0]: 最後に通常で倒した場合の最大経験値 (i体目まで)
# dp[i][1]: 最後に2倍報酬を得て倒した場合の最大経験値 (i体目まで)
# ただし、「2倍報酬」は偶数番目の倒したモンスターにしかつかないので、
# 状態として「これまでに奇数体倒したか（次が偶数番目か）」を管理する必要がある。
#
# 上のコードの誤り: 
# 「dp[i][1] = dp[i-1][0] + 2*s[i-1]」としているが、
# 2倍報酬を得るには「そのモンスターが偶数番目に倒された」ことが必要。
# つまり、それまでに奇数体のモンスターを倒していなければならない。
# しかし、dp[i-1][0] は直前の行動が通常倒しとは限らず、状態設計が間違っている。

# 正しいアプローチ:
# dp[i][j] := i体目まで見て、今まで倒したモンスターの数が j (mod 2) のときの最大経験値
# j=0: 偶数体倒した, j=1: 奇数体倒した

dp = [[0, 0] for _ in range(n+1)]
# 初期状態: 0体倒した → 偶数体倒したとみなす
dp[0][0] = 0
dp[0][1] = -10**18  # 不可能

for i in range(n):
    # モンスターi (0-indexed) を見ている
    for j in range(2):
        if dp[i][j] == -10**18:
            continue
        # 行動1: モンスターを逃す
        dp[i+1][j] = max(dp[i+1][j], dp[i][j])
        
        # 行動2: モンスターを倒す
        new_j = 1 - j  # 倒すと個数の偶奇が反転
        exp = a[i]
        if j == 1:  # 今が奇数体 → 次は偶数番目 → 2倍報酬
            exp += a[i]
        dp[i+1][new_j] = max(dp[i+1][new_j], dp[i][j] + exp)

print(max(dp[n]))
```
