To fix this code, we need to properly handle the counting of selected elements and ensure that the doubling of values is done correctly based on whether the count of selected elements is even or odd. The issue in the original code was that it didn't correctly account for the scenarios where selecting an element could lead to a higher total value when considering the doubling effect.

### Approaches
1. **Dynamic Programming (DP) Setup**: Use a DP array where `dp[i][0]` represents the maximum value up to the i-th element without selecting it, and `dp[i][1]` represents the maximum value up to the i-th element with selecting it.
2. **Count Tracking**: Track the number of selected elements (`cnt`) to determine whether the next selected element should be doubled (if the count is even) or not (if the count is odd).
3. **Transitions**:
   - If we don't select the i-th element, the maximum value is the maximum of the previous state (either selected or not selected).
   - If we select the i-th element, we check the count of selected elements from the previous state and decide whether to double the current element's value based on whether the count is even or odd.

### Solution Code
```cpp
#include<bits/stdc++.h>
#define INF 0x3f3f3f3f3f3f3f3f
using namespace std;
typedef long long ll;
const int MAXN=2e5+10;
ll a[MAXN], dp[MAXN][2];
int cnt[MAXN][2];
int main(){
    ll n;
    scanf("%lld", &n);
    for(ll i=1; i<=n; i++){
        scanf("%lld", &a[i]);
    }
    dp[0][0] = 0;
    dp[0][1] = -INF;
    cnt[0][0] = 0;
    cnt[0][1] = 0;
    for(ll i=1; i<=n; i++){
        // Transition for not selecting a[i]
        if (dp[i-1][0] > dp[i-1][1]) {
            dp[i][0] = dp[i-1][0];
            cnt[i][0] = cnt[i-1][0];
        } else {
            dp[i][0] = dp[i-1][1];
            cnt[i][0] = cnt[i-1][1];
        }
        // Transition for selecting a[i]
        ll val1, val2;
        int new_cnt1 = cnt[i-1][0] + 1;
        if (new_cnt1 % 2 == 0) {
            val1 = a[i] * 2;
        } else {
            val1 = a[i];
        }
        ll sum1 = dp[i-1][0] + val1;
        
        int new_cnt2 = cnt[i-1][1] + 1;
        if (new_cnt2 % 2 == 0) {
            val2 = a[i] * 2;
        } else {
            val2 = a[i];
        }
        ll sum2 = dp[i-1][1] + val2;
        
        if (sum1 > sum2) {
            dp[i][1] = sum1;
            cnt[i][1] = new_cnt1;
        } else {
            dp[i][1] = sum2;
            cnt[i][1] = new_cnt2;
        }
    }
    printf("%lld\n", max(dp[n][0], dp[n][1]));
    return 0;
}
```

### Explanation
1. **Initialization**: The DP arrays `dp` and `cnt` are initialized to store the maximum values and counts of selected elements up to each position.
2. **Transitions**:
   - **Not Selecting the i-th Element**: The value is the maximum from the previous state (either selected or not), and the count remains the same.
   - **Selecting the i-th Element**: The value is computed by considering the count of selected elements from the previous state. If the count is even, the current element is doubled; otherwise, it is taken as-is. The new count is incremented by 1.
3. **Result**: The maximum value between the last state (selected or not) is printed, which gives the optimal solution considering the doubling rules.

This approach ensures that all possible scenarios are considered, and the solution is computed efficiently with a time complexity of O(N).
