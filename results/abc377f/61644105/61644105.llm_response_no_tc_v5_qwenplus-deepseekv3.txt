### Approaches
1. **Understand the Problem**: The task is to count the number of empty squares in an N x N grid where a new piece cannot be captured by any existing pieces. The existing pieces can capture along rows, columns, and both diagonals (i+j = a+b and i-j = a-b).

2. **Inclusion-Exclusion Principle**: We need to count all safe squares by:
   - Total squares: \( N^2 \)
   - Subtract squares under threat (rows, columns, diagonals)
   - Add back squares subtracted multiple times (intersections of rows, columns, diagonals)
   - Subtract intersections of three conditions, and so on.

3. **Efficient Handling of Conditions**: Given constraints \( N \leq 10^9 \) and \( M \leq 10^3 \), we need to efficiently compute the intersections without iterating through all squares. We can use sets to track unique rows, columns, and diagonals, and then compute their intersections.

### Solution Code
```cpp
#include <bits/stdc++.h>
using namespace std;

typedef long long ll;

int main() {
    ll N, M;
    cin >> N >> M;

    set<ll> rows, cols, diag1, diag2;
    for (int i = 0; i < M; ++i) {
        ll a, b;
        cin >> a >> b;
        rows.insert(a);
        cols.insert(b);
        diag1.insert(a + b);
        diag2.insert(a - b);
    }

    ll unsafe = 0;

    // Rows and columns
    unsafe += rows.size() * N;
    unsafe += cols.size() * N;
    unsafe -= rows.size() * cols.size(); // Intersection of rows and columns

    // Diagonals 1 (a + b)
    for (ll d : diag1) {
        ll lower = max(1LL, d - N);
        ll upper = min(N, d - 1);
        if (lower <= upper) {
            unsafe += upper - lower + 1;
        }
    }

    // Diagonals 2 (a - b)
    for (ll d : diag2) {
        ll lower = max(1LL, 1 - d);
        ll upper = min(N, N - d);
        if (lower <= upper) {
            unsafe += upper - lower + 1;
        }
    }

    // Subtract intersections of rows and diagonals
    for (ll r : rows) {
        for (ll d : diag1) {
            ll b = d - r;
            if (1 <= b && b <= N && cols.count(b)) {
                unsafe--;
            }
        }
        for (ll d : diag2) {
            ll b = r - d;
            if (1 <= b && b <= N && cols.count(b)) {
                unsafe--;
            }
        }
    }

    // Subtract intersections of columns and diagonals
    for (ll c : cols) {
        for (ll d : diag1) {
            ll a = d - c;
            if (1 <= a && a <= N && rows.count(a)) {
                unsafe--;
            }
        }
        for (ll d : diag2) {
            ll a = c + d;
            if (1 <= a && a <= N && rows.count(a)) {
                unsafe--;
            }
        }
    }

    // Add intersections of rows, columns, and diagonals (subtracted too many times)
    for (ll r : rows) {
        for (ll c : cols) {
            if (diag1.count(r + c) && diag2.count(r - c)) {
                unsafe++;
            }
        }
    }

    ll total = N * N;
    ll safe = total - unsafe;
    cout << safe << endl;

    return 0;
}
```

### Explanation
1. **Initialization**: We read the grid size \( N \) and the number of pieces \( M \). We use sets to store unique rows, columns, and diagonals (sum and difference diagonals) of the existing pieces.

2. **Unsafe Squares Calculation**: 
   - **Rows and Columns**: Each row and column covered by existing pieces contributes \( N \) unsafe squares. We then adjust for overlaps where rows and columns intersect.
   - **Diagonals**: For each diagonal (sum and difference), we calculate the range of valid squares and count them. Each diagonal contributes its valid range of squares to the unsafe count.
   - **Intersections**: We adjust for overlaps between rows and diagonals, columns and diagonals by checking if the intersection points are already counted in multiple conditions. Finally, we adjust for triple overlaps (rows, columns, and diagonals).

3. **Result Calculation**: The total safe squares are computed by subtracting the unsafe squares from the total squares \( N^2 \). The result is printed.

This approach efficiently handles the constraints using set operations and mathematical range calculations to avoid brute-force checks, making it suitable for large \( N \) values.
