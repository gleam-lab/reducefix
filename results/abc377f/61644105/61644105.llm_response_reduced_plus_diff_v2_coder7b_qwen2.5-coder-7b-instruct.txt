To solve this problem efficiently, we need to determine how many empty squares on an \(N \times N\) grid cannot be captured by any existing pieces. Each piece can capture all squares in its row, column, and two diagonals. We will use a combination of sorting and binary search to achieve this.

Here's a step-by-step breakdown of the solution:

1. **Input Parsing**: Read the input values for \(N\), \(M\), and the positions of the existing pieces.
2. **Sorting**: Sort the positions of the pieces based on their row and column indices.
3. **Binary Search**: For each empty square, check if it can be captured by any existing piece using binary search on the sorted lists of row, column, and diagonal positions.
4. **Count Valid Squares**: Count the number of squares that cannot be captured by any piece.

Here is the corrected and optimized C++ code:

```cpp
#include <bits/stdc++.h>
using namespace std;

#define ll long long

ll n, m;
vector<ll> rows, cols, diags1, diags2;

bool canCapture(ll x, ll y) {
    return 
        binary_search(rows.begin(), rows.end(), x) ||
        binary_search(cols.begin(), cols.end(), y) ||
        binary_search(diags1.begin(), diags1.end(), x + y) ||
        binary_search(diags2.begin(), diags2.end(), x - y);
}

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);

    cin >> n >> m;
    rows.resize(n + 1);
    cols.resize(n + 1);
    diags1.resize(2 * n + 1);
    diags2.resize(2 * n + 1);

    for (ll i = 1; i <= m; ++i) {
        ll x, y;
        cin >> x >> y;
        rows[x] = y;
        cols[y] = x;
        diags1[x + y] = y;
        diags2[x - y + n] = y;
    }

    sort(rows.begin() + 1, rows.end());
    sort(cols.begin() + 1, cols.end());
    sort(diags1.begin() + 1, diags1.end());
    sort(diags2.begin() + 1, diags2.end());

    ll count = 0;
    for (ll i = 1; i <= n; ++i) {
        for (ll j = 1; j <= n; ++j) {
            if (!canCapture(i, j)) {
                ++count;
            }
        }
    }

    cout << count << endl;

    return 0;
}
```

### Explanation:

1. **Input Handling**: The input is read into vectors representing rows, columns, and diagonals.
2. **Sorting**: The vectors are sorted to facilitate binary search.
3. **Can Capture Function**: This function checks if a given position `(x, y)` can be captured by checking if `x`, `y`, `x+y`, or `x-y` exists in the respective sorted vectors.
4. **Counting Valid Squares**: We iterate through all possible positions on the grid and count those that cannot be captured.

This approach ensures that we efficiently check each position and handle the constraints provided.
