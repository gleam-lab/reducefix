
This problem is about finding the number of empty squares in an \(N \times N\) grid where a new piece can be placed such that it is not captured by any of the existing pieces. The grid and the positions of the existing pieces are given.

To solve this, we need to understand the capturing patterns:

1. A piece can capture other pieces in its row.
2. A piece can capture other pieces in its column.
3. A piece can capture other pieces on the diagonal where \(i + j = \text{constant}\).
4. A piece can capture other pieces on the anti-diagonal where \(i - j = \text{constant}\).

We aim to subtract the number of squares that are captured by the existing pieces from the total number of squares in the grid.

### Analysis

1. **Total Squares:** \(N^2\)
2. **Squares in Rows and Columns:** Each row and each column will have \(N\) squares, and if a row or column is completely captured by an existing piece, all squares in that row or column are captured.
3. **Diagonal and Anti-Diagonal Captures:** We need to find the unique diagonals and anti-diagonals that are captured by the existing pieces. This involves checking for each pair of existing pieces whether they lie on the same diagonal or anti-diagonal. If they do, then any new piece placed on that diagonal or anti-diagonal will be captured.
4. **Overlap of Captures:** We have to handle overlaps between row, column, diagonal, and anti-diagonal captures. For example, if a piece captures both a row and a diagonal, the intersection of that row and diagonal would be captured twice.

### Steps to Solve

1. **Initialize Data Structures:** Use sets to store unique rows, columns, diagonals (\(i + j\)), and anti-diagonals (\(i - j\)) that are captured.
2. **Process Existing Pieces:** For each existing piece, check if it captures any row, column, diagonal, or anti-diagonal and add these to the sets.
3. **Calculate Captured Squares:** Calculate the total number of captured squares by considering intersections and avoiding double-counting.
4. **Calculate Remaining Squares:** Subtract the number of captured squares from the total number of squares in the grid.

### Implementation

```cpp
#include <iostream>
#include <vector>
#include <set>

using namespace std;

int N, M;
vector<pair<int, int>> P;

int main() {
    cin >> N >> M;
    P.resize(M);
    for (auto &p : P) cin >> p.first >> p.second;

    set<int> rows, cols, diag1, diag2;
    set<pair<int, int>, greater<pair<int, int>>> diag1_set, diag2_set;

    // Collect captured rows, columns, diagonals, and anti-diagonals
    for (auto &p : P) {
        rows.insert(p.first);
        cols.insert(p.second);
        diag1.insert(p.first + p.second);
        diag2.insert(p.first - p.second);
    }

    // Calculate captured squares
    int captured = 0;
    captured += rows.size() * N; // Captured by rows
    captured += cols.size() * N; // Captured by columns
    captured += diag1.size();    // Captured by diagonals i+j
    captured += diag2.size();    // Captured by diagonals i-j

    // Subtract overlaps
    // Overlaps between rows and columns
    captured -= rows.size() * cols.size();

    // Overlaps between diagonals and anti-diagonals
    for (auto &d1 : diag1) {
        auto it = diag2.lower_bound({-d1.first, 0});
        if (it != diag2.end() && it->first == d1.first) {
            captured -= 2; // Overlap at (d1.first, -d1.first)
        }
    }

    // Total squares minus captured squares
    cout << N * N - captured << endl;

    return 0;
}
```

This implementation initializes data structures to store the captured rows, columns, and diagonals. It processes each piece to update these sets and calculate the total number of captured squares, taking care to handle overlaps correctly. Finally, it outputs the number of empty squares where a new piece can be placed without being captured.
