### Approaches
1. **Identify Attack Lines**: The existing pieces can attack along rows, columns, diagonals (i + j = constant), and anti-diagonals (i - j = constant).
2. **Collect Attack Lines**: For each existing piece, note the row, column, diagonal, and anti-diagonal it influences.
3. **Calculate Attacked Squares**: For each attack line, determine how many squares it affects. However, we must account for overlaps where multiple lines might attack the same square.
4. **Compute Safe Squares**: Subtract the total attacked squares from the total number of squares (N^2), but adjust for overlaps and ensure we only count empty squares not occupied by existing pieces.

    safe = N * N - total_attacked;
    
    // Now, subtract the squares already occupied by pieces (since they are not empty)
    safe -= M;
    
    cout << max(0LL, safe) << endl;
    
    return 0;

### Explanation
1. **Input Reading**: The code reads the grid size N and the number of existing pieces M, followed by their positions.
2. **Attack Lines Identification**: The existing pieces' positions are used to identify rows, columns, diagonals (i - j), and anti-diagonals (i + j) that are under attack.
3. **Inclusion-Exclusion Principle**: The code calculates the total number of squares under attack by applying the inclusion-exclusion principle to avoid double-counting squares attacked by multiple lines.
4. **Safe Squares Calculation**: The total safe squares are computed by subtracting the attacked squares and existing pieces from the total grid squares (N^2).
5. **Output**: The result is printed, ensuring it is non-negative.

This approach efficiently computes the safe squares by leveraging mathematical principles to handle large constraints without excessive memory usage.
