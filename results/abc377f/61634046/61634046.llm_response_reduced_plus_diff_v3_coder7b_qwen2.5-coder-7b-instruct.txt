To solve this problem efficiently, we need to determine how many empty squares on an \(N \times N\) grid cannot be captured by any of the \(M\) existing pieces. The key observation here is that each piece captures squares in four specific diagonals: one main diagonal, one anti-diagonal, and two off-diagonals.

Given these constraints, our approach should focus on counting the number of squares that fall into these capture zones and then subtracting this from the total number of squares to get the number of safe squares.

Here's the corrected and optimized C++ code:

```cpp
#include <bits/stdc++.h>
using namespace std;

const int MAX_N = 1e9 + 10;
const int MAX_M = 1e3 + 10;

int n, m;
set<int> rows, cols, diags1, diags2;

int main() {
    cin >> n >> m;
    
    for (int i = 0; i < m; ++i) {
        int x, y;
        cin >> x >> y;
        rows.insert(x);
        cols.insert(y);
        diags1.insert(x + y);
        diags2.insert(x - y);
    }
    
    long long total_squares = 1LL * n * n;
    long long captured_squares = 0;
    
    captured_squares += rows.size();
    captured_squares += cols.size();
    captured_squares += diags1.size();
    captured_squares += diags2.size();
    
    // Each intersection point is counted twice, so subtract them once
    for (auto x : rows) {
        for (auto y : cols) {
            if (diags1.count(x + y) && diags2.count(x - y)) {
                captured_squares--;
            }
        }
    }
    
    cout << total_squares - captured_squares << endl;
    return 0;
}
```

### Explanation:
1. **Initialization**: We initialize sets to store unique rows, columns, and diagonals (`diags1` and `diags2`) occupied by the pieces.
2. **Reading Input**: We read the grid size `n` and the number of pieces `m`. For each piece, we insert its coordinates into the respective sets.
3. **Counting Captured Squares**: 
   - We count the number of unique rows, columns, and diagonals.
   - Each intersection point (where both a row and a column intersect) is counted twice in the initial sum, so we adjust for this by subtracting the number of intersections.
4. **Output**: Finally, we calculate the number of empty squares as the total number of squares minus the number of captured squares and print the result.

This approach ensures that we handle all edge cases and constraints efficiently, making it suitable for large inputs within the given limits.
