To address the problem, we need to determine how many empty squares on an \( N \times N \) grid remain unoccupied after placing \( M \) pieces on specific positions. A piece cannot be captured if there is no other piece in the same row, column, or diagonal.

Here's a step-by-step approach to solving this problem:

1. **Track Occupied Rows, Columns, and Diagonals**: We'll use sets to keep track of which rows, columns, and diagonals already have pieces.

2. **Count Available Squares**: For each square on the grid, check if it lies on any of the occupied rows, columns, or diagonals. If it does not, count it as available.

3. **Calculate the Result**: The total number of available squares will be \( N^2 \) minus the sum of squares in each occupied row, column, and diagonal.

Here is the corrected and optimized C++ code:

```cpp
#include <bits/stdc++.h>

using namespace std;

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);

    int n, m;
    cin >> n >> m;

    set<int> rows, cols, diags1, diags2;

    for (int i = 0; i < m; ++i) {
        int x, y;
        cin >> x >> y;
        rows.insert(x);
        cols.insert(y);
        diags1.insert(x + y);
        diags2.insert(x - y);
    }

    int ans = n * n - rows.size() - cols.size();

    for (auto diag1 : diags1) {
        int count = 0;
        for (auto row : rows) {
            if (diag1 - row >= 1 && diag1 - row <= n) {
                count++;
            }
        }
        ans -= count;
    }

    for (auto diag2 : diags2) {
        int count = 0;
        for (auto col : cols) {
            if (diag2 + col >= 1 && diag2 + col <= n) {
                count++;
            }
        }
        ans -= count;
    }

    cout << ans << endl;

    return 0;
}
```

### Explanation:
1. **Input Handling**: Read \( N \) and \( M \). Then, read the positions of the \( M \) pieces and store them in appropriate sets (`rows`, `cols`, `diags1`, `diags2`).

2. **Initial Calculation**: Calculate the initial number of available squares by subtracting the sizes of the sets representing occupied rows and columns from \( N^2 \).

3. **Diagonal Adjustments**: For each diagonal, calculate the number of intersections with occupied rows and columns. Subtract these counts from the result.

This approach ensures that we efficiently track and adjust for all occupied spaces, providing the correct number of available squares.
