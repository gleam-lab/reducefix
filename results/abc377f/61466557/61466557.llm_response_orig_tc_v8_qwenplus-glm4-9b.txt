
The problem is about placing a piece on a grid such that it is not captured by any of the existing pieces. The capturing conditions are based on rows, columns, and diagonals of the grid. Given the constraints, especially the large possible size of the grid (up to \(10^9 \times 10^9\)), a direct simulation of all possible movements is infeasible. Therefore, we need a more analytical approach to solve the problem.

To determine the number of safe squares where you can place your piece, we need to account for the following:

1. **Rows and Columns**: Pieces on a square can capture other pieces on the same row and column. Thus, we need to find unique rows and columns that have pieces on them.
2. **Diagonals**: There are two sets of diagonals, one for the "main" diagonals (where \(i + j = a + b\)) and one for the "secondary" diagonals (where \(i - j = a - b\)). We need to account for the unique diagonals that have pieces on them.

Let's break down the solution into steps:

### Step 1: Collect Rows and Columns
- For each piece, add its row and column to two separate sets. This will automatically handle duplicates as the `insert` operation for sets discards duplicates.

### Step 2: Calculate Captured Rows and Columns
- The number of unique rows and columns captured by existing pieces is the size of the two sets.
- We subtract these from the total number of rows and columns (`N` each) to find the number of unaffected rows and columns.

### Step 3: Collect Diagonals
- For the "main" diagonals, compute the difference `a - b` for each piece and store this in a set.
- For the "secondary" diagonals, compute the difference `a + b` for each piece and store this in a set.

### Step 4: Calculate Captured Diagonals
- The number of unique âmainâ and âsecondaryâ diagonals captured by existing pieces is the size of the two sets.
- We subtract these from the total number of "main" and "secondary" diagonals (`N` each) to find the number of unaffected diagonals.

### Step 5: Calculate Safe Squares
- The number of safe squares is the total number of squares minus the number of squares in captured rows, columns, and diagonals. This is calculated as:
  \[
  \text{Safe Squares} = (N \times N) - \text{Captured Rows} \times N - \text{Captured Columns} \times N + \text{Captured Rows} \times \text{Captured Columns}
  \]
- Here, we add back the squares that are captured by both rows and columns since they were subtracted twice.

### Sample Code Implementation

```cpp
#include <iostream>
#include <unordered_set>
#include <vector>

using namespace std;

int main() {
    long long N, M;
    cin >> N >> M;

    unordered_set<int> rows, cols, main_diags, secondary_diags;

    for (int i = 0; i < M; ++i) {
        int a, b;
        cin >> a >> b;
        rows.insert(a);
        cols.insert(b);
        main_diags.insert(a - b);
        secondary_diags.insert(a + b);
    }

    long long captured_rows = rows.size();
    long long captured_cols = cols.size();
    long long captured_main_diags = main_diags.size();
    long long captured_secondary_diags = secondary_diags.size();

    long long safe_squares = N * N - captured_rows * N - captured_cols * N + captured_rows * captured_cols;

    cout << safe_squares << endl;

    return 0;
}
```

This solution efficiently calculates the number of safe squares by leveraging the properties of sets to handle uniqueness and avoid duplicates, which is crucial given the large possible size of the grid.
