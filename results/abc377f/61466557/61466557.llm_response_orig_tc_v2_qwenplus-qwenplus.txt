Here's the **corrected and optimized** C++ code that fixes the logical errors in your original implementation. The main issues were:

1. **Overlapping counts** due to improper handling of intersections between attacked rows, columns, and diagonals.
2. **Incorrect assumptions about exclusivity** when subtracting overlapping areas.
3. **Double-counting or undercounting** captured squares due to not using a unified set for tracking already attacked positions.

The new approach uses a **finite number of existing pieces (M ≤ 1000)** to our advantage by:
- Collecting all attacked cells via each piece.
- Using a hash-based set (`unordered_set`) to store unique attacked coordinates.
- Subtracting the size of this set from total unoccupied positions.

This ensures correctness and avoids overcomplicated math while remaining efficient.

```cpp
#include <bits/stdc++.h>
using namespace std;

using ll = long long;
using pll = pair<ll, ll>;

// Hash function for pair<ll, ll> to be used in unordered_set
struct pair_hash {
    template <class T1, class T2>
    size_t operator()(const pair<T1, T2>& p) const {
        return hash<T1>{}(p.first) * 92837149 + hash<T2>{}(p.second);
    }
};

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);

    ll N, M;
    cin >> N >> M;

    unordered_set<pll, pair_hash> attacked;
    vector<pll> pieces(M);

    for (int i = 0; i < M; ++i) {
        ll x, y;
        cin >> x >> y;
        pieces[i] = {x, y};
    }

    // Add all attacked positions to the set
    for (auto [x, y] : pieces) {
        // Row
        for (ll j = 1; j <= N; ++j)
            attacked.insert({x, j});
        // Column
        for (ll i = 1; i <= N; ++i)
            attacked.insert({i, y});
        // Diagonal 1: x - y = const
        {
            ll d = x - y;
            ll start = max(1LL, d + 1), end = min(N, N + d);
            for (ll i = start; i <= end; ++i)
                attacked.insert({i, i - d});
        }
        // Diagonal 2: x + y = const
        {
            ll s = x + y;
            ll start = max(1LL, s - N), end = min(N, s - 1);
            for (ll i = start; i <= end; ++i)
                attacked.insert({i, s - i});
        }
    }

    // Count how many of those are within bounds and actually exist
    ll totalAttacked = 0;
    for (auto [x, y] : attacked) {
        if (1 <= x && x <= N && 1 <= y && y <= N)
            ++totalAttacked;
    }

    // Total empty squares minus attacked ones
    ll totalSquares = N * N;
    cout << max(0LL, totalSquares - totalAttacked) << '\n';

    return 0;
}
```

### Key Improvements:
- Uses `unordered_set` with custom hashing for fast lookups and uniqueness.
- Efficiently computes attacked cells row-wise, column-wise, and along both diagonal directions.
- Correctly accounts for grid boundaries.
- Handles overlap naturally via the set structure.

### Time Complexity:
Since M ≤ 1000 and for each piece we attack up to 4N squares, but we use a set which automatically de-duplicates, the actual number of insertions is significantly less than 4MN. This is feasible even for large N due to small M.

### Space Complexity:
Proportional to the number of unique attacked cells, which is bounded by N², but acceptable due to constraints on M.
