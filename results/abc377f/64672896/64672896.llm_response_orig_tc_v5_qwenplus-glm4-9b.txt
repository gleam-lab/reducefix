
The problem is to find the number of empty squares on an \( N \times N \) grid where a new piece can be placed without being captured by any of the existing \( M \) pieces. The capturing rules are based on rows, columns, and diagonals.

### Key Observations
1. **Diagonals**: The capturing rules involve diagonals of two types: \( i + j = a + b \) and \( i - j = a - b \). This implies that each diagonal can be uniquely identified by its starting and ending points.
2. **Unique Diagonals**: Not all diagonals are unique. The grid has a limited number of diagonals, and many pieces can be on the same diagonal.
3. **Capture Checks**: A piece placed at any position \( (i, j) \) can capture pieces on several rows, columns, and diagonals. The challenge is to efficiently compute the number of squares that are not under attack by any existing pieces.

### Simplifications
1. **Large \( N \)**: Given the constraints \( 1 \leq N \leq 10^9 \), it's impractical to simulate the grid explicitly. Instead, we should focus on the capturing logic.
2. **Small \( M \)**: Since \( 1 \leq M \leq 10^3 \), we can afford to keep track of each piece's capturing area.

### Strategy
1. **Identify Capturing Areas**: For each piece, identify the rows, columns, and diagonals that would be captured if a piece were placed at that position.
2. **Uniquify Diagonals**: Create a unique identifier for each diagonal to avoid duplicate checks.
3. **Count Free Squares**: Iterate over each square and check if it's within the capturing area of any piece. Count the squares that are not captured.

### Implementation Steps
1. Parse the input to get the size \( N \) and positions of the \( M \) pieces.
2. For each piece, compute the rows, columns, and diagonals that it captures.
3. Use a set or map to keep track of these capturing areas.
4. Iterate over each square on the grid and check if it's captured by any of the pieces. If not, increment the count of free squares.
5. Print the number of free squares.

### Python Code Implementation

```python
from collections import defaultdict

def main():
    import sys
    input = sys.stdin.read
    data = input().split()
    
    N = int(data[0])
    M = int(data[1])
    
    # Read the positions of the M pieces
    pieces = []
    index = 2
    for _ in range(M):
        a = int(data[index])
        b = int(data[index + 1])
        pieces.append((a, b))
        index += 2
    
    # Dictionary to store capturing areas
    captured = defaultdict(set)
    
    # Calculate capturing areas for each piece
    for a, b in pieces:
        # Rows and columns
        for row in range(1, N + 1):
            captured[row].add((a, b))
        for col in range(1, N + 1):
            captured[col].add((a, b))
        
        # Diagonals
        # i+j = a+b
        for k in range(-N + 1, N):
            captured[a + b + k].add((a, b))
        
        # i-j = a-b
        for k in range(-N + 1, N):
            captured[a - b + k].add((a, b))
    
    # Count free squares
    free_squares = N * N
    for row in range(1, N + 1):
        for col in range(1, N + 1):
            if (row, col) in captured:
                free_squares -= 1
    
    # Output the result
    print(free_squares)

if __name__ == "__main__":
    main()
```

This implementation calculates the capturing areas for each piece and then counts the number of free squares by checking each square against these areas. It assumes that \( N \) is large and \( M \) is small, focusing on capturing logic rather than grid simulation.
