To solve this problem efficiently, we need to determine how many empty squares remain unattacked by the existing pieces. Each piece can attack squares based on its position along rows, columns, diagonals (both positive and negative slopes). We will use sets to store all attacked positions and then count the number of empty squares that are not in these sets.

Here's the corrected and optimized C++ code:

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    long long N, M;
    cin >> N >> M;

    set<pair<long long, long long>> attacked_squares;

    for (long long i = 0; i < M; ++i) {
        long long a, b;
        cin >> a >> b;
        
        // Mark all squares attacked by the current piece
        for (long long j = 1; j <= N; ++j) {
            attacked_squares.insert({a, j}); // Same row
            attacked_squares.insert({j, b}); // Same column
            attacked_squares.insert({a + j - b, j}); // Positive diagonal
            attacked_squares.insert({a - j + b, j}); // Negative diagonal
        }
    }

    // Calculate the number of empty squares
    long long empty_squares = N * N - attacked_squares.size();

    cout << empty_squares << '\n';

    return 0;
}
```

### Explanation:

1. **Initialization**: We initialize a set `attacked_squares` to keep track of all squares that are attacked by any piece.

2. **Processing Pieces**: For each piece at position `(a, b)`:
   - We mark all squares in the same row as attacked.
   - We mark all squares in the same column as attacked.
   - We mark all squares on the same positive diagonal (`a + j - b`) as attacked.
   - We mark all squares on the same negative diagonal (`a - j + b`) as attacked.

3. **Counting Empty Squares**: The total number of empty squares is calculated by subtracting the size of the `attacked_squares` set from the total number of squares (`N * N`).

This approach ensures that we efficiently count all attacked squares using a set, which provides average O(log N) time complexity for insertions and lookups. This solution handles all constraints and edge cases effectively.
