To fix this code, we need to ensure that the `cost_for_W` function correctly calculates the minimum cost required to achieve a given W for each stage, considering both types of machines (S and T) optimally. The previous implementation had some issues in the way it calculated the optimal number of machines of each type (x and y) for each stage, leading to incorrect results in certain cases.

### Approaches
1. **Optimal Machine Selection**: For each stage, determine the optimal number of S machines (x) and T machines (y) such that the total productivity of the stage is at least W, and the cost is minimized.
2. **Binary Search**: Use binary search to find the maximum W that can be achieved within the given budget X. The binary search will check for each mid value `W` whether it can be achieved with a cost less than or equal to X.

### Solution Code
```cpp
#include <bits/stdc++.h>
using namespace std;
using ll = long long;

ll cost_for_W(int N, ll W, ll X,
    const vector<ll>& A, const vector<ll>& P,
    const vector<ll>& B, const vector<ll>& Q)
{
    ll total = 0;
    for (int i = 0; i < N; ++i) {
        ll best_cost = X + 1;
        ll max_x = (W + A[i] - 1) / A[i];
        ll x_star = (W * Q[i]) / (A[i] * Q[i] + B[i] * P[i]);
        
        for (ll x = max(0LL, x_star - 2); x <= min(max_x, x_star + 2); ++x) {
            ll rem = W - A[i] * x;
            ll y = (rem <= 0) ? 0 : (rem + B[i] - 1) / B[i];
            ll cost = P[i] * x + Q[i] * y;
            if (cost < best_cost) {
                best_cost = cost;
            }
        }
        
        total += best_cost;
        if (total > X) {
            return X + 1;
        }
    }
    return total;
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int N;
    ll X;
    cin >> N >> X;
    vector<ll> A(N), P(N), B(N), Q(N);
    for (int i = 0; i < N; ++i) {
        cin >> A[i] >> P[i] >> B[i] >> Q[i];
    }

    ll lo = 0;
    ll hi = 0;
    for (int i = 0; i < N; ++i) {
        ll max_contribution = max((X / P[i]) * A[i], (X / Q[i]) * B[i]);
        hi += max_contribution;
    }
    hi += 1;

    while (lo < hi) {
        ll mid = lo + (hi - lo) / 2;
        if (cost_for_W(N, mid, X, A, P, B, Q) <= X) {
            lo = mid + 1;
        } else {
            hi = mid;
        }
    }

    cout << lo - 1 << "\n";
    return 0;
}
```

### Explanation
1. **Optimal Machine Selection**: The `cost_for_W` function calculates the minimal cost to achieve a productivity of at least W for each stage. For each stage, it checks a range of possible numbers of S machines (x) around an optimal point (x_star) derived from the cost and productivity ratios of S and T machines. For each x, it computes the required number of T machines (y) to meet the productivity requirement and calculates the total cost.
2. **Binary Search**: The binary search in the `main` function determines the maximum W that can be achieved within the budget X. The search range is initialized based on the maximum possible contributions from each type of machine in all stages. The search checks whether a given W can be achieved with the budget and adjusts the search range accordingly. The result is the highest W that meets the budget constraint.
