To fix this code, we need to ensure that for each engineering step, we correctly calculate the minimal cost to achieve a certain work capacity (W) without exceeding the budget (X). The original code had some issues in how it calculated the optimal number of machines of each type (S and T) for each step, leading to an incorrect total work capacity.

### Approaches
1. **Binary Search**: We use binary search to determine the maximum possible work capacity (W) that can be achieved within the budget (X). The binary search operates between a lower bound (lo) and an upper bound (hi), where hi is initially set based on the most efficient machine type.
2. **Cost Calculation**: For each midpoint (mid) in the binary search, we calculate the total cost required to achieve a work capacity of mid. If the total cost is within the budget, we adjust the lower bound; otherwise, we adjust the upper bound.
3. **Optimization Per Step**: For each engineering step, we determine the optimal combination of machines of types S and T that minimizes the cost while meeting or exceeding the required work capacity for that step. This involves:
   - Calculating possible values of x (number of S machines) that could potentially yield the minimal cost.
   - For each x, compute the corresponding y (number of T machines) needed to meet the remaining work capacity.
   - Track the minimal cost for the step and ensure it does not exceed the remaining budget.

### Solution Code
```cpp
#include <bits/stdc++.h>
using namespace std;
using ll = long long;

// Function to calculate the minimal cost to achieve work W for each step
ll cost_for_W(int N, ll W, ll X,
    const vector<ll>& A, const vector<ll>& P,
    const vector<ll>& B, const vector<ll>& Q)
{
    ll total = 0;
    for(int i = 0; i < N; i++){
        ll best = X + 1;  // Initialize to a value larger than X

        // The maximum number of S machines that could be needed
        ll max_x = (W + A[i] - 1) / A[i];

        // Function to calculate cost for a given x
        auto calculate_cost = [&](ll x) {
            if (x < 0) return LLONG_MAX;
            ll rem = W - A[i] * x;
            ll y = rem <= 0 ? 0 : (rem + B[i] - 1) / B[i];
            return P[i] * x + Q[i] * y;
        };

        // Check x = 0 (all T machines)
        best = min(best, calculate_cost(0));

        // Check x where all S machines are used
        best = min(best, calculate_cost(max_x));

        // Check x derived from the optimal point (continuous relaxation)
        if (A[i] != 0 && B[i] != 0) {
            long double x_star = (long double)W * Q[i] / ( (long double)A[i]*Q[i] + (long double)B[i]*P[i] );
            ll x_optimal = round(x_star);
            for (ll x = max(0LL, x_optimal - 2); x <= min(max_x, x_optimal + 2); ++x) {
                best = min(best, calculate_cost(x));
            }
        }

        total += best;
        if(total > X) return X + 1;  // Early exit if budget exceeded
    }
    return total;
}

int main(){
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int N;
    ll X;
    cin >> N >> X;
    vector<ll> A(N), P(N), B(N), Q(N);
    for(int i = 0; i < N; i++){
        cin >> A[i] >> P[i] >> B[i] >> Q[i];
    }

    // Determine the upper bound for binary search
    ll max_W = 0;
    for(int i = 0; i < N; i++){
        ll max_A = (X / P[i]) * A[i];
        ll max_B = (X / Q[i]) * B[i];
        max_W += max(max_A, max_B);
    }
    ll lo = 0, hi = max_W + 1;

    // Binary search to find the maximum W
    while(lo < hi){
        ll mid = lo + (hi - lo + 1) / 2;
        if(cost_for_W(N, mid, X, A, P, B, Q) <= X){
            lo = mid;
        } else {
            hi = mid - 1;
        }
    }

    cout << lo << "\n";
    return 0;
}
```

### Explanation
1. **Binary Search Setup**: The binary search is initialized with `lo` set to 0 and `hi` set to an upper bound derived from the maximum possible work capacity achievable by spending the entire budget on the most efficient machine type for each step.
2. **Cost Calculation**: For each midpoint `mid` during the binary search, the function `cost_for_W` calculates the total cost required to achieve `mid` work capacity across all steps. The function checks various potential values of x (number of S machines) for each step, including boundary values and values around the optimal point derived from continuous relaxation.
3. **Early Termination**: If at any point the cumulative cost exceeds the budget, the function returns early to avoid unnecessary calculations.
4. **Binary Search Adjustment**: Based on whether the total cost for `mid` is within the budget, the binary search adjusts `lo` or `hi` to narrow down the maximum achievable work capacity.

This approach ensures that we efficiently and correctly determine the maximum work capacity achievable within the given budget, handling all edge cases and constraints appropriately.
