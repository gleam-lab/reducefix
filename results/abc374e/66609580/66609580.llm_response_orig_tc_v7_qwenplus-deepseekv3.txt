To fix this code, we need to ensure that the binary search correctly identifies the maximum time `v` such that the total cost does not exceed `x`. The issue likely lies in the `check` function where the calculation of the cost might be off by one in certain cases, leading to an incorrect binary search result.

### Approaches
1. **Binary Search**: We perform a binary search on the possible time values to find the maximum time `v` such that the total cost is at most `x`.
2. **Cost Calculation**: For each time `v`, we calculate the cost for each item by considering both the optimal purchases of a combination of `a[i]` and `b[i]` units.
3. **LCM Calculation**: The least common multiple (LCM) of `a[i]` and `b[i]` helps in determining the optimal purchase cycles.
4. **Remaining Time Handling**: After accounting for full cycles, the remaining time is used to determine the cost of the remaining units, either by buying more `a[i]` or `b[i]`.

### Solution Code
```cpp
#include <bits/stdc++.h>
using namespace std;

int n;
long long a[110], b[110], p[110], q[110], x;

inline bool check(long long v) {
    long long total_cost = 0;
    for (int i = 1; i <= n; ++i) {
        long long gcd_ab = __gcd(a[i], b[i]);
        long long lcm_ab = a[i] / gcd_ab * b[i];
        
        long long full_cycles = v / lcm_ab;
        long long cost_full_cycle = min(full_cycles * p[i] * (lcm_ab / a[i]), 
                                        full_cycles * q[i] * (lcm_ab / b[i]));
        long long remaining_time = v % lcm_ab;
        
        long long min_remaining_cost = LLONG_MAX;
        for (long long j = 0; j * a[i] <= remaining_time; ++j) {
            long long remaining_after_a = remaining_time - j * a[i];
            long long k = (remaining_after_a + b[i] - 1) / b[i]; // ceil division
            long long current_cost = j * p[i] + k * q[i];
            if (current_cost < min_remaining_cost) {
                min_remaining_cost = current_cost;
            }
        }
        
        total_cost += cost_full_cycle + min_remaining_cost;
        if (total_cost > x) {
            return false;
        }
    }
    return true;
}

int main() {
    ios::sync_with_stdio(0);
    cin.tie(0), cout.tie(0);
    cin >> n >> x;
    for (int i = 1; i <= n; ++i) {
        cin >> a[i] >> p[i] >> b[i] >> q[i];
    }
    
    long long left = 0;
    long long right = 1LL << 60; // Sufficiently large upper bound
    
    while (left < right) {
        long long mid = (left + right + 1) / 2;
        if (check(mid)) {
            left = mid;
        } else {
            right = mid - 1;
        }
    }
    cout << left << endl;
    return 0;
}
```

### Explanation
1. **Binary Search Setup**: The binary search is initialized with `left` as 0 and `right` as a very large value (`1LL << 60`) to cover all possible time values.
2. **Cost Calculation in `check` function**: For each item, the cost is computed by first determining the cost of full cycles (multiples of LCM of `a[i]` and `b[i]`), then the cost for the remaining time is computed by trying all possible combinations of buying `a[i]` and `b[i]` units.
3. **Binary Search Adjustment**: The binary search adjusts `left` and `right` based on whether the total cost for the mid time value is within the budget `x`. The loop continues until the maximum feasible time is found.
4. **Output**: The result is the maximum time `left` for which the total cost does not exceed `x`.

This approach ensures that the solution is both efficient and correct, handling all edge cases and constraints properly.
