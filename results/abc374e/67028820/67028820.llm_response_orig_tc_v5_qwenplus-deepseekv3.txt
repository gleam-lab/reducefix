The problem involves minimizing the cost to produce a certain number of items, and the binary search needs to accurately determine the maximum number of items that can be produced within the budget.

### Approaches
1. **Binary Search**: We use binary search to determine the maximum number of items (`mid`) that can be produced within the budget `x`.
2. **Cost Calculation**: For each item type, calculate the cost to produce `mid` items by first using the most cost-effective bundles (either `a[i]` items for `p[i]` cost or `b[i]` items for `q[i]` cost) and then filling the remaining items with the next best option.
3. **Precision Handling**: To avoid floating-point precision issues, we should use integer arithmetic wherever possible. Specifically, the cost per item for the remaining items should be calculated by comparing `p[i] / a[i]` and `q[i] / b[i]` using cross-multiplication to avoid division.

### Solution Code
```cpp
#include <bits/stdc++.h>

using namespace std;

#define int long long

int n, x;
vector<int> a, b, p, q;

bool canMake(int mid) {
    int total_cost = 0;
    for (int i = 0; i < n; i++) {
        int s_num = p[i];
        int s_den = a[i];
        int t_num = q[i];
        int t_den = b[i];

        if (s_num * t_den < t_num * s_den) {
            int bundles = mid / a[i];
            total_cost += bundles * p[i];
            int remaining = mid - bundles * a[i];
            if (remaining > 0) {
                int cost_per_item = (t_num + t_den - 1) / t_den;
                total_cost += remaining * cost_per_item;
            }
        } else {
            int bundles = mid / b[i];
            total_cost += bundles * q[i];
            int remaining = mid - bundles * b[i];
            if (remaining > 0) {
                int cost_per_item = (s_num + s_den - 1) / s_den;
                total_cost += remaining * cost_per_item;
            }
        }
        if (total_cost > x) {
            return false;
        }
    }
    return total_cost <= x;
}

signed main() {
    cin >> n >> x;
    a.resize(n); b.resize(n); p.resize(n); q.resize(n);
    for (int i = 0; i < n; i++) cin >> a[i] >> p[i] >> b[i] >> q[i];

    int left = 0, right = 1e18;
    while (left < right) {
        int mid = (left + right + 1) / 2;
        if (canMake(mid)) {
            left = mid;
        } else {
            right = mid - 1;
        }
    }

    cout << left;

    return 0;
}
```

### Explanation
1. **Binary Search Setup**: The binary search is initialized with `left` as 0 and `right` as a large value (1e18) to cover the possible range of items.
2. **Cost Calculation**: In the `canMake` function, for each item type, we compare the cost per item of two options (`a[i]` items for `p[i]` cost and `b[i]` items for `q[i]` cost) using cross-multiplication to avoid floating-point division. This ensures precision.
3. **Bundles and Remaining Items**: For the optimal option, we calculate the number of full bundles that can be used and the remaining items. The cost for the remaining items is calculated by rounding up the per-item cost to ensure we stay within budget.
4. **Early Termination**: If at any point the total cost exceeds the budget `x`, the function returns early to save computation.
5. **Binary Search Adjustment**: The binary search adjusts the bounds based on whether the current `mid` value is feasible (can be produced within the budget) or not, ensuring the maximum feasible value is found.

This approach efficiently narrows down the maximum number of items that can be produced within the budget while avoiding precision errors and ensuring correctness.
