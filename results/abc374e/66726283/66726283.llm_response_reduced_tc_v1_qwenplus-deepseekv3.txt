### Approaches
1. **Binary Search Setup**: The binary search should be performed over the possible range of days, starting from 0 to a sufficiently large upper bound, which can be derived based on the constraints.
2. **Cost Calculation**: For each day `mid`, compute the minimum cost for each item by considering the optimal purchase strategy between two types of purchases (using `a[i]` and `b[i]`).
3. **Check Function**: The `check` function should verify if the total cost for all items, given `mid` days, is within the budget `x`.
4. **Binary Search Execution**: Adjust the binary search bounds based on whether the current `mid` is feasible (cost <= x) or not.

### Solution Code
```cpp
#include <bits/stdc++.h>
using namespace std;
#define int long long
const int N = 105;
int n, x;
int a[N], b[N], p[N], q[N];

bool check(int mid) {
    int total_cost = 0;
    for (int i = 1; i <= n; ++i) {
        int min_cost = LLONG_MAX;
        // Try all possible j where j is the number of times we buy type a[i]
        for (int j = 0; j <= mid / a[i] + 2; ++j) {
            if (j * a[i] > mid) continue;
            int remaining = mid - j * a[i];
            int k = (remaining + b[i] - 1) / b[i]; // ceil(remaining / b[i])
            if (k < 0) k = 0;
            int cost = j * p[i] + k * q[i];
            if (cost < min_cost) {
                min_cost = cost;
            }
        }
        // Try all possible j where j is the number of times we buy type b[i]
        for (int j = 0; j <= mid / b[i] + 2; ++j) {
            if (j * b[i] > mid) continue;
            int remaining = mid - j * b[i];
            int k = (remaining + a[i] - 1) / a[i]; // ceil(remaining / a[i])
            if (k < 0) k = 0;
            int cost = j * q[i] + k * p[i];
            if (cost < min_cost) {
                min_cost = cost;
            }
        }
        total_cost += min_cost;
        if (total_cost > x) {
            return false;
        }
    }
    return total_cost <= x;
}

signed main() {
    ios::sync_with_stdio(0);
    cin.tie(0);
    cout.tie(0);
    cin >> n >> x;
    for (int i = 1; i <= n; ++i) {
        cin >> a[i] >> p[i] >> b[i] >> q[i];
    }

    int left = 0;
    int right = 1e18; // Sufficiently large upper bound
    int answer = 0;
    while (left <= right) {
        int mid = left + (right - left) / 2;
        if (check(mid)) {
            answer = mid;
            left = mid + 1;
        } else {
            right = mid - 1;
        }
    }
    cout << answer << endl;
    return 0;
}
```

### Explanation
1. **Binary Search Setup**: The binary search is initialized with `left` as 0 and `right` as a very large number (1e18) to cover all possible days.
2. **Cost Calculation**: For each item, the code calculates the minimum cost by considering all possible combinations of purchasing the item either at rate `a[i]` or `b[i]`. This involves iterating through possible counts of each purchase type and computing the remaining days needed, adjusting for ceiling division.
3. **Check Function**: The function sums the minimum costs for all items. If at any point the total exceeds the budget `x`, it returns false; otherwise, it returns true if the total is within the budget.
4. **Binary Search Execution**: The binary search adjusts the bounds based on the feasibility of the current midpoint (`mid`). If feasible, it searches for a higher value; otherwise, it searches lower. The maximum feasible value is tracked and printed as the result.

This approach ensures that the solution is both efficient and correct, handling all edge cases and constraints effectively.
