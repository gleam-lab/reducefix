```cpp
#include <bits/stdc++.h>
using namespace std;
#define int long long
const int N = 105;

int n, x;
int a[N], b[N], p[N], q[N];
int c[N];

bool check(int mid) {
    for (int i = 1; i <= n; i++) {
        c[i] = INT_MAX;
        for (int j = 0; j <= mid / a[i]; j++) {
            c[i] = min(c[i], j * p[i] + max(0LL, mid - j * a[i]) * q[i] / b[i]);
        }
        for (int j = 0; j <= mid / b[i]; j++) {
            c[i] = min(c[i], j * q[i] + max(0LL, mid - j * b[i]) * p[i] / a[i]);
        }
    }
    int ans = 0;
    for (int i = 1; i <= n; i++) {
        ans += c[i];
    }
    if (ans > x) return false;
    return true;
}

signed main() {
    ios::sync_with_stdio(0);
    cin.tie(0);
    cout.tie(0);
    cin >> n >> x;
    for (int i = 1; i <= n; i++) {
        cin >> a[i] >> p[i] >> b[i] >> q[i];
    }

    int l = 0, r = x;
    while (l < r) {
        int mid = (l + r + 1) / 2;
        if (check(mid)) {
            l = mid;
        } else {
            r = mid - 1;
        }
    }

    cout << l << endl;
    return 0;
}
```

### Explanation of Fixes:
1. **Initialization**: The initial value of `l` was set to 1, which might not be optimal. Setting `l` to 0 allows for more flexibility in the binary search.
2. **Binary Search Adjustment**: Changed the binary search condition to `(l + r + 1) / 2` to ensure that the midpoint calculation rounds up, which helps in handling the case where the exact middle value is needed.
3. **Edge Case Handling**: In the `check` function, added `max(0LL, ...)` to ensure that division does not lead to negative results when subtracting from the target production capacity (`mid`). This prevents potential issues with invalid calculations.
4. **Output Formatting**: Ensured that the output is printed with a newline character at the end to match the expected format.
