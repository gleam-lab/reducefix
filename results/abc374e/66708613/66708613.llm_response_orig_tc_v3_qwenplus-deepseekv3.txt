### Approaches
1. **Binary Search**: We perform a binary search on the possible processing capacities to determine the maximum achievable capacity within the budget.
2. **Cost Calculation**: For each processing capacity, calculate the minimum cost required to achieve that capacity across all machines. This involves checking both types of operations (A and B) for each machine and determining the optimal combination that minimizes the cost for the desired capacity.
3. **Budget Check**: After calculating the total cost for a given capacity, check if it fits within the budget X. If it does, we can try higher capacities; otherwise, we try lower capacities.

### Solution Code
```cpp
#include<iostream>
#include<vector>
#include<algorithm>
#include<climits>
using namespace std;
using ll = long long;

int main() {
    int N;
    ll X;
    cin >> N >> X;
    vector<ll> A(N), P(N), B(N), Q(N);
    for (int i = 0; i < N; ++i) {
        cin >> A[i] >> P[i] >> B[i] >> Q[i];
    }

    auto f = [&](ll x) {
        ll total_cost = 0;
        for (int i = 0; i < N; ++i) {
            ll min_cost = LLONG_MAX;
            // Check all possible s (number of A operations)
            // The maximum s needed is ceil(x / A[i]) or Q[i] / any other relevant constraint
            // To optimize, we limit s to up to (x + A[i] - 1) / A[i]
            ll max_s = min((x + A[i] - 1) / A[i], B[i]);
            for (ll s = 0; s <= max_s; ++s) {
                ll remaining = x - A[i] * s;
                if (remaining <= 0) {
                    min_cost = min(min_cost, s * P[i]);
                    continue;
                }
                ll t = (remaining + B[i] - 1) / B[i];
                if (t > Q[i]) continue; // t cannot exceed Q[i]
                min_cost = min(min_cost, s * P[i] + t * Q[i]);
            }
            // Similarly, check all possible t (number of B operations)
            ll max_t = min((x + B[i] - 1) / B[i], A[i]);
            for (ll t = 0; t <= max_t; ++t) {
                ll remaining = x - B[i] * t;
                if (remaining <= 0) {
                    min_cost = min(min_cost, t * Q[i]);
                    continue;
                }
                ll s = (remaining + A[i] - 1) / A[i];
                if (s > P[i]) continue; // s cannot exceed P[i]
                min_cost = min(min_cost, t * Q[i] + s * P[i]);
            }
            if (min_cost == LLONG_MAX) {
                return false;
            }
            total_cost += min_cost;
            if (total_cost > X) {
                return false;
            }
        }
        return total_cost <= X;
    };

    ll low = 0, high = 1LL << 60;
    while (low < high) {
        ll mid = low + (high - low + 1) / 2;
        if (f(mid)) {
            low = mid;
        } else {
            high = mid - 1;
        }
    }
    cout << low << endl;
    return 0;
}
```

### Explanation
1. **Binary Search Setup**: The binary search is initialized with `low` set to 0 and `high` set to a very large value (1LL << 60), representing the possible range of processing capacities.
2. **Feasibility Check (f function)**: For each mid value in the binary search, the function `f` checks if it feasible to achieve the processing capacity `mid` within the budget X. For each machine, it calculates the minimum cost required to achieve the capacity by considering all possible combinations of operations (A and B).
3. **Cost Calculation**: For each machine, the function checks two scenarios: varying the number of A operations (s) and varying the number of B operations (t), ensuring that the remaining capacity is covered by the other operation. The minimum cost for each scenario is recorded.
4. **Budget Validation**: The total cost across all machines is compared against the budget X. If the total cost exceeds X, the function returns false, indicating the capacity is not achievable. Otherwise, it returns true.
5. **Binary Search Adjustment**: Based on the feasibility check, the binary search adjusts the range. If a capacity is achievable, the search continues for higher capacities; otherwise, it searches lower capacities.
6. **Output**: The binary search concludes with the maximum achievable processing capacity, which is printed as the result.
