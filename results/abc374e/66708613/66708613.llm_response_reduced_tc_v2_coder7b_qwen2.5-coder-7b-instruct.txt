```cpp
#include <iostream>
#include <vector>
#include <algorithm>

using namespace std;

bool canAchieveCapacity(const vector<pair<long long, long long>>& machines, long long budget, long long targetCapacity) {
    for (const auto& [A, P] : machines) {
        long long units = max(0LL, targetCapacity / A) + 1;
        long long cost = units * P;
        long long remainingBudget = budget - cost;
        if (remainingBudget < 0) {
            return false;
        }
    }
    return true;
}

int main() {
    int N;
    long long X;
    cin >> N >> X;
    vector<pair<long long, long long>> machinesS(N), machinesT(N);

    for (int i = 0; i < N; ++i) {
        long long A, B, P, Q;
        cin >> A >> P >> B >> Q;
        machinesS[i] = {A, P};
        machinesT[i] = {B, Q};
    }

    long long low = 0, high = 1e14;
    while (low < high) {
        long long mid = (low + high + 1) / 2;
        if (canAchieveCapacity(machinesS, X, mid) || canAchieveCapacity(machinesT, X, mid)) {
            low = mid;
        } else {
            high = mid - 1;
        }
    }

    cout << low << endl;
    return 0;
}
```

### Explanation:
1. **Function `canAchieveCapacity`**: This function checks if it's possible to achieve at least the target capacity using the given budget. It iterates over each type of machine and calculates the minimum number of units needed to meet the target capacity. It then checks if the budget allows purchasing those units.
   
2. **Main Function**:
   - Reads the number of processes \(N\) and the total budget \(X\).
   - Stores the processing rates and costs for both types of machines.
   - Uses binary search to find the maximum achievable capacity. The search range starts from 0 to a large value (1e14 in this case).
   - For each midpoint in the binary search, it checks if it's possible to achieve that capacity using either type of machine. If it is, it updates the lower bound (`low`). Otherwise, it updates the upper bound (`high`).
   
This approach ensures that the solution is efficient and handles all edge cases correctly.
