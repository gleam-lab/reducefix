
The problem can be solved by using a binary search on the answer. 

The minimum value of the answer is \(0\), and the maximum value is \(N-1\). 

1. The idea is to use binary search for each possible answer, and check if it's possible to remove \(K\) elements such that the maximum and minimum of the remaining sequence are at least \(x\) (the current guess of the answer).
2. For the check, we maintain a frequency array that counts how many times each element appears. We iterate over the numbers in descending order and greedily remove them until we can no longer remove \(K\) elements while keeping the remaining elements at least \(x\).
3. If we can remove \(K\) elements, then the current guess is possible; otherwise, it's not.
4. We continue the binary search based on whether the current guess was possible or not.

bool check(int x, int n, int k, vector<int> &num) {
    vector<int> freq(n + 1);
    for (int num_val : num) {
        freq[num_val]++; 
    }
    
    int counts = 0;
    for (int i = n; i >= x; i--) {
        counts += freq[i];
        if (counts >= k) return true;
    }
    return false;
}

    int left = 0, right = n - 1, ans = 0;
    while (left <= right) {
        int mid = left + (right - left) / 2;
        if (check(mid, n, k, num)) {
            ans = mid;
            right = mid - 1;
        } else {
