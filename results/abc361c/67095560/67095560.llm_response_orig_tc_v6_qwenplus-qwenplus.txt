To solve this problem optimally, we need to **minimize the difference between the maximum and minimum values in a subsequence of length N-K**, formed by removing exactly K elements from the original sequence.

### Key Insight:
- The best strategy is to consider removing K elements such that the remaining elements have the smallest possible range (max - min).
- Since order matters (we must keep the relative order), we can't just sort the array and pick the closest N-K numbers.
- However, if we sort the array first, then any contiguous subsequence of length N-K gives a candidate for the minimal range.

This leads us to the following efficient solution:

### Efficient Algorithm:
1. Sort the array → O(N log N)
2. Slide a window of size N-K over the sorted array.
3. For each window, compute `A[i + (N-K-1)] - A[i]` and track the minimum.

This approach guarantees an O(N log N) solution which works within the constraints.

Here's the corrected and optimized code:

```cpp
#include <bits/stdc++.h>
using namespace std;

#define endl "\n"
#define int long long

void solve() {
    int N, K;
    cin >> N >> K;
    vector<int> A(N);
    for (int i = 0; i < N; ++i) {
        cin >> A[i];
    }

    // Sort the array
    sort(A.begin(), A.end());

    int minDiff = LLONG_MAX;
    // Slide a window of size N-K
    for (int i = 0; i + (N - K - 1) < N; ++i) {
        minDiff = min(minDiff, A[i + (N - K - 1)] - A[i]);
    }

    cout << minDiff << endl;
}

signed main() {
    ios_base::sync_with_stdio(false);
    cin.tie(NULL);
    solve();
    return 0;
}
```
