To solve the problem correctly, we need to **remove exactly K elements** from the list of N integers such that the **difference between the maximum and minimum of the remaining elements is minimized**.

### Key Observations:

- We are allowed to remove **any K elements**, not necessarily contiguous.
- The final sequence B must consist of **N-K elements**, in their original order — but for our purposes (since we only care about the min and max), we can ignore the ordering and just focus on selecting a subset of N-K elements with minimal `max - min`.

### Strategy:

1. **Sort the array**: Sorting helps us consider contiguous windows of size `N-K`, because removing K elements optimally means choosing a subset of N-K elements with the smallest spread.
2. **Sliding window over sorted array**:
   - Consider all windows of size `N-K` in the sorted array.
   - For each window, compute `max - min`.
   - Keep track of the minimum difference found.

This approach has **O(N log N)** time due to sorting, and is efficient enough given constraints (`N ≤ 200000`).

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    int N, K;
    cin >> N >> K;
    vector<long long> A(N);
    for (int i = 0; i < N; ++i) {
        cin >> A[i];
    }

    sort(A.begin(), A.end());

    long long result = LLONG_MAX;
    int window_size = N - K;

    // Slide window of size N-K over the sorted array
    for (int i = 0; i + window_size <= N; ++i) {
        result = min(result, A[i + window_size - 1] - A[i]);
    }

    cout << result << endl;
    return 0;
}
```
