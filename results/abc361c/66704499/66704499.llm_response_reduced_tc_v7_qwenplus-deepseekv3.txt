### Correct Approach
The correct approach involves:
1. **Sliding Window Technique**: Since we need to maintain the original order, we can consider all possible windows of size (N - K) in the original array. For each window, we calculate the difference between the maximum and minimum values within that window.
2. **Efficient Calculation**: To efficiently find the min and max in each window, we can use a data structure that allows O(1) or O(log N) min and max queries, such as a deque (for sliding window maximum/minimum) or a segment tree.

However, given the constraints (N up to 2 × 10^5), an O(N log N) solution is acceptable. We can sort the array and then consider all possible windows of size (N - K) in the sorted array to find the minimum difference between the max and min in any such window. int main() {

    vector<int> A(N);

    }
    sort(A.begin(), A.end());
    int min_diff = INT_MAX;
    for (int i = 0; i + (N - K) - 1 < N; ++i) {
        int current_diff = A[i + (N - K) - 1] - A[i];
        if (current_diff < min_diff) {

        }
    }

    return 0;
}
