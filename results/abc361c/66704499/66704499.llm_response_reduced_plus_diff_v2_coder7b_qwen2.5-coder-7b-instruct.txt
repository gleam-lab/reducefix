To solve this problem, we need to find the minimum possible value of the difference between the maximum and minimum values of the sequence after removing `K` elements. The key observation here is that the optimal way to minimize this difference is to keep the smallest `N-K` elements and discard the largest `K` elements.

Here's the step-by-step approach:

1. **Sort the Sequence**: Sorting helps us easily identify the smallest and largest elements.
2. **Calculate the Difference**: After sorting, the smallest `N-K` elements will be at the beginning of the array, and the largest `K` elements will be at the end. The difference between the maximum and minimum of these two groups will give us the desired result.

Let's implement this in C++:

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(0);

    int n, k;
    cin >> n >> k;

    vector<int> nums(n);
    for (int i = 0; i < n; i++) {
        cin >> nums[i];
    }

    sort(nums.begin(), nums.end());

    // The minimum possible value of the difference
    int min_diff = nums[n-1] - nums[k];

    cout << min_diff << endl;

    return 0;
}
```

### Explanation:
1. **Sorting**: We sort the array `nums`.
2. **Finding the Minimum Difference**: After sorting, the smallest `N-K` elements are from index `0` to `K-1`, and the largest `K` elements are from index `N-K` to `N-1`. The minimum possible value of the difference between the maximum and minimum values of the remaining elements is simply the difference between `nums[N-1]` (the largest element) and `nums[K]` (the `(K+1)`-th smallest element).

This solution ensures that we always consider the smallest possible range of elements after removing `K` elements, thus minimizing the difference. The time complexity is dominated by the sorting step, which is \(O(N \log N)\), making it efficient enough for the given constraints.
