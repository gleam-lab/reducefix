```cpp
#include <bits/stdc++.h>
using namespace std;

int nums[200000];

int main(void){
    int n, k;
    cin >> n >> k;

    for (int i = 0; i < n; i++){
        cin >> nums[i];
    }

    sort(nums, nums + n);

    int ans = INT_MAX;
    for(int i = 0; i <= k; i++) {
        int left_max = max(nums[i], nums[n-k+i-1]);
        int right_min = min(nums[k-i], nums[n-1-i]);
        ans = min(ans, left_max - right_min);
    }

    cout << ans << endl;

    return 0;
}
```

This solution works by iterating over all possible pairs of indices `(i, n-k+i-1)` that could potentially be the boundaries of the resulting sequence after removing `K` elements. For each pair, it calculates the potential maximum (`left_max`) and minimum (`right_min`) values of the resulting sequence and updates the answer with the smallest difference between these two values found so far. This approach ensures that we consider all possible ways to partition the array into two parts after removing `K` elements and find the configuration that minimizes the difference between the maximum and minimum values of the resulting sequence. The time complexity of this solution is O(N log N) due to sorting, followed by a linear scan, which is efficient enough given the problem constraints.
