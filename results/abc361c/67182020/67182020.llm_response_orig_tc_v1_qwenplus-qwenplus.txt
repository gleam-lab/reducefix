To solve this problem, we must **remove exactly K elements** from a sequence of N integers such that the **difference between the maximum and minimum values in the remaining elements is minimized**.

---

### Key Insight:

After removing K elements from the list of N elements, we are left with (N - K) elements. We want to choose these (N - K) elements such that the difference between the **maximum** and **minimum** among them is as small as possible.

Since the order of the remaining elements must be preserved (they're concatenated in original order), but only their values matter for max-min, the optimal approach is:

- Sort the array.
- Consider all **consecutive subarrays of length (N-K)**.
- For each such subarray, compute `max - min` (which is simply `A[i + (N-K) - 1] - A[i]` after sorting).
- Return the **minimum** value among all these differences.

This runs in O(N log N) time due to sorting, which is acceptable given constraints up to 200,000.

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);

    int N, K;
    cin >> N >> K;
    vector<int> A(N);
    for (int i = 0; i < N; ++i) {
        cin >> A[i];
    }

    sort(A.begin(), A.end());

    int window_size = N - K;
    long long min_diff = LLONG_MAX;

    for (int i = 0; i <= N - window_size; ++i) {
        min_diff = min(min_diff, 1LL * A[i + window_size - 1] - A[i]);
    }

    cout << min_diff << endl;

    return 0;
}
```
