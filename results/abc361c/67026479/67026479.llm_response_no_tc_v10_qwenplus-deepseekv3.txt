3. **Efficient Calculation**: To find the minimal difference, we can use a sliding window approach or consider all possible contiguous windows of size (N - K) in the original array. ### Solution Code
```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    int N, K;
    cin >> N >> K;
    vector<int> A(N);
    for (int i = 0; i < N; ++i) {
        cin >> A[i];
    }
    
    sort(A.begin(), A.end());
    int min_diff = INT_MAX;
    int m = N - K;
    
    for (int i = 0; i + m - 1 < N; ++i) {
        int current_diff = A[i + m - 1] - A[i];
        if (current_diff < min_diff) {
            min_diff = current_diff;
        }
    }
    
    cout << min_diff << endl;
    return 0;
}
```

### Explanation
1. **Sorting the Array**: By sorting the array, we can easily access the smallest and largest elements in any contiguous subarray of size (N - K).
2. **Sliding Window Check**: After sorting, we check all possible contiguous windows of size (N - K). The difference between the last and first elements in each window gives the (max - min) for that window.
3. **Finding Minimum Difference**: We iterate through all possible windows and keep track of the smallest difference encountered, which is our answer.

This approach ensures that we efficiently find the minimal possible difference by leveraging the sorted order to simplify the calculation of max and min in each window. The complexity is O(N log N) due to the sorting step, which is acceptable given the constraints.
