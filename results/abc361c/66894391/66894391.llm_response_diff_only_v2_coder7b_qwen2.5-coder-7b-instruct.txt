```cpp
#include <bits/stdc++.h>
using namespace std;

int nums[200000];

int main(void){
    int n, k;
    cin >> n >> k;

    for (int i = 0; i < n; i++){
        cin >> nums[i];
    }

    sort(nums, nums + n);

    int l = 0;
    int r = n - 1;

    while (k--) {
        int median = nums[(r+l+1) / 2];
        int dl = abs(median - nums[l]);
        int dr = abs(median - nums[r]);

        if (dl >= dr){ // Changed '>' to '>=' to handle equal distances correctly
            l++;
        } else {
            r--;
        }
    }

    cout << nums[r] - nums[l] << endl;

    return 0;
}
```

Explanation:
- The issue with the original code was that it used `>` instead of `>=` when comparing distances (`dl` and `dr`). This could lead to incorrect decisions when `dl` equals `dr`, causing unnecessary removals.
- By changing the condition to `>=`, we ensure that both left and right pointers move appropriately even when the distances are equal, thus maintaining the goal of minimizing the difference between the maximum and minimum values in the resulting sequence.
