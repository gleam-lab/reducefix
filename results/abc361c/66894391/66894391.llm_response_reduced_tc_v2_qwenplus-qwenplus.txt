To solve this problem, we are given a sequence of integers and asked to remove exactly `K` elements such that the **difference between the maximum and minimum** of the remaining elements is minimized.

---

### Key Observations:

- We must **retain `N - K` elements**, and they should be in the **same order** as in the original array (but not necessarily contiguous).
- The optimal strategy involves selecting a **subsequence** of `N - K` elements with the smallest possible difference between the max and min.

A brute-force approach would try all combinations of `N - K` elements, which is computationally infeasible for large `N`.

---

### Efficient Strategy:

1. **Sort the array**: Sorting helps us consider candidate windows of size `N - K`.
2. **Sliding window over sorted array**:
   - For each window of size `N - K`, compute `max - min` (which is simply `A[i + (N-K) - 1] - A[i]` in the sorted array).
   - Track the minimum such difference across all valid windows.

This is an `O(N log N)` solution due to sorting, followed by a linear scan.

    }

    sort(A.begin(), A.end());

    long long min_diff = numeric_limits<long long>::max();
    int window_size = n - k;

    for (int i = 0; i + window_size <= n; ++i) {
        min_diff = min(min_diff, A[i + window_size - 1] - A[i]);
    }

    cout << min_diff << endl;

    return 0;
}
