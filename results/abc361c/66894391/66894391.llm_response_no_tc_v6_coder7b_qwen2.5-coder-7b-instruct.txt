```cpp
#include <bits/stdc++.h>
using namespace std;

int nums[200000];

int main(void){
    int n, k;
    cin >> n >> k;

    for (int i = 0; i < n; i++){
        cin >> nums[i];
    }

    sort(nums, nums + n);

    int l = 0;
    int r = n - 1;

    while (k--) {
        int dl = nums[l + 1] - nums[l];
        int dr = nums[r] - nums[r - 1];
        if (dl <= dr){
            l++;
        }
        else {
            r--;
        }
    }

    cout << nums[r] - nums[l] << endl;

    return 0;
}
```

In the original code, the strategy to find the optimal elements to remove was flawed. Instead of choosing elements based on their absolute difference from a median, the correct approach involves comparing the differences between adjacent elements and deciding whether to keep the larger gap or the smaller one, depending on which minimizes the range when elements are removed. This ensures that we are always removing the least impactful elements to the potential range of the final sequence.
