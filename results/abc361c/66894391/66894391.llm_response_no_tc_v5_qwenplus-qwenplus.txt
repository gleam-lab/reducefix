```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    int n, k;
    cin >> n >> k;
    vector<int> A(n);
    for (int i = 0; i < n; ++i) {
        cin >> A[i];
    }

    // Sliding window to find minimum range after removing K elements
    int min_diff = INT_MAX;
    int window_size = n - k;

    // We'll consider all subarrays of size `window_size` in the original array
    // Each such subarray corresponds to removing exactly K elements
    for (int i = 0; i + window_size <= n; ++i) {
        int current_min = A[i];
        int current_max = A[i];
        for (int j = i; j < i + window_size; ++j) {
            current_min = min(current_min, A[j]);
            current_max = max(current_max, A[j]);
        }
        min_diff = min(min_diff, current_max - current_min);
    }

    cout << min_diff << endl;
    return 0;
}
```

---

### Explanation of Fix

Your original approach sorted the array and tried a greedy "shrink from both sides" method. This is incorrect because:

- The problem requires **preserving the order** of remaining elements.
- You cannot sort or reorder the array since the resulting sequence B must maintain the original order of elements in A after removing K elements.

### Correct Approach:
- Since we are to remove **exactly K elements**, there will be **N - K elements left**.
- Try all windows of size **N - K** in the original array.
- For each such window (without changing elements' order), compute `max - min`.
- Return the minimum of these values.

### Time Complexity:
- O(N * window_size) = O(N * (N - K)) which is acceptable since N ≤ 2 × 10^5 and K can be close to N, making window_size small.

This brute-force sliding window over the original array guarantees correctness while maintaining readability and performance within constraints.
