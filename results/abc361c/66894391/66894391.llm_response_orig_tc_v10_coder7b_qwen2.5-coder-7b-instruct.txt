```cpp
#include <bits/stdc++.h>
using namespace std;

int nums[200000];

int main(void){
    int n, k;
    cin >> n >> k;

    for (int i = 0; i < n; i++){
        cin >> nums[i];
    }

    sort(nums, nums + n);

    int l = 0;
    int r = n - 1;

    while (k--) {
        int dl = nums[l + 1] - nums[l];
        int dr = nums[r] - nums[r - 1];
        if (dl <= dr){
            l++;
        } else {
            r--;
        }
    }

    cout << nums[r] - nums[l] << endl;

    return 0;
}
```

1. **Choosing the Correct Elements to Remove**: 
   - The goal is to minimize the difference between the maximum and minimum values in the resulting sequence `B`.
   - By removing the smallest and largest elements, we can achieve this.
   - However, instead of always removing the middle element (which might not be optimal), we need to consider the differences between adjacent elements.
   - If the difference between the next smallest element (`nums[l + 1]`) and the current smallest element (`nums[l]`) is less than or equal to the difference between the current largest element (`nums[r]`) and the previous largest element (`nums[r - 1]`), it is more beneficial to remove the smaller element (`nums[l]`). Otherwise, remove the larger element (`nums[r]`).

2. **Efficiency**:
   - Sorting the array takes \(O(N \log N)\) time.
   - The loop runs \(K\) times, which is efficient as \(K\) is much smaller than \(N\).

This approach ensures that you are always making the optimal choice at each step, leading to the correct solution.
