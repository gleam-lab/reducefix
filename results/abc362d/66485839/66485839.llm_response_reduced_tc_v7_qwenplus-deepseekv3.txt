2. **Priority Queue**: Use a priority queue to process vertices in order of increasing current known distance. This ensures that once a vertex is processed, its distance is the shortest possible.
3. **Edge Relaxation**: For each edge from a vertex `u` to `v`, check if the path through `u` improves the known distance to `v`. The total weight of the path through `u` is `d[u] + B_j + A_v`, where `B_j` is the edge weight and `A_v` is the vertex weight at `v`.

        auto [x, w] = pq.top();

        if (vis[x]) continue;

                d[y] = d[x] + dw + a[y];
                pq.push({y, d[y]});

2. **Priority Queue Processing**: The priority queue is initialized with the starting vertex. For each vertex processed, we check all its neighbors. If a shorter path is found through the current vertex, we update the neighbor's distance and push it into the queue.
3. **Edge Relaxation**: For each edge from vertex `x` to `y`, the potential new distance to `y` is `d[x] + edge_weight + a[y]`. If this new distance is shorter than the currently known distance to `y`, we update `d[y]` and push `y` into the priority queue.
