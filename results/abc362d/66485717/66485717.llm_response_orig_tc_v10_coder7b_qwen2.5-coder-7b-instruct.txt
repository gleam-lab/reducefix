To solve this problem efficiently, we need to use Dijkstra's algorithm to find the shortest path from vertex 1 to every other vertex in the graph. Here’s how you can implement it:

1. **Graph Representation**: Use an adjacency list to represent the graph.
2. **Dijkstra's Algorithm**: Implement Dijkstra's algorithm to find the shortest paths from vertex 1 to all other vertices.

Here is the corrected and optimized C++ code:

```cpp
#include <iostream>
#include <vector>
#include <queue>
#include <climits>

using namespace std;

typedef pair<long long, int> pii;

struct Edge {
    int to;
    long long weight;
};

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(NULL);

    int n, m;
    cin >> n >> m;

    vector<vector<Edge>> adj(n + 1);
    vector<long long> dist(n + 1, LLONG_MAX);
    vector<bool> visited(n + 1, false);

    for (int i = 0; i < m; ++i) {
        int u, v, b;
        cin >> u >> v >> b;
        adj[u].push_back({v, b});
        adj[v].push_back({u, b});
    }

    priority_queue<pii, vector<pii>, greater<pii>> pq;
    pq.push({0, 1});
    dist[1] = 0;

    while (!pq.empty()) {
        int u = pq.top().second;
        pq.pop();

        if (visited[u]) continue;
        visited[u] = true;

        for (auto &edge : adj[u]) {
            int v = edge.to;
            long long weight = edge.weight;

            if (!visited[v] && dist[u] + weight < dist[v]) {
                dist[v] = dist[u] + weight;
                pq.push({dist[v], v});
            }
        }
    }

    for (int i = 2; i <= n; ++i) {
        cout << dist[i] << " ";
    }
    cout << endl;

    return 0;
}
```

### Explanation:
1. **Graph Representation**:
   - We use an adjacency list `adj` where `adj[u]` contains all edges going from node `u`.

2. **Distance Initialization**:
   - We initialize the distance array `dist` with `LLONG_MAX` to represent infinity, except for the starting node `1`, which is set to `0`.

3. **Priority Queue**:
   - We use a min-priority queue (`pq`) to always process the node with the smallest current distance first.

4. **Dijkstra's Algorithm**:
   - For each node `u`, we explore its neighbors. If visiting neighbor `v` through `u` results in a shorter path, we update the distance and push `v` into the priority queue.

5. **Output**:
   - Finally, we print the distances for all nodes from `2` to `n`.

This approach ensures that we correctly compute the shortest paths using Dijkstra's algorithm, which runs in \(O((N + M) \log N)\) time complexity, making it efficient enough for large graphs within the given constraints.
