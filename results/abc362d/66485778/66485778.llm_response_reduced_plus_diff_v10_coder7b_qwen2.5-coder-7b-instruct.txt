To solve the problem efficiently, we can use Dijkstra's algorithm to find the shortest path from vertex 1 to every other vertex in the graph. This approach ensures that we handle the constraints and complexities effectively.

Here's the corrected and optimized C++ code:

```cpp
#include <bits/stdc++.h>
using namespace std;

#define INF 1e18

struct Edge {
    int v, w;
};

vector<Edge> adj[200001];

void dijkstra(int n, int src, vector<long long>& dist) {
    priority_queue<pair<long long, int>, vector<pair<long long, int>>, greater<>> pq;
    dist[src] = 0;
    pq.push({0, src});
    
    while (!pq.empty()) {
        int u = pq.top().second;
        long long d = pq.top().first;
        pq.pop();
        
        if (d > dist[u]) continue;
        
        for (auto& e : adj[u]) {
            int v = e.v;
            long long w = e.w;
            
            if (dist[u] + w < dist[v]) {
                dist[v] = dist[u] + w;
                pq.push({dist[v], v});
            }
        }
    }
}

int main() {
    ios_base::sync_with_stdio(false); cin.tie(NULL);

    int n, m;
    cin >> n >> m;
    
    vector<long long> a(n + 1), dist(n + 1, INF);
    
    for (int i = 1; i <= n; ++i) {
        cin >> a[i];
    }
    
    for (int i = 0; i < m; ++i) {
        int u, v, b;
        cin >> u >> v >> b;
        adj[u].push_back({v, b});
        adj[v].push_back({u, b});
    }
    
    dijkstra(n, 1, dist);
    
    for (int i = 2; i <= n; ++i) {
        cout << dist[i] + a[i] << " ";
    }
    cout << "\n";
    
    return 0;
}
```

### Explanation:
1. **Graph Representation**: We represent the graph using an adjacency list where each node stores its neighbors and the corresponding edge weights.
2. **Dijkstra's Algorithm**:
   - Initialize a distance array `dist` with infinity (`INF`) for all nodes except the source node which is set to 0.
   - Use a min-priority queue to always expand the least distant unvisited node first.
   - For each node, update the distances to its neighbors if a shorter path is found.
3. **Output**: After computing the shortest paths, print the result for each vertex from 2 to N.

This solution ensures that we efficiently compute the shortest paths using Dijkstra's algorithm, which runs in \(O((N + M) \log N)\) time complexity, making it suitable for large graphs up to the specified constraints.
