To solve this problem, we need to compute the **minimum path weight** from vertex `1` (0-based index) to all other vertices in a **weighted undirected graph**. Each path's weight is defined as the **sum of vertex weights and edge weights** along that path.

---

### 🔍 Key Observations

- This is essentially a **shortest path problem**, where the "cost" of a path is not just the sum of edge weights but also includes **vertex weights**.
- Since all weights are non-negative, we can use **Dijkstra’s algorithm**, which is efficient for large graphs with non-negative weights.

---

### 🧠 Algorithm Strategy

We define the cost of reaching a node `v` as:

```
path_weight = A_1 + B_1 + A_2 + B_2 + ... + A_v
```

This means when traversing from node `u` to `v`, we add:
- Edge weight `B_j` between `u` and `v`
- Vertex weight `A_v` of the destination node

So, during traversal:
- When moving from `u` to `v` over an edge with weight `B`, the total added cost is `B + A[v]`.

However, note that the **starting node 1** should have its vertex weight included only once at the beginning.

Thus, we initialize Dijkstra's algorithm with distance to node `0` as `A[0]`.

---

### ✅ Implementation Details

- Use an adjacency list to store the graph.
- Use a priority queue (`priority_queue`) for Dijkstra's algorithm.
- Maintain a distance array initialized to infinity.
- Mark visited nodes to avoid reprocessing.
- For each neighbor, relax the edge if a shorter path is found.

---

### 🚀 Final Code

```cpp
#include <bits/stdc++.h>
using namespace std;

#define vec vector
#define ll long long
#define INF 5LL << 60

template <typename T>
void chmin(T& x, const T& y) { x = min(x, y); }

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    ll n, m;
    cin >> n >> m;
    vec<ll> A(n);
    for (ll i = 0; i < n; ++i) cin >> A[i];

    // Build graph: adjacency list of (neighbor, edge weight)
    vec<vec<pair<ll, ll>>> graph(n);
    for (ll i = 0; i < m; ++i) {
        ll u, v, b;
        cin >> u >> v >> b;
        u--; v--;
        graph[u].push_back({v, b});
        graph[v].push_back({u, b});
    }

    // Dijkstra setup
    vec<ll> dist(n, INF);
    dist[0] = A[0]; // Start from node 1 (0-based), include its own weight

    using P = pair<ll, ll>; // (distance, node)
    priority_queue<P, vec<P>, greater<P>> pq;
    pq.push({dist[0], 0});

    while (!pq.empty()) {
        auto [d, u] = pq.top(); pq.pop();
        if (d > dist[u]) continue;

        for (auto [v, b] : graph[u]) {
            ll new_cost = d + b + A[v]; // Add edge weight and destination vertex weight
            if (new_cost < dist[v]) {
                dist[v] = new_cost;
                pq.push({dist[v], v});
            }
        }
    }

    // Output results for nodes 2 to N
    for (ll i = 1; i < n; ++i) {
        cout << dist[i] << " ";
    }
    cout << '\n';

    return 0;
}
```

- The algorithm correctly models the path weight as specified.
- It avoids unnecessary overhead by not storing full paths.
- Efficient enough for constraints up to `2 * 10^5` nodes/edges due to O(M log N) complexity.
