To solve this problem efficiently, we need to compute the shortest paths from vertex 1 to all other vertices in the graph. This can be achieved using Dijkstra's algorithm, which is well-suited for graphs with non-negative edge weights. We will use a priority queue to implement this algorithm.

Here is the corrected and optimized C++ code:

```cpp
#include <bits/stdc++.h>
using namespace std;

typedef pair<int, int> pii;
const int INF = 1e9;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n, m;
    cin >> n >> m;
    vector<int> a(n);
    for(int i = 0; i < n; ++i) cin >> a[i];

    vector<vector<pii>> adj(n);
    for(int i = 0; i < m; ++i) {
        int u, v, b;
        cin >> u >> v >> b;
        --u, --v;
        adj[u].emplace_back(v, b);
        adj[v].emplace_back(u, b);
    }

    vector<int> dist(n, INF);
    dist[0] = a[0];
    priority_queue<pii, vector<pii>, greater<pii>> pq;
    pq.emplace(a[0], 0);

    while(!pq.empty()) {
        auto [cost, u] = pq.top(); pq.pop();
        if(cost > dist[u]) continue;

        for(auto [v, weight] : adj[u]) {
            int new_cost = cost + a[v] + weight;
            if(new_cost < dist[v]) {
                dist[v] = new_cost;
                pq.emplace(dist[v], v);
            }
        }
    }

    for(int i = 1; i < n; ++i) {
        cout << dist[i] << " ";
    }
    cout << endl;

    return 0;
}
```

### Explanation:
1. **Graph Representation**: We represent the graph using an adjacency list where each node stores pairs of its neighbors and the corresponding edge weights.
2. **Distance Initialization**: Initialize the distance from vertex 0 to itself as `a[0]` and set all other distances to infinity (`INF`).
3. **Priority Queue**: Use a min-priority queue to always expand the least costly path first.
4. **Dijkstra's Algorithm**:
   - While there are nodes to process, extract the node with the smallest distance.
   - For each neighbor of the current node, calculate the new distance.
   - If the new distance is smaller than the previously known distance, update it and push the neighbor into the priority queue.
5. **Output**: Print the distances from vertex 0 to all other vertices.

This implementation ensures that we correctly compute the minimum path costs from vertex 1 to all other vertices efficiently. The time complexity is \(O(E \log V)\), which is efficient for the given constraints.
