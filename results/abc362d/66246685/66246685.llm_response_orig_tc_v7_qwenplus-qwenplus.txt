To solve this problem, we must compute the **minimum path weight** from node 1 to every other node in a graph. The path weight is defined as the **sum of vertex weights and edge weights** along the path.

The key observation is that the path cost includes both vertices and edges, which slightly changes how we model the problem compared to standard shortest path algorithms like Dijkstra's.

### Key Points:
- Each vertex has a fixed weight `A[i]`.
- Each edge has a weight `B[j]`.
- A path's total cost = sum of all **vertex weights** on the path + sum of all **edge weights** on the path.
- Since each node appears only once on a simple path, the vertex weight of each node is added exactly once.
- Therefore, the cost from node u to v is similar to a weighted graph where the **cost to enter a node includes its vertex weight**.

We can treat this as a **modified Dijkstra’s algorithm**, where:
- We store the cumulative cost to reach each node (including its own vertex weight).
- When moving from node `u` to node `v`, we add:
  - The edge weight `B_j`
  - The vertex weight of `v` (since it's newly visited)

However, since the input size (`N`, `M`) is large (~2e5), we need an efficient implementation using a priority queue (min-heap) for optimal performance.

Here is the corrected and optimized C++ code:

```cpp
#include <bits/stdc++.h>
using namespace std;
using ll = long long;

const ll INF = 1e18;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    ll N, M;
    cin >> N >> M;

    vector<ll> A(N);
    for (ll i = 0; i < N; ++i) {
        cin >> A[i];
    }

    vector<vector<pair<ll, ll>>> G(N);
    for (ll i = 0; i < M; ++i) {
        ll u, v, b;
        cin >> u >> v >> b;
        u--; v--;
        G[u].push_back({v, b});
        G[v].push_back({u, b});
    }

    vector<ll> dist(N, INF);
    priority_queue<pair<ll, ll>, vector<pair<ll, ll>>, greater<>> pq;

    // Start from node 0 with initial cost A[0]
    dist[0] = A[0];
    pq.push({dist[0], 0});

    while (!pq.empty()) {
        auto [d, u] = pq.top();
        pq.pop();

        if (d > dist[u]) continue;

        for (auto [v, b] : G[u]) {
            ll new_cost = d + b + A[v]; // current cost + edge weight + next node weight
            if (new_cost < dist[v]) {
                dist[v] = new_cost;
                pq.push({new_cost, v});
            }
        }
    }

    for (ll i = 1; i < N; ++i) {
        cout << dist[i] << " ";
    }
    cout << '\n';
}
```

### Explanation:
- We initialize the starting node's cost as `A[0]` because it's included in any path starting from it.
- For each move from `u` to `v`, we add:
  - The edge weight `b`
  - The vertex weight `A[v]`
- This ensures the correct modeling of the path cost including both nodes and edges.
- We use a **priority queue** to always expand the minimum-cost node next, ensuring optimal performance.
- Time complexity: `O(M log N)` — suitable for the given constraints.
