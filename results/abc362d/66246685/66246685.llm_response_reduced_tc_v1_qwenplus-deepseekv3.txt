To fix this code, we need to properly handle the path weight calculation and ensure that we explore all possible paths efficiently. The original code uses a BFS approach, which is not suitable for graphs with weighted edges where the shortest path might not be the first one found. Instead, we should use Dijkstra's algorithm, which is more appropriate for finding the shortest paths in weighted graphs.

### Approaches
1. **Graph Representation**: Use an adjacency list to represent the graph, where each node points to its neighbors along with the edge weights.
2. **Priority Queue**: Utilize a priority queue (min-heap) to always expand the node with the current minimum path weight.
3. **Distance Tracking**: Maintain a distance array to keep track of the minimum path weights from the source node (vertex 1) to all other nodes.
4. **Relaxation Process**: For each node, relax all its adjacent edges, updating the distance array if a shorter path is found.

### Solution Code
```cpp
#include <bits/stdc++.h>
using namespace std;
using ll = long long;
using P = pair<ll, ll>;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    ll N, M;
    cin >> N >> M;
    
    vector<ll> A(N);
    for (int i = 0; i < N; i++) {
        cin >> A[i];
    }
    
    vector<vector<pair<ll, ll>>> G(N);
    for (int i = 0; i < M; i++) {
        ll u, v, b;
        cin >> u >> v >> b;
        u--; v--;
        G[u].emplace_back(v, b);
        G[v].emplace_back(u, b);
    }
    
    vector<ll> dist(N, LLONG_MAX);
    dist[0] = A[0];
    priority_queue<P, vector<P>, greater<P>> pq;
    pq.emplace(dist[0], 0);
    
    while (!pq.empty()) {
        auto [current_dist, u] = pq.top();
        pq.pop();
        
        if (current_dist > dist[u]) continue;
        
        for (auto [v, b] : G[u]) {
            ll new_dist = current_dist + b + A[v];
            if (new_dist < dist[v]) {
                dist[v] = new_dist;
                pq.emplace(new_dist, v);
            }
        }
    }
    
    for (int i = 1; i < N; i++) {
        cout << dist[i] << " ";
    }
    cout << endl;
    
    return 0;
}
```

### Explanation
1. **Graph Representation**: The graph is represented using an adjacency list where each node `u` has a list of pairs `(v, b)`, indicating a bidirectional edge to node `v` with weight `b`.
2. **Priority Queue Initialization**: The priority queue is initialized with the source node (vertex 1) having an initial distance equal to its vertex weight `A[0]`.
3. **Dijkstra's Algorithm**: The algorithm processes nodes in order of increasing path weight. For each node `u`, it checks all adjacent nodes `v` and updates their distances if a shorter path through `u` is found.
4. **Distance Update**: The distance to node `v` is updated to `current_dist + b + A[v]`, where `current_dist` is the distance to `u`, `b` is the edge weight, and `A[v]` is the vertex weight of `v`.
5. **Output**: After processing all nodes, the distances from vertex 1 to all other vertices are printed, providing the minimum path weights for each vertex.

This approach ensures that the shortest paths are found efficiently, leveraging the priority queue to always expand the most promising nodes first, which is crucial for handling the large constraints specified in the problem.
