To fix this code, we need to address the algorithm used for finding the minimum weight path from vertex 1 to all other vertices in the graph. The initial approach used a BFS with a queue, which is not suitable for finding shortest paths in graphs with varying edge weights (since BFS is optimal only for unweighted graphs). Instead, we should use Dijkstra's algorithm, which is efficient for graphs with non-negative weights.

### Approaches
1. **Graph Representation**: Use an adjacency list to represent the graph, where each vertex maintains a list of its neighbors along with the edge weights.
2. **Priority Queue**: Utilize a priority queue (min-heap) to always expand the vertex with the current minimum path weight. This ensures that the first time we visit a vertex, we have found the shortest path to it.
3. **Distance Array**: Maintain an array to keep track of the minimum distance from the source vertex (vertex 1) to all other vertices. Initialize this array with a large value (infinity) except for the source vertex, which is initialized to its vertex weight.
4. **Relaxation**: For each vertex extracted from the priority queue, relax all its adjacent edges. If a shorter path is found to an adjacent vertex, update its distance and add it to the priority queue.

### Solution Code
```cpp
#include <bits/stdc++.h>
using namespace std;
using ll = long long;
using P = pair<ll, ll>;

const ll INF = 1e18;

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);

    ll N, M;
    cin >> N >> M;
    
    vector<ll> A(N);
    for (int i = 0; i < N; ++i) {
        cin >> A[i];
    }
    
    vector<vector<pair<ll, ll>>> G(N);
    for (int i = 0; i < M; ++i) {
        ll u, v, b;
        cin >> u >> v >> b;
        u--; v--;
        G[u].emplace_back(v, b);
        G[v].emplace_back(u, b);
    }
    
    vector<ll> dist(N, INF);
    dist[0] = A[0];
    priority_queue<P, vector<P>, greater<P>> pq;
    pq.emplace(dist[0], 0);
    
    while (!pq.empty()) {
        auto [current_dist, u] = pq.top();
        pq.pop();
        
        if (current_dist > dist[u]) continue;
        
        for (auto [v, b] : G[u]) {
            ll new_dist = current_dist + b + A[v];
            if (new_dist < dist[v]) {
                dist[v] = new_dist;
                pq.emplace(new_dist, v);
            }
        }
    }
    
    for (int i = 1; i < N; ++i) {
        cout << dist[i] << " ";
    }
    cout << endl;
    
    return 0;
}
```

### Explanation
1. **Graph Representation**: The graph is represented using an adjacency list `G`, where each entry `G[u]` contains pairs `(v, b)`, indicating an edge from vertex `u` to vertex `v` with weight `b`.
2. **Priority Queue**: A min-heap (`priority_queue`) is used to process vertices in order of increasing current distance from the source vertex. This ensures that the shortest path to each vertex is found first.
3. **Distance Array**: The `dist` array is initialized to `INF` (a very large value) to represent initially unknown distances. The distance to the source vertex (vertex 1) is set to its vertex weight `A[0]`.
4. **Relaxation Process**: For each vertex `u` extracted from the priority queue, the algorithm checks all adjacent vertices `v`. If the path from `u` to `v` via edge `b` offers a shorter path than the currently known distance to `v`, the distance is updated, and `v` is added to the priority queue for further processing.
5. **Output**: After processing all vertices, the `dist` array contains the shortest path weights from vertex 1 to all other vertices, which are printed in the required format.

This approach efficiently computes the shortest paths using Dijkstra's algorithm, which is optimal for graphs with non-negative edge weights, ensuring correctness and performance within the problem constraints.
