To fix this code, we need to address the issue of correctly calculating the minimum path weight from vertex 1 to all other vertices in the graph. The problem arises because the initial approach uses a BFS-like traversal with a queue, which is not suitable for graphs with weighted edges where the shortest path might not be found first. Instead, we should use Dijkstra's algorithm, which is more appropriate for finding shortest paths in graphs with non-negative edge weights.

### Approaches
1. **Graph Representation**: We represent the graph using an adjacency list where each vertex points to its neighbors along with the edge weights.
2. **Priority Queue for Dijkstra's Algorithm**: We use a priority queue (min-heap) to always expand the vertex with the current minimum path weight. This ensures that once we visit a vertex, we have found the shortest path to it.
3. **Distance Initialization**: Initialize the distance from vertex 1 to itself as the vertex's weight (`A[0]`). For all other vertices, initialize the distance to infinity.
4. **Relaxation**: For each vertex, we relax all its outgoing edges, updating the distances if a shorter path is found.

### Solution Code
```cpp
#include <bits/stdc++.h>
using namespace std;
using ll = long long;
using P = pair<ll, int>;
const ll INF = LLONG_MAX;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    ll N, M;
    cin >> N >> M;
    
    vector<ll> A(N);
    for (int i = 0; i < N; ++i) {
        cin >> A[i];
    }
    
    vector<vector<pair<int, ll>>> G(N);
    for (int i = 0; i < M; ++i) {
        int u, v;
        ll b;
        cin >> u >> v >> b;
        u--; v--;
        G[u].emplace_back(v, b);
        G[v].emplace_back(u, b);
    }
    
    vector<ll> dist(N, INF);
    dist[0] = A[0];
    priority_queue<P, vector<P>, greater<P>> pq;
    pq.emplace(dist[0], 0);
    
    while (!pq.empty()) {
        auto [current_dist, u] = pq.top();
        pq.pop();
        
        if (current_dist > dist[u]) continue;
        
        for (auto [v, b] : G[u]) {
            ll new_dist = current_dist + b + A[v];
            if (new_dist < dist[v]) {
                dist[v] = new_dist;
                pq.emplace(new_dist, v);
            }
        }
    }
    
    for (int i = 1; i < N; ++i) {
        cout << dist[i] << (i == N - 1 ? "\n" : " ");
    }
    
    return 0;
}
```

### Explanation
1. **Initialization**: The graph is represented using an adjacency list `G`, where each entry holds a neighbor vertex and the connecting edge's weight.
2. **Dijkstra's Algorithm**: We initialize the distance array `dist` with infinity, except for the starting vertex (vertex 1, index 0), which is set to its vertex weight `A[0]`.
3. **Priority Queue**: The priority queue (min-heap) is used to process vertices in order of increasing distance from the source. This ensures that the first time a vertex is processed, its distance is the shortest possible.
4. **Relaxation**: For each vertex `u` extracted from the queue, we check all its neighbors. If a shorter path to a neighbor `v` is found (via `u`), we update the distance and push the new distance into the queue.
5. **Output**: After processing all vertices, the distances from vertex 1 to all other vertices are printed.

This approach efficiently computes the shortest paths using Dijkstra's algorithm, which is optimal for graphs with non-negative weights, ensuring correctness and performance within the problem constraints.
