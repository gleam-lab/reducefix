4. **Correction**: The vertex weight of the current node should be added to the path only when moving to the next node, not for the current node's edges. Additionally, handle cases where there are no edges by ensuring the distance remains as infinity if no path exists.

typedef long long ll;
typedef pair<ll, int> pli;
const int N = 2e5 + 5;
const ll INF = LLONG_MAX;

int n, m;
ll a[N];
ll dis[N];
vector<pli> g[N];

void dijkstra() {
    for (int i = 1; i <= n; ++i) {
        dis[i] = INF;
    }
    dis[1] = 0;
    priority_queue<pli, vector<pli>, greater<pli>> pq;
    pq.push({0, 1});
    while (!pq.empty()) {
        ll current_dist = pq.top().first;
        int u = pq.top().second;
        pq.pop();
        if (current_dist > dis[u]) continue;
        for (pli edge : g[u]) {
            int v = edge.second;
            ll w = edge.first;
            if (dis[v] > dis[u] + w + a[u]) {
                dis[v] = dis[u] + w + a[u];
                pq.push({dis[v], v});
            }
        }
    }
}

### Explanation
1. **Initialization**: The distance array `dis` is initialized to infinity (`INF`), representing initially unreachable vertices. Vertex 1 is set to distance 0.

3. **Edge Relaxation**: For each edge from vertex `u` to `v` with weight `w`, the algorithm checks if the path through `u` to `v` offers a shorter distance. The vertex weight `a[u]` is added when moving from `u` to `v`.
