To fix this code, we need to address the issue in the Dijkstra's algorithm implementation. The main problem lies in how the vertex weights are incorporated into the path cost. The current implementation incorrectly adds the vertex weight when processing adjacent nodes, which leads to incorrect path costs. 

### Approaches
1. **Initialization**: The initial distance for vertex 1 should be the weight of vertex 1 itself since the path starts here.
2. **Priority Queue**: Use a min-heap (priority queue) to always expand the node with the current minimum distance.
3. **Relaxation**: For each edge, the new distance should be the current distance plus the edge weight and the vertex weight of the adjacent node (if not already included). This ensures that each vertex's weight is added only once when it is first visited.

### Solution Code
```cpp
#include<bits/stdc++.h>
using namespace std;
#define int long long 
#define pii pair<int,int>

vector<pii> G[200005];
int dis[200005];
int a[200005];
int vis[200005];

void dij(int u) {
    priority_queue<pii, vector<pii>, greater<pii>> q;
    q.push({a[u], u});
    dis[u] = a[u];
    while (!q.empty()) {
        int u = q.top().second;
        int d = q.top().first;
        q.pop();
        if (vis[u]) continue;
        vis[u] = 1;
        for (auto [v, w] : G[u]) {
            if (dis[v] > d + w + a[v]) {
                dis[v] = d + w + a[v];
                q.push({dis[v], v});
            }
        }
    }
}

signed main() {
    ios_base::sync_with_stdio(false);
    cin.tie(NULL);
    int n, m;
    cin >> n >> m;
    for (int i = 1; i <= n; i++) {
        cin >> a[i];
    }
    for (int i = 1; i <= m; i++) {
        int u, v, w;
        cin >> u >> v >> w;
        G[u].push_back({v, w});
        G[v].push_back({u, w});
    }
    memset(dis, 0x3f, sizeof dis);
    memset(vis, 0, sizeof vis);
    dij(1);
    for (int i = 2; i <= n; i++) {
        cout << dis[i] << ' ';
    }
    cout << "\n";
    return 0;
}
```

### Explanation
1. **Initialization**: The distance array `dis` is initialized to a large value, and the starting vertex's distance is set to its own weight `a[u]`.
2. **Priority Queue**: A min-heap is used to process nodes in order of increasing distance. This ensures that once a node is processed, its distance is the shortest possible.
3. **Relaxation**: For each neighbor of the current node, the algorithm checks if the path through the current node to the neighbor is shorter than the known distance. If so, the neighbor's distance is updated, and it is added to the priority queue for further processing.
4. **Output**: After processing all nodes, the distances from vertex 1 to all other vertices are printed.

This approach ensures that each vertex's weight is included exactly once in the path cost, and the use of Dijkstra's algorithm guarantees the shortest path in a graph with non-negative edge weights.
