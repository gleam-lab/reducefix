To address the problem of finding the minimum weight of paths from vertex 1 to all other vertices in a graph, we can use Dijkstra's algorithm. This algorithm efficiently finds the shortest paths from a single source node to all other nodes in a weighted graph. Given the constraints, using Dijkstra's algorithm ensures that we handle large graphs within the time limits.

Here's the corrected C++ code:

```cpp
#include<bits/stdc++.h>
using namespace std;

#define ll long long

vector<vector<pair<ll, ll>>> adjList;
vector<ll> dist;
const ll INF = 1e18;

void dijkstra(ll src) {
    priority_queue<pair<ll, ll>, vector<pair<ll, ll>>, greater<pair<ll, ll>>> pq;
    pq.push({src, 0});
    
    while (!pq.empty()) {
        auto [node, d] = pq.top();
        pq.pop();
        
        if (d > dist[node]) continue;
        
        for (auto &[neighbor, weight] : adjList[node]) {
            ll newDist = d + weight;
            
            if (newDist < dist[neighbor]) {
                dist[neighbor] = newDist;
                pq.push({neighbor, newDist});
            }
        }
    }
}

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);
    
    ll n, m;
    cin >> n >> m;
    
    adjList.resize(n + 1);
    dist.assign(n + 1, INF);
    
    for (ll i = 1; i <= n; ++i) {
        cin >> dist[i];
    }
    
    for (ll i = 0; i < m; ++i) {
        ll u, v, w;
        cin >> u >> v >> w;
        adjList[u].emplace_back(v, w);
        adjList[v].emplace_back(u, w); // Since the graph is undirected
    }
    
    dist[1] = 0; // Starting point
    
    dijkstra(1);
    
    for (ll i = 2; i <= n; ++i) {
        cout << min(dist[i], dist[i] + dist[1]) << " ";
    }
    
    return 0;
}
```

### Explanation:
1. **Graph Representation**: We use an adjacency list to represent the graph. Each node stores a list of pairs representing its neighbors and the corresponding edge weights.

2. **Distance Initialization**: We initialize the distance array `dist` with infinity (`INF`) to indicate that initially, all distances are unknown except for the starting node which is set to 0.

3. **Dijkstra's Algorithm**:
   - We use a priority queue to always process the node with the smallest known distance.
   - For each node, we update the distances to its neighbors if a shorter path is found.
   
4. **Handling the Source Node**: The distance to the starting node (vertex 1) is set to 0 before running the Dijkstra's algorithm.

5. **Output**: For each vertex from 2 to N, we print the minimum of its direct distance from the source and the distance through the source (which should be the same but calculated for completeness).

This approach ensures that we correctly find the minimum weight paths from vertex 1 to all other vertices efficiently.
