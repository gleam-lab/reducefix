### Explanation
The original code had two main issues:
1. It used a max-heap (priority queue with `greater`) instead of a min-heap, which is incorrect for finding the shortest paths using Dijkstra's algorithm.
2. It did not correctly handle the relaxation condition within the loop.

These issues have been addressed in the corrected version, where I've changed the priority queue to use `greater` for min-heap behavior and added a check to skip nodes that have already been processed or whose distance cannot be improved further.
