To fix this code, we need to ensure that the shortest path algorithm correctly accounts for the weights of both vertices and edges. The initial approach uses a queue-based BFS, which doesn't guarantee the shortest path in a weighted graph. Instead, we should use Dijkstra's algorithm with a priority queue to efficiently find the shortest paths.

### Approaches
1. **Graph Representation**: Represent the graph using an adjacency list where each node points to its neighbors along with the edge weights.
2. **Priority Queue**: Use a priority queue (min-heap) to always process the node with the current shortest distance. This ensures that once a node is processed, its shortest distance is finalized.
3. **Distance Initialization**: Initialize distances from the source node (1) to all other nodes as infinity, except for the source node itself, which is set to the weight of the source vertex.
4. **Relaxation**: For each neighbor of the current node, update the distance if a shorter path is found. The updated distance is the sum of the current node's distance, the neighbor's vertex weight, and the edge weight between them.

### Solution Code
```cpp
#include<bits/stdc++.h>
using namespace std;
#define int long long 
const int INF = 1e18;

vector<pair<int,int>> G[200005];
int dis[200005], a[200005], n, m;

void dij(int u) {
    priority_queue<pair<int,int>, vector<pair<int,int>>, greater<pair<int,int>>> pq;
    pq.push({a[u], u});
    dis[u] = a[u];
    
    while (!pq.empty()) {
        int d = pq.top().first;
        int u = pq.top().second;
        pq.pop();
        
        if (d > dis[u]) continue;
        
        for (auto [v, w] : G[u]) {
            if (dis[v] > dis[u] + w + a[v]) {
                dis[v] = dis[u] + w + a[v];
                pq.push({dis[v], v});
            }
        }
    }
}

signed main() {
    scanf("%lld%lld", &n, &m);
    for (int i = 1; i <= n; i++) {
        scanf("%lld", &a[i]);
    }
    for (int i = 1; i <= m; i++) {
        int u, v, w;
        scanf("%lld%lld%lld", &u, &v, &w);
        G[u].push_back({v, w});
        G[v].push_back({u, w});
    }
    
    fill(dis, dis + n + 1, INF);
    dij(1);
    
    for (int i = 2; i <= n; i++) {
        printf("%lld ", dis[i]);
    }
    printf("\n");
    
    return 0;
}
```

### Explanation
1. **Initialization**: The adjacency list `G` is initialized to store the graph. The distance array `dis` is initialized to infinity, except for the source node (1), which is set to its vertex weight.
2. **Priority Queue**: The priority queue (min-heap) is used to process nodes in order of increasing distance from the source. This ensures that the shortest path is found efficiently.
3. **Relaxation**: For each node processed, we check all its neighbors. If a shorter path to a neighbor is found (current node's distance plus edge weight plus neighbor's vertex weight), the neighbor's distance is updated, and it is added to the priority queue.
4. **Output**: After processing all nodes, the distances from the source node to all other nodes are printed, starting from node 2 to node N.

This approach ensures that the shortest paths are found efficiently using Dijkstra's algorithm, which is optimal for graphs with non-negative edge weights. The complexity is O(M log N), where M is the number of edges and N is the number of vertices, making it suitable for large graphs as specified in the problem constraints.
