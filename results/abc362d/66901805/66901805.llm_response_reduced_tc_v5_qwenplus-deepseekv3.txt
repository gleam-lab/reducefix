To fix this code, we need to address several issues. The primary problem is the incorrect implementation of Dijkstra's algorithm, which is essential for finding the shortest path in a graph with non-negative edge weights. The current implementation uses a queue, which does not guarantee that the node with the smallest current distance is processed next, leading to incorrect results. Additionally, the initialization of distances and handling of vertex weights needs to be corrected.

### Approaches
1. **Algorithm Selection**: Use Dijkstra's algorithm with a priority queue (min-heap) to ensure that the node with the current smallest distance is processed next.
2. **Vertex Weights**: The weight of a path includes the sum of the weights of the vertices and edges. The vertex weight of the starting node should be added once. For subsequent nodes, the vertex weight is added when the node is visited.
3. **Initialization**: Initialize the distance array with a large value (infinity) except for the starting node, which should be set to its vertex weight.
4. **Priority Queue**: Use a priority queue to efficiently retrieve and update the smallest distances.

### Solution Code
```cpp
#include<bits/stdc++.h>
using namespace std;
#define int long long 
const int INF = 1e18;

vector<pair<int,int>> G[200005];
int dis[200005];
int a[200005];
int n, m;

void dijkstra(int u) {
    priority_queue<pair<int, int>, vector<pair<int, int>>, greater<pair<int, int>>> pq;
    dis[u] = a[u];
    pq.push({dis[u], u});
    while (!pq.empty()) {
        int current_dist = pq.top().first;
        int u = pq.top().second;
        pq.pop();
        if (current_dist > dis[u]) continue;
        for (auto [v, w] : G[u]) {
            int new_dist = current_dist + a[v] + w;
            if (new_dist < dis[v]) {
                dis[v] = new_dist;
                pq.push({dis[v], v});
            }
        }
    }
}

signed main() {
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);
    cout.tie(nullptr);
    
    cin >> n >> m;
    for (int i = 1; i <= n; i++) {
        cin >> a[i];
    }
    for (int i = 1; i <= m; i++) {
        int u, v, w;
        cin >> u >> v >> w;
        G[u].push_back({v, w});
        G[v].push_back({u, w});
    }
    fill(dis, dis + n + 1, INF);
    dijkstra(1);
    for (int i = 2; i <= n; i++) {
        cout << (dis[i] == INF ? -1 : dis[i]) << ' ';
    }
    cout << '\n';
    return 0;
}
```

### Explanation
1. **Initialization**: The distance array `dis` is initialized to a large value (INF) to represent infinity. The starting node's distance is set to its vertex weight `a[1]`.
2. **Priority Queue**: The priority queue (min-heap) is used to process nodes in order of increasing current distance. This ensures that once a node is processed, its distance is the shortest possible.
3. **Relaxation**: For each neighbor of the current node, the algorithm checks if the path through the current node offers a shorter distance. If so, the neighbor's distance is updated, and it is added to the priority queue.
4. **Output**: After processing all nodes, the distances from node 1 to all other nodes are printed. If a node remains unreachable (distance remains INF), it is treated as -1 in the output, though the problem guarantees connectivity, so this is a safeguard.

This approach ensures correctness and efficiency, leveraging the properties of Dijkstra's algorithm to handle large input sizes within feasible time complexity.
