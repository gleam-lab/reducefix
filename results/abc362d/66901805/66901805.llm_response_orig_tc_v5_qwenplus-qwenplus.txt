To solve this problem correctly, we must compute the **minimum path weight** from node 1 to all other nodes in a graph. The **path weight** is defined as the sum of vertex weights and edge weights along that path.

The main issue with your code is that it incorrectly initializes the starting distance as `A[1]` (the weight of node 1), but then when exploring neighbors, it adds both `a[v]` and `w`, leading to **double-counting** the destination node's weight (`a[v]`) on every update.

---

### Correct Approach:

- Use **Dijkstra's algorithm**, since all weights are non-negative.
- Define the cost of moving from node `u` to node `v` via edge weight `w` as:  
  `cost = current_cost[u] + B_edge`
- Do **not** add the weight of node `v` twice — it should be added **only once** at the destination during initialization.
- Initialize `dis[1] = A[1]` (since we start at node 1).
- For each neighbor `(v, w)` of current node `u`, update:
  ```cpp
  if (dis[u] + w < dis[v]) {
      dis[v] = dis[u] + w;
      q.push({dis[v], v});
  }
  ```

This ensures that each node’s weight is only included once in the total path cost, and edges contribute their weights properly.

Here's the corrected and efficient C++ implementation:

```cpp
#include <bits/stdc++.h>
using namespace std;

#define int long long

const int INF = 1e18;

vector<vector<pair<int, int>>> G(200005); // G[u] = list of {v, edge_weight}
int dis[200005];
bool vis[200005];

signed main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int N, M;
    cin >> N >> M;

    vector<int> A(N + 1);
    for (int i = 1; i <= N; ++i) {
        cin >> A[i];
    }

    for (int i = 0; i < M; ++i) {
        int u, v, w;
        cin >> u >> v >> w;
        G[u].push_back({v, w});
        G[v].push_back({u, w});
    }

    // Initialize distances
    fill(dis + 1, dis + N + 1, INF);
    dis[1] = A[1]; // Include node 1's weight only once

    // Dijkstra using priority_queue (min-heap)
    priority_queue<pair<int, int>, vector<pair<int, int>>, greater<>> pq;
    pq.push({dis[1], 1});

    while (!pq.empty()) {
        auto [d, u] = pq.top();
        pq.pop();

        if (vis[u]) continue;
        vis[u] = true;

        for (auto [v, w] : G[u]) {
            if (dis[v] > d + w) {
                dis[v] = d + w;
                pq.push({dis[v], v});
            }
        }
    }

    // Output result for nodes 2 through N
    for (int i = 2; i <= N; ++i) {
        cout << dis[i] << ' ';
    }
    cout << '\n';

    return 0;
}
```

- **Time Complexity:** `O((N + M) * log N)` due to Dijkstra's algorithm using a priority queue.
- **Space Complexity:** `O(N + M)` for storing the graph and distances.
- **Correctness:** Ensures no double counting of node weights.
- **Robustness:** Handles large inputs efficiently due to optimized data structures.
