To solve this problem correctly, we must compute the **minimum path weight** from vertex 1 to every other vertex in a graph where:

- Each vertex has a weight (A[i])
- Each edge has a weight (B[j])

The total **path cost** is defined as the **sum of all vertex and edge weights along the path**.

---

### Key Observations

Each time you move from one node to another:
- You pay for the **edge weight**
- You also pay for the **vertex weight at the destination**, but **not** the source again if you already passed through it

Hence, when modeling the graph:
- When traversing an edge `(u -> v)` with weight `b`, the cost should be:  
  `b + A[v]` — because `v` is visited after `u`

But since Dijkstra’s algorithm starts from node 1, we have to include its vertex weight too. So, we initialize `dist[0] = A[0]`.

---

### Issues in Original Code

1. **Incorrect Initialization**: The code initializes `dist[0] = 0`, but should start with `dist[0] = A[0]` because node 1's own value contributes to any path starting from it.
2. **Wrong Edge Weight Addition**: When building the adjacency list, the code adds `val[u]` to the edge weight. That's incorrect because the vertex weight should only be added once per visit to a node, not on each edge into it.

---

### Fix Summary

- Initialize distance to node 1 (index 0) as `A[0]`
- Build adjacency list without adding `A[u]` to the edge weight
- Use Dijkstra's algorithm to find shortest paths, and for each node `v`, update cost as:  
  `dist[u] + B_uv + A[v]` — when moving from `u` to `v` via edge with weight `B_uv`

### Fixed C++ Code

```cpp
#include <iostream>
#include <vector>
#include <queue>
#include <algorithm>
#include <limits>

using namespace std;

typedef long long ll;
typedef pair<ll, int> pli;

const ll INF = numeric_limits<ll>::max();

void solve() {
    int n, m;
    cin >> n >> m;

    vector<ll> A(n);
    for (int i = 0; i < n; ++i)
        cin >> A[i];

    vector<vector<pli>> adj(n);
    for (int i = 0; i < m; ++i) {
        int u, v;
        ll b;
        cin >> u >> v >> b;
        u--, v--;
        adj[u].push_back({v, b});
        adj[v].push_back({u, b});
    }

    vector<ll> dist(n, INF);
    priority_queue<pli, vector<pli>, greater<pli>> pq;

    dist[0] = A[0]; // Start node includes its own vertex weight
    pq.push({dist[0], 0});

    while (!pq.empty()) {
        auto [d, u] = pq.top();
        pq.pop();

        if (d > dist[u]) continue;

        for (auto [v, b] : adj[u]) {
            ll new_cost = d + b + A[v]; // Add edge weight and target vertex weight
            if (new_cost < dist[v]) {
                dist[v] = new_cost;
                pq.push({dist[v], v});
            }
        }
    }

    for (int i = 1; i < n; ++i)
        cout << dist[i] << " ";
    cout << endl;
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);
    cout.tie(nullptr);

    solve();

    return 0;
}
```

### Time Complexity

- **Time Complexity**: `O((N + M) log N)` due to Dijkstra's with a priority queue
- **Space Complexity**: `O(N + M)` for storing graph and distances
