### Approaches
1. **Data Structures**: Use sets to keep track of the walls in each row and column. This allows efficient insertion, deletion, and searching operations.
2. **Bomb Processing**: For each bomb:
   - If there is a wall at the bomb's location, remove it and continue.
   - If there is no wall, find the nearest walls in the four directions (up, down, left, right) and remove them if they exist.
3. **Efficiency**: The use of sets ensures that each insertion, deletion, and search operation is O(log N), making the solution efficient for large inputs.

### Explanation
1. **Initialization**: We initialize sets for each row and column to keep track of walls. Each set starts with all column indices for rows and all row indices for columns.
2. **Bomb Processing**: For each query:
   - If there's a wall at the bomb's location, it is removed.
   - If there's no wall, we search for the nearest walls in the up, down, left, and right directions using `lower_bound` to efficiently find the closest walls.
3. **Erasure**: The `erase` function checks if a wall exists at a given location before removing it to avoid double-counting and updates the count of destroyed walls.
4. **Output**: The result is calculated by subtracting the count of destroyed walls from the initial total number of walls.

This approach efficiently handles the destruction of walls with logarithmic time complexity for each operation, ensuring the solution is optimal for large inputs.
