### Approaches
1. **Initialization**: We start by initializing sets for each row and column to keep track of the walls. This allows efficient searching for adjacent walls.
2. **Handling Queries**: For each query:

   - If the cell is empty, we search for the nearest walls in the four cardinal directions (up, down, left, right) and remove them if they exist.
3. **Efficiency**: Using sets ensures that searching and removal operations are efficient (O(log N) per operation).

        it = col[C].lower_bound(R);

        it = row[R].lower_bound(C);

### Explanation
1. **Initialization**: We use two vectors of sets, `row` and `col`, to track the walls in each row and column. This allows efficient insertion, deletion, and searching operations.
2. **Query Processing**: For each query:
   - If the cell `(R, C)` has a wall, it is removed directly.
   - If the cell is empty, the code searches for the nearest walls in the four directions (up, down, left, right) using the `lower_bound` function on the sets. This function efficiently locates the first wall in the specified direction.
   - The walls found are removed if they exist, and the count of destroyed walls is incremented.
3. **Output**: The result is the total initial walls minus the count of walls destroyed, printed after processing all queries.
