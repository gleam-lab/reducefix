### Approaches
1. **Initial Setup**: Initialize sets for each row and column to keep track of the walls present. This allows efficient lookup and updates.
2. **Handling Queries**: For each query:
   - If the cell has a wall, remove it.
   - If the cell does not have a wall, find the closest walls in each of the four directions (up, down, left, right) and remove them if they exist.
3. **Efficiency**: Using sets allows for O(log N) time complexity for insertions, deletions, and searches, which is crucial given the constraints.

```cpp
int main() {
    int H, W, Q;
    cin >> H >> W >> Q;
    vector<set<int>> row(H+1), col(W+1);
    rep(i, H+1) rep(j, W+1) {
        if (i >= 1 && j >= 1) {
            row[i].insert(j);
            col[j].insert(i);
        }
    }
    ll ans = (ll)H * W;
    ll cnt = 0;
    auto erase = [&](int r, int c) {
        if (row[r].count(c)) {
            cnt++;
            row[r].erase(c);
            col[c].erase(r);
        }
    };
    rep(itr, Q) {
        int R, C;
        cin >> R >> C;
        if (row[R].count(C)) {
            erase(R, C);
            continue;
        }
        // Up
        auto it = col[C].lower_bound(R);
        if (it != col[C].begin()) {
            --it;
            erase(*it, C);
        }
        // Down
        it = col[C].lower_bound(R);
        if (it != col[C].end()) {
            erase(*it, C);
        }
        // Left
        it = row[R].lower_bound(C);
        if (it != row[R].begin()) {
            --it;
            erase(R, *it);
        }
        // Right
        it = row[R].lower_bound(C);
        if (it != row[R].end()) {
            erase(R, *it);
        }
    }
    cout << ans - cnt << endl;
    return 0;
}
```

### Explanation
1. **Initialization**: The sets `row` and `col` are initialized to 1-based indexing, with each cell (i, j) marked as having a wall by inserting `j` into `row[i]` and `i` into `col[j]`.
2. **Query Processing**: For each query (R, C):
   - If the cell (R, C) has a wall, it is removed, and `cnt` is incremented.
   - If there is no wall, the code searches for the nearest walls in each of the four directions (up, down, left, right) using `lower_bound` to efficiently find the closest walls. These walls are then removed if they exist.
3. **Output**: The result is the total number of walls initially minus the number of walls destroyed (`cnt`), printed as the remaining walls.

This approach ensures that each query is processed in logarithmic time relative to the grid dimensions, making it efficient for large grids and queries.
