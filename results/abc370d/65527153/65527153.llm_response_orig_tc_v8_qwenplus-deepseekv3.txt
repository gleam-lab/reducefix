The problem involves a grid where each cell initially has a wall, and we process queries to destroy walls either at the given cell or the nearest walls in the four cardinal directions if the cell is empty. The incorrect solution uses sets to track remaining walls, but it fails to correctly handle the simultaneous destruction of the first walls in all four directions when the bomb is placed on an empty cell.

### Analysis
1. **Problem Understanding**: Each query specifies a cell (R_q, C_q). If there's a wall at this cell, it is destroyed. If not, the first walls in each direction (up, down, left, right) from this cell are destroyed simultaneously.
2. **Data Structures**: The incorrect solution uses separate sets for rows and columns, which doesn't efficiently track the nearest walls in all directions. We need a more efficient way to find the first walls in each direction.
3. **Algorithm**: For each query, we need to:
   - Check if the cell has a wall. If yes, destroy it.
   - If not, find the nearest walls in each direction and destroy them.
4. **Complexity**: The solution must handle up to 200,000 queries efficiently. Using sets for each row and column gives O(log W) or O(log H) per query, but the incorrect implementation may not correctly find the nearest walls.

### Solution Code
```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);
    
    int H, W, Q;
    cin >> H >> W >> Q;
    
    vector<set<int>> row_walls(H + 1);
    vector<set<int>> col_walls(W + 1);
    
    for (int i = 1; i <= H; ++i) {
        for (int j = 1; j <= W; ++j) {
            row_walls[i].insert(j);
            col_walls[j].insert(i);
        }
    }
    
    long long remaining = (long long)H * W;
    
    while (Q--) {
        int r, c;
        cin >> r >> c;
        
        if (row_walls[r].count(c)) {
            row_walls[r].erase(c);
            col_walls[c].erase(r);
            remaining--;
        } else {
            // Up
            auto it_up = col_walls[c].lower_bound(r);
            if (it_up != col_walls[c].begin()) {
                --it_up;
                int up_r = *it_up;
                row_walls[up_r].erase(c);
                col_walls[c].erase(up_r);
                remaining--;
            }
            
            // Down
            auto it_down = col_walls[c].lower_bound(r);
            if (it_down != col_walls[c].end()) {
                int down_r = *it_down;
                row_walls[down_r].erase(c);
                col_walls[c].erase(down_r);
                remaining--;
            }
            
            // Left
            auto it_left = row_walls[r].lower_bound(c);
            if (it_left != row_walls[r].begin()) {
                --it_left;
                int left_c = *it_left;
                row_walls[r].erase(left_c);
                col_walls[left_c].erase(r);
                remaining--;
            }
            
            // Right
            auto it_right = row_walls[r].lower_bound(c);
            if (it_right != row_walls[r].end()) {
                int right_c = *it_right;
                row_walls[r].erase(right_c);
                col_walls[right_c].erase(r);
                remaining--;
            }
        }
    }
    
    cout << remaining << "\n";
    
    return 0;
}
```

### Explanation
1. **Initialization**: We initialize sets for each row and column to keep track of walls. Each cell (i, j) is added to both the row set `row_walls[i]` and the column set `col_walls[j]`.
2. **Query Processing**: For each query:
   - If the cell (R_q, C_q) has a wall, it is removed from both sets, and the remaining count is decremented.
   - If the cell is empty, we find the nearest walls in all four directions:
     - **Up**: The first wall above the cell in the same column.
     - **Down**: The first wall below the cell in the same column.
     - **Left**: The first wall to the left of the cell in the same row.
     - **Right**: The first wall to the right of the cell in the same row.
   - Each found wall is removed from both sets, and the remaining count is decremented accordingly.
3. **Output**: After processing all queries, the remaining number of walls is printed.

This approach ensures that we efficiently track and destroy walls using set operations, maintaining logarithmic time complexity for each query, which is optimal for the given constraints.
