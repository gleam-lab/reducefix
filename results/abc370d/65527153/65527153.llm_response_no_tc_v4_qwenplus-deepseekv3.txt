To fix this code, we need to ensure that the logic correctly handles the destruction of walls according to the problem's specifications. The original code attempts to use sets to track walls, but there are issues in how it handles the directions (up, down, left, right) when the bomb is placed in an empty cell. Specifically, the code incorrectly decrements iterators without checking if the set is empty and doesn't properly handle all four directions.

### Approaches

1. **Data Structures**: Use separate sets for each row and column to efficiently track walls. Each row's set will track the columns with walls, and each column's set will track the rows with walls.
2. **Bomb Placement Handling**: 
   - If the bomb is placed on a wall, remove that wall.
   - If the bomb is placed on an empty cell, remove the nearest walls in all four directions:
     - **Up**: Find the first wall above the bomb.
     - **Down**: Find the first wall below the bomb.
     - **Left**: Find the first wall to the left of the bomb.
     - **Right**: Find the first wall to the right of the bomb.
3. **Efficiency**: This approach ensures that each query is handled in logarithmic time due to the use of ordered sets.

### Solution Code

```cpp
#include <bits/stdc++.h>
using namespace std;

void solve() {
    int H, W, Q;
    cin >> H >> W >> Q;
    vector<set<int>> row(H + 1), col(W + 1);
    
    // Initialize all cells as walls
    for (int i = 1; i <= H; ++i) {
        for (int j = 1; j <= W; ++j) {
            row[i].insert(j);
            col[j].insert(i);
        }
    }

    while (Q--) {
        int R, C;
        cin >> R >> C;
        
        if (row[R].count(C)) {
            // Wall exists at (R, C), destroy it
            row[R].erase(C);
            col[C].erase(R);
        } else {
            // No wall at (R, C), destroy nearest walls in four directions
            // Up
            auto it_up = col[C].lower_bound(R);
            if (it_up != col[C].begin()) {
                --it_up;
                row[*it_up].erase(C);
                col[C].erase(it_up);
            }
            // Down
            auto it_down = col[C].lower_bound(R);
            if (it_down != col[C].end()) {
                row[*it_down].erase(C);
                col[C].erase(it_down);
            }
            // Left
            auto it_left = row[R].lower_bound(C);
            if (it_left != row[R].begin()) {
                --it_left;
                col[*it_left].erase(R);
                row[R].erase(it_left);
            }
            // Right
            auto it_right = row[R].lower_bound(C);
            if (it_right != row[R].end()) {
                col[*it_right].erase(R);
                row[R].erase(it_right);
            }
        }
    }

    // Count remaining walls
    long long ans = 0;
    for (int i = 1; i <= H; ++i) {
        ans += row[i].size();
    }
    cout << ans << "\n";
}

int main() {
    ios::sync_with_stdio(0);
    cin.tie(0);
    solve();
    return 0;
}
```

### Explanation

1. **Initialization**: We initialize sets for each row and column to keep track of walls. Each cell (i, j) is added to both the row set of i and the column set of j.
2. **Query Processing**: For each query:
   - If the cell (R, C) has a wall, it is removed from the respective row and column sets.
   - If the cell is empty, we find and remove the nearest walls in the four cardinal directions using `lower_bound` to efficiently locate these walls in logarithmic time.
3. **Counting Remaining Walls**: After processing all queries, we iterate through each row's set to count the remaining walls, summing their sizes to get the total count.

This solution efficiently handles the constraints and ensures correct wall destruction as specified in the problem.
