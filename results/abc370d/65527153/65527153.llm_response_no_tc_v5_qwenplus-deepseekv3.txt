To fix this code, we need to ensure that the logic correctly handles the destruction of walls as per the problem statement. The original code attempted to use sets to track the remaining walls in each row and column, but it had some logical flaws in how it handled the destruction of walls in the four directions (up, down, left, right) when the bomb was placed on an empty cell.

### Approaches
1. **Track Walls**: Use separate sets for each row and column to keep track of the remaining walls. This allows efficient look-up and deletion operations.
2. **Bomb Placement**: 
   - If a bomb is placed on a cell with a wall, simply remove that wall.
   - If the cell is empty, find the nearest walls in the four cardinal directions (up, down, left, right) and remove them.
3. **Efficiency**: Given the constraints (H × W ≤ 4 × 10^5 and Q ≤ 2 × 10^5), using sets provides O(log n) time complexity for insertion, deletion, and search operations, which is efficient enough.

### Solution Code
```cpp
#include <bits/stdc++.h>
using namespace std;

void solve() {
    int H, W, Q;
    cin >> H >> W >> Q;
    
    vector<set<int>> row_walls(H + 1);
    vector<set<int>> col_walls(W + 1);
    
    for (int i = 1; i <= H; ++i) {
        for (int j = 1; j <= W; ++j) {
            row_walls[i].insert(j);
            col_walls[j].insert(i);
        }
    }
    
    auto erase_wall = [&](int r, int c) {
        row_walls[r].erase(c);
        col_walls[c].erase(r);
    };
    
    while (Q--) {
        int R, C;
        cin >> R >> C;
        
        if (row_walls[R].count(C)) {
            erase_wall(R, C);
            continue;
        }
        
        // Look left
        auto it_left = row_walls[R].lower_bound(C);
        if (it_left != row_walls[R].begin()) {
            --it_left;
            erase_wall(R, *it_left);
        }
        
        // Look right
        auto it_right = row_walls[R].lower_bound(C);
        if (it_right != row_walls[R].end()) {
            erase_wall(R, *it_right);
        }
        
        // Look up
        auto it_up = col_walls[C].lower_bound(R);
        if (it_up != col_walls[C].begin()) {
            --it_up;
            erase_wall(*it_up, C);
        }
        
        // Look down
        auto it_down = col_walls[C].lower_bound(R);
        if (it_down != col_walls[C].end()) {
            erase_wall(*it_down, C);
        }
    }
    
    long long remaining_walls = 0;
    for (int i = 1; i <= H; ++i) {
        remaining_walls += row_walls[i].size();
    }
    cout << remaining_walls << "\n";
}

int main() {
    ios::sync_with_stdio(0);
    cin.tie(0);
    
    solve();
    
    return 0;
}
```

### Explanation
1. **Initialization**: We initialize sets for each row and column to keep track of walls. Each cell (i, j) is added to both the row set for row `i` and the column set for column `j`.
2. **Query Processing**: For each query:
   - If the cell (R, C) has a wall, it is removed.
   - If the cell is empty, we find the nearest walls in the four directions (left, right, up, down) using `lower_bound` on the sets. The walls found are then removed.
3. **Result Calculation**: After processing all queries, the total remaining walls are counted by summing the sizes of the row sets (or column sets, as they should match).

This approach ensures that the operations are efficient, leveraging the logarithmic time complexity of set operations, making it suitable for large input sizes as specified in the problem constraints.
