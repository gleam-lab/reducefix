To fix this code, we need to correctly handle the bomb placements and the destruction of walls as described in the problem statement. The initial approach tried to use sets to track walls in each row and column, but it had some logical errors in how it handled the four directional searches for walls to destroy when the bomb cell was already empty.

### Approaches
1. **Data Structures**: Use separate sets for each row and column to keep track of the remaining walls. This allows efficient insertion, deletion, and searching operations.
2. **Query Processing**:
   - For each query, first check if the specified cell has a wall. If it does, remove it.
   - If the cell is already empty, search in the four directions (up, down, left, right) for the nearest wall and remove the first encountered wall in each direction.
3. **Efficient Search**: Utilize the ordered nature of sets to perform binary searches (using `lower_bound` and `upper_bound`) to find the nearest walls efficiently.

### Solution Code
```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int H, W, Q;
    cin >> H >> W >> Q;

    vector<set<int>> row_walls(H + 1);
    vector<set<int>> col_walls(W + 1);

    for (int i = 1; i <= H; ++i) {
        for (int j = 1; j <= W; ++j) {
            row_walls[i].insert(j);
            col_walls[j].insert(i);
        }
    }

    while (Q--) {
        int R, C;
        cin >> R >> C;

        if (row_walls[R].count(C)) {
            row_walls[R].erase(C);
            col_walls[C].erase(R);
            continue;
        }

        // Check up
        auto it_up = col_walls[C].lower_bound(R);
        if (it_up != col_walls[C].begin()) {
            --it_up;
            int r = *it_up;
            row_walls[r].erase(C);
            col_walls[C].erase(r);
        }

        // Check down
        auto it_down = col_walls[C].upper_bound(R);
        if (it_down != col_walls[C].end()) {
            int r = *it_down;
            row_walls[r].erase(C);
            col_walls[C].erase(r);
        }

        // Check left
        auto it_left = row_walls[R].lower_bound(C);
        if (it_left != row_walls[R].begin()) {
            --it_left;
            int c = *it_left;
            row_walls[R].erase(c);
            col_walls[c].erase(R);
        }

        // Check right
        auto it_right = row_walls[R].upper_bound(C);
        if (it_right != row_walls[R].end()) {
            int c = *it_right;
            row_walls[R].erase(c);
            col_walls[c].erase(R);
        }
    }

    long long ans = 0;
    for (int i = 1; i <= H; ++i) {
        ans += row_walls[i].size();
    }
    cout << ans << "\n";

    return 0;
}
```

### Explanation
- **Initial Setup**: We initialize sets for each row and column to keep track of the walls. Each cell (i, j) is added to the sets for row i and column j.
- **Query Processing**: For each query (R, C):
  - If the cell (R, C) has a wall, it is removed from the respective row and column sets.
  - If the cell is empty, we search in the four directions (up, down, left, right) for the nearest wall:
    - **Up**: Find the largest row number less than R in column C's set (using `lower_bound` and decrementing).
    - **Down**: Find the smallest row number greater than R in column C's set (using `upper_bound`).
    - **Left**: Find the largest column number less than C in row R's set (using `lower_bound` and decrementing).
    - **Right**: Find the smallest column number greater than C in row R's set (using `upper_bound`).
  - Each found wall in these directions is removed from the sets.
- **Result Calculation**: After processing all queries, the total remaining walls are counted by summing the sizes of all row sets (or column sets, as they are consistent).

This approach efficiently handles the queries with logarithmic time complexity for each operation, making it suitable for large input sizes.
