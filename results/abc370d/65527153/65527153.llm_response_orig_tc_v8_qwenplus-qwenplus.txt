To solve this problem efficiently, we need to simulate placing bombs on a grid and destroying walls in specific directions. The challenge lies in maintaining an efficient way to track and update the remaining walls after each bomb placement.

### Key Observations:
1. **Each cell initially has one wall**.
2. When a bomb is placed:
   - If there's still a wall at that cell, it gets destroyed.
   - Otherwise, we destroy the first walls in the four cardinal directions (up, down, left, right) from that cell.

### Efficient Data Structure:
We use `std::set` for each row and column to store the positions of cells with intact walls. This allows us to perform range searches and deletions in logarithmic time.

### Approach:
- Maintain two sets per row and column: one for rows (`row_walls`) and one for columns (`col_walls`).
- For each query:
  - Check if the current cell has a wall.
    - If yes, remove it.
    - If not, find and remove the nearest undestroyed wall in all four directions using `lower_bound` and `upper_bound`.

This ensures each operation takes O(log N), which is efficient for up to 200,000 queries.

Here is the corrected and optimized C++ code:

```cpp
#include <bits/stdc++.h>
using namespace std;
using ll = long long;

void solve() {
    int H, W, Q;
    cin >> H >> W >> Q;

    vector<set<int>> row_walls(H + 1);
    vector<set<int>> col_walls(W + 1);

    // Initialize all walls present
    for (int i = 1; i <= H; ++i) {
        for (int j = 1; j <= W; ++j) {
            row_walls[i].insert(j);
            col_walls[j].insert(i);
        }
    }

    auto destroy = [&](int r, int c) {
        row_walls[r].erase(c);
        col_walls[c].erase(r);
    };

    while (Q--) {
        int r, c;
        cin >> r >> c;

        if (row_walls[r].find(c) == row_walls[r].end()) {
            // Already destroyed, so try to destroy neighbors

            // Left
            {
                auto it = row_walls[r].lower_bound(c);
                if (it != row_walls[r].begin()) {
                    --it;
                    destroy(r, *it);
                }
            }

            // Right
            {
                auto it = row_walls[r].lower_bound(c);
                if (it != row_walls[r].end() && *it != c)
                    destroy(r, *it);
            }

            // Up
            {
                auto it = col_walls[c].lower_bound(r);
                if (it != col_walls[c].begin()) {
                    --it;
                    destroy(*it, c);
                }
            }

            // Down
            {
                auto it = col_walls[c].lower_bound(r);
                if (it != col_walls[c].end() && *it != r)
                    destroy(*it, c);
            }
        } else {
            // Current cell has a wall, destroy it
            destroy(r, c);
        }
    }

    ll ans = 0;
    for (int i = 1; i <= H; ++i)
        ans += row_walls[i].size();
    cout << ans << '\n';
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);
    cout.tie(nullptr);

    int T = 1;
    while (T--)
        solve();

    return 0;
}
```

### Explanation:
- We use `set<int>` for each row and column to maintain sorted indices where walls exist.
- `destroy()` removes a wall from both its row and column set.
- In each query:
  - If the cell already has no wall, we look in the 4 directions:
    - Left: `--lower_bound(...)`
    - Right: same `lower_bound(...)` but only if not equal to `c`.
    - Up: similar logic on column set.
    - Down: same as above.

This approach guarantees efficiency and correctness even under maximum constraints.
