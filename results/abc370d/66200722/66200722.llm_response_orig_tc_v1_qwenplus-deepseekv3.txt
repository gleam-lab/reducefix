To fix this code, we need to address the core issue of efficiently simulating the destruction of walls in a grid based on the given queries. The original code attempted to use a Union-Find data structure with some custom logic, but it didn't correctly handle the simultaneous destruction of walls in the four directions (up, down, left, right) when a bomb is placed in an empty cell.

### Approaches
1. **Data Structures**: Use separate structures to track the walls in each row and column. Specifically:
   - For each row, maintain a set of columns that still have walls.
   - For each column, maintain a set of rows that still have walls.
2. **Query Processing**:
   - If the bomb is placed on a cell with a wall, simply remove that wall.
   - If the bomb is placed on an empty cell:
     - For the row direction, find the first wall to the left and the first wall to the right.
     - For the column direction, find the first wall above and the first wall below.
     - Remove these walls if they exist.
3. **Efficiency**: Using sets (or similar data structures) allows efficient insertion, deletion, and searching operations, which are crucial for handling large input sizes within time constraints.

### Solution Code
```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int H, W, Q;
    cin >> H >> W >> Q;
    
    vector<set<int>> row_walls(H + 1);
    vector<set<int>> col_walls(W + 1);
    for (int i = 1; i <= H; ++i) {
        row_walls[i].insert(0);
        row_walls[i].insert(W + 1);
    }
    for (int j = 1; j <= W; ++j) {
        col_walls[j].insert(0);
        col_walls[j].insert(H + 1);
    }

    long long remaining = (long long)H * W;

    while (Q--) {
        int R, C;
        cin >> R >> C;
        
        if (row_walls[R].count(C)) {
            // Already destroyed, do nothing
            continue;
        }

        // Check if current cell has a wall
        if (row_walls[R].insert(C).second && col_walls[C].insert(R).second) {
            remaining--;
        } else {
            // No wall here, destroy adjacent walls
            // Left
            auto it_left = row_walls[R].lower_bound(C);
            --it_left;
            if (*it_left > 0) {
                int left_col = *it_left;
                if (row_walls[R].erase(left_col) && col_walls[left_col].erase(R)) {
                    remaining--;
                }
            }
            // Right
            auto it_right = row_walls[R].upper_bound(C);
            if (*it_right <= W) {
                int right_col = *it_right;
                if (row_walls[R].erase(right_col) && col_walls[right_col].erase(R)) {
                    remaining--;
                }
            }
            // Up
            auto it_up = col_walls[C].lower_bound(R);
            --it_up;
            if (*it_up > 0) {
                int up_row = *it_up;
                if (col_walls[C].erase(up_row) && row_walls[up_row].erase(C)) {
                    remaining--;
                }
            }
            // Down
            auto it_down = col_walls[C].upper_bound(R);
            if (*it_down <= H) {
                int down_row = *it_down;
                if (col_walls[C].erase(down_row) && row_walls[down_row].erase(C)) {
                    remaining--;
                }
            }
        }
    }

    cout << remaining << '\n';

    return 0;
}
```

### Explanation
1. **Initialization**: The code initializes sets for each row and column to keep track of walls. Each set includes boundary markers (0 and W+1 for rows, 0 and H+1 for columns) to simplify boundary checks.
2. **Query Processing**:
   - For each query, it checks if the cell (R, C) has a wall. If it does, the wall is removed, and the count of remaining walls is decremented.
   - If the cell is empty, the code finds the nearest walls in all four directions (left, right, up, down) using set operations and removes them if they exist, also updating the remaining wall count.
3. **Efficiency**: The use of sets ensures that each insertion, deletion, and search operation is O(log N), making the solution efficient enough for large input sizes (H, W up to 4×10^5 and Q up to 2×10^5).

This approach ensures that the solution correctly handles both types of bomb placements and efficiently tracks the remaining walls after each query.
