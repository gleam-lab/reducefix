### Analysis
The original code attempts to solve the problem by maintaining disjoint sets (Union-Find) to track walls and their destruction. However, it has several issues:
1. **Incorrect Union-Find Usage**: The code tries to use Union-Find to manage walls in rows and columns separately but doesn't correctly handle the dependencies between rows and columns.
2. **Complexity Issues**: The approach with Union-Find and maps leads to high time complexity, especially given the constraints (H × W ≤ 4 × 10^5, Q ≤ 2 × 10^5).
3. **Logical Errors**: The logic for finding the first walls in each direction when a bomb is placed on an empty cell is not correctly implemented.

### Correct Approach
To efficiently solve this problem, we should:
1. **Track Walls**: Use separate data structures to track walls in rows and columns (e.g., sets for each row and column).
2. **Query Handling**: For each query, check if the cell has a wall. If it does, remove it. If not, find the first walls in each direction using binary search in the respective row/column sets.
3. **Efficiency**: Using sets allows for O(log N) operations per query, which is efficient enough given the constraints.

### Fixed Code
```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int h, w, q;
    cin >> h >> w >> q;
    
    vector<set<int>> rows(h + 2); // 1-based to h
    vector<set<int>> cols(w + 2); // 1-based to w
    
    // Initialize all cells as having walls
    for (int i = 1; i <= h; ++i) {
        for (int j = 1; j <= w; ++j) {
            rows[i].insert(j);
            cols[j].insert(i);
        }
    }
    
    long long remaining = (long long)h * w;
    
    while (q--) {
        int r, c;
        cin >> r >> c;
        
        if (rows[r].count(c)) {
            // Remove the wall at (r, c)
            rows[r].erase(c);
            cols[c].erase(r);
            remaining--;
        } else {
            // Find first walls in each direction
            // Up
            auto it_up = cols[c].lower_bound(r);
            if (it_up != cols[c].begin()) {
                --it_up;
                int wall_r = *it_up;
                rows[wall_r].erase(c);
                cols[c].erase(wall_r);
                remaining--;
            }
            // Down
            it_up = cols[c].lower_bound(r);
            if (it_up != cols[c].end()) {
                int wall_r = *it_up;
                rows[wall_r].erase(c);
                cols[c].erase(wall_r);
                remaining--;
            }
            // Left
            auto it_left = rows[r].lower_bound(c);
            if (it_left != rows[r].begin()) {
                --it_left;
                int wall_c = *it_left;
                rows[r].erase(wall_c);
                cols[wall_c].erase(r);
                remaining--;
            }
            // Right
            it_left = rows[r].lower_bound(c);
            if (it_left != rows[r].end()) {
                int wall_c = *it_left;
                rows[r].erase(wall_c);
                cols[wall_c].erase(r);
                remaining--;
            }
        }
    }
    
    cout << remaining << '\n';
    return 0;
}
```

### Explanation
1. **Initialization**: We initialize sets for each row and column to keep track of walls.
2. **Query Processing**: For each query:
   - If the cell has a wall, we remove it from the respective row and column sets.
   - If the cell is empty, we find the first walls in all four directions using binary search (`lower_bound`) in the sets and remove them.
3. **Efficiency**: Using sets ensures that each insertion, deletion, and search operation is O(log N), making the solution efficient for large inputs. The total time complexity is O(Q log (H + W)), which is acceptable given the constraints.
