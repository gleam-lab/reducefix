To fix this code, we need to correctly handle the destruction of walls based on the given queries. The original code attempted to use a Union-Find data structure to manage connected components of walls, but it didn't correctly simulate the process of destroying walls in the grid.

### Approaches
1. **Grid Representation**: Use a 2D array or hash set to keep track of remaining walls.
2. **Query Processing**: For each query:
   - If there is a wall at the given cell, remove it.
   - If there is no wall, search in the four directions (up, down, left, right) for the first wall in each direction and remove it.
3. **Efficiency**: Given the constraints (H × W ≤ 4 × 10^5 and Q ≤ 2 × 10^5), we need an efficient way to check and remove walls. Using hash sets for rows and columns (or a 2D boolean array) allows O(1) or O(log n) operations for checking and removing walls.

### Solution Code
```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int H, W, Q;
    cin >> H >> W >> Q;
    
    vector<set<int>> row_walls(H + 2), col_walls(W + 2);
    for (int i = 1; i <= H; ++i) {
        row_walls[i].insert(0);
        row_walls[i].insert(W + 1);
    }
    for (int j = 1; j <= W; ++j) {
        col_walls[j].insert(0);
        col_walls[j].insert(H + 1);
    }

    long long remaining = (long long)H * W;

    while (Q--) {
        int R, C;
        cin >> R >> C;
        
        if (!row_walls[R].count(C)) {
            // Wall is present, destroy it
            row_walls[R].insert(C);
            col_walls[C].insert(R);
            --remaining;
        } else {
            // No wall, destroy the first in each direction
            // Up
            auto it_up = col_walls[C].upper_bound(R);
            int up = *it_up;
            if (up != H + 1 && !row_walls[up].count(C)) {
                row_walls[up].insert(C);
                col_walls[C].insert(up);
                --remaining;
            }
            // Down
            it_up = col_walls[C].lower_bound(R);
            --it_up;
            int down = *it_up;
            if (down != 0 && !row_walls[down].count(C)) {
                row_walls[down].insert(C);
                col_walls[C].insert(down);
                --remaining;
            }
            // Left
            auto it_left = row_walls[R].upper_bound(C);
            int left = *it_left;
            if (left != W + 1 && !col_walls[left].count(R)) {
                col_walls[left].insert(R);
                row_walls[R].insert(left);
                --remaining;
            }
            // Right
            it_left = row_walls[R].lower_bound(C);
            --it_left;
            int right = *it_left;
            if (right != 0 && !col_walls[right].count(R)) {
                col_walls[right].insert(R);
                row_walls[R].insert(right);
                --remaining;
            }
        }
    }

    cout << remaining << '\n';

    return 0;
}
```

### Explanation
1. **Initialization**: We initialize sets to keep track of walls in each row and column. Boundary walls (0 and W+1 for rows, 0 and H+1 for columns) are added to avoid out-of-bounds checks.
2. **Query Processing**: For each query:
   - If the cell has a wall, it is removed (marked as destroyed) by inserting it into the respective row and column sets.
   - If the cell does not have a wall, the first walls in the four directions (up, down, left, right) are found using set operations (upper_bound and lower_bound) and removed if they exist.
3. **Efficiency**: The use of sets allows O(log n) time complexity for insertion and search operations, making the solution efficient enough to handle the input constraints. The remaining walls count is decremented each time a wall is destroyed.

This approach ensures that each query is processed efficiently, and the total remaining walls are accurately counted after all queries are processed.
