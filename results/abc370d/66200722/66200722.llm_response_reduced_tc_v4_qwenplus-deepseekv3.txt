### Approaches
1. **Problem Analysis**: The grid has H rows and W columns, each initially containing a wall. We need to process Q queries where each query either destroys the wall at (R_q, C_q) or, if there's no wall there, destroys the first walls in the four directions (up, down, left, right) from (R_q, C_q).

2. **Key Insight**: For each query, we need to efficiently check if the cell (R_q, C_q) has a wall. If it does, we remove it. If not, we find the first walls in each of the four directions and remove them. This requires maintaining data structures that can quickly access and update the walls in each row and column.

3. **Data Structures**: We'll use separate maps for each row and column to keep track of the walls. For each row, we'll maintain a map of column indices where walls exist, and similarly for each column, we'll maintain a map of row indices where walls exist.

4. **Algorithm**: For each query:
   - If the cell (R_q, C_q) has a wall, remove it from the row and column maps.
   - If it doesn't have a wall, find the first walls in the four directions (up, down, left, right) and remove them from the respective maps.

### Solution Code
```cpp
#include <bits/stdc++.h>
using namespace std;

void solve() {
    int H, W, Q;
    cin >> H >> W >> Q;

    vector<set<int>> row_walls(H + 2);
    vector<set<int>> col_walls(W + 2);
    for (int i = 1; i <= H; ++i) {
        row_walls[i].insert(0);
        row_walls[i].insert(W + 1);
    }
    for (int j = 1; j <= W; ++j) {
        col_walls[j].insert(0);
        col_walls[j].insert(H + 1);
    }

    long long remaining = (long long)H * W;

    while (Q--) {
        int R, C;
        cin >> R >> C;

        if (row_walls[R].count(C)) {
            row_walls[R].erase(C);
            col_walls[C].erase(R);
            remaining++;
        } else {
            // Check up
            auto it = col_walls[C].lower_bound(R);
            --it;
            int up = *it;
            if (up >= 1 && up <= H && row_walls[up].count(C)) {
                row_walls[up].erase(C);
                col_walls[C].erase(up);
                remaining++;
            }

            // Check down
            it = col_walls[C].upper_bound(R);
            int down = *it;
            if (down >= 1 && down <= H && row_walls[down].count(C)) {
                row_walls[down].erase(C);
                col_walls[C].erase(down);
                remaining++;
            }

            // Check left
            it = row_walls[R].lower_bound(C);
            --it;
            int left = *it;
            if (left >= 1 && left <= W && col_walls[left].count(R)) {
                row_walls[R].erase(left);
                col_walls[left].erase(R);
                remaining++;
            }

            // Check right
            it = row_walls[R].upper_bound(C);
            int right = *it;
            if (right >= 1 && right <= W && col_walls[right].count(R)) {
                row_walls[R].erase(right);
                col_walls[right].erase(R);
                remaining++;
            }
        }
    }

    cout << remaining << endl;
}

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);
    solve();
    return 0;
}
```

### Explanation
1. **Initialization**: We start by reading the dimensions of the grid (H, W) and the number of queries Q. We initialize sets for each row and column to keep track of walls. Each row and column is initialized with boundary values (0 and W+1 for rows, 0 and H+1 for columns) to simplify boundary checks.

2. **Processing Queries**: For each query:
   - **Wall Exists**: If there's a wall at (R, C), we remove it from the respective row and column sets and increment the remaining wall count (since we initially set remaining to H*W and decrement when walls are destroyed).
   - **No Wall**: If there's no wall at (R, C), we check the first walls in the four directions (up, down, left, right) using binary search (lower_bound and upper_bound) on the sets. If a wall is found in any direction, it is removed, and the remaining count is incremented.

3. **Output**: After processing all queries, we print the number of remaining walls.

This approach efficiently handles each query in logarithmic time due to the use of sets (which are typically implemented as balanced binary search trees), making the overall complexity O(Q log(H + W)), which is optimal for the given constraints.
