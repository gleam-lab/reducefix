```cpp
#include <iostream>
#include <vector>
#include <unordered_set>
#include <queue>

using namespace std;

struct Query {
    int r, c;
};

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);

    int h, w, q;
    cin >> h >> w >> q;

    vector<unordered_set<int>> rows(h);
    vector<unordered_set<int>> cols(w);

    queue<pair<int, int>> todo;

    for (int i = 0; i < q; ++i) {
        int r, c;
        cin >> r >> c;
        --r, --c;

        if (!rows[r].empty()) {
            for (int cc : rows[r]) {
                cols[cc].erase(r);
            }
            rows[r].clear();
        }

        if (!cols[c].empty()) {
            for (int rr : cols[c]) {
                rows[rr].erase(c);
            }
            cols[c].clear();
        }

        rows[r].insert(c);
        cols[c].insert(r);

        todo.push({r, c});
    }

    int walls = h * w;

    while (!todo.empty()) {
        int r = todo.front().first;
        int c = todo.front().second;
        todo.pop();

        walls--;

        if (rows[r].size() == 1 && !cols[*rows[r].begin()].empty()) {
            walls--;
            cols[*rows[r].begin()].erase(r);
            rows[r].clear();
        }

        if (cols[c].size() == 1 && !rows[*cols[c].begin()].empty()) {
            walls--;
            rows[*cols[c].begin()].erase(c);
            cols[c].clear();
        }
    }

    cout << walls << endl;

    return 0;
}
```

### Explanation:

1. **Data Structures**:
   - `rows` and `cols` are vectors of unordered sets. Each set represents the columns or rows that have a wall in a specific row or column, respectively.
   - `todo` is a queue used to keep track of cells where walls were destroyed during previous steps.

2. **Processing Queries**:
   - For each query, we check if there are any walls in the current row (`r`) and column (`c`). If so, we remove them and update the corresponding sets.
   - We then add the current cell to the `todo` queue.

3. **Destroying Walls**:
   - After processing all queries, we iterate through the `todo` queue to ensure that any newly exposed walls are also removed.
   - Specifically, if a row or column has only one wall left and another wall is present in its adjacent row or column, we remove both walls.

4. **Counting Remaining Walls**:
   - Finally, we calculate the total number of remaining walls by subtracting the number of destroyed walls from the initial total (`h * w`).

This approach ensures that we efficiently handle the destruction of walls and accurately count the remaining ones, even under the constraints provided.
