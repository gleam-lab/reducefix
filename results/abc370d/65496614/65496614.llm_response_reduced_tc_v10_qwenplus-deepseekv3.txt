The problem involves simulating the destruction of walls in a grid based on a series of bomb placements. The key operations involve checking and removing walls either at the bomb's location or the nearest walls in the four cardinal directions if the bomb's location is already empty.

### Approaches
1. **Data Structures**: Use sets to efficiently track the positions of remaining walls in each row and column. This allows us to quickly find the nearest walls in the four directions if the bomb's location is empty.
2. **Handling Queries**: For each query:
   - **If there's a wall** at the bomb's location, remove it.
   - **If there's no wall**, find the nearest walls in the up, down, left, and right directions using binary search (via `lower_bound` in sets) and remove those walls.
3. **Efficiency**: The use of sets ensures that each query operation (insertion, deletion, and search) is handled in logarithmic time, making the solution efficient for large inputs.

### Solution Code
```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int h, w, q;
    cin >> h >> w >> q;

    vector<set<int>> row(h + 2); // rows 1..h, columns 1..w
    vector<set<int>> col(w + 2); // columns 1..w, rows 1..h

    for (int i = 1; i <= h; ++i) {
        for (int j = 1; j <= w; ++j) {
            row[i].insert(j);
            col[j].insert(i);
        }
    }

    int remaining = h * w;

    while (q--) {
        int r, c;
        cin >> r >> c;

        if (row[r].count(c)) {
            row[r].erase(c);
            col[c].erase(r);
            remaining--;
        } else {
            // Check up
            auto it = col[c].lower_bound(r);
            if (it != col[c].begin()) {
                --it;
                int i = *it;
                if (row[i].count(c)) {
                    row[i].erase(c);
                    col[c].erase(i);
                    remaining--;
                }
                ++it;
            }

            // Check down
            it = col[c].lower_bound(r);
            if (it != col[c].end()) {
                int i = *it;
                if (row[i].count(c)) {
                    row[i].erase(c);
                    col[c].erase(i);
                    remaining--;
                }
            }

            // Check left
            it = row[r].lower_bound(c);
            if (it != row[r].begin()) {
                --it;
                int j = *it;
                if (col[j].count(r)) {
                    col[j].erase(r);
                    row[r].erase(j);
                    remaining--;
                }
                ++it;
            }

            // Check right
            it = row[r].lower_bound(c);
            if (it != row[r].end()) {
                int j = *it;
                if (col[j].count(r)) {
                    col[j].erase(r);
                    row[r].erase(j);
                    remaining--;
                }
            }
        }
    }

    cout << remaining << '\n';
    return 0;
}
```

### Explanation
1. **Initialization**: The grid is initialized with walls at every position. Sets `row` and `col` are used to keep track of the columns and rows that still have walls, respectively.
2. **Query Processing**: For each query:
   - If there's a wall at `(r, c)`, it is removed from both sets, and the remaining wall count is decremented.
   - If there's no wall, the code checks the nearest walls in the four directions (up, down, left, right) using binary search in the sets. If such walls exist, they are removed, and the remaining count is adjusted accordingly.
3. **Output**: After processing all queries, the remaining number of walls is printed.

This approach efficiently handles the constraints by leveraging set operations to maintain and query the state of the grid dynamically.
