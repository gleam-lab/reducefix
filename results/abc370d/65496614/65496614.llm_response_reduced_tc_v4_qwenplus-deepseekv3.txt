To fix this code, we need to ensure that the bomb placement and wall destruction process is correctly implemented. The original code fails to correctly handle the destruction of walls in all four directions when the bomb is placed on a cell without a wall. The incorrect output for the simple test case where a single bomb is placed on the only cell shows that the wall destruction process is not being correctly executed.

### Approaches
1. **Initial Setup**: We start with a grid where all cells have walls. We need to track the presence of walls in each cell efficiently.
2. **Handling Queries**: For each query:
   - **If there is a wall at (R_q, C_q)**: Simply destroy that wall.
   - **If there is no wall at (R_q, C_q)**:
     - **Up Direction**: Find the first wall above (R_q, C_q) and destroy it.
     - **Down Direction**: Find the first wall below (R_q, C_q) and destroy it.
     - **Left Direction**: Find the first wall to the left of (R_q, C_q) and destroy it.
     - **Right Direction**: Find the first wall to the right of (R_q, C_q) and destroy it.
3. **Efficient Data Structures**: Use sets to keep track of the walls in each row and column to efficiently find the nearest walls in the four directions.

### Solution Code
```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int h, w, q;
    cin >> h >> w >> q;

    vector<set<int>> row_walls(h + 2);
    vector<set<int>> col_walls(w + 2);

    for (int i = 1; i <= h; ++i) {
        for (int j = 1; j <= w; ++j) {
            row_walls[i].insert(j);
            col_walls[j].insert(i);
        }
    }

    int total_walls = h * w;

    while (q--) {
        int r, c;
        cin >> r >> c;

        if (row_walls[r].count(c)) {
            row_walls[r].erase(c);
            col_walls[c].erase(r);
            total_walls--;
        } else {
            auto it = row_walls[r].lower_bound(c);
            if (it != row_walls[r].begin()) {
                --it;
                int j = *it;
                row_walls[r].erase(j);
                col_walls[j].erase(r);
                total_walls--;
                it = row_walls[r].lower_bound(c);
            }
            it = row_walls[r].upper_bound(c);
            if (it != row_walls[r].end()) {
                int j = *it;
                row_walls[r].erase(j);
                col_walls[j].erase(r);
                total_walls--;
            }

            it = col_walls[c].lower_bound(r);
            if (it != col_walls[c].begin()) {
                --it;
                int i = *it;
                col_walls[c].erase(i);
                row_walls[i].erase(c);
                total_walls--;
                it = col_walls[c].lower_bound(r);
            }
            it = col_walls[c].upper_bound(r);
            if (it != col_walls[c].end()) {
                int i = *it;
                col_walls[c].erase(i);
                row_walls[i].erase(c);
                total_walls--;
            }
        }
    }

    cout << total_walls << '\n';

    return 0;
}
```

### Explanation
1. **Initialization**: The grid is initialized with walls in every cell. Sets `row_walls` and `col_walls` keep track of the columns and rows that still have walls, respectively.
2. **Handling Queries**:
   - **Direct Hit**: If a bomb is placed on a cell with a wall, the wall is removed from the sets, and the total wall count is decremented.
   - **Indirect Hit**: If the bomb is placed on a cell without a wall, the nearest walls in all four directions (up, down, left, right) are removed from the sets, and the total wall count is decremented for each destroyed wall.
3. **Efficiency**: Using sets allows for efficient insertion, deletion, and searching operations (O(log n) per operation), making the solution scalable for large grids and many queries.

This approach ensures that all test cases, including edge cases, are handled correctly and efficiently.
