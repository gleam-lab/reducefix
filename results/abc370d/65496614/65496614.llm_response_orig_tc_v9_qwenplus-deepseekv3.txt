To fix this code, we need to handle the grid operations more efficiently, especially considering the constraints (H×W up to 400,000 and Q up to 200,000). The initial approach using sets for rows and columns might not be efficient enough for large grids. Instead, we can use binary search to find the nearest walls in the four directions (up, down, left, right) for each query.

### Approaches
1. **Tracking Walls**:
   - For each row, maintain a sorted list of columns that still have walls.
   - For each column, maintain a sorted list of rows that still have walls.
   - Use binary search (via `lower_bound` and `upper_bound`) to quickly find the nearest walls in each direction.

2. **Handling Queries**:
   - If the query cell (R, C) has a wall, remove it.
   - If not, find the first walls in the four directions (up, down, left, right) and remove them if they exist.

### Solution Code
```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int H, W, Q;
    cin >> H >> W >> Q;

    vector<set<int>> rows(H + 2);
    vector<set<int>> cols(W + 2);

    for (int i = 1; i <= H; ++i) {
        rows[i].insert(0);
        rows[i].insert(W + 1);
    }
    for (int j = 1; j <= W; ++j) {
        cols[j].insert(0);
        cols[j].insert(H + 1);
    }

    int remaining = H * W;

    while (Q--) {
        int R, C;
        cin >> R >> C;

        if (rows[R].count(C) == 0) {
            // The cell has no wall, find adjacent walls
            auto it = rows[R].lower_bound(C);
            int right = *it;
            --it;
            int left = *it;

            if (left + 1 <= C - 1 && cols[C - 1].count(R)) {
                cols[C - 1].erase(R);
                --remaining;
            }
            if (C + 1 <= right - 1 && cols[C + 1].count(R)) {
                cols[C + 1].erase(R);
                --remaining;
            }

            it = cols[C].lower_bound(R);
            int down = *it;
            --it;
            int up = *it;

            if (up + 1 <= R - 1 && rows[R - 1].count(C)) {
                rows[R - 1].erase(C);
                --remaining;
            }
            if (R + 1 <= down - 1 && rows[R + 1].count(C)) {
                rows[R + 1].erase(C);
                --remaining;
            }
        } else {
            // The cell has a wall, remove it
            rows[R].erase(C);
            cols[C].erase(R);
            --remaining;
        }
    }

    cout << remaining << '\n';
    return 0;
}
```

### Explanation
1. **Initialization**:
   - Initialize sets for each row and column. These sets will keep track of the walls present. Boundary values (0 and W+1 for rows, 0 and H+1 for columns) are added to handle edge cases.

2. **Processing Queries**:
   - For each query, check if the cell (R, C) has a wall. If it does, remove it from the respective row and column sets.
   - If there's no wall at (R, C), find the nearest walls in the four directions (left, right, up, down) using binary search. If any of these walls exist, remove them.

3. **Efficiency**:
   - Using sets allows for O(log N) insertion and deletion operations.
   - Binary search operations (via `lower_bound` and `upper_bound`) ensure efficient nearest neighbor queries.
   - The total complexity is O(Q log N), where N is the maximum of H and W, making it suitable for large input sizes.
