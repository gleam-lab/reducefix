To solve this problem efficiently, we need to keep track of the walls in the grid and handle the bomb placement queries effectively. The key idea is to use sets to maintain the positions of walls in each row and column. This allows us to quickly identify and remove walls in a sorted manner.

Here's the corrected C++ code:

```cpp
#include <iostream>
#include <vector>
#include <set>

using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);
    
    int h, w, q;
    cin >> h >> w >> q;
    
    vector<set<int>> rows(h + 2);
    vector<set<int>> cols(w + 2);
    
    for (int i = 1; i <= h; ++i) {
        for (int j = 1; j <= w; ++j) {
            rows[i].insert(j);
            cols[j].insert(i);
        }
    }
    
    for (int qi = 0; qi < q; ++qi) {
        int r, c;
        cin >> r >> c;
        
        if (!rows[r].count(c)) {
            // Place a bomb
            for (auto& s : rows) s.erase(c);
            for (auto& s : cols) s.erase(r);
        } else {
            // Destroy the wall at (r, c)
            rows[r].erase(c);
            cols[c].erase(r);
        }
    }
    
    int remaining_walls = 0;
    for (const auto& s : rows) remaining_walls += s.size();
    for (const auto& s : cols) remaining_walls += s.size();
    
    cout << remaining_walls / 2 << endl;
    
    return 0;
}
```

### Explanation:
1. **Initialization**: We initialize two vectors of sets, `rows` and `cols`, where each set represents the columns or rows containing walls.
2. **Adding Initial Walls**: For each cell `(i, j)` in the grid, we add `j` to `rows[i]` and `i` to `cols[j]`.
3. **Processing Queries**:
   - If the wall at `(r, c)` does not exist (`!rows[r].count(c)`), we simulate placing a bomb by removing the wall at `(r, c)` from all rows and columns.
   - If the wall at `(r, c)` exists, we simply remove it from its respective row and column.
4. **Counting Remaining Walls**: Finally, we count the total number of walls by summing the sizes of all sets in `rows` and `cols`. Since each wall is counted twice (once in its row and once in its column), we divide the total by 2.

This approach ensures that we efficiently handle the queries and maintain the correct state of the grid using sets, which provide logarithmic time complexity for insertions and deletions.
