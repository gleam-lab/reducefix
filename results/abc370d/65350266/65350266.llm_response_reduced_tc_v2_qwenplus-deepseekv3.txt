To fix this code, we need to address the incorrect handling of the bomb placement queries, especially when the bomb is placed on a cell that has no wall. The issue in the original code stems from improper checks for the existence of walls in the four directions (up, down, left, right) when the bomb is placed on an empty cell, leading to incorrect wall removals.

### Approaches
1. **Data Structures**: Use `set` to maintain walls in each row and column for efficient insertion, deletion, and searching (logarithmic time complexity).
2. **Handling Queries**:
   - **Wall Present**: If the cell (R, C) has a wall, remove it.
   - **No Wall Present**: For each of the four directions (up, down, left, right):
     - **Up**: Find the first wall above (R, C) and remove it.
     - **Down**: Find the first wall below (R, C) and remove it.
     - **Left**: Find the first wall to the left of (R, C) and remove it.
     - **Right**: Find the first wall to the right of (R, C) and remove it.
3. **Efficiency**: The use of `set` ensures that each operation (search, insertion, deletion) is O(log N), making the solution efficient enough for large inputs.

### Solution Code
```cpp
#include <bits/stdc++.h>
using namespace std;
#define rep(i,n) for(int i = 0; i < (n); i++)
using ll = long long;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int h, w, q;
    cin >> h >> w >> q;
    vector<set<int>> row_walls(h), col_walls(w);
    rep(i, h) rep(j, w) {
        row_walls[i].insert(j);
        col_walls[j].insert(i);
    }

    rep(qi, q) {
        int r, c;
        cin >> r >> c;
        r--; c--;

        if (row_walls[r].count(c)) {
            row_walls[r].erase(c);
            col_walls[c].erase(r);
        } else {
            // Up
            auto it_up = col_walls[c].lower_bound(r);
            if (it_up != col_walls[c].begin()) {
                --it_up;
                int i = *it_up;
                row_walls[i].erase(c);
                col_walls[c].erase(i);
            }
            // Down
            auto it_down = col_walls[c].lower_bound(r);
            if (it_down != col_walls[c].end()) {
                int i = *it_down;
                row_walls[i].erase(c);
                col_walls[c].erase(i);
            }
            // Left
            auto it_left = row_walls[r].lower_bound(c);
            if (it_left != row_walls[r].begin()) {
                --it_left;
                int j = *it_left;
                row_walls[r].erase(j);
                col_walls[j].erase(r);
            }
            // Right
            auto it_right = row_walls[r].lower_bound(c);
            if (it_right != row_walls[r].end()) {
                int j = *it_right;
                row_walls[r].erase(j);
                col_walls[j].erase(r);
            }
        }
    }

    int ans = 0;
    rep(i, h) ans += row_walls[i].size();
    cout << ans << '\n';

    return 0;
}
```

### Explanation
1. **Initialization**: We initialize sets for each row and column to keep track of walls. Each cell (i, j) is added to the respective row and column sets.
2. **Processing Queries**:
   - **Wall Present**: If the cell (R, C) has a wall, it is removed from both the row and column sets.
   - **No Wall Present**: For each direction (up, down, left, right), we use `lower_bound` to find the nearest wall. If found, it is removed from both the row and column sets.
3. **Counting Remaining Walls**: After processing all queries, the total number of remaining walls is counted by summing the sizes of all row sets (or column sets, as they should be consistent).

This approach ensures efficient handling of each query in logarithmic time, making the solution suitable for large input sizes as specified in the problem constraints.
