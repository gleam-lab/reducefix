To fix this code, we need to ensure that the logic correctly handles both cases of the query: when there is a wall at the specified position and when there isn't. The main issue in the original code is that it incorrectly handles the case where there is no wall at the specified position by not properly checking the closest walls in all four directions (up, down, left, right). The code also fails to correctly handle the scenario where there are no walls in one or more of these directions.

### Approaches

1. **Initial Setup**: The grid is represented with sets for each row and column to keep track of the walls. This allows efficient look-up and deletion operations.
2. **Handling Queries**:
   - **If there's a wall at (R, C)**: Remove the wall from both the row and column sets.
   - **If there's no wall at (R, C)**:
     - **Up Direction**: Find the closest wall above (R, C) in the same column.
     - **Down Direction**: Find the closest wall below (R, C) in the same column.
     - **Left Direction**: Find the closest wall to the left of (R, C) in the same row.
     - **Right Direction**: Find the closest wall to the right of (R, C) in the same row.
3. **Efficiency**: Using sets allows each insertion, deletion, and search operation to be performed in logarithmic time (O(log n)), which is efficient given the constraints.

### Solution Code

```cpp
#include <bits/stdc++.h>
using namespace std;
#define rep(i,n) for(int i = 0; i < (n); i++)
using ll = long long;

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);
    
    int h, w, q;
    cin >> h >> w >> q;
    vector<set<int>> row_walls(h), col_walls(w);
    
    rep(i, h) rep(j, w) {
        row_walls[i].insert(j);
        col_walls[j].insert(i);
    }
    
    int total_walls = h * w;
    
    rep(qi, q) {
        int r, c;
        cin >> r >> c;
        r--; c--;
        
        if (row_walls[r].count(c)) {
            row_walls[r].erase(c);
            col_walls[c].erase(r);
            total_walls--;
        } else {
            // Check up
            auto it_up = col_walls[c].lower_bound(r);
            if (it_up != col_walls[c].begin()) {
                --it_up;
                int wall_row = *it_up;
                row_walls[wall_row].erase(c);
                col_walls[c].erase(wall_row);
                total_walls--;
            }
            
            // Check down
            auto it_down = col_walls[c].lower_bound(r);
            if (it_down != col_walls[c].end()) {
                int wall_row = *it_down;
                row_walls[wall_row].erase(c);
                col_walls[c].erase(wall_row);
                total_walls--;
            }
            
            // Check left
            auto it_left = row_walls[r].lower_bound(c);
            if (it_left != row_walls[r].begin()) {
                --it_left;
                int wall_col = *it_left;
                row_walls[r].erase(wall_col);
                col_walls[wall_col].erase(r);
                total_walls--;
            }
            
            // Check right
            auto it_right = row_walls[r].lower_bound(c);
            if (it_right != row_walls[r].end()) {
                int wall_col = *it_right;
                row_walls[r].erase(wall_col);
                col_walls[wall_col].erase(r);
                total_walls--;
            }
        }
    }
    
    cout << total_walls << "\n";
    return 0;
}
```

### Explanation

1. **Initialization**: We start by reading the grid dimensions and the number of queries. We initialize sets for each row and column to keep track of walls. Each wall is added to both the corresponding row and column sets.
2. **Query Processing**:
   - **Wall Present**: If there's a wall at the queried position, it is removed from both sets, and the total wall count is decremented.
   - **Wall Absent**: The code checks the closest walls in all four directions (up, down, left, right) by using the `lower_bound` method to efficiently find the nearest walls. If found, these walls are removed from the sets, and the total count is adjusted.
3. **Output**: After processing all queries, the total remaining walls are printed.

This approach ensures that each query is processed efficiently, leveraging the ordered nature of sets to maintain logarithmic time complexity for each operation. The solution correctly handles all edge cases, including queries where no walls exist in one or more directions.
