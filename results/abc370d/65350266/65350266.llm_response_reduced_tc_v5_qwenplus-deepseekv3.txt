To fix this code, we need to address the incorrect handling of the query processing, particularly when the bomb is placed on a cell that has no wall. The original code attempts to find and remove the first walls in the four directions (up, down, left, right) but fails to correctly identify these walls in some cases, especially when the set of walls is empty in a particular direction.

### Approaches
1. **Initial Setup**: Initialize sets to keep track of walls in each row and column. Each row will have a set of columns with walls, and each column will have a set of rows with walls.
2. **Query Processing**:
   - **Wall Present**: If the bomb is placed on a cell with a wall, simply remove it from the respective row and column sets.
   - **Wall Absent**: If there is no wall at the bomb's location, search for the nearest walls in all four directions (up, down, left, right) and remove them if they exist. This involves:
     - **Up**: Find the largest row less than the current row in the column's set.
     - **Down**: Find the smallest row greater than the current row in the column's set.
     - **Left**: Find the largest column less than the current column in the row's set.
     - **Right**: Find the smallest column greater than the current column in the row's set.
3. **Result Calculation**: After processing all queries, sum the remaining walls from either the row sets or column sets.

### Solution Code
```cpp
#include <bits/stdc++.h>
using namespace std;
#define rep(i,n) for(int i = 0; i < (n); i++)
using ll = long long;

int main() {
    int h, w, q;
    cin >> h >> w >> q;
    vector<set<int>> row_walls(h), col_walls(w);
    rep(i, h) rep(j, w) {
        row_walls[i].insert(j);
        col_walls[j].insert(i);
    }
    rep(qi, q) {
        int r, c;
        cin >> r >> c;
        r--; c--;
        if (row_walls[r].count(c)) {
            row_walls[r].erase(c);
            col_walls[c].erase(r);
        } else {
            // Up
            auto it_up = col_walls[c].lower_bound(r);
            if (it_up != col_walls[c].begin()) {
                it_up--;
                int up_row = *it_up;
                row_walls[up_row].erase(c);
                col_walls[c].erase(up_row);
            }
            // Down
            auto it_down = col_walls[c].lower_bound(r);
            if (it_down != col_walls[c].end()) {
                int down_row = *it_down;
                row_walls[down_row].erase(c);
                col_walls[c].erase(down_row);
            }
            // Left
            auto it_left = row_walls[r].lower_bound(c);
            if (it_left != row_walls[r].begin()) {
                it_left--;
                int left_col = *it_left;
                row_walls[r].erase(left_col);
                col_walls[left_col].erase(r);
            }
            // Right
            auto it_right = row_walls[r].lower_bound(c);
            if (it_right != row_walls[r].end()) {
                int right_col = *it_right;
                row_walls[r].erase(right_col);
                col_walls[right_col].erase(r);
            }
        }
    }
    ll ans = 0;
    for (auto& row : row_walls) ans += row.size();
    cout << ans << "\n";
    return 0;
}
```

### Explanation
1. **Initialization**: We initialize `row_walls` and `col_walls` to keep track of the walls present in each row and column. Initially, all cells have walls.
2. **Query Processing**:
   - **Wall Present**: If the cell (R, C) has a wall, it is removed from both `row_walls[R]` and `col_walls[C]`.
   - **Wall Absent**: The code searches for the nearest walls in all four directions:
     - **Up**: The largest row number less than R in `col_walls[C]`.
     - **Down**: The smallest row number greater than R in `col_walls[C]`.
     - **Left**: The largest column number less than C in `row_walls[R]`.
     - **Right**: The smallest column number greater than C in `row_walls[R]`.
     If such walls exist, they are removed from the respective sets.
3. **Result Calculation**: The total remaining walls are counted by summing the sizes of all `row_walls` sets (or `col_walls` sets), which gives the correct answer after all queries are processed.

This approach efficiently handles each query in logarithmic time relative to the number of walls, ensuring optimal performance even for the upper constraint limits.
