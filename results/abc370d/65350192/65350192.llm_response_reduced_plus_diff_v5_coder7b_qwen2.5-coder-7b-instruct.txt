To solve this problem efficiently, we need to handle the destruction of walls based on the given rules. The key observation here is that we can use a combination of sets and maps to keep track of the positions of walls in both horizontal and vertical directions. This allows us to quickly identify and remove walls as required.

Here's the corrected C++ code:

```cpp
#include <bits/stdc++.h>
using namespace std;
#define rep(i,n) for(int i = 0; i < (n); i++)
using ll = long long;

int main(void) {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int h, w, q;
    cin >> h >> w >> q;
    vector<set<int>> tate(w), yoko(h);
    rep(i, h) rep(j, w) {
        tate[j].insert(i);
        yoko[i].insert(j);
    }

    rep(qi, q) {
        int r, c;
        cin >> r >> c;
        r--, c--;

        if (tate[c].count(r)) {
            tate[c].erase(r);
            yoko[r].erase(c);
        } else {
            // Find the nearest wall above
            auto it_up = tate[c].upper_bound(r);
            if (it_up != tate[c].begin()) {
                --it_up;
                tate[c].erase(it_up);
                yoko[*it_up].erase(c);
            }

            // Find the nearest wall below
            auto it_down = tate[c].lower_bound(r);
            if (it_down != tate[c].end()) {
                tate[c].erase(it_down);
                yoko[*it_down].erase(c);
            }

            // Find the nearest wall to the left
            auto it_left = yoko[r].upper_bound(c);
            if (it_left != yoko[r].begin()) {
                --it_left;
                yoko[r].erase(it_left);
                tate[*it_left].erase(r);
            }

            // Find the nearest wall to the right
            auto it_right = yoko[r].lower_bound(c);
            if (it_right != yoko[r].end()) {
                yoko[r].erase(it_right);
                tate[*it_right].erase(r);
            }
        }
    }

    int ans = 0;
    rep(i, w) {
        ans += tate[i].size();
    }
    cout << ans << "\n";

    return 0;
}
```

### Explanation
1. **Initialization**: We initialize two vectors of sets, `tate` and `yoko`, to store the positions of walls in each column and row respectively.

2. **Processing Queries**:
   - For each query, we check if there is a wall at the specified position `(r, c)`.
   - If there is a wall, we remove it from both the `tate` and `yoko` sets.
   - If there is no wall, we find the nearest walls in the four directions (above, below, left, right) using the `upper_bound` and `lower_bound` functions of the set. We then remove these nearest walls from their respective sets.

3. **Counting Remaining Walls**: Finally, we count the total number of walls remaining in the grid by summing the sizes of all sets in the `tate` vector.

This approach ensures that we efficiently handle the operations in \(O(\log n)\) time per query, making it suitable for large inputs within the given constraints.
