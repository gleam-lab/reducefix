To fix this code, we need to ensure that the logic correctly handles the destruction of walls in all four directions (up, down, left, right) when a bomb is placed at a cell that does not contain a wall. The original code had some issues in the logic for finding the first walls in each direction, particularly in handling the boundaries and ensuring that the correct walls are removed.

### Approaches
1. **Data Structures**: Use sets to keep track of the walls in each row and column. This allows efficient insertion, deletion, and searching (in logarithmic time) for the nearest walls.
2. **Processing Queries**: For each query, check if the cell has a wall:
   - If it does, remove the wall.
   - If it does not, find the nearest walls in each of the four directions (up, down, left, right) and remove them if they exist.
3. **Efficiency**: The use of sets ensures that each insertion, deletion, and search operation is O(log N), making the overall complexity O(Q log N), where N is the maximum of H and W.

### Solution Code
```cpp
#include <bits/stdc++.h>
using namespace std;
#define rep(i,n) for(int i = 0; i < (n); i++)
using ll = long long;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int h, w, q;
    cin >> h >> w >> q;

    vector<set<int>> row_walls(h), col_walls(w);
    rep(i, h) rep(j, w) {
        row_walls[i].insert(j);
        col_walls[j].insert(i);
    }

    int remaining = h * w;

    rep(qi, q) {
        int r, c;
        cin >> r >> c;
        r--; c--;

        if (row_walls[r].count(c)) {
            row_walls[r].erase(c);
            col_walls[c].erase(r);
            remaining--;
        } else {
            // Check up
            auto it_up = col_walls[c].lower_bound(r);
            if (it_up != col_walls[c].begin()) {
                --it_up;
                int i = *it_up;
                row_walls[i].erase(c);
                col_walls[c].erase(i);
                remaining--;
            }

            // Check down
            auto it_down = col_walls[c].lower_bound(r);
            if (it_down != col_walls[c].end()) {
                int i = *it_down;
                row_walls[i].erase(c);
                col_walls[c].erase(i);
                remaining--;
            }

            // Check left
            auto it_left = row_walls[r].lower_bound(c);
            if (it_left != row_walls[r].begin()) {
                --it_left;
                int j = *it_left;
                row_walls[r].erase(j);
                col_walls[j].erase(r);
                remaining--;
            }

            // Check right
            auto it_right = row_walls[r].lower_bound(c);
            if (it_right != row_walls[r].end()) {
                int j = *it_right;
                row_walls[r].erase(j);
                col_walls[j].erase(r);
                remaining--;
            }
        }
    }

    cout << remaining << "\n";
    return 0;
}
```

### Explanation
1. **Initial Setup**: We initialize sets for each row and column to keep track of the walls. Initially, every cell has a wall, so all sets are populated accordingly.
2. **Processing Queries**: For each query:
   - If the cell (r, c) has a wall, it is removed from both the row and column sets.
   - If the cell does not have a wall, we search for the nearest walls in the four directions using `lower_bound` to find the appropriate positions in the sets. If found, these walls are removed.
3. **Efficiency**: The use of sets ensures that each operation (insertion, deletion, search) is O(log N), making the solution efficient even for large inputs.
4. **Output**: The remaining walls are counted by the variable `remaining`, which is decremented each time a wall is removed, and printed at the end.
