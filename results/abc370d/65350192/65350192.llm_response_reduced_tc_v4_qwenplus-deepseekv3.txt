To fix this code, we need to ensure that the logic correctly handles both scenarios when a bomb is placed at a cell with a wall and when it is placed at a cell without a wall. The previous implementation had issues with the logic for finding the first walls in the four directions when there was no wall at the bomb's location. Specifically, the checks for the existence of walls before proceeding with the erasures were incorrect, leading to potential segmentation faults or incorrect erasures.

### Approaches
1. **Initialization**: Use sets to keep track of walls in each row and column. This allows efficient insertion, deletion, and search operations.
2. **Query Processing**:
   - **Wall Present**: If there's a wall at the bomb's location, remove it from both the row and column sets.
   - **No Wall Present**: For each of the four directions (up, down, left, right), find the nearest wall in that direction and remove it if it exists. This involves checking the sets for the nearest elements in the respective directions.
3. **Efficiency**: Using sets ensures that each insertion, deletion, and search operation is O(log n), making the solution efficient enough for large inputs.

### Solution Code
```cpp
#include <bits/stdc++.h>
using namespace std;
#define rep(i,n) for(int i = 0; i < (n); i++)
using ll = long long;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);
    
    int h, w, q;
    cin >> h >> w >> q;
    vector<set<int>> row_walls(h), col_walls(w);
    
    rep(i, h) rep(j, w) {
        row_walls[i].insert(j);
        col_walls[j].insert(i);
    }
    
    rep(qi, q) {
        int r, c;
        cin >> r >> c;
        r--; c--;
        
        if (row_walls[r].count(c)) {
            row_walls[r].erase(c);
            col_walls[c].erase(r);
        } else {
            // Up
            auto it_up = col_walls[c].lower_bound(r);
            if (it_up != col_walls[c].begin()) {
                --it_up;
                int i = *it_up;
                row_walls[i].erase(c);
                col_walls[c].erase(i);
            }
            
            // Down
            auto it_down = col_walls[c].lower_bound(r);
            if (it_down != col_walls[c].end()) {
                int i = *it_down;
                row_walls[i].erase(c);
                col_walls[c].erase(i);
            }
            
            // Left
            auto it_left = row_walls[r].lower_bound(c);
            if (it_left != row_walls[r].begin()) {
                --it_left;
                int j = *it_left;
                row_walls[r].erase(j);
                col_walls[j].erase(r);
            }
            
            // Right
            auto it_right = row_walls[r].lower_bound(c);
            if (it_right != row_walls[r].end()) {
                int j = *it_right;
                row_walls[r].erase(j);
                col_walls[j].erase(r);
            }
        }
    }
    
    int ans = 0;
    rep(i, h) ans += row_walls[i].size();
    cout << ans << '\n';
    
    return 0;
}
```

### Explanation
1. **Initialization**: We initialize sets for each row and column to keep track of walls. Each cell (i, j) is added to the sets for row i and column j.
2. **Query Processing**:
   - **Wall Present**: If the bomb's cell (r, c) has a wall, it is removed from both the row and column sets.
   - **No Wall Present**: For each direction (up, down, left, right), we check for the nearest wall in the respective set. If found, the wall is removed from both the row and column sets.
3. **Output**: The total remaining walls are counted by summing the sizes of all row sets (or column sets, as they should be consistent).

This approach efficiently handles each query in logarithmic time, ensuring the solution meets the problem's constraints.
