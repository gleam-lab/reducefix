To fix this code, we need to address the issue where the code fails to correctly identify the first walls in the four directions when the bomb is placed on an already empty cell. The current implementation has flaws in how it handles the search for the nearest walls in each direction, leading to incorrect deletions or missed deletions.

### Approaches
1. **Data Structures**: Use sets to maintain the positions of walls in each row and column. This allows efficient insertion, deletion, and searching operations.
2. **Query Processing**:
   - **Direct Hit**: If the bomb's position contains a wall, remove it directly.
   - **Indirect Hit**: If the bomb's position does not contain a wall, search for the nearest walls in each of the four directions (up, down, left, right) and remove them if they exist.
3. **Efficient Search**: Utilize the `upper_bound` and `lower_bound` functions to find the nearest walls in logarithmic time relative to the number of elements in the sets.

### Solution Code
```cpp
#include <bits/stdc++.h>
using namespace std;
#define rep(i,n) for(int i = 0; i < (n); i++)
using ll = long long;

int main(void) {
    int h, w, q;
    cin >> h >> w >> q;
    vector<set<int>> tate(w), yoko(h);
    rep(i,h) rep(j,w) {
        tate[j].insert(i);
        yoko[i].insert(j);
    }
    rep(qi,q) {
        int r, c;
        cin >> r >> c;
        r--; c--;
        if (tate[c].count(r)) {
            tate[c].erase(r);
            yoko[r].erase(c);
        } else {
            // Up
            auto it_up = tate[c].lower_bound(r);
            if (it_up != tate[c].begin()) {
                it_up--;
                int tmp = *it_up;
                tate[c].erase(tmp);
                yoko[tmp].erase(c);
            }
            // Down
            auto it_down = tate[c].upper_bound(r);
            if (it_down != tate[c].end()) {
                int tmp = *it_down;
                tate[c].erase(tmp);
                yoko[tmp].erase(c);
            }
            // Left
            auto it_left = yoko[r].lower_bound(c);
            if (it_left != yoko[r].begin()) {
                it_left--;
                int tmp = *it_left;
                yoko[r].erase(tmp);
                tate[tmp].erase(r);
            }
            // Right
            auto it_right = yoko[r].upper_bound(c);
            if (it_right != yoko[r].end()) {
                int tmp = *it_right;
                yoko[r].erase(tmp);
                tate[tmp].erase(r);
            }
        }
    }
    int ans = 0;
    rep(i,w) {
        ans += tate[i].size();
    }
    cout << ans << "\n";
    return 0;
}
```

### Explanation
- **Data Structures**: The `tate` vector of sets keeps track of the rows that have walls in each column, and the `yoko` vector of sets keeps track of the columns that have walls in each row.
- **Query Processing**:
  - **Direct Hit**: If the query's position `(r, c)` has a wall (checked via `tate[c].count(r)`), it is directly removed from both the column and row sets.
  - **Indirect Hit**: If the position is empty, the code searches for the nearest walls in all four directions:
    - **Up**: Uses `lower_bound` to find the first wall at or below `r`, then checks if there is a wall above by decrementing the iterator.
    - **Down**: Uses `upper_bound` to find the first wall strictly above `r`.
    - **Left**: Similar to up, but searches in the row set for columns left of `c`.
    - **Right**: Similar to down, but searches in the row set for columns right of `c`.
- **Efficiency**: Each operation (insertion, deletion, search) is performed in logarithmic time relative to the number of walls in the respective row or column, ensuring the solution handles the constraints efficiently. The total number of remaining walls is computed by summing the sizes of all column sets.
