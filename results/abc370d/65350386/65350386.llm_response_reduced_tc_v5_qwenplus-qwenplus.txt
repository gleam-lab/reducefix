To solve this problem efficiently, we need to simulate placing bombs on a grid and destroying walls in specific directions. The challenge lies in efficiently tracking which walls remain after each bomb placement.

### Key Observations:
1. **Each cell initially has a wall**.
2. When a bomb is placed on a cell that still has its wall, it simply destroys that wall.
3. If the cell already has no wall (i.e., previously destroyed), it will attempt to destroy the first walls found in the four cardinal directions (up, down, left, right).
4. We must perform these operations **efficiently**, especially considering large constraints (`H * W <= 4e5`, `Q <= 2e5`).

### Efficient Data Structures:
- Use `std::set` for each row and column to maintain the positions of remaining walls.
- This allows us to use `lower_bound` to find nearest walls in O(log N) time.

### Strategy:
- Maintain two arrays of sets:
  - `rows[i]`: set of columns where walls exist in row i
  - `cols[j]`: set of rows where walls exist in column j
- For each query:
  - Check if there's a wall at `(r, c)` using both sets.
  - If present: remove it from both row and column sets.
  - If not: search in each direction using `lower_bound` and erase the first wall found (if any).

Here’s the corrected and efficient implementation:

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    int H, W, Q;
    cin >> H >> W >> Q;

    vector<set<int>> rows(H), cols(W);

    // Initialize all cells as having walls
    for (int i = 0; i < H; ++i) {
        for (int j = 0; j < W; ++j) {
            rows[i].insert(j);
            cols[j].insert(i);
        }
    }

    while (Q--) {
        int r, c;
        cin >> r >> c;
        r--; c--;

        if (cols[c].count(r) == 0) {
            // Bomb at (r,c) already exploded, do directional destruction
            auto& row = rows[r];
            auto& col = cols[c];

            // Up
            {
                auto it = col.lower_bound(r);
                if (it != col.begin()) {
                    --it;
                    int x = *it;
                    cols[c].erase(x);
                    rows[x].erase(c);
                }
            }

            // Down
            {
                auto it = col.lower_bound(r);
                if (it != col.end() && *it == r) ++it;
                if (it != col.end()) {
                    int x = *it;
                    cols[c].erase(x);
                    rows[x].erase(c);
                }
            }

            // Left
            {
                auto it = row.lower_bound(c);
                if (it != row.begin()) {
                    --it;
                    int y = *it;
                    rows[r].erase(y);
                    cols[y].erase(r);
                }
            }

            // Right
            {
                auto it = row.lower_bound(c);
                if (it != row.end() && *it == c) ++it;
                if (it != row.end()) {
                    int y = *it;
                    rows[r].erase(y);
                    cols[y].erase(r);
                }
            }
        } else {
            // Bomb hits a wall directly
            rows[r].erase(c);
            cols[c].erase(r);
        }
    }

    int total = 0;
    for (int i = 0; i < H; ++i)
        total += rows[i].size();

    cout << total << '\n';
    return 0;
}
```

### Explanation of Fixes:
- **Proper handling of duplicate check**: Used `cols[c].count(r)` to determine if a wall exists at `(r, c)`.
- **Directional explosion logic**:
  - In "up", we go backward from `r` in `cols[c]` to find the first existing wall above.
  - In "down", we go forward from `r` in `cols[c]`.
  - Similarly for "left" and "right" using `rows[r]`.

### Time Complexity:
- Each operation uses `set::lower_bound`, which is `O(log N)`.
- Total complexity per query is `O(log N)`, leading to an overall acceptable performance for up to `2e5` queries.
