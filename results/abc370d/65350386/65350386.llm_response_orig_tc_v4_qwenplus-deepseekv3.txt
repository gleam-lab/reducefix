### Approaches

1. **Initialization**: We start by initializing a grid where each cell has a wall. We'll use two sets for each row and column to keep track of the walls. This helps in efficiently finding the nearest walls in any direction.

2. **Processing Queries**: For each query:
   - If there's a wall at the given cell, remove it.
   - If there's no wall, look for the nearest walls in the four directions (up, down, left, right) and remove them if they exist.

3. **Efficiency**: Using sets allows us to efficiently perform insertion, deletion, and search operations in logarithmic time, which is crucial given the constraints.

### Solution Code

```cpp
#include <bits/stdc++.h>
using namespace std;
#define rep(i, n) for(int i = 0; i < (n); i++)
using ll = long long;

int main() {
    int h, w, q;
    cin >> h >> w >> q;
    vector<set<int>> row(h), col(w);
    rep(i, h) rep(j, w) {
        row[i].insert(j);
        col[j].insert(i);
    }
    int ans = h * w;
    rep(qi, q) {
        int r, c;
        cin >> r >> c;
        r--; c--;
        if (row[r].count(c)) {
            row[r].erase(c);
            col[c].erase(r);
            ans--;
        } else {
            auto it_up = col[c].lower_bound(r);
            if (it_up != col[c].begin()) {
                it_up--;
                int i = *it_up;
                row[i].erase(c);
                col[c].erase(it_up);
                ans--;
            }
            auto it_down = col[c].lower_bound(r);
            if (it_down != col[c].end()) {
                int i = *it_down;
                row[i].erase(c);
                col[c].erase(it_down);
                ans--;
            }
            auto it_left = row[r].lower_bound(c);
            if (it_left != row[r].begin()) {
                it_left--;
                int j = *it_left;
                row[r].erase(j);
                col[j].erase(r);
                ans--;
            }
            auto it_right = row[r].lower_bound(c);
            if (it_right != row[r].end()) {
                int j = *it_right;
                row[r].erase(j);
                col[j].erase(r);
                ans--;
            }
        }
    }
    cout << ans << "\n";
    return 0;
}
```

### Explanation

1. **Initialization**: We initialize `row` and `col` as vectors of sets. Each set in `row[i]` contains column indices where walls exist in the i-th row, and similarly, each set in `col[j]` contains row indices where walls exist in the j-th column. Initially, all cells have walls, so all sets are populated with their respective indices.

2. **Processing Queries**: For each query:
   - **Wall Exists at (r, c)**: If a wall is found, it is removed from both the row and column sets, and the wall count `ans` is decremented.
   - **No Wall at (r, c)**:
     - **Up Direction**: We use `lower_bound` to find the first wall below (r, c) and then move to the previous element to get the first wall above. If found, it is removed.
     - **Down Direction**: The first wall below (r, c) is found and removed if it exists.
     - **Left Direction**: The first wall to the left of (r, c) is found and removed if it exists.
     - **Right Direction**: The first wall to the right of (r, c) is found and removed if it exists.

3. **Output**: After processing all queries, the remaining number of walls is printed.

This approach efficiently handles each query in logarithmic time per operation, leveraging the ordered nature of sets to quickly locate the nearest walls in each direction. The solution ensures correctness and optimal performance within the problem constraints.
