
The problem involves processing a grid where each cell has walls and a series of queries that place bombs that destroy walls. The goal is to determine the number of walls left after processing all queries.

To solve this problem efficiently, especially given the constraints, we need to manage the grid operations such that we don't explicitly iterate over the entire grid for each query.

Here is a structured approach to solving the problem:

1. **Initialization**: Create data structures to efficiently manage wall existence and deletion. For this, two maps (`tate` and `yoko`) are used to keep track of vertical and horizontal walls respectively.

2. **Query Handling**:
   - If a bomb is placed at a cell with a wall, remove that wall.
   - If the cell is already bombarded, we need to handle the deletion of surrounding walls. This requires efficient querying and deletion from the `tate` and `yoko` maps.

3. **Efficient Wall Handling**:
   - For deletion in the vertical direction, use set data structures to quickly find and remove walls at specific rows for a given column.
   - For deletion in the horizontal direction, use set data structures to quickly find and remove walls at specific columns for a given row.

The current approach has several inefficiencies, particularly in the deletion of surrounding walls. We can utilize a more efficient approach by using data structures like segment trees or binary indexed trees to manage the range deletion of walls.

### Revised Approach:
1. **Data Structures**:
   - Use segment trees for efficient range deletion queries.
   - Maintain segment trees for each row and column to handle range deletions in both directions.

2. **Query Processing**:
   - For each query, if the cell has a wall, remove it.
   - If the cell is bombarded, update the segment trees to mark the removal of walls in the surrounding range for both rows and columns.

3. **Calculation of Remaining Walls**:
   - After processing all queries, iterate through all cells in the grid and use the segment trees to determine if there are any walls left in each cell.

struct SegmentTree {
    vector<int> tree;
    int n;

    SegmentTree(int size) {
        n = size;
        tree.resize(4 * n);
    }

    void update(int idx, int l, int r, int val) {
        if (l <= idx && r >= idx) {
            tree[idx] += val;
        } else {
            if (l <= n && r >= n) {
                update(2 * idx, l, r, val);
                update(2 * idx + 1, l, r, val);
