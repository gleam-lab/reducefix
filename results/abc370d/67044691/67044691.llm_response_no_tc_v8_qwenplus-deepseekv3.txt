To fix this code, we need to address the issue where the code does not correctly handle the destruction of walls in all four directions (up, down, left, right) when there is no wall at the bomb's position. The original code attempts to find the nearest walls in each direction but fails to correctly identify them in some cases, especially when the bomb's position is at the boundary of the grid. 

### Approaches
1. **Grid Representation**: Use sets to keep track of the walls in each row and column. This allows efficient insertion, deletion, and searching operations.
2. **Handling Queries**: For each query, check if there is a wall at the bomb's position. If there is, remove it. If there isn't, find the nearest walls in each of the four directions (up, down, left, right) and remove them if they exist.
3. **Efficient Search**: Utilize the `upper_bound` and `lower_bound` methods to efficiently find the nearest walls in the specified directions.

### Solution Code
```cpp
#include <bits/stdc++.h>
using namespace std;
#define ll long long

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);

    ll h, w, q;
    cin >> h >> w >> q;
    vector<set<ll>> row(h), col(w);
    
    for (ll i = 0; i < h; ++i) {
        for (ll j = 0; j < w; ++j) {
            row[i].insert(j);
        }
    }
    for (ll j = 0; j < w; ++j) {
        for (ll i = 0; i < h; ++i) {
            col[j].insert(i);
        }
    }

    while (q--) {
        ll r, c;
        cin >> r >> c;
        r--; c--;

        if (row[r].count(c)) {
            row[r].erase(c);
            col[c].erase(r);
        } else {
            // Find the nearest walls in four directions
            vector<pair<ll, ll>> to_remove;
            
            // Left
            auto it_left = row[r].lower_bound(c);
            if (it_left != row[r].begin()) {
                --it_left;
                to_remove.emplace_back(r, *it_left);
            }
            
            // Right
            auto it_right = row[r].upper_bound(c);
            if (it_right != row[r].end()) {
                to_remove.emplace_back(r, *it_right);
            }
            
            // Up
            auto it_up = col[c].lower_bound(r);
            if (it_up != col[c].begin()) {
                --it_up;
                to_remove.emplace_back(*it_up, c);
            }
            
            // Down
            auto it_down = col[c].upper_bound(r);
            if (it_down != col[c].end()) {
                to_remove.emplace_back(*it_down, c);
            }
            
            // Remove the walls found
            for (auto [i, j] : to_remove) {
                row[i].erase(j);
                col[j].erase(i);
            }
        }
    }

    ll ans = 0;
    for (ll i = 0; i < h; ++i) {
        ans += row[i].size();
    }
    cout << ans << endl;
}
```

### Explanation
1. **Initialization**: The grid is initialized with walls in every cell. Rows and columns are represented using sets to store the positions of walls for efficient access and modification.
2. **Query Processing**: For each query:
   - If there is a wall at the specified position, it is removed.
   - If there is no wall, the nearest walls in the four directions (left, right, up, down) are found using `lower_bound` and `upper_bound` methods. These walls are then removed.
3. **Result Calculation**: After processing all queries, the total number of remaining walls is computed by summing the sizes of the sets representing each row.

This approach ensures that each query is processed efficiently, leveraging the properties of sets to maintain and update the grid state dynamically. The solution efficiently handles the constraints and ensures correct results for all possible inputs.
