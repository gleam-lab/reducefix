To fix this code, we need to address the incorrect handling of the bomb's effect when a cell is already empty. The original code attempts to find the first walls in each direction (up, down, left, right) but fails to correctly identify the closest walls in each direction due to incorrect set operations and boundary checks.

### Approaches
1. **Data Structures**: Use sets to keep track of walls in each row and column. This allows efficient searching for the closest walls in each direction.
2. **Query Processing**:
   - For each query, check if the cell (R_q, C_q) has a wall. If it does, remove it.
   - If the cell is already empty, find the closest walls in each of the four directions (up, down, left, right) and remove them if they exist.
3. **Efficiency**: Ensure the operations are efficient by leveraging the ordered nature of sets to perform upper_bound and lower_bound operations, which run in O(log N) time.

### Solution Code
```cpp
#include <bits/stdc++.h>
using namespace std;
#define ll long long

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);
    ll h, w, q;
    cin >> h >> w >> q;
    vector<set<ll>> row(h), col(w);
    for (ll i = 0; i < h; ++i) {
        for (ll j = 0; j < w; ++j) {
            row[i].insert(j);
            col[j].insert(i);
        }
    }
    while (q--) {
        ll r, c;
        cin >> r >> c;
        --r; --c;
        if (row[r].count(c)) {
            row[r].erase(c);
            col[c].erase(r);
        } else {
            // Find up
            auto it_up = col[c].lower_bound(r);
            if (it_up != col[c].begin()) {
                --it_up;
                ll i = *it_up;
                row[i].erase(c);
                col[c].erase(i);
            }
            // Find down
            it_up = col[c].upper_bound(r);
            if (it_up != col[c].end()) {
                ll i = *it_up;
                row[i].erase(c);
                col[c].erase(i);
            }
            // Find left
            auto it_left = row[r].lower_bound(c);
            if (it_left != row[r].begin()) {
                --it_left;
                ll j = *it_left;
                row[r].erase(j);
                col[j].erase(r);
            }
            // Find right
            it_left = row[r].upper_bound(c);
            if (it_left != row[r].end()) {
                ll j = *it_left;
                row[r].erase(j);
                col[j].erase(r);
            }
        }
    }
    ll ans = 0;
    for (ll i = 0; i < h; ++i) {
        ans += row[i].size();
    }
    cout << ans << '\n';
    return 0;
}
```

### Explanation
1. **Initialization**: We initialize sets `row` and `col` to keep track of walls in each row and column. Each cell (i, j) is added to both `row[i]` and `col[j]`.
2. **Query Processing**:
   - For each query, if the cell (R_q, C_q) contains a wall, it is removed from both `row` and `col`.
   - If the cell is empty, we search for the closest walls in each direction:
     - **Up**: The largest value less than `r` in `col[c]`.
     - **Down**: The smallest value greater than `r` in `col[c]`.
     - **Left**: The largest value less than `c` in `row[r]`.
     - **Right**: The smallest value greater than `c` in `row[r]`.
   - Each found wall is removed from both `row` and `col`.
3. **Result Calculation**: After processing all queries, we count the remaining walls by summing the sizes of all `row` sets.

This approach ensures efficient processing of each query in logarithmic time relative to the number of walls, making it suitable for large grid sizes and query counts.
