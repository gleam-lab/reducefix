To fix this code, we need to correctly handle the queries where a bomb is placed at a cell (R_q, C_q). The issue with the original code is that it doesn't correctly identify the first walls in the four directions (up, down, left, right) when the bomb is placed on an empty cell. The original code attempts to find these walls using upper_bound and lower_bound operations on sets, but it may not correctly identify the closest walls in all cases.

### Approaches
1. **Data Structures**: Use sets to keep track of the walls in each row and each column. This allows efficient insertion, deletion, and searching operations.
2. **Handling Queries**:
   - If the bomb is placed on a cell with a wall, remove that wall from the sets.
   - If the bomb is placed on an empty cell, find the nearest walls in all four directions (up, down, left, right) and remove them from the sets.
3. **Finding Nearest Walls**:
   - For the left direction, find the largest column less than the current column in the same row.
   - For the right direction, find the smallest column greater than the current column in the same row.
   - For the up direction, find the largest row less than the current row in the same column.
   - For the down direction, find the smallest row greater than the current row in the same column.

### Solution Code
```cpp
#include <bits/stdc++.h>
using namespace std;
#define ll long long

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);

    ll h, w, q;
    cin >> h >> w >> q;
    vector<set<ll>> row(h + 1), col(w + 1);
    for (ll i = 1; i <= h; ++i) {
        for (ll j = 1; j <= w; ++j) {
            row[i].insert(j);
            col[j].insert(i);
        }
    }
    set<pair<ll, ll>> walls;
    for (ll i = 1; i <= h; ++i) {
        for (ll j = 1; j <= w; ++j) {
            walls.insert({i, j});
        }
    }
    while (q--) {
        ll r, c;
        cin >> r >> c;
        if (walls.find({r, c}) != walls.end()) {
            walls.erase({r, c});
            row[r].erase(c);
            col[c].erase(r);
        } else {
            vector<pair<ll, ll>> to_remove;
            // Left
            auto it_left = row[r].lower_bound(c);
            if (it_left != row[r].begin()) {
                --it_left;
                to_remove.emplace_back(r, *it_left);
            }
            // Right
            auto it_right = row[r].upper_bound(c);
            if (it_right != row[r].end()) {
                to_remove.emplace_back(r, *it_right);
            }
            // Up
            auto it_up = col[c].lower_bound(r);
            if (it_up != col[c].begin()) {
                --it_up;
                to_remove.emplace_back(*it_up, c);
            }
            // Down
            auto it_down = col[c].upper_bound(r);
            if (it_down != col[c].end()) {
                to_remove.emplace_back(*it_down, c);
            }
            for (auto &p : to_remove) {
                ll x = p.first, y = p.second;
                if (walls.find({x, y}) != walls.end()) {
                    walls.erase({x, y});
                    row[x].erase(y);
                    col[y].erase(x);
                }
            }
        }
    }
    cout << walls.size() << '\n';
    return 0;
}
```

### Explanation
1. **Initialization**: The code initializes sets for each row and column to keep track of walls. A global set `walls` is used to store all wall positions for quick lookup.
2. **Processing Queries**:
   - **Wall Removal**: If the queried position has a wall, it is removed from the sets and the `walls` set.
   - **Bomb Effect on Empty Cell**: For each direction (left, right, up, down), the code finds the nearest wall using set operations (`lower_bound` and `upper_bound`). These walls are then removed from the sets and the `walls` set.
3. **Output**: The size of the `walls` set, representing the remaining walls after all queries, is printed.

This approach ensures that each query is processed efficiently, leveraging the properties of sets for logarithmic-time operations, making the solution both correct and optimal for large input sizes.
