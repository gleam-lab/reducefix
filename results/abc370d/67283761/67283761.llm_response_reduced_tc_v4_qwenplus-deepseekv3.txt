To fix this code, we need to address the incorrect handling of the destruction process when a bomb is placed at a cell that no longer has a wall. The original code attempts to cache the distances to the nearest walls, but this approach fails to correctly identify the first walls in each direction when the bomb is placed on an empty cell. 

### Approaches
1. **Initial Setup**: The grid is initialized with all cells having walls (True).
2. **Handling Queries**:
   - **If the cell has a wall**, it is destroyed, and the count of destroyed walls is incremented.
   - **If the cell does not have a wall**, we need to search in four directions (up, down, left, right) for the nearest wall and destroy it. This involves:
     - **Up Direction**: Search for the first wall above the current cell.
     - **Down Direction**: Search for the first wall below the current cell.
     - **Left Direction**: Search for the first wall to the left of the current cell.
     - **Right Direction**: Search for the first wall to the right of the current cell.
3. **Efficiency**: For each query, we check each direction sequentially. The overall complexity is O(Q * (H + W)) in the worst case, but with constraints ensuring H * W ≤ 4 × 10^5 and Q ≤ 2 × 10^5, this approach is feasible.

### Solution Code
```cpp
#include <vector>
#include <iostream>
using namespace std;

int main() {
    int H, W, Q;
    cin >> H >> W >> Q;
    vector<vector<bool>> map(H, vector<bool>(W, true));
    int broken = 0;

    for (int i = 0; i < Q; ++i) {
        int r, c;
        cin >> r >> c;
        --r; --c;

        if (map[r][c]) {
            map[r][c] = false;
            ++broken;
        } else {
            // Up
            for (int i = r - 1; i >= 0; --i) {
                if (map[i][c]) {
                    map[i][c] = false;
                    ++broken;
                    break;
                }
            }
            // Down
            for (int i = r + 1; i < H; ++i) {
                if (map[i][c]) {
                    map[i][c] = false;
                    ++broken;
                    break;
                }
            }
            // Left
            for (int j = c - 1; j >= 0; --j) {
                if (map[r][j]) {
                    map[r][j] = false;
                    ++broken;
                    break;
                }
            }
            // Right
            for (int j = c + 1; j < W; ++j) {
                if (map[r][j]) {
                    map[r][j] = false;
                    ++broken;
                    break;
                }
            }
        }
    }
    cout << (H * W - broken) << endl;
    return 0;
}
```

### Explanation
- **Initialization**: The grid is initialized with all cells set to `true`, indicating walls are present.
- **Query Processing**:
  - For each query, if the cell (R_q, C_q) has a wall, it is destroyed, and the count of destroyed walls (`broken`) is incremented.
  - If the cell does not have a wall, the code searches in each of the four directions (up, down, left, right) for the nearest wall. The first wall encountered in each direction is destroyed.
- **Output**: The total number of remaining walls is calculated as the total cells (H × W) minus the count of destroyed walls (`broken`), which is printed as the result.

This approach ensures that each query is processed correctly, and the solution efficiently handles the constraints by directly searching for the nearest walls in each direction when necessary.
