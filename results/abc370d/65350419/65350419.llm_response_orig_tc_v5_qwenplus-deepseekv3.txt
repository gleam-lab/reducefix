To fix this code, we need to ensure that the logic correctly handles both the destruction of walls at the bomb's location and the destruction of the first walls in the four directions if the bomb's location is already empty. The issue in the initial code likely stems from incorrect handling of the lower_bound and range checks when searching for adjacent walls to destroy.

### Approaches
1. **Data Structures**: Use sets to keep track of walls in each row and column. This allows efficient insertion, deletion, and searching operations.
2. **Processing Queries**:
   - **If the cell has a wall**: Remove it from the sets.
   - **If the cell is empty**: Find the nearest walls in the four directions (up, down, left, right) and remove them from the sets.
3. **Efficiency**: Using sets ensures that each operation (insertion, deletion, search) is O(log n), making the overall complexity O(Q log N) where N is the grid size.

### Solution Code
```cpp
#include <bits/stdc++.h>
using namespace std;
#define rep(i,n) for(int i = 0; i < (n); i++)
using ll = long long;

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(NULL);
    
    int h, w, q;
    cin >> h >> w >> q;
    vector<set<int>> row_walls(h), col_walls(w);
    
    rep(i, h) rep(j, w) {
        row_walls[i].insert(j);
        col_walls[j].insert(i);
    }
    
    int ans = h * w;
    
    rep(qi, q) {
        int r, c;
        cin >> r >> c;
        r--; c--;
        
        if (row_walls[r].count(c)) {
            row_walls[r].erase(c);
            col_walls[c].erase(r);
            ans--;
        } else {
            // Check up
            auto it_up = col_walls[c].lower_bound(r);
            if (it_up != col_walls[c].begin()) {
                it_up--;
                int i = *it_up;
                if (row_walls[i].count(c)) {
                    row_walls[i].erase(c);
                    col_walls[c].erase(i);
                    ans--;
                }
            }
            
            // Check down
            auto it_down = col_walls[c].lower_bound(r);
            if (it_down != col_walls[c].end()) {
                int i = *it_down;
                if (row_walls[i].count(c)) {
                    row_walls[i].erase(c);
                    col_walls[c].erase(i);
                    ans--;
                }
            }
            
            // Check left
            auto it_left = row_walls[r].lower_bound(c);
            if (it_left != row_walls[r].begin()) {
                it_left--;
                int j = *it_left;
                if (col_walls[j].count(r)) {
                    col_walls[j].erase(r);
                    row_walls[r].erase(j);
                    ans--;
                }
            }
            
            // Check right
            auto it_right = row_walls[r].lower_bound(c);
            if (it_right != row_walls[r].end()) {
                int j = *it_right;
                if (col_walls[j].count(r)) {
                    col_walls[j].erase(r);
                    row_walls[r].erase(j);
                    ans--;
                }
            }
        }
    }
    
    cout << ans << "\n";
    return 0;
}
```

### Explanation
1. **Initialization**: We initialize sets for each row and column to keep track of walls. Initially, every cell has a wall, so each row set contains all column indices and each column set contains all row indices.
2. **Processing Queries**: For each query:
   - **If the cell has a wall**: Remove the cell from the row and column sets and decrement the wall count.
   - **If the cell is empty**: Search for the nearest walls in all four directions using `lower_bound`:
     - **Up**: Check the first wall above the cell in the same column.
     - **Down**: Check the first wall below the cell in the same column.
     - **Left**: Check the first wall to the left of the cell in the same row.
     - **Right**: Check the first wall to the right of the cell in the same row.
   - Remove any found walls from the sets and decrement the wall count.
3. **Output**: The remaining wall count is printed after processing all queries.

This approach efficiently manages the grid state and ensures correct handling of both direct and adjacent wall destruction with optimal time complexity.
