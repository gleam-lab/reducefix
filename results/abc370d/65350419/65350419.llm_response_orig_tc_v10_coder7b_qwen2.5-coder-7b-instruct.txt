   - We'll maintain separate sets for walls in each row (`yoko`) and each column (`tate`).

2. **Query Processing**:
   - For each query, check if there's a wall at the specified position `(r, c)`.
   - If a wall exists at `(r, c)`, simply remove it.
   - If no wall exists at `(r, c)`, traverse the corresponding row and column to find and remove the nearest walls according to the rules.

3. **Efficiency**:
   - Using sets ensures that finding and removing elements is logarithmic in the number of elements.
   - By maintaining separate sets for rows and columns, we can handle each query efficiently.

                if (up_col < w && tate[up_col].find(r) == tate[up_col].end()) {
                    yoko[r].erase(up_col);
                    tate[up_col].erase(r);
                }
            }

            auto iter_down = yoko[r].lower_bound(c);
            if (iter_down != yoko[r].end()) {
                int down_col = *iter_down;
                if (down_col < w && tate[down_col].find(r) == tate[down_col].end()) {
                    yoko[r].erase(down_col);
                    tate[down_col].erase(r);
                }
            }

            auto iter_left = tate[c].upper_bound(r);
            if (iter_left != tate[c].begin()) {
                iter_left--;
                int left_row = *iter_left;

                    tate[c].erase(left_row);

                }
            }

            auto iter_right = tate[c].lower_bound(r);
            if (iter_right != tate[c].end()) {

                if (right_row < h && yoko[right_row].find(c) == yoko[right_row].end()) {
