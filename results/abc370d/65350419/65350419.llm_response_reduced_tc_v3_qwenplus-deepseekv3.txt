The problem involves processing queries on a grid where each query can either destroy a wall at a specific location or destroy the nearest walls in the four cardinal directions if the specified location is already empty.

### Approaches
1. **Initialization**: We need to track the walls in each row and column. For this, we can use sets to maintain the walls' positions for efficient insertion and deletion operations.
2. **Processing Queries**: For each query:
   - If the specified cell has a wall, we remove it from both row and column sets.
   - If the cell is empty, we find the nearest walls in each of the four directions (up, down, left, right) and remove them if they exist.
3. **Efficiency**: Using sets allows us to perform these operations in logarithmic time, ensuring the solution is efficient enough for large input sizes.

### Solution Code
```cpp
#include <bits/stdc++.h>
using namespace std;
#define rep(i, n) for(int i = 0; i < (n); i++)
using ll = long long;

int main() {
    int h, w, q;
    cin >> h >> w >> q;
    vector<set<int>> row_walls(h), col_walls(w);
    rep(i, h) rep(j, w) {
        row_walls[i].insert(j);
        col_walls[j].insert(i);
    }
    int ans = h * w;
    rep(qi, q) {
        int r, c;
        cin >> r >> c;
        r--; c--;
        if (row_walls[r].count(c)) {
            row_walls[r].erase(c);
            col_walls[c].erase(r);
            ans--;
        } else {
            auto it_up = col_walls[c].lower_bound(r);
            if (it_up != col_walls[c].begin()) {
                it_up--;
                int i = *it_up;
                if (row_walls[i].count(c)) {
                    row_walls[i].erase(c);
                    col_walls[c].erase(i);
                    ans--;
                }
            }
            auto it_down = col_walls[c].lower_bound(r);
            if (it_down != col_walls[c].end()) {
                int i = *it_down;
                if (row_walls[i].count(c)) {
                    row_walls[i].erase(c);
                    col_walls[c].erase(i);
                    ans--;
                }
            }
            auto it_left = row_walls[r].lower_bound(c);
            if (it_left != row_walls[r].begin()) {
                it_left--;
                int j = *it_left;
                if (col_walls[j].count(r)) {
                    col_walls[j].erase(r);
                    row_walls[r].erase(j);
                    ans--;
                }
            }
            auto it_right = row_walls[r].lower_bound(c);
            if (it_right != row_walls[r].end()) {
                int j = *it_right;
                if (col_walls[j].count(r)) {
                    col_walls[j].erase(r);
                    row_walls[r].erase(j);
                    ans--;
                }
            }
        }
    }
    cout << ans << "\n";
    return 0;
}
```

### Explanation
1. **Initialization**: We initialize sets for each row and column to keep track of wall positions. Initially, all cells have walls, so we populate these sets accordingly.
2. **Processing Queries**: For each query:
   - **Direct Hit**: If the queried cell has a wall, we remove it from both the row and column sets and decrement the wall count.
   - **Indirect Hit (Nearest Walls)**: If the cell is empty, we search for the nearest walls in each direction (up, down, left, right) using the `lower_bound` method to efficiently locate the nearest walls. If found, we remove these walls from the sets and decrement the wall count.
3. **Output**: After processing all queries, we print the remaining wall count, which was tracked and updated during the query processing.

This approach ensures that each operation is performed in logarithmic time, making the solution efficient and scalable for large inputs.
