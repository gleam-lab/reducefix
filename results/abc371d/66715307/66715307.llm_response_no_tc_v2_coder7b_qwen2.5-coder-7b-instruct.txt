```cpp
int n, x[200050], p[200050], q, a[200050];

int main() {
    cin >> n;
    for (int i = 1; i <= n; ++i) cin >> x[i];
    for (int i = 1; i <= n; ++i) {
        cin >> p[i];
        a[i] = a[i - 1] + p[i];
    }
    cin >> q;
    while (q--) {
        int l, r; cin >> l >> r;
        auto it1 = lower_bound(x + 1, x + n + 1, l);
        auto it2 = upper_bound(x + 1, x + n + 1, r);
        cout << a[it2 - x - 1] - a[it1 - x] << endl;
    }
    return 0;
}
```

### Explanation of Changes:
1. **Use `lower_bound` and `upper_bound`**: These functions help in finding the first position that is not less than `l` and the first position greater than `r`, respectively. This simplifies the search process and makes the code cleaner and more efficient.
2. **Correct Indexing**: Adjusted the indexing to be zero-based as `lower_bound` and `upper_bound` work with arrays starting from index 0.
3. **Removed Unnecessary Variables**: Removed variables like `l1`, `r1`, `l2`, `r2` which were used for binary search but are no longer needed due to the use of `lower_bound` and `upper_bound`.
4. **Efficiency**: The use of `lower_bound` and `upper_bound` reduces the complexity of searching within sorted arrays, making the solution more efficient especially when dealing with large inputs.
