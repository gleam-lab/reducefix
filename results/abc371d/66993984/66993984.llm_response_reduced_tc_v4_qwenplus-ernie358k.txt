```cpp
#include <iostream>
#include <vector>
#include <map>
#include <algorithm>

using namespace std;

int main() {
    int n, q;
    cin >> n;
    
    vector<long long> x(n + 1), p(n + 1); // Use long long to handle large numbers
    map<long long, long long> prefixSum; // Using map to store prefix sums efficiently
    
    for (int i = 1; i <= n; i++) {
        cin >> x[i];
    }
    
    long long totalPower = 0;
    for (int i = 1; i <= n; i++) {
        cin >> p[i];
        totalPower += p[i];
        prefixSum[x[i]] = totalPower; // Store prefix sum at each x[i] position
    }
    
    cin >> q;
    while (q--) {
        long long L, R;
        cin >> L >> R;
        
        // Use lower_bound and upper_bound to find the positions in sorted x array
        auto itL = lower_bound(x.begin() + 1, x.end(), L);
        auto itR = upper_bound(x.begin() + 1, x.end(), R);
        
        // Convert iterators to indices
        int indexL = itL - x.begin();
        int indexR = itR - x.begin();
        
        // If indexL is still pointing to the beginning (i.e., L is smaller than all x[i]), use prefix sum at 0 (implicitly 0)
        // If indexR has gone past the end (i.e., R is larger than all x[i]), use the total prefix sum
        long long sumL = (indexL == 1 ? 0 : prefixSum[*(itL - 1)]);
        long long sumR = (indexR == n + 1 ? totalPower : prefixSum[*(itR - 1)]);
        
        cout << sumR - sumL << endl;
    }
    
    return 0;
}
```

### Explanation:
1. **Data Types**: Changed the arrays `x` and `p` to `long long` to accommodate the large input values.
2. **Prefix Sum Storage**: Used a `map<long long, long long>` to store prefix sums at each `x[i]` value. This allows for efficient look-up of prefix sums for any given value in `x`.
3. **Handling Boundaries**: Ensured that when `lower_bound` or `upper_bound` return iterators at the beginning or end of the array, the correct prefix sum values are used (0 for the beginning and total sum for the end).
4. **Iterator to Index Conversion**: Converted the iterators returned by `lower_bound` and `upper_bound` to indices to access the correct prefix sum values from the map.

This solution ensures algorithmic correctness and handles large input values and edge cases effectively.
