```cpp
#include<bits/stdc++.h>
using namespace std;

int n, q, x[200005], p[200005];
map<int, int> prefixSum;

int main() {
    cin >> n;
    for (int i = 1; i <= n; i++) {
        cin >> x[i];
    }
    for (int i = 1; i <= n; i++) {
        cin >> p[i];
        prefixSum[x[i]] += p[i];
    }
    cin >> q;
    while (q--) {
        int L, R;
        cin >> L >> R;
        auto itL = prefixSum.lower_bound(L);
        auto itR = prefixSum.upper_bound(R);
        int result = 0;
        if (itL != prefixSum.end()) {
            result -= itL->second;
        }
        if (itR != prefixSum.begin()) {
            itR--;
            result += itR->second;
        }
        cout << result << endl;
    }
    return 0;
}
```

### Explanation of Fixes:
1. **Prefix Sum Calculation**: The original code incorrectly used `sum` as an array to store prefix sums based on indices, which is not suitable because the values in `x` might be out of bounds or repeated. Instead, a map (`prefixSum`) is used where keys are the unique elements from `x` and values are their cumulative sums.
2. **Lower and Upper Bound Handling**: When calculating the range sum using `lower_bound` and `upper_bound`, special care must be taken to handle the case when `upper_bound` points to the end of the map. In such cases, we adjust the pointer to point to the last valid element before the upper bound.
3. **Result Adjustment**: Since `lower_bound` returns the first element that is not less than `L`, and `upper_bound` returns the first element greater than `R`, we need to adjust the results accordingly to correctly compute the sum within the specified range. This involves subtracting the contribution of the lower bound value and adding back the contribution of the element just below the upper bound.
