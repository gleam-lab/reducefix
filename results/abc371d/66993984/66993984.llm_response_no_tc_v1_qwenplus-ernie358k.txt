```cpp
#include <iostream>
#include <vector>
#include <map>
#include <algorithm>

using namespace std;

int main() {
    int n, q;
    cin >> n;
    vector<int> x(n + 1), p(n + 1); // Using vectors for better memory management and bounds safety
    map<int, int> prefixSum;

    // Read x values (0-indexed for easier range management, though we only use 1 to n)
    for (int i = 1; i <= n; ++i) {
        cin >> x[i];
    }

    // Read p values and compute prefix sums using a map for sparse updates
    for (int i = 1; i <= n; ++i) {
        cin >> p[i];
        prefixSum[x[i]] += p[i];
    }

    // Since prefix sums are stored based on x values directly, we need to maintain a cumulative sum
    map<int, int> cumulativeSum;
    int cumSum = 0;
    for (const auto& pair : prefixSum) {
        cumSum += pair.second;
        cumulativeSum[pair.first] = cumSum;
    }

    cin >> q;
    while (q--) {
        int L, R;
        cin >> L >> R;

        // Use lower_bound and upper_bound on the x values (since they are sorted implicitly by their insertion order in map keys)
        auto itL = cumulativeSum.lower_bound(L);
        auto itR = cumulativeSum.upper_bound(R);

        // If itL points to a key greater than R, it means there are no elements in the range [L, R]
        if (itL == cumulativeSum.end() || itL->first > R) {
            cout << 0 << endl;
        } else {
            int sumL = (itL == cumulativeSum.begin()) ? 0 : (--itL)->second;
            int sumR = (itR != cumulativeSum.end()) ? itR->second : cumulativeSum.rbegin()->second;
            cout << sumR - sumL << endl;
        }
    }

    return 0;
}
```

### Explanation:
1. **Vectors for `x` and `p`**: Using `vector` instead of raw arrays to avoid potential out-of-bounds issues and to simplify memory management.
2. **Using `map` for prefix sums**: Since we need to sum based on specific `x` values, using a `map` allows us to store only necessary `x` values and their cumulative `p` values.
3. **Cumulative sum maintenance**: Instead of using a simple prefix sum array, we maintain a cumulative sum map to efficiently query sums for any given range of `x` values.
4. **Range query using `lower_bound` and `upper_bound`**: We use these functions to find the appropriate bounds in the cumulative sum map, ensuring efficient O(log n) lookups.
5. **Edge case handling**: If `lower_bound` returns an iterator pointing beyond the end of the map or to a key greater than `R`, it means there are no elements in the specified range, and we output `0`.

This solution is efficient and handles edge cases correctly within the given constraints.
