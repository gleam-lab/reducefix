### Explanation:
1. **Input Reading and Initialization**: The code reads the input as described, initializes the prefix sums in `P`, and stores the mapping from `X` values to their indices in `x_to_index`.

2. **Lower and Upper Bound Handling**:

   - `upper_bound(R)` is used to find the smallest index `i` such that `X[i] > R`.

3. **Edge Case for Upper Bound**:
   - Since `upper_bound` returns an iterator one past the last element that is not greater than `R`, we use `prev(it_r)` to get the iterator to the largest element that is less than or equal to `R`. This is important to correctly calculate the prefix sum range.

4. **Prefix Sum Calculation**:
   - The answer is computed as `P[r_index + 1] - P[l_index]`, which gives the sum of all `p` values corresponding to `X` values between `L` and `R` (inclusive of `L` but exclusive of the element right after `R`).
