2. **Handling Lower and Upper Bounds**: Your code correctly uses `lower_bound` and `upper_bound` but doesn't correctly handle the cases where these bounds are beyond the end of the map. Specifically, you need to adjust the indices `l` and `r` to reflect the correct segment boundaries when the bounds are not found in the map.

3. **Efficient Data Structure**: Given the constraints, a more memory-efficient approach is needed. Instead of using a `map`, you can use a sorted vector to store the $X_i$ values and use binary search to find the corresponding indices. This will reduce memory usage significantly.

#include <iostream>
#include <vector>
#include <algorithm>

using namespace std;
using ll = long long;

int main() {
    int N;
    cin >> N;
    vector<ll> P(N + 1, 0);
    vector<int> X(N);
    rep(i, N) cin >> X[i];
    rep(i, N) {
        int p;
        cin >> p;
        P[i + 1] = P[i] + p;
    }

    // Sort X and create an index array for binary search
    vector<int> sorted_X = X;
    sort(sorted_X.begin(), sorted_X.end());
    vector<int> original_indices(N);
    rep(i, N) original_indices[i] = i;

    // Function to perform binary search and return the original index
    auto find_index = [&](int val) -> int {
        auto it = lower_bound(sorted_X.begin(), sorted_X.end(), val);
        if (it == sorted_X.end()) return N; // If val is greater than all elements
        // Find the original index of the found element
        int idx = distance(sorted_X.begin(), it);
        return original_indices[idx];
    };

    int Q;
    cin >> Q;
    rep(q, Q) {
        int L, R;
        cin >> L >> R;
        int l = find_index(L);
        int r = find_index(R + 1); // Use R + 1 to find the first element greater than R
        if (r > l) {
            r--; // Adjust r to the last valid index within the range
        }
        ll ans = P[r + 1] - P[l];
        cout << ans << endl;
    }

    return 0;
}

### Explanation:
1. **Sorted Vector**: We sort the `X` values and store their original indices in `original_indices`. This allows us to map back to the original indices after binary searching in the sorted vector.

2. **Binary Search**: We use `lower_bound` to find the closest index in the sorted vector. For the right bound, we search for `R + 1` to ensure we get the first element greater than `R`.

3. **Adjust Indices**: After finding `l` and `r`, we adjust `r` to ensure it points to the last valid index within the range $[L, R]$.

4. **Efficient Memory Usage**: By avoiding the use of a `map`, we significantly reduce memory overhead, making the code more suitable for large inputs.
