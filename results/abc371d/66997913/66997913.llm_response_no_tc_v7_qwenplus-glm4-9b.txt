
```cpp
#include <bits/stdc++.h>
using namespace std;
using ll = long long;

int main() {
  int n;
  cin >> n;
  vector<int> x(n);
  for (int i = 0; i < n; i++) cin >> x[i];
  vector<ll> prefix(n + 1, 0);
  for (int i = 0; i < n; i++) {
    cin >> prefix[i + 1];
    prefix[i + 1] += prefix[i];
  }
  
  int q;
  cin >> q;
  for (int i = 0; i < q; i++) {
    int l, r;
    cin >> l >> r;
    int rt = upper_bound(x.begin(), x.end(), r) - x.begin();
    int lt = lower_bound(x.begin(), x.end(), l) - x.begin();
    if (lt >= rt) {
      cout << "0\n"; // Ensure that we do not have an empty range
    } else {
      cout << prefix[rt] - prefix[lt] << "\n";
    }
  }
  return 0;
}
```

### Explanation:
1. **Prefix Sum Calculation**: The prefix sum array `prefix` is computed in a single loop, ensuring that we do not repeatedly calculate the sum from the start for each query.
2. **Range Query Handling**: The lower bound (`lt`) finds the first element greater than or equal to `l`, and the upper bound (`rt`) finds the first element greater than `r`. If `lt` is greater than or equal to `rt`, it means the range is invalid, and we should output `0`.
3. **Edge Case**: If the range is valid, the difference between the prefix sums at `rt` and `lt` gives the sum of elements within the range `[l, r]`.
4. **Correctness and Efficiency**: The solution handles even the worst-case scenarios efficiently with O(n + q * log(n)) time complexity, where `n` is the number of elements in the input array and `q` is the number of queries.
