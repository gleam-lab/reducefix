
```cpp
#include <iostream>
#include <vector>
#include <algorithm>

using namespace std;
using ll = long long;

int main() {
    int n;
    cin >> n;
    vector<int> x(n);
    for (int i = 0; i < n; i++) cin >> x[i];

    vector<ll> p(n), s(n + 1);
    for (int i = 0; i < n; i++) {
        cin >> p[i];
        s[i + 1] = s[i] + p[i];
    }
    
    int q;
    cin >> q;
    for (int i = 0; i < q; i++) {
        int l, r;
        cin >> l >> r;
        // Finding the position of r and l in the vector x
        int rt = upper_bound(x.begin(), x.end(), r) - x.begin() - 1; // Subtracting 1 to convert to 0-based index
        int lt = lower_bound(x.begin(), x.end(), l) - x.begin(); // This gives the first element greater or equal to l, hence no -1 needed
        
        // Calculate the sum from l to r using the prefix sums
        ll ans = (rt == -1) ? 0 : s[rt + 1] - s[lt];
        cout << ans << endl;
    }
    return 0;
}
```

### Explanation:
1. **Index Adjustment**: `upper_bound` and `lower_bound` return iterators to the first element that is **greater than** the query element (`r`) and the first element that is **greater or equal to** the query element (`l`), respectively. Since the problem expects 0-based indexing, we subtract 1 from the index returned by `upper_bound` for the right boundary.
2. **Edge Case Handling**: When `rt` is -1 after subtracting 1, it means there's no element greater than `r` in the array `x`, so the sum from `l` to `r` is 0.
3. **Prefix Sum Usage**: The sum from `l` to `r` can be computed by subtracting the prefix sum at `lt` (exclusive) from the prefix sum at `rt+1` (inclusive). If `rt` is -1, the subtraction results in 0, which is the correct answer.
4. **Efficiency**: The solution uses binary search (via `upper_bound` and `lower_bound`) which operates in O(log n) time for each query. Given that `q` queries are to be handled, the complexity for the query handling part is O(q log n). The preprocessing (computing prefix sums) is O(n). Thus, the overall complexity remains O(n + q log n).
