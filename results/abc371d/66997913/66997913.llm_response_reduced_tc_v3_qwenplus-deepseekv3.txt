### Solution Code
```cpp
#include <bits/stdc++.h>
using namespace std;
using ll = long long;

int main() {
  int n;
  cin >> n;
  vector<int> x(n);
  for (int i = 0; i < n; i++) cin >> x[i];
  vector<ll> p(n), s(n+1);
  for (int i = 0; i < n; i++) {
    cin >> p[i];
    s[i+1] = s[i] + p[i];
  }
  
  int q;
  cin >> q;
  for (int i = 0; i < q; i++) {
    int l, r;
    cin >> l >> r;
    int lt = lower_bound(x.begin(), x.end(), l) - x.begin();
    int rt = upper_bound(x.begin(), x.end(), r) - x.begin();
    ll ans = s[rt] - s[lt];
    cout << ans << endl;
  }
  return 0;
}
```

### Explanation
1. **Prefix Sum Array**: The prefix sum array `s` is constructed such that `s[i]` contains the sum of the first `i` elements of `p`. This allows for O(1) range sum queries.
2. **Query Processing**: For each query `(l, r)`, `lower_bound` finds the first position in `x` where the element is not less than `l`, and `upper_bound` finds the first position where the element is greater than `r`. The sum of elements in `p` from `lt` to `rt-1` is then `s[rt] - s[lt]`.
3. **Overflow Prevention**: By using `long long` for the sum variables, we avoid integer overflow issues that could arise from summing large numbers.

This approach efficiently handles the queries in logarithmic time per query (due to the binary search operations), making it suitable for large input sizes.
