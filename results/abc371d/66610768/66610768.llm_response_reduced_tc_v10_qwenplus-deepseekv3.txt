3. **Range Sum Calculation**: Calculate the sum of elements in `b` between the indices found in step 2. If the upper bound `r` is greater than the maximum value in `a`, the sum should be from the lower bound index to the last element of `b`.

    }
    for(int i = 1; i <= n; ++i) {

        c[i] = b[i] + c[i-1];

        ll l, r;

        auto l_it = lower_bound(a.begin()+1, a.end(), l);
        auto r_it = upper_bound(a.begin()+1, a.end(), r);
        int i = l_it - a.begin();
        int j = r_it - a.begin();
        ll sum = c[j-1] - c[i-1];

    }
    return 0;
}

1. **Input Reading**: The code reads the size of the arrays `n`, followed by the elements of arrays `a` and `b`. It then computes the prefix sum array `c` for efficient range sum queries.
2. **Binary Search for Indices**: For each query `[l, r]`, the code uses `lower_bound` to find the first element in `a` that is not less than `l` and `upper_bound` to find the first element in `a` that is greater than `r`. These operations efficiently locate the relevant indices in logarithmic time.
3. **Range Sum Calculation**: The sum of elements in `b` between the indices found by the binary search is computed using the prefix sum array `c`. This ensures the sum is calculated in constant time, making the solution efficient even for large input sizes.
4. **Edge Cases Handling**: The solution naturally handles edge cases, such as when `l` is less than all elements in `a` (resulting in `i` being 1) or when `r` is greater than all elements in `a` (resulting in `j` being `n+1`), by using the prefix sums correctly.
