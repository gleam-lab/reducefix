To solve this problem, we need to efficiently handle both adding edges and querying the k-th largest vertex connected to a given vertex. Here's a revised version of your code that addresses the issues and ensures it meets the requirements:

```cpp
#include<bits/stdc++.h>
using namespace std;

const int MAXN = 2e5 + 5;
vector<int> adj[MAXN];
set<int> st[MAXN];

void addEdge(int u, int v) {
    adj[u].push_back(v);
    adj[v].push_back(u);
    st[u].insert(v);
    st[v].insert(u);
}

int kthLargest(int v, int k) {
    if (st[v].size() < k) return -1;
    auto it = st[v].rbegin();
    advance(it, k - 1);
    return *it;
}

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);

    int n, q;
    cin >> n >> q;

    while (q--) {
        int type, u, v, k;
        cin >> type;
        if (type == 1) {
            cin >> u >> v;
            addEdge(u, v);
        } else {
            cin >> u >> k;
            cout << kthLargest(u, k) << '\n';
        }
    }

    return 0;
}
```

### Explanation:

1. **Graph Representation**: We use adjacency lists (`adj`) and sets (`st`) to store the connections and sorted neighbors respectively. Sets allow us to maintain the neighbors in sorted order automatically.

2. **Adding Edges**: The `addEdge` function adds an edge between vertices `u` and `v`, updating both the adjacency list and the set of neighbors.

3. **K-th Largest Neighbor**: The `kthLargest` function checks if there are at least `k` neighbors. If not, it returns `-1`. Otherwise, it uses a reverse iterator to find the k-th largest element in the set.

4. **Main Function**: The main function reads the input and processes each query. For type 1 queries, it calls `addEdge`. For type 2 queries, it calls `kthLargest` and prints the result.

This approach ensures efficient addition of edges and quick retrieval of the k-th largest neighbor using the properties of sets. The overall time complexity is dominated by the operations on the sets, which are O(log N) per operation. This makes the solution suitable for the given constraints.
