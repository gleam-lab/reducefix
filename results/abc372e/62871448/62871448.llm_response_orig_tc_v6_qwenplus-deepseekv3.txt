To fix this code, we need to address the incorrect output for certain test cases. The problem involves handling a series of queries on an undirected graph, where we need to add edges and query the k-th largest vertex in a connected component. The original code attempts to maintain the top 10 largest vertices for each connected component, but it fails to correctly merge these lists when combining components.

### Approaches
1. **Union-Find Data Structure**: We'll use the Union-Find (Disjoint Set Union, DSU) data structure to efficiently manage the connected components. This helps in quickly merging sets and finding the root of any element.
2. **Maintaining Sorted Lists**: For each connected component, we'll maintain a sorted list (in descending order) of the vertices. This allows us to efficiently access the k-th largest vertex when queried.
3. **Efficient Merging**: When merging two components, we need to combine their sorted lists while keeping only the top 10 elements to optimize both time and space complexity.

### Solution Code
```cpp
#include <bits/stdc++.h>
using namespace std;

const int MAXN = 2e5 + 5;
vector<int> parent(MAXN);
vector<vector<int>> top10(MAXN);

int find(int u) {
    if (parent[u] != u) {
        parent[u] = find(parent[u]);
    }
    return parent[u];
}

void unite(int u, int v) {
    u = find(u);
    v = find(v);
    if (u == v) return;
    if (top10[u].size() < top10[v].size()) {
        swap(u, v);
    }
    parent[v] = u;
    top10[u].insert(top10[u].end(), top10[v].begin(), top10[v].end());
    sort(top10[u].rbegin(), top10[u].rend());
    if (top10[u].size() > 10) {
        top10[u].resize(10);
    }
}

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);
    
    int N, Q;
    cin >> N >> Q;
    
    for (int i = 1; i <= N; ++i) {
        parent[i] = i;
        top10[i] = {i};
    }
    
    while (Q--) {
        int op, u, v;
        cin >> op >> u;
        if (op == 1) {
            cin >> v;
            unite(u, v);
        } else {
            int k;
            cin >> k;
            int root = find(u);
            if (top10[root].size() >= k) {
                cout << top10[root][k - 1] << '\n';
            } else {
                cout << "-1\n";
            }
        }
    }
    
    return 0;
}
```

### Explanation
1. **Union-Find Data Structure**: The `parent` array keeps track of the root parent of each node, and the `find` function uses path compression to efficiently find the root of any node.
2. **Maintaining Sorted Lists**: The `top10` array of vectors maintains the top 10 largest vertices in each connected component in descending order. This allows direct access to the k-th largest vertex in constant time if it exists.
3. **Efficient Merging**: The `unite` function merges two components by combining their `top10` lists. The larger list absorbs the smaller one, and the combined list is sorted and truncated to maintain only the top 10 elements. This ensures that the merging process is efficient and the stored lists remain manageable in size.
4. **Query Handling**: For each query of type 2, the code checks the size of the `top10` list of the connected component's root. If the size is at least `k`, it outputs the k-th largest vertex; otherwise, it outputs `-1`.

This approach ensures that the operations are handled efficiently, with the Union-Find structure providing near-constant time complexity for the find and union operations, and the sorted lists ensuring quick access to the required vertices during queries.
