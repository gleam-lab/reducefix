To fix this code, we need to ensure that the data structure correctly maintains the top 10 largest vertices in each connected component and efficiently merges these sets during union operations. The previous implementation had issues with incorrectly merging the sets and not handling the top 10 elements properly.

### Approaches
1. **Union-Find Data Structure**: This helps efficiently manage and merge connected components.
2. **Top 10 Elements**: For each component, maintain a sorted list (or a priority queue) of the top 10 largest vertices. During a union operation, merge these lists while keeping only the top 10 elements.
3. **Handling Queries**: For type 2 queries, directly access the k-th largest element from the stored top 10 list of the component.

### Solution Code
```cpp
#include <bits/stdc++.h>
using namespace std;

const int MAXN = 2e5 + 5;
int parent[MAXN];
vector<int> top10[MAXN];

int find(int u) {
    if (parent[u] != u) {
        parent[u] = find(parent[u]);
    }
    return parent[u];
}

void merge(int u, int v) {
    u = find(u);
    v = find(v);
    if (u == v) return;
    if (top10[u].size() < top10[v].size()) {
        swap(u, v);
    }
    vector<int> merged;
    merge(top10[u].begin(), top10[u].end(), top10[v].begin(), top10[v].end(), back_inserter(merged));
    sort(merged.rbegin(), merged.rend());
    if (merged.size() > 10) {
        merged.resize(10);
    }
    top10[u] = merged;
    parent[v] = u;
}

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);
    int N, Q;
    cin >> N >> Q;
    for (int i = 1; i <= N; ++i) {
        parent[i] = i;
        top10[i] = {i};
    }
    while (Q--) {
        int op, u, v;
        cin >> op >> u >> v;
        if (op == 1) {
            merge(u, v);
        } else {
            int root = find(u);
            if (top10[root].size() < v) {
                cout << "-1\n";
            } else {
                cout << top10[root][v - 1] << "\n";
            }
        }
    }
    return 0;
}
```

### Explanation
1. **Union-Find Initialization**: Each node is initially its own parent, and its top 10 list contains only itself.
2. **Merge Operation**: When merging two components, we combine their top 10 lists. The combined list is sorted in descending order and truncated to keep only the top 10 elements. The parent of the smaller component is updated to point to the larger component's parent.
3. **Query Handling**: For a type 2 query, we find the root of the component containing the queried vertex. If the component's top 10 list has at least `k` elements, the k-th largest is output; otherwise, `-1` is printed.

This approach ensures efficient handling of dynamic connectivity and top-k queries with optimal time complexity for the given constraints.
