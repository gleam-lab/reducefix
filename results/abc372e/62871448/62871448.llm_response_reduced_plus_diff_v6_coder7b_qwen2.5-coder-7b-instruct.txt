To solve this problem efficiently, we need to handle both adding edges and querying the k-th largest neighbor for each vertex. We can use a combination of union-find and a priority queue to manage the neighbors and their weights.

Here's the corrected code:

```cpp
#include<bits/stdc++.h>
using namespace std;

const int MAXN = 2 * 100005;
vector<int> adj[MAXN];
priority_queue<pair<int, int>, vector<pair<int, int>>, greater<pair<int, int>>> pq[MAXN];

void addEdge(int u, int v) {
    adj[u].push_back(v);
    adj[v].push_back(u);
}

void updateQueue(int u, int v) {
    pq[u].emplace(-v, v);
    pq[v].emplace(-u, u);
}

int kthLargest(int u, int k) {
    while (!pq[u].empty() && pq[u].top().second != u) {
        pq[u].pop();
    }
    if (pq[u].size() < k) return -1;
    return -pq[u].nth_element(k-1)->second;
}

int main() {
    ios::sync_with_stdio(false); cin.tie(nullptr);
    int n, q;
    cin >> n >> q;
    
    for (int i = 0; i < q; ++i) {
        int op, u, v, k;
        cin >> op;
        if (op == 1) {
            cin >> u >> v;
            addEdge(u, v);
            updateQueue(u, v);
        } else {
            cin >> u >> k;
            cout << kthLargest(u, k) << '\n';
        }
    }
    
    return 0;
}
```

### Explanation:

1. **Union-Find Data Structure**: While the union-find structure is used to manage connected components, it's not necessary for this problem since we don't need to merge sets of nodes based on edge connections.

2. **Priority Queue for Neighbors**:
   - For each vertex, maintain a min-priority queue (`pq`) that stores its neighbors along with their distances.
   - When adding an edge `(u, v)`, update both `pq[u]` and `pq[v]` to include the new neighbor.

3. **K-th Largest Neighbor Query**:
   - To find the k-th largest neighbor for a vertex `u`, ensure the top element of `pq[u]` is `u`.
   - Pop elements until you reach the k-th largest element. If there aren't enough elements, return `-1`.

4. **Main Function**:
   - Read the number of vertices `n` and queries `q`.
   - Process each query:
     - For type 1 queries, add the edge and update the priority queues.
     - For type 2 queries, output the k-th largest neighbor using the priority queue.

This approach ensures efficient handling of both operations with a time complexity of approximately O((Q + E) log N), where E is the number of edges. This is suitable within the given constraints.
