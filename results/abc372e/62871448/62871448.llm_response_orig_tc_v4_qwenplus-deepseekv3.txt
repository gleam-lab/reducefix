To fix this code, we need to accurately handle the union-find operations and maintain the top 10 largest elements in each connected component. The provided solution had several issues, including incorrect handling of the merging of top elements and incorrect initialization of the arrays.

### Approaches
1. **Union-Find Data Structure**: This will help efficiently manage the connected components of the graph. Each node will have a parent, and we will use path compression to optimize the find operation.
2. **Top-k Largest Elements**: For each connected component, we will maintain a list of the top 10 largest elements. When two components are merged, we will merge their top 10 lists and keep only the top 10 elements.
3. **Handling Queries**: For each query, we will either perform a union operation (Type 1) or retrieve the k-th largest element from the connected component (Type 2). If the component has fewer than k elements, we return -1.

### Solution Code
```cpp
#include <bits/stdc++.h>
using namespace std;

const int MAXN = 200005;
vector<int> parent(MAXN);
vector<vector<int>> top_elements(MAXN);

int find(int u) {
    if (parent[u] != u) {
        parent[u] = find(parent[u]);
    }
    return parent[u];
}

void unite(int u, int v) {
    u = find(u);
    v = find(v);
    if (u == v) return;
    if (top_elements[u].size() < top_elements[v].size()) {
        swap(u, v);
    }
    parent[v] = u;
    vector<int> merged;
    merged.reserve(top_elements[u].size() + top_elements[v].size());
    merge(top_elements[u].begin(), top_elements[u].end(),
          top_elements[v].begin(), top_elements[v].end(),
          back_inserter(merged), greater<int>());
    if (merged.size() > 10) {
        merged.resize(10);
    }
    top_elements[u] = merged;
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int N, Q;
    cin >> N >> Q;

    for (int i = 1; i <= N; ++i) {
        parent[i] = i;
        top_elements[i] = {i};
    }

    while (Q--) {
        int op, u, v;
        cin >> op >> u >> v;
        if (op == 1) {
            unite(u, v);
        } else {
            int root = find(u);
            if (top_elements[root].size() < v) {
                cout << "-1\n";
            } else {
                cout << top_elements[root][v - 1] << '\n';
            }
        }
    }

    return 0;
}
```

### Explanation
1. **Union-Find Data Structure**: The `find` function uses path compression to efficiently locate the root of a node. The `unite` function merges two components by size, ensuring the smaller component's top elements are merged into the larger one.
2. **Top-k Elements**: Each node's connected component maintains a list of its top 10 largest elements. When merging two components, their top elements are combined and sorted in descending order, then truncated to keep only the top 10.
3. **Query Handling**: For Type 1 queries, the `unite` function merges the components. For Type 2 queries, the k-th largest element is retrieved from the root's top elements list, or -1 if there are fewer than k elements.

This solution efficiently handles the constraints by leveraging union-find with path compression and maintaining sorted top elements lists, ensuring optimal performance for large inputs.
