To fix this code, we need to address the issue where the incorrect output was produced for a specific test case. The problem involves handling dynamic connectivity and efficiently querying the k-th largest vertex in a connected component. The initial approach used a Union-Find (Disjoint Set Union, DSU) data structure combined with sets to keep track of the vertices in each component. However, there was a mistake in how the sets were merged and how the k-th largest element was queried.

### Approaches
1. **Union-Find Data Structure**: This helps in efficiently managing the connected components and merging them when new edges are added.
2. **Balanced Binary Search Trees (BSTs)** using `std::set`: Each component's vertices are stored in a set to allow efficient insertion, deletion, and querying of the k-th largest element.
3. **Size-based Merging**: When merging two components, the smaller set is merged into the larger one to maintain efficiency (similar to the union by size heuristic in DSU).

### Solution Code
```cpp
#include <bits/stdc++.h>
using namespace std;
#define rep(i, n) for(int i = 0; i < (n); i++)
#define all(a) a.begin(), a.end()
#define rall(a) a.rbegin(), a.rend()
using ll = long long;
using pi = pair<int, int>;
const int INF = 1e9;

struct UnionFind {
    vector<int> parent, rank;
    vector<set<int>> elements;
    UnionFind(int n) : parent(n), rank(n, 0), elements(n) {
        for (int i = 0; i < n; i++) {
            parent[i] = i;
            elements[i].insert(i);
        }
    }
    int findRoot(int x) {
        if (parent[x] != x) 
            parent[x] = findRoot(parent[x]);
        return parent[x];
    }
    void unite(int x, int y) {
        int rx = findRoot(x), ry = findRoot(y);
        if (rx == ry) return;
        if (elements[rx].size() < elements[ry].size()) swap(rx, ry);
        parent[ry] = rx;
        elements[rx].insert(all(elements[ry]));
        elements[ry].clear();
    }
};

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);
    int n, q;
    cin >> n >> q;
    UnionFind uf(n);
    vector<int> ans;
    rep(qi, q) {
        int com;
        cin >> com;
        if (com == 1) {
            int u, v;
            cin >> u >> v;
            u--; v--;
            uf.unite(u, v);
        } else if (com == 2) {
            int v, k;
            cin >> v >> k;
            v--;
            int root = uf.findRoot(v);
            if ((int)uf.elements[root].size() < k) {
                ans.push_back(-1);
            } else {
                auto it = uf.elements[root].rbegin();
                advance(it, k - 1);
                ans.push_back(*it + 1);
            }
        }
    }
    for (auto x : ans) cout << x << "\n";
    return 0;
}
```

### Explanation
1. **Initialization**: The `UnionFind` structure is initialized with each vertex as its own parent and each vertex's set containing only itself.
2. **Handling Queries**:
   - **Type 1 (Add Edge)**: The `unite` function merges the sets of the two vertices involved in the edge. It uses size-based merging to ensure efficiency, merging the smaller set into the larger one.
   - **Type 2 (Query k-th Largest)**: The function checks if the connected component has at least `k` vertices. If not, it appends `-1` to the answer. Otherwise, it retrieves the k-th largest vertex by using a reverse iterator on the set (which is sorted in ascending order by default) and advances it to the appropriate position.
3. **Output**: The results of all Type 2 queries are printed in order.

This approach efficiently handles dynamic connectivity and queries with a time complexity that is nearly optimal for the given constraints, leveraging the union-find data structure and balanced BST operations.
