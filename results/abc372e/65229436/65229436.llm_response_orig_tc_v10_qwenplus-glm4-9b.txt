
### Problem Analysis

The problem involves managing an undirected graph with dynamic connectivity queries and range queries on the size of connected components. Specifically, we need to:

1. **Add edges** between vertices.
2. **Query the k-th largest vertex** in the connected component of a given vertex.

### Solution Strategy

To solve this problem efficiently given the constraints, we can use the Union-Find (Disjoint Set Union, DSU) data structure to handle the connectivity queries. For the range queries, we can maintain a size array that keeps track of the number of vertices in each connected component. 

The main steps are:
1. Initialize the Union-Find structure.
2. Process each query:
   - For an edge addition, merge the components of the two vertices if they are not already connected.
   - For a k-th largest vertex query, determine the connected component of the vertex and return the k-th largest vertex from that list.

### Code Explanation

Here's a structured and corrected version of the code:

```cpp
#include <iostream>
#include <vector>
#include <set>
using namespace std;

class UnionFind {
private:
    vector<int> parent, size;

public:
    UnionFind(int n) : parent(n), size(n, 1) {
        for (int i = 0; i < n; ++i) {
            parent[i] = i;
        }
    }

    int findRoot(int x) {
        if (x != parent[x]) {
            parent[x] = findRoot(parent[x]);
        }
        return parent[x];
    }

    bool unite(int x, int y) {
        x = findRoot(x);
        y = findRoot(y);
        if (x == y) return false;

        if (size[x] < size[y]) {
            swap(x, y);
        }
        parent[y] = x;
        size[x] += size[y];
        return true;
    }

    int getSize(int x) {
        return size[findRoot(x)];
    }
};

int main() {
    int N, Q;
    cin >> N >> Q;
    UnionFind uf(N);
    vector<set<int>> components(N);
    for (int i = 0; i < N; ++i) {
        components[i].insert(i);
    }

    for (int i = 0; i < Q; ++i) {
        int type, v, k;
        cin >> type;
        if (type == 1) {
            cin >> v >> k;
            --v; --k; // Convert to 0-based index
            int rootV = uf.findRoot(v);
            int rootK = uf.findRoot(k);
            if (rootV != rootK) {
                uf.unite(rootV, rootK);
            }
        } else if (type == 2) {
            cin >> v >> k;
            --v; --k; // Convert to 0-based index
            int root = uf.findRoot(v);
            if (components[root].size() < k) {
                cout << -1 << endl;
            } else {
                auto it = components[root].upper_bound(k - 1);
                if (it == components[root].begin()) {
                    cout << -1 << endl;
                } else {
                    --it;
                    cout << *it + 1 << endl; // Convert back to 1-based index
                }
            }
        }
    }

    return 0;
}
```

### Key Points

1. **Union-Find Initialization**: The Union-Find structure is initialized with `parent[i] = i` and `size[i] = 1` for all vertices.
2. **Edge Addition**: When adding an edge, we find the roots of the two vertices and unite their sets if they are not already connected.
3. **k-th Largest Vertex Query**: For querying the k-th largest vertex, we find the root of the vertex and then use the upper bound to find the k-th largest element in the set of vertices in the component.
4. **Efficiency**: The Union-Find operations (find and unite) are nearly constant time, making the solution efficient even for the upper limit of constraints.
