### Analysis of the Original Code

The original code attempts to solve the problem by using a Union-Find (Disjoint Set Union, DSU) data structure to manage connected components and a separate set for each component to maintain the vertices in sorted order. However, the code has several issues:

1. **Incorrect Output Handling**: The code incorrectly outputs `x+1` where `x` is the answer. While 0-based indexing is used internally, the problem expects 1-based indexing. The correction should be to output the value directly when it's positive and `-1` when the k-th largest vertex does not exist.

2. **Edge Case Handling**: The code incorrectly handles the case where the number of vertices in the connected component is less than `k` by pushing `-2` into the answer vector, which is later adjusted to `-1` (due to `x+1`). This should be directly handled by pushing `-1`.

3. **Efficiency Issues**: The code merges sets by iterating over all elements of the smaller set and inserting them into the larger set. While this approach is correct in theory, the constraints (N and Q up to 2×10^5) make this approach too slow due to its O(N log N) time complexity for each merge operation, leading to a total time complexity of O(Q N log N), which is not feasible.

### Optimized Approach

To handle the problem efficiently, we need to optimize the merging process. The key observation is that each query of type 2 only asks for the k-th largest vertex, with k up to 10. Therefore, we can maintain for each component only the top 10 largest vertices, instead of storing all vertices in a sorted set. This reduces the merging time to O(k), making the overall time complexity O(Q k), which is acceptable given the constraints.

#include <bits/stdc++.h>
using namespace std;
#define rep(i, n) for(int i=0;i<(n);i++)
#define all(a) a.begin(), a.end()
#define rall(a) a.rbegin(), a.rend()
using ll = long long;
using pi = pair<int, int>;
const int di[]={1,-1,0,0};
const int dj[]={0,0,1,-1};
const int INF=1e9;
struct UnionFind {
    vector<int> parent, rank;
    vector<vector<int>> top; // Stores up to 10 largest vertices in each component
    UnionFind(int n) : parent(n), rank(n, 0), top(n) {
        for (int i = 0; i < n; i++) {
            parent[i] = i;
            top[i].push_back(i);
        }
    }
    int findRoot(int x) {
        if (parent[x] != x) 
            parent[x] = findRoot(parent[x]);
        return parent[x];
    }
    void unite(int x, int y) {
        int rx = findRoot(x), ry = findRoot(y);
        if (rx == ry) return;
        if (rank[rx] < rank[ry]) {
            parent[rx] = ry;
            top[ry].insert(top[ry].end(), all(top[rx]));
            sort(rall(top[ry]));
            if (top[ry].size() > 10) top[ry].resize(10);
            top[rx].clear();
        } else {
            parent[ry] = rx;
            top[rx].insert(top[rx].end(), all(top[ry]));
            sort(rall(top[rx]));
            if (top[rx].size() > 10) top[rx].resize(10);
            top[ry].clear();
            if (rank[rx] == rank[ry])
                rank[rx]++;
        }
    }
};
int main(){
    ios::sync_with_stdio(false);
    cin.tie(nullptr);
    int n, q;
    cin >> n >> q;
    UnionFind uf(n);
    vector<int> ans;
    rep(qi, q) {
        int com;
        cin >> com;
        if(com==1) {
            int u, v;
            cin >> u >> v;
            u--; v--;
            uf.unite(u, v);
        }
        if(com==2) {
            int v, k;
            cin >> v >> k;
            v--;
            int root = uf.findRoot(v);
            if(uf.top[root].size() < k) {
                ans.push_back(-1);
            } else {
                ans.push_back(uf.top[root][k-1] + 1);
            }
        }
    }
    for(auto x:ans) cout << x << endl;
    return 0;
}

### Explanation

- **UnionFind Structure**: The Union-Find data structure is enhanced to store, for each component, the top 10 largest vertices (in descending order). This is done by maintaining a vector `top` where `top[i]` holds the largest vertices in the component of vertex `i`.

- **Merging Components**: When merging two components, the top vertices of both components are combined, sorted in descending order, and truncated to keep only the top 10. This ensures that each merge operation is O(k log k), which is efficient since k is at most 10.

- **Query Handling**: For type 2 queries, the code checks if the component has at least `k` vertices. If so, it retrieves the `k-1`-th index from the top list (since the list is sorted in descending order) and adds 1 to convert from 0-based to 1-based indexing. If there are fewer than `k` vertices, it directly outputs `-1`.

This approach ensures that the solution is both correct and efficient, handling the constraints effectively.
