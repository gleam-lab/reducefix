The problem arises from how the connected components' vertices are being tracked and merged. The original approach uses a `UnionFind` data structure combined with a `vector<set<int>>` to keep track of connected vertices. However, the merging logic was flawed, leading to incorrect tracking of vertices after union operations.

### Approaches
1. **UnionFind with Tracking**: Utilize the Union-Find (Disjoint Set Union, DSU) data structure to manage connected components efficiently.
2. **Balanced Merging**: When merging two sets, ensure the larger set absorbs the smaller one to maintain efficiency.
3. **Maintaining Sorted Elements**: Keep the vertices of each connected component in a sorted manner (using a set or a multiset) to efficiently access the k-th largest element when queried.

#include <bits/stdc++.h>

#define all(a) a.begin(), a.end()

struct UnionFind {
    vector<int> parent, rank;
    vector<multiset<int>> elements;
    UnionFind(int n) : parent(n), rank(n, 0), elements(n) {
        for (int i = 0; i < n; i++) {
            parent[i] = i;
            elements[i].insert(i);
        }
    }
    int findRoot(int x) {
        if (parent[x] != x) 
            parent[x] = findRoot(parent[x]);
        return parent[x];
    }
    void unite(int x, int y) {
        int rx = findRoot(x);
        int ry = findRoot(y);
        if (rx == ry) return;
        if (elements[rx].size() < elements[ry].size()) {
            swap(rx, ry);
        }
        parent[ry] = rx;
        elements[rx].insert(all(elements[ry]));
        elements[ry].clear();
    }
};
int main(){

### Explanation
1. **UnionFind Structure**: The `UnionFind` structure is enhanced to maintain a `multiset` of vertices for each component. This allows efficient merging and sorting of vertices.
2. **Unite Operation**: The `unite` method merges the smaller set into the larger one to maintain efficiency. After merging, the vertices of the smaller set are moved to the larger set and the smaller set is cleared.
3. **Query Handling**: For a Type 2 query, the root of the vertex is found, and the k-th largest vertex is retrieved by traversing the multiset in reverse order (descending). If the component size is smaller than k, `-1` is returned.
4. **Efficiency**: The use of balanced merging ensures each vertex is moved at most O(log n) times, keeping the overall complexity manageable for large inputs.

This approach efficiently handles the dynamic connectivity and k-th largest queries, ensuring correctness and optimal performance.
