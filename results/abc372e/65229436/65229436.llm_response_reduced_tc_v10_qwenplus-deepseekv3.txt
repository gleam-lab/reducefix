To fix this code, we need to ensure that the data structure used to maintain the connected components and their vertices is efficient and correctly updated when unions occur. The original code uses a Union-Find (Disjoint Set Union) structure combined with a vector of sets to keep track of vertices in each component. However, the issue arises when merging sets during the union operation, specifically in how the root is determined and the sets are merged.

### Approaches
1. **Union-Find Data Structure**: This helps efficiently manage the connected components and perform union operations.
2. **Balanced Tree for Each Component**: Each component's vertices are stored in a set (or a similar structure) to maintain sorted order, allowing efficient access to the k-th largest element.
3. **Merge by Size**: When merging two components, the smaller set is merged into the larger one to ensure efficiency.

### Solution Code
```cpp
#include <bits/stdc++.h>
using namespace std;
#define rep(i, n) for(int i=0;i<(n);i++)
#define all(a) a.begin(), a.end()
#define rall(a) a.rbegin(), a.rend()
using ll = long long;
using pi = pair<int, int>;
const int INF=1e9;
struct UnionFind {
    vector<int> parent;
    vector<multiset<int>> components;
    UnionFind(int n) : parent(n), components(n) {
        for (int i = 0; i < n; i++) {
            parent[i] = i;
            components[i].insert(i);
        }
    }
    int findRoot(int x) {
        if (parent[x] != x) 
            parent[x] = findRoot(parent[x]);
        return parent[x];
    }
    void unite(int x, int y) {
        int rx = findRoot(x);
        int ry = findRoot(y);
        if (rx == ry) return;
        if (components[rx].size() < components[ry].size()) {
            swap(rx, ry);
        }
        parent[ry] = rx;
        components[rx].insert(all(components[ry]));
        components[ry].clear();
    }
};
int main(){
    ios::sync_with_stdio(false);
    cin.tie(nullptr);
    int n, q;
    cin >> n >> q;
    UnionFind uf(n);
    vector<int> ans;
    rep(qi, q) {
        int com;
        cin >> com;
        if(com == 1) {
            int u, v;
            cin >> u >> v;
            u--; v--;
            uf.unite(u, v);
        }
        else if(com == 2) {
            int v, k;
            cin >> v >> k;
            v--;
            int root = uf.findRoot(v);
            if ((int)uf.components[root].size() < k) {
                ans.push_back(-1);
            }
            else {
                auto it = uf.components[root].end();
                advance(it, -k);
                ans.push_back(*it + 1);
            }
        }
    }
    for (auto x : ans) {
        cout << x << endl;
    }
    return 0;
}
```

### Explanation
1. **Union-Find Initialization**: The `UnionFind` structure is initialized with each vertex as its own parent and a component set containing only itself.
2. **Union Operation**: When vertices `u` and `v` are united, their roots are found. The smaller component's set is merged into the larger one's set, and the smaller component's set is cleared to save space.
3. **Query Handling**: For a query asking for the k-th largest vertex in a component, the root of the vertex is found, and then the k-th largest element is retrieved from the component's set. If the component size is less than `k`, `-1` is returned.
4. **Efficiency**: By merging smaller sets into larger ones and using a balanced tree (multiset) for each component, operations remain efficient (O(log N) per element merge and O(log N) for k-th element access).

This approach ensures that the solution is both correct and efficient, handling the constraints adequately.
