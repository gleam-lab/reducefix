### Approaches

1. **Union-Find Data Structure**: We use the Union-Find (Disjoint Set Union, DSU) data structure to efficiently manage and merge connected components. Each node starts as its own parent, and we use path compression and union by rank to optimize operations.

2. **Tracking Elements in Sets**: For each root in the Union-Find structure, we maintain a set (or a multiset) of elements in that component. When two components are merged, the smaller set is merged into the larger one to maintain efficiency.

3. **Query Handling**: For type 2 queries, we check the size of the connected component of the given vertex. If the size is less than `k`, we output `-1`. Otherwise, we find the k-th largest element in the set associated with the component's root.

struct UnionFind {
    vector<int> parent, rank;
    vector<set<int>> sets;
    UnionFind(int n) : parent(n), rank(n, 0), sets(n) {
        for (int i = 0; i < n; i++) {
            parent[i] = i;
            sets[i].insert(i);
        }
    }
    int findRoot(int x) {
        if (parent[x] != x) 
            parent[x] = findRoot(parent[x]);
        return parent[x];
    }
    bool unite(int x, int y) {
        int rx = findRoot(x), ry = findRoot(y);
        if (rx == ry) return false;
        if (rank[rx] < rank[ry]) {
            parent[rx] = ry;
            for (int num : sets[rx]) {
                sets[ry].insert(num);
            }
            sets[rx].clear();
        } else {
            parent[ry] = rx;
            for (int num : sets[ry]) {
                sets[rx].insert(num);
            }
            sets[ry].clear();
            if (rank[rx] == rank[ry])
                rank[rx]++;
        }
        return true;
    }
};

### Explanation

1. **Union-Find Initialization**: The `UnionFind` structure is initialized with each node as its own parent and a set containing only itself. This helps in tracking the elements of each component.

2. **Unite Operation**: When merging two components, the algorithm checks the roots of both nodes. If they are different, it merges the smaller set into the larger one to maintain efficiency. Path compression and union by rank are used to keep the operations nearly constant time.

3. **Query Handling**: For type 2 queries, the root of the given node is found. If the size of the component (set size) is less than `k`, `-1` is added to the answer. Otherwise, the k-th largest element is found by navigating the reverse iterator of the set and advancing it by `k-1` positions.

This approach ensures that the operations are efficient and correct, handling all edge cases and large constraints appropriately. The use of sets allows for efficient k-th largest element queries, while Union-Find ensures that component merges are handled optimally.
