To fix this code, we need to ensure that we correctly handle the operations of adding edges and querying the k-th largest vertex in the connected component of a given vertex. The provided code uses a Union-Find data structure with path compression and union by rank, but there was an issue in how the connected components' vertices were being managed and queried.

### Approaches
1. **Union-Find Data Structure**: Use Union-Find to manage the connected components. Each component will have a root, and we maintain a set of vertices for each root.
2. **Handling Queries**: 
   - For Type 1 queries (adding edges), we merge the sets of vertices of the connected components of the two vertices involved.
   - For Type 2 queries (querying the k-th largest vertex), we access the set of vertices of the connected component of the queried vertex and check if the size is at least k. If it is, we retrieve the k-th largest vertex; otherwise, return -1.

struct UnionFind {
    vector<int> parent, rank;
    vector<set<int>> vertices;
    UnionFind(int n) : parent(n), rank(n, 0), vertices(n) {
        for (int i = 0; i < n; i++) {
            parent[i] = i;
            vertices[i].insert(i);
        }
    }
    int findRoot(int x) {
        if (parent[x] != x) 
            parent[x] = findRoot(parent[x]);
        return parent[x];
    }
    bool unite(int x, int y) {
        int rx = findRoot(x), ry = findRoot(y);
        if (rx == ry) return false;
        if (rank[rx] < rank[ry]) {
            parent[rx] = ry;
            for (int v : vertices[rx]) 
                vertices[ry].insert(v);
            vertices[rx].clear();
        } else {
            parent[ry] = rx;
            for (int v : vertices[ry]) 
                vertices[rx].insert(v);
            vertices[ry].clear();
            if (rank[rx] == rank[ry])
                rank[rx]++;
        }
        return true;
    }
};

### Explanation
1. **Union-Find Initialization**: The Union-Find structure is initialized with each vertex as its own parent and a set containing only itself.
2. **Unite Operation**: When merging two sets, the vertices of the smaller set are merged into the larger set. This ensures efficient union operations with a time complexity of O(log n) per operation.
3. **Query Handling**: For Type 2 queries, the root of the queried vertex is found. The set of vertices in this root's component is then checked for size. If the size is sufficient, the k-th largest vertex is retrieved by moving an iterator to the appropriate position from the end of the set.

This approach efficiently handles the operations within the constraints, ensuring correct results for all test cases.
