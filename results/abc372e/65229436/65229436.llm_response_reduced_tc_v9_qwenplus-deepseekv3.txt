To fix this code, we need to address the primary issue causing the incorrect output. The original code uses a Union-Find (Disjoint Set Union, DSU) data structure with path compression and union by rank, but the way it handles the connected components' sets is flawed. Specifically, when merging two sets, it incorrectly assumes the root of the merged set will remain the same, which is not always true due to path compression in the DSU. 

### Approaches

1. **Union-Find Data Structure**: We use the Union-Find data structure to manage the connected components of the graph. This structure helps efficiently find the root of any vertex and merge components.
2. **Tracking Connected Components**: For each component, we maintain a set of vertices sorted in descending order. This allows us to quickly access the k-th largest vertex in the component.
3. **Efficient Merging**: When merging two components, we ensure the smaller set is merged into the larger one to maintain efficiency (similar to the union by size heuristic).
4. **Handling Queries**: For type 2 queries, we check if the component size is sufficient to have a k-th largest element. If it is, we retrieve it from the set; otherwise, we return -1.

### Solution Code

```cpp
#include <bits/stdc++.h>
using namespace std;
#define rep(i, n) for(int i=0;i<(n);i++)
#define all(a) a.begin(), a.end()
#define rall(a) a.rbegin(), a.rend()
using ll = long long;
using pi = pair<int, int>;
const int INF=1e9;

struct UnionFind {
    vector<int> parent;
    vector<multiset<int, greater<int>>> sets;

    UnionFind(int n) : parent(n), sets(n) {
        for (int i = 0; i < n; i++) {
            parent[i] = i;
            sets[i].insert(i);
        }
    }

    int findRoot(int x) {
        if (parent[x] != x)
            parent[x] = findRoot(parent[x]);
        return parent[x];
    }

    void unite(int x, int y) {
        int rx = findRoot(x);
        int ry = findRoot(y);
        if (rx == ry) return;

        if (sets[rx].size() < sets[ry].size()) {
            swap(rx, ry);
        }

        parent[ry] = rx;
        sets[rx].insert(all(sets[ry]));
        sets[ry].clear();
    }
};

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);
    int n, q;
    cin >> n >> q;
    UnionFind uf(n);
    vector<int> ans;

    rep(qi, q) {
        int com;
        cin >> com;
        if (com == 1) {
            int u, v;
            cin >> u >> v;
            u--; v--;
            uf.unite(u, v);
        } else if (com == 2) {
            int v, k;
            cin >> v >> k;
            v--;
            int root = uf.findRoot(v);
            if (uf.sets[root].size() < k) {
                ans.push_back(-1);
            } else {
                auto it = uf.sets[root].begin();
                advance(it, k - 1);
                ans.push_back(*it + 1);
            }
        }
    }

    for (auto x : ans) {
        cout << x << '\n';
    }

    return 0;
}
```

### Explanation

1. **Union-Find Initialization**: We initialize the Union-Find structure with each vertex as its own parent and a set containing only itself.
2. **Unite Operation**: When merging two components, we find the roots of both vertices. If they are different, we merge the smaller set into the larger one to maintain efficiency. The sets are merged by inserting all elements of the smaller set into the larger set.
3. **Type 2 Query Handling**: For a query asking for the k-th largest vertex in the component of `v`, we first find the root of `v`. If the component size is less than `k`, we return -1. Otherwise, we access the k-th largest element by leveraging the ordered set (sorted in descending order) and return the element after adjusting for 1-based indexing.
4. **Output**: The results of type 2 queries are collected in a vector and printed at the end.

This approach ensures efficient handling of both union and query operations, leveraging the Union-Find data structure and ordered sets for optimal performance.
