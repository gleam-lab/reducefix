I analyzed the code and found the main issue is with the handling of the connected components and the ordered sets. The original code attempts to use a policy-based data structure (ordered_set) to keep track of connected vertices, but it has several problems:

1. The Union-Find (Disjoint Set Union) implementation is correct, but the way it merges sets is inefficient for large N (up to 2×10^5).
2. The ordered_set approach isn't optimal for keeping track of connected vertices since it's too memory-intensive for large k values (up to 10).
3. The code inserts negative values to simulate descending order, which is unnecessary and confusing.

Here's the fixed version that:
- Uses standard Union-Find with path compression
- Maintains a separate set for each component's vertices
- Optimizes the merging process by always merging smaller sets into larger ones
- Uses vectors instead of ordered_set for better performance
- Limits the stored vertices to at most 10 (since k is up to 10)

```cpp
#include <iostream>
#include <vector>
#include <algorithm>
using namespace std;

const int N = 2e5 + 10;
int parent[N];
vector<int> vertices[N];

int Find(int u) {
    if (parent[u] != u) {
        parent[u] = Find(parent[u]);
    }
    return parent[u];
}

void Union(int u, int v) {
    u = Find(u);
    v = Find(v);
    if (u == v) return;
    
    if (vertices[u].size() < vertices[v].size()) {
        swap(u, v);
    }
    
    parent[v] = u;
    for (int x : vertices[v]) {
        vertices[u].push_back(x);
    }
    
    // Keep only top 10 elements to optimize memory
    sort(vertices[u].rbegin(), vertices[u].rend());
    if (vertices[u].size() > 10) {
        vertices[u].resize(10);
    }
}

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);
    
    int n, q;
    cin >> n >> q;
    
    for (int i = 1; i <= n; ++i) {
        parent[i] = i;
        vertices[i] = {i};
    }
    
    while (q--) {
        int cmd;
        cin >> cmd;
        if (cmd == 1) {
            int u, v;
            cin >> u >> v;
            Union(u, v);
        } else {
            int v, k;
            cin >> v >> k;
            v = Find(v);
            if (vertices[v].size() < k) {
                cout << "-1\n";
            } else {
                cout << vertices[v][k-1] << '\n';
            }
        }
    }
    
    return 0;
}
```

Key improvements:
1. Simplified Union-Find implementation
2. Each component maintains its own sorted list of vertices
3. Only keeps up to 10 largest vertices per component since k ≤ 10
4. Uses standard vectors instead of policy-based data structures
5. Merges smaller sets into larger ones for efficiency
6. Proper sorting in descending order for quick k-th largest access

This solution should handle the constraints efficiently (O(N α(N)) for Union-Find operations and O(k log k) for sorting small sets during merges).
