To solve this problem correctly and efficiently, we need to handle dynamic connectivity queries on a graph using the **Disjoint Set Union (DSU)** data structure (also known as Union-Find). Additionally, for each query of type 2 (`v k`), we must return the `k-th largest` vertex in the connected component that includes `v`.

The issue with your original code is that during the union operation, it does not maintain the correct order or size when merging sets, which leads to incorrect results for the `k-th largest` query.

---

### Key Improvements:
1. **Use a map (or sorted list) per component to store vertices in descending order**.
2. **Path compression and size-based union** for efficient DSU operations.
3. Maintain a mapping from node to its root and from root to the sorted list of nodes in its component.
4. When performing union, merge the smaller set into the larger one to keep time complexity close to `O(log N)` per operation.

Here's the fixed and optimized version:

```cpp
#include <bits/stdc++.h>
using namespace std;

typedef long long ll;
typedef vector<int> vi;

const int MAXN = 2 * 100000 + 5;

int parent[MAXN];
set<int> components[MAXN]; // Each root has its own set of nodes in decreasing order

int find(int u) {
    if (parent[u] != u)
        parent[u] = find(parent[u]);
    return parent[u];
}

void unite(int u, int v) {
    int root_u = find(u);
    int root_v = find(v);

    if (root_u == root_v)
        return;

    // Always merge smaller set into larger set
    if (components[root_u].size() < components[root_v].size()) {
        swap(root_u, root_v);
    }

    // Merge component_v into component_u
    for (int x : components[root_v]) {
        components[root_u].insert(x);
    }
    components[root_v].clear();
    parent[root_v] = root_u;
}

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);

    int N, Q;
    cin >> N >> Q;

    // Initialize DSU
    for (int i = 1; i <= N; ++i) {
        parent[i] = i;
        components[i].insert(-i); // Store negative values for descending order
    }

    while (Q--) {
        int type;
        cin >> type;
        if (type == 1) {
            int u, v;
            cin >> u >> v;
            unite(u, v);
        } else {
            int v, k;
            cin >> v >> k;
            int root = find(v);
            if ((int)components[root].size() < k) {
                cout << -1 << "\n";
            } else {
                auto it = components[root].begin();
                advance(it, k - 1);
                cout << -(*it) << "\n"; // negate back to get actual value
            }
        }
    }

    return 0;
}
```

---

### Explanation:

- **DSU (Union-Find)**: Efficiently groups nodes into connected components.
- **Set of Negatives**: Since C++'s `std::set` gives ascending order by default, storing negatives allows us to treat it like a descending-order container.
- **Efficient Merging**: We always merge the smaller component into the larger one to ensure near-logarithmic performance per union.

This approach ensures correctness and efficiency even under tight constraints (`N, Q ≤ 2 * 10^5`).

---

### Time Complexity:
- Union: O(α(N) * log K), where α is the inverse Ackermann function and K is the size of the smaller set.
- Query: O(log K)
