To solve this problem, we need to efficiently manage the addition of edges and querying the k-th largest element among the neighbors of a specific node. The key challenge here is to handle these operations within the constraints using appropriate data structures.

### Approach
1. **Union-Find Data Structure**: This will help in managing connected components and merging them when new edges are added.
2. **Ordered Set**: For each connected component, we maintain an ordered set to store the vertex numbers. This allows us to quickly find the k-th largest element.

### Detailed Steps
1. **Initialize Union-Find**: Each vertex starts as its own parent.
2. **Union Operation**: When adding an edge, merge the sets of the two connected components.
3. **Query Operation**: For a given vertex, find its root (representative of the connected component) and use the ordered set to find the k-th largest element.

### Implementation
We'll use a combination of Union-Find and ordered sets to achieve the desired functionality.

```cpp
#include <iostream>
#include <vector>
#include <set>
#include <algorithm>
#include <cassert>

using namespace std;

const int MAXN = 2e5 + 10;

int parent[MAXN];
set<int> adjacencySets[MAXN];

int findSet(int u) {
    if (parent[u] == u) return u;
    return parent[u] = findSet(parent[u]);
}

void unionSets(int u, int v) {
    u = findSet(u);
    v = findSet(v);
    if (u != v) {
        if (adjacencySets[u].size() > adjacencySets[v].size()) swap(u, v);
        for (int neighbor : adjacencySets[u]) {
            adjacencySets[v].insert(neighbor);
        }
        parent[u] = v;
    }
}

int kthLargest(int u, int k) {
    u = findSet(u);
    if (k <= adjacencySets[u].size()) {
        auto it = adjacencySets[u].rbegin();
        advance(it, k - 1);
        return *it;
    } else {
        return -1;
    }
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);
    
    int n, q;
    cin >> n >> q;
    
    for (int i = 1; i <= n; ++i) {
        parent[i] = i;
    }
    
    while (q--) {
        int type;
        cin >> type;
        
        if (type == 1) {
            int u, v;
            cin >> u >> v;
            unionSets(u, v);
        } else if (type == 2) {
            int u, k;
            cin >> u >> k;
            cout << kthLargest(u, k) << "\n";
        }
    }
    
    return 0;
}
```

### Explanation
1. **Initialization**:
   - `parent` array is initialized such that each node is its own parent initially.
   
2. **Find Operation**:
   - Uses path compression to keep the tree flat.

3. **Union Operation**:
   - Merges the smaller set into the larger set to keep the overall height of the tree low.
   - Updates the parent pointers accordingly.

4. **Kth Largest Query**:
   - Finds the representative of the set using `findSet`.
   - Checks if k is within the bounds of the set size.
   - Returns the k-th largest element if valid, otherwise returns -1.

This approach ensures efficient management of connected components and quick queries, making it suitable for large inputs up to the specified limits.
