### Solution
Instead of storing all vertices, maintain a priority queue (min-heap) for each component that stores the vertices in increasing order. When processing a query, simply pop from the heap until we get the k-th largest vertex.

### Solution
When adding an edge, check if the vertices belong to the same set. If they do, no action is needed. If they don't, perform a union operation only if the sets are not already connected.
