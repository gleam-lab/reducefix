To fix this code, we need to address several issues related to the incorrect handling of union operations and the management of the top k largest elements in each connected component. The problem requires efficiently maintaining the connected components of a graph and quickly querying the k-th largest vertex in a component.

### Approaches
1. **Union-Find Data Structure**: This helps efficiently manage the connected components and perform union operations.
2. **Balanced Binary Search Trees (BSTs)** (specifically `std::set`): Each component's root will maintain a set of its vertices to quickly access the k-th largest element.
3. **Handling Union Operations**: When two components are merged, we combine their sets and keep only the top 10 elements to ensure efficient queries.

### Solution Code
```cpp
#include <bits/stdc++.h>
using namespace std;

const int MAXN = 2e5 + 5;
int parent[MAXN];
set<int, greater<int>> components[MAXN];

int find(int u) {
    if (parent[u] != u) {
        parent[u] = find(parent[u]);
    }
    return parent[u];
}

void unite(int u, int v) {
    int root_u = find(u);
    int root_v = find(v);
    if (root_u == root_v) return;
    
    if (components[root_u].size() < components[root_v].size()) {
        swap(root_u, root_v);
    }
    
    for (int x : components[root_v]) {
        components[root_u].insert(x);
        if (components[root_u].size() > 10) {
            components[root_u].erase(prev(components[root_u].end()));
        }
    }
    parent[root_v] = root_u;
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);
    
    int N, Q;
    cin >> N >> Q;
    
    for (int i = 1; i <= N; ++i) {
        parent[i] = i;
        components[i].insert(i);
    }
    
    while (Q--) {
        int op, u, v;
        cin >> op >> u >> v;
        if (op == 1) {
            unite(u, v);
        } else {
            int root = find(u);
            if (components[root].size() < v) {
                cout << "-1\n";
            } else {
                auto it = components[root].begin();
                advance(it, v - 1);
                cout << *it << '\n';
            }
        }
    }
    
    return 0;
}
```

### Explanation
1. **Union-Find Data Structure**: The `parent` array keeps track of the root parent of each node, enabling efficient union and find operations with path compression.
2. **Component Management**: Each root node maintains a set (`std::set`) of its vertices sorted in descending order. This allows quick access to the k-th largest element.
3. **Union Operation**: When merging two components (`unite`), the smaller set is merged into the larger one to optimize performance. The merged set retains only the top 10 elements to ensure queries remain efficient.
4. **Query Handling**: For each query of type 2, the root of the component is found. If the component has at least `v` elements, the k-th largest element is retrieved; otherwise, `-1` is returned.

This approach ensures efficient handling of union operations and queries, adhering to the problem constraints and providing correct results for all test cases.
