To fix this code, we need to properly handle the union-find data structure and maintain the top 10 largest elements in each connected component. The previous implementation tried to merge sorted lists but had several issues, including incorrect merging and not properly maintaining the top 10 elements.

### Approaches
1. **Union-Find with Path Compression**: This helps in efficiently finding the root of each node and merging sets.
2. **Maintaining Top 10 Elements**: For each root node, maintain a sorted list (in descending order) of the top 10 largest elements in its connected component. When merging two sets, combine their top 10 lists, sort them, and keep only the top 10 elements.

### Solution Code
```cpp
#include<bits/stdc++.h>
using namespace std;

const int MAXN = 200005;

vector<int> parent(MAXN);
vector<vector<int>> top(MAXN);

int find(int u) {
    if (parent[u] != u) {
        parent[u] = find(parent[u]);
    }
    return parent[u];
}

void merge(int u, int v) {
    int rootU = find(u);
    int rootV = find(v);
    if (rootU == rootV) return;

    if (top[rootU].size() < top[rootV].size()) {
        swap(rootU, rootV);
    }

    parent[rootV] = rootU;
    top[rootU].insert(top[rootU].end(), top[rootV].begin(), top[rootV].end());
    sort(top[rootU].rbegin(), top[rootU].rend());
    if (top[rootU].size() > 10) {
        top[rootU].resize(10);
    }
}

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);

    int N, Q;
    cin >> N >> Q;

    for (int i = 1; i <= N; ++i) {
        parent[i] = i;
        top[i] = {i};
    }

    while (Q--) {
        int op, u, v;
        cin >> op >> u >> v;
        if (op == 1) {
            merge(u, v);
        } else {
            int root = find(u);
            if (v > top[root].size()) {
                cout << "-1\n";
            } else {
                cout << top[root][v - 1] << "\n";
            }
        }
    }

    return 0;
}
```

### Explanation
1. **Initialization**: We initialize each node as its own parent and set its top list to contain only itself.
2. **Find Operation**: This operation uses path compression to efficiently find the root of a node.
3. **Merge Operation**: When merging two sets, we combine their top 10 lists, sort them in descending order, and keep only the top 10 elements. This ensures that each root's top list always contains the largest elements in its connected component.
4. **Query Handling**: For each query of type 2, we check if the requested element exists in the top list of the connected component. If it does, we return the element; otherwise, we return -1.

This approach ensures that we efficiently handle both union and find operations while maintaining the top 10 elements in each connected component, providing the correct results for all queries.
