
This problem is asking you to manage an undirected graph with dynamic edge additions and queries about the vertices connected to certain nodes.

Here's a breakdown of the problem:

1. **Graph Initialization**: Start with a graph with N nodes and no edges. Nodes are labeled from 1 to N.

2. **Dynamic Edge Additions**: Process a series of queries where you either add an edge between two nodes or perform a query on the neighbors of a node.

3. **Edge Addition Query**: For a query of type 1 (`1 u v`), add an edge between nodes u and v. If u and v are already connected, no action is taken. This would mean they're in the same connected component.

4. **Neighborhood Query**: For a query of type 2 (`2 v k`), find the k-th largest connected vertex to node v. If there are fewer than k vertices connected to v, print `-1`.

### Key Points for the Solution:

1. **Union-Find Data Structure**: Use the union-find (or disjoint set) data structure to manage and query connected components efficiently.

2. **Path Compression**: Implement path compression for the union-find structure to ensure that operations on nodes look up the root in logarithmic time.

3. **Union by Rank**: Always attach the smaller tree under the root of the larger tree during a union operation to keep the tree shallow.

4. **Tracking Neighbors**: When adding an edge, update the lists of neighbors for each connected component. Sorting these lists allows fast retrieval of the k-th largest element.

5. **Efficient Queries**: Maintain a sorted list of vertices for each connected component and update this list with new vertices when an edge is added.

### Example Solution Steps:

1. Initialize a union-find structure for N nodes.

2. For each query:
   - If it's an edge addition:
     - Use find operations to check if u and v are in the same component.
     - If not, union them and update the neighbors of each component.
   - If it's a neighborhood query:
     - Find the component containing v.
     - Retrieve the sorted list of neighbors.
     - Output the k-th largest element, or `-1` if there are fewer than k elements.
