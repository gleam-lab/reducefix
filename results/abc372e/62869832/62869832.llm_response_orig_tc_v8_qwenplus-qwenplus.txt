To solve this problem efficiently, we need to handle two types of queries on an initially disconnected undirected graph:

1. **Union Operation (Type 1)**: Connect two nodes `u` and `v`.
2. **Query Operation (Type 2)**: Retrieve the k-th largest node in the connected component containing node `v`.

### Key Requirements:
- Efficiently manage dynamic connectivity between nodes.
- Quickly retrieve top-k largest elements from a connected component.

### Optimal Data Structure:
We'll use **Disjoint Set Union (DSU / Union-Find)** with path compression for efficient union and find operations. Additionally, each set will maintain a sorted list (in descending order) of its elements using a `multiset<int>` or a vector that is kept sorted.

This ensures:
- Fast merging of sets (`O(α(N))`, where α is the inverse Ackermann function).
- Efficient retrieval of the k-th largest element (`O(log N)` per query due to sorted structure).

### Strategy:
- Each DSU root maintains a sorted multiset of all nodes in its set.
- When uniting two components, merge the smaller set into the larger one to keep complexity low (union by size/rank).
- For Type 2 queries, access the k-th largest value directly from the multiset (or vector) maintained at the root.

Here's the complete working solution:

```cpp
#include <bits/stdc++.h>
using namespace std;

const int MAXN = 2e5 + 5;

int parent[MAXN];
set<int, greater<int>> components[MAXN];

int find(int u) {
    if (parent[u] != u)
        parent[u] = find(parent[u]);
    return parent[u];
}

void unite(int u, int v) {
    int rootU = find(u);
    int rootV = find(v);
    if (rootU == rootV) return;

    // Always merge smaller into larger to keep time complexity low
    if (components[rootU].size() < components[rootV].size()) {
        swap(rootU, rootV);
    }

    // Merge the smaller set into the larger set
    for (int val : components[rootV]) {
        components[rootU].insert(val);
    }
    components[rootV].clear();
    parent[rootV] = rootU;
}

int kthLargest(set<int, greater<int>>& s, int k) {
    auto it = s.begin();
    for (int i = 0; i < k - 1 && it != s.end(); ++i) {
        ++it;
    }
    return (it != s.end()) ? *it : -1;
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int N, Q;
    cin >> N >> Q;

    // Initialize DSU
    for (int i = 1; i <= N; ++i) {
        parent[i] = i;
        components[i].insert(i);
    }

    while (Q--) {
        int type;
        cin >> type;
        if (type == 1) {
            int u, v;
            cin >> u >> v;
            unite(u, v);
        } else if (type == 2) {
            int v, k;
            cin >> v >> k;
            int root = find(v);
            cout << kthLargest(components[root], k) << '\n';
        }
    }

    return 0;
}
```

### Why This Works:
- The union-by-size heuristic ensures that the cost of merging is proportional to the size of the smaller set, leading to amortized logarithmic performance.
- Using `set<int, greater<int>>` maintains a sorted collection allowing O(log n) insertion and k-th element access.
- Path compression in DSU keeps tree heights nearly constant, making find operations effectively O(1).

### Complexity:
- Each operation is `O(α(N))` for DSU and `O(k)` worst-case for k-th element access, but since `k ≤ 10`, this is negligible.
- Total time complexity remains acceptable even for maximum constraints.
