To solve this problem efficiently, we need to handle both adding edges and querying the k-th largest vertex connected to a specific vertex. The key challenge is to maintain connectivity information and efficiently find the k-th largest element in a set of connected vertices.

Here's a revised approach:

1. **Union-Find Data Structure**: This will help manage the connected components efficiently.
2. **Sorted Set for Each Component**: Maintain a sorted set for each component to quickly find the k-th largest element.

### Steps to Fix the Code

1. **Union-Find Initialization**: Initialize parent arrays and size arrays for union-find operations.
2. **Union Operation**: Merge components using union-find.
3. **Add Edge**: When adding an edge, merge the components containing the two endpoints and update the sorted sets accordingly.
4. **Query Operation**: For each query, find the root of the component and return the k-th largest element if it exists.

Here's the corrected code:

```cpp
#include<bits/stdc++.h>
using namespace std;

const int MAXN = 200005;
vector<int> parent(MAXN), rank(MAXN), sortedSet[MAXN];

int find(int x) {
    if (parent[x] != x) {
        parent[x] = find(parent[x]);
    }
    return parent[x];
}

void unite(int x, int y) {
    int px = find(x), py = find(y);
    if (px == py) return;
    
    if (rank[px] > rank[py]) {
        parent[py] = px;
        sortedSet[px].insert(sortedSet[py].begin(), sortedSet[py].end());
        sortedSet[py].clear();
    } else {
        parent[px] = py;
        sortedSet[py].insert(sortedSet[px].begin(), sortedSet[px].end());
        sortedSet[px].clear();
        
        if (rank[px] == rank[py]) {
            rank[py]++;
        }
    }
}

int main() {
    ios_base::sync_with_stdio(false); cin.tie(NULL); cout.tie(NULL);

    int n, q;
    cin >> n >> q;

    for (int i = 1; i <= n; ++i) {
        parent[i] = i;
        rank[i] = 0;
        sortedSet[i].push_back(i);
    }

    while (q--) {
        int type, u, v, k;
        cin >> type;

        if (type == 1) {
            cin >> u >> v;
            unite(u, v);
        } else {
            cin >> u >> k;
            int root = find(u);
            if (sortedSet[root].size() >= k) {
                cout << *next(sortedSet[root].rbegin(), k - 1) << "\n";
            } else {
                cout << "-1\n";
            }
        }
    }

    return 0;
}
```

### Explanation

1. **Initialization**:
   - `parent` array stores the parent of each node.
   - `rank` array stores the depth of the tree rooted at each node.
   - `sortedSet` array stores the sorted list of nodes in each connected component.

2. **Find Function**:
   - Uses path compression to find the root of a node.

3. **Unite Function**:
   - Merges two components based on their ranks and updates the sorted sets.

4. **Main Loop**:
   - Reads each query and processes it according to its type.
   - For type 1 queries, it adds an edge and merges the components.
   - For type 2 queries, it finds the k-th largest node in the connected component of the queried node.

This solution ensures that both union and find operations are efficient, maintaining the overall time complexity as close to O(N log N) as possible.
