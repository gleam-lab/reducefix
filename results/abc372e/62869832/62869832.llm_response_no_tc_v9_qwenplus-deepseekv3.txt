To fix this code, we need to address several issues related to handling connected components and efficiently maintaining the top k largest vertices in each component. The original code attempts to use a Union-Find data structure with a sorted array for each component, but it has several flaws such as incorrect merging logic and inefficient sorting operations.

### Approaches
1. **Union-Find with Path Compression**: This helps in efficiently finding the root of each component.
2. **Maintaining Sorted Lists**: For each component, we maintain a sorted list of the top 10 largest vertices. This is crucial for efficiently answering the k-th largest queries.
3. **Efficient Merging**: When merging two components, we merge their sorted lists and keep only the top 10 elements to ensure operations remain efficient.

### Solution Code
```cpp
#include <bits/stdc++.h>
using namespace std;

const int MAXN = 200005;

vector<int> components[MAXN];
int parent[MAXN];

int find(int u) {
    if (parent[u] != u) {
        parent[u] = find(parent[u]);
    }
    return parent[u];
}

void merge(int u, int v) {
    u = find(u);
    v = find(v);
    if (u == v) return;
    if (components[u].size() < components[v].size()) {
        swap(u, v);
    }
    components[u].insert(components[u].end(), components[v].begin(), components[v].end());
    sort(components[u].begin(), components[u].end(), greater<int>());
    if (components[u].size() > 10) {
        components[u].resize(10);
    }
    parent[v] = u;
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int N, Q;
    cin >> N >> Q;

    for (int i = 1; i <= N; ++i) {
        parent[i] = i;
        components[i] = {i};
    }

    while (Q--) {
        int op, u, v;
        cin >> op >> u >> v;
        if (op == 1) {
            merge(u, v);
        } else {
            int root = find(u);
            if (v > components[root].size()) {
                cout << -1 << '\n';
            } else {
                cout << components[root][v - 1] << '\n';
            }
        }
    }

    return 0;
}
```

### Explanation
1. **Initialization**: We initialize each vertex as its own parent and its own component containing only itself.
2. **Union-Find with Path Compression**: The `find` function uses path compression to efficiently find the root of a vertex, which represents the component it belongs to.
3. **Merging Components**: The `merge` function combines two components by merging their sorted lists of vertices. It ensures the merged list is sorted in descending order and truncated to the top 10 elements to maintain efficiency.
4. **Handling Queries**: For a Type 2 query, we find the root of the given vertex and check if the component size is sufficient to answer the k-th largest query. If not, we return -1; otherwise, we return the k-th largest vertex from the sorted list.

This approach ensures that each merge operation and query is handled efficiently, making it suitable for large input sizes as specified in the problem constraints.
