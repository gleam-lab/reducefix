To fix this code, we need to ensure that the Union-Find data structure correctly maintains the connected components and that the sets of vertices are properly merged when unions occur. The original code attempts to maintain a set of vertices for each root, but it incorrectly handles the merging process, leading to potential errors in query responses.

### Approaches

1. **Union-Find Data Structure**: This helps efficiently manage and query the connected components of the graph.
2. **Maintaining Sorted Lists**: For each root in the Union-Find structure, maintain a sorted list (or a multiset) of vertex numbers in its component. This allows efficient access to the k-th largest element.
3. **Efficient Merging**: When two components are merged, the smaller set is merged into the larger one to maintain efficiency (often referred to as the "smaller to larger" strategy).

### Solution Code

```cpp
#include <bits/stdc++.h>
using namespace std;
#define rep(i, n) for(int i = 0; i < (n); i++)
#define all(a) a.begin(), a.end()
#define rall(a) a.rbegin(), a.rend()
using ll = long long;
using pi = pair<int, int>;
const int INF = 1e9;
struct UnionFind {
    vector<int> parent;
    vector<int> rank;
    vector<multiset<int>> vertices;
    UnionFind(int n) : parent(n), rank(n, 0), vertices(n) {
        for (int i = 0; i < n; i++) {
            parent[i] = i;
            vertices[i].insert(i);
        }
    }
    int findRoot(int x) {
        if (parent[x] != x) 
            parent[x] = findRoot(parent[x]);
        return parent[x];
    }
    bool unite(int x, int y) {
        int rx = findRoot(x);
        int ry = findRoot(y);
        if (rx == ry) return false;
        if (rank[rx] < rank[ry]) {
            parent[rx] = ry;
            vertices[ry].insert(all(vertices[rx]));
            vertices[rx].clear();
        } else {
            parent[ry] = rx;
            vertices[rx].insert(all(vertices[ry]));
            vertices[ry].clear();
            if (rank[rx] == rank[ry])
                rank[rx]++;
        }
        return true;
    }
};
int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);
    int n, q;
    cin >> n >> q;
    UnionFind uf(n);
    vector<int> ans;
    rep(qi, q) {
        int com;
        cin >> com;
        if (com == 1) {
            int u, v;
            cin >> u >> v;
            u--; v--;
            uf.unite(u, v);
        }
        if (com == 2) {
            int v, k;
            cin >> v >> k;
            v--;
            int root = uf.findRoot(v);
            if ((int)uf.vertices[root].size() < k) {
                ans.push_back(-1);
            } else {
                auto it = uf.vertices[root].rbegin();
                advance(it, k - 1);
                ans.push_back(*it + 1);
            }
        }
    }
    for (auto x : ans) cout << x << endl;
    return 0;
}
```

### Explanation

1. **Union-Find Structure**: The `UnionFind` class is initialized with `parent`, `rank`, and `vertices` arrays. The `parent` array keeps track of the root parent of each vertex, `rank` helps in union operations to keep the tree flat, and `vertices` is a list of multisets that store the vertices in each component.
2. **Union Operation**: The `unite` method merges two components. It uses the "smaller to larger" strategy to merge the smaller set into the larger one, ensuring efficiency. The vertices from the smaller component are inserted into the larger component's multiset.
3. **Query Handling**: For each type 2 query, the code checks the size of the component containing the queried vertex. If the size is smaller than `k`, it returns `-1`. Otherwise, it accesses the k-th largest element by traversing the multiset in reverse order.
4. **Efficiency**: The operations are efficient due to the use of Union-Find with path compression and union by rank, and the "smaller to larger" merging strategy ensures that each vertex is merged at most `log N` times, keeping the overall complexity manageable for large inputs.
