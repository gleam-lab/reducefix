To fix this code, we need to address the issue of efficiently maintaining and querying the k-th largest vertex in the connected component of a given vertex while handling dynamic edge additions. The original code uses a Union-Find (Disjoint Set Union, DSU) data structure along with a vector of sets to keep track of the vertices in each component. However, the approach has efficiency issues, especially with the high constraints (N and Q up to 2×10^5 and k up to 10).

### Approaches

1. **Union-Find with Small to Large Merging**: The key observation here is that when merging two sets, we always merge the smaller set into the larger one to ensure that each element is moved at most O(log N) times. This optimizes the merging process.
   
2. **Maintaining Sorted Elements**: For each root in the Union-Find structure, we maintain a sorted list (or a multiset) of vertex numbers in the component. This allows us to efficiently query the k-th largest element by accessing the elements in reverse order.

3. **Handling Queries Efficiently**: For each type 2 query, we first find the root of the vertex, then check if the size of the component is at least k. If it is, we access the k-th largest element by traversing the sorted list from the end.

### Solution Code

```cpp
#include <bits/stdc++.h>
using namespace std;
#define rep(i, n) for(int i = 0; i < (n); i++)
#define all(a) a.begin(), a.end()
#define rall(a) a.rbegin(), a.rend()
using ll = long long;
using pi = pair<int, int>;
const int INF = 1e9;
struct UnionFind {
    vector<int> parent, rank;
    vector<multiset<int>> components;
    UnionFind(int n) : parent(n), rank(n, 0), components(n) {
        for (int i = 0; i < n; i++) {
            parent[i] = i;
            components[i].insert(i);
        }
    }
    int findRoot(int x) {
        if (parent[x] != x) 
            parent[x] = findRoot(parent[x]);
        return parent[x];
    }
    bool unite(int x, int y) {
        int rx = findRoot(x), ry = findRoot(y);
        if (rx == ry) return false;
        if (components[rx].size() < components[ry].size()) {
            swap(rx, ry);
        }
        parent[ry] = rx;
        components[rx].insert(all(components[ry]));
        components[ry].clear();
        return true;
    }
};
int main(){
    ios::sync_with_stdio(false);
    cin.tie(nullptr);
    int n, q;
    cin >> n >> q;
    UnionFind uf(n);
    vector<int> ans;
    rep(qi, q) {
        int com;
        cin >> com;
        if(com == 1) {
            int u, v;
            cin >> u >> v;
            u--; v--;
            uf.unite(u, v);
        }
        if(com == 2) {
            int v, k;
            cin >> v >> k;
            v--;
            int root = uf.findRoot(v);
            if((int)uf.components[root].size() < k) {
                ans.push_back(-1);
            } else {
                auto it = uf.components[root].rbegin();
                advance(it, k - 1);
                ans.push_back(*it + 1);
            }
        }
    }
    for(auto x : ans) cout << x << endl;
    return 0;
}
```

### Explanation

1. **Union-Find Structure**: The Union-Find data structure keeps track of connected components. Each component is represented by its root, and the `findRoot` method uses path compression to ensure efficient lookups.

2. **Components Storage**: Each root maintains a multiset of vertices in its component. This allows efficient insertion and merging of components, leveraging the "small to large" merging strategy to ensure each vertex is moved O(log N) times across sets.

3. **Handling Queries**: For type 2 queries, the root of the vertex is found, and then the size of the component is checked. If the size is sufficient, the k-th largest vertex is retrieved by traversing the multiset in reverse order (from largest to smallest).

This approach ensures that both union operations and queries are handled efficiently, meeting the problem constraints effectively.
