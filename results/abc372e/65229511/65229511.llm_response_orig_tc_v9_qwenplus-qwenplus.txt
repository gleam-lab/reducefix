To fix the issue in your current code, we need to address a subtle bug in how connected components are maintained and queried.

### Problem Summary Recap:
You're given an undirected graph starting with zero edges. You process two types of queries:
- **Type 1**: Connect two nodes.
- **Type 2**: Ask for the `k`-th largest node number among all nodes connected to a given node.

This is a classic use case for the **Disjoint Set Union (DSU)** or **Union-Find** data structure with **additional information tracking**, specifically, the set of all nodes in each connected component.

---

### Issues in Your Code:

Your current implementation uses a `set<int>` per root to maintain the elements in the component. This allows efficient access to k-th largest using reverse iterators.

However, there's a **critical flaw** in this block:
```cpp
if(cnt[ru].size()<cnt[rv].size()) swap(ru, rv);
uf.unite(ru, rv);
int rnew=uf.findRoot(ru);
int rold=(rnew==ru ? rv:ru);
for (int x:cnt[rv]) cnt[rnew].insert(x);
```

Here’s what goes wrong:
- You call `unite(ru, rv)` which merges `ru` and `rv`, but the actual root returned by `findRoot()` afterward may not be `ru` or `rv` unless you carefully track it.
- The assignment `int rnew = uf.findRoot(ru);` is redundant since `uf.unite()` already does path compression.
- You insert into `cnt[rnew]` while possibly clearing from `cnt[rold]`. However, if `rnew` is not the same as the one used earlier due to internal DSU logic, then `cnt[rnew]` might be uninitialized or invalid.

---

### Fix Strategy:

1. **Always query the root after union**: Use `findRoot()` on either node post-union to get the true root.
2. **Merge smaller sets into larger ones**: To ensure logarithmic performance, always merge the smaller component into the larger one.
3. **Use ordered data structures**: We'll continue using `std::multiset` (or `std::set`) to allow fast access to k-th largest.

```cpp
#include <bits/stdc++.h>
using namespace std;

#define rep(i, n) for(int i = 0; i < (n); ++i)
#define all(v) v.begin(), v.end()
#define pb push_back
using ll = long long;
using pii = pair<int, int>;

struct UnionFind {
    vector<int> parent, rank;
    UnionFind(int n) : parent(n), rank(n, 0) {
        for (int i = 0; i < n; ++i) parent[i] = i;
    }
    int find(int x) {
        if (parent[x] != x) parent[x] = find(parent[x]);
        return parent[x];
    }
    bool unite(int x, int y) {
        int rx = find(x), ry = find(y);
        if (rx == ry) return false;
        if (rank[rx] < rank[ry]) swap(rx, ry);
        parent[ry] = rx;
        if (rank[rx] == rank[ry]) rank[rx]++;
        return true;
    }
};

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int N, Q;
    cin >> N >> Q;

    UnionFind uf(N);
    vector<multiset<int>> comp(N);

    for (int i = 0; i < N; ++i) {
        comp[i].insert(i);
    }

    vector<int> ans;

    while (Q--) {
        int type;
        cin >> type;

        if (type == 1) {
            int u, v;
            cin >> u >> v;
            u--; v--;

            int ru = uf.find(u), rv = uf.find(v);
            if (ru != rv) {
                if (comp[ru].size() < comp[rv].size()) swap(ru, rv);
                // Merge rv into ru
                for (int x : comp[rv]) {
                    comp[ru].insert(x);
                }
                comp[rv].clear();
                uf.parent[rv] = ru;
            }
        } else {
            int v, k;
            cin >> v >> k;
            v--;
            int root = uf.find(v);
            if (comp[root].size() < k) {
                ans.push_back(-1);
            } else {
                auto it = comp[root].rbegin();
                advance(it, k - 1);
                ans.push_back(*it + 1);
            }
        }
    }

    for (int x : ans) cout << x << '\n';
}
```

### Key Improvements:

- **Correct DSU management**: After `unite`, always use `find()` to get the current root.
- **Efficient merging**: Always merge the smaller component into the larger one to keep complexity near O(log N).
- **Safe access**: Using `rbegin()` ensures correct access to the k-th largest element in descending order.

---

### Time Complexity:

Each operation runs in **O(log N)** amortized time due to:
- DSU path compression.
- Merging small sets into large ones.
- Set operations (insert, size, rbegin, advance) are logarithmic in multiset size.
