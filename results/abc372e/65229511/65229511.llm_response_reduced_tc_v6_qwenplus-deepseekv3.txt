To fix this code, we need to address the issue where the k-th largest vertex number among the connected vertices was not being correctly identified. The original approach used a Union-Find data structure with a set to keep track of the vertices in each connected component, but there was a mistake in how the sets were merged, leading to incorrect results for some queries.

### Approaches
1. **Union-Find Data Structure**: This structure helps in efficiently managing and merging connected components.
2. **Balanced Merging**: When merging two sets, the smaller set is merged into the larger one to maintain efficiency.
3. **Handling Queries**: For Type 2 queries, we need to find the k-th largest vertex in the connected component of the given vertex. This involves accessing the sorted elements in descending order and checking if the component size is sufficient to have a k-th element.

### Solution Code
```cpp
#include <bits/stdc++.h>
using namespace std;
#define rep(i, n) for(int i=0;i<(n);i++)
#define all(a) a.begin(), a.end()
#define rall(a) a.rbegin(), a.rend()
using ll = long long;
using pi = pair<int, int>;
const int di[]={1,-1,0,0};
const int dj[]={0,0,1,-1};
const int INF=1e9;
struct UnionFind {
    vector<int> parent, rank;
    UnionFind(int n) : parent(n), rank(n, 0) {
        for (int i = 0; i < n; i++) 
            parent[i] = i;
    }
    int findRoot(int x) {
        if (parent[x] != x) 
            parent[x] = findRoot(parent[x]);
        return parent[x];
    }
    bool unite(int x, int y) {
        int rx = findRoot(x), ry = findRoot(y);
        if (rx == ry) return false;
        if (rank[rx] < rank[ry]) {
            parent[rx] = ry;
        } else {
            parent[ry] = rx;
            if (rank[rx] == rank[ry])
                rank[rx]++;
        }
        return true;
    }
};
int main(){
    ios::sync_with_stdio(false);
    cin.tie(nullptr);
    int n, q;
    cin >> n >> q;
    UnionFind uf(n);
    vector<set<int>> cnt(n);
    rep(i, n) cnt[i].insert(i+1);  // Storing 1-based indices
    vector<int> ans;
    rep(qi, q) {
        int com;
        cin >> com;
        if(com==1) {
            int u, v;
            cin >> u >> v;
            int ru = uf.findRoot(u-1);
            int rv = uf.findRoot(v-1);
            if(ru != rv) {
                if(cnt[ru].size() < cnt[rv].size()) {
                    swap(ru, rv);
                }
                uf.unite(u-1, v-1);
                for (int x : cnt[rv]) {
                    cnt[ru].insert(x);
                }
                cnt[rv].clear();
            }
        }
        if(com==2) {
            int v, k;
            cin >> v >> k;
            int root = uf.findRoot(v-1);
            if (cnt[root].size() < k) {
                ans.push_back(-1);
            } else {
                auto it = cnt[root].rbegin();
                advance(it, k-1);
                ans.push_back(*it);
            }
        }
    }
    for(auto x : ans) cout << x << endl;
    return 0;
}
```

### Explanation
1. **Initialization**: The `UnionFind` structure is initialized to manage connected components. Each vertex starts in its own component.
2. **Type 1 Query (Edge Addition)**:
   - The roots of the two vertices are found.
   - If the roots are different, the smaller set is merged into the larger one to maintain efficiency.
   - The set of vertices in the smaller component is merged into the larger component's set, and the smaller set is cleared.
3. **Type 2 Query (k-th Largest Vertex)**:
   - The root of the given vertex is determined.
   - The size of the root's set is checked to see if it contains at least `k` elements.
   - If it does, the k-th largest element is found by iterating from the end of the set (rbegin) and advancing by `k-1` positions.
   - The result is stored in the answer vector.
4. **Output**: The answers to Type 2 queries are printed in order.

This approach ensures that the operations are efficient and correct, handling all edge cases and large input sizes as specified in the constraints.
