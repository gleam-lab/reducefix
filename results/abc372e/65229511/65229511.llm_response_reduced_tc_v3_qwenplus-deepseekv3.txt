To fix this code, we need to ensure that the data structures and algorithms correctly handle the dynamic connectivity and k-th largest element queries efficiently. The primary issue in the original code was the incorrect merging of sets and the handling of the Union-Find data structure, leading to incorrect answers for certain queries.

### Approaches
1. **Union-Find with Path Compression**: This helps in efficiently finding the root of any element and merging sets.
2. **Balanced Tree Structures (Sets)**: Each root maintains a set of its connected vertices. When merging two sets, the smaller set is merged into the larger one to maintain efficiency.
3. **Handling Queries**: For Type 2 queries, we directly access the set of the root of the given vertex and find the k-th largest element by leveraging the ordered nature of the set.

### Solution Code
```cpp
#include <bits/stdc++.h>
using namespace std;
#define rep(i, n) for(int i = 0; i < (n); i++)
#define all(a) a.begin(), a.end()
#define rall(a) a.rbegin(), a.rend()
using ll = long long;
using pi = pair<int, int>;
const int INF = 1e9;

struct UnionFind {
    vector<int> parent;
    vector<set<int>> sets;
    UnionFind(int n) : parent(n), sets(n) {
        for (int i = 0; i < n; i++) {
            parent[i] = i;
            sets[i].insert(i);
        }
    }
    int findRoot(int x) {
        if (parent[x] != x) {
            parent[x] = findRoot(parent[x]);
        }
        return parent[x];
    }
    void unite(int x, int y) {
        int rx = findRoot(x);
        int ry = findRoot(y);
        if (rx == ry) return;
        if (sets[rx].size() < sets[ry].size()) {
            swap(rx, ry);
        }
        for (int elem : sets[ry]) {
            sets[rx].insert(elem);
        }
        sets[ry].clear();
        parent[ry] = rx;
    }
};

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);
    int n, q;
    cin >> n >> q;
    UnionFind uf(n);
    vector<int> ans;
    rep(qi, q) {
        int com;
        cin >> com;
        if (com == 1) {
            int u, v;
            cin >> u >> v;
            u--; v--;
            uf.unite(u, v);
        } else if (com == 2) {
            int v, k;
            cin >> v >> k;
            v--;
            int root = uf.findRoot(v);
            if (uf.sets[root].size() < k) {
                ans.push_back(-1);
            } else {
                auto it = uf.sets[root].rbegin();
                advance(it, k - 1);
                ans.push_back(*it + 1);
            }
        }
    }
    for (auto x : ans) cout << x << endl;
    return 0;
}
```

### Explanation
1. **Union-Find Data Structure**: This structure is used to efficiently manage and merge disjoint sets. Path compression is employed to keep the tree flat, ensuring efficient root finding.
2. **Sets for Connected Components**: Each root in the Union-Find structure maintains a set of vertices in its component. This set is kept ordered, allowing efficient access to the k-th largest element.
3. **Merging Sets**: When merging two sets, the smaller set is merged into the larger one to ensure that the operation is done in logarithmic time relative to the size of the sets.
4. **Handling Queries**: For Type 1 queries, the sets are merged. For Type 2 queries, the set of the root of the given vertex is accessed, and the k-th largest element is found by traversing the set in reverse order (since sets are ordered in ascending order).

This approach ensures that both the union and query operations are handled efficiently, adhering to the constraints and providing correct results for all test cases.
