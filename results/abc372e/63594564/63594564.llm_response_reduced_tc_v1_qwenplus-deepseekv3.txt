To fix this code, we need to ensure that when merging two sets, the merged set maintains the top 10 largest elements in sorted order. The previous implementation had a flaw where it was merging two sorted lists but not correctly maintaining the top 10 elements after merging, which led to incorrect results for certain queries.

### Approaches
1. **Union-Find Data Structure**: This will help in efficiently managing the connected components of the graph.
2. **Maintaining Sorted Lists**: For each component, we maintain a sorted list (in descending order) of the top 10 largest vertex numbers. This ensures that when we merge two components, we can efficiently combine their lists and keep only the top 10 elements.
3. **Merge Operation**: When merging two components, we merge their sorted lists and retain only the largest 10 elements to keep the operations efficient.

### Solution Code
```cpp
#include "bits/stdc++.h"

using namespace std;

#define ff first
#define ss second
#define ll long long
#define SZ(s) (int)s.size()

int const N = 2e5 + 5;

vector <vector <int>> v;

vector <int> p;

int par(int x) {
    if(p[x] == x) return x;
    return p[x] = par(p[x]);
}

void uni(int a, int b) {
    a = par(a), b = par(b);
    if(a == b) return;
    
    vector<int> merged;
    int i = 0, j = 0;
    while ((i < v[a].size() || j < v[b].size()) && merged.size() < 10) {
        if (j >= v[b].size() || (i < v[a].size() && v[a][i] >= v[b][j])) {
            merged.push_back(v[a][i]);
            i++;
        } else {
            merged.push_back(v[b][j]);
            j++;
        }
    }
    
    v[a] = merged;
    p[b] = a;
    return;
}

signed main() {
    ios_base::sync_with_stdio(false); cin.tie(nullptr);

    int n, q;
    cin >> n >> q;
    p.resize(n+1);
    v.resize(n+1);
    for(int i = 1; i <= n; i++) {
        p[i] = i;
        v[i].push_back(i);
    }
    while(q--) {
        int t, a, b;
        cin >> t >> a >> b;
        if(t == 1) {
            uni(a, b);
        }
        else {
            a = par(a);
            cout << (SZ(v[a]) < b ? -1 : v[a][b-1]) << '\n';
        }
    }

    return 0;
}
```

### Explanation
1. **Union-Find Initialization**: The `p` array is initialized to represent each vertex as its own parent, and each vertex initially has a list containing only itself in `v`.
2. **Merge Operation (uni function)**:
   - **Parent Check**: If the parents of `a` and `b` are the same, they are already in the same component, so we return early.
   - **Merging Lists**: The lists of the two components are merged by comparing the elements from both lists and selecting the larger elements until the merged list has 10 elements or all elements are exhausted.
   - **Parent Update**: The parent of `b` is set to `a` to merge the components.
3. **Query Handling**:
   - **Type 1 Query**: Calls the `uni` function to merge the components of `u` and `v`.
   - **Type 2 Query**: Retrieves the parent of `v` and checks if the size of the list is at least `k`. If so, it returns the `k`-th largest element (adjusted for 0-based indexing); otherwise, it returns `-1`.

This approach ensures that each merge operation is efficient (O(10) time per merge) and the queries are handled in constant time, making the solution suitable for large input sizes.
