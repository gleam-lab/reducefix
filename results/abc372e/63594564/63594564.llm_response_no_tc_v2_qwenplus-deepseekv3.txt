To fix this code, we need to address the inefficiency and incorrectness in handling the merging of sorted vectors during the union operation. The current approach merges two sorted vectors and then truncates the result to the top 10 elements, but it's inefficient and doesn't handle the merging correctly, especially for large inputs. Instead, we should merge the top 10 elements of both vectors directly to maintain efficiency.

### Approaches
1. **Union-Find Data Structure**: Use the union-find (disjoint set) data structure to manage connected components.
2. **Top K Elements**: For each connected component, maintain a sorted list of the largest 10 vertices (since k is at most 10). This allows efficient merging by only considering the top 10 elements from each component during a union operation.
3. **Efficient Merging**: When merging two components, merge their top 10 lists in sorted order and keep only the top 10 elements.

### Solution Code
```cpp
#include <bits/stdc++.h>

using namespace std;

#define ff first
#define ss second
#define ll long long
#define SZ(s) (int)s.size()

int const N = 2e5 + 5;

vector<vector<int>> v;

vector<int> p;

int par(int x) {
    if(p[x] == x) return x;
    return p[x] = par(p[x]);
}

void uni(int a, int b) {
    a = par(a), b = par(b);
    if(a == b) return;
    if(SZ(v[a]) < SZ(v[b])) swap(a, b);
    vector<int> merged;
    int i = 0, j = 0;
    while((i < SZ(v[a]) || j < SZ(v[b])) && SZ(merged) < 10) {
        if(j >= SZ(v[b]) || (i < SZ(v[a]) && v[a][i] >= v[b][j])) {
            merged.push_back(v[a][i]);
            i++;
        } else {
            merged.push_back(v[b][j]);
            j++;
        }
    }
    v[a] = merged;
    p[b] = a;
    return;
}

signed main() {
    ios_base::sync_with_stdio(false); cin.tie(nullptr);

    int n, q;
    cin >> n >> q;
    p.resize(n+1);
    v.resize(n+1);
    for(int i = 1; i <= n; i++) {
        p[i] = i;
        v[i] = {i};
    }
    while(q--) {
        int t, a, b;
        cin >> t >> a >> b;
        if(t == 1) {
            uni(a, b);
        }
        else {
            a = par(a);
            if(SZ(v[a]) < b) {
                cout << -1 << '\n';
            } else {
                cout << v[a][b-1] << '\n';
            }
        }
    }

    return 0;
}
```

### Explanation
1. **Initialization**: The vertices are initialized as individual components, each containing only themselves.
2. **Union Operation**: When merging two components, the code checks if they are already connected. If not, it merges the top 10 elements of both components into a new sorted list, ensuring that only the largest 10 elements are kept. This merging is done in linear time (O(10)), which is efficient.
3. **Query Operation**: For a query, it checks the size of the connected component of the vertex. If the component has at least `k` vertices, it returns the `k`-th largest vertex; otherwise, it returns `-1`.

This approach ensures that the operations are efficient and handle the constraints effectively, providing the correct results for each query.
