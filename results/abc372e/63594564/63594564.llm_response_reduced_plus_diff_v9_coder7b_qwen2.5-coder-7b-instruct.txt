To solve this problem, we need to handle two types of operations efficiently:
1. Adding an edge between two vertices.
2. Finding the k-th largest vertex connected to a specific vertex.

Given the constraints, using a simple adjacency list representation might not suffice due to potential performance issues with frequent updates and queries. Instead, we can use a combination of union-find data structure to manage connectivity and a sorted set to maintain the connected vertices in a sorted manner.

Here's the corrected and optimized C++ code:

```cpp
#include "bits/stdc++.h"
using namespace std;

const int MAXN = 2e5 + 5;
int parent[MAXN], rank[MAXN];
set<int> connections[MAXN];

int find(int x) {
    if (parent[x] != x) 
        parent[x] = find(parent[x]);
    return parent[x];
}

void unite(int x, int y) {
    x = find(x);
    y = find(y);
    if (x == y) return;
    
    // Always attach smaller tree under larger tree
    if (rank[x] < rank[y]) swap(x, y);
    connections[x].insert(connections[y].begin(), connections[y].end());
    connections[x].insert(y);
    parent[y] = x;
    if (rank[x] == rank[y]) rank[x]++;
}

int kthLargest(vector<int>& vec, int k) {
    if (k > vec.size()) return -1;
    auto it = prev(vec.end(), k);
    return *it;
}

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);

    int n, q;
    cin >> n >> q;
    
    for (int i = 1; i <= n; ++i) {
        parent[i] = i;
        rank[i] = 0;
    }

    while (q--) {
        int type, u, v;
        cin >> type >> u >> v;

        if (type == 1) {
            unite(u, v);
        } else {
            int root = find(u);
            if (connections[root].size() < v) {
                cout << "-1\n";
            } else {
                cout << kthLargest(connections[root], v) << "\n";
            }
        }
    }

    return 0;
}
```

### Explanation:
1. **Union-Find Data Structure**:
   - We use a union-find structure with path compression and union by rank to manage the connected components.
   
2. **Sorted Set for Connected Vertices**:
   - For each component, we maintain a set of its connected vertices. This allows us to quickly add new vertices and sort them.

3. **Finding K-th Largest Element**:
   - When queried for the k-th largest element, we simply access the appropriate position in the sorted set.

This approach ensures efficient handling of both adding edges and querying the k-th largest connected vertex, making it suitable for large inputs within the given constraints.
