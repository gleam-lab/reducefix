To fix this code, we need to ensure that the union operation correctly merges the sets of connected vertices while maintaining the top 10 largest vertices in each set. The initial approach had a bug in the merging process where it didn't correctly handle the merging of two sorted lists, leading to incorrect results in certain cases.

### Approaches
1. **Union-Find Data Structure**: This will help efficiently manage the connected components of the graph.
2. **Maintaining Sorted Lists**: For each connected component, we maintain a sorted list (in descending order) of the top 10 largest vertices. This allows us to quickly access the k-th largest vertex in the component.
3. **Efficient Merging**: When merging two components, we merge their sorted lists and retain only the top 10 elements. This ensures that the list remains manageable and operations remain efficient.

### Solution Code
```cpp
#include "bits/stdc++.h"

using namespace std;

#define ff first
#define ss second
#define ll long long
#define SZ(s) (int)s.size()

int const N = 2e5 + 5;

vector<vector<int>> v;

vector<int> p;

int par(int x) {
    if(p[x] == x) return x;
    return p[x] = par(p[x]);
}

void uni(int a, int b) {
    a = par(a), b = par(b);
    if(a == b) return;
    if(SZ(v[a]) < SZ(v[b])) swap(a, b);
    vector<int> merged;
    int i = 0, j = 0;
    while((i < SZ(v[a]) || j < SZ(v[b])) && merged.size() < 10) {
        if(j >= SZ(v[b]) || (i < SZ(v[a]) && v[a][i] > v[b][j])) {
            merged.push_back(v[a][i]);
            i++;
        } else {
            merged.push_back(v[b][j]);
            j++;
        }
    }
    v[a] = merged;
    p[b] = a;
    return;
}

signed main() {
    ios_base::sync_with_stdio(false); cin.tie(nullptr);

    int n, q;
    cin >> n >> q;
    p.resize(n+1);
    v.resize(n+1);
    for(int i = 1; i <= n; i++) {
        p[i] = i;
        v[i].push_back(i);
    }
    while(q--) {
        int t, a, b;
        cin >> t >> a >> b;
        if(t == 1) {
            uni(a, b);
        }
        else {
            a = par(a);
            cout << (SZ(v[a]) < b ? -1 : v[a][b-1]) << '\n';
        }
    }

    return 0;
}
```

### Explanation
1. **Initialization**: We initialize the parent array `p` and the list of vertices `v` for each node. Each node initially points to itself and contains only itself in its list.
2. **Union Operation**: When merging two components, we ensure the larger component absorbs the smaller one to maintain efficiency. We merge their sorted lists while keeping only the top 10 largest elements. This is done using a two-pointer approach to merge the lists in descending order.
3. **Query Processing**: For each query of type 2, we find the root of the component containing the given vertex and directly access the k-th largest element from the pre-maintained list. If the list is smaller than `k`, we return `-1`.

This approach ensures that each union operation is handled in logarithmic time due to path compression in the Union-Find structure, and the merging of lists is linear in the size of the lists (limited to 10 elements), making the overall complexity efficient enough for the given constraints.
