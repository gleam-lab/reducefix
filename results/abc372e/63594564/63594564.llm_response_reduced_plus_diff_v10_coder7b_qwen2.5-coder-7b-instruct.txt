int par(int x) {

    return p[x] = par(p[x]);

void uni(int a, int b) {
    a = par(a), b = par(b);

        sorted_v[b].insert(it, x);

    return;
}

### Explanation of Changes:
1. **Union-Find Structure**: 
   - We use a union-find structure where each node points to its parent (`p`).
   - We also maintain a separate vector `sorted_v` to keep track of the sorted neighbors for each component.

2. **Union Operation**:
   - When merging two components, we ensure that the smaller component is always merged into the larger one to minimize the size of `sorted_v`.
   - We insert all elements from the smaller component into the larger component's sorted list and then resize it to at most 10 elements.

3. **Find Operation**:
   - For finding the k-th largest neighbor, we simply check if the size of the sorted list is less than `k`. If so, we output `-1`. Otherwise, we output the k-th largest element.

This approach ensures that both union and find operations are efficient, keeping the overall complexity manageable even for large inputs.
