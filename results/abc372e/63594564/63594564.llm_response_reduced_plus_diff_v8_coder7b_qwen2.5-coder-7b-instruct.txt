To solve this problem efficiently, we need to handle both adding edges and querying the k-th largest vertex connected to a given vertex. This requires maintaining adjacency lists and being able to quickly access and sort the neighbors of each vertex.

Given the constraints, using a balanced binary search tree (like `std::multiset`) for storing the neighbors can help maintain efficient operations. Here's how you can implement it:

```cpp
#include "bits/stdc++.h"

using namespace std;

#define ff first
#define ss second
#define ll long long
#define SZ(s) (int)s.size()

int const N = 2e5 + 5;

vector<multiset<int>> adj;

void addEdge(int u, int v) {
    adj[u].insert(v);
    adj[v].insert(u);
}

int kthLargest(int v, int k) {
    auto it = adj[v].rbegin();
    advance(it, k - 1);
    return *it;
}

signed main() {
    ios_base::sync_with_stdio(false); cin.tie(nullptr);

    int n, q;
    cin >> n >> q;
    adj.resize(n + 1);

    while(q--) {
        int t, a, b;
        cin >> t >> a >> b;
        if(t == 1) {
            addEdge(a, b);
        } else {
            if(adj[a].size() < k) {
                cout << "-1\n";
            } else {
                cout << kthLargest(a, k) << '\n';
            }
        }
    }

    return 0;
}
```

### Explanation:
1. **Adjacency List Representation**: We use a vector of multiset to represent the graph. A multiset allows us to store duplicate values (in case multiple edges between the same pair of nodes are added) and provides efficient insertion and deletion operations.

2. **Adding Edges**: The `addEdge` function inserts the vertices into each other's adjacency list.

3. **Finding k-th Largest Neighbor**: The `kthLargest` function uses reverse iterators to find the k-th largest element in the multiset. Since multiset stores elements in sorted order, `rbegin()` gives us the largest element, and advancing it `k-1` steps gives us the k-th largest element.

4. **Query Handling**: For each query, if the size of the adjacency list for the given vertex is less than `k`, we output `-1`. Otherwise, we output the k-th largest neighbor.

This approach ensures that all operations are performed efficiently within the given constraints. The time complexity for adding an edge is \(O(\log N)\) and for finding the k-th largest neighbor is also \(O(\log N)\). Therefore, the overall complexity for processing `Q` queries is \(O(Q \log N)\), which is efficient enough for the problem constraints.
