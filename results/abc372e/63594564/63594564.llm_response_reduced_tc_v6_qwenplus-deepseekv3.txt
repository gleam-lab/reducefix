To fix this code, we need to address the issue in how the connected components and their vertices are being managed. The problem requires maintaining the connected components efficiently and being able to quickly find the k-th largest vertex in the component. The initial approach merges the sorted lists of vertices but fails to maintain the correct order and size, leading to incorrect results.

### Approaches
1. **Union-Find Data Structure**: This helps in efficiently managing the connected components and finding the root of any vertex.
2. **Sorted Lists for Each Component**: Each component maintains a sorted list of its vertices. When two components are merged, the two sorted lists are merged efficiently, ensuring the merged list remains sorted. To keep the operations efficient, we limit the size of the sorted list to a maximum of 10 elements since `k` is at most 10 in the queries.

### Solution Code
```cpp
#include <bits/stdc++.h>

using namespace std;

#define ff first
#define ss second
#define ll long long
#define SZ(s) (int)s.size()

int const N = 2e5 + 5;

vector <vector <int>> v;

vector <int> p;

int par(int x) {
    if (p[x] == x) return x;
    return p[x] = par(p[x]);
}

void uni(int a, int b) {
    a = par(a), b = par(b);
    if (a == b) return;
    if (SZ(v[a]) < SZ(v[b])) swap(a, b);
    vector<int> merged;
    int i = 0, j = 0;
    while (i < SZ(v[a]) && j < SZ(v[b]) && merged.size() < 10) {
        if (v[a][i] > v[b][j]) {
            merged.push_back(v[a][i]);
            i++;
        } else {
            merged.push_back(v[b][j]);
            j++;
        }
    }
    while (i < SZ(v[a]) && merged.size() < 10) {
        merged.push_back(v[a][i]);
        i++;
    }
    while (j < SZ(v[b]) && merged.size() < 10) {
        merged.push_back(v[b][j]);
        j++;
    }
    v[a] = merged;
    p[b] = a;
    return;
}

signed main() {
    ios_base::sync_with_stdio(false); cin.tie(nullptr);

    int n, q;
    cin >> n >> q;
    p.resize(n + 1);
    v.resize(n + 1);
    for (int i = 1; i <= n; i++) {
        p[i] = i;
        v[i].push_back(i);
    }
    while (q--) {
        int t, a, b;
        cin >> t >> a >> b;
        if (t == 1) {
            uni(a, b);
        }
        else {
            a = par(a);
            if (SZ(v[a]) < b) {
                cout << -1 << '\n';
            }
            else {
                cout << v[a][b - 1] << '\n';
            }
        }
    }

    return 0;
}
```

### Explanation
1. **Union-Find Data Structure**: The `par` function implements path compression to efficiently find the root of any vertex. The `uni` function merges two components by size, ensuring the smaller component is merged into the larger one for efficiency.
2. **Sorted Lists Management**: Each component maintains a sorted list of its vertices in descending order. When merging two components, the `uni` function merges their sorted lists while keeping only the top 10 elements, which ensures the operation is efficient and meets the problem constraints.
3. **Query Handling**: For Type 2 queries, the code checks if the component has at least `k` vertices. If it does, it returns the k-th largest vertex; otherwise, it returns `-1`.

This approach ensures the solution is efficient and correct, handling all edge cases as specified in the problem constraints.
