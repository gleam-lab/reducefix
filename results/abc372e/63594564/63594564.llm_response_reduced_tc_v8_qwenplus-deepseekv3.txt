To fix the code, we need to ensure that when merging two sets, the top 10 largest elements are correctly maintained in the merged set. The original code attempts to merge two sorted lists and then truncate to the top 10 elements, but this approach may not always maintain the correct order or the top elements efficiently, especially when merging larger sets.

### Approaches
1. **Union-Find Data Structure**: This helps efficiently manage and merge disjoint sets. Each set will keep track of its elements in a sorted manner, specifically the top 10 largest elements.
2. **Maintaining Top Elements**: When merging two sets, we merge their top elements, sort them in descending order, and keep only the top 10 elements. This ensures that each set always has the largest elements readily available for queries.

### Solution Code
```cpp
#include "bits/stdc++.h"

using namespace std;

#define ff first
#define ss second
#define ll long long
#define SZ(s) (int)s.size()

int const N = 2e5 + 5;

vector <vector <int>> v;

vector <int> p;

int par(int x) {
    if(p[x] == x) return x;
    return p[x] = par(p[x]);
}

void uni(int a, int b) {
    a = par(a), b = par(b);
    if(a == b) return;
    if (SZ(v[a]) < SZ(v[b])) {
        swap(a, b);
    }
    vector<int> merged;
    merged.reserve(v[a].size() + v[b].size());
    merge(v[a].begin(), v[a].end(), v[b].begin(), v[b].end(), back_inserter(merged), greater<int>());
    if (SZ(merged) > 10) {
        merged.resize(10);
    }
    v[a] = merged;
    p[b] = a;
    return;
}

signed main() {
    ios_base::sync_with_stdio(false); cin.tie(nullptr);

    int n, q;
    cin >> n >> q;
    p.resize(n+1);
    v.resize(n+1);
    for(int i = 1; i <= n; i++) {
        p[i] = i;
        v[i].push_back(i);
    }
    while(q--) {
        int t, a, b;
        cin >> t >> a >> b;
        if(t == 1) {
            uni(a, b);
        }
        else {
            a = par(a);
            cout << (SZ(v[a]) < b ? -1 : v[a][b-1]) << '\n';
        }
    }

    return 0;
}
```

### Explanation
1. **Union-Find Data Structure**: The `par` function implements path compression to efficiently find the root parent of a node. The `uni` function merges two sets by size, ensuring the smaller set is merged into the larger one to maintain efficiency.
2. **Merging Top Elements**: The `uni` function merges the top elements of two sets using the `merge` function from the standard library, which combines two sorted ranges into a single sorted range in descending order. The merged list is then truncated to the top 10 elements to maintain efficiency.
3. **Query Handling**: For type 2 queries, the code checks the size of the connected component of the queried vertex. If the size is less than `k`, it returns `-1`; otherwise, it returns the `k-th` largest element from the stored top elements.

This approach ensures that each merge operation is efficient and that the top elements are always readily available for queries, maintaining the required performance constraints.
