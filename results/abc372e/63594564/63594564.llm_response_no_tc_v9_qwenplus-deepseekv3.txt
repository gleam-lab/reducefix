To fix this code, we need to address the inefficiency and incorrectness in handling the merging of sets and maintaining the top 10 largest elements in each connected component. The original code attempts to merge two sorted lists and keep only the top 10 elements, but this approach is too slow for large inputs (up to 200,000 queries) due to its O(N) time complexity per merge operation.

### Approaches
1. **Union-Find Data Structure**: Use the Union-Find (Disjoint Set Union, DSU) data structure to manage connected components efficiently.
2. **Merge Smallest into Largest**: When merging two sets, always merge the smaller set into the larger one to keep the operations efficient (Union by Size).
3. **Maintain Top Elements**: For each root node, maintain a sorted list (or a priority queue) of the top 10 largest elements in its component. When merging two components, combine their top elements, sort them, and keep only the top 10.

### Solution Code
```cpp
#include "bits/stdc++.h"

using namespace std;

#define ff first
#define ss second
#define ll long long
#define SZ(s) (int)s.size()

int const N = 2e5 + 5;

vector<set<int>> v;
vector<int> p;

int par(int x) {
    if (p[x] == x) return x;
    return p[x] = par(p[x]);
}

void uni(int a, int b) {
    a = par(a), b = par(b);
    if (a == b) return;
    if (v[a].size() < v[b].size()) swap(a, b);
    for (int num : v[b]) {
        v[a].insert(num);
        if (v[a].size() > 10) {
            v[a].erase(v[a].begin());
        }
    }
    p[b] = a;
    return;
}

signed main() {
    ios_base::sync_with_stdio(false); cin.tie(nullptr);

    int n, q;
    cin >> n >> q;
    p.resize(n + 1);
    v.resize(n + 1);
    for (int i = 1; i <= n; i++) {
        p[i] = i;
        v[i].insert(i);
    }
    while (q--) {
        int t, a, b;
        cin >> t >> a >> b;
        if (t == 1) {
            uni(a, b);
        }
        else {
            a = par(a);
            if (v[a].size() < b) {
                cout << -1 << '\n';
            } else {
                auto it = v[a].end();
                advance(it, -b);
                cout << *it << '\n';
            }
        }
    }

    return 0;
}
```

### Explanation
1. **Union-Find Data Structure**: The `par` function uses path compression to efficiently find the root of a node, ensuring near-constant time complexity for each find operation.
2. **Merge Strategy**: The `uni` function merges the smaller set into the larger one to keep the operations efficient. It also ensures that each root node's set (`v[a]`) contains the top 10 largest elements after merging by inserting elements from the smaller set and trimming if the set exceeds 10 elements.
3. **Top Elements Maintenance**: For each query of type 2, the code checks if the connected component of the given node has at least `k` elements. If not, it returns `-1`. Otherwise, it retrieves the `k-th` largest element by accessing the appropriate position in the sorted set.

This approach ensures that each merge operation is efficient (O(10 log N) per operation) and each query is answered in O(1) time after the set operations, making the solution suitable for large input sizes.
