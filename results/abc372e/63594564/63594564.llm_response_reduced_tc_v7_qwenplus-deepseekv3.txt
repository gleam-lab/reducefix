To fix this code, we need to address the issue with the merging of the sets and the way the k-th largest element is being maintained. The current approach merges the sets and then tries to keep only the top 10 elements, but there might be issues with how the merging is done, especially with maintaining the sorted order and correctly keeping the top elements.

### Approaches
1. **Union-Find Data Structure**: This will help manage the connected components efficiently.
2. **Maintaining Sorted Lists**: For each connected component, maintain a sorted list (or a priority queue) of the vertices. When two components are merged, merge their respective lists and keep only the top 10 largest elements. This ensures that we can quickly access the k-th largest element when needed.
3. **Efficient Merging**: When merging two components, ensure that the merging process is efficient (O(1) or O(log N) per operation) and does not degrade performance.

### Solution Code
```cpp
#include <bits/stdc++.h>

using namespace std;

#define ff first
#define ss second
#define ll long long
#define SZ(s) (int)s.size()

int const N = 2e5 + 5;

vector<set<int, greater<int>>> v;

vector<int> p;

int par(int x) {
    if (p[x] == x) return x;
    return p[x] = par(p[x]);
}

void uni(int a, int b) {
    a = par(a), b = par(b);
    if (a == b) return;
    if (SZ(v[a]) < SZ(v[b])) swap(a, b);
    for (auto x : v[b]) {
        v[a].insert(x);
        if (SZ(v[a]) > 10) {
            auto it = prev(v[a].end());
            v[a].erase(it);
        }
    }
    p[b] = a;
    return;
}

signed main() {
    ios_base::sync_with_stdio(false); cin.tie(nullptr);

    int n, q;
    cin >> n >> q;
    p.resize(n + 1);
    v.resize(n + 1);
    for (int i = 1; i <= n; i++) {
        p[i] = i;
        v[i].insert(i);
    }
    while (q--) {
        int t, a, b;
        cin >> t >> a >> b;
        if (t == 1) {
            uni(a, b);
        }
        else {
            a = par(a);
            if (SZ(v[a]) < b) {
                cout << -1 << '\n';
            }
            else {
                auto it = v[a].begin();
                advance(it, b - 1);
                cout << *it << '\n';
            }
        }
    }

    return 0;
}
```

### Explanation
1. **Union-Find Data Structure**: The `par` function helps in finding the parent of a node with path compression, ensuring efficient lookups.
2. **Maintaining Sorted Lists**: The `set<int, greater<int>>` is used for each component to maintain the vertices in descending order. This allows for efficient access to the k-th largest element.
3. **Efficient Merging**: The `uni` function merges two components by moving all elements from the smaller set to the larger one, ensuring the merge operation is efficient. After merging, it ensures the set does not exceed 10 elements by removing the smallest elements if necessary.
4. **Query Handling**: For type 2 queries, it checks the size of the set corresponding to the component of the given vertex. If the size is less than k, it returns -1; otherwise, it retrieves the k-th largest element from the set.

This approach ensures that the operations are handled efficiently, even for large inputs, by leveraging the union-find data structure and maintaining sorted sets of limited size.
