1. **Merging Sorted Lists**: The code attempts to merge two sorted lists by comparing elements and merging them in descending order. However, it doesn't handle the case where one list is exhausted before the other correctly, leading to incorrect merged lists.
2. **Maintaining Top 10 Elements**: The code only keeps the top 10 elements after merging, but the merging process itself is flawed, which can lead to incorrect top elements being retained.
3. **Efficiency**: The merging process is O(n) per union operation, which can be too slow for large N and Q (up to 2×10^5 each), leading to a potential O(N^2) time complexity in the worst case.

### Solution Approach
1. **Efficient Union-Find with Path Compression**: Use the standard union-find data structure with path compression to efficiently manage connected components.
2. **Maintaining Sorted Lists**: For each component, maintain a sorted list (in descending order) of the vertices. When merging two components, merge their sorted lists and keep only the top 10 elements to ensure queries can be answered quickly.
3. **Handling Queries**: For each type 2 query, check if the component has at least k elements. If so, return the k-th largest element; otherwise, return -1.

#include <bits/stdc++.h>

using namespace std;

vector<int> parent;
vector<vector<int>> components;

int find(int x) {
    if (parent[x] != x) {
        parent[x] = find(parent[x]);
    }
    return parent[x];
}

void unite(int a, int b) {
    a = find(a);
    b = find(b);
    if (a == b) return;
    if (components[a].size() < components[b].size()) {
        swap(a, b);
    }
    // Merge the smaller component into the larger one
    vector<int> merged;
    int i = 0, j = 0;
    while (i < components[a].size() && j < components[b].size() && merged.size() < 10) {
        if (components[a][i] > components[b][j]) {
            merged.push_back(components[a][i]);
            i++;

            merged.push_back(components[b][j]);
            j++;
        }
    }
    while (i < components[a].size() && merged.size() < 10) {
        merged.push_back(components[a][i]);
        i++;
    }
    while (j < components[b].size() && merged.size() < 10) {
        merged.push_back(components[b][j]);
        j++;
    }
    components[a] = merged;
    parent[b] = a;
}

int main() {

    cin.tie(nullptr);

    int n, q;
    cin >> n >> q;
    parent.resize(n + 1);
    components.resize(n + 1);
