To solve this problem efficiently, we need to manage dynamic connectivity in a graph with the following operations:

1. **Add an edge** between two nodes.
2. **Query** the k-th largest node number connected to a given node.

This is a classic use case for the **Disjoint Set Union (DSU)** data structure (also known as Union-Find), but with a twist: we need to maintain information about the top-k largest nodes connected to any given node.

---

### Key Observations

- Since each query of type 2 only asks for the **top 10 largest** nodes connected to a vertex, we don't need to store the full list of connected nodes — just the **top 10 largest ones**.
- Merging sets while keeping only the top 10 largest elements can be done efficiently using **priority queues or sorting and trimming**.
- We can optimize union operations by always merging smaller sets into larger ones (union-by-size) to reduce overhead.

---

### Solution Strategy

We'll implement:

- A DSU with path compression and union-by-size.
- Each set will maintain a sorted list (descending order) of up to **10 largest vertices** in that component.
- On union, we merge the smaller set's top 10 list into the larger set's list, sort it, and keep only the top 10.

---

### Efficient Implementation Details

- Use `vector<int>` for each representative to store the top 10 largest vertices.
- When merging:
  - Always merge the smaller component into the larger one.
  - Merge their top lists, sort, and trim to keep only the top 10.
- For queries, simply access the list at index `k-1` if it exists.

### Corrected C++ Code

```cpp
#include <bits/stdc++.h>

using namespace std;

#define ff first
#define ss second
#define ll long long
#define SZ(s) ((int)(s).size())

const int N = 2 * 100000 + 5;

vector<vector<int>> top_nodes;
vector<int> parent, size;

int find(int x) {
    if (parent[x] != x)
        parent[x] = find(parent[x]);
    return parent[x];
}

void unite(int u, int v) {
    int root_u = find(u);
    int root_v = find(v);

    if (root_u == root_v)
        return;

    // Ensure root_u has the larger set
    if (SZ(top_nodes[root_u]) < SZ(top_nodes[root_v])) {
        swap(root_u, root_v);
    }

    // Merge the smaller set into the larger one
    for (int x : top_nodes[root_v]) {
        top_nodes[root_u].push_back(x);
    }

    // Sort in descending order and keep top 10
    sort(top_nodes[root_u].begin(), top_nodes[root_u].end(), greater<int>());
    if (SZ(top_nodes[root_u]) > 10) {
        top_nodes[root_u].resize(10);
    }

    // Update DSU parent
    parent[root_v] = root_u;
    size[root_u] += size[root_v];
}

int main() {
    ios_base::sync_with_stdio(false); cin.tie(nullptr);

    int n, q;
    cin >> n >> q;

    top_nodes.resize(n + 1);
    parent.resize(n + 1);
    size.resize(n + 1);

    for (int i = 1; i <= n; ++i) {
        parent[i] = i;
        size[i] = 1;
        top_nodes[i].push_back(i);
    }

    while (q--) {
        int t;
        cin >> t;

        if (t == 1) {
            int u, v;
            cin >> u >> v;
            unite(u, v);
        } else {
            int v, k;
            cin >> v >> k;
            int root = find(v);
            if (SZ(top_nodes[root]) >= k) {
                cout << top_nodes[root][k - 1] << '\n';
            } else {
                cout << -1 << '\n';
            }
        }
    }

    return 0;
}
```

### Complexity Analysis

- **Union Operation**: O(log N) amortized due to union-by-size and maintaining at most 10 elements.
- **Find Operation**: O(α(N)) due to path compression.
- **Query Operation**: O(1) since we directly access the k-th element.
