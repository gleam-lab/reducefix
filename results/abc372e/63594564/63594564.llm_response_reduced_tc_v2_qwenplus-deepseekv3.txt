To fix this code, we need to ensure that the k-th largest vertex number among the connected vertices is correctly identified and returned. The initial approach tried to maintain sorted lists of vertices in each connected component, but it had a flaw in merging the lists correctly, leading to incorrect results for certain queries.

### Approaches
1. **Union-Find Data Structure**: This will help in efficiently managing the connected components and merging them when edges are added.
2. **Maintaining Sorted Lists**: For each connected component, maintain a sorted list (or a priority queue) of the largest up to 10 vertices. This ensures that when we need to find the k-th largest vertex, we can quickly access it from the maintained list.
3. **Efficient Merging**: When two components are merged, combine their respective lists while keeping only the top 10 largest elements. This ensures that the lists remain manageable and queries can be answered in constant time.

### Solution Code
```cpp
#include <bits/stdc++.h>

using namespace std;

#define ff first
#define ss second
#define ll long long
#define SZ(s) (int)s.size()

int const N = 2e5 + 5;

vector<priority_queue<int, vector<int>, greater<int>>> v; // Min-heap to keep top 10 largest
vector<int> p;

int par(int x) {
    if (p[x] == x) return x;
    return p[x] = par(p[x]);
}

void uni(int a, int b) {
    a = par(a), b = par(b);
    if (a == b) return;
    if (SZ(v[a]) < SZ(v[b])) swap(a, b);
    while (!v[b].empty()) {
        v[a].push(v[b].top());
        v[b].pop();
        if (SZ(v[a]) > 10) v[a].pop();
    }
    p[b] = a;
}

signed main() {
    ios_base::sync_with_stdio(false); cin.tie(nullptr);

    int n, q;
    cin >> n >> q;
    p.resize(n + 1);
    v.resize(n + 1);
    for (int i = 1; i <= n; i++) {
        p[i] = i;
        v[i].push(i);
    }
    while (q--) {
        int t, a, b;
        cin >> t >> a >> b;
        if (t == 1) {
            uni(a, b);
        } else {
            a = par(a);
            if (SZ(v[a]) < b) {
                cout << -1 << '\n';
            } else {
                vector<int> temp;
                while (!v[a].empty()) {
                    temp.push_back(v[a].top());
                    v[a].pop();
                }
                cout << temp[SZ(temp) - b] << '\n';
                for (int num : temp) {
                    v[a].push(num);
                }
            }
        }
    }

    return 0;
}
```

### Explanation
1. **Union-Find Initialization**: The `par` function uses path compression to efficiently find the root of a vertex. The `p` array keeps track of parent nodes.
2. **Heap Management**: Each vertex initially has a min-heap containing only itself. When merging two components, the heaps are combined, and only the top 10 largest elements are retained. This ensures the heap size never exceeds 10, making operations efficient.
3. **Query Handling**: For a Type 2 query, the connected component's heap is checked. If the heap size is less than `k`, `-1` is returned. Otherwise, the elements are temporarily extracted to a vector, and the `k-th` largest element (from the end of the sorted vector) is returned. The elements are then restored back to the heap.

This approach ensures that each union operation is handled in logarithmic time relative to the heap size, and each query is processed in constant time, making the solution efficient and scalable for large inputs.
