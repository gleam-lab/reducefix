To solve this problem efficiently, we need to handle the union-find structure correctly to manage the connected components and maintain a sorted list of vertices for each component. We also need to use a balanced binary search tree (like set or multiset) to quickly find the k-th largest element in the sorted list of connected vertices.

Here's the corrected C++ code:

```cpp
#include <iostream>
#include <vector>
#include <set>

using namespace std;

const int MAX_N = 2 * 100000 + 5;

vector<int> parent(MAX_N);
vector<set<int>> adjList(MAX_N);

int findParent(int node) {
    if (parent[node] != node) {
        parent[node] = findParent(parent[node]);
    }
    return parent[node];
}

void unionSets(int u, int v) {
    int pu = findParent(u);
    int pv = findParent(v);
    
    if (pu == pv) return;
    
    // Merge smaller set into larger set
    if (adjList[pu].size() > adjList[pv].size()) {
        swap(pu, pv);
    }
    
    for (auto &neighbor : adjList[pu]) {
        adjList[pv].insert(neighbor);
    }
    adjList[pu].clear();
    parent[pu] = pv;
}

int kthLargest(int u, int k) {
    int pu = findParent(u);
    auto it = adjList[pu].rbegin();
    advance(it, k - 1);
    if (it != adjList[pu].rend()) {
        return *it;
    }
    return -1;
}

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);

    int n, q;
    cin >> n >> q;

    for (int i = 1; i <= n; ++i) {
        parent[i] = i;
        adjList[i].insert(i);
    }

    for (int i = 0; i < q; ++i) {
        int type, u, v;
        cin >> type >> u >> v;

        if (type == 1) {
            unionSets(u, v);
        } else {
            int result = kthLargest(v, k);
            cout << result << '\n';
        }
    }

    return 0;
}
```

### Explanation:
1. **Union-Find Structure**:
   - `findParent` function to find the root of the set containing `node`.
   - `unionSets` function to merge two sets. It ensures that the smaller set is merged into the larger set to keep the structure balanced.

2. **Adjacency List with Sets**:
   - Each node has an associated set (`adjList`) that stores all nodes connected to it.
   - When merging two sets, the smaller set is added to the larger set, and the smaller set is cleared.

3. **K-th Largest Element**:
   - `kthLargest` function finds the k-th largest element in the set of connected vertices using reverse iterators.

This approach ensures efficient handling of both union operations and finding the k-th largest element in the connected components. The time complexity for union and find operations is nearly optimal due to path compression and union by rank, making the solution suitable for large inputs.
