The constraints are relatively large, so an efficient algorithm is needed. ### Issues:

3. **Finding `k`-th Largest Vertex**: The current approach is to maintain a sorted list of vertices for each component, which is updated every time an edge is added. This is inefficient and can lead to performance issues as the number of vertices grows.

### Proposed Improvements:
