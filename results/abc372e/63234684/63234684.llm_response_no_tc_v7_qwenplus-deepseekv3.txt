### Approaches
1. **Union-Find with Path Compression and Union by Size**: This ensures efficient merging and finding operations.
2. **Maintaining Sorted Lists**: For each root, maintain a sorted list of the top 10 largest vertices in its connected component. This allows quick access to the k-th largest vertex by simply indexing into the list.
3. **Efficient Merging**: When merging two components, merge their sorted lists and keep only the top 10 elements to ensure the lists remain manageable in size.

### Solution Code
```cpp
#include<bits/stdc++.h>
using namespace std;
using ll = long long;
#define rep(i,n) for(int i = 0;i<n;i++)
using P = pair<int,int>;
using T = tuple<int,int,int>;

struct Union_Find {
    vector<int> par;
    vector<vector<int>> top;

    Union_Find(int n) {
        par.resize(n, -1);
        top.resize(n);
        rep(i, n) top[i].push_back(i + 1); // 1-based
    }

    int root(int v) {
        if (par[v] == -1) return v;
        else return par[v] = root(par[v]);
    }

    bool issame(int x, int y) {
        return root(x) == root(y);
    }

    void merge(int x, int y) {
        x = root(x);
        y = root(y);
        if (x == y) return;
        if (top[x].size() < top[y].size()) swap(x, y);
        par[y] = x;
        top[x].insert(top[x].end(), top[y].begin(), top[y].end());
        sort(top[x].rbegin(), top[x].rend());
        if (top[x].size() > 10) top[x].resize(10);
    }
};

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);

    int n, q;
    cin >> n >> q;
    Union_Find uf(n);

    rep(i, q) {
        int ty, a, b;
        cin >> ty >> a >> b;
        if (ty == 1) {
            uf.merge(a - 1, b - 1); // converting to 0-based
        } else {
            int root = uf.root(a - 1);
            if (uf.top[root].size() < b) {
                cout << -1 << '\n';
            } else {
                cout << uf.top[root][b - 1] << '\n';
            }
        }
    }
    return 0;
}
```

### Explanation
1. **Union-Find Structure**: The `Union_Find` class includes path compression and union by size to efficiently manage the connected components.
2. **Top Elements Maintenance**: Each root maintains a sorted list of the top 10 largest vertices in its component. This list is updated during merges by combining the lists of the two components, sorting them in descending order, and truncating to the top 10 elements.
3. **Query Handling**: For Type 2 queries, the code checks if the connected component has at least `k` vertices. If not, it outputs `-1`. Otherwise, it retrieves the k-th largest vertex from the maintained list.
4. **Efficiency**: The approach ensures that each merge operation is O(1) amortized due to path compression, and maintaining small sorted lists (size ≤ 10) keeps operations efficient within the problem constraints.
