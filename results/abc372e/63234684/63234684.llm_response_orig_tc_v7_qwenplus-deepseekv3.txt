To fix this code, we need to address the incorrect handling of the k-th largest vertex number in connected components. The issue arises because the original code attempts to maintain a sorted list of vertices in each component, but the merging process is inefficient and doesn't correctly handle large inputs within the constraints.

### Approaches
1. **Union-Find Data Structure**: The Union-Find (Disjoint Set Union, DSU) is used to efficiently manage the connected components of the graph.
2. **Efficient Merging**: For each component, maintain a sorted list of vertices. When merging two components, merge their lists and keep only the top 10 largest elements to ensure queries can be answered in constant time.
3. **Handling Queries**: For each type 2 query, check if the component has at least `k` elements and return the k-th largest element if it exists.

### Solution Code
```cpp
#include<bits/stdc++.h>
using namespace std;
using ll = long long;
#define rep(i,n) for(int i = 0;i<n;i++)
using P = pair<int,int>;
using T = tuple<int,int,int>;

struct Union_Find {
    vector<int> sz, par;
    vector<vector<int>> uni;
    
    Union_Find(int n) {
        sz.resize(n, 1);
        par.resize(n, -1);
        uni.resize(n);
        rep(i, n) uni[i].push_back(i);
    }
    
    int root(int v) {
        if(par[v] == -1) return v;
        else return par[v] = root(par[v]);
    }
    
    bool issame(int x, int y) {
        return root(x) == root(y);
    }
    
    void merge(int x, int y) {
        x = root(x);
        y = root(y);
        if(x == y) return;
        if(sz[x] < sz[y]) swap(x, y);
        par[y] = x;
        sz[x] += sz[y];
        uni[x].insert(uni[x].end(), uni[y].begin(), uni[y].end());
        sort(uni[x].rbegin(), uni[x].rend());
        if(uni[x].size() > 10) uni[x].resize(10);
        uni[y].clear();
    }
};

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);
    int n, q;
    cin >> n >> q;
    Union_Find uf(n);
    
    rep(i, q) {
        int ty, a, b;
        cin >> ty >> a >> b;
        a--, b--;
        if(ty == 1) {
            uf.merge(a, b);
        } else {
            int r = uf.root(a);
            if(uf.uni[r].size() < b + 1) {
                cout << -1 << '\n';
            } else {
                cout << uf.uni[r][b] + 1 << '\n';
            }
        }
    }
}
```

### Explanation
1. **Union-Find Initialization**: The `Union_Find` structure is initialized with each vertex as its own parent and a list containing only itself.
2. **Root Finding**: The `root` method uses path compression to efficiently find the root of any vertex.
3. **Merging Components**: The `merge` method combines two components by size (smaller into larger). It merges their vertex lists, sorts them in descending order, and keeps only the top 10 elements to optimize space and query time.
4. **Query Handling**: For type 2 queries, the code checks the size of the component's vertex list. If the list is large enough, it returns the k-th largest vertex; otherwise, it returns `-1`.

This approach ensures efficient handling of both union and query operations, adhering to the problem constraints and providing correct results for all test cases.
