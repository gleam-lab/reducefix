To solve this problem correctly and efficiently, we need to:

1. **Track connected components** of the graph as edges are added using a **Disjoint Set Union (DSU)** or **Union-Find** data structure.
2. For each query of Type 2 (`2 v k`), retrieve the `k`-th largest vertex in the component that includes vertex `v`.

### Key Observations:
- Since only `Type 2` queries ask for the `k`-th largest node in a connected component, and `k <= 10`, we only need to maintain **up to 10 largest elements** per set in DSU.
- Merging sets must preserve these top-k values efficiently.

### Improvements over Your Code:
- You were maintaining a sorted list of up to 10 nodes per root, but sorting after every merge is expensive and unnecessary.
- Instead, we should **merge smaller lists into larger ones**, and then keep only the **top 10 largest vertices** in the merged list.

Here's the corrected and optimized C++ implementation:

```cpp
#include <bits/stdc++.h>
using namespace std;

using vi = vector<int>;

struct DSU {
    vi parent, size;
    vector<vi> top_nodes;

    DSU(int n) {
        parent.resize(n);
        size.assign(n, 1);
        iota(parent.begin(), parent.end(), 0);
        top_nodes.resize(n);
        for (int i = 0; i < n; ++i) {
            top_nodes[i].push_back(i);
        }
    }

    int find(int x) {
        if (parent[x] != x)
            parent[x] = find(parent[x]);
        return parent[x];
    }

    void unite(int x, int y) {
        int xr = find(x), yr = find(y);
        if (xr == yr) return;

        // Merge smaller set into larger set
        if (size[xr] < size[yr]) swap(xr, yr);
        parent[yr] = xr;
        size[xr] += size[yr];

        // Merge top nodes
        auto& large = top_nodes[xr];
        auto& small = top_nodes[yr];

        large.insert(large.end(), small.begin(), small.end());
        sort(large.rbegin(), large.rend());

        // Keep only top 10
        if (large.size() > 10) {
            large.resize(10);
        }
    }
};

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int N, Q;
    cin >> N >> Q;

    DSU dsu(N);

    while (Q--) {
        int type, u, v;
        cin >> type >> u >> v;
        u--; v--;

        if (type == 1) {
            dsu.unite(u, v);
        } else {
            int root = dsu.find(u);
            const auto& nodes = dsu.top_nodes[root];
            if (v >= (int)nodes.size()) {
                cout << -1 << '\n';
            } else {
                cout << nodes[v] + 1 << '\n'; // Convert from 0-indexed to 1-indexed
            }
        }
    }

    return 0;
}
```

---

### Explanation:
- **DSU Initialization**: Each node starts as its own parent with size 1. The `top_nodes` list for each node initially contains just itself.
- **Unite Operation**:
  - We ensure the smaller set merges into the larger one to maintain efficiency.
  - After merging, we sort the combined list in descending order and truncate it to at most 10 elements.
- **Query Handling**:
  - For Type 2 queries, we find the root of the component containing `u`, and return the `v`-th element (adjusted for 1-based output).
  - If there are fewer than `v+1` elements in the list, we return `-1`.

This approach ensures that both union and query operations run efficiently within time constraints.
