
The problem involves managing an undirected graph using the Union-Find data structure with two main operations: adding edges between vertices and querying for the k-th largest vertex in the connected component of a given vertex.

Here are the areas where the current solution is falling short and the proposed improvements:

### Issues in the current solution:
1. **Incorrect Union-Find Representation**: The current `Union_Find` structure uses an additional vector `uni` which stores the vertices in the connected components. This is not the standard way to represent a Union-Find structure and leads to inefficient operations.
2. **Incorrect Component Size Calculation**: The current solution calculates the size of the connected component in a roundabout way by populating and resizing multiple vectors which can be costly.
3. **Efficiency**: Using a more complex data structure such as a vector of vectors inside the Union-Find does not efficiently support the required operations of the problem, especially when k is small.

### Proposed Improvements:
To address the above issues, we can follow these steps:

1. **Optimize the Union-Find Structure**: Instead of using a complex data structure, we can use a simple array to represent the parent of each node and an array to store the size of each component.
2. **Efficient Find and Union Operations**: The find operation needs to be path compression, which flattens the structure of the tree whenever find is called, ensuring that the height of the trees remains small. The union operation needs to be size-based, which merges the smaller tree under the root of the larger tree.
3. **Efficient Query Handling**: For handling queries, we just need to find the root of the given vertex and then retrieve the k-th largest element. This can be better managed by using a priority queue to keep track of the vertices in each component.

### Refined Solution:
```cpp
#include <iostream>
#include <vector>
#include <queue>
using namespace std;

struct UnionFind {
    vector<int> parent;
    vector<int> size;
    vector<vector<int>> components;

    UnionFind(int n) {
        parent.resize(n);
        size.resize(n, 1); // size[i] represents the size of the component containing i
        for (int i = 0; i < n; ++i) {
            parent[i] = i;
        }
    }

    int find(int u) {
        if (parent[u] != u) parent[u] = find(parent[u]);
        return parent[u];
    }

    void unite(int u, int v) {
        int root_u = find(u);
        int root_v = find(v);
        if (root_u != root_v) {
            if (size[root_u] < size[root_v]) {
                swap(root_u, root_v);
            }
            parent[root_v] = root_u;
            size[root_u] += size[root_v];
        }
    }

    void addEdge(int u, int v) {
        unite(u, v);
    }

    void getComponent(int v, priority_queue<int, vector<int>, greater<int>>& pq) {
        int root = find(v);
        for (int u : components[root]) {
            pq.push(u);
        }
    }
};

int main() {
    int n, q;
    cin >> n >> q;
    UnionFind uf(n);
    
    while (q--) {
        int ty, u, v, k;
        cin >> ty >> u >> v;
        if (ty == 1) {
            uf.addEdge(u, v);
        } else if (ty == 2) {
            cin >> k;
            priority_queue<int, vector<int>, greater<int>> pq;
            uf.getComponent(v, pq);
            if (k > pq.size()) {
                cout << -1 << endl;
            } else {
                while (pq.size() > k) pq.pop();
                cout << pq.top() << endl;
            }
        }
    }
    return 0;
}
```

### Explanation:
- The `UnionFind` structure is simplified to manage parent and size arrays.
- We use a priority queue to efficiently find the k-th largest vertex in the component connected to a given vertex.
- For merging components, we use the standard path compression and size-based union techniques.
- For queries, we retrieve the k-th largest element by popping elements from the priority queue until we reach the k-th largest element or until the queue is empty (in which case we return -1).
