To fix this code, we need to address the issue where the incorrect output is produced for certain queries. The problem involves maintaining connected components in a graph and efficiently answering queries about the k-th largest vertex in a component. The initial approach used a Union-Find data structure with a sorted list of vertices for each component, but it failed to correctly handle the merging of these lists, leading to incorrect results.

### Approaches
1. **Union-Find with Sorted Lists**: The idea is to maintain each connected component as a sorted list (in descending order) of its vertices. When merging two components, we merge their sorted lists efficiently.
2. **Efficient Merging**: The merging of two sorted lists should be done in O(m + n) time, where m and n are the sizes of the lists. However, since k can be up to 10, we only need to keep the top 10 elements in each merged list to answer queries efficiently.
3. **Handling Queries**: For a Type 2 query, we access the sorted list of the component containing the vertex `v` and return the k-th largest element if it exists; otherwise, return -1.

### Solution Code
```cpp
#include "bits/stdc++.h"

using namespace std;

#define ff first
#define ss second
#define ll long long
#define SZ(s) (int)s.size()

int const N = 2e5 + 5;

vector <vector <int>> v;

vector <int> p;

int par(int x) {
    if(p[x] == x) return x;
    return p[x] = par(p[x]);
}

void uni(int a, int b) {
    a = par(a), b = par(b);
    if(a == b) return;
    if (SZ(v[a]) < SZ(v[b])) {
        swap(a, b);
    }
    vector<int> merged;
    int i = 0, j = 0;
    while ((i < SZ(v[a]) || j < SZ(v[b])) && SZ(merged) < 10) {
        if (j == SZ(v[b]) || (i < SZ(v[a]) && v[a][i] >= v[b][j])) {
            merged.push_back(v[a][i]);
            i++;
        } else {
            merged.push_back(v[b][j]);
            j++;
        }
    }
    v[a] = merged;
    p[b] = a;
    return;
}

signed main() {
    ios_base::sync_with_stdio(false); cin.tie(nullptr);

    int n, q;
    cin >> n >> q;
    p.resize(n+1);
    v.resize(n+1);
    for(int i = 1; i <= n; i++) {
        p[i] = i;
        v[i].push_back(i);
    }
    while(q--) {
        int t, a, b;
        cin >> t >> a >> b;
        if(t == 1) {
            uni(a, b);
        }
        else {
            a = par(a);
            cout << (SZ(v[a]) < b ? -1 : v[a][b-1]) << '\n';
        }
    }

    return 0;
}
```

### Explanation
1. **Union-Find Setup**: The `par` function implements path compression to efficiently find the root of a vertex. The `uni` function merges two components by their roots, ensuring the larger component absorbs the smaller one to maintain efficiency.
2. **Merging Sorted Lists**: During the union operation, the sorted lists of the two components are merged into a single list, keeping only the top 10 elements. This ensures that any subsequent query for the k-th largest element (where k ≤ 10) can be answered in constant time.
3. **Query Handling**: For Type 2 queries, the code checks the size of the sorted list of the component containing the queried vertex. If the list has at least `k` elements, it returns the k-th largest; otherwise, it returns -1.

This approach efficiently handles the dynamic nature of the graph while ensuring queries are answered in optimal time, adhering to the problem constraints.
