To fix this code, we need to address the issue where the k-th largest vertex number among the connected components is not correctly identified. The original code attempts to maintain a sorted list of vertex numbers for each connected component and merges these lists during union operations. However, the merging process and the way the lists are maintained are flawed, leading to incorrect results.

### Approaches
1. **Union-Find Data Structure**: We use the Union-Find (Disjoint Set Union, DSU) data structure to manage the connected components efficiently.
2. **Maintaining Sorted Lists**: For each connected component, we maintain a sorted list (in descending order) of the vertex numbers. This allows us to quickly access the k-th largest element.
3. **Efficient Merging**: When merging two components, we merge their sorted lists while keeping the combined list sorted. Given the constraint that k ≤ 10, we only need to keep the top 10 elements in each list to answer queries efficiently.

### Solution Code
```cpp
#include <bits/stdc++.h>

using namespace std;

#define ff first
#define ss second
#define ll long long
#define SZ(s) (int)s.size()

int const N = 2e5 + 5;

vector<vector<int>> v;
vector<int> p;

int par(int x) {
    if (p[x] == x) return x;
    return p[x] = par(p[x]);
}

void uni(int a, int b) {
    a = par(a), b = par(b);
    if (a == b) return;
    if (SZ(v[a]) < SZ(v[b])) swap(a, b);
    vector<int> merged;
    int i = 0, j = 0;
    while ((i < SZ(v[a]) || j < SZ(v[b])) && SZ(merged) < 10) {
        if (j >= SZ(v[b]) || (i < SZ(v[a]) && v[a][i] >= v[b][j])) {
            merged.push_back(v[a][i]);
            i++;
        } else {
            merged.push_back(v[b][j]);
            j++;
        }
    }
    v[a] = merged;
    p[b] = a;
}

int main() {
    ios_base::sync_with_stdio(false); cin.tie(nullptr);

    int n, q;
    cin >> n >> q;
    p.resize(n + 1);
    v.resize(n + 1);
    for (int i = 1; i <= n; i++) {
        p[i] = i;
        v[i] = {i};
    }
    while (q--) {
        int t, a, b;
        cin >> t >> a >> b;
        if (t == 1) {
            uni(a, b);
        } else {
            a = par(a);
            if (SZ(v[a]) < b) {
                cout << -1 << '\n';
            } else {
                cout << v[a][b - 1] << '\n';
            }
        }
    }

    return 0;
}
```

### Explanation
1. **Union-Find Data Structure**: The `par` function implements path compression to efficiently find the root of a vertex. The `uni` function merges two sets by size, ensuring the smaller set is merged into the larger one for efficiency.
2. **Maintaining Sorted Lists**: Each vertex initially has a list containing only itself. During a union operation, the lists of the two sets being merged are combined into a single sorted list (in descending order). We only keep the top 10 elements to optimize for queries where k ≤ 10.
3. **Query Handling**: For a type 2 query, the code checks the size of the connected component's list. If the list size is less than k, it returns -1; otherwise, it returns the k-th largest element (0-indexed as `v[a][k-1]`).

This approach ensures efficient handling of both union and query operations, adhering to the constraints and providing correct results for all test cases.
