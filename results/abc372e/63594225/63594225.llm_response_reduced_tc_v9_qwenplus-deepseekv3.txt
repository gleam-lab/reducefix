To fix this code, we need to ensure that the union operation correctly merges the sorted lists of vertices connected to each node and maintains the top k elements efficiently. The incorrect output in the failing case suggests that the merging process is not correctly maintaining the sorted order or is not handling the top k elements properly.

### Approaches
1. **Union-Find Data Structure**: This helps efficiently manage the connected components of the graph.
2. **Maintaining Sorted Lists**: For each component, maintain a sorted list of vertices in descending order. This allows quick access to the k-th largest element.
3. **Efficient Merging**: When merging two components, merge their sorted lists and keep only the top 10 elements to ensure queries are answered in O(1) time.

### Solution Code
```cpp
#include <bits/stdc++.h>

using namespace std;

#define ff first
#define ss second
#define ll long long
#define SZ(s) (int)s.size()

int const N = 2e5 + 5;

vector<set<int, greater<int>>> v;

vector<int> p;

int par(int x) {
    if (p[x] == x) return x;
    return p[x] = par(p[x]);
}

void uni(int a, int b) {
    a = par(a), b = par(b);
    if (a == b) return;
    if (SZ(v[a]) < SZ(v[b])) swap(a, b);
    for (int x : v[b]) {
        v[a].insert(x);
        if (SZ(v[a]) > 10) {
            auto it = prev(v[a].end());
            v[a].erase(it);
        }
    }
    p[b] = a;
    return;
}

signed main() {
    ios_base::sync_with_stdio(false); cin.tie(nullptr);

    int n, q;
    cin >> n >> q;
    p.resize(n + 1);
    v.resize(n + 1);
    for (int i = 1; i <= n; i++) {
        p[i] = i;
        v[i].insert(i);
    }
    while (q--) {
        int t, a, b;
        cin >> t >> a >> b;
        if (t == 1) {
            uni(a, b);
        }
        else {
            a = par(a);
            if (SZ(v[a]) < b) {
                cout << -1 << '\n';
            }
            else {
                auto it = v[a].begin();
                advance(it, b - 1);
                cout << *it << '\n';
            }
        }
    }

    return 0;
}
```

### Explanation
1. **Union-Find Initialization**: The `p` array is initialized to point each node to itself, indicating that each node is its own parent initially. The `v` array is a vector of sets, with each set initialized to contain only the node itself.
2. **Union Operation**: When merging two components, the larger set absorbs the smaller one. The elements of the smaller set are inserted into the larger set, maintaining the sorted order in descending order. If the size exceeds 10, the smallest elements are removed to keep only the top 10 elements.
3. **Query Handling**: For a query of type 2, the parent of the node is found, and if the size of the set is sufficient, the k-th largest element is retrieved by advancing the iterator to the appropriate position in the set.

This approach ensures that each union operation is handled efficiently (O(1) for smaller sets and O(log n) for larger sets due to the set operations), and queries are answered in O(1) time by directly accessing the k-th element in the sorted set. The solution efficiently handles the constraints while maintaining correctness.
