To fix this code, we need to address the inefficiency and correctness issues in the original implementation. The main problem lies in the way the connected components are merged and the top k elements are maintained. The original approach merges two sorted lists and keeps only the top 100 elements, which is not efficient and can lead to incorrect results if the merged list is not properly maintained.

### Approaches
1. **Union-Find Data Structure**: This will help efficiently manage the connected components.
2. **Maintaining Top Elements**: For each connected component, maintain a set (or a priority queue) of the top k elements (where k is up to 10 as per the problem constraints). This ensures quick access to the k-th largest element.
3. **Efficient Merging**: When merging two components, combine their top elements and keep only the top 10 to maintain efficiency.

### Solution Code
```cpp
#include "bits/stdc++.h"

using namespace std;

#define ff first
#define ss second
#define ll long long
#define SZ(s) (int)s.size()

int const N = 2e5 + 5;

vector <set<int>> top_set;
vector <int> p;

int par(int x) {
    if(p[x] == x) return x;
    return p[x] = par(p[x]);
}

void uni(int a, int b) {
    a = par(a), b = par(b);
    if(a == b) return;
    if (top_set[a].size() < top_set[b].size()) {
        swap(a, b);
    }
    // Merge smaller set into larger set
    for (int num : top_set[b]) {
        top_set[a].insert(num);
        if (top_set[a].size() > 10) {
            top_set[a].erase(top_set[a].begin());
        }
    }
    p[b] = a;
    return;
}

int main() {
    ios_base::sync_with_stdio(false); cin.tie(nullptr);

    int n, q;
    cin >> n >> q;
    p.resize(n+1);
    top_set.resize(n+1);
    for(int i = 1; i <= n; i++) {
        p[i] = i;
        top_set[i].insert(i);
    }
    while(q--) {
        int t, a, b;
        cin >> t >> a >> b;
        if(t == 1) {
            uni(a, b);
        }
        else {
            a = par(a);
            if (top_set[a].size() < b) {
                cout << "-1\n";
            } else {
                auto it = top_set[a].end();
                advance(it, -b);
                cout << *it << '\n';
            }
        }
    }

    return 0;
}
```

### Explanation
1. **Union-Find Data Structure**: The `par` function finds the root parent of a node with path compression for efficiency. The `uni` function merges two sets by size, ensuring the smaller set is merged into the larger one to keep operations efficient.
2. **Top Elements Maintenance**: Each node's connected component maintains a set of the top 10 largest elements. During a merge, elements from the smaller set are inserted into the larger set, and if the size exceeds 10, the smallest elements are removed to maintain the top 10.
3. **Query Handling**: For Type 2 queries, the k-th largest element is accessed from the set in reverse order. If the set size is smaller than k, -1 is returned.

This approach ensures efficient handling of both union and query operations, adhering to the problem constraints and providing correct results.
