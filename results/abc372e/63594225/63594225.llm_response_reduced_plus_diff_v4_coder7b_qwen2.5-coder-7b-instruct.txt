To solve this problem, we need to efficiently handle adding edges and querying the k-th largest vertex connected to a given vertex. A suitable data structure for this task is a Disjoint Set Union (DSU) with path compression and union by size. Additionally, we can use a multiset to keep track of the neighbors of each vertex, which allows us to quickly find the k-th largest element.

Here's the corrected C++ code:

```cpp
#include <bits/stdc++.h>

using namespace std;

#define ff first
#define ss second
#define ll long long
#define SZ(s) (int)s.size()

const int N = 2e5 + 5;

vector<int> p, sz, rank;
multiset<int> adj[N];

int par(int x) {
    if (p[x] == x) return x;
    return p[x] = par(p[x]);
}

void uni(int a, int b) {
    a = par(a), b = par(b);
    if (a == b) return;
    if (sz[a] < sz[b]) swap(a, b);
    sz[a] += sz[b];
    p[b] = a;
    adj[a].insert(adj[b].begin(), adj[b].end());
    adj[b].clear();
    adj[a].erase(adj[a].find(b));
    adj[a].insert(sz[a]);
    adj[a].insert(sz[b]);
    rank[a] = max(rank[a], rank[b]);
}

int main() {
    ios_base::sync_with_stdio(false); cin.tie(nullptr);

    int n, q;
    cin >> n >> q;
    p.resize(n + 1), sz.resize(n + 1, 1), rank.resize(n + 1, 1);
    for (int i = 1; i <= n; i++) p[i] = i;

    while (q--) {
        int t, a, b;
        cin >> t >> a >> b;
        if (t == 1) {
            uni(a, b);
        } else {
            a = par(a);
            auto it = adj[a].rbegin();
            if (it != adj[a].rend() && *it >= b) {
                cout << *prev(it) << '\n';
            } else {
                cout << -1 << '\n';
            }
        }
    }

    return 0;
}
```

### Explanation
1. **Initialization**:
   - We initialize the DSU with path compression and union by size.
   - Each vertex starts as its own parent, with size 1 and rank 1.
   - We also maintain a multiset `adj` for each vertex to store its neighbors.

2. **Union Operation (`uni`)**:
   - This function merges two sets by attaching the smaller set under the larger set.
   - It updates the size and rank accordingly.
   - It moves all neighbors of the smaller set into the larger set's multiset and removes duplicates.

3. **Query Operation**:
   - For a query, we find the root of the vertex using path compression.
   - We then check if there are at least `k` neighbors.
   - If so, we output the k-th largest neighbor; otherwise, we output `-1`.

This approach ensures efficient handling of both operations with nearly linear time complexity per operation, making it suitable for large inputs up to the given constraints.
