To fix this code, we need to address the issue where the incorrect output is generated for certain queries. The problem arises because the merging of sorted lists during the union operation is not correctly maintaining the top k largest elements in each connected component. 

### Approaches
The correct approach involves:
1. **Union-Find Data Structure**: This helps in efficiently managing the connected components.
2. **Sorted Lists**: For each connected component, maintain a sorted list of the vertices (in descending order) to quickly access the k-th largest element.
3. **Efficient Merging**: When merging two components, merge their sorted lists and keep only the top 10 elements to ensure the operation remains efficient.

### Solution Code
```cpp
#include <bits/stdc++.h>

using namespace std;

#define ff first
#define ss second
#define ll long long
#define SZ(s) (int)s.size()

int const N = 2e5 + 5;

vector<vector<int>> v;

vector<int> p;

int par(int x) {
    if(p[x] == x) return x;
    return p[x] = par(p[x]);
}

void uni(int a, int b) {
    a = par(a), b = par(b);
    if(a == b) return;
    if (SZ(v[a]) < SZ(v[b])) {
        swap(a, b);
    }
    vector<int> merged;
    int ai = 0, bi = 0;
    while ((ai < SZ(v[a]) || bi < SZ(v[b])) && SZ(merged) < 10) {
        if (bi == SZ(v[b]) || (ai < SZ(v[a]) && v[a][ai] > v[b][bi])) {
            merged.push_back(v[a][ai]);
            ai++;
        } else {
            merged.push_back(v[b][bi]);
            bi++;
        }
    }
    v[a] = merged;
    p[b] = a;
    return;
}

int main() {
    ios_base::sync_with_stdio(false); cin.tie(nullptr);

    int n, q;
    cin >> n >> q;
    p.resize(n+1);
    v.resize(n+1);
    for(int i = 1; i <= n; i++) {
        p[i] = i;
        v[i].push_back(i);
    }
    while(q--) {
        int t, a, b;
        cin >> t >> a >> b;
        if(t == 1) {
            uni(a, b);
        }
        else {
            a = par(a);
            cout << (SZ(v[a]) < b ? -1 : v[a][b-1]) << '\n';
        }
    }

    return 0;
}
```

### Explanation
1. **Union-Find Data Structure**: The `par` function uses path compression to find the root of a vertex efficiently.
2. **Sorted Lists**: Each vertex initially has its own list containing only itself. During a union operation, the lists of the two components are merged while maintaining the order in descending order.
3. **Efficient Merging**: The `uni` function merges the lists of the two components and keeps only the top 10 elements, ensuring that the operations remain efficient and within the constraints.
4. **Query Handling**: For type 2 queries, the code checks if the connected component has at least `k` elements and returns the k-th largest (or -1 if not enough elements).

This approach ensures that the solution is both time and space efficient, handling the constraints effectively.
