To fix this code, we need to address several issues related to efficiency and correctness, especially considering the constraints where both N and Q can be up to 2×10^5. The original approach attempts to maintain a sorted list of connected vertices for each component, but it has several flaws:

1. **Merging Lists Inefficiently**: The merging of two sorted lists is done in O(n + m) time, which is too slow for large inputs.
2. **Excessive Memory Usage**: Keeping lists of up to 100 elements for each component is unnecessary and can lead to high memory usage.
3. **Incorrect Handling of Duplicates**: The merging process might not correctly handle duplicates or maintain the order properly.

### Approaches
The optimal approach involves using Union-Find (Disjoint Set Union, DSU) with path compression and union by size or rank. Additionally, for each component, we can maintain a sorted list of the top 10 largest vertices. This ensures that the operations are efficient and within the problem constraints.

1. **Union-Find Data Structure**: To efficiently manage the connected components.
2. **Maintaining Top K Elements**: For each component, maintain a sorted list (in descending order) of the top 10 largest vertices. This allows quick access to the k-th largest vertex in O(1) time after merging.

### Solution Code
```cpp
#include <bits/stdc++.h>
using namespace std;

const int N = 2e5 + 5;

vector<int> parent;
vector<priority_queue<int, vector<int>, greater<int>>> top_k;

int find_set(int v) {
    if (v == parent[v])
        return v;
    return parent[v] = find_set(parent[v]);
}

void union_sets(int a, int b) {
    a = find_set(a);
    b = find_set(b);
    if (a != b) {
        if (top_k[a].size() < top_k[b].size())
            swap(a, b);
        parent[b] = a;
        while (!top_k[b].empty()) {
            top_k[a].push(top_k[b].top());
            top_k[b].pop();
            if (top_k[a].size() > 10)
                top_k[a].pop();
        }
    }
}

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);

    int n, q;
    cin >> n >> q;
    parent.resize(n + 1);
    top_k.resize(n + 1);
    for (int i = 1; i <= n; ++i) {
        parent[i] = i;
        top_k[i].push(i);
    }

    while (q--) {
        int t, a, b;
        cin >> t >> a >> b;
        if (t == 1) {
            union_sets(a, b);
        } else {
            a = find_set(a);
            if (top_k[a].size() < b) {
                cout << -1 << '\n';
            } else {
                vector<int> temp;
                while (!top_k[a].empty()) {
                    temp.push_back(top_k[a].top());
                    top_k[a].pop();
                }
                cout << temp[temp.size() - b] << '\n';
                for (int num : temp) {
                    top_k[a].push(num);
                }
            }
        }
    }

    return 0;
}
```

### Explanation
1. **Union-Find with Path Compression**: The `find_set` function uses path compression to ensure that future queries are faster. The `union_sets` function merges two sets by size, ensuring the larger set absorbs the smaller one.
2. **Top K Elements**: Each component's vertices are stored in a min-heap (priority queue) limited to the top 10 largest vertices. This allows efficient merging and access.
3. **Query Handling**: For type 2 queries, the k-th largest vertex is retrieved by temporarily storing the elements in a vector, accessing the k-th largest element, and then restoring the heap.

This approach ensures that each operation is performed in logarithmic time relative to the size of the components, making it efficient for large inputs.
