To fix this code, we need to address several issues related to efficiency and correctness in handling the union operations and queries. The original code attempts to maintain sorted lists of vertices in each connected component, but it does this inefficiently and fails to handle large inputs within the constraints.

### Approaches
1. **Union-Find Data Structure**: We use the union-find (disjoint set) data structure to efficiently manage and merge connected components.
2. **Maintaining Sorted Lists**: For each component, we maintain a sorted list (in descending order) of the vertices. When two components are merged, we merge their sorted lists efficiently using a two-pointer technique and keep only the top 10 elements (since `k` is at most 10).
3. **Handling Queries**: For type 2 queries, we directly access the k-th largest element from the sorted list of the component's vertices, if it exists.

### Solution Code
```cpp
#include <bits/stdc++.h>

using namespace std;

#define ff first
#define ss second
#define ll long long
#define SZ(s) (int)s.size()

int const N = 2e5 + 5;

vector<vector<int>> v;
vector<int> p;

int par(int x) {
    if (p[x] == x) return x;
    return p[x] = par(p[x]);
}

void uni(int a, int b) {
    a = par(a), b = par(b);
    if (a == b) return;
    if (SZ(v[a]) < SZ(v[b])) swap(a, b);
    vector<int> merged;
    int ai = 0, bi = 0;
    while ((ai < SZ(v[a]) || bi < SZ(v[b])) && merged.size() < 10) {
        if (bi >= SZ(v[b]) || (ai < SZ(v[a]) && v[a][ai] > v[b][bi])) {
            merged.push_back(v[a][ai]);
            ai++;
        } else {
            merged.push_back(v[b][bi]);
            bi++;
        }
    }
    v[a] = merged;
    p[b] = a;
}

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);

    int n, q;
    cin >> n >> q;
    p.resize(n + 1);
    v.resize(n + 1);
    for (int i = 1; i <= n; i++) {
        p[i] = i;
        v[i] = {i};
    }
    while (q--) {
        int t, a, b;
        cin >> t >> a >> b;
        if (t == 1) {
            uni(a, b);
        } else {
            a = par(a);
            if (SZ(v[a]) < b) {
                cout << -1 << '\n';
            } else {
                cout << v[a][b - 1] << '\n';
            }
        }
    }

    return 0;
}
```

### Explanation
1. **Initialization**: We initialize the union-find structure (`p`) and a vector of vectors (`v`) to hold sorted lists of vertices for each component.
2. **Union Operation**: When merging two components, we merge their sorted lists while keeping only the top 10 elements. This is done efficiently using a two-pointer approach to merge the lists in descending order.
3. **Query Handling**: For type 2 queries, after finding the root of the component containing vertex `v`, we check if the component has at least `k` vertices. If so, we return the k-th largest vertex; otherwise, we return `-1`.

This approach ensures that the operations are efficient and the solution handles the constraints effectively, especially the large input sizes. The merging of sorted lists is optimized to keep only the necessary elements, which keeps the operations within feasible time limits.
