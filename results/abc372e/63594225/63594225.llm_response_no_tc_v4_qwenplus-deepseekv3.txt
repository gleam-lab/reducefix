To fix this code, we need to address the inefficiency in merging the connected component lists and ensure that the implementation correctly handles the constraints, especially the large input sizes. The original code attempts to merge two sorted lists every time a union operation is performed, which can lead to a time complexity that is too high for the given constraints (N and Q up to 2×10^5). Instead, we should use a more efficient approach, such as maintaining a priority queue or a set for each connected component to keep track of the top k elements.

### Approaches
1. **Union-Find Data Structure**: This will help efficiently manage the connected components of the graph.
2. **Maintaining Sorted Lists**: For each connected component, maintain a sorted list (or a set) of the vertices in descending order. When merging two components, merge their corresponding sorted lists and keep only the top 10 elements to answer queries efficiently.
3. **Efficient Merging**: Instead of merging the entire lists every time, which can be costly, we can merge the top elements from both lists and keep only the top 10 elements to ensure quick access during queries.

### Solution Code
```cpp
#include <bits/stdc++.h>

using namespace std;

#define ff first
#define ss second
#define ll long long
#define SZ(s) (int)s.size()

int const N = 2e5 + 5;

vector<set<int>> v;
vector<int> p;

int par(int x) {
    if (p[x] == x) return x;
    return p[x] = par(p[x]);
}

void uni(int a, int b) {
    a = par(a), b = par(b);
    if (a == b) return;
    
    if (v[a].size() < v[b].size()) {
        swap(a, b);
    }
    
    for (int num : v[b]) {
        v[a].insert(num);
    }
    v[b].clear();
    
    while (v[a].size() > 20) {
        v[a].erase(v[a].begin());
    }
    
    p[b] = a;
}

signed main() {
    ios_base::sync_with_stdio(false); cin.tie(nullptr);

    int n, q;
    cin >> n >> q;
    p.resize(n + 1);
    v.resize(n + 1);
    for (int i = 1; i <= n; i++) {
        p[i] = i;
        v[i].insert(i);
    }
    while (q--) {
        int t, a, b;
        cin >> t >> a >> b;
        if (t == 1) {
            uni(a, b);
        } else {
            a = par(a);
            if (SZ(v[a]) < b) {
                cout << -1 << '\n';
            } else {
                auto it = v[a].rbegin();
                advance(it, b - 1);
                cout << *it << '\n';
            }
        }
    }

    return 0;
}
```

### Explanation
1. **Union-Find Data Structure**: The `par` function finds the root of a vertex using path compression to optimize future queries. The `uni` function merges two sets by always attaching the smaller set to the larger one to keep the tree flat.
2. **Maintaining Sorted Lists**: Each connected component maintains a set of vertex numbers in descending order. This allows efficient insertion and deletion operations.
3. **Efficient Merging**: When merging two sets, the smaller set is merged into the larger one to minimize the number of operations. The merged set is then trimmed to keep only the top 20 elements (more than the required 10 for safety) to ensure quick access during queries.
4. **Query Handling**: For each query of type 2, the code checks if the connected component has at least `k` elements. If not, it returns -1; otherwise, it retrieves the `k`-th largest element from the set in descending order.

This approach ensures that the solution is efficient and handles the constraints effectively, providing quick responses to both types of queries.
