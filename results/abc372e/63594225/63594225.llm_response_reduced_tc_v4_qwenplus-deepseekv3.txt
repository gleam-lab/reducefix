To fix this code, we need to ensure that the k-th largest vertex in the connected component of vertex v is correctly identified. The issue in the original code arises from the way the connected components are merged and how the vertices are stored and sorted. The merging process should maintain a sorted list of vertices in descending order, and the k-th largest element should be accessed correctly.

### Approaches
1. **Union-Find Data Structure**: This will help efficiently manage the connected components.
2. **Maintaining Sorted Lists**: Each component will maintain a sorted list (in descending order) of its vertices. When two components are merged, their lists are merged while maintaining the sorted order.
3. **Efficient Merging**: To handle large inputs efficiently, we limit the size of the lists to the top 10 elements (since k can be at most 10), which reduces the complexity of merging operations.

### Solution Code
```cpp
#include "bits/stdc++.h"

using namespace std;

#define ff first
#define ss second
#define ll long long
#define SZ(s) (int)s.size()

int const N = 2e5 + 5;

vector <vector <int>> v;

vector <int> p;

int par(int x) {
    if(p[x] == x) return x;
    return p[x] = par(p[x]);
}

void uni(int a, int b) {
    a = par(a), b = par(b);
    if(a == b) return;
    if (SZ(v[a]) < SZ(v[b])) {
        swap(a, b);
    }
    vector<int> merged;
    int i = 0, j = 0;
    while (i < SZ(v[a]) && j < SZ(v[b]) && merged.size() < 10) {
        if (v[a][i] > v[b][j]) {
            merged.push_back(v[a][i]);
            i++;
        } else {
            merged.push_back(v[b][j]);
            j++;
        }
    }
    while (i < SZ(v[a]) && merged.size() < 10) {
        merged.push_back(v[a][i]);
        i++;
    }
    while (j < SZ(v[b]) && merged.size() < 10) {
        merged.push_back(v[b][j]);
        j++;
    }
    v[a] = merged;
    p[b] = a;
    return;
}

signed main() {
    ios_base::sync_with_stdio(false); cin.tie(nullptr);

    int n, q;
    cin >> n >> q;
    p.resize(n+1);
    v.resize(n+1);
    for(int i = 1; i <= n; i++) {
        p[i] = i;
        v[i].push_back(i);
    }
    while(q--) {
        int t, a, b;
        cin >> t >> a >> b;
        if(t == 1) {
            uni(a, b);
        }
        else {
            a = par(a);
            cout << (SZ(v[a]) < b ? -1 : v[a][b-1]) << '\n';
        }
    }

    return 0;
}
```

### Explanation
1. **Union-Find Data Structure**: The `par` function finds the root parent of a vertex, and the `uni` function merges two components. This helps efficient connectivity checks and component merging.
2. **Maintaining Sorted Lists**: Each component's vertices are stored in a list sorted in descending order. When merging two components, we merge their lists while keeping only the top 10 elements, ensuring efficient operations.
3. **Handling Queries**: For each query, if it is a type 1 (edge addition), we merge the components of the two vertices. If it is a type 2 query, we access the k-th largest vertex in the component of the given vertex by checking the maintained sorted list.

This approach ensures that the operations are efficient and correct, handling all edge cases and constraints effectively.
