To fix this code, we need to ensure that the union operation correctly merges the sorted lists of vertices while maintaining the top k elements efficiently. The original approach tried to merge two sorted lists and keep only the top 100 elements, but this method might not always preserve the correct order or enough elements to handle queries involving the k-th largest vertex, especially when k is up to 10.

### Approaches
1. **Union-Find Data Structure**: This will help efficiently manage the connected components of the graph.
2. **Balanced Binary Search Trees (BSTs)**: For each connected component, maintain a sorted list of vertices. When merging two components, we merge their BSTs to keep the vertices sorted. This allows efficient access to the k-th largest element.
3. **Efficient Merging**: Instead of merging lists directly, we use the BSTs (specifically `std::set` in C++) to maintain the vertices in sorted order. Merging two sets can be done efficiently by inserting elements from the smaller set into the larger one.

### Solution Code
```cpp
#include "bits/stdc++.h"

using namespace std;

#define ff first
#define ss second
#define ll long long
#define SZ(s) (int)s.size()

int const N = 2e5 + 5;

vector<set<int, greater<int>>> v;

vector<int> p;

int par(int x) {
    if (p[x] == x) return x;
    return p[x] = par(p[x]);
}

void uni(int a, int b) {
    a = par(a), b = par(b);
    if (a == b) return;
    if (v[a].size() < v[b].size()) swap(a, b);
    for (auto x : v[b]) {
        v[a].insert(x);
    }
    v[b].clear();
    p[b] = a;
    return;
}

signed main() {
    ios_base::sync_with_stdio(false); cin.tie(nullptr);

    int n, q;
    cin >> n >> q;
    p.resize(n + 1);
    v.resize(n + 1);
    for (int i = 1; i <= n; i++) {
        p[i] = i;
        v[i].insert(i);
    }
    while (q--) {
        int t, a, b;
        cin >> t >> a >> b;
        if (t == 1) {
            uni(a, b);
        }
        else {
            a = par(a);
            if (v[a].size() < b) {
                cout << -1 << '\n';
            }
            else {
                auto it = v[a].begin();
                advance(it, b - 1);
                cout << *it << '\n';
            }
        }
    }

    return 0;
}
```

### Explanation
1. **Union-Find Data Structure**: The `par` function implements path compression to efficiently find the root parent of each vertex.
2. **Balanced BSTs**: Each connected component is represented by a `set` sorted in descending order. This allows efficient insertion and retrieval operations.
3. **Merging Components**: The `uni` function merges two sets by inserting all elements of the smaller set into the larger one. This ensures that the merging operation is efficient, with a time complexity of O(M log N), where M is the size of the smaller set.
4. **Query Handling**: For each Type 2 query, the code checks the size of the set associated with the vertex's component. If the size is sufficient, it retrieves the k-th largest element by advancing the iterator to the appropriate position.

This approach ensures that the operations are efficient and the solution handles the constraints effectively. The use of BSTs guarantees that the vertices are always maintained in a sorted order, allowing quick access to the k-th largest element.
