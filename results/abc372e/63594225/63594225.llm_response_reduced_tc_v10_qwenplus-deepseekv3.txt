To fix this code, we need to ensure that when merging two sets in the Union-Find data structure, the merged set maintains the top-k largest elements correctly, where k is up to 10 as per the problem constraints. The original code attempts to keep up to 100 elements, but this is inefficient and unnecessary. Instead, we should only keep the top 10 elements to match the query constraints (k ≤ 10).

### Approaches
1. **Union-Find with Path Compression**: This helps in efficiently finding the root of a vertex.
2. **Maintaining Top-k Elements**: For each set, we maintain a sorted list of the top 10 largest elements in descending order. When merging two sets, we merge their top elements and keep only the top 10 to optimize performance.
3. **Efficient Merging**: When merging two sets, we use a two-pointer technique to merge their top elements in O(k) time, where k is 10, ensuring the operation is O(1) per merge.

### Solution Code
```cpp
#include "bits/stdc++.h"

using namespace std;

#define ff first
#define ss second
#define ll long long
#define SZ(s) (int)s.size()

int const N = 2e5 + 5;

vector <vector <int>> v;

vector <int> p;

int par(int x) {
    if(p[x] == x) return x;
    return p[x] = par(p[x]);
}

void uni(int a, int b) {
    a = par(a), b = par(b);
    if(a == b) return;
    if (SZ(v[a]) < SZ(v[b])) {
        swap(a, b);
    }
    vector<int> merged;
    int i = 0, j = 0;
    while ((i < SZ(v[a]) || j < SZ(v[b])) && SZ(merged) < 10) {
        if (j >= SZ(v[b]) || (i < SZ(v[a]) && v[a][i] >= v[b][j])) {
            merged.push_back(v[a][i]);
            i++;
        } else {
            merged.push_back(v[b][j]);
            j++;
        }
    }
    v[a] = merged;
    p[b] = a;
    return;
}

signed main() {
    ios_base::sync_with_stdio(false); cin.tie(nullptr);

    int n, q;
    cin >> n >> q;
    p.resize(n+1);
    v.resize(n+1);
    for(int i = 1; i <= n; i++) {
        p[i] = i;
        v[i].push_back(i);
    }
    while(q--) {
        int t, a, b;
        cin >> t >> a >> b;
        if(t == 1) {
            uni(a, b);
        }
        else {
            a = par(a);
            cout << (SZ(v[a]) < b ? -1 : v[a][b-1]) << '\n';
        }
    }

    return 0;
}
```

### Explanation
1. **Union-Find Data Structure**: This is used to manage the connected components. The `par` function implements path compression to ensure efficient root finding.
2. **Top-k Maintenance**: Each component's root maintains a list of the top 10 largest vertices in descending order. The `uni` function merges two components by combining their top 10 lists, ensuring the merged list is also in descending order and contains only the top 10 elements.
3. **Query Processing**: For a Type 2 query, the code checks if the requested k-th largest element exists in the top 10 list of the component and returns it or -1 if k exceeds the list size.

This approach ensures efficient handling of up to 200,000 vertices and queries, adhering to the problem constraints and providing optimal performance.
