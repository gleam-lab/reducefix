To solve this problem efficiently, we need to support two types of queries on an initially disconnected undirected graph:

1. **Add an edge** between two vertices.
2. **Query the k-th largest vertex number** in the connected component that includes a given vertex.

Given the constraints (up to 200,000 vertices and 200,000 queries), a naive approach won't work. We must use a data structure that supports efficient union operations and retrieval of top-k elements from a set.

### Approach:
We use the **Disjoint Set Union (DSU)** with path compression for managing connected components efficiently. Each component maintains a sorted list (or multiset) of its vertices in descending order so that we can answer "k-th largest" queries quickly.

To manage these sorted lists, we utilize `std::multiset` in C++, which allows:
- Insertion in O(log N)
- Merging two sets efficiently by moving elements from the smaller set into the larger one (ensuring amortized logarithmic complexity overall).

Each DSU node keeps track of:
- Its parent (`parent[]`)
- A multiset containing all the vertices in its connected component (`component[]`)

When merging two components:
- Always merge the smaller into the larger to reduce overhead
- This ensures amortized efficiency

For Type 2 queries:
- If the size of the component is less than `k`, output `-1`
- Otherwise, move a pointer `k-1` steps forward from the beginning of the multiset and return that value

This solution ensures acceptable performance within the constraints.

---

Here's the corrected and optimized implementation:

```cpp
#include <bits/stdc++.h>
using namespace std;

const int MAXN = 2e5 + 5;

int parent[MAXN];
multiset<int, greater<int>> component[MAXN];

// Find with path compression
int find(int x) {
    if (parent[x] != x)
        parent[x] = find(parent[x]);
    return parent[x];
}

// Union by size
void unite(int u, int v) {
    int root_u = find(u);
    int root_v = find(v);

    if (root_u == root_v)
        return;

    // Always merge smaller into larger
    if (component[root_u].size() < component[root_v].size()) {
        swap(root_u, root_v);
        swap(component[root_u], component[root_v]);
    }

    // Merge the smaller set into the larger one
    component[root_u].insert(component[root_v].begin(), component[root_v].end());
    parent[root_v] = root_u;
    component[root_v].clear(); // Clear the merged set
}

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);

    int N, Q;
    cin >> N >> Q;

    // Initialize DSU and each component
    for (int i = 1; i <= N; ++i) {
        parent[i] = i;
        component[i].insert(i);
    }

    while (Q--) {
        int type;
        cin >> type;

        if (type == 1) {
            int u, v;
            cin >> u >> v;
            unite(u, v);
        } else {
            int v, k;
            cin >> v >> k;

            int root = find(v);
            auto& comp = component[root];

            if (comp.size() < (size_t)k) {
                cout << -1 << '\n';
            } else {
                auto it = comp.begin();
                advance(it, k - 1);
                cout << *it << '\n';
            }
        }
    }

    return 0;
}
```

### Key Points:
- Efficient merging strategy avoids unnecessary copying.
- Uses `std::multiset` with reverse ordering (`greater<int>`) for fast access to top-k values.
- Path-compressed DSU ensures almost constant time per operation.
- Amortized complexity is acceptable for large inputs.
