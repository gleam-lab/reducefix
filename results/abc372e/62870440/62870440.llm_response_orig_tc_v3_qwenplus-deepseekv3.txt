To fix this code, we need to address the incorrect handling of the Union-Find data structure and the way the connected vertices are tracked and sorted. The original code attempts to maintain a sorted list of connected vertices, but it fails to correctly merge these lists when two sets are unioned. 

### Approaches
1. **Union-Find Data Structure**: This will help efficiently manage the connected components of the graph. Each node will have a parent pointer, and path compression will be used to keep the structure efficient.
2. **Tracking Connected Vertices**: For each connected component (represented by its root), maintain a sorted list (in descending order) of the vertices in that component. This allows quick access to the k-th largest vertex.
3. **Merging Components**: When two components are merged, combine their sorted lists, keep only the top 10 elements (since k is at most 10), and sort them in descending order to maintain the required order.

### Solution Code
```cpp
#include<bits/stdc++.h>
using namespace std;

const int MAXN = 200005;
vector<int> as[MAXN];
int fa[MAXN];

int find(int x) {
    if (fa[x] != x) {
        fa[x] = find(fa[x]);
    }
    return fa[x];
}

void unite(int u, int v) {
    u = find(u);
    v = find(v);
    if (u == v) return;
    if (as[u].size() < as[v].size()) {
        swap(u, v);
    }
    fa[v] = u;
    as[u].insert(as[u].end(), as[v].begin(), as[v].end());
    sort(as[u].begin(), as[u].end(), greater<int>());
    if (as[u].size() > 10) {
        as[u].resize(10);
    }
}

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(NULL);
    
    int N, Q;
    cin >> N >> Q;
    
    for (int i = 1; i <= N; ++i) {
        fa[i] = i;
        as[i] = {i};
    }
    
    while (Q--) {
        int op, u, v;
        cin >> op >> u >> v;
        if (op == 1) {
            unite(u, v);
        } else {
            int root = find(u);
            if (v > as[root].size()) {
                cout << "-1\n";
            } else {
                cout << as[root][v-1] << "\n";
            }
        }
    }
    return 0;
}
```

### Explanation
1. **Initialization**: The `fa` array is initialized such that each node is its own parent. The `as` array (stands for "associated set") is initialized to contain each node itself in a list.
2. **Union Operation**: The `unite` function merges two sets. It ensures the larger set absorbs the smaller one to maintain efficiency. After merging, it combines their lists, sorts them in descending order, and keeps only the top 10 elements to optimize for queries where k is at most 10.
3. **Find Operation**: The `find` function uses path compression to flatten the structure of the tree, making future queries faster.
4. **Query Handling**: For each query of type 2, it checks the size of the connected component. If the requested k-th largest element exists, it outputs it; otherwise, it outputs -1.

This approach ensures efficient handling of both union and query operations, maintaining the constraints and providing correct results for all test cases.
