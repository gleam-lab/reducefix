To solve this problem, we need to efficiently handle both adding edges and querying the k-th largest vertex connected to a given vertex. This requires maintaining adjacency lists and being able to quickly access and sort these lists. Here's how you can achieve it using a combination of adjacency lists and heaps:

1. **Adjacency List Representation**: Use an adjacency list to store connections between nodes.
2. **Min Heap for K-th Largest Element**: For each node, use a min heap to keep track of the top k largest neighbors. This allows efficient updates and retrieval of the k-th largest element.

Here’s the corrected and optimized C++ code:

```cpp
#include <bits/stdc++.h>
using namespace std;

const int MAXN = 2 * 100005;
vector<int> adj[MAXN];
priority_queue<int, vector<int>, greater<int>> pq[MAXN];

void addEdge(int u, int v) {
    adj[u].push_back(v);
    adj[v].push_back(u);
}

void updateHeap(int v, int k) {
    while (!pq[v].empty() && pq[v].size() > k) {
        pq[v].pop();
    }
}

void processQueries(int n, int q) {
    for (int i = 0; i < q; ++i) {
        int op, v, k;
        cin >> op >> v >> k;
        if (op == 1) {
            int u = v;
            cin >> v;
            addEdge(u, v);

            // Update the heaps
            updateHeap(u, k);
            updateHeap(v, k);

            for (auto neighbor : adj[u]) {
                pq[neighbor].push(v);
                updateHeap(neighbor, k);
            }

            for (auto neighbor : adj[v]) {
                pq[neighbor].push(u);
                updateHeap(neighbor, k);
            }
        } else {
            if (adj[v].size() < k) {
                cout << "-1" << endl;
            } else {
                auto it = pq[v].begin();
                advance(it, k - 1);
                cout << *it << endl;
            }
        }
    }
}

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);

    int n, q;
    cin >> n >> q;

    processQueries(n, q);

    return 0;
}
```

### Explanation:
1. **Graph Representation**:
   - We use an array of vectors (`adj`) where `adj[i]` stores all neighbors of node `i`.
   
2. **Heaps**:
   - For each node `v`, we maintain a min heap (`pq[v]`) that contains at most `k` elements. This heap helps us keep track of the k largest neighbors of `v`.

3. **Adding Edges**:
   - When adding an edge between nodes `u` and `v`, we update the heaps for both nodes to ensure they only contain the k largest neighbors.

4. **Processing Queries**:
   - For each query, if it's a type 1 query, we add the edge and update the heaps accordingly.
   - If it's a type 2 query, we check if there are at least `k` neighbors. If so, we output the k-th largest neighbor using the heap. Otherwise, we output `-1`.

This approach ensures that both adding edges and querying the k-th largest neighbor are efficient, making it suitable for large inputs within the given constraints.
