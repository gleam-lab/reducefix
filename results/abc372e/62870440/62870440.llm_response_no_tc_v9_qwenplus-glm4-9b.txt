int parent[200005];
int size[200005];

vector<int> tree[200005]; // To store the vertices in the connected component

int find(int v) {
    if (parent[v] == v) return v;
    parent[v] = find(parent[v]); // Path compression
    return parent[v];
}

### Explanation

1. **Union-Find Data Structure**: The code uses the union-find data structure to keep track of connected components in the graph. The `find` function performs path compression to flatten the structure, making it efficient for finding roots. The `unionSets` function merges two components and stores all vertices in a sorted order within the root of the merged component.

2. **Edge Cases**: The code checks if the sizes of the two components are equal or not and always attaches the smaller component under the larger one to keep the tree balanced and prevent a linear union-find structure that would degrade performance.

3. **Efficiency**: The use of path compression and union by size (merging smaller component under larger) ensures that the operations are almost constant time in practice, even for the maximum input sizes.

4. **Output**: For Type 2 queries, the code checks if there are at least `k` vertices in the connected component and prints the `k`-th largest vertex. If there are fewer than `k` vertices, it prints `-1`.
