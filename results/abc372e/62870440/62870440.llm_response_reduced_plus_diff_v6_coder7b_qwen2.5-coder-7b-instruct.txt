To solve this problem efficiently, we need to handle both adding edges and querying the k-th largest connected node in constant time per operation. We can achieve this using Disjoint Set Union (DSU) with path compression and union by rank, along with maintaining a multiset for each set to store the connected nodes' values.

Here's the corrected and optimized C++ code:

```cpp
#include<bits/stdc++.h>
using namespace std;

const int MAXN = 200005;
vector<int> parent(MAXN), size(MAXN), *sets[MAXN];

void init(int n) {
    iota(parent.begin(), parent.end(), 0);
    fill(size.begin(), size.end(), 1);
    for (int i = 0; i <= n; ++i) sets[i].clear();
}

int find(int u) {
    if (parent[u] != u) return parent[u] = find(parent[u]);
    return u;
}

void merge(int u, int v) {
    u = find(u), v = find(v);
    if (u == v) return;
    
    if (size[u] > size[v]) swap(u, v);
    parent[u] = v;
    size[v] += size[u];
    sets[v].insert(sets[u].begin(), sets[u].end());
    sets[v].insert(v);
    while (sets[v].size() > 10) sets[v].erase(sets[v].begin());
}

int kthLargest(int v, int k) {
    v = find(v);
    if (v == -1 || k > 10 || sets[v].size() < k) return -1;
    return *next(sets[v].rbegin(), k - 1);
}

int main() {
    ios::sync_with_stdio(false); cin.tie(nullptr);
    
    int n, q;
    cin >> n >> q;
    init(n);
    
    for (int i = 0; i < q; ++i) {
        int op, u, v;
        cin >> op >> u >> v;
        
        if (op == 1) {
            merge(u, v);
        } else {
            cout << kthLargest(v, v) << '\n';
        }
    }
    
    return 0;
}
```

### Explanation
1. **Initialization**: 
   - `init` function initializes the DSU structure and clears all sets.

2. **Find Operation**:
   - `find` function uses path compression to find the root of the set containing `u`.

3. **Merge Operation**:
   - `merge` function merges two sets by connecting their roots. It ensures that the smaller set is always merged into the larger set to keep the tree flat.
   - After merging, it combines the multisets of both sets, ensuring that only the top 10 largest elements are kept.

4. **K-th Largest Query**:
   - `kthLargest` function finds the k-th largest element in the set containing `v`.
   - It first finds the root of the set containing `v` and then checks if the required element exists in the sorted multiset.

This approach ensures efficient handling of both operations within the constraints.
