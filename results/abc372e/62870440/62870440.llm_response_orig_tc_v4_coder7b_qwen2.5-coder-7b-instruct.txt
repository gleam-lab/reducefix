To solve this problem efficiently, we need to maintain a list of connected vertices for each vertex and keep track of these lists sorted to quickly find the k-th largest element. Here's how you can achieve this using a combination of Union-Find and sorting techniques:

1. **Union-Find Data Structure**: This helps in managing the connectivity of vertices efficiently.
2. **Sorted Lists**: For each vertex, maintain a sorted list of connected vertices. This allows us to quickly access the k-th largest element.

Here's the corrected C++ code that implements the above approach:

```cpp
#include<bits/stdc++.h>
using namespace std;

const int MAXN = 200005;
vector<int> parent(MAXN), rank(MAXN);
vector<vector<int>> adjList(MAXN);

void init(int n) {
    iota(parent.begin(), parent.end(), 0);
    fill(rank.begin(), rank.end(), 1);
}

int find(int x) {
    if (parent[x] != x) parent[x] = find(parent[x]);
    return parent[x];
}

void unite(int x, int y) {
    x = find(x);
    y = find(y);
    if (x == y) return;
    if (rank[x] > rank[y]) swap(x, y);
    parent[x] = y;
    rank[y] += rank[x];
    adjList[y].insert(adjList[y].end(), adjList[x].begin(), adjList[x].end());
    sort(adjList[y].begin(), adjList[y].end(), greater<int>());
    adjList[x].clear();
}

int kthLargest(vector<int>& vec, int k) {
    if (vec.size() < k) return -1;
    return vec[k - 1];
}

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(NULL);

    int n, q;
    cin >> n >> q;
    init(n);

    while (q--) {
        int type, u, v, k;
        cin >> type;
        if (type == 1) {
            cin >> u >> v;
            --u, --v;
            unite(u, v);
        } else {
            cin >> u >> k;
            --u;
            cout << kthLargest(adjList[find(u)], k) << "\n";
        }
    }

    return 0;
}
```

### Explanation:
1. **Initialization**:
   - We initialize the Union-Find structure with `init(n)` which sets each node as its own parent and initializes ranks to 1.

2. **Find Function**:
   - The `find` function uses path compression to optimize future queries.

3. **Unite Function**:
   - The `unite` function merges two sets. It ensures that the smaller set is always attached to the larger set to keep the tree flat, which helps in maintaining efficient operations.
   - After merging, it updates the adjacency list of the root node with the combined list of both nodes' adjacency lists and sorts it in descending order to facilitate quick retrieval of the k-th largest element.

4. **Kth Largest Element**:
   - The `kthLargest` function checks if there are at least k elements in the list. If so, it returns the k-th largest element (considering 1-based index); otherwise, it returns -1.

5. **Main Logic**:
   - The main loop processes each query. For type 1 queries, it calls `unite`, and for type 2 queries, it finds the root of the queried vertex and retrieves the k-th largest element from its adjacency list.

This solution ensures that each operation (union and finding the k-th largest element) is efficient, making it suitable for large inputs within the given constraints.
