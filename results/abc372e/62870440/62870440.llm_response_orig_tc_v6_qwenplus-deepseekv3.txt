To fix this code, we need to address the incorrect output for several test cases. The primary issue lies in how the code handles the union-find operations and maintains the sorted lists of connected vertices. The incorrect approach in the original code led to wrong answers in some cases.

### Approaches
1. **Union-Find Data Structure**: Use a union-find (disjoint set) data structure to efficiently manage the connected components of the graph.
2. **Sorted Lists**: For each connected component, maintain a sorted list of vertex numbers in descending order. This allows quick access to the k-th largest vertex when needed.
3. **Merge Operation**: When merging two components, combine their sorted lists and keep only the top 10 elements to ensure efficiency, as the problem constraints limit k to 10.

### Solution Code
```cpp
#include <bits/stdc++.h>
using namespace std;

const int MAXN = 200005;
vector<int> parent(MAXN);
vector<vector<int>> components(MAXN);

void initialize(int n) {
    for (int i = 1; i <= n; ++i) {
        parent[i] = i;
        components[i].clear();
        components[i].push_back(i);
    }
}

int find(int u) {
    if (parent[u] != u) {
        parent[u] = find(parent[u]);
    }
    return parent[u];
}

void unionSets(int u, int v) {
    u = find(u);
    v = find(v);
    if (u == v) return;
    if (components[u].size() < components[v].size()) {
        swap(u, v);
    }
    parent[v] = u;
    components[u].insert(components[u].end(), components[v].begin(), components[v].end());
    sort(components[u].begin(), components[u].end(), greater<int>());
    if (components[u].size() > 10) {
        components[u].resize(10);
    }
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int N, Q;
    cin >> N >> Q;
    initialize(N);

    while (Q--) {
        int op, u, v;
        cin >> op >> u >> v;
        if (op == 1) {
            unionSets(u, v);
        } else {
            int k = v;
            u = find(u);
            if (components[u].size() < k) {
                cout << -1 << '\n';
            } else {
                cout << components[u][k - 1] << '\n';
            }
        }
    }

    return 0;
}
```

### Explanation
1. **Initialization**: The `initialize` function sets up each vertex as its own parent and initializes its component list with itself.
2. **Find Operation**: The `find` function uses path compression to efficiently determine the root parent of a vertex.
3. **Union Operation**: The `unionSets` function merges two sets. It ensures the smaller component is merged into the larger one to optimize performance. The merged component's list is sorted in descending order and truncated to keep only the top 10 elements.
4. **Query Handling**: For type 2 queries, the code checks if the connected component has enough elements to return the k-th largest vertex. If not, it returns -1; otherwise, it outputs the appropriate element from the sorted list.

This solution efficiently handles the dynamic connectivity and query requirements of the problem while ensuring optimal performance for large inputs.
