To fix this code, we need to ensure that the union-find data structure correctly maintains the connected components and that for each component, we keep track of the top K largest vertices (where K is up to 10, as per the problem constraints). The incorrect code was attempting to merge sorted arrays but didn't handle the merging and sorting correctly, leading to incorrect answers.

### Approaches
1. **Union-Find Data Structure**: This will help efficiently manage the connected components of the graph.
2. **Maintaining Top K Elements**: For each component, we need to maintain a sorted list of the largest vertices. When two components are united, we merge their top K lists and keep only the top K elements.
3. **Efficient Merging**: When merging two components, we combine their top K lists, sort them in descending order, and keep the first K elements. This ensures that we always have the top K largest vertices for each component.

### Solution Code
```cpp
#include<bits/stdc++.h>
using namespace std;

const int MAXN = 200005;
const int K = 10;

int parent[MAXN];
vector<int> topK[MAXN];

int find(int u) {
    if (parent[u] != u) {
        parent[u] = find(parent[u]);
    }
    return parent[u];
}

void unite(int u, int v) {
    u = find(u);
    v = find(v);
    if (u == v) return;
    if (topK[u].size() < topK[v].size()) {
        swap(u, v);
    }
    parent[v] = u;
    vector<int> merged;
    merged.reserve(topK[u].size() + topK[v].size());
    merge(topK[u].begin(), topK[u].end(), topK[v].begin(), topK[v].end(), back_inserter(merged), greater<int>());
    if (merged.size() > K) {
        merged.resize(K);
    }
    topK[u] = merged;
}

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);

    int N, Q;
    cin >> N >> Q;

    for (int i = 1; i <= N; ++i) {
        parent[i] = i;
        topK[i] = {i};
    }

    while (Q--) {
        int op, u, v;
        cin >> op >> u >> v;
        if (op == 1) {
            unite(u, v);
        } else {
            int root = find(u);
            if (v > topK[root].size()) {
                cout << -1 << '\n';
            } else {
                cout << topK[root][v - 1] << '\n';
            }
        }
    }

    return 0;
}
```

### Explanation
1. **Initialization**: We initialize the parent array for the union-find structure and the topK arrays for each vertex. Each vertex initially is its own parent and has a topK list containing only itself.
2. **Union Operation**: When merging two vertices, we find their roots. If they are not already in the same component, we merge the smaller topK list into the larger one. The merged list is sorted in descending order, and we keep only the top K elements.
3. **Query Operation**: For a query, we find the root of the vertex. If the requested k-th largest element exists in the topK list of the root, we return it; otherwise, we return -1.
4. **Efficiency**: The union operation efficiently merges the top K lists using the merge function from the STL, which operates in O(N) time where N is the total number of elements in both lists. Since K is small (up to 10), this operation is effectively O(1) for practical purposes.

This approach ensures that we correctly maintain the connected components and efficiently answer queries about the k-th largest vertex in each component.
