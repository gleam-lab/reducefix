To solve this problem efficiently, we need to manage dynamic connectivity and frequent queries about the k-th largest connected node. This is a classic application of the **Disjoint Set Union (DSU)** with **augmented information**.

We'll maintain for each component:
- The 10 largest vertex numbers in descending order. This allows us to answer Type 2 queries in O(1) time per query.

### Key Concepts Used:

1. **Union-Find (Disjoint Set Union - DSU)**: Efficiently merge components.
2. **Augmented DSU**: Each root maintains a sorted list of up to 10 largest nodes in its component.
3. **Efficient Merging**: When two sets are merged, we merge their top-10 lists, sort them, and keep only the top 10.

This approach ensures:
- Each union operation runs in O(1), because merging two sorted lists of size ≤10 takes constant time.
- Each query of type 2 is answered in O(1).

---

### ✅ Final Working C++ Code:

```cpp
#include <bits/stdc++.h>
using namespace std;

const int MAXN = 2e5 + 5;
const int K = 10;

int parent[MAXN];
vector<int> top_components[MAXN];

int find(int x) {
    if (parent[x] != x)
        parent[x] = find(parent[x]);
    return parent[x];
}

void unite(int u, int v) {
    int pu = find(u);
    int pv = find(v);
    if (pu == pv)
        return;

    // Always attach smaller tree to bigger tree
    if ((int)top_components[pu].size() > (int)top_components[pv].size())
        swap(pu, pv);

    // Merge top_components
    for (int val : top_components[pu]) {
        top_components[pv].push_back(val);
    }

    // Sort and keep top K elements
    sort(top_components[pv].rbegin(), top_components[pv].rend());
    if (top_components[pv].size() > K)
        top_components[pv].resize(K);

    parent[pu] = pv;
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int N, Q;
    cin >> N >> Q;

    // Initialize DSU
    for (int i = 1; i <= N; ++i) {
        parent[i] = i;
        top_components[i].push_back(i);
    }

    while (Q--) {
        int type;
        cin >> type;
        if (type == 1) {
            int u, v;
            cin >> u >> v;
            unite(u, v);
        } else {
            int v, k;
            cin >> v >> k;
            int root = find(v);
            if ((int)top_components[root].size() < k)
                cout << -1 << "\n";
            else
                cout << top_components[root][k - 1] << "\n";
        }
    }

    return 0;
}
```

---

### ✅ Why This Works Correctly

- **Merging Components**: We always merge the smaller component into the larger one for efficiency.
- **Maintaining Top-K Values**: After merging, we re-sort and trim the list to keep only the top 10 values.
- **Query Handling**: Directly access the k-th value from the precomputed list at the root.

---

### ✅ Time Complexity

- **Initialization**: O(N)
- **Each Query**:
  - Type 1: O(1) amortized (due to path compression and small merge sizes)
  - Type 2: O(1)

Total complexity is efficient enough for N, Q ≤ 2 × 10⁵.
