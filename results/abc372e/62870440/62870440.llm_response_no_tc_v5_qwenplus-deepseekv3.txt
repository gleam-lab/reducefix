To fix this code, we need to address several issues. The main problems in the original code include incorrect handling of the union-find data structure, inefficient sorting and merging of adjacency lists, and incorrect handling of the k-th largest element query.

### Approaches
1. **Union-Find Data Structure**: This helps efficiently manage the connected components of the graph. Each component will have a root, and merging two components involves updating the root of one component to point to the root of the other.
2. **Maintaining Sorted Lists**: For each component, we maintain a sorted list (in descending order) of the vertices connected to it. This allows efficient access to the k-th largest element.
3. **Efficient Merging**: When merging two components, we combine their sorted lists and keep only the top 10 elements (since k is at most 10). This ensures that each merge operation is efficient and doesn't involve sorting large lists repeatedly.

### Solution Code
```cpp
#include <bits/stdc++.h>
using namespace std;

const int MAX_N = 200005;

int parent[MAX_N];
vector<int> components[MAX_N];

int find_root(int x) {
    if (parent[x] != x) {
        parent[x] = find_root(parent[x]);
    }
    return parent[x];
}

void merge(int u, int v) {
    int root_u = find_root(u);
    int root_v = find_root(v);
    if (root_u == root_v) return;
    if (components[root_u].size() < components[root_v].size()) {
        swap(root_u, root_v);
    }
    parent[root_v] = root_u;
    components[root_u].insert(components[root_u].end(), components[root_v].begin(), components[root_v].end());
    sort(components[root_u].rbegin(), components[root_u].rend());
    if (components[root_u].size() > 10) {
        components[root_u].resize(10);
    }
    components[root_v].clear();
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);
    
    int N, Q;
    cin >> N >> Q;
    
    for (int i = 1; i <= N; ++i) {
        parent[i] = i;
        components[i] = {i};
    }
    
    while (Q--) {
        int op, u, v;
        cin >> op >> u >> v;
        if (op == 1) {
            merge(u, v);
        } else {
            int root = find_root(u);
            if (v > (int)components[root].size()) {
                cout << -1 << '\n';
            } else {
                cout << components[root][v - 1] << '\n';
            }
        }
    }
    
    return 0;
}
```

### Explanation
1. **Union-Find Data Structure**: The `find_root` function uses path compression to efficiently find the root of any vertex. The `merge` function combines two components by linking the root of the smaller component to the root of the larger one (union by size).
2. **Maintaining Sorted Lists**: Each component's vertices are stored in a vector sorted in descending order. This allows direct access to the k-th largest element via indexing.
3. **Efficient Merging**: When merging two components, their vectors are combined, resorted, and trimmed to keep only the top 10 elements. This ensures that the merge operation remains efficient even with large graphs.
4. **Query Handling**: For each query, if it's a merge request, the components are merged. If it's a k-th largest query, the result is directly retrieved from the sorted list of the component's root, or `-1` is printed if k exceeds the list size.

This approach ensures that the solution is both efficient and correct, handling the constraints effectively.
