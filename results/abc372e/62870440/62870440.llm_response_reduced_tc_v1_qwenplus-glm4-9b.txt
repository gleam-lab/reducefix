1. Add an edge between vertices u and v.
2. Find the k-th largest connected vertex for vertex v.

2. **Path Compression**: You are using path compression in `gtf`, which is good for the union-find structure. However, you are not updating the path compression when merging the sets. You need to apply path compression both ways when merging.

3. **Size Array**: Your code doesn't maintain a size array which keeps track of the number of vertices in each set. This can be useful for finding the k-th largest vertex efficiently.

4. **Data Structure for Storing Connected Vertices**: Currently, you are using a 2D array `as` to store the connected vertices. However, you are not managing this array efficiently. For each union operation, you are copying all elements from the smaller array to the larger one, which is inefficient.

5. **Sorting**: You are sorting the connected vertices every time you add an edge. This is expensive and unnecessary. You can use a data structure like a balanced binary search tree (e.g., `std::set` in C++) to keep the vertices sorted efficiently.

6. **Indexing**: When finding the k-th largest vertex, you are using `as[gtf(ix)][iy-1]`. This indexing is incorrect and assumes fixed-size arrays, which can lead to errors.

int n, q;
vector<int> parent;
vector<int> rank;
vector<set<int>> components;

int find_set(int v) {
    if (parent[v] != v) {
        parent[v] = find_set(parent[v]); // Path compression
    }
    return parent[v];
}

void union_sets(int u, int v) {
    int root_u = find_set(u);
    int root_v = find_set(v);

    if (root_u != root_v) {
        // Union by rank
        if (rank[root_u] > rank[root_v]) {
            parent[root_v] = root_u;
            components[root_u].insert(v);
        } else if (rank[root_u] < rank[root_v]) {

        } else {
            parent[root_v] = root_u;
            components[root_u].insert(v);
            rank[root_u]++;
        }
    }
}
