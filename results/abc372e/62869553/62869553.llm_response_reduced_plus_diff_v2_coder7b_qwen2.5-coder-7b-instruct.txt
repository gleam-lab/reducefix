To solve this problem, we need to efficiently manage and query the connections between vertices in an undirected graph. The key operations are adding edges and querying the k-th largest neighbor of a vertex. Here's a revised version of your code that addresses the issues and meets the requirements:

```cpp
#include<bits/stdc++.h>
using namespace std;

vector<int> adj[200005];
int n, q;

void addEdge(int u, int v) {
    adj[u].push_back(v);
    adj[v].push_back(u);
}

int findKthLargest(vector<int>& vec, int k) {
    sort(vec.begin(), vec.end());
    if (vec.size() < k) return -1;
    return vec[k-1];
}

int main() {
    cin >> q >> n;
    while(q--) {
        int op, ix, iy;
        cin >> op >> ix >> iy;
        if (op == 1) {
            addEdge(ix, iy);
        } else {
            vector<int> neighbors = adj[ix];
            cout << findKthLargest(neighbors, iy) << endl;
        }
    }
    return 0;
}
```

### Explanation

1. **Graph Representation**: We use an adjacency list to represent the graph. This allows efficient addition of edges and querying of neighbors.

2. **Adding Edges**: The `addEdge` function adds an undirected edge between vertices `u` and `v` by appending each to the other's adjacency list.

3. **Finding K-th Largest Neighbor**: The `findKthLargest` function sorts the neighbors of a vertex and returns the k-th largest value. If there are fewer than k neighbors, it returns `-1`.

4. **Main Function**:
   - Read the number of queries `q` and vertices `n`.
   - Process each query:
     - If the operation is type 1 (`op == 1`), add an edge between vertices `ix` and `iy`.
     - If the operation is type 2 (`op == 2`), retrieve the neighbors of vertex `ix`, sort them, and output the k-th largest neighbor (where `iy` is the position in the sorted list).

This approach ensures that the graph operations are handled efficiently, and the queries can be answered correctly within the constraints.
