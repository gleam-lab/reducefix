To solve this problem efficiently, we need to handle dynamic connectivity queries and retrieve the k-th largest node in a connected component. This is a classic use case for the **Disjoint Set Union (DSU)** data structure with path compression and union by size/rank.

However, since we also need to support `k-th` largest queries efficiently (with `k <= 10`), we can augment each DSU set to maintain a sorted list of the top 10 elements in descending order. Since `k` is small, maintaining only the top 10 per set is efficient.

### Key Ideas:
- Use **DSU** with path compression and union by size.
- For each root in the DSU, keep an array/list of up to 10 largest vertices in that set, sorted in descending order.
- On union, merge the smaller list into the larger one to keep operations fast.
- On query, simply return the k-th element from the root's list or -1 if there are fewer than k elements.

Here is the corrected and efficient C++ implementation:

```cpp
#include <bits/stdc++.h>
using namespace std;

const int MAXN = 2 * 1e5 + 5;
const int K_MAX = 10;

int parent[MAXN];
vector<int> top_members[MAXN];

int find(int x) {
    if (parent[x] != x)
        parent[x] = find(parent[x]);
    return parent[x];
}

void unite(int x, int y) {
    int rootX = find(x);
    int rootY = find(y);
    if (rootX == rootY)
        return;

    // Always merge smaller into larger
    if (top_members[rootX].size() < top_members[rootY].size())
        swap(rootX, rootY);

    // Merge members of rootY into rootX
    for (int v : top_members[rootY]) {
        top_members[rootX].push_back(v);
    }

    // Sort and retain only top K_MAX elements
    sort(top_members[rootX].begin(), top_members[rootX].end(), greater<int>());
    if (top_members[rootX].size() > K_MAX)
        top_members[rootX].resize(K_MAX);

    parent[rootY] = rootX;
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int N, Q;
    cin >> N >> Q;

    // Initialize DSU
    for (int i = 1; i <= N; ++i) {
        parent[i] = i;
        top_members[i].push_back(i);
    }

    while (Q--) {
        int op;
        cin >> op;

        if (op == 1) {
            int u, v;
            cin >> u >> v;
            unite(u, v);
        } else if (op == 2) {
            int v, k;
            cin >> v >> k;
            int root = find(v);
            if (k <= (int)top_members[root].size())
                cout << top_members[root][k - 1] << '\n';
            else
                cout << "-1\n";
        }
    }

    return 0;
}
```

### Explanation:
- **DSU Initialization**: Each node starts as its own parent, and each node's list contains just itself.
- **Union Operation**: When uniting two sets, we merge the lists of top members and keep only the top 10 values.
- **Find Operation**: Standard path compression used for efficiency.
- **Query Handling**: We simply check the root's cached list of top members and output the k-th largest or -1 if not enough nodes.

This approach ensures that all operations are nearly constant time due to path compression and the limited number of elements kept per set (`<= 10`).

### Complexity:
- Time complexity per operation: amortized O(α(N)) for DSU with path compression.
- Extra work during union is at most O(k log k) where k <= 20 (due to merging and sorting top 10s).
- Total time complexity: Efficient enough for constraints `N, Q <= 2 * 1e5`.
