To fix this code, we need to ensure that the union-find data structure correctly maintains the top 10 largest elements in each connected component. The previous implementation had several issues, including incorrect merging of the top elements and improper handling of the k-th largest query.

### Approaches
1. **Union-Find Data Structure**: This will help efficiently manage the connected components and their unions.
2. **Maintaining Top Elements**: For each component, we maintain a sorted list of the top 10 largest elements. When two components are merged, we merge their top elements lists and keep only the top 10 elements.
3. **Handling Queries**: For type 2 queries, we directly access the k-th largest element from the sorted list of the component's top elements.

### Solution Code
```cpp
#include <bits/stdc++.h>
using namespace std;

const int MAXN = 2e5 + 5;
int parent[MAXN];
vector<int> top_elements[MAXN];

int find(int u) {
    if (parent[u] != u) {
        parent[u] = find(parent[u]);
    }
    return parent[u];
}

void unite(int u, int v) {
    u = find(u);
    v = find(v);
    if (u == v) return;
    if (top_elements[u].size() < top_elements[v].size()) {
        swap(u, v);
    }
    parent[v] = u;
    top_elements[u].insert(top_elements[u].end(), top_elements[v].begin(), top_elements[v].end());
    sort(top_elements[u].rbegin(), top_elements[u].rend());
    if (top_elements[u].size() > 10) {
        top_elements[u].resize(10);
    }
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int N, Q;
    cin >> N >> Q;

    for (int i = 1; i <= N; ++i) {
        parent[i] = i;
        top_elements[i].push_back(i);
    }

    while (Q--) {
        int op, u, v;
        cin >> op >> u >> v;
        if (op == 1) {
            unite(u, v);
        } else {
            int root = find(u);
            if (top_elements[root].size() < v) {
                cout << "-1\n";
            } else {
                cout << top_elements[root][v - 1] << "\n";
            }
        }
    }

    return 0;
}
```

### Explanation
1. **Initialization**: We initialize the parent array for the union-find structure and the top_elements vector for each node to store its own value initially.
2. **Union Operation**: The `unite` function merges two components. It ensures the larger component absorbs the smaller one to maintain efficiency. The top elements of both components are merged and sorted in descending order, keeping only the top 10 elements.
3. **Find Operation**: The `find` function uses path compression to optimize future queries, ensuring efficient access to the root of any node.
4. **Query Handling**: For type 2 queries, we check the size of the top elements list of the component's root. If the list is smaller than the requested k-th element, we output -1; otherwise, we output the k-th largest element from the sorted list.

This approach efficiently handles the dynamic merging of components and ensures quick access to the k-th largest element in each component, adhering to the constraints and requirements.
