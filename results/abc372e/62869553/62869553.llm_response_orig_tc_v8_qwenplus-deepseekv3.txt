To fix this code, we need to ensure that the union-find data structure correctly maintains the top 10 largest elements in each connected component. The previous implementation had several issues, including incorrect merging of the top elements and incorrect handling of the union operation.

### Approaches
1. **Union-Find Data Structure**: This will help in efficiently managing the connected components of the graph.
2. **Maintaining Top Elements**: Each root of a component will maintain a sorted list of the top 10 largest elements in its component. When two components are merged, the top elements are combined and sorted to maintain the top 10 elements.
3. **Handling Queries**: For each query of type 2, we need to check the root of the given vertex and retrieve the k-th largest element from its maintained list.

### Solution Code
```cpp
#include <bits/stdc++.h>
using namespace std;

const int MAXN = 2e5 + 5;

int parent[MAXN];
vector<int> top[MAXN];

int find(int u) {
    if (parent[u] != u) {
        parent[u] = find(parent[u]);
    }
    return parent[u];
}

void unite(int u, int v) {
    u = find(u);
    v = find(v);
    if (u == v) return;
    if (top[u].size() < top[v].size()) {
        swap(u, v);
    }
    parent[v] = u;
    top[u].insert(top[u].end(), top[v].begin(), top[v].end());
    sort(top[u].begin(), top[u].end(), greater<int>());
    if (top[u].size() > 10) {
        top[u].resize(10);
    }
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);
    int N, Q;
    cin >> N >> Q;
    for (int i = 1; i <= N; ++i) {
        parent[i] = i;
        top[i] = {i};
    }
    while (Q--) {
        int op, u, v;
        cin >> op >> u >> v;
        if (op == 1) {
            unite(u, v);
        } else {
            int root = find(u);
            if (top[root].size() < v) {
                cout << "-1\n";
            } else {
                cout << top[root][v - 1] << "\n";
            }
        }
    }
    return 0;
}
```

### Explanation
1. **Initialization**: We initialize each node to be its own parent and its top list to contain only itself.
2. **Union Operation**: When merging two components, the larger component absorbs the smaller one. The top elements of both components are combined, sorted in descending order, and truncated to maintain only the top 10 elements.
3. **Query Handling**: For each type 2 query, we find the root of the given node and check if the requested k-th largest element exists in the top list. If it does, we print it; otherwise, we print -1.

This approach ensures that each operation is handled efficiently, maintaining the correct top elements for each connected component and answering queries in constant time after the union-find operations. The complexity is optimized to handle the constraints effectively.
