
The problem involves managing an undirected graph with dynamic connectivity and vertex queries. The solution must efficiently handle union-find data structures and priority queues to address the two types of queries: adding edges and finding the k-th largest connected vertex.

### Key Observations:
1. **Union-Find Data Structure**: This will be used to manage and merge connected components of the graph.
2. **Sorting**: We need to maintain sorted lists of vertices for each connected component to efficiently fetch the k-th largest vertex.
3. **Efficient Updates and Queries**: We need to ensure that the operations (edge addition, component merging, and sorting) are efficient enough to handle up to 200,000 operations.

### Solution Approach:
1. **Initialization**: Initialize the union-find structure and priority queues for each vertex. This will allow us to efficiently merge components and perform queries.
2. **Edge Addition (Type 1)**: When adding an edge, we merge the components of the two vertices. Update the priority queues for both vertices to reflect the merged component.
3. **Vertex Query (Type 2)**: To answer the k-th largest vertex query, we need to fetch the k-th largest element from the priority queue of the connected component's list. If the list size is less than k, return `-1`.

### Code Implementation:
```cpp
#include <iostream>
#include <vector>
#include <queue>
using namespace std;

const int MAX_N = 200005;
vector<int> parent(MAX_N), size(MAX_N), vertices(MAX_N);
priority_queue<int> pq[MAX_N];

void findSet(int x) {
    if (parent[x] != x) {
        parent[x] = findSet(parent[x]);
    }
    return parent[x];
}

void unionSets(int x, int y) {
    x = findSet(x);
    y = findSet(y);
    if (x != y) {
        if (size[x] < size[y]) {
            swap(x, y);
        }
        parent[y] = x;
        size[x] += size[y];
        // Merging the priority queues
        while (!pq[y].empty()) {
            pq[x].push(pq[y].top());
            pq[y].pop();
        }
    }
}

int solveKthLargest(int v, int k) {
    v = findSet(v);
    if (size[v] < k) {
        return -1;
    }
    while (k--) {
        if (!pq[v].empty()) {
            pq[v].pop();
        }
    }
    if (!pq[v].empty()) {
        return pq[v].top();
    }
    return -1;
}

int main() {
    int N, Q, u, v, k;
    cin >> N >> Q;

    for (int i = 1; i <= N; i++) {
        parent[i] = i;
        size[i] = 1;
        vertices[i] = i;
    }

    for (int i = 0; i < Q; i++) {
        cin >> u >> v;
        if (u == 1) {
            unionSets(u, v);
        } else {
            cin >> k;
            cout << solveKthLargest(v, k) << endl;
        }
    }

    return 0;
}
```

### Explanation:
- **Union-Find with Path Compression**: `findSet` and `unionSets` are used to manage the connected components.
- **Priority Queue**: Each component maintains a priority queue `pq[v]` to keep vertices sorted. This allows efficient retrieval of the k-th largest element.
- **Edge and Query Processing**: For edge addition, the union operation is performed, and for vertex queries, the k-th largest element is fetched from the respective priority queue.
