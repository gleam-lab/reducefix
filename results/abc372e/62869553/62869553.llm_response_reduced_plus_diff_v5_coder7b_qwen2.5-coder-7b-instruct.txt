To solve this problem, we need to efficiently manage the graph and handle the operations of adding edges and querying the k-th largest neighbor for a given vertex. Here's how we can approach it:

1. **Graph Representation**: We'll use an adjacency list to represent the graph.
2. **Union-Find Data Structure**: This will help us manage connected components efficiently.
3. **Sorting**: For each component, we'll keep track of its neighbors and sort them to quickly find the k-th largest neighbor.

### Detailed Steps:

1. **Initialization**:
   - Initialize the adjacency list and union-find data structure.
   
2. **Adding Edges**:
   - When adding an edge, merge the components of the two vertices using the union-find structure.
   - Sort the neighbors of the merged component to maintain the k-th largest property.

3. **Querying k-th Largest Neighbor**:
   - Find the component of the given vertex using the union-find structure.
   - Return the k-th largest neighbor from the sorted list of neighbors.

### Implementation:

```cpp
#include<bits/stdc++.h>
using namespace std;

const int MAXN = 2 * 100005;

vector<int> adj[MAXN];
int parent[MAXN], rank[MAXN];

void init(int n) {
    for (int i = 1; i <= n; ++i) {
        parent[i] = i;
        rank[i] = 1;
    }
}

int find(int x) {
    if (parent[x] != x) {
        parent[x] = find(parent[x]);
    }
    return parent[x];
}

void unite(int x, int y) {
    int px = find(x), py = find(y);
    if (px == py) return;
    if (rank[px] > rank[py]) swap(px, py);
    parent[px] = py;
    if (rank[px] == rank[py]) rank[py]++;
    // Merge smaller set into larger set and sort
    adj[py].insert(adj[py].end(), adj[px].begin(), adj[px].end());
    adj[px].clear();
    sort(adj[py].begin(), adj[py].end());
}

int getKthLargest(int v, int k) {
    int root = find(v);
    if (k > adj[root].size()) return -1;
    return adj[root][k - 1];
}

int main() {
    ios::sync_with_stdio(false); cin.tie(0); cout.tie(0);

    int n, q;
    cin >> n >> q;
    init(n);

    while (q--) {
        int op, u, v;
        cin >> op >> u >> v;
        if (op == 1) {
            unite(u, v);
        } else {
            cout << getKthLargest(v, u) << '\n';
        }
    }

    return 0;
}
```

### Explanation:

1. **Initialization (`init`)**:
   - Initializes the parent array where each node is its own parent initially, and rank array to keep track of tree heights.

2. **Find Function (`find`)**:
   - Uses path compression to flatten the structure, making future queries faster.

3. **Unite Function (`unite`)**:
   - Merges two sets and ensures that the smaller set is always merged into the larger one to keep the depth small.
   - After merging, it sorts the combined set of neighbors to maintain the k-th largest property.

4. **Get K-th Largest Neighbor (`getKthLargest`)**:
   - Finds the root of the given vertex and checks if the k-th element exists in the sorted list of neighbors.

This solution handles all given constraints and ensures efficient operations for both adding edges and querying k-th largest neighbors.
