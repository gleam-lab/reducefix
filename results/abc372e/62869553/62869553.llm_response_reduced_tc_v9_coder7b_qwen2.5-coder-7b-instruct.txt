To solve this problem, we need to efficiently manage the connectivity of vertices and handle queries related to finding the k-th largest connected vertex. We can use a combination of Union-Find (Disjoint Set Union) and a priority queue to achieve this.

Here's a step-by-step breakdown of the solution:

1. **Union-Find Data Structure**: This will help us manage the connected components of the graph. It supports operations like `find` (to find the root of a set), `union_set` (to merge two sets), and `size` (to get the size of a set).

2. **Priority Queue**: For each connected component, we'll maintain a priority queue to store the sizes of its connected vertices. This allows us to quickly find the k-th largest vertex in a connected component.

3. **Handling Queries**:
   - For type 1 queries, we union the two sets containing the given vertices and update the priority queues accordingly.
   - For type 2 queries, we find the connected component of the given vertex and return the k-th largest vertex from its priority queue.

Here's the corrected and optimized C++ code:

```cpp
#include <bits/stdc++.h>
using namespace std;

const int MAXN = 200005;
vector<int> parent(MAXN), rank(MAXN), size(MAXN);
priority_queue<int> pq[MAXN];

void init(int n) {
    for (int i = 1; i <= n; ++i) {
        parent[i] = i;
        rank[i] = 1;
        size[i] = 1;
        pq[i].push(-i);
    }
}

int find(int x) {
    if (parent[x] != x) {
        parent[x] = find(parent[x]);
    }
    return parent[x];
}

void union_sets(int x, int y) {
    x = find(x);
    y = find(y);
    if (x == y) return;
    if (rank[x] > rank[y]) {
        swap(x, y);
    }
    parent[x] = y;
    rank[y] += rank[x];
    size[y] += size[x];
    pq[y].merge(pq[x]);
}

int kth_largest(int x, int k) {
    x = find(x);
    if (pq[x].size() < k) return -1;
    auto it = pq[x].begin();
    advance(it, k - 1);
    return -(*it);
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n, q;
    cin >> n >> q;
    init(n);

    while (q--) {
        int op, u, v;
        cin >> op >> u >> v;
        if (op == 1) {
            union_sets(u, v);
        } else {
            cout << kth_largest(v, u) << '\n';
        }
    }

    return 0;
}
```

### Explanation:
1. **Initialization (`init` function)**:
   - Initializes the parent array, rank array, and size array for Union-Find.
   - Initializes each priority queue with a single element representing the vertex itself.

2. **Find Function (`find` function)**:
   - Uses path compression to find the root of the set containing `x`.

3. **Union Sets Function (`union_sets` function)**:
   - Merges the sets containing `x` and `y`.
   - Updates the parent and rank arrays.
   - Merges the priority queues of the two sets.

4. **Kth Largest Function (`kth_largest` function)**:
   - Finds the connected component of `x`.
   - Returns the k-th largest vertex from the priority queue of that component.

This approach ensures efficient handling of both union and query operations, making it suitable for large inputs within the given constraints.
