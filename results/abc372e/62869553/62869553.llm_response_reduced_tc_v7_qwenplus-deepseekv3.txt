To fix this code, we need to ensure that the algorithm correctly maintains and queries the k-th largest vertex in connected components of an undirected graph. The initial approach attempted to use a Union-Find (Disjoint Set Union) data structure with an array to keep track of the top vertices, but it had several flaws in how it managed and merged these lists.

### Approaches
1. **Union-Find Data Structure**: This is used to efficiently manage the connected components of the graph. Each set's root will maintain a sorted list of its vertices.
2. **Sorted Lists Management**: For each connected component, we will keep a sorted list (in descending order) of the vertices. This allows efficient access to the k-th largest vertex.
3. **Efficient Merging**: When two components are merged, their sorted lists are combined and then truncated to keep only the top 10 elements, as the problem constraints specify k ≤ 10.

### Solution Code
```cpp
#include <bits/stdc++.h>
using namespace std;

const int MAXN = 200005;

vector<int> parent(MAXN);
vector<int> rank_(MAXN);
vector<vector<int>> top_nodes(MAXN);

void make_set(int v) {
    parent[v] = v;
    rank_[v] =0;
    top_nodes[v] = {v};
}

int find_set(int v) {
    if (v == parent[v])
        return v;
    return parent[v] = find_set(parent[v]);
}

void union_sets(int a, int b) {
    a = find_set(a);
    b = find_set(b);
    if (a != b) {
        if (rank_[a] < rank_[b])
            swap(a, b);
        parent[b] = a;
        top_nodes[a].insert(top_nodes[a].end(), top_nodes[b].begin(), top_nodes[b].end());
        sort(top_nodes[a].rbegin(), top_nodes[a].rend());
        if (top_nodes[a].size() > 10)
            top_nodes[a].resize(10);
        if (rank_[a] == rank_[b])
            rank_[a]++;
    }
}

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);

    int N, Q;
    cin >> N >> Q;

    for (int i = 1; i <= N; ++i) {
        make_set(i);
    }

    while (Q--) {
        int type, u, v;
        cin >> type >> u;
        if (type == 1) {
            cin >> v;
            union_sets(u, v);
        } else {
            int k;
            cin >> k;
            int root = find_set(u);
            if (top_nodes[root].size() >= k) {
                cout << top_nodes[root][k - 1] << '\n';
            } else {
                cout << "-1\n";
            }
        }
    }

    return 0;
}
```

### Explanation
1. **Initialization**: The `make_set` function initializes each vertex as its own parent, sets its rank to 0, and starts its `top_nodes` list with itself.
2. **Union-Find Operations**: The `find_set` function uses path compression to efficiently find the root of a vertex. The `union_sets` function merges two sets by rank and combines their `top_nodes` lists, ensuring the combined list stays sorted and contains only the top 10 elements.
3. **Query Handling**: For each query:
   - **Type 1 (Union)**: Calls `union_sets` to merge the sets containing vertices `u` and `v`.
   - **Type 2 (Query)**: Finds the root of vertex `u` and checks if the `top_nodes` list of the root has at least `k` elements. If so, it outputs the k-th largest element; otherwise, it outputs `-1`.

This approach ensures efficient handling of unions and queries, adhering to the problem constraints and providing correct results for all test cases.
