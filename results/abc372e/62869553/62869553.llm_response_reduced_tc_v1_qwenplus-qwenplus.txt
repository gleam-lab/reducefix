To solve this problem efficiently, we need to manage a dynamic undirected graph where:

- Vertices are numbered from 1 to N.
- We process queries of two types:
  - Type 1: Add an edge between vertices `u` and `v`.
  - Type 2: Find the k-th largest vertex among all vertices connected to `v`.

### Key Observations:
1. **Union-Find (Disjoint Set Union - DSU)** is ideal for managing connected components dynamically.
2. Each component should maintain a sorted list of its vertices in descending order to quickly answer the k-th largest query.
3. To merge sets efficiently while maintaining sorted order, use a `multiset` or `set` that supports O(log n) insertion and merging.

### Strategy:
- Use DSU with path compression and union by rank for efficiency.
- Maintain a `set<int, greater<>>` per representative (root) to store all vertices in the component in descending order.
- When merging sets (`union(u, v)`), always merge the smaller set into the larger one (small-to-large technique) to ensure amortized logarithmic performance.

Here's the corrected and optimized C++ code:

```cpp
#include <bits/stdc++.h>
using namespace std;

const int MAXN = 2e5 + 5;

int parent[MAXN];
int sz[MAXN];
set<int, greater<>> components[MAXN];

int find(int x) {
    if (parent[x] != x)
        parent[x] = find(parent[x]);
    return parent[x];
}

void unite(int u, int v) {
    int root_u = find(u);
    int root_v = find(v);

    if (root_u == root_v)
        return;

    // Merge smaller set into larger
    if (sz[root_u] < sz[root_v]) {
        swap(root_u, root_v);
    }

    for (int val : components[root_v]) {
        components[root_u].insert(val);
    }

    components[root_v].clear();
    parent[root_v] = root_u;
    sz[root_u] += sz[root_v];
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int N, Q;
    cin >> N >> Q;

    for (int i = 1; i <= N; ++i) {
        parent[i] = i;
        sz[i] = 1;
        components[i].insert(i);
    }

    while (Q--) {
        int type;
        cin >> type;

        if (type == 1) {
            int u, v;
            cin >> u >> v;
            unite(u, v);
        } else if (type == 2) {
            int v, k;
            cin >> v >> k;

            int root = find(v);
            if ((int)components[root].size() < k) {
                cout << -1 << "\n";
            } else {
                auto it = components[root].begin();
                advance(it, k - 1);
                cout << *it << "\n";
            }
        }
    }

    return 0;
}
```

### Key Points:
- Efficient data structure usage: `set<int, greater<>>` allows fast access to k-th largest.
- Small-to-large merging ensures overall time complexity near O(Q log N).
- Path compression in DSU ensures almost constant time per operation.
- Handles up to 2×10⁵ vertices and queries efficiently.
