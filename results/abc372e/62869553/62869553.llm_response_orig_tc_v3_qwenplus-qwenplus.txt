To solve this problem efficiently, we need to:

- Track connected components of an undirected graph.
- Support two operations:
  1. **Add an edge** between two vertices.
  2. **Query the k-th largest vertex number** in the component that a given vertex belongs to.

### Key Observations

- Since queries can be up to `2 * 10^5`, and for each query of type 2 we might need the top 10 elements of a set, it's important to keep these top-k values accessible quickly.
- The Disjoint Set Union (Union-Find) data structure is ideal for managing connected components dynamically.
- To support fast retrieval of the top-k vertex numbers within a component, we can maintain a sorted list (or vector) of the top 10 node IDs in descending order for each root in the DSU.

### Strategy

1. Use **DSU with path compression** to manage connected components efficiently.
2. Each root maintains a sorted array/vector of the **top 10 node values** from its component.
3. When merging two components:
   - Merge their top 10 lists.
   - Sort them in descending order.
   - Keep only the top 10 again.
4. For a query of type 2:
   - Find the root of the component.
   - Return the k-th value if available, else return `-1`.

Here's the corrected and efficient implementation:

```cpp
#include <bits/stdc++.h>
using namespace std;

const int MAXN = 200005;
const int TOP_K = 10;

int n, q;
int parent[MAXN];
vector<int> top_vals[MAXN];

// DSU find with path compression
int find(int x) {
    if (parent[x] != x)
        parent[x] = find(parent[x]);
    return parent[x];
}

// Merge the two sets: merge y into x
void unite(int x, int y) {
    int fx = find(x), fy = find(y);
    if (fx == fy) return;

    // Always merge smaller into larger to optimize performance
    if (top_vals[fx].size() < top_vals[fy].size())
        swap(fx, fy);

    // Merge top_vals
    for (int val : top_vals[fy])
        top_vals[fx].push_back(val);

    // Sort and keep top 10
    sort(top_vals[fx].rbegin(), top_vals[fx].rend());
    if (top_vals[fx].size() > TOP_K)
        top_vals[fx].resize(TOP_K);

    parent[fy] = fx;
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(0);

    cin >> n >> q;

    // Initialize DSU
    for (int i = 1; i <= n; ++i) {
        parent[i] = i;
        top_vals[i].push_back(i);
    }

    while (q--) {
        int op;
        cin >> op;

        if (op == 1) {
            int u, v;
            cin >> u >> v;
            unite(u, v);
        } else {
            int v, k;
            cin >> v >> k;
            int root = find(v);
            if (k <= top_vals[root].size())
                cout << top_vals[root][k - 1] << '\n';
            else
                cout << "-1\n";
        }
    }

    return 0;
}
```

### Why This Works

- **Efficiency**: Each union operation merges at most 10 elements, so even with large N and Q, complexity remains manageable (`O(Q log N)`).
- **Correctness**: By maintaining the top 10 elements per root, and sorting after merge, we always get the correct k-th largest node in the component.
- **Robustness**: Handles edge cases like small components or querying k greater than size.
