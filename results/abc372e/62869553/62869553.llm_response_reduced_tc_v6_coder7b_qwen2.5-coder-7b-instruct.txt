To solve this problem, we need to maintain a graph and efficiently handle two types of operations: adding an edge and querying the k-th largest connected vertex. We can use Disjoint Set Union (DSU) to manage the connectivity of vertices and keep track of degrees of each vertex. For efficient querying, we will store the degrees of connected components in a sorted manner using a Fenwick Tree (Binary Indexed Tree).

Here's the corrected and optimized C++ code:

```cpp
#include<bits/stdc++.h>
using namespace std;

const int MAXN = 2e5 + 5;

struct BIT {
    vector<int> tree;
    BIT(int size) : tree(size + 1) {}

    void update(int idx, int val) {
        while (idx <= tree.size()) {
            tree[idx] += val;
            idx += idx & -idx;
        }
    }

    int query(int idx) {
        int sum = 0;
        while (idx > 0) {
            sum += tree[idx];
            idx -= idx & -idx;
        }
        return sum;
    }
};

vector<vector<int>> adj(MAXN);
BIT bit(MAXN);

void addEdge(int u, int v) {
    adj[u].push_back(v);
    adj[v].push_back(u);
    bit.update(deg[u], -1);
    bit.update(deg[v], -1);
    deg[u]++;
    deg[v]++;
    bit.update(deg[u], 1);
    bit.update(deg[v], 1);
}

int getKthLargest(int v, int k) {
    if (deg[v] < k) return -1;
    return k + bit.query(k) - 1;
}

int main() {
    int n, q;
    cin >> n >> q;
    vector<int> deg(n + 1, 0);
    
    for (int i = 0; i < q; i++) {
        int type, u, v, k;
        cin >> type;
        
        if (type == 1) {
            cin >> u >> v;
            addEdge(u, v);
        } else {
            cin >> u >> k;
            cout << getKthLargest(u, k) << "\n";
        }
    }
    
    return 0;
}
```

### Explanation:
1. **Data Structures**:
   - `adj`: Adjacency list to represent the graph.
   - `bit`: Fenwick Tree to efficiently manage degree counts.
   - `deg`: Array to store the degree of each vertex.

2. **Operations**:
   - `addEdge(u, v)`: Adds an edge between vertices `u` and `v`, updating the degrees and the Fenwick Tree.
   - `getKthLargest(v, k)`: Returns the k-th largest degree connected to vertex `v`. Uses the Fenwick Tree to find the correct position.

3. **Main Function**:
   - Reads the number of vertices `n` and queries `q`.
   - Processes each query, either adding an edge or querying the k-th largest degree.

This approach ensures that both operations are handled efficiently, maintaining a time complexity of approximately O(log N) per operation, which is suitable given the constraints.
