### 关键改进：使用无偏估计

之前使用的是**有偏估计** (取前k个，任一通过即通过)，现在改为**无偏估计**：

```
pass@k = 1 - C(n-c, k) / C(n, k)
```

其中：
- `n` = 总候选数 (10个patch)
- `c` = 通过的候选数
- `k` = 采样数 (1, 5, 或 10)

这个公式计算的是：从n个候选中随机选择k个时，至少有一个通过的概率。

### 为什么之前不匹配？

1. **有偏估计**: 简单地取前k个，如果有任何一个通过就算通过
   - 这种方法**依赖候选顺序**
   - 如果通过的patch在前k个中，结果偏高

2. **无偏估计**: 假设从n个候选中**随机采样**k个
   - 考虑所有可能的k个样本组合
   - 计算期望通过率，不依赖顺序

### 示例说明

假设有10个候选patch，其中2个通过：

| 指标 | 有偏估计 | 无偏估计 |
|------|----------|----------|
| pass@1 | 如果第1个通过: 100%<br>否则: 0% | 20% (期望值) |
| pass@5 | 如果前5个中有通过: 100%<br>否则: 0% | 77.8% (期望值) |
| pass@10 | 100% (全部采样) | 100% (全部采样) |

可以看到，**pass@10 两种方法相同**（因为全采样），这就是为什么之前我们的 pass@10 数据完全匹配！

### 技术细节

#### 修改的文件

`analyze_rq2_table.py`:

```python
from math import comb

def calculate_passk_unbiased(versions, k):
    """
    Calculate unbiased pass@k using the formula:
    pass@k = 1 - C(n-c, k) / C(n, k)
    """
    if not versions:
        return 0.0
    
    n = len(versions)
    c = sum(1 for v in versions if v.get('passed', False))
    
    if c == 0:
        return 0.0
    if n - c < k:
        return 1.0
    
    numerator = comb(n - c, k)  # Ways to choose k from non-passing
    denominator = comb(n, k)     # Total ways to choose k
    return 1.0 - (numerator / denominator)
```
