```python
n = int(input())
a = list(map(int, input().split()))

# dp[i][0]: 最も最近に defeated したモンスターが奇数番目（1st, 3rd, ...）のときの最大経験値
# dp[i][1]: 最も最近に defeated したモンスターが偶数番目（2nd, 4th, ...）のときの最大経験値
# 状態は「何体倒したか」ではなく、「最後に倒した順番が奇数か偶数か」に基づく

dp0 = 0  # 前回までに倒したモンスターの数が偶数（つまり次に倒せば奇数番目）
dp1 = -10**18  # 前回までに倒したモンスターの数が奇数（つまり次に倒せば偶数番目）

for i in range(n):
    strength = a[i]
    # モンスターiを倒さない場合：状態は変化しない
    # モンスターiを倒す場合：
    #   - dp0 の状態から：倒したのは奇数番目 → 経験値 + strength
    #   - dp1 の状態から：倒したのは偶数番目 → 経験値 + 2 * strength
    
    new_dp0 = max(dp0, dp1)  # このモンスターを倒さない
    new_dp1 = dp0 + strength  # 前が偶数個で、今回が奇数個目になる
    new_dp2 = dp1 + 2 * strength  # 前が奇数個で、今回が偶数個目になる
    
    # new_dp1 と new_dp2 を統合：現在の最終倒数順が奇数か偶数かで管理
    # 実際には new_dp1 は「奇数個目を倒した直後」→ 次は偶数番目
    # new_dp2 は「偶数個目を倒した直後」→ 次は奇数番目
    # しかし、状態としては「これまでに奇数個 or 偶数個倒したか」ではなく、
    # 「最後の倒したのが奇数番目か偶数番目か」を保持したい。
    
    # よって、
    # dp0: 最後の倒したのが偶数番目（＝次に倒すと奇数番目）← 実はこの解釈は不自然
    # 改めて：
    # dp_even: 最後に倒したのが偶数番目（つまり次は奇数番目）
    # dp_odd: 最後に倒したのが奇数番目（つまり次は偶数番目）
    
    # 再定義：
    # dp0: 最後に倒したのが奇数番目（1st, 3rd, ...）→ 次に倒したら偶数番目
    # dp1: 最後に倒したのが偶数番目（2nd, 4th, ...）→ 次に倒したら奇数番目
    
    # だが、上記の初期設定がわかりにくい。
    # よって、以下のようにリフレーズ：

    # 現在の選択：
    # - 奇数番目の倒し方：前の状態は「偶数個倒した」（つまり最後が偶数番目）または未倒し
    # - 偶数番目の倒し方：前の状態は「奇数個倒した」（つまり最後が奇数番目）

    # 正しい更新：
    # new_odd = max(現時点での偶数個倒した状態 + strength, 現時点での奇数個倒した状態（使えない）)
    # new_even = 現時点での奇数個倒した状態 + 2 * strength

    # ただし、最初の倒しは必ず奇数番目。

# 正しく再実装：

# dp0: 最後に倒したのが奇数番目（1st, 3rd, ...）→ 次に倒すと偶数番目
# dp1: 最後に倒したのが偶数番目（2nd, 4th, ...）→ 次に倒すと奇数番目
# 初期状態：何も倒していない → 次に倒すと奇数番目なので、dp1 は無効、dp0 は 0？ いや違う。

# 初期状態：まだ倒していない → 次に倒すと1体目（奇数番目）→ なので、前状態として「偶数個倒した」状態がある。
# よって：
# even_count_state = 0  # 倒した数が偶数（0,2,4,...）→ 次に倒すと奇数番目
# odd_count_state = -inf  # 倒した数が奇数（1,3,5,...）→ 次に倒すと偶数番目

even_state = 0  # 倒した数が偶数（0,2,4,...）→ 次に倒すと奇数番目（報酬: X）
odd_state = -10**18  # 倒した数が奇数（1,3,5,...）→ 次に倒すと偶数番目（報酬: 2X）

for strength in a:
    # 選択肢1: このモンスターを倒さない → 状態はそのまま
    # 選択肢2: このモンスターを倒す
    #   - 偶数個倒した状態から → 奇数個目を倒す → 報酬 + strength → 新たな奇数個状態
    #   - 奇数個倒した状態から → 偶数個目を倒す → 報酬 + 2*strength → 新たな偶数個状態
    
    new_even = max(even_state, odd_state)  # このモンスターを倒さない
    new_odd_from_even = even_state + strength  # 偶数個状態から奇数個目を倒す
    new_even_from_odd = odd_state + 2 * strength  # 奇数個状態から偶数個目を倒す
    
    # 新たな状態：倒した数が奇数になるのは new_odd_from_even
    # 新たな状態：倒した数が偶数になるのは new_even_from_odd
    even_state = max(new_even, new_even_from_odd)
    odd_state = max(odd_state, new_odd_from_even)  # ただし new_odd_from_even は新しい奇数状態

# 上記は誤り。new_odd_from_even は奇数個状態になるので、odd_state に代入すべき。

# 正しい更新：
# 新しい偶数個状態：max(倒さない, 奇数個状態から偶数個目を倒す)
# 新しい奇数個状態：max(倒さない, 偶数個状態から奇数個目を倒す)

# だが、倒さない場合は、元の偶数状態と奇数状態の両方が維持される。
# つまり、新しい偶数状態 = max(元の偶数状態（倒さない）, 元の奇数状態 + 2*strength)
# 新しい奇数状態 = max(元の奇数状態（倒さない）, 元の偶数状態 + strength)

# しかし、倒さない場合、偶数状態は偶数状態のまま、奇数状態は奇数状態のまま。
# なので、更新は同時に行う必要がある。

next_even = max(even_state, odd_state + 2 * strength)  # 倒さない or 奇数状態から偶数個目を倒す
next_odd = max(odd_state, even_state + strength)       # 倒さない or 偶数状態から奇数個目を倒す

# だが、これは論理的に正しくない。なぜなら、あるモンスターについて、
# 「倒さない」を選ぶと、状態は変わらないが、
# 「倒す」を選ぶと、状態が変わる。
# そして、次の状態は、どちらの行動を選んでも到達できる。

# 正しくは：
# 新しい偶数個状態（= 倒した数が偶数）は、以下のいずれか：
#   - 以前の偶数個状態で、このモンスターを倒さなかった
#   - 以前の奇数個状態で、このモンスターを倒して偶数個目にした
# 新しい奇数個状態（= 倒した数が奇数）は、以下のいずれか：
#   - 以前の奇数個状態で、このモンスターを倒さなかった
#   - 以前の偶数個状態で、このモンスターを倒して奇数個目にした

# よって：
# next_even = max(even_state, odd_state + 2 * strength)
# next_odd = max(odd_state, even_state + strength)

# しかし、これは「倒さない」場合、状態が維持されることを意味する。
# つまり、even_state は「倒さない」ことで next_even に引き継がれる。
# 同様に、odd_state は「倒さない」ことで next_odd に引き継がれる。

# これは正しい。

# ただし、初期状態：倒した数が0（偶数）→ even_state = 0, odd_state = -inf（到達不能）

even_state = 0
odd_state = -10**18

for strength in a:
    next_even = max(even_state, odd_state + 2 * strength)
    next_odd = max(odd_state, even_state + strength)
    even_state = next_even
    odd_state = next_odd

print(max(even_state, odd_state))
```
